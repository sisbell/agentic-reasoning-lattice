**GlobalUniqueness (GlobalUniqueness).** No two distinct allocations, anywhere in the system, at any time, produce the same address.

*Proof.* We must show that for any two addresses `a` and `b` produced by distinct allocation events — whether by the same allocator, different allocators at the same level, or allocators at different levels of the hierarchy — `a ≠ b`. The argument partitions all pairs of distinct allocation events into four exhaustive cases based on the relationship between the owning allocators.

We first establish the taxonomy that assigns every address to an allocator. An *allocation event* is any invocation of `inc(t, k)` within a conforming system. Each allocator `A` with base address `t₀` has a *domain* `dom(A) = {tₙ : n ≥ 0}` where `tₙ₊₁ = inc(tₙ, 0)` — the base address together with every sibling output of the `inc(·, 0)` chain. When a parent allocator executes `inc(t, k')` with `k' > 0`, the resulting address `c₀ = inc(t, k')` becomes the base of a new child allocator and belongs to that child's domain, not the parent's. This assignment is well-defined and exhaustive: every `inc` output belongs to exactly one domain — sibling outputs (`k = 0`) to the producing allocator's domain, child-spawning outputs (`k > 0`) to the newly created child's domain. The *owning allocator* of an address `a` is the unique `A` satisfying `a ∈ dom(A)`.

For a non-root allocator `A` spawned by `c₀ = inc(t, k')` with `k' > 0`, the *domain prefix* of `A` is the parent sibling `t` from which the child-spawning increment was performed. The root allocator has no domain prefix; its outputs have the shortest uniform length in the hierarchy (T10a.1), and every non-root output is strictly longer (T10a.3), so all root-vs-non-root pairs are separated by length and fall under Cases 3–4.

*Case 1: Same owning allocator.* Both `a` and `b` belong to `dom(A)` for the same allocator `A`. By the domain definition, `dom(A)` is the sequence `t₀, inc(t₀, 0), inc(inc(t₀, 0), 0), ...` — exactly the stream over which T9 (forward allocation) is proved. Since the allocation events are distinct, `a` and `b` are distinct elements of this sequence; without loss of generality, `allocated_before(a, b)`. By T9, `allocated_before(a, b)` implies `a < b`. Since `a < b`, irreflexivity of the strict order (T1, part (a)) gives `a ≠ b`.

*Case 2: Different non-root allocators with non-nesting domain prefixes.* The two allocators `A₁` and `A₂` are both non-root, with domain prefixes `p₁` and `p₂` satisfying `p₁ ⋠ p₂ ∧ p₂ ⋠ p₁` (Prefix). This arises whenever the allocators descend from different branches of the hierarchy — two sub-partitions whose ancestral spawning points diverge at some level.

We must establish that every output of a non-root allocator extends its domain prefix in the ≼ sense, so that T10's preconditions are met. Let `A` be a non-root allocator with domain prefix `t` and base address `c₀ = inc(t, k')` for `k' > 0`. By TA5(b), `c₀` agrees with `t` on all positions `1 ≤ i ≤ #t`; by TA5(d), `#c₀ = #t + k' > #t`. These two facts — agreement on all `#t` positions and strictly greater length — satisfy the definition of `t ≼ c₀` (Prefix). For subsequent siblings: each `cₙ₊₁ = inc(cₙ, 0)` modifies only position `sig(cₙ)` (TA5(c)). Since every output of a conforming allocator satisfies T4 (T10a.4), `sig(cₙ) = #cₙ` (TA5-SigValid), and `#cₙ = #c₀ = #t + k'` (T10a.1). The modified position `#t + k'` exceeds `#t`, so positions `1, …, #t` are unchanged across every sibling step. By induction on `n`, every `cₙ ∈ dom(A)` agrees with `t` on positions `1, …, #t` and has length `#t + k' > #t`, whence `t ≼ cₙ` for every output of `A`.

Now let `a ∈ dom(A₁)` and `b ∈ dom(A₂)`. The extension property gives `p₁ ≼ a` and `p₂ ≼ b`. Since `p₁ ⋠ p₂ ∧ p₂ ⋠ p₁`, T10 (partition independence) applies directly: its proof locates a position `k` where `p₁ₖ ≠ p₂ₖ`, transfers this divergence to `aₖ ≠ bₖ`, and concludes `a ≠ b` via T3 (canonical representation).

*Case 3: Different allocators with nesting prefixes and different zero counts.* One allocator's prefix nests within the other's, and the two allocators produce addresses at different hierarchical levels. By T4c (LevelDetermination), the zero count `zeros(t)` — the number of zero-valued field-separator components — uniquely determines the hierarchical level via a bijection on `{0, 1, 2, 3}`: `zeros = 0` for node, `zeros = 1` for user, `zeros = 2` for document, `zeros = 3` for element. The injective correspondence between levels and zero counts means allocators at different levels produce addresses with `zeros(a) ≠ zeros(b)`.

We show `a ≠ b` by contradiction. Suppose `a = b`. By T3, `a = b` requires `#a = #b` and `aᵢ = bᵢ` at every position `1 ≤ i ≤ #a`. If the components are identical at every position, then `{i : aᵢ = 0} = {i : bᵢ = 0}`, giving `zeros(a) = zeros(b)` — contradicting the hypothesis that the allocators operate at different hierarchical levels. Therefore `a ≠ b`.

*Case 4: Different allocators with nesting prefixes and the same zero count.* This is the structurally subtle case: a parent and a descendant allocator both produce addresses at the same hierarchical level (same zero count). We show that length separation makes collision impossible.

Let the parent allocator have base address `t₀` with `#t₀ = γ`. By T10a (allocator discipline), the parent produces its sibling outputs exclusively by repeated application of `inc(·, 0)`. By TA5(c), `inc(t, 0)` preserves length: `#inc(t, 0) = #t`. Applying this inductively — as established in T10a Consequence 1 — every parent sibling output has uniform length `γ`.

To spawn a child allocator, the parent performs one `inc(t, k')` with `k' > 0` for some parent sibling `t` with `#t = γ`. By TA5(d), `c₀ = inc(t, k')` has length `#c₀ = γ + k'`. Since `k' ≥ 1`, this gives `#c₀ ≥ γ + 1`. By the domain taxonomy, `c₀` belongs to the child's domain: it is the child allocator's base address — the initial element of `dom(child)` — produced by the parent's deep increment but assigned to the child upon creation. The child allocator then produces its siblings by `inc(·, 0)` (T10a), and by TA5(c) applied inductively, all child outputs — including `c₀` itself — have uniform length `γ + k'`.

We now establish `a ≠ b`. Every parent output has length `γ`; every child output — including the child's base address `c₀` — has length `γ + k'` with `k' ≥ 1`, so `γ + k' > γ`. For any parent output `a` and any child output `b` (or vice versa), `#a ≠ #b`, and by T3 (tumblers of different lengths are distinct), `a ≠ b`.

The length separation is additive across nesting levels. Each child-spawning step via `inc(·, k')` with `k' ≥ 1` adds at least one component (TA5(d)). A descendant `d` nesting levels below the parent produces outputs of length at least `γ + d > γ`. Allocators at different nesting depths produce outputs of different lengths, so they cannot collide by T3. Allocators at the same depth but on different branches have non-nesting prefixes and are handled by Case 2.

*Exhaustiveness.* The domain taxonomy assigns every address to exactly one allocator, so every pair of distinct addresses has well-defined owning allocators. If both share the same owning allocator: Case 1. If the owning allocators differ and one is the root: the root's outputs are strictly shorter than any non-root output (T10a.3), so the pair falls under Case 3 or 4. If both are non-root: their domain prefixes either nest or do not — if non-nesting, Case 2; if nesting (one allocator is an ancestor of the other along the spawning lineage), the addresses have different zero counts (Case 3) or the same zero count (Case 4). The four cases are exhaustive and mutually exclusive. Child-spawning outputs — produced by `inc(·, k')` with `k' > 0`, outside the scope of T9 — are assigned to the child's domain as its base address. Their distinctness from parent-domain addresses follows from length separation (Case 4); their distinctness from the child's subsequent siblings follows from Case 1, since the child's base is the initial element of `dom(child)`.

*Critical dependence on T10a.* The argument in Case 4 depends on T10a's constraint that sibling allocations use `k = 0`. If a parent allocator could use `k > 0` for siblings, its outputs would have varying lengths — each deep increment extends the tumbler by TA5(d). Some parent output could then match the length of a child output, collapsing the length separation. T10a's necessity proof demonstrates this failure mode explicitly: `inc(t₁, 1)` produces a sibling that is a proper prefix of the next, violating the non-nesting precondition of T10. ∎

This theorem is the foundation of the addressing architecture. Every subsequent guarantee — link stability, transclusion identity, royalty tracing — depends on the uniqueness of addresses. And uniqueness is achieved without any distributed consensus, simply by the structure of the names.

*Formal Contract:*
- *Preconditions:* `a, b ∈ T` produced by distinct allocation events within a system conforming to T10a (allocator discipline). Each address is assigned an owning allocator via the domain taxonomy: `dom(A) = {tₙ : n ≥ 0, tₙ₊₁ = inc(tₙ, 0)}` for allocator `A` with base `t₀`; child-spawning outputs (`k > 0`) belong to the child's domain. The domain prefix of a non-root allocator `A`, spawned by `c₀ = inc(t, k')`, is the parent sibling `t`; every `a ∈ dom(A)` satisfies `t ≼ a` (by TA5(b), TA5(d), TA5-SigValid, and T10a.1).
- *Invariant:* For every pair of addresses `a, b` produced by distinct allocation events in any reachable system state: `a ≠ b`.
