**GlobalUniqueness (GlobalUniqueness).** No two distinct allocations, anywhere in the system, at any time, produce the same address.

*Proof.* We must show that for any two addresses `a` and `b` produced by distinct allocation events — whether by the same allocator, different allocators at the same level, or allocators at different levels of the hierarchy — `a ≠ b`. The proof proceeds by strong induction on allocator tree depth; the five cases below supply the case analysis at each inductive step.

We first establish the taxonomy that assigns every allocation event's output to a producing allocator. An *allocation event* is either the initialization of the root allocator — establishing its base address `t₀` satisfying T4 (as specified by T10a) — or any invocation of `inc(t, k)` within a conforming system. Each allocator `A` with base address `t₀` has a *domain* `dom(A) = {tₙ : n ≥ 0}` where `tₙ₊₁ = inc(tₙ, 0)` — the base address together with every sibling output of the `inc(·, 0)` chain. When a parent allocator executes `inc(t, k')` with `k' > 0`, the resulting address `c₀ = inc(t, k')` becomes the base of a new child allocator and is assigned to that child's domain, not the parent's. The *producing allocator* of an address is the allocator to which the generating event assigns it: root initialization assigns `t₀` to root; an `inc(t, 0)` invocation assigns the output to the executing allocator; an `inc(t, k')` invocation with `k' > 0` assigns the output to the newly created child. This per-event assignment is well-defined — each allocation event has exactly one producing allocator — but two distinct events might a priori produce the same address value, placing it in two domains; ruling that out is precisely what this theorem establishes.

For a non-root allocator `A` spawned by `c₀ = inc(t, k')` with `k' > 0`, the *domain prefix* of `A` is the parent domain element `t` from which the child-spawning increment was performed. The root allocator has no domain prefix; root-vs-non-root pairs receive separate treatment in Case 2.

Define the *depth* of an allocator as the number of child-spawning steps from the root: the root has depth 0, and a child spawned by a depth-*d* allocator has depth *d* + 1. The inductive claim `U(d)` is: every pair of distinct allocation events whose producing allocators both have depth ≤ *d* yields distinct outputs. *Base case* (`d = 0`): the root is the sole depth-0 allocator, so every pair of distinct events at depth 0 shares a single producing allocator; Case 1 gives `a ≠ b`. *Inductive step*: assume `U(d)` holds; we prove `U(d + 1)`. Pairs where both producing allocators have depth ≤ *d* are covered by the hypothesis. For pairs where the maximum producing-allocator depth is *d* + 1, the following five cases apply; their arguments are self-contained except for the `p₁ = p₂` exhaustiveness routing, which invokes `U(d)`.

*Case 1: Same producing allocator.* Both `a` and `b` belong to `dom(A)` for the same allocator `A`. By the domain definition, `dom(A)` is the sequence `t₀, inc(t₀, 0), inc(inc(t₀, 0), 0), ...` — exactly the stream over which T9 (forward allocation) is proved. Since the allocation events are distinct, `a` and `b` are distinct elements of this sequence; without loss of generality, `allocated_before(a, b)`. By T9, `allocated_before(a, b)` implies `a < b`. Since `a < b`, irreflexivity of the strict order (T1, part (a)) gives `a ≠ b`.

*Case 2: Root allocator vs non-root allocator.* One address `a` belongs to `dom(root)` and the other `b` belongs to `dom(A)` for some non-root allocator `A`. By T10a.1, every root output has uniform length `γ = #t₀`. Every non-root allocator descends from the root through one or more child-spawning steps; by T10a.3, a descendant at depth `d ≥ 1` produces outputs of length at least `γ + d ≥ γ + 1`. Hence `#a = γ` and `#b ≥ γ + 1 > γ`, so `#a ≠ #b`, and by T3, `a ≠ b`.

*Case 3: Different non-root allocators with non-nesting domain prefixes.* The two allocators `A₁` and `A₂` are both non-root, with domain prefixes `p₁` and `p₂` satisfying `p₁ ⋠ p₂ ∧ p₂ ⋠ p₁` (Prefix). This arises whenever the allocators descend from different branches of the hierarchy — two sub-partitions whose ancestral spawning points diverge at some level.

We must establish that every output of a non-root allocator extends its domain prefix in the ≼ sense, so that T10's preconditions are met. Let `A` be a non-root allocator with domain prefix `t` and base address `c₀ = inc(t, k')` for `k' > 0`. By TA5(b), `c₀` agrees with `t` on all positions `1 ≤ i ≤ #t`; by TA5(d), `#c₀ = #t + k' > #t`. These two facts — agreement on all `#t` positions and strictly greater length — satisfy the definition of `t ≼ c₀` (Prefix). For subsequent siblings: each `cₙ₊₁ = inc(cₙ, 0)` modifies only position `sig(cₙ)` (TA5(c)). Since every output of a conforming allocator satisfies T4 (T10a.4), `sig(cₙ) = #cₙ` (TA5-SigValid), and `#cₙ = #c₀ = #t + k'` (T10a.1). The modified position `#t + k'` exceeds `#t`, so positions `1, …, #t` are unchanged across every sibling step. By induction on `n`, every `cₙ ∈ dom(A)` agrees with `t` on positions `1, …, #t` and has length `#t + k' > #t`, whence `t ≼ cₙ` for every output of `A`.

Now let `a ∈ dom(A₁)` and `b ∈ dom(A₂)`. The extension property gives `p₁ ≼ a` and `p₂ ≼ b`. Since `p₁ ⋠ p₂ ∧ p₂ ⋠ p₁`, T10 (partition independence) applies directly: its proof locates a position `k` where `p₁ₖ ≠ p₂ₖ`, transfers this divergence to `aₖ ≠ bₖ`, and concludes `a ≠ b` via T3 (canonical representation).

*Case 4: Different allocators with nesting prefixes and different zero counts.* One allocator's prefix nests within the other's, and the addresses satisfy `zeros(a) ≠ zeros(b)`. We show `a ≠ b` by contradiction. Suppose `a = b`. By T3, `a = b` requires `aᵢ = bᵢ` at every position `1 ≤ i ≤ #a`, so `{i : aᵢ = 0} = {i : bᵢ = 0}`, giving `zeros(a) = zeros(b)` — contradicting `zeros(a) ≠ zeros(b)`. Therefore `a ≠ b`.

*Case 5: Different allocators with nesting prefixes and the same zero count.* This is the structurally subtle case: two non-root allocators whose domain prefixes nest — `p₁ ≼ p₂` with `p₁ ≠ p₂` — yet whose outputs share the same zero count. Nesting of domain prefixes does not in general imply an ancestor–descendant relationship between the producing allocators; an uncle allocator and its nephew may share nesting prefixes when the uncle and the nephew's parent are *sibling allocators* in the sense of T10a's *Derived tree relations* — two distinct children of a common grandparent in 𝒯 — rather than standing in any ancestor–descendant relation to one another. The argument must therefore apply to arbitrary nesting-prefix pairs. We show that length separation makes collision impossible.

Let the parent allocator have base address `t₀` with `#t₀ = γ`. By T10a (allocator discipline), the parent produces its sibling outputs exclusively by repeated application of `inc(·, 0)`. By TA5(c), `inc(t, 0)` preserves length: `#inc(t, 0) = #t`. Applying this inductively — as established in T10a Consequence 1 — every parent sibling output has uniform length `γ`.

To spawn a child allocator, the parent performs one `inc(t, k')` with `k' > 0` for some parent domain element `t` with `#t = γ`. By TA5(d), `c₀ = inc(t, k')` has length `#c₀ = γ + k'`. Since `k' ≥ 1`, this gives `#c₀ ≥ γ + 1`. By the domain taxonomy, `c₀` belongs to the child's domain: it is the child allocator's base address — the initial element of `dom(child)` — produced by the parent's deep increment but assigned to the child upon creation. The child allocator then produces its siblings by `inc(·, 0)` (T10a), and by TA5(c) applied inductively, all child outputs — including `c₀` itself — have uniform length `γ + k'`.

We now establish `a ≠ b`. Every parent output has length `γ`; every child output — including the child's base address `c₀` — has length `γ + k'` with `k' ≥ 1`, so `γ + k' > γ`. For any parent output `a` and any child output `b` (or vice versa), `#a ≠ #b`, and by T3 (tumblers of different lengths are distinct), `a ≠ b`.

The preceding argument handles a parent and its immediate child. We now show that the same-zero-count assumption of Case 5 excludes equal-length outputs for any nesting-prefix pair, reducing the general case entirely to length separation. Let A₁ and A₂ be two non-root allocators with domain prefixes p₁ and p₂ where p₁ ≼ p₂ and p₁ ≠ p₂. Each Aᵢ, spawned with parameter k'ᵢ ∈ {1, 2} (per T10a), produces outputs of uniform length #pᵢ + k'ᵢ (by TA5(d) and T10a.1). Suppose for contradiction that #p₁ + k'₁ = #p₂ + k'₂. Since p₁ is a proper prefix of p₂, Prefix's derived postcondition `p ≺ q ⟹ #p < #q` gives #p₁ < #p₂. The composed inference "from #p₁ + k'₁ = #p₂ + k'₂ ∧ #p₁ < #p₂ conclude k'₁ > k'₂" is a non-trivial ℕ-arithmetic step that we discharge from explicit NAT-* axioms rather than leave to background arithmetic, matching the per-step citation convention applied in T10a-N, TA1, and TS3 to comparable ℕ-level inferences. By NAT-order's trichotomy on the pair (k'₁, k'₂), exactly one of k'₁ < k'₂, k'₁ = k'₂, k'₁ > k'₂ holds; we rule out the first two. *Sub-case k'₁ = k'₂:* substituting into the assumed equality gives #p₁ + k'₁ = #p₂ + k'₁; NAT-cancel's right cancellation (`n + m = p + m ⟹ n = p`, instantiated at m = k'₁, n = #p₁, p = #p₂) yields #p₁ = #p₂, contradicting #p₁ < #p₂ via NAT-order's trichotomy, which forbids `<` and `=` from holding simultaneously. *Sub-case k'₁ < k'₂:* NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` weakens k'₁ < k'₂ to k'₁ ≤ k'₂, and NAT-addcompat's left order compatibility (`n ≥ p ⟹ m + n ≥ m + p`, instantiated at m = #p₁, n = k'₂, p = k'₁) lifts it to #p₁ + k'₁ ≤ #p₁ + k'₂. The same NAT-order weakening turns #p₁ < #p₂ into #p₁ ≤ #p₂, and NAT-addcompat's right order compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at m = k'₂, n = #p₂, p = #p₁) lifts it to #p₁ + k'₂ ≤ #p₂ + k'₂; this latter step sharpens to strict because NAT-cancel's right cancellation rules out the equality #p₁ + k'₂ = #p₂ + k'₂ (else #p₁ = #p₂, again contradicting #p₁ < #p₂), and NAT-order's defining clause then converts non-strict ≤ together with non-equality into the strict #p₁ + k'₂ < #p₂ + k'₂. NAT-order's transitivity chains #p₁ + k'₁ ≤ #p₁ + k'₂ < #p₂ + k'₂ into the strict #p₁ + k'₁ < #p₂ + k'₂, contradicting the assumed equality #p₁ + k'₁ = #p₂ + k'₂ via NAT-order's irreflexivity. The only remaining trichotomy outcome is k'₁ > k'₂; with both values in {1, 2}, the only possibility is k'₁ = 2 and k'₂ = 1, whence the assumed equality reads #p₁ + 2 = #p₂ + 1. Rewriting 2 as 1 + 1 — a definitional unfolding of the numeral 2 as the successor of 1, with successor coinciding with +1 — and applying NAT-addassoc's associativity (`(m + n) + p = m + (n + p)`, instantiated at m = #p₁, n = 1, p = 1) gives #p₁ + 2 = #p₁ + (1 + 1) = (#p₁ + 1) + 1, so the equality becomes (#p₁ + 1) + 1 = #p₂ + 1; NAT-cancel's right cancellation then yields #p₂ = #p₁ + 1. The prefix p₂ therefore extends p₁ by exactly one position — position #p₂. By T10a.4, every domain prefix is a T4-valid address (it is a domain element of a conforming allocator). By T4's axiom — specifically clause (iii) of its field-segment constraint, `t_{#t} ≠ 0`, imposed on every admitted address tumbler — the last component of a T4-valid address is non-zero: p₂[#p₂] ≠ 0. The single extending position contributes no zero, so zeros(p₂) = zeros(p₁). Now compute the output zero counts. TA5(d) fixes the zero count of each allocator's base address: with k'₁ = 2, A₁'s base `c₀(A₁) = inc(p₁, 2)` inserts a single zero field separator at position #p₁ + 1 (TA5(d)'s `k − 1 = 1` intermediate zero before the final 1), giving zeros(c₀(A₁)) = zeros(p₁) + 1; with k'₂ = 1, A₂'s base `c₀(A₂) = inc(p₂, 1)` inserts no separator (TA5(d)'s `k − 1 = 0` intermediate zeros), giving zeros(c₀(A₂)) = zeros(p₂) = zeros(p₁). TA5(d) alone fixes only these base zero counts; the arbitrary `a ∈ dom(A₁)` and `b ∈ dom(A₂)` under analysis need not be the bases. T10a.8 (UniformSiblingZeroCount) lifts each base zero count to every sibling in the corresponding allocator's domain — no `inc(·, 0)` step alters the zero count, because TA5-SigValid and T10a.4 pin the modified position to `sig(t) = #t`, which T4's field-segment constraint keeps strictly positive before and after the `+1`. Therefore `zeros(a) = zeros(c₀(A₁)) = zeros(p₁) + 1` for every `a ∈ dom(A₁)`, and `zeros(b) = zeros(c₀(A₂)) = zeros(p₁)` for every `b ∈ dom(A₂)`, so `zeros(a) ≠ zeros(b)` for every such pair — routing this pair to Case 4 and contradicting the same-zero-count assumption of Case 5. The equal-length scenario is therefore excluded from this case, and every pair in Case 5 satisfies #a ≠ #b; by T3, a ≠ b. Allocators at the same depth but on different branches have non-nesting prefixes and are handled by Case 3.

*Exhaustiveness.* The producing-allocator assignment is well-defined for every allocation event: root initialization assigns `t₀` to root; each `inc` invocation assigns its output to exactly one allocator by the event-taxonomy rule. Every pair of distinct allocation events therefore has well-defined producing allocators. If both events assign to the same producing allocator: Case 1. If the producing allocators differ and one is the root: Case 2. If both are non-root: their domain prefixes either nest or do not — if non-nesting, Case 3. If the domain prefixes are value-identical (`p₁ = p₂`): we must first establish that both allocators were spawned by the same parent. Let `P₁` and `P₂` be the respective parent allocators; each has depth strictly less than its child, hence depth ≤ *d*. The prefix value `p₁` belongs to `dom(P₁)` by virtue of the event — call it `E₁` — that produced it: the per-event producing-allocator taxonomy introduced at the start of the proof (and recorded in the Preconditions) assigns each `inc` output and the root's initialization output to exactly one allocator, and `P₁` is the unique allocator to which `E₁` assigns `p₁`. Analogously, `p₂ = p₁` belongs to `dom(P₂)` by virtue of an event `E₂` whose unique producing-allocator assignment is `P₂`. If `P₁ ≠ P₂`, then `E₁ ≠ E₂`, because the taxonomy's *exactly-one-allocator* clause forbids a single event from having two distinct producing allocators; the equality `E₁ = E₂` would force `P₁ = P₂` via functionality of the assignment, contradicting `P₁ ≠ P₂`. Hence `E₁` and `E₂` are two distinct allocation events whose producing allocators (`P₁` and `P₂`) both have depth ≤ *d*, yet both events yield the same output value `p₁`, contradicting `U(d)`. Therefore `P₁ = P₂`: both child-spawning events were performed by the same parent, applying `inc` to the same parent domain element `p₁` with respective parameters `k'₁, k'₂ ∈ {1, 2}`. T10a's per-parent uniqueness constraint — each `(t, k')` pair spawns at most one child — now applies and excludes `k'₁ = k'₂`, so `k'₁ ≠ k'₂`; since one of `{k'₁, k'₂}` is `1` and the other `2`, the allocator spawned with `k' = 2` acquires one additional field separator zero in its base relative to the allocator spawned with `k' = 1` (the TA5(d) argument of Case 5, applied to the two base addresses `inc(p₁, 2)` and `inc(p₁, 1)`), and T10a.8 (UniformSiblingZeroCount) lifts each base zero count to every sibling in the corresponding allocator's domain — yielding different zero counts for every pair `a`, `b` with `a` drawn from one allocator's domain and `b` from the other, routing the pair to Case 4. If the domain prefixes nest strictly (`p₁ ≼ p₂` with `p₁ ≠ p₂`, or symmetrically): the addresses have different zero counts (Case 4) or the same zero count (Case 5). The five cases are exhaustive and mutually exclusive, completing the inductive step: `U(d)` implies `U(d + 1)`. By induction, `U(d)` holds for all `d ≥ 0`; since every allocator has finite depth, GlobalUniqueness follows. Child-spawning outputs — produced by `inc(·, k')` with `k' > 0`, outside the scope of T9 — are assigned to the child's domain as its base address. Their distinctness from parent-domain addresses follows from length separation (Case 5); their distinctness from the child's subsequent siblings follows from Case 1, since the child's base is the initial element of `dom(child)`.

*Critical dependence on T10a.* The argument in Case 5 depends on T10a's constraint that sibling allocations use `k = 0`. If a parent allocator could use `k > 0` for siblings, its outputs would have varying lengths — each deep increment extends the tumbler by TA5(d). Some parent output could then match the length of a child output, collapsing the length separation. T10a's necessity proof demonstrates this failure mode explicitly: `inc(t₁, 1)` produces a sibling that is a proper prefix of the next, violating the non-nesting precondition of T10. ∎

*Corollary (Domain Disjointness).* For distinct allocators `A₁ ≠ A₂`, `dom(A₁) ∩ dom(A₂) = ∅`. If some address `a` belonged to both `dom(A₁)` and `dom(A₂)`, then `a` was produced by two distinct allocation events — one assigning it to `A₁`, one to `A₂` — yet both events produced the same value `a`, contradicting GlobalUniqueness. Hence each address value belongs to at most one allocator's domain, and the per-event producing-allocator assignment induces a well-defined *owning allocator* per address value: the unique `A` satisfying `a ∈ dom(A)`.

This theorem is the foundation of the addressing architecture. Every subsequent guarantee — link stability, transclusion identity, royalty tracing — depends on the uniqueness of addresses. And uniqueness is achieved without any distributed consensus, simply by the structure of the names.

*Formal Contract:*
- *Preconditions:* `a, b ∈ T` produced by distinct allocation events — where an allocation event is either root initialization or an invocation of `inc(t, k)` — within a system conforming to T10a (allocator discipline). Each address is assigned a producing allocator by its generating event: the root's base address to root by initialization; each `inc` output to a single allocator via the event-taxonomy rule (`k = 0` to the executing allocator, `k > 0` to the newly created child). The domain prefix of a non-root allocator `A`, spawned by `c₀ = inc(t, k')`, is the parent domain element `t`; every `a ∈ dom(A)` satisfies `t ≼ a` (by TA5(b), TA5(d), TA5-SigValid, and T10a.1).
- *Depends:* AllocatedSet (AllocatedSet) — supplies the producing-allocator taxonomy the proof rests on: *allocation event* as root initialization or `inc(t, k)` invocation in a reachable system state, grounding "distinct allocation events" in the Preconditions and underwriting the per-event assignment of each output to a producing allocator. T9 (ForwardAllocation) — Case 1: `allocated_before(a, b)` implies `a < b`, supplying the strict ordering from which distinctness is extracted. T1 (LexicographicOrder) — Case 1: part (a) irreflexivity converts `a < b` to `a ≠ b`. T10 (PartitionIndependence) — Case 3: applied directly once `p₁ ⋠ p₂ ∧ p₂ ⋠ p₁`, `p₁ ≼ a`, and `p₂ ≼ b` are established; T10's proof locates a divergence position, transfers it to `a` and `b`, and concludes `a ≠ b`. T10a (AllocatorDiscipline) — Case 5 and the exhaustiveness routing: the `inc(·, 0)`-only sibling restriction underwrites the uniform-length argument; the child-spawning bound `k' ∈ {1, 2}` pins down the values in the Case 5 length and zero-count computations; the per-parent uniqueness constraint (each `(t, k')` spawns at most one child) excludes `k'₁ = k'₂` in the `p₁ = p₂` routing, forcing distinct zero counts and directing the pair to Case 4. T10a.1 (Uniform sibling length) — Cases 2 and 5: every sibling shares the allocator's base length, letting the proof assign length `γ` to root outputs and `γ + k'` to any child's full sibling stream. T10a.3 (Length separation) — Case 2: a descendant at depth `d ≥ 1` produces outputs of length `≥ γ + d ≥ γ + 1`, yielding `#a ≠ #b` between root and non-root outputs. T10a.4 (T4 preservation) — every output of a conforming allocator satisfies T4, so every domain prefix is T4-valid; this supplies the precondition for TA5-SigValid (fixing `sig(cₙ) = #cₙ`) in the Case 5 extension argument and the T4-validity of p₂ required before T4's field-segment clause (iii) can be invoked to fix `p₂[#p₂] ≠ 0` in the Case 5 length-collision argument. T10a.8 (Uniform sibling zero count) — Case 5 length-collision argument in the `(k'₁, k'₂) = (2, 1)` subcase, and the exhaustiveness routing for `p₁ = p₂`: lifts the base-address zero counts fixed by TA5(d) (`zeros(c₀(A₁)) = zeros(p₁) + 1` when `k' = 2`, `zeros(c₀(A₂)) = zeros(p₂)` when `k' = 1`) to every sibling in the corresponding domain, so the Case 4 zero-count distinctness `zeros(a) ≠ zeros(b)` holds for the arbitrary pair `a ∈ dom(A₁)`, `b ∈ dom(A₂)` under analysis rather than only for the two base addresses. Without T10a.8 the routing would rely on an unstated within-allocator inductive argument (TA5(b)/TA5(c)/TA5-SigValid/T10a.4) at each use site; with T10a.8 the same chain is discharged once. T3 (CanonicalRepresentation) — Cases 2, 3, 4, 5: tumbler equality requires component-by-component agreement at every position of a shared length, whence `#a ≠ #b` gives `a ≠ b` (Cases 2, 5), position-wise divergence at some `k` gives `a ≠ b` (Case 3), and equal zero sets are a necessary consequence of equality (Case 4). T4 (HierarchicalParsing) — invoked through T10a's root initialization constraint (every root output and descendant is T4-valid) and through T10a.4's extension of T4 to all domain prefixes. In the Case 5 length-collision argument, T4's field-segment clause (iii) (`t_{#t} ≠ 0`, imposed by T4's axiom on every admitted address tumbler) directly supplies `p₂[#p₂] ≠ 0` once T10a.4 establishes p₂'s T4-validity, whence the single extending position at index #p₂ contributes no zero and `zeros(p₂) = zeros(p₁)` follows. The load-bearing edge at this site is T4, not T4a: T4a proves the equivalence between T4's positional clauses (i)–(iii) and segment non-emptiness, but clause (iii) itself is T4's axiom content, so extracting `p₂[#p₂] ≠ 0` requires only T4 — routing through T4a would demand an independent prior derivation of segment non-emptiness for p₂ before T4a's Forward direction could recover (iii), a detour that T4 makes unnecessary. TA5 (HierarchicalIncrement) — TA5(b) supplies agreement on positions `1 ≤ i ≤ #t` between `inc(t, k)` and `t` for `k > 0`, underwriting `t ≼ c₀` (extension argument) and the preservation of prefix content through child-spawning; TA5(c) supplies `#inc(t, 0) = #t` and single-position modification at `sig`, used for the uniform-length induction in Case 5; TA5(d) supplies `#inc(t, k') = #t + k'` for `k' > 0`, furnishing the length expansion across child-spawning (Cases 2, 5) and the zero-separator bookkeeping (one separator when `k' = 2`, none when `k' = 1`) that distinguishes zero counts in the exhaustiveness routing. TA5-SigValid (TA5-SigValid) — Case 5 extension argument: for T4-valid `cₙ`, `sig(cₙ) = #cₙ`, so each `inc(cₙ, 0)` modifies only the terminal position, leaving positions `1, …, #t` fixed across every sibling step. Prefix (PrefixRelation) — Cases 3 and 5 extension argument: the ≼ definition (agreement on shorter-length positions together with strict length excess) specifies non-nesting (Case 3: `p₁ ⋠ p₂ ∧ p₂ ⋠ p₁`) and proper prefixing (`t ≼ c₀` derived from TA5(b) and TA5(d)) on which T10's precondition and the case-5 analysis rest; Prefix's derived postcondition `p ≺ q ⟹ #p < #q` supplies the strict length inequality #p₁ < #p₂ from `p₁ ≺ p₂` that the Case 5 length-collision argument consumes. NAT-addcompat (NatAdditionOrderAndSuccessor) — Case 5 length-collision argument, sub-case k'₁ < k'₂: two clauses are cited. NAT-addcompat's left order compatibility (`n ≥ p ⟹ m + n ≥ m + p`, instantiated at m = #p₁, n = k'₂, p = k'₁) lifts the weakened k'₁ ≤ k'₂ to #p₁ + k'₁ ≤ #p₁ + k'₂; NAT-addcompat's right order compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at m = k'₂, n = #p₂, p = #p₁) lifts the weakened #p₁ ≤ #p₂ to #p₁ + k'₂ ≤ #p₂ + k'₂. Both clauses are needed because the variable summand sits on opposite sides in the two steps — left for the (k'₁, k'₂) comparison and right for the (#p₁, #p₂) comparison — matching the symmetry rationale that NAT-addcompat axiomatises both forms; this is the same per-step convention applied in T10a-N (left order compatibility on `1 ≤ k`) and TA1 (right order compatibility on `aₖ < bₖ`). NAT-cancel (NatAdditionCancellation) — Case 5 length-collision argument: right cancellation (`n + m = p + m ⟹ n = p`) is invoked three times. (a) Sub-case k'₁ = k'₂: cancelling the common k'₁ from #p₁ + k'₁ = #p₂ + k'₁ yields #p₁ = #p₂, contradicting #p₁ < #p₂. (b) Sub-case k'₁ < k'₂: cancelling the common k'₂ rules out the equality #p₁ + k'₂ = #p₂ + k'₂, sharpening the non-strict ≤ supplied by NAT-addcompat into the strict <. (c) After determining (k'₁, k'₂) = (2, 1): cancelling the trailing +1 from (#p₁ + 1) + 1 = #p₂ + 1 yields #p₂ = #p₁ + 1. NAT-order (NatStrictTotalOrder) — Case 5 length-collision argument: discharges four roles. (i) Trichotomy is invoked at the start to enumerate the three cases on (k'₁, k'₂) and again to forbid the simultaneous holding of #p₁ < #p₂ and #p₁ = #p₂ in the sub-case k'₁ = k'₂ contradiction. (ii) The defining clause `m ≤ n ⟺ m < n ∨ m = n` weakens strict inequalities (k'₁ < k'₂ to k'₁ ≤ k'₂; #p₁ < #p₂ to #p₁ ≤ #p₂) to the non-strict form NAT-addcompat consumes, and converts non-strict-with-non-equality back into strict for the chain step. (iii) Transitivity chains #p₁ + k'₁ ≤ #p₁ + k'₂ < #p₂ + k'₂ into the strict #p₁ + k'₁ < #p₂ + k'₂. (iv) Irreflexivity discharges the final contradiction with the assumed equality #p₁ + k'₁ = #p₂ + k'₂. NAT-addassoc (NatAdditionAssociative) — Case 5 length-collision argument, (k'₁, k'₂) = (2, 1) extraction: associativity (`(m + n) + p = m + (n + p)`, instantiated at m = #p₁, n = 1, p = 1) regroups #p₁ + (1 + 1) = (#p₁ + 1) + 1, allowing #p₁ + 2 to be rewritten as (#p₁ + 1) + 1 so that NAT-cancel's right cancellation can extract #p₂ = #p₁ + 1 from #p₁ + 2 = #p₂ + 1.
- *Invariant:* For every pair of addresses `a, b` arising from distinct allocation events (root initialization or `inc` invocations) in any reachable system state: `a ≠ b`.
- *Postconditions:* (1) Domain Disjointness — for distinct allocators `A₁ ≠ A₂`, `dom(A₁) ∩ dom(A₂) = ∅`. A shared address would require two distinct allocation events producing the same value, contradicting the invariant. (2) Well-defined owning allocator — each address value belongs to at most one allocator's domain (by Domain Disjointness), so the per-event producing-allocator assignment induces a unique owning allocator per address value: the unique `A` satisfying `a ∈ dom(A)`.
- *Proof structure:* Strong induction on allocator tree depth *d*. Claim `U(d)`: all pairs at depth ≤ *d* produce distinct outputs. Base (`d = 0`): sole root allocator, Case 1. Step (`d → d + 1`): Cases 1–5 are self-contained; the `p₁ = p₂` exhaustiveness routing invokes `U(d)` to establish shared parentage — identical prefix values held by distinct parents at depth ≤ *d* would be a repeated output contradicting `U(d)` — then applies T10a's per-parent uniqueness to separate spawning parameters.
