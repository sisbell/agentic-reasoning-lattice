## Tumbler arithmetic

We now turn to the arithmetic operations. The system requires operations that advance a position by a displacement (for computing span endpoints and shifting positions) and that recover the displacement between two positions (for computing span widths). These operations ‚ÄĒ tumbler addition (‚äē, constructed in TumblerAdd) and subtraction (‚äĖ, constructed in TumblerSub, ¬ß Tumbler subtraction above) ‚ÄĒ are not arithmetic on numbers but position-advance operations in a hierarchical address space.

A displacement `w` is a tumbler whose leading zeros say "stay at these hierarchical levels" and whose first nonzero component says "advance here." Components after the advance point describe the structure of the landing position within the target region.

### Definition of ‚äē

Tumbler addition is not arithmetic addition ‚ÄĒ it is a **position-advance operation**: given a start position `a` and a displacement `w`, compute where you land. The displacement encodes both the distance and the hierarchical level at which the advance occurs.

```
START:  1.0.3.0.2.0.1.777
  DIF:  0.0.0.0.0.0.0.300
        ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ
AFTER:  1.0.3.0.2.0.1.1077
```

Reading the displacement `[0,0,0,0,0,0,0,300]`: seven leading zeros mean "same server, same account, same document, same subspace." Component 8 is 300: "advance 300 elements." No trailing components: the landing position has no further sub-structure.

A displacement that acts at a higher level:

```
START:  1.0.3.0.2.0.1.777
  DIF:  0.0.0.0.3.0.1.1
        ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ‚ĒÄ
AFTER:  1.0.3.0.5.0.1.1
```

Reading `[0,0,0,0,3,0,1,1]`: four leading zeros mean "same server, same account." Component 5 is 3: "advance 3 documents." Trailing `[0,1,1]`: "land at element 1.1 in the target document." The start position's element field `[1,777]` is replaced by the displacement's trailing structure `[1,1]`.

**TumblerAdd (TumblerAdd).** Let `a = [a‚āĀ, ..., a‚āė]` and `w = [w‚āĀ, ..., w‚āô]` with `a, w ‚ąą T` and `Pos(w)`. By ActionPoint, the precondition `Pos(w)` yields a well-defined action point `k = actionPoint(w)` ‚ÄĒ the least index with a nonzero component ‚ÄĒ satisfying `1 ‚Č§ k ‚Č§ n` and `w‚āĖ ‚Č• 1`. Require `k ‚Č§ m`: the action point must fall within the start position's length.

```
         ‚éß aŠĶĘ           if i < k        (copy from start)
rŠĶĘ   =  ‚é® a‚āĖ + w‚āĖ      if i = k        (single-component advance)
         ‚é© wŠĶĘ           if i > k        (copy from displacement)
```

The result `a ‚äē w = [r‚āĀ, ..., r‚āö]` has length `p = (k - 1) + 1 + (n - k) = n = #w`, where the partial-subtraction terms `k - 1 ‚ąą ‚Ąē` and `n - k ‚ąą ‚Ąē` are well-defined by NAT-sub's conditional closure (the former under `k ‚Č• 1` from ActionPoint, the latter under `n ‚Č• k` supplied by ActionPoint's `k ‚Č§ #w = n`), and the collapses `(k - 1) + 1 = k` and `k + (n - k) = n` are NAT-sub's right-inverse characterisation `(m ‚ąí n) + n = m` (instantiated at `m = k, n = 1` under `k ‚Č• 1`) and left-inverse characterisation `n + (m ‚ąí n) = m` (instantiated at `m = n, n = k` under `n ‚Č• k`) respectively. Since `w ‚ąą T` requires `n ‚Č• 1`, the result has at least one component. We record this as the *result-length identity*: **`#(a ‚äē w) = #w`** ‚ÄĒ the length of the sum is determined entirely by the displacement, not the start position.

Each component of the result is a natural number: for `i < k`, `rŠĶĘ = aŠĶĘ ‚ąą ‚Ąē` since `a ‚ąą T` and `k ‚Č§ m` ensures position `i` exists within `a`; at the action point, `r‚āĖ = a‚āĖ + w‚āĖ ‚ąą ‚Ąē` by NAT-closure's closure of ‚Ąē under addition; for `i > k`, `rŠĶĘ = wŠĶĘ ‚ąą ‚Ąē` since `w ‚ąą T`. The result is therefore a finite sequence over ‚Ąē with length ‚Č• 1, hence **`a ‚äē w ‚ąą T`** by T0.

The construction also yields strict advancement. ActionPoint supplies `w‚āĖ ‚Č• 1` (the minimum-nonzero value at the action point), from which we derive `r‚āĖ > a‚āĖ` via the chain `r‚āĖ = a‚āĖ + w‚āĖ ‚Č• a‚āĖ + 1 > a‚āĖ`, each link explicitly sourced: the equality `r‚āĖ = a‚āĖ + w‚āĖ` is the action-point clause of the construction; NAT-addcompat's left order-compatibility (`n ‚Č• p ‚üĻ m + n ‚Č• m + p`, instantiated at `m = a‚āĖ, n = w‚āĖ, p = 1`) lifts `w‚āĖ ‚Č• 1` to the non-strict `a‚āĖ + w‚āĖ ‚Č• a‚āĖ + 1`; NAT-addcompat's strict successor inequality (`n < n + 1`, instantiated at `n = a‚āĖ`) supplies the strict `a‚āĖ + 1 > a‚āĖ`; and NAT-order composes the non-strict `a‚āĖ + w‚āĖ ‚Č• a‚āĖ + 1` with the strict `a‚āĖ + 1 > a‚āĖ` into the strict `a‚āĖ + w‚āĖ > a‚āĖ` ‚ÄĒ NAT-order's defining clause `m ‚Č§ n ‚üļ m < n ‚ą® m = n` instantiated at `m = a‚āĖ + 1, n = a‚āĖ + w‚āĖ` unfolds the `‚Č•` into `a‚āĖ + w‚āĖ > a‚āĖ + 1 ‚ą® a‚āĖ + w‚āĖ = a‚āĖ + 1`; the first disjunct chains with `a‚āĖ + 1 > a‚āĖ` through NAT-order's transitivity of `<` to yield `a‚āĖ + w‚āĖ > a‚āĖ`; the second disjunct substitutes `a‚āĖ + w‚āĖ = a‚āĖ + 1` into `a‚āĖ + 1 > a‚āĖ` to yield the same conclusion. NAT-addcompat's order-compatibility clauses deliver *only* non-strict conclusions, so this `‚Č•`/`>` composition to strict is discharged from NAT-order ‚ÄĒ matching the per-step citation discipline T1 and TA1 apply at their analogous strict-promotion sites. For all `i` with `1 ‚Č§ i < k`, `rŠĶĘ = aŠĶĘ` by the construction. The precondition `k ‚Č§ m` gives `k ‚Č§ #a`, and the result-length identity gives `k ‚Č§ n = #(a ‚äē w)`, so `k ‚Č§ min(#a, #(a ‚äē w))` and both tumblers have a component at position `k`. T1 case (i) with divergence position `k` ‚ÄĒ agreement on positions `1, ..., k - 1` and strict inequality `a‚āĖ < r‚āĖ` ‚ÄĒ yields `a < a ‚äē w`. We record this as the *ordering guarantee*: **`a ‚äē w > a`** ‚ÄĒ tumbler addition strictly advances the start position.

The construction also yields dominance over the displacement. Since `#(a ‚äē w) = #w` (result-length identity), the T1 comparison of `a ‚äē w` and `w` reduces to finding the first position where `rŠĶĘ ‚Č† wŠĶĘ`. For `i < k`, `rŠĶĘ = aŠĶĘ` and `wŠĶĘ = 0` (by definition of action point). The top-level case split on whether some `a‚Īľ > 0` for `j < k` is a dichotomy at `0` on ‚Ąē, sourced per position: for each `j` with `1 ‚Č§ j < k`, the precondition `k ‚Č§ m` places `j` within `a`'s length so T0 gives `a‚Īľ ‚ąą ‚Ąē`; NAT-zero's axiom `(A n ‚ąą ‚Ąē :: 0 ‚Č§ n)` supplies the lower bound `0 ‚Č§ a‚Īľ`; NAT-order's defining clause `m ‚Č§ n ‚üļ m < n ‚ą® m = n` instantiated at `m = 0, n = a‚Īľ` unfolds this lower bound into `0 < a‚Īľ ‚ą® 0 = a‚Īľ`, i.e., `a‚Īľ > 0 ‚ą® a‚Īľ = 0`; the two top-level cases are the existential and universal closures of these two disjuncts over `1 ‚Č§ j < k`. If some `a‚Īľ > 0` for `j < k`, the least such `j` is a divergence point with `r‚Īľ > w‚Īľ`, so T1 case (i) gives `r > w`. If instead `aŠĶĘ = 0` for all `i < k`, then at position `k` we have `r‚āĖ = a‚āĖ + w‚āĖ`, and ActionPoint's `w‚āĖ ‚Č• 1` yields `w‚āĖ > 0` via the chain `w‚āĖ ‚Č• 1 > 0`: NAT-order's defining clause at `m = 1, n = w‚āĖ` unfolds `w‚āĖ ‚Č• 1` into `1 < w‚āĖ ‚ą® 1 = w‚āĖ`; NAT-addcompat's strict successor inequality at `n = 0`, together with NAT-closure's additive identity `0 + 1 = 1`, supplies `0 < 1`; NAT-order composes these through its transitivity of `<` (under the strict disjunct `1 < w‚āĖ`) or by substitution (under the equality disjunct `1 = w‚āĖ`) to yield `0 < w‚āĖ`, i.e., `w‚āĖ > 0`. The inner case split `a‚āĖ > 0 ‚ą® a‚āĖ = 0` at the action point is the same NAT-zero + NAT-order unfolding applied at `m = 0, n = a‚āĖ`, with `a‚āĖ ‚ąą ‚Ąē` because `k ‚Č§ m` places position `k` within `a`: NAT-zero supplies `0 ‚Č§ a‚āĖ`, and NAT-order's defining clause unfolds this into `0 < a‚āĖ ‚ą® 0 = a‚āĖ`. When `a‚āĖ > 0`, `r‚āĖ > w‚āĖ` and T1 case (i) again gives `r > w`; when `a‚āĖ = 0`, `r‚āĖ = w‚āĖ`; combined with `rŠĶĘ = aŠĶĘ = 0 = wŠĶĘ` for `i < k` and `rŠĶĘ = wŠĶĘ` for `i > k`, every component agrees, and `#r = #w` by the result-length identity, so `r = w` by T3, hence `r ‚Č• w`. In every case: **`a ‚äē w ‚Č• w`** ‚ÄĒ the result of tumbler addition dominates the displacement in T1 ordering.

These four results are load-bearing for subsequent properties that depend on TumblerAdd.

Three properties of this definition require explicit statement:

**No carry propagation:** The sum `a‚āĖ + w‚āĖ` at the action point is a single natural-number addition. There is no carry into position `k - 1`. This is why the operation is fast ‚ÄĒ constant time regardless of tumbler length.

**Tail replacement, not tail addition:** Components after the action point come entirely from `w`. The start position's components at positions `k + 1, ..., m` are discarded. `a ‚äē w` does not add corresponding components pairwise ‚ÄĒ it replaces the start's sub-structure with the displacement's sub-structure below the action point.

**The many-to-one property:** Because trailing components of `a` are discarded, distinct start positions can produce the same result:

```
[1, 1] ‚äē [0, 2]       = [1, 3]
[1, 1, 5] ‚äē [0, 2]    = [1, 3]
[1, 1, 999] ‚äē [0, 2]  = [1, 3]
```

This is correct and intentional: advancing to "the beginning of the next chapter" lands at the same place regardless of where you were within the current chapter. ‚ąé
