## Tumbler arithmetic

We now turn to the arithmetic operations. The system requires operations that advance a position by a displacement (for computing span endpoints and shifting positions) and that recover the displacement between two positions (for computing span widths). These operations — tumbler addition (⊕, constructed in TumblerAdd) and subtraction (⊖, constructed in TumblerSub, § Tumbler subtraction above) — are not arithmetic on numbers but position-advance operations in a hierarchical address space.

A displacement `w` is a tumbler whose leading zeros say "stay at these hierarchical levels" and whose first nonzero component says "advance here." Components after the advance point describe the structure of the landing position within the target region.

### Definition of ⊕

Tumbler addition is not arithmetic addition — it is a **position-advance operation**: given a start position `a` and a displacement `w`, compute where you land. The displacement encodes both the distance and the hierarchical level at which the advance occurs.

```
START:  1.0.3.0.2.0.1.777
  DIF:  0.0.0.0.0.0.0.300
        ──────────────────
AFTER:  1.0.3.0.2.0.1.1077
```

Reading the displacement `[0,0,0,0,0,0,0,300]`: seven leading zeros mean "same server, same account, same document, same subspace." Component 8 is 300: "advance 300 elements." No trailing components: the landing position has no further sub-structure.

A displacement that acts at a higher level:

```
START:  1.0.3.0.2.0.1.777
  DIF:  0.0.0.0.3.0.1.1
        ──────────────────
AFTER:  1.0.3.0.5.0.1.1
```

Reading `[0,0,0,0,3,0,1,1]`: four leading zeros mean "same server, same account." Component 5 is 3: "advance 3 documents." Trailing `[0,1,1]`: "land at element 1.1 in the target document." The start position's element field `[1,777]` is replaced by the displacement's trailing structure `[1,1]`.

**TumblerAdd (TumblerAdd).** Let `a = [a₁, ..., aₘ]` and `w = [w₁, ..., wₙ]` with `a, w ∈ T` and `Pos(w)`. By ActionPoint, the precondition `Pos(w)` yields a well-defined action point `k = actionPoint(w)` — the least index with a nonzero component — satisfying `1 ≤ k ≤ n` and `wₖ ≥ 1`. Require `k ≤ m`: the action point must fall within the start position's length.

```
         ⎧ aᵢ           if i < k        (copy from start)
rᵢ   =  ⎨ aₖ + wₖ      if i = k        (single-component advance)
         ⎩ wᵢ           if i > k        (copy from displacement)
```

The result `a ⊕ w = [r₁, ..., rₚ]` has length `p = (k - 1) + 1 + (n - k) = n = #w`, where the partial-subtraction terms `k - 1 ∈ ℕ` and `n - k ∈ ℕ` are well-defined by NAT-sub's conditional closure (the former under `k ≥ 1` from ActionPoint, the latter under `n ≥ k` supplied by ActionPoint's `k ≤ #w = n`), and the collapses `(k - 1) + 1 = k` and `k + (n - k) = n` are NAT-sub's right-inverse characterisation `(m − n) + n = m` (instantiated at `m = k, n = 1` under `k ≥ 1`) and left-inverse characterisation `n + (m − n) = m` (instantiated at `m = n, n = k` under `n ≥ k`) respectively. Since `w ∈ T` requires `n ≥ 1`, the result has at least one component. We record this as the *result-length identity*: **`#(a ⊕ w) = #w`** — the length of the sum is determined entirely by the displacement, not the start position.

Each component of the result is a natural number: for `i < k`, `rᵢ = aᵢ ∈ ℕ` since `a ∈ T` and `k ≤ m` ensures position `i` exists within `a`; at the action point, `rₖ = aₖ + wₖ ∈ ℕ` by NAT-closure's closure of ℕ under addition; for `i > k`, `rᵢ = wᵢ ∈ ℕ` since `w ∈ T`. The result is therefore a finite sequence over ℕ with length ≥ 1, hence **`a ⊕ w ∈ T`** by T0.

The construction also yields strict advancement. ActionPoint supplies `wₖ ≥ 1` (the minimum-nonzero value at the action point), from which we derive `rₖ > aₖ` via the chain `rₖ = aₖ + wₖ ≥ aₖ + 1 > aₖ`, each link explicitly sourced: the equality `rₖ = aₖ + wₖ` is the action-point clause of the construction; NAT-addcompat's left order-compatibility (`n ≥ p ⟹ m + n ≥ m + p`, instantiated at `m = aₖ, n = wₖ, p = 1`) lifts `wₖ ≥ 1` to the non-strict `aₖ + wₖ ≥ aₖ + 1`; NAT-addcompat's strict successor inequality (`n < n + 1`, instantiated at `n = aₖ`) supplies the strict `aₖ + 1 > aₖ`; and NAT-order composes the non-strict `aₖ + wₖ ≥ aₖ + 1` with the strict `aₖ + 1 > aₖ` into the strict `aₖ + wₖ > aₖ` — NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = aₖ + 1, n = aₖ + wₖ` unfolds the `≥` into `aₖ + wₖ > aₖ + 1 ∨ aₖ + wₖ = aₖ + 1`; the first disjunct chains with `aₖ + 1 > aₖ` through NAT-order's transitivity of `<` to yield `aₖ + wₖ > aₖ`; the second disjunct substitutes `aₖ + wₖ = aₖ + 1` into `aₖ + 1 > aₖ` to yield the same conclusion. NAT-addcompat's order-compatibility clauses deliver *only* non-strict conclusions, so this `≥`/`>` composition to strict is discharged from NAT-order — matching the per-step citation discipline T1 and TA1 apply at their analogous strict-promotion sites. For all `i` with `1 ≤ i < k`, `rᵢ = aᵢ` by the construction. The precondition `k ≤ m` gives `k ≤ #a`, and the result-length identity gives `k ≤ n = #(a ⊕ w)`, so `k ≤ min(#a, #(a ⊕ w))` and both tumblers have a component at position `k`. T1 case (i) with divergence position `k` — agreement on positions `1, ..., k - 1` and strict inequality `aₖ < rₖ` — yields `a < a ⊕ w`. We record this as the *ordering guarantee*: **`a ⊕ w > a`** — tumbler addition strictly advances the start position.

The construction also yields dominance over the displacement. Since `#(a ⊕ w) = #w` (result-length identity), the T1 comparison of `a ⊕ w` and `w` reduces to finding the first position where `rᵢ ≠ wᵢ`. For `i < k`, `rᵢ = aᵢ` and `wᵢ = 0` (by definition of action point). The top-level case split on whether some `aⱼ > 0` for `j < k` is a dichotomy at `0` on ℕ, sourced per position: for each `j` with `1 ≤ j < k`, the precondition `k ≤ m` places `j` within `a`'s length so T0 gives `aⱼ ∈ ℕ`; NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` supplies the lower bound `0 ≤ aⱼ`; NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = 0, n = aⱼ` unfolds this lower bound into `0 < aⱼ ∨ 0 = aⱼ`, i.e., `aⱼ > 0 ∨ aⱼ = 0`; the two top-level cases are the existential and universal closures of these two disjuncts over `1 ≤ j < k`. If some `aⱼ > 0` for `j < k`, the least such `j` is a divergence point with `rⱼ > wⱼ`, so T1 case (i) gives `r > w`. If instead `aᵢ = 0` for all `i < k`, then at position `k` we have `rₖ = aₖ + wₖ`, and ActionPoint's `wₖ ≥ 1` yields `wₖ > 0` via the chain `wₖ ≥ 1 > 0`: NAT-order's defining clause at `m = 1, n = wₖ` unfolds `wₖ ≥ 1` into `1 < wₖ ∨ 1 = wₖ`; NAT-addcompat's strict successor inequality at `n = 0`, together with NAT-closure's additive identity `0 + 1 = 1`, supplies `0 < 1`; NAT-order composes these through its transitivity of `<` (under the strict disjunct `1 < wₖ`) or by substitution (under the equality disjunct `1 = wₖ`) to yield `0 < wₖ`, i.e., `wₖ > 0`. The inner case split `aₖ > 0 ∨ aₖ = 0` at the action point is the same NAT-zero + NAT-order unfolding applied at `m = 0, n = aₖ`, with `aₖ ∈ ℕ` because `k ≤ m` places position `k` within `a`: NAT-zero supplies `0 ≤ aₖ`, and NAT-order's defining clause unfolds this into `0 < aₖ ∨ 0 = aₖ`. When `aₖ > 0`, we derive `rₖ > wₖ` — i.e., `aₖ + wₖ > wₖ` — via a four-step chain consuming NAT-addcompat, NAT-closure, NAT-cancel, and NAT-order together (with the NAT-zero lower bound already supplied for the dichotomy above): the symmetric transfer from the earlier strict-advancement chain `rₖ = aₖ + wₖ ≥ aₖ + 1 > aₖ` is unavailable because the mirrored route would require `1 + wₖ > wₖ` against NAT-addcompat's strict successor inequality `wₖ < wₖ + 1`, and T0's exhaustive NAT-* enumeration does not include commutativity of addition on ℕ. NAT-addcompat's right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at `n = aₖ, p = 0, m = wₖ`) lifts NAT-zero's lower bound `0 ≤ aₖ` (the same instance the inner dichotomy consumes) into the non-strict `aₖ + wₖ ≥ 0 + wₖ`; NAT-closure's additive identity `0 + wₖ = wₖ` rewrites the right-hand side, yielding `aₖ + wₖ ≥ wₖ`; NAT-cancel's summand absorption in its symmetric form `n + m = m ⟹ n = 0` (instantiated at `n = aₖ, m = wₖ`) rules out the equality `aₖ + wₖ = wₖ` — that equality would force `aₖ = 0` and contradict the sub-case hypothesis `aₖ > 0` via NAT-order's irreflexivity (substituting `aₖ = 0` into `0 < aₖ` yields `0 < 0`); and NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = wₖ, n = aₖ + wₖ`, applied to the non-strict `aₖ + wₖ ≥ wₖ` together with the just-discharged non-equality, returns the strict `aₖ + wₖ > wₖ`, i.e., `rₖ > wₖ` — the same strict-through-addition pattern TA1's sub-case `j = k` applies at its analogous site. T1 case (i) with divergence position `k` again gives `r > w`; when `aₖ = 0`, `rₖ = wₖ`; combined with `rᵢ = aᵢ = 0 = wᵢ` for `i < k` and `rᵢ = wᵢ` for `i > k`, every component agrees, and `#r = #w` by the result-length identity, so `r = w` by T3, hence `r ≥ w`. In every case: **`a ⊕ w ≥ w`** — the result of tumbler addition dominates the displacement in T1 ordering.

These four results are load-bearing for subsequent properties that depend on TumblerAdd.

Three properties of this definition require explicit statement:

**No carry propagation:** The sum `aₖ + wₖ` at the action point is a single natural-number addition. There is no carry into position `k - 1`. This is why the operation is fast — constant time regardless of tumbler length.

**Tail replacement, not tail addition:** Components after the action point come entirely from `w`. The start position's components at positions `k + 1, ..., m` are discarded. `a ⊕ w` does not add corresponding components pairwise — it replaces the start's sub-structure with the displacement's sub-structure below the action point.

**The many-to-one property:** Because trailing components of `a` are discarded, distinct start positions can produce the same result:

```
[1, 1] ⊕ [0, 2]       = [1, 3]
[1, 1, 5] ⊕ [0, 2]    = [1, 3]
[1, 1, 999] ⊕ [0, 2]  = [1, 3]
```

This is correct and intentional: advancing to "the beginning of the next chapter" lands at the same place regardless of where you were within the current chapter. ∎

*Formal Contract:*
- *Preconditions:* a ∈ T, w ∈ T, Pos(w) (TA-Pos), actionPoint(w) ≤ #a (ActionPoint)
- *Definition:* k = actionPoint(w) (ActionPoint); rᵢ = aᵢ if i < k; rₖ = aₖ + wₖ; rᵢ = wᵢ if i > k
- *Depends:* T0 (CarrierSetDefinition) — membership `a ⊕ w ∈ T` is concluded via T0's characterisation of T as finite sequences over ℕ with length ≥ 1; the dominance proof's two dichotomy sites (`aⱼ > 0 ∨ aⱼ = 0` for `j < k`, and `aₖ > 0 ∨ aₖ = 0` at the action point) also read `aⱼ ∈ ℕ` and `aₖ ∈ ℕ` off T0's carrier so that NAT-zero's `(A n ∈ ℕ :: 0 ≤ n)` applies at those components. NAT-closure (NatArithmeticClosure) — the "each component of the result is a natural number" step at the action point (`rₖ = aₖ + wₖ ∈ ℕ`) invokes NAT-closure's closure of ℕ under addition; the dominance proof's equality sub-case (`aₖ = 0 ⟹ rₖ = wₖ`) and strict-advancement sub-case (`aₖ > 0 ⟹ aₖ + wₖ > wₖ`) both invoke NAT-closure's additive identity (`0 + wₖ = wₖ`) — in the equality sub-case to substitute `rₖ = aₖ + wₖ = 0 + wₖ = wₖ` directly, in the strict-advancement sub-case to rewrite the right-hand side of NAT-addcompat's right-order-compatibility conclusion `aₖ + wₖ ≥ 0 + wₖ` as the non-strict `aₖ + wₖ ≥ wₖ` that the remaining NAT-cancel + NAT-order promotion consumes; the dominance proof's `wₖ ≥ 1 ⟹ wₖ > 0` chain invokes the additive identity again (at the substitution `0 + 1 = 1`) so that NAT-addcompat's strict successor inequality at `n = 0` reads as `0 < 1` rather than the unsimplified `0 < 0 + 1`. NAT-addcompat (NatAdditionOrderCompatibility) — the strict-advancement chain `rₖ = aₖ + wₖ ≥ aₖ + 1 > aₖ` invokes two NAT-addcompat clauses: left order-compatibility (`n ≥ p ⟹ m + n ≥ m + p`, instantiated at `m = aₖ, n = wₖ, p = 1`) lifts `wₖ ≥ 1` into the non-strict `aₖ + wₖ ≥ aₖ + 1`, and the strict successor inequality (`n < n + 1`, instantiated at `n = aₖ`) supplies the strict `aₖ + 1 > aₖ`. The dominance proof's `wₖ ≥ 1 ⟹ wₖ > 0` chain invokes NAT-addcompat's strict successor inequality at `n = 0` (together with NAT-closure's additive identity `0 + 1 = 1`) to discharge `0 < 1`. The dominance proof's inner sub-case `aₖ > 0` invokes NAT-addcompat's right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at `n = aₖ, p = 0, m = wₖ`) to lift NAT-zero's lower bound `0 ≤ aₖ` into the non-strict `aₖ + wₖ ≥ 0 + wₖ` — the first step of the strict-through-addition chain that derives `aₖ + wₖ > wₖ` (equivalently `rₖ > wₖ`) for the sub-case's conclusion; right order-compatibility is the form required here because the symmetric route through left order-compatibility with `aₖ ≥ 1` would need the strict `1 + wₖ > wₖ`, not available from NAT-addcompat's strict successor inequality `wₖ < wₖ + 1` without commutativity of addition on ℕ (which T0's exhaustive NAT-* enumeration does not include). NAT-addcompat's order-compatibility clauses deliver *only* non-strict conclusions — as NAT-addcompat's own contract articulates — so the composition of the non-strict `aₖ + wₖ ≥ aₖ + 1` with the strict `aₖ + 1 > aₖ` into the strict `aₖ + wₖ > aₖ`, the composition of ActionPoint's `wₖ ≥ 1` with `0 < 1` into `wₖ > 0`, and the promotion of the non-strict `aₖ + wₖ ≥ wₖ` (from right order-compatibility + NAT-closure's additive identity) to the strict `aₖ + wₖ > wₖ`, are routed through NAT-order (with NAT-cancel additionally ruling out the equality disjunct in the last case) rather than discharged by NAT-addcompat alone. NAT-cancel (NatAdditionCancellation) — the dominance proof's inner sub-case `aₖ > 0` invokes NAT-cancel's summand absorption in its symmetric form (`(A m, n ∈ ℕ : n + m = m : n = 0)`, instantiated at `n = aₖ, m = wₖ`) to rule out the equality disjunct `aₖ + wₖ = wₖ` inside NAT-order's defining-clause unfolding of the non-strict `aₖ + wₖ ≥ wₖ` — itself obtained from NAT-zero's lower bound `0 ≤ aₖ` via NAT-addcompat's right order-compatibility at `n = aₖ, p = 0, m = wₖ` and NAT-closure's additive identity `0 + wₖ = wₖ`. The equality `aₖ + wₖ = wₖ` would force `aₖ = 0` and contradict the sub-case hypothesis `aₖ > 0` via NAT-order's irreflexivity (substituting `aₖ = 0` into `0 < aₖ` yields `0 < 0`), leaving the strict disjunct `aₖ + wₖ > wₖ` as the sub-case's `rₖ > wₖ` conclusion. NAT-addcompat's order-compatibility clauses deliver only non-strict conclusions, so the strict promotion from `aₖ + wₖ ≥ wₖ` to `aₖ + wₖ > wₖ` must be discharged through NAT-cancel (for the equality-ruling-out sub-step) together with NAT-order (for the defining-clause unfolding and irreflexivity) rather than NAT-addcompat alone — the same strict-through-addition pattern TA1's sub-case `j = k` applies at its analogous site (with right cancellation instead of summand absorption), and the only route to `aₖ + wₖ > wₖ` that respects T0's exhaustive NAT-* enumeration: the symmetric chain through left order-compatibility with `aₖ ≥ 1` would require `1 + wₖ > wₖ` against NAT-addcompat's strict successor inequality `wₖ < wₖ + 1`, which differ by a commutativity step absent from the enumeration. NAT-zero (NatZeroMinimum) — invoked at the dominance proof's two dichotomy sites to fix the lower bound on the ℕ-valued components being case-split, and additionally consumed at the strict-through-addition chain in the inner sub-case `aₖ > 0`. (i) For each `j` with `1 ≤ j < k`, the precondition `k ≤ m` places `j` within `a`'s length so T0 gives `aⱼ ∈ ℕ`, and NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` supplies `0 ≤ aⱼ` — the lower-bound premise that NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` at `m = 0, n = aⱼ` consumes to unfold into `0 < aⱼ ∨ 0 = aⱼ`, i.e., the `aⱼ > 0 ∨ aⱼ = 0` dichotomy the top-level case split consumes over `j < k`. (ii) At the action point, `k ≤ m` places position `k` within `a` so T0 gives `aₖ ∈ ℕ`, and NAT-zero supplies `0 ≤ aₖ` — the lower-bound premise the same NAT-order unfolding at `m = 0, n = aₖ` consumes to deliver the inner `aₖ > 0 ∨ aₖ = 0` dichotomy; in the inner sub-case `aₖ > 0`, the same `0 ≤ aₖ` instance additionally feeds NAT-addcompat's right order-compatibility at `n = aₖ, p = 0, m = wₖ` to lift into `aₖ + wₖ ≥ 0 + wₖ` — the first step of the strict-through-addition chain that derives `aₖ + wₖ > wₖ` (equivalently `rₖ > wₖ`) for the sub-case's conclusion, a reuse of the same NAT-zero instance rather than a fresh site. No other listed NAT-* axiom supplies this lower bound: NAT-order's trichotomy compares `0` and `aⱼ` (resp. `aₖ`) without fixing the winner, and NAT-wellorder's least-element principle gives ℕ a minimum without naming it `0`. NAT-zero's axiom discharges exactly the `0 ≤ ·` premise at each site, matching the per-step citation convention TA-Pos's Case `#z ≥ k` and ActionPoint's third postcondition establish for the structurally identical `0 ≤ t_⋅` step. NAT-order (NatStrictTotalOrder) — supplies four steps in the strict-advancement and dominance proofs. (i) The strict-advancement chain `rₖ = aₖ + wₖ ≥ aₖ + 1 > aₖ` composes the non-strict `aₖ + wₖ ≥ aₖ + 1` (from NAT-addcompat's left order-compatibility) with the strict `aₖ + 1 > aₖ` (from NAT-addcompat's strict successor inequality) into the strict `aₖ + wₖ > aₖ` — NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = aₖ + 1, n = aₖ + wₖ` unfolds the `≥` into `aₖ + wₖ > aₖ + 1 ∨ aₖ + wₖ = aₖ + 1`, the first disjunct chains with `aₖ + 1 > aₖ` through NAT-order's transitivity of `<` to yield `aₖ + wₖ > aₖ`, and the second disjunct substitutes `aₖ + wₖ = aₖ + 1` into `aₖ + 1 > aₖ` to yield the same conclusion. Since NAT-addcompat's order-compatibility clauses deliver only non-strict conclusions, this `≥`/`>` composition to strict must be routed through NAT-order — matching the per-step citation discipline T1 and TA1 apply at their analogous strict-promotion sites. (ii) The dominance proof's `wₖ ≥ 1 ⟹ wₖ > 0` step at the action point: NAT-order's defining clause at `m = 1, n = wₖ` unfolds ActionPoint's `wₖ ≥ 1` into `1 < wₖ ∨ 1 = wₖ`; NAT-order then composes these disjuncts with `0 < 1` (discharged above from NAT-addcompat + NAT-closure) — through transitivity of `<` under the strict disjunct `1 < wₖ`, or by substitution under the equality disjunct `1 = wₖ` — to yield `0 < wₖ`, i.e., `wₖ > 0`, the same NAT-order chain TA-Pos's Case `#z ≥ k` and ActionPoint's third postcondition use for the analogous `0 ≤ · ⟹ 0 < ·` step. (iii) The dominance proof's dichotomy at `0` on ℕ, at both instances (`aⱼ > 0 ∨ aⱼ = 0` for `j < k`, and the inner `aₖ > 0 ∨ aₖ = 0` at the action point): NAT-order's defining clause instantiated at `m = 0, n = aⱼ` (resp. `n = aₖ`) unfolds the lower bound `0 ≤ aⱼ` (resp. `0 ≤ aₖ`) supplied by NAT-zero into the disjunct `0 < aⱼ ∨ 0 = aⱼ` (resp. `0 < aₖ ∨ 0 = aₖ`) that each dichotomy site consumes. Without NAT-order the dichotomies would appeal to background `<`/`=` trichotomy on ℕ, breaking the per-step citation convention that TA-Pos and ActionPoint already enforce for analogous dichotomies. (iv) The dominance proof's inner sub-case `aₖ > 0` step `aₖ + wₖ > wₖ`: NAT-order's defining clause instantiated at `m = wₖ, n = aₖ + wₖ` unfolds the non-strict `aₖ + wₖ ≥ wₖ` (obtained from NAT-zero's lower bound `0 ≤ aₖ` via NAT-addcompat's right order-compatibility and NAT-closure's additive identity) into `aₖ + wₖ > wₖ ∨ aₖ + wₖ = wₖ`; NAT-cancel's summand absorption in its symmetric form rules out the equality disjunct by forcing `aₖ = 0`, which contradicts the sub-case hypothesis `aₖ > 0` via NAT-order's irreflexivity (substituting `aₖ = 0` into `0 < aₖ` yields `0 < 0`), so the strict disjunct `aₖ + wₖ > wₖ` is the remaining outcome — equivalently `rₖ > wₖ`, the strict component inequality T1 case (i) with divergence position `k` consumes to give `a ⊕ w > w`. Since NAT-addcompat's order-compatibility clauses deliver only non-strict conclusions, this strict promotion from `aₖ + wₖ ≥ wₖ` to `aₖ + wₖ > wₖ` must be routed through NAT-order (with NAT-cancel ruling out the equality disjunct) rather than NAT-addcompat alone — matching the strict-through-addition pattern TA1's sub-case `j = k` applies at its analogous site, and the only route that respects T0's exhaustive NAT-* enumeration: the symmetric chain through NAT-addcompat's left order-compatibility with `aₖ ≥ 1` would require `1 + wₖ > wₖ` against the strict successor inequality `wₖ < wₖ + 1`, which differ by a commutativity step absent from the enumeration. NAT-wellorder (NatWellOrdering) — the dominance proof's divergence sub-case step "the least such `j` is a divergence point" invokes the well-ordering of ℕ applied to the nonempty subset `{j : 1 ≤ j < k ∧ aⱼ > 0} ⊆ ℕ` — nonempty by the sub-case hypothesis that some `aⱼ > 0` for `j < k`, a subset of ℕ because `j` ranges over natural-number indices `1 ≤ j < k` — to supply the least element named by "the least such `j`", a use of well-ordering independent of the well-ordering propagated into TumblerAdd through `actionPoint(w)` (which `ActionPoint` discharges internally over the set `{i : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0}`), so the "least such `j`" claim is discharged from NAT-wellorder rather than left implicit, matching the per-step citation convention established for `TA-Pos` and `ActionPoint`. NAT-sub (NatPartialSubtraction) — the result-length identity `p = (k - 1) + 1 + (n - k) = n` consumes NAT-sub at three sites enumerated at the granularity TumblerSub, D0, D1, D2, TA3, TA4, TA3-strict, TA-LC, TA7a, ReverseInverse, and TA5a apply for partial subtraction on ℕ: (i) conditional closure discharges `k - 1 ∈ ℕ` under the precondition `k ≥ 1` (supplied by ActionPoint's lower bound `1 ≤ k`); (ii) conditional closure discharges `n - k ∈ ℕ` under the precondition `n ≥ k` (supplied by ActionPoint's upper bound `k ≤ #w` together with the typing `n = #w`); (iii) the right-inverse characterisation `(m − n) + n = m` discharges the collapse `(k - 1) + 1 = k` (instantiated at `m = k, n = 1` under `k ≥ 1`), and the left-inverse characterisation `n + (m − n) = m` discharges the collapse `k + (n - k) = n` (instantiated at `m = n, n = k` under `n ≥ k`). The result-length identity is load-bearing — promulgated as the postcondition `#(a ⊕ w) = #w` and consumed by TA0, Definition (Span), and every downstream user of `a ⊕ w`'s length — so without NAT-sub the identity's arithmetic would tacitly rest on background subtraction on ℕ, leaving a reviser tightening NAT-sub without Depends-backed visibility into TumblerAdd. ActionPoint (ActionPoint) — supplies `k = actionPoint(w)` with bounds `1 ≤ k ≤ n`, the zeros-below-action-point fact `wᵢ = 0 for i < actionPoint(w)`, and the minimum-nonzero value `wₖ ≥ 1`; the bounds `1 ≤ k` and `k ≤ n` also discharge the preconditions of NAT-sub's conditional-closure clauses for `k - 1 ∈ ℕ` and `n - k ∈ ℕ` in the result-length identity; the dominance proof `a ⊕ w ≥ w` uses the zeros-below-k fact to license both the divergence case (`wⱼ = 0 < aⱼ = rⱼ` at the least `j < k` with `aⱼ > 0`) and the equality sub-case (`rᵢ = aᵢ = 0 = wᵢ` for `i < k` when `aᵢ = 0` throughout).
