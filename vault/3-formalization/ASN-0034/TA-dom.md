**TA-dom (DisplacementDominance).** `(A a, w ∈ T : Pos(w) ∧ actionPoint(w) ≤ #a : a ⊕ w ≥ w)`.

TumblerAdd's fourth postcondition is precisely `a ⊕ w ≥ w (T1, T3)`: the *dominance guarantee* paragraph inside TumblerAdd discharges it by case analysis on whether any position below the action point of `a` is nonzero. The top-level dichotomy at `0` on ℕ is sourced per position — T0's carrier characterisation places each `aⱼ ∈ ℕ`, NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` supplies `0 ≤ aⱼ`, and NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = 0, n = aⱼ` unfolds this into `aⱼ > 0 ∨ aⱼ = 0`. In the divergence sub-case (some `aⱼ > 0` for `j < k`), NAT-wellorder applied to the nonempty subset `{j : 1 ≤ j < k ∧ aⱼ > 0} ⊆ ℕ` supplies the least such `j`; T1 case (i) at divergence position `j` — with agreement-zeros on `1 ≤ i < j` (minimality excluding the strict disjunct of the per-position dichotomy, ActionPoint's zeros-below-action-point fact giving `wᵢ = 0`), bounds `j ≤ #w` and `j ≤ #r` (chaining `j < k` with `k ≤ #w` via NAT-order's transitivity and defining clause, then substituting the result-length identity `#r = #w`), and the strict component inequality `rⱼ > wⱼ` (ActionPoint's zeros-below-action-point fact giving `wⱼ = 0`, construction giving `rⱼ = aⱼ`, sub-case hypothesis giving `aⱼ > 0`) — yields `r > w`. In the equality sub-case (`aᵢ = 0` for all `i < k`), a second dichotomy at the action point `aₖ > 0 ∨ aₖ = 0` (the same NAT-zero + NAT-order unfolding at `m = 0, n = aₖ`) splits into two branches: when `aₖ > 0`, a four-step chain (NAT-addcompat's right order-compatibility lifting `0 ≤ aₖ` into `aₖ + wₖ ≥ 0 + wₖ`, NAT-closure's additive identity rewriting the right-hand side to `aₖ + wₖ ≥ wₖ`, NAT-cancel's summand absorption ruling out the equality disjunct `aₖ + wₖ = wₖ` via contradiction with the sub-case hypothesis, and NAT-order's defining clause folding the result into the strict `aₖ + wₖ > wₖ`) delivers `rₖ > wₖ` and T1 case (i) gives `r > w`; when `aₖ = 0`, `rₖ = aₖ + wₖ = 0 + wₖ = wₖ` via NAT-closure's additive identity, every component of `r` agrees with the corresponding component of `w`, the result-length identity gives `#r = #w`, and T3's canonical-representation principle yields `r = w`. In every case the non-strict `r ≥ w` follows — by T1's non-strict-order definition `a ≤ b ⟺ a < b ∨ a = b` applied in the dual `a ≥ b ⟺ b ≤ a` form, the strict disjunct `r > w` from the divergence sub-case and the inner `aₖ > 0` sub-case folds into `r ≥ w`, and the equality `r = w` from the inner `aₖ = 0` sub-case likewise folds in.

TA-dom does not re-prove this — it exports the dominance guarantee as a single labelled fact so that downstream users (chiefly TA4 (ReverseCancellation), whose Step 2 cites the dominance postcondition to discharge TumblerSub's `r ≥ w` precondition) can reference one corollary rather than cherry-pick from TumblerAdd's four-postcondition list. The dominance of the result over the displacement — the observation that tumbler addition never produces a result lexicographically below the displacement it applied — is carried by TumblerAdd's construction itself: the tail-copy rule for `i > k` forces `rᵢ = wᵢ` above the action point, and the action-point and prefix-copy regions together either match or exceed `w`'s components below and at `k`, so TA-dom is a theorem derived from TumblerAdd, not an independent axiom.

*Proof.* Immediate from TumblerAdd's *dominance guarantee* postcondition `a ⊕ w ≥ w (T1, T3)` under the preconditions `a, w ∈ T`, `Pos(w)`, `actionPoint(w) ≤ #a`, which are exactly TA-dom's hypotheses. ∎

*Formal Contract:*
- *Preconditions:* `a ∈ T`, `w ∈ T`, `Pos(w)`, `k ≤ #a` where `k` is the action point of `w`
- *Depends:* TumblerAdd (TumblerAdd) — the sole arithmetic source; TumblerAdd's *dominance guarantee* paragraph derives `a ⊕ w ≥ w (T1, T3)` as one of its four postconditions, and TA-dom invokes this postcondition unchanged, so the per-step axioms that TumblerAdd cites (T0's carrier characterisation for the `aⱼ ∈ ℕ` and `aₖ ∈ ℕ` readings that underpin both dichotomy sites; NAT-zero for the ℕ lower bound `0 ≤ ·` at each dichotomy site and at the inner sub-case `aₖ > 0`'s strict-through-addition chain; NAT-order for the defining-clause unfoldings at the dichotomy sites, for the `≥`/`>` composition in the strict-through-addition chain, and for the transitivity composition `j < k ≤ #w` in the divergence sub-case; NAT-addcompat's right order-compatibility for the lift `0 ≤ aₖ ⟹ aₖ + wₖ ≥ 0 + wₖ` in the inner sub-case `aₖ > 0`; NAT-closure's additive identity for both the rewrite `0 + wₖ = wₖ` in the same chain and the equality-sub-case collapse `rₖ = aₖ + wₖ = 0 + wₖ = wₖ`; NAT-cancel's summand absorption for ruling out the equality disjunct `aₖ + wₖ = wₖ` inside the same chain's final NAT-order fold; NAT-wellorder for the least element of `{j : 1 ≤ j < k ∧ aⱼ > 0}` that names the divergence position and underwrites the agreement-zeros argument at the sub-range `1 ≤ i < j`; and NAT-sub, via the result-length identity, for the bound `j ≤ #r` that the divergence sub-case's T1 case (i) consumes) are transitively sourced through TumblerAdd rather than re-listed here — TA-dom introduces no fresh arithmetic step beyond the postcondition lookup. TA-Pos (PositiveTumbler) — invoked through the precondition `Pos(w)`, which TumblerAdd's precondition list requires and which is the entry condition for the action-point construction; without `Pos(w)`, the action point does not exist and TumblerAdd's dominance-guarantee postcondition does not engage. ActionPoint (ActionPoint) — invoked through the precondition `actionPoint(w) ≤ #a`; ActionPoint's defining minimum-position formula is the construct used to express this bound, and its lower bound `1 ≤ k`, its upper bound `k ≤ #w`, and its zeros-below-action-point fact are silent premises of TumblerAdd's dominance-guarantee sub-proof — the lower bound underwriting the three-region split, the upper bound feeding the divergence sub-case's `j ≤ #w` chaining through NAT-order, and the zeros-below-action-point fact supplying `wᵢ = 0` at both the divergence sub-case's agreement-zeros argument (at every index `i` with `1 ≤ i < j < k`) and the divergence-position witness's strict component inequality (at index `j`) — that underwrite the postcondition TA-dom exports. TA0 (WellDefinedAddition) — invoked to discharge both the membership `a ⊕ w ∈ T` that the target inequality `a ⊕ w ≥ w` presupposes so that T1's lexicographic ordering applies to its left-hand side, and the result-length identity `#(a ⊕ w) = #w` that TumblerAdd's dominance-guarantee paragraph consumes at two sites: the divergence sub-case's length bound `j ≤ #r` (obtained by substituting `#r = #w` into the NAT-order-derived `j ≤ #w`) and the equality sub-case's T3 closure from component-wise agreement to `r = w` (which requires `#r = #w` as one of T3's two premises). The cascade TA-dom → TA0 → TumblerAdd is deliberate rather than a relabelling artefact: TA0 is the ASN's canonical citation handle for "a ⊕ w is well-defined", exporting exactly TumblerAdd's first two postconditions (`a ⊕ w ∈ T` and `#(a ⊕ w) = #w`) as a single labelled well-definedness fact, so that TA-dom — which needs both the membership half (to license T1's `≥` on the left-hand side) and the length half (to license T3's equality-from-agreement step in the equality sub-case), not the strict-ordering guarantee that shares TumblerAdd's postcondition list — routes through TA0 rather than cherry-picking two of the four postconditions out of TumblerAdd's postcondition list, mirroring TA-strict's own role as the labelled handle for the third postcondition. A reviser tightening TumblerAdd's membership or length arguments updates TA0's postconditions in lockstep, and TA-dom inherits the change through the handle without re-audit. T1 (LexicographicOrder) — supplies both the meaning of the non-strict ordering `≥` through its non-strict-order definition `a ≤ b ⟺ a < b ∨ a = b` (with the dual `a ≥ b ⟺ b ≤ a`) and the case-(i) divergence-position witness for the two strict sub-cases of the dominance-guarantee proof (divergence sub-case at position `j`, inner sub-case `aₖ > 0` at position `k`); TumblerAdd annotates the dominance-guarantee postcondition `(T1, T3)` at the lifts from `rⱼ > wⱼ` (divergence sub-case) and `rₖ > wₖ` (inner sub-case `aₖ > 0`) to `r > w`, and at the final `r > w ∨ r = w ⟹ r ≥ w` fold, and TA-dom's postcondition re-exports the same ordering under the same T1 semantics. T3 (CanonicalRepresentation) — supplies the equality conclusion `r = w` in the inner sub-case `aₖ = 0` of the equality sub-case: TumblerAdd's dominance-guarantee paragraph establishes component-wise agreement `rᵢ = wᵢ` at every position `1 ≤ i ≤ n = #w` (below the action point from the construction's prefix-copy rule combined with the sub-case hypotheses `aᵢ = 0` for `i < k` and ActionPoint's `wᵢ = 0` for `i < k`; at the action point from NAT-closure's additive identity collapsing `rₖ = aₖ + wₖ = 0 + wₖ = wₖ`; above the action point from the construction's tail-copy rule `rᵢ = wᵢ`) together with the result-length identity `#r = #w` (via TA0), and T3's canonical-representation principle — that tumblers agreeing in length and at every component are equal — converts this into the tumbler equality `r = w` which T1's non-strict-order definition then folds into `r ≥ w` via its equality disjunct; TA-dom's postcondition re-exports the same equality-to-dominance conversion under the same T3 semantics. The T3 citation is what the `(T1, T3)` annotation on TumblerAdd's fourth postcondition names — T1 alone would suffice for the two strict sub-cases, but T3 is required to close the equality sub-case where the result coincides with the displacement.
- *Postconditions:* `a ⊕ w ≥ w`
