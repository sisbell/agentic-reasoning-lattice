## Zero tumblers and positivity

Under T3, the tumblers `[0]`, `[0, 0]`, `[0, 0, 0]`, etc., are *distinct* elements of T ‚Äî they have different lengths. Under T1, they form a chain: `[0] < [0, 0] < [0, 0, 0] < ...` by the prefix rule. There is no single "zero tumbler"; there are infinitely many all-zero tumblers. (This distinctness remark is motivational commentary, not a load-bearing claim: the postcondition's universal quantification over zero tumblers `z ‚àà T` would hold whether length-varying zero sequences denote distinct elements of T or collapse to a single representative ‚Äî if two `z`'s coincided as elements of T, the claim `z < t` would still need to be discharged pointwise, and the proof below would still discharge it. T3 is therefore referenced here for motivation only and is not cited in the formal Depends below.)

**TA-Pos (PositiveTumbler).** A tumbler `t ‚àà T` is *positive*, written `Pos(t)`, iff at least one of its components is nonzero: `(E i : 1 ‚â§ i ‚â§ #t : t·µ¢ ‚âÝ 0)`. A tumbler is a *zero tumbler* iff every component is zero: `(A i : 1 ‚â§ i ‚â§ #t : t·µ¢ = 0)`.

The predicate `Pos(t)` is not written `t > 0`, because `>` already denotes T1's lexicographic ordering, and the two notions diverge. The all-zero tumbler `[0, 0]` exhibits the divergence concretely: under T1, `[0] < [0, 0]` by case (ii) ‚Äî the prefix rule, since `[0]` is a proper prefix of `[0, 0]` ‚Äî yet `¬¨Pos([0, 0])`, since every component of `[0, 0]` is zero. The converse direction does hold: `Pos(t)` implies `t` is T1-greater than every zero tumbler, as the following proof establishes.

Every positive tumbler is greater than every zero tumbler under T1.

*Proof.* Let `t ‚àà T` with `Pos(t)` and let `z ‚àà T` be a zero tumbler; we show `z < t`. Since `Pos(t)`, ActionPoint applies to `t` ‚Äî its sole precondition is `Pos(w)`, which `t` satisfies ‚Äî and supplies the index `actionPoint(t)` together with three postconditions: (i) `1 ‚â§ actionPoint(t) ‚â§ #t`; (ii) `t·µ¢ = 0` for all `1 ‚â§ i < actionPoint(t)`; (iii) `t_{actionPoint(t)} ‚â• 1`. Earlier drafts of this proof reconstructed these three facts from a direct NAT-wellorder appeal to the set `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ ‚âÝ 0}` ‚Äî least element via well-ordering, minimality-yields-zeros-below corollary, and nonzero-at-the-least-index ‚Äî but that reconstruction duplicates ActionPoint's construction verbatim (same set, same axiom, same minimality-yields-zeros-below corollary), so the proof below instead consumes `actionPoint(t)` as the T1 divergence witness and delegates the construction to ActionPoint, paralleling TA-strict's delegation of its ordering postcondition to TumblerAdd rather than quoting TumblerAdd's four-postcondition list. Write `k` for `actionPoint(t)`. Since `z` is a zero tumbler, `z·µ¢ = 0` for all `i ‚â§ #z`. We distinguish two cases by the relationship between `#z` and `k`.

*Case `#z ‚â• k`.* For `1 ‚â§ i < k` we have `z·µ¢ = 0` (since `z` is a zero tumbler) and `t·µ¢ = 0` (ActionPoint's postcondition (ii)), so `z·µ¢ = t·µ¢`, establishing the T1 agreement condition. The case hypothesis `#z ‚â• k` supplies `k ‚â§ #z`, and ActionPoint's postcondition (i) supplies `k ‚â§ #t`, establishing the pair `k ‚â§ #z ‚àß k ‚â§ #t` required by T1 case (i). We now establish `z‚Çñ = 0 < t‚Çñ`: `z‚Çñ = 0` since `z` is a zero tumbler, and ActionPoint's postcondition (iii) supplies `1 ‚â§ t‚Çñ`. NAT-addcompat's strict successor inequality `n < n + 1` instantiated at `n = 0` supplies `0 < 0 + 1`, equivalently `0 < 1`; NAT-order's definition `m ‚â§ n ‚ü∫ m < n ‚à® m = n` instantiated at `m = 1, n = t‚Çñ` unfolds `1 ‚â§ t‚Çñ` to `1 < t‚Çñ ‚à® 1 = t‚Çñ`, and the two disjuncts compose with `0 < 1` to deliver `0 < t‚Çñ` ‚Äî strict disjunct via NAT-order's transitivity of `<` on ‚Ñï composing `0 < 1 < t‚Çñ` into `0 < t‚Çñ`, equality disjunct via substitution of `1 = t‚Çñ` into `0 < 1`. By T1 case (i) with witness `k`, `z < t`.

*Case `#z < k`.* For `1 ‚â§ i ‚â§ #z` we have `i < k` (since `i ‚â§ #z < k`), whence `t·µ¢ = 0` (ActionPoint's postcondition (ii)) and `z·µ¢ = 0` (since `z` is a zero tumbler), so `t·µ¢ = 0 = z·µ¢`, establishing the T1 agreement condition at every position of `z`. From `#z < k` and ActionPoint's postcondition (i)'s `k ‚â§ #t` ‚Äî composed via NAT-order's `<`/`‚â§` transitivity (the strict disjunct `k < #t` of the unpacked `k ‚â§ #t` composes with `#z < k` by NAT-order's transitivity of `<` to yield `#z < #t`; the equality disjunct `k = #t` substitutes into `#z < k` to yield `#z < #t` directly) ‚Äî we obtain `#z < #t`. NAT-discrete's axiom `m ‚â§ n < m + 1 ‚üπ n = m` in its contrapositive form ‚Äî for `m, n ‚àà ‚Ñï`, `m < n ‚üπ m + 1 ‚â§ n`, since the contrary assumption `¬¨(m + 1 ‚â§ n)` gives `n < m + 1` by NAT-order's trichotomy, combined with `m < n` yields `m ‚â§ n < m + 1`, whence NAT-discrete forces `n = m`, contradicting `m < n` by NAT-order's irreflexivity ‚Äî instantiated at `m = #z, n = #t` yields `#z + 1 ‚â§ #t`. By T1 case (ii) with witness `#z + 1`, `z < t`. ‚àé

*Formal Contract:*
- *Definition:* `Pos(t)` (positive) iff `(E i : 1 ‚â§ i ‚â§ #t : t·µ¢ ‚âÝ 0)`. Zero tumbler: `(A i : 1 ‚â§ i ‚â§ #t : t·µ¢ = 0)`.
- *Depends:* T0 (CarrierSetDefinition) ‚Äî the carrier set `T`, the length `#t`, and the component projection `t·µ¢` used in the Definition (`Pos(t)` iff `(E i : 1 ‚â§ i ‚â§ #t : t·µ¢ ‚âÝ 0)`) and in the zero-tumbler companion definition (`(A i : 1 ‚â§ i ‚â§ #t : t·µ¢ = 0)`) all come from T0's characterisation of T as finite sequences over ‚Ñï with length ‚â• 1. ActionPoint (ActionPoint) ‚Äî the postcondition proof consumes `actionPoint(t)` as its T1 divergence witness; ActionPoint's three postconditions discharge, respectively, the bound `1 ‚â§ actionPoint(t) ‚â§ #t` used in Case `#z ‚â• k`'s T1 case (i) conjunct `k ‚â§ #t` and in Case `#z < k`'s NAT-order composition `#z < k ‚â§ #t` (postcondition (i)), the agreement zeros `t·µ¢ = 0` for `1 ‚â§ i < actionPoint(t)` used in both cases' T1 agreement condition (postcondition (ii)), and the lower bound `1 ‚â§ t_{actionPoint(t)}` used in Case `#z ‚â• k`'s derivation `z‚Çñ = 0 < t‚Çñ` (postcondition (iii)). The delegation is deliberate: ActionPoint is the ASN's canonical site of the least-element-of-`{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ ‚âÝ 0}` construction ‚Äî including the NAT-wellorder appeal that names the least element and the minimality-yields-zeros-below corollary ‚Äî and TA-Pos's postcondition reads its witness and the three accompanying facts off ActionPoint rather than reconstructing the construction in parallel, paralleling TA-strict's delegation of its ordering postcondition to TumblerAdd rather than quoting TumblerAdd's four-postcondition list. The inverse citation ‚Äî ActionPoint's `depends` listing TA-Pos for the `Pos(w)` predicate ‚Äî is logically non-circular: `Pos(t)` is a T0-level existential definition requiring no further axiomatic input, ActionPoint constructs `actionPoint(w)` from `Pos(w)` via NAT-wellorder, and TA-Pos's postcondition theorem proves `z < t` under `Pos(t)` using that construction, so the two YAML `depends` entries reflect two logical layers ‚Äî the definition consumption ActionPoint ‚Üí TA-Pos and the theorem consumption TA-Pos ‚Üí ActionPoint ‚Äî flattened into a mutual citation without a content-level cycle. NAT-wellorder (NatWellOrdering) ‚Äî retained as a transitive sourcing handle through ActionPoint: ActionPoint's construction of `actionPoint(t)` as the least element of `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ ‚âÝ 0}` invokes NAT-wellorder's well-ordering of ‚Ñï, and TA-Pos's postcondition reads that least element off ActionPoint rather than invoking NAT-wellorder directly. Listing NAT-wellorder here preserves the Depends-backed signal a reviser needs when tightening NAT-wellorder's invocation pattern at ActionPoint, so that changes to the action-point construction propagate to TA-Pos's postcondition through this handle without re-audit; the former direct prose invocation inside TA-Pos's proof ‚Äî "NAT-wellorder's well-ordering of ‚Ñï furnishes a smallest element `k` with `t‚Çñ ‚âÝ 0`" ‚Äî is now replaced by the ActionPoint citation that routes through this transitive handle. NAT-zero (NatZeroMinimum) ‚Äî retained as a transitive sourcing handle through ActionPoint: ActionPoint's derivation of postcondition (iii) `w_{actionPoint(w)} ‚â• 1` threads through NAT-zero's `(A n ‚àà ‚Ñï :: 0 ‚â§ n)` as the `0 ‚â§ w_{actionPoint(w)}` premise that NAT-order's ‚â§-unfolding consumes en route to the `0 < w_{actionPoint(w)}` intermediate that NAT-discrete then upgrades to `‚â• 1`. Listing NAT-zero here preserves the Depends-backed handle for this cascade so that revisions to ActionPoint's postcondition (iii) adjusting the NAT-zero premise pattern propagate to TA-Pos's Case `#z ‚â• k` without re-audit; the former direct invocation inside TA-Pos's proof ‚Äî "NAT-zero furnishes `0 ‚â§ t‚Çñ` since `t‚Çñ ‚àà ‚Ñï`" ‚Äî is now replaced by the ActionPoint citation of postcondition (iii), which already incorporates the NAT-zero step. NAT-addcompat (NatAdditionOrderAndSuccessor) ‚Äî Case `#z ‚â• k`'s step from ActionPoint's postcondition (iii) `1 ‚â§ t‚Çñ` to `0 < t‚Çñ` threads through `0 < 1` as the stepping stone, which NAT-addcompat's strict successor inequality `n < n + 1` instantiated at `n = 0` supplies directly as `0 < 0 + 1`. No other listed axiom produces `0 < 1` in TA-Pos's scope: NAT-wellorder's least-element principle does not name `0`'s successor, NAT-zero's `(A n ‚àà ‚Ñï :: 0 ‚â§ n)` fixes `0` as a lower bound without producing a strict comparison, and ActionPoint's postcondition (iii) supplies the form `1 ‚â§ t‚Çñ` into which `0 < 1` composes rather than the `0 < 1` stepping stone itself. NAT-discrete (NatDiscreteness) ‚Äî Case `#z < k`'s passage from `#z < #t` (obtained by composing `#z < k` with ActionPoint's postcondition (i)'s `k ‚â§ #t` via NAT-order's `<`/`‚â§` transitivity) to `#z + 1 ‚â§ #t` is licensed by NAT-discrete in its contrapositive form ‚Äî for `m, n ‚àà ‚Ñï`, `m < n ‚üπ m + 1 ‚â§ n`, since the contrary assumption `¬¨(m + 1 ‚â§ n)` gives `n < m + 1` by NAT-order's trichotomy, combined with `m < n` yields `m ‚â§ n < m + 1`, whence NAT-discrete forces `n = m`, contradicting `m < n` by NAT-order's irreflexivity ‚Äî instantiated at `m = #z, n = #t` yields the T1 case (ii) witness `#z + 1 ‚â§ #t`, so the `m < n ‚üπ m + 1 ‚â§ n` inference is discharged from NAT-discrete (with NAT-order) rather than left implicit. NAT-order (NatStrictTotalOrder) ‚Äî supplies (i) the definition of `‚â§` used in Case `#z ‚â• k` to unfold `1 ‚â§ t‚Çñ` into `1 < t‚Çñ ‚à® 1 = t‚Çñ`, whose two disjuncts compose with NAT-addcompat's `0 < 1` to yield `0 < t‚Çñ` ‚Äî strict disjunct via NAT-order's transitivity of `<` on ‚Ñï composing `0 < 1 < t‚Çñ` into `0 < t‚Çñ`, equality disjunct via substitution of `1 = t‚Çñ` into `0 < 1` ‚Äî the step that replaces the former NAT-zero-routed `0 ‚â§ t‚Çñ ‚à® 0 = t‚Çñ` unfolding; (ii) the `<`/`‚â§` composition in Case `#z < k` deriving `#z < #t` from `#z < k` and `k ‚â§ #t` ‚Äî NAT-order's transitivity of `<` composes the strict disjunct `k < #t` of the unpacked `k ‚â§ #t` with `#z < k`, and substitution handles the equality disjunct `k = #t` ‚Äî this step replaces the former direct derivation of `#z + 1 ‚â§ #t` from `#z < k ‚â§ #t` that conflated `<` and `‚â§` within a single unpacking; and (iii) the trichotomy, irreflexivity, and `‚â§`-transitivity steps composed with NAT-discrete in the Case `#z < k` contrapositive argument producing `#z + 1 ‚â§ #t` from `#z < #t`. T1 (LexicographicOrder) ‚Äî the postcondition proof invokes T1 case (i) when `#z ‚â• k` to conclude `z < t` from `z‚Çñ = 0 < t‚Çñ` together with the bound `k ‚â§ #z ‚àß k ‚â§ #t`, and T1 case (ii) when `#z < k` to conclude `z < t` from `z` being a proper prefix of `t` with witness `#z + 1`.
- *Postconditions:* `(A t ‚àà T, z ‚àà T : Pos(t) ‚àß (A i : 1 ‚â§ i ‚â§ #z : z·µ¢ = 0) :: z < t)` ‚Äî every positive tumbler is strictly greater under T1 than every zero tumbler of any length.
