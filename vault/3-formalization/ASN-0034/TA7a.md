## Subspace closure

When arithmetic advances a position within one element subspace, the result must remain in that subspace. Text positions must not cross into link space, and vice versa.

An element-local position within subspace `S` has two components: the subspace identifier `N` and the ordinal `x`. A natural first attempt at an element-local displacement is `w = [0, n]` — action point `k = 2`, preserving the subspace identifier and advancing the ordinal. Addition works: `[N, x] ⊕ [0, n] = [N, x + n]`, preserving the subspace. But subtraction exposes a subtlety: `[N, x] ⊖ [0, n]` finds the first divergence at position 1 (where `N ≠ 0`), not at position 2 where the intended action lies. The subtraction produces `[N - 0, x] = [N, x]` — a no-op. The abstract `⊖` cannot shift a position backward by a displacement that disagrees with the position at the subspace identifier.

Gregory's implementation reveals the resolution. The operands passed to the arithmetic during shifts are not full element-local positions; they are *within-subspace ordinals* — the second component alone. The subspace identifier is not an operand to the shift; it is structural context that determines *which* positions are subject to the shift. The arithmetic receives ordinals, not full positions.

**TA7a (SubspaceClosure).** The canonical representation for shift arithmetic is the *ordinal-only* formulation: a position in a subspace with identifier `N` and ordinal `o = [o₁, ..., oₘ]` (where `m ≥ 1`) is represented as the tumbler `o` for arithmetic purposes, with `N` held as structural context. Define **S** = {o ∈ T : #o ≥ 1 ∧ (A i : 1 ≤ i ≤ #o : oᵢ > 0)} — ordinals with all positive components, matching the shape of an element field under T4 (a non-zero run delimited by separators; positivity of non-zero entries is supplied by T0's carrier ℕ). An element-local displacement is a positive tumbler `w` with action point `k` satisfying `1 ≤ k ≤ m`. In this formulation:

  `(A o ∈ S, Pos(w) : k ≤ #o ⟹ o ⊕ w ∈ T)`

  `(A o ∈ S, Pos(w) : o ≥ w ⟹ o ⊖ w ∈ T)`

Both claims assert closure in T: arithmetic on ordinals, with the subspace identifier held as structural context, produces results that remain in T. The subspace identifier is not an operand — it determines *which* positions are subject to the shift, but never enters the arithmetic. This design ensures that no shift can escape the subspace.

The ordinal-only formulation is not arbitrary. The natural 2-component formulation `[N, x]` fails for subtraction: `[N, x] ⊖ [0, n]` finds the divergence at position 1 (where `N > 0 = 0`), producing `[N, x]` — a no-op rather than a genuine shift. Stripping the subspace identifier from the operands avoids this degeneracy.

*Proof.* We prove each conjunct of TA7a, then analyze the finer question of S-membership.

Let `o = [o₁, ..., oₘ]` with `o ∈ S`, so `m ≥ 1` and every `oᵢ > 0`. Let `w` be a positive displacement with action point `k = min({i : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0})`.

**Conjunct 1** (`⊕`-closure in T). The precondition gives `o ∈ T`, `w ∈ T`, `Pos(w)`, and `k ≤ #o = m`. These are exactly the preconditions of TA0 (well-defined addition). By TA0, `o ⊕ w ∈ T`, with `#(o ⊕ w) = #w`. The subspace identifier, held as structural context outside the operands, is untouched.

A stronger result holds for S-membership. By TumblerAdd's constructive definition, the result `r = o ⊕ w` has components: `rᵢ = oᵢ > 0` for `1 ≤ i < k` (prefix copied from `o ∈ S`); `rₖ = oₖ + wₖ > 0` (since `oₖ > 0` because `o ∈ S`, and `wₖ > 0` because `k` is the action point of `w`); and `rᵢ = wᵢ` for `k < i ≤ #w` (tail copied from the displacement). Components before and at the action point are positive. The result is in S precisely when every tail component `wᵢ` (for `i > k`) is also positive. For single-component ordinals — the common case — `[x] ⊕ [n] = [x + n]`, which is unconditionally in S since both `x > 0` and `n > 0`.

For example, spanning from ordinal `[1, 3, 2]` to `[1, 5, 7]` requires displacement `[0, 2, 7]` (action point `k = 2`). TumblerAdd produces `[1, 3 + 2, 7] = [1, 5, 7]` — position 1 of the ordinal is copied from the start, preserving the ordinal prefix.

**Conjunct 2** (`⊖`-closure in T). The precondition gives `o ∈ T`, `w ∈ T`, `Pos(w)`, and `o ≥ w`. The triple `o ∈ T`, `w ∈ T`, `o ≥ w` matches the preconditions of TA2 (well-defined subtraction); the additional `Pos(w)` is *not* consumed by TA2 — TA2's contract delivers T-closure for any `w ∈ T` satisfying `o ≥ w`, irrespective of positivity — but is held in reserve for the action-point case analysis below, where the action point `k = actionPoint(w)` is well-defined only under `Pos(w)` (per ActionPoint's contract, whose own precondition list pairs `w ∈ T` with `Pos(w)`). By TA2 applied at the triple, `o ⊖ w ∈ T`. The subspace identifier is again untouched.

The S-membership question for `⊖` is more delicate. We analyze by action point and divergence position, using TumblerSub's constructive definition: zero-pad to length `L` — the longer of `#o` and `#w`, named by NAT-order's trichotomy on `(#o, #w)` per TumblerSub's Definition (sub-case (α) `#o = #w` with `L = #o`; sub-case (β) `#o < #w` with `L = #w`; sub-case (γ) `#w < #o` with `L = #o`) rather than by a primitive binary-maximum operator on ℕ, matching the per-step convention TumblerSub and TA2 apply at the structurally identical length-pair dispatch — find the divergence position `d` (the first position where the padded sequences differ), then set `rᵢ = 0` for `i < d`, `r_d = o_d - w_d`, and `rᵢ = oᵢ` (zero-padded) for `i > d`.

*Preliminary: when `#w > m`.* The hypothesis `#w > m` is the same as `#o < #w` at `#o = m`, placing the length pair `(#o, #w) = (m, #w)` in sub-case (β) of TumblerSub's NAT-order trichotomy dispatch — where `#o < #w` and `L = #w` — so TumblerSub produces a result of length `L = #w > m` rather than naming the length by a primitive binary-maximum operator on ℕ. The zero-padded minuend has zeros at positions `m + 1` through `#w`, so the result inherits trailing zeros at those positions and lies in T \ S. The cases below assume `#w ≤ m`.

*Case `k ≥ 2`:* The displacement has `wᵢ = 0` for all `i < k`, so in particular `w₁ = 0`. Since `o ∈ S`, `o₁ > 0`. Therefore `o₁ ≠ w₁` and the divergence falls at `d = 1`. TumblerSub produces: `r₁ = o₁ − 0 = o₁ > 0` — the identity `o₁ − 0 = o₁` is obtained by instantiating NAT-sub's left-inverse characterisation `n + (m − n) = m` at `m = o₁, n = 0`, whose precondition `o₁ ≥ 0` is supplied by NAT-zero's lower-bound axiom, yielding `0 + (o₁ − 0) = o₁`; NAT-closure's additive identity `0 + n = n` at `n = o₁ − 0` then rewrites the left-hand side to `o₁ − 0`, closing the identity `o₁ − 0 = o₁`. Routing through the left-inverse form rather than NAT-sub's right-telescoping clause `(o₁ + 0) − 0 = o₁` is forced by the NAT-* citation discipline: right-telescoping would require `o₁ + 0 = o₁` (the right additive identity) to collapse, and NAT-closure supplies only the left identity `0 + o₁ = o₁`, so a right-telescoping route would have to appeal to commutativity of addition on ℕ — an axiom that T0's NAT-* enumeration excludes, and that NAT-sub's own contract avoids precisely by stating both inverse forms explicitly. For `1 < i ≤ m`, `rᵢ = oᵢ > 0` (copied from the minuend since `i > d = 1`). When `#w ≤ m`, the result has length `m` and equals `o` itself — a no-op. The result is trivially in S. This is the vacuous closure: TumblerSub finds the mismatch at the ordinal's first positive component rather than at the displacement's intended action point.

*Case `k = 1`, divergence `d = 1`:* The displacement has `w₁ > 0`, and `o₁ ≠ w₁`. Since `o ≥ w`, the ordering at the first divergence position requires `o₁ > w₁` (T1). TumblerSub produces: `r₁ = o₁ - w₁ > 0` by NAT-sub (strict positivity), whose precondition `o₁ > w₁` is the strict inequality just derived; `r₁ ≥ 1` since `o₁ > w₁ ≥ 1`. For `1 < i ≤ m`, `rᵢ = oᵢ > 0` (copied from the minuend). When `#w ≤ m`, all components are positive and the result is in S.

*Case `k = 1`, divergence `d > 1`:* The displacement has `w₁ > 0`, and `o₁ = w₁` (the operands agree at position 1, with divergence at some later `d > 1`). TumblerSub zeros all positions before `d`: `rᵢ = 0` for `1 ≤ i < d`. In particular `r₁ = 0`, so the result has a zero first component and is not in S. Counterexample: `o = [5, 3]`, `w = [5, 1]` (action point `k = 1`, divergence `d = 2`). TumblerSub yields `r = [0, 3 - 1] = [0, 2]` — here `3 - 1 = 2` is NAT-sub's conditional-closure and right-inverse clauses on the naturals `3, 1 ∈ ℕ` with `3 ≥ 1`. We have `[0, 2] ∈ T` (confirming the T-closure claim) but `[0, 2] ∉ S ∪ Z`. This sub-case arises when `o` and `w` share a leading prefix — the subtraction produces a displacement-like tumbler with leading zeros rather than a valid ordinal position.

For single-component ordinals, the `d > 1` sub-case cannot arise (there is only one position), and `⊖` gives closure in S ∪ Z: `[x] ⊖ [n]` is `[x - n] ∈ S` when `x > n`, or `[0] ∈ Z` when `x = n` (a sentinel, TA6).

In every case, the result lies in T. The subspace identifier, held as structural context outside the operands, is never modified by either operation. TA7a holds. ∎

The restriction to element-local displacements is necessary. An unrestricted displacement whose action point falls at the subspace-identifier position could produce an address in a different subspace — TA7a cannot hold for arbitrary `w`.

*Formal Contract:*
- *Preconditions:* For `⊕`: `o ∈ S`, `w ∈ T`, `Pos(w)`, `actionPoint(w) ≤ #o`. For `⊖`: `o ∈ S`, `w ∈ T`, `Pos(w)`, `o ≥ w`.
- *Depends:* T0 (CarrierSetDefinition) — invoked at the **S** definition and the proof opening to source positivity from the carrier ℕ ("positivity of non-zero entries is supplied by T0's carrier ℕ"); without T0 the all-positive characterization of S would be unsourced. T4 (HierarchicalParsing) — invoked at the **S** definition to anchor the subspace shape against a structural reference ("matching the shape of an element field under T4 (a non-zero run delimited by separators)"). T1 (LexicographicOrder) — invoked in *Case `k = 1`, divergence `d = 1`* to convert the dominance precondition `o ≥ w` into the strict inequality at the divergence position ("Since `o ≥ w`, the ordering at the first divergence position requires `o₁ > w₁` (T1)"); without T1 the step from `o ≥ w` and `o₁ ≠ w₁` to `o₁ > w₁` would be unjustified. TA-Pos (PositiveTumbler) — invoked at the proof opening to license the existence of an action point for `w` ("Let `w` be a positive displacement with action point `k = min({i : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0})`") and quoted in both conjunct preconditions through `Pos(w)`. Additionally consumed for the set-form **Z** = {t ∈ T : Zero(t)} that TA-Pos introduces as the reification of its `Zero` predicate: `Z` names the zero-tumbler set referenced in *Case `k = 1`, divergence `d > 1`*'s counterexample ("`[0, 2] ∉ S ∪ Z`"), in the single-component-ordinal closing note ("`⊖` gives closure in S ∪ Z" and "`[0] ∈ Z` when `x = n`"), and in the postcondition clauses for `⊖` on single-component ordinals ("the result is in `S ∪ Z`: `[x] ⊖ [n] ∈ S` when `x > n`, and `[x] ⊖ [n] ∈ Z` when `x = n`"). Without TA-Pos's set-form reification the bare symbol `Z` would reference no defined object; the set is bound at TA-Pos rather than respecified inline here so that the union `S ∪ Z` names a single set-theoretic composition (the ordinals whose components are all positive together with the zero tumblers) across every consumer of TA7a's postcondition. ActionPoint (ActionPoint) — invoked at the proof opening to fix the action point as the least-non-zero-position formula ("`k = min({i : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0})`") and used throughout to identify the prefix-zero region of `w` (so that `wᵢ = 0` for `i < k` is available in case `k ≥ 2`). TumblerAdd (TumblerAdd) — invoked at Conjunct 1 for the constructive three-region rule ("By TumblerAdd's constructive definition, the result `r = o ⊕ w` has components: `rᵢ = oᵢ > 0` for `1 ≤ i < k` (prefix copied from `o ∈ S`); `rₖ = oₖ + wₖ > 0`; and `rᵢ = wᵢ` for `k < i ≤ #w` (tail copied from the displacement)") used to derive S-membership component-by-component. TumblerSub (TumblerSub) — invoked at Conjunct 2 for the divergence-discovery rule ("using TumblerSub's constructive definition: zero-pad to length `L` — the longer of `#o` and `#w`, named by NAT-order's trichotomy on `(#o, #w)` per TumblerSub's Definition … — find the divergence position `d`, then set `rᵢ = 0` for `i < d`, `r_d = o_d - w_d`, and `rᵢ = oᵢ` for `i > d`") and at every sub-case to compute the result's components and decide S-membership. TumblerSub's Definition is also consumed at the Preliminary case `#w > m` to furnish the length-pair dispatch sub-case (β) (`#o < #w`, `L = #w`) that names the result length `L = #w > m` in place of a primitive `max(m, #w)`, matching the per-step convention TA2 applies for the structurally identical sub-case identification. NAT-order (NatStrictTotalOrder) — invoked at the length pair `(#o, #w)` at two sites to discharge the length-pair dispatch that names the result length `L` of `o ⊖ w` rather than invoking `max(#o, #w)` as an unsourced primitive binary-maximum operator on ℕ, matching the per-step convention TumblerSub and TA2 apply at the structurally identical length-pair dispatch for the same operand pair. (i) *Conjunct 2 opening (result-length recitation)*: trichotomy's exactly-one clause selects one of sub-case (α) `#o = #w` with `L = #o`, sub-case (β) `#o < #w` with `L = #w`, or sub-case (γ) `#w < #o` with `L = #o`, naming `L` explicitly in the construction TA7a's proof recites from TumblerSub's Definition. (ii) *Preliminary case `#w > m`*: the same trichotomy axiom, under the hypothesis `#o = m < #w`, places the length pair in sub-case (β), whose longer length `L = #w` furnishes the result length `#(o ⊖ w) = #w > m` consumed to identify the trailing-zero region of positions `m + 1` through `#w` that excludes the result from S; without NAT-order's trichotomy there is no source for the length identity `L = #w` at this hypothesis without appeal to a primitive `max` operator on ℕ. The two sites are enumerated together under this single NAT-order citation, matching the per-step convention TumblerSub and TA2 apply when a single axiom is invoked at multiple sites within a single proof. TA0 (WellDefinedAddition) — invoked at Conjunct 1 to discharge `⊕`-closure in T ("These are exactly the preconditions of TA0 (well-defined addition). By TA0, `o ⊕ w ∈ T`, with `#(o ⊕ w) = #w`"); without TA0 the T-closure of `⊕` would not be sourced. TA2 (WellDefinedSubtraction) — invoked at Conjunct 2 to discharge `⊖`-closure in T ("These are exactly the preconditions of TA2 (well-defined subtraction). By TA2, `o ⊖ w ∈ T`"). TA6 (ZeroTumblers) — invoked at the closing single-component-ordinal note to interpret `[0]` as a sentinel rather than a position ("`[0] ∈ Z` when `x = n` (a sentinel, TA6)"); without TA6 the appearance of `[0]` in S ∪ Z would be unexplained. NAT-sub (NatPartialSubtraction) — left-inverse characterisation discharges `o₁ − 0 = o₁` in Case `k ≥ 2` by instantiation at `m = o₁, n = 0` (with the precondition `o₁ ≥ 0` supplied by NAT-zero), yielding `0 + (o₁ − 0) = o₁` which NAT-closure's left additive identity then rewrites to `o₁ − 0 = o₁`; the route is through the left-inverse form rather than right-telescoping because right-telescoping at `n = 0` yields `(o₁ + 0) − 0 = o₁` whose collapse to `o₁ − 0 = o₁` would require the right additive identity `o₁ + 0 = o₁` that NAT-closure does not supply, so the right-telescoping route would tacitly appeal to commutativity of addition on ℕ — an axiom absent from T0's NAT-* enumeration and one that NAT-sub's own contract avoids by stating both inverse forms explicitly. Strict positivity discharges `o₁ - w₁ > 0` in Case `k = 1, d = 1` (from `o₁ > w₁`); conditional closure licenses every divergence-point subtraction in the counterexample and elsewhere. NAT-closure (NatArithmeticClosureAndIdentity) — supplies the left additive identity `0 + n = n` at `n = o₁ − 0` to rewrite the left-hand side of `0 + (o₁ − 0) = o₁` (obtained from NAT-sub's left-inverse characterisation at `m = o₁, n = 0`) to `o₁ − 0`, closing the identity `o₁ − 0 = o₁` in Case `k ≥ 2`. NAT-zero (NatZeroMinimum) — supplies the precondition `o₁ ≥ 0` required by NAT-sub's left-inverse characterisation at `m = o₁, n = 0` in Case `k ≥ 2`; without NAT-zero's lower-bound axiom `(A n ∈ ℕ :: 0 ≤ n)` the instantiation would lack the `m ≥ n` side-condition that NAT-sub's left-inverse clause quantifies under, and the derivation of `o₁ − 0 = o₁` could not invoke the left-inverse form that the rest of the ASN's subtraction-identity discipline favours over right-telescoping.
- *Postconditions:* `o ⊕ w ∈ T`, `#(o ⊕ w) = #w`. `o ⊖ w ∈ T`. For `⊕`, the result is in S when all tail components of `w` (after the action point) are positive. For `⊖` with `actionPoint(w) ≥ 2` and `#w ≤ #o`: the divergence falls at position 1, TumblerSub produces `o` itself (a no-op), and the result is in S. For `⊖` with `actionPoint(w) = 1` and divergence at position `d = 1` (i.e., `o₁ ≠ w₁`): `r₁ = o₁ - w₁ > 0` and `rᵢ = oᵢ > 0` for `i > 1`, so the result is in S when `#w ≤ #o`. For `⊖` with `actionPoint(w) = 1` and divergence at position `d > 1` (i.e., `o₁ = w₁`): the result has `r₁ = 0` and lies in `T \ S` (counterexample: `[5, 3] ⊖ [5, 1] = [0, 2]`). For `⊖` when `#w > #o`: the result inherits trailing zeros at positions `#o + 1` through `#w` and lies in `T \ S`. For `⊖` on single-component ordinals (`#o = 1`, `#w = 1`): the result is in `S ∪ Z`: `[x] ⊖ [n] ∈ S` when `x > n`, and `[x] ⊖ [n] ∈ Z` when `x = n`.
