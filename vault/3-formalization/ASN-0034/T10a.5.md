**T10a.5 (CrossAllocatorIncomparability).** For any two allocators X and Y not in an ancestor-descendant relationship, every output of X is prefix-incomparable with every output of Y. This alone establishes T10's non-nesting precondition `p₁ ⋠ p₂ ∧ p₂ ⋠ p₁` at the domain prefixes of every non-ancestor-descendant allocator pair in the tree, since T10's precondition is a cross-allocator condition and T10a.2 — the disjoint within-allocator incomparability of distinct siblings of one allocator — does not participate in supplying it. (Ancestor-descendant pairs inherently nest by TA5(b) and TA5(d); their output distinctness follows from length separation via T10a.3, not from T10.)

*Proof.* Two allocators not in an ancestor-descendant relationship share a lowest common ancestor C. Let C_X be the child of C on the path to X (with X = C_X when X is a direct child), and C_Y the child on the path to Y. Since C is the lowest common ancestor and X, Y are not ancestor-descendant related, C_X ≠ C_Y. Let s_X be the output of C from which C_X was spawned, and s_Y the output from which C_Y was spawned.

We require a preliminary fact.

**Positional inheritance.** Let allocator A have base b of length m, and let p be a position with 1 ≤ p < m. Then every output of every allocator in the subtree rooted at A has value b_p at position p.

For A itself: every output has length m by T10a.1, and inc(·, 0) modifies only position sig(·) = m (by TA5-SigValid, since every output satisfies T4 by T10a.4). Since p < m, position p is unmodified: every output of A has value b_p at position p. For a child of A spawned from output u via inc(u, k'') with k'' ∈ {1, 2}: the child's base c has length m + k'' and agrees with u on positions 1 through m by TA5(b). Since u_p = b_p (established for A's outputs), c_p = b_p. Since p < m < m + k'', position p satisfies p < #c, and the argument applies to the child's subtree. By induction on subtree depth, every output of every descendant has value b_p at position p. ∎ (positional inheritance)

The main proof splits on whether the two lineages diverge at different parent outputs or at the same parent output with different spawning parameters.

**Case 1: s_X ≠ s_Y.** Both are sibling outputs of C, so #s_X = #s_Y = L by T10a.1. Since s_X ≠ s_Y and both have length L, T3 yields a position j with 1 ≤ j ≤ L where (s_X)_j ≠ (s_Y)_j.

The base of C_X is inc(s_X, k'_X) for some k'_X ∈ {1, 2}, which has length L + k'_X and agrees with s_X on positions 1 through L by TA5(b). Since j ≤ L < L + k'_X, positional inheritance gives: every output in C_X's subtree has value (s_X)_j at position j. The base of C_Y is inc(s_Y, k'_Y) for some k'_Y ∈ {1, 2}, which has length L + k'_Y and agrees with s_Y on positions 1 through L by TA5(b). Since j ≤ L < L + k'_Y, positional inheritance gives: every output in C_Y's subtree has value (s_Y)_j at position j.

For any output x of X and y of Y: x_j = (s_X)_j ≠ (s_Y)_j = y_j. Every output in C_X's subtree has length at least L + 1, since C_X's own outputs have length L + k'_X ≥ L + 1 by T10a.1 and deeper descendants are strictly longer by T10a.3; the same holds for C_Y's subtree. So j ≤ L < L + 1 ≤ min(#x, #y), placing position j within the range of both tumblers. The disagreement at position j excludes both x ≼ y and y ≼ x by the Prefix definition. ∎ (Case 1)

**Case 2: s_X = s_Y = s.** The at-most-once child-spawning constraint of T10a requires that each (t, k') pair produces at most one child. Since C_X ≠ C_Y are both spawned from s, they must use distinct spawning parameters: k'_X ≠ k'_Y with {k'_X, k'_Y} ⊆ {1, 2}. Label them so that k'_X = 1 and k'_Y = 2.

The base of C_X is inc(s, 1), which has length #s + 1 with (inc(s, 1))_{#s+1} = 1 by TA5(d) — the final position of the extension is set to 1. The base of C_Y is inc(s, 2), which has length #s + 2 with (inc(s, 2))_{#s+1} = 0 (field separator) and (inc(s, 2))_{#s+2} = 1 by TA5(d).

We show that every output in C_X's subtree has value ≥ 1 at position #s + 1, while every output in C_Y's subtree has value 0 there.

*Outputs of C_X at position #s + 1.* Position #s + 1 is the last position of C_X's outputs (length #s + 1 by T10a.1). The base has value 1 there. Each application of inc(·, 0) advances this position by 1 (TA5(c)), producing values 1, 2, 3, .... Every output of C_X has value ≥ 1 at position #s + 1.

*Descendants of C_X at position #s + 1.* Any child of C_X is spawned from an output u of C_X with u_{#s+1} ≥ 1. The child's base has length #s + 1 + k'' ≥ #s + 2 for some k'' ∈ {1, 2}, and agrees with u on positions 1 through #s + 1 by TA5(b), inheriting a value ≥ 1 at position #s + 1. Since #s + 1 < #s + 1 + k'', positional inheritance propagates this value through the child's entire subtree. By induction on depth, every output of every descendant of C_X has value ≥ 1 at position #s + 1.

*All outputs in C_Y's subtree at position #s + 1.* The base of C_Y has value 0 at position #s + 1, and #s + 1 < #s + 2 = length of C_Y's base. Positional inheritance gives: every output of every allocator in C_Y's subtree has value 0 at position #s + 1.

For any output x of X and y of Y: x_{#s+1} ≥ 1 and y_{#s+1} = 0, so x_{#s+1} ≠ y_{#s+1}. Every output in C_X's subtree has length ≥ #s + 1, and every output in C_Y's subtree has length ≥ #s + 2 > #s + 1, so position #s + 1 lies within the range of both x and y. The disagreement at position #s + 1 excludes both x ≼ y and y ≼ x by the Prefix definition. ∎

*Formal Contract:*
- *Precondition:* Allocators X and Y conforming to T10a, not in an ancestor-descendant relationship.
- *Postcondition:* For all x ∈ domain(X) and y ∈ domain(Y): x ⋠ y ∧ y ⋠ x.
- *Depends:* T10a (at-most-once child-spawning constraint), T10a.1 (uniform sibling length), T10a.3 (length separation across depths), T10a.4 (T4 preservation for TA5-SigValid applicability), T4 (HierarchicalParsing — TA5-SigValid precondition: `sig(t) = #t` holds only for T4-valid addresses), TA5 (postconditions (b), (c), (d)), TA5-SigValid (sig = length for T4-valid addresses), T3 (distinct same-length tumblers diverge at some position), Prefix (definition of ≼). This alone delivers T10's non-nesting precondition `p₁ ⋠ p₂ ∧ p₂ ⋠ p₁` at the domain prefixes of every non-ancestor-descendant allocator pair in the tree, since T10's precondition is a cross-allocator condition and T10a.2 — the within-allocator incomparability of co-siblings — does not participate in supplying it. Ancestor-descendant pairs inherently nest (TA5(b), TA5(d)); their output distinctness follows from length separation (T10a.3).
