**T10a.8 (UniformSiblingZeroCount).** All siblings produced by a single allocator have the same zero count as its base address.

Let an allocator have base address `t₀` and produce siblings by repeated application of `inc(·, 0)`: define `tₙ₊₁ = inc(tₙ, 0)` for `n ≥ 0`. We prove `zeros(tₙ) = zeros(t₀)` for all `n ≥ 0` by induction on `n`.

*Base case.* `n = 0`: `zeros(t₀) = zeros(t₀)` holds trivially.

*Inductive step.* Assume `zeros(tₙ) = zeros(t₀)`. Every output of a conforming allocator satisfies T4 (T10a.4), so TA5-SigValid gives `sig(tₙ) = #tₙ`; T4's field-segment constraint then forces `(tₙ)_{#tₙ} ≠ 0`. The inference "non-zero ⇒ strictly positive on ℕ" is sourced per the convention TA5a's case-analysis preface, T4a's opening, T4c's exhaustion step, TA5-SigValid, and TA5 already follow: T0 fixes the carrier so `(tₙ)_{#tₙ} ∈ ℕ`, NAT-zero supplies the lower bound `0 ≤ (tₙ)_{#tₙ}`, and NAT-discrete (instantiated at `m = 0`) rules out the interval `0 ≤ (tₙ)_{#tₙ} < 1` under `(tₙ)_{#tₙ} ≠ 0`, so `(tₙ)_{#tₙ} ≥ 1 > 0` — the component at position `sig(tₙ) = #tₙ` is strictly positive, not merely non-zero. The step `tₙ₊₁ = inc(tₙ, 0)` modifies only position `sig(tₙ)`: TA5(b) (for `k = 0`) preserves every position `i ≠ sig(tₙ)` with `1 ≤ i ≤ #tₙ`, and TA5(c) sets `(tₙ₊₁)_{sig(tₙ)} = (tₙ)_{sig(tₙ)} + 1` while preserving length `#tₙ₊₁ = #tₙ`. That the post-increment value `(tₙ)_{sig(tₙ)} + 1` remains strictly positive is discharged per the same per-step convention TA5a's case `k = 0` applies to the structurally identical sibling step: NAT-closure's closure of ℕ under successor places `(tₙ)_{sig(tₙ)} + 1 ∈ ℕ`, NAT-zero supplies `0 ≤ (tₙ)_{sig(tₙ)}`, and NAT-addcompat's strict successor inequality `n < n + 1` for every `n ∈ ℕ` gives `(tₙ)_{sig(tₙ)} < (tₙ)_{sig(tₙ)} + 1`, so `(tₙ₊₁)_{sig(tₙ)} = (tₙ)_{sig(tₙ)} + 1 > (tₙ)_{sig(tₙ)} ≥ 0` — the modified position is non-zero after the `+1`, and hence strictly positive by the same NAT-zero / NAT-discrete inference applied above. The set `{i : 1 ≤ i ≤ #tₙ ∧ (tₙ)ᵢ = 0}` is therefore unchanged across the step: no index enters the zero set (the sole modified position was positive and stays positive) and none leaves it (every other position is held fixed by TA5(b)); length preservation (`#tₙ₊₁ = #tₙ`) rules out new indices that could carry a zero. Hence `zeros(tₙ₊₁) = zeros(tₙ) = zeros(t₀)` by the inductive hypothesis. ∎

*Formal Contract:*
- *Precondition:* Allocator with base address `t₀`, producing siblings by `inc(·, 0)`, conforming to T10a.
- *Postcondition:* `(A n ≥ 0 : zeros(tₙ) = zeros(t₀))` — every sibling has the same zero count as the base address.
- *Depends:* T10a (AllocatorDiscipline) — supplies the base address `t₀` and restricts sibling production to `tₙ₊₁ = inc(tₙ, 0)`, making the enumeration on which the induction runs well-defined. T10a.4 (T4PreservationUnderDiscipline) — every output of a conforming allocator satisfies T4, supplying both TA5-SigValid's precondition and T4's field-segment constraint at every step of the induction. T4 (HierarchicalParsing) — the field-segment constraint `(tₙ)_{#tₙ} ≠ 0` delivers the non-zeroness of the pre-increment component at position `sig(tₙ) = #tₙ`, which the chain T0 + NAT-zero + NAT-discrete then lifts to strict positivity `(tₙ)_{sig(tₙ)} > 0`, ensuring the advanced component stays strictly positive after `+1` and so does not enter or leave the zero set. T0 (CarrierSetDefinition) — fixes the carrier as ℕ so that every component `(tₙ)ᵢ ∈ ℕ`, supplying the underlying set on which the NAT-* axioms speak; T0 on its own does not license the step from non-zero to strictly positive — that step is discharged jointly with NAT-zero and NAT-discrete below. NAT-zero (NatZeroMinimum) — invoked twice: at the pre-increment step to supply the lower bound `0 ≤ (tₙ)_{#tₙ}` required to instantiate NAT-discrete at `m = 0` for the "non-zero ⇒ strictly positive on ℕ" inference, and at the post-increment step to supply `0 ≤ (tₙ)_{sig(tₙ)}` which, combined with NAT-addcompat's strict successor inequality, yields `(tₙ)_{sig(tₙ)} + 1 > 0`; without NAT-zero neither the NAT-discrete interval `0 ≤ (tₙ)_{#tₙ} < 1` could be formed nor the post-increment chained-inequality conclusion sourced. Matches the per-step citation convention TA5a, T4a, T4c, TA5-SigValid, and TA5 already follow for this same inference. NAT-discrete (NatDiscreteness) — invoked jointly with NAT-zero to convert the lower bound `0 ≤ (tₙ)_{#tₙ}` together with `(tₙ)_{#tₙ} ≠ 0` into strict positivity `(tₙ)_{#tₙ} ≥ 1`; without NAT-discrete the equivalence "non-zero ⇔ strictly positive" on ℕ would fail (the reals ℝ admit `0 < r < 1` and satisfy NAT-zero but not NAT-discrete), and the positivity attribution would revert to the T0-only form the ASN has moved away from. Matches the per-step citation convention established in T4a, T4c, TA5-SigValid, TA5, and TA5a for this same inference. NAT-closure (NatArithmeticClosure) — invoked at the `+1` step to confirm that `(tₙ)_{sig(tₙ)} + 1 ∈ ℕ` via closure of ℕ under successor; without NAT-closure the post-increment component would not be secured as a natural, and the subsequent strict-positivity claim would speak about a quantity outside T0's carrier. Matches TA5 and TA5a's citation of NAT-closure for the structurally identical sibling-construction step `(tₙ₊₁)_{sig(tₙ)} = (tₙ)_{sig(tₙ)} + 1`. NAT-addcompat (NatAdditionOrderCompatibility) — invoked at the `+1` step to supply the strict successor inequality `n < n + 1` for every `n ∈ ℕ` at `(tₙ)_{sig(tₙ)} < (tₙ)_{sig(tₙ)} + 1`; combined with NAT-zero's lower bound this yields `(tₙ)_{sig(tₙ)} + 1 > 0`, hence non-zero, so the "successor preserves positivity" inference is discharged per-step rather than left as an implicit appeal to ℕ-successor behaviour. Matches TA5 and TA5a's case `k = 0` citation of NAT-addcompat for the structurally identical sibling step. TA5 (HierarchicalIncrement) — TA5(b) for `k = 0` preserves every position other than `sig(tₙ)`, and TA5(c) sets `(tₙ₊₁)_{sig(tₙ)} = (tₙ)_{sig(tₙ)} + 1` with `#tₙ₊₁ = #tₙ`; together these restrict the step's change to a single position whose zero/non-zero status is preserved, and they rule out new indices that could carry a zero. TA5-SigValid (SigOnValidAddresses) — fixes `sig(tₙ) = #tₙ` for T4-valid `tₙ`, identifying the position modified by TA5(b)/TA5(c) as the terminal component — exactly the component T4's field-segment constraint forbids from being zero.
