**T10a.8 (UniformSiblingZeroCount).** All siblings produced by a single allocator have the same zero count as its base address.

Let an allocator have base address `t‚āÄ` and produce siblings by repeated application of `inc(¬∑, 0)`: define `t‚āô‚āä‚āĀ = inc(t‚āô, 0)` for `n ‚Č• 0`. We prove `zeros(t‚āô) = zeros(t‚āÄ)` for all `n ‚Č• 0` by induction on `n`.

*Base case.* `n = 0`: `zeros(t‚āÄ) = zeros(t‚āÄ)` holds trivially.

*Inductive step.* Assume `zeros(t‚āô) = zeros(t‚āÄ)`. Every output of a conforming allocator satisfies T4 (T10a.4), so TA5-SigValid gives `sig(t‚āô) = #t‚āô`; T4's field-segment constraint then forces `(t‚āô)_{#t‚āô} ‚Č† 0`, and since `(t‚āô)_{#t‚āô} ‚ąą ‚Ąē` by T0's carrier, `(t‚āô)_{sig(t‚āô)} > 0` ‚ÄĒ the component at position `sig(t‚āô)` is strictly positive. The step `t‚āô‚āä‚āĀ = inc(t‚āô, 0)` modifies only position `sig(t‚āô)`: TA5(b) (for `k = 0`) preserves every position `i ‚Č† sig(t‚āô)` with `1 ‚Č§ i ‚Č§ #t‚āô`, and TA5(c) sets `(t‚āô‚āä‚āĀ)_{sig(t‚āô)} = (t‚āô)_{sig(t‚āô)} + 1` while preserving length `#t‚āô‚āä‚āĀ = #t‚āô`. Since `(t‚āô)_{sig(t‚āô)} > 0`, we have `(t‚āô‚āä‚āĀ)_{sig(t‚āô)} = (t‚āô)_{sig(t‚āô)} + 1 > 0` as well ‚ÄĒ the modified position remains strictly positive. The set `{i : 1 ‚Č§ i ‚Č§ #t‚āô ‚ąß (t‚āô)ŠĶĘ = 0}` is therefore unchanged across the step: no index enters the zero set (the sole modified position was positive and stays positive) and none leaves it (every other position is held fixed by TA5(b)); length preservation (`#t‚āô‚āä‚āĀ = #t‚āô`) rules out new indices that could carry a zero. Hence `zeros(t‚āô‚āä‚āĀ) = zeros(t‚āô) = zeros(t‚āÄ)` by the inductive hypothesis. ‚ąé

*Formal Contract:*
- *Precondition:* Allocator with base address `t‚āÄ`, producing siblings by `inc(¬∑, 0)`, conforming to T10a.
- *Postcondition:* `(A n ‚Č• 0 : zeros(t‚āô) = zeros(t‚āÄ))` ‚ÄĒ every sibling has the same zero count as the base address.
- *Depends:* T10a (AllocatorDiscipline) ‚ÄĒ supplies the base address `t‚āÄ` and restricts sibling production to `t‚āô‚āä‚āĀ = inc(t‚āô, 0)`, making the enumeration on which the induction runs well-defined. T10a.4 (T4PreservationUnderDiscipline) ‚ÄĒ every output of a conforming allocator satisfies T4, supplying both TA5-SigValid's precondition and T4's field-segment constraint at every step of the induction. T4 (HierarchicalParsing) ‚ÄĒ the field-segment constraint `(t‚āô)_{#t‚āô} ‚Č† 0` combined with T0's carrier ‚Ąē delivers `(t‚āô)_{sig(t‚āô)} > 0`, ensuring the advanced component stays strictly positive after `+1` and so does not enter or leave the zero set. TA5 (HierarchicalIncrement) ‚ÄĒ TA5(b) for `k = 0` preserves every position other than `sig(t‚āô)`, and TA5(c) sets `(t‚āô‚āä‚āĀ)_{sig(t‚āô)} = (t‚āô)_{sig(t‚āô)} + 1` with `#t‚āô‚āä‚āĀ = #t‚āô`; together these restrict the step's change to a single position whose zero/non-zero status is preserved, and they rule out new indices that could carry a zero. TA5-SigValid (SigOnValidAddresses) ‚ÄĒ fixes `sig(t‚āô) = #t‚āô` for T4-valid `t‚āô`, identifying the position modified by TA5(b)/TA5(c) as the terminal component ‚ÄĒ exactly the component T4's field-segment constraint forbids from being zero.
