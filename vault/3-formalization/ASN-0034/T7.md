## Subspace disjointness

Within a document's element space, the first component after the third zero delimiter identifies the *subspace*: 1 for text, 2 for links. Nelson also mentions that the link subspace "could be further subdivided." The critical property is permanent separation:

**T7 (SubspaceDisjointness).** The subspace identifier (the first component of the element field) permanently separates the address space into disjoint regions. No T4-valid element-level tumbler in subspace `s₁` can equal or be confused with a T4-valid element-level tumbler in subspace `s₂ ≠ s₁`. The notation `t.E₁` is supplied by T4b, whose preconditions require `t` to satisfy the T4 constraints — at most three zero-valued components, no two zeros adjacent, `t₁ ≠ 0`, `t_{#t} ≠ 0`; under those constraints T4b's case analysis makes `t.E₁` defined exactly when `zeros(t) = 3`. Without them the equivalence fails: tumblers such as `[0, 1, 0, 2, 0, 3]` (violating `t₁ ≠ 0`) or `[1, 0, 0, 2, 0, 3]` (violating the no-adjacent-zeros clause) have `zeros(·) = 3` yet lie outside T4b's domain, so `·.E₁` has no licensed meaning there. We therefore quantify over T4-valid element-level tumblers, carrying the T4 constraints explicitly into the restriction on the bound variables:

  `(A a, b ∈ T : a, b satisfy T4 ∧ zeros(a) = zeros(b) = 3 : a.E₁ ≠ b.E₁ ⟹ a ≠ b)`

*Proof.* We are given two T4-valid tumblers `a, b ∈ T` with `zeros(a) = zeros(b) = 3` — the precise preconditions under which T4b's case analysis for `zeros(·) = 3` makes `E(a)` and `E(b)` non-empty and `a.E₁`, `b.E₁` denote well-defined components — whose first element-field components differ: `a.E₁ ≠ b.E₁`. We must show `a ≠ b`.

The T4-validity of `a` and `b` ensures, by T4's definitional role-assignment, that every zero in either tumbler is unambiguously a field separator; the field-segment constraint (no two zeros adjacent, `a₁ ≠ 0`, `a_{#a} ≠ 0`, and likewise for `b`) forces each of the four field segments — node, user, document, element — to be non-empty. By T0, every component lies in ℕ, so every non-separator component is strictly positive.

We establish notation. Write the field lengths of `a` as `(α, β, γ, δ)` — the node field has `α` components, the user field `β`, the document field `γ`, and the element field `δ`. The three separators sit at positions `α + 1`, `α + β + 2`, and `α + β + γ + 3` in the raw component sequence. The first element-field component `E₁` therefore occupies position `pₐ = α + β + γ + 4`. Analogously, write the field lengths of `b` as `(α', β', γ', δ')`, so that `b.E₁` sits at position `p_b = α' + β' + γ' + 4`.

We proceed by case analysis on whether `pₐ = p_b`.

*Case 1* (`pₐ = p_b`). Both tumblers have their first element-field component at the same position `p = pₐ = p_b`. By hypothesis, `a[p] = a.E₁ ≠ b.E₁ = b[p]`. The tumblers differ at position `p`. By T3 (canonical representation — two tumblers are equal if and only if they have the same length and agree at every position), `a ≠ b`.

*Case 2* (`pₐ ≠ p_b`). The first element-field components sit at different positions, so the field-length triples `(α, β, γ)` and `(α', β', γ')` are not all equal. We consider two sub-cases.

*Sub-case 2a* (`#a ≠ #b`). The tumblers have different lengths. By T3 (distinct lengths entail distinct tumblers), `a ≠ b`.

*Sub-case 2b* (`#a = #b`). The tumblers have equal length but their separator positions differ. The separator positions of `a` are `{α + 1, α + β + 2, α + β + γ + 3}` and those of `b` are `{α' + 1, α' + β' + 2, α' + β' + γ' + 3}`. We show these sets cannot coincide. Suppose for contradiction they are identical. Matching the three elements pairwise in ascending order extracts `α = α'`, `β = β'`, and `γ = γ'` through three ℕ-cancellation steps, each discharged from NAT-cancel with the intermediate regroupings discharged from NAT-addassoc — the same per-step citation convention that GlobalUniqueness applies to the structurally identical `(#p₁ + 1) + 1 = #p₂ + 1 ⟹ #p₂ = #p₁ + 1` step and that TA-LC and TA-MTO apply to cancellation at the action point.

Matching the smallest elements: `α + 1 = α' + 1`. NAT-cancel's right cancellation (`n + m = p + m ⟹ n = p`, instantiated at `m = 1`, `n = α`, `p = α'`) yields `α = α'`.

Matching the middle elements and substituting the previous result: `α + β + 2 = α + β' + 2`. NAT-addassoc's associativity (`(m + n) + p = m + (n + p)`, instantiated at `m = α`, `n = β`, `p = 2`, and again with `β` replaced by `β'`) licenses the regroupings `α + β + 2 = (α + β) + 2` and `α + β' + 2 = (α + β') + 2`, so the equality reads `(α + β) + 2 = (α + β') + 2`. NAT-cancel's right cancellation (instantiated at `m = 2`, `n = α + β`, `p = α + β'`) yields `α + β = α + β'`, and NAT-cancel's left cancellation (`m + n = m + p ⟹ n = p`, instantiated at `m = α`, `n = β`, `p = β'`) then yields `β = β'`.

Matching the largest elements and substituting the previous two results: `α + β + γ + 3 = α + β + γ' + 3`. NAT-addassoc's associativity (instantiated at `m = α + β`, `n = γ`, `p = 3`, and again with `γ` replaced by `γ'`) licenses the regroupings `α + β + γ + 3 = (α + β + γ) + 3` and `α + β + γ' + 3 = (α + β + γ') + 3`, so the equality reads `(α + β + γ) + 3 = (α + β + γ') + 3`. NAT-cancel's right cancellation (instantiated at `m = 3`, `n = α + β + γ`, `p = α + β + γ'`) yields `α + β + γ = α + β + γ'`, and NAT-cancel's left cancellation (instantiated at `m = α + β`, `n = γ`, `p = γ'`) then yields `γ = γ'`.

But then `pₐ = α + β + γ + 4 = α' + β' + γ' + 4 = p_b`, contradicting the case hypothesis `pₐ ≠ p_b`.

Therefore the separator-position sets differ: there exists a position `j` that is a separator in one tumbler but not the other. At position `j`, one tumbler has value `0` (it is a separator). The other tumbler has a field component at position `j`, which is non-zero by T4's role-assignment and hence strictly positive since all components lie in ℕ (T0). Hence `a[j] ≠ b[j]`, and by T3, `a ≠ b`.

All cases yield `a ≠ b`. ∎

*Formal Contract:*
- *Preconditions:* `a, b ∈ T` satisfy the T4 constraints — at most three zero-valued components, no two zeros adjacent, `a₁ ≠ 0`, `a_{#a} ≠ 0` (and likewise for `b`) — and have `zeros(a) = zeros(b) = 3`. Under these constraints both are element-level addresses with well-formed field structure, and by T4b (UniqueParse) the component-access notation `a.E₁`, `b.E₁` denotes well-defined components.
- *Depends:* T0 (CarrierSetDefinition) — invoked at the opening structural setup ("By T0, every component lies in ℕ, so every non-separator component is strictly positive") and again in Sub-case 2b's separator-vs-component contradiction ("non-zero by T4's role-assignment and hence strictly positive since all components lie in ℕ (T0)"); T0 supplies the strict positivity that lets the proof contrast `0` (separator) with non-zero values (field components) at a position where the two tumblers' separator sets disagree. T3 (CanonicalRepresentation) — invoked at three sites: in Case 1 ("By T3 (canonical representation — two tumblers are equal if and only if they have the same length and agree at every position), `a ≠ b`"), in Sub-case 2a ("By T3 (distinct lengths entail distinct tumblers), `a ≠ b`"), and in Sub-case 2b ("by T3, `a ≠ b`"); each invocation converts a positional or length disagreement into the tumbler inequality that the postcondition asserts. T4 (HierarchicalParsing) — invoked in the statement and precondition to constrain the bound variables ("we quantify over T4-valid element-level tumblers" with the explicit side-condition "`a, b` satisfy T4"), in the proof's opening ("We are given two T4-valid tumblers `a, b ∈ T` with `zeros(a) = zeros(b) = 3`"), at the role-assignment claim ("The T4-validity of `a` and `b` ensures, by T4's definitional role-assignment, that every zero in either tumbler is unambiguously a field separator"), at the field-segment-constraint claim used to fix the four field lengths as non-empty ("the field-segment constraint (no two zeros adjacent, `a₁ ≠ 0`, `a_{#a} ≠ 0`, and likewise for `b`) forces each of the four field segments — node, user, document, element — to be non-empty"), and in Sub-case 2b's separator-vs-component contradiction ("non-zero by T4's role-assignment"); T4 supplies both the well-formedness constraints that license T4b's case analysis on `a` and `b` and the separator semantics on which the field-position arithmetic depends. T4b (UniqueParse) — invoked in the statement's quantifier-validity remark ("The notation `t.E₁` is supplied by T4b, whose preconditions require `t` to satisfy the T4 constraints ... under those constraints T4b's case analysis makes `t.E₁` defined exactly when `zeros(t) = 3`") and in the proof's interpretation step ("T4b's case analysis for `zeros(·) = 3` makes `E(a)` and `E(b)` non-empty and `a.E₁`, `b.E₁` denote well-defined components"); T4b licenses the well-definedness of the `t.E₁` notation that the statement and proof both use, but only on T4-valid inputs, which is why T4-validity is carried through the quantifier and preconditions. NAT-cancel (NatAdditionCancellation) — invoked in Sub-case 2b's separator-position-set contradiction to discharge the three pairwise-matching "so" extractions as explicit ℕ-axiom steps rather than background arithmetic, matching the per-step citation convention that GlobalUniqueness applies to the structurally identical `(#p₁ + 1) + 1 = #p₂ + 1 ⟹ #p₂ = #p₁ + 1` step and that TA-LC and TA-MTO apply to cancellation at the action point. Three sites: (a) right cancellation (`n + m = p + m ⟹ n = p`, instantiated at `m = 1`, `n = α`, `p = α'`) yields `α = α'` from `α + 1 = α' + 1`; (b) right cancellation at `m = 2`, `n = α + β`, `p = α + β'` yields `α + β = α + β'` from `(α + β) + 2 = (α + β') + 2`, then left cancellation (`m + n = m + p ⟹ n = p`, instantiated at `m = α`, `n = β`, `p = β'`) yields `β = β'`; (c) right cancellation at `m = 3`, `n = α + β + γ`, `p = α + β + γ'` yields `α + β + γ = α + β + γ'` from `(α + β + γ) + 3 = (α + β + γ') + 3`, then left cancellation (instantiated at `m = α + β`, `n = γ`, `p = γ'`) yields `γ = γ'`; without NAT-cancel these appeals would be left to background ℕ arithmetic, contrary to T0's convention that each proof cites only the ℕ facts it actually uses. NAT-addassoc (NatAdditionAssociative) — invoked in Sub-case 2b's second and third cancellation steps to supply the regroupings under which NAT-cancel's right cancellation applies at `m = 2` and `m = 3`. Two sites: associativity (`(m + n) + p = m + (n + p)`, instantiated at `m = α`, `n = β`, `p = 2`, and again with `β` replaced by `β'`) licenses `α + β + 2 = (α + β) + 2` and `α + β' + 2 = (α + β') + 2` so the middle-element equality reads `(α + β) + 2 = (α + β') + 2`; associativity (instantiated at `m = α + β`, `n = γ`, `p = 3`, and again with `γ` replaced by `γ'`) licenses `α + β + γ + 3 = (α + β + γ) + 3` and `α + β + γ' + 3 = (α + β + γ') + 3` so the largest-element equality reads `(α + β + γ) + 3 = (α + β + γ') + 3`; without NAT-addassoc these re-bracketings would be left to background ℕ arithmetic, contrary to T0's per-step citation convention (the same convention TS3 and TA-assoc follow when reconciling `(vₘ + n₁) + n₂` with `vₘ + (n₁ + n₂)` and `(aₖ + bₖ) + cₖ` with `aₖ + (bₖ + cₖ)`).
- *Postconditions:* `a.E₁ ≠ b.E₁ ⟹ a ≠ b`.

We state T7 explicitly because it is load-bearing for the guarantee that operations within one content type do not interfere with another. T7 is the structural basis — arithmetic within subspace 1 cannot produce addresses in subspace 2, because the subspace identifier is part of the address, not metadata.

The ordering T1 places all text addresses (subspace 1) before all link addresses (subspace 2) within the same document, because `1 < 2` at the subspace position. This is a consequence, not an assumption — it falls out of the lexicographic order.
