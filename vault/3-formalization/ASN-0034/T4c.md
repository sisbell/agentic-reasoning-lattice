**T4c (LevelDetermination).** Define `zeros(t) = #{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}`, the count of zero-valued components in a tumbler `t`. Under the constraints of T4, the function `zeros` determines the hierarchical level of an address tumbler, and the mapping from `{0, 1, 2, 3}` to the four hierarchical levels is a bijection.

By T4, valid address tumblers have at most three zero-valued components, so `zeros(t) ∈ {0, 1, 2, 3}`. By T4b (UniqueParse), every zero in `t` is a field separator and every separator is a zero — T4's definitional role-assignment fixes zeros as the separator marker, and T0's carrier ℕ leaves no other value available to play that role. Therefore `zeros(t)` counts exactly the number of field separators, and the number of fields present equals `zeros(t) + 1`, since `k` separators delimit `k + 1` fields.

The mapping from zero count to hierarchical level is defined by the number of fields:

  - `zeros(t) = 0` → 1 field (node only) → node address,
  - `zeros(t) = 1` → 2 fields (node, user) → user address,
  - `zeros(t) = 2` → 3 fields (node, user, document) → document address,
  - `zeros(t) = 3` → 4 fields (node, user, document, element) → element address.

*Injectivity.* Distinct zero counts produce distinct field counts — `zeros(t) + 1` is injective on `{0, 1, 2, 3}` — and the field-count-to-level mapping is itself injective, since each level is defined by a specific number of fields. Therefore `zeros(a) ≠ zeros(b)` implies `a` and `b` belong to different hierarchical levels.

*Surjectivity.* Each of the four levels is realized: `zeros(t) = 0, 1, 2, 3` are all values permitted by T4, and each corresponds to exactly one level. The mapping is therefore bijective on `{0, 1, 2, 3}`.

We note the role of T4's definitional choice in this result. T4 assigns zero the role of field separator and nothing else; T0's carrier ℕ ensures that every non-zero component is strictly positive, so no value other than zero can play the separator role and no separator can be anything but zero. A tumbler like `[1, 0, 0, 3]` is excluded not by a positivity clause — which would be vacuous over ℕ — but by the field-segment constraint's no-adjacent-zeros clause, which forbids the two zeros at positions 2 and 3; the zeros are unambiguously separators in every admissible tumbler, so the parse is unique. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints (at most three zero-valued components, field-segment constraint — no two zeros adjacent, `t₁ ≠ 0`, `t_{#t} ≠ 0`). Positivity of non-zero components is supplied by T0's carrier ℕ. `t` satisfies T4b (UniqueParse): every zero in `t` is a field separator and every separator is a zero, so the separator positions are exactly the zero-valued positions.
