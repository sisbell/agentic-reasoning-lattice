**T4c (LevelDetermination).** Define `zeros(t) = #{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}`, the count of zero-valued components in a tumbler `t`. On the T4-valid subset of `T` (tumblers `t ∈ T` satisfying `zeros(t) ≤ 3`, no two zeros adjacent, `t₁ ≠ 0`, `t_{#t} ≠ 0`), the function `zeros` determines the hierarchical level of an address tumbler, and the mapping from `{0, 1, 2, 3}` to the four hierarchical levels is a bijection. Outside this subdomain — tumblers with `zeros(t) ≥ 4` or with `zeros(t) ≤ 3` but violating the field-segment constraint — this corollary assigns no hierarchical level, so every citation of a level from `zeros(t)` must carry T4-validity as a precondition.

By T4, valid address tumblers have at most three zero-valued components, so `zeros(t) ∈ {0, 1, 2, 3}`. By T4's definitional role-assignment, a position `i` of `t` is a field separator iff `tᵢ = 0` — T4's axiom carries both directions of this biconditional (`tᵢ = 0 ⇒ i` is a separator, and `i` is a separator `⇒ tᵢ = 0`) as a clause of the axiom itself, and T0's carrier ℕ leaves no other value available to play the separator role. Therefore `zeros(t)` counts exactly the number of field separators, and the number of fields present equals `zeros(t) + 1`, since `k` separators delimit `k + 1` fields.

The mapping from zero count to hierarchical level is defined by the number of fields:

  - `zeros(t) = 0` → 1 field (node only) → node address,
  - `zeros(t) = 1` → 2 fields (node, user) → user address,
  - `zeros(t) = 2` → 3 fields (node, user, document) → document address,
  - `zeros(t) = 3` → 4 fields (node, user, document, element) → element address.

*Injectivity.* Distinct zero counts produce distinct field counts — `zeros(t) + 1` is injective on `{0, 1, 2, 3}` — and the field-count-to-level mapping is itself injective, since each level is defined by a specific number of fields. Therefore `zeros(a) ≠ zeros(b)` implies `a` and `b` belong to different hierarchical levels.

*Surjectivity.* Each of the four levels is realized: `zeros(t) = 0, 1, 2, 3` are all values permitted by T4, and each corresponds to exactly one level. The mapping is therefore bijective on `{0, 1, 2, 3}`.

We note the role of T4's definitional choice in this result. T4 assigns zero the role of field separator and nothing else; T0's carrier ℕ ensures that every non-zero component is strictly positive, so no value other than zero can play the separator role and no separator can be anything but zero. A tumbler like `[1, 0, 0, 3]` is excluded not by a positivity clause — which would be vacuous over ℕ — but by the field-segment constraint's no-adjacent-zeros clause, which forbids the two zeros at positions 2 and 3; the zeros are unambiguously separators in every admissible tumbler, so the parse is unique. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints (at most three zero-valued components, field-segment constraint — no two zeros adjacent, `t₁ ≠ 0`, `t_{#t} ≠ 0`). Positivity of non-zero components is supplied by T0's carrier ℕ. T4's definitional role-assignment supplies the biconditional `tᵢ = 0 ⇔ i` is a field separator, so the separator positions are exactly the zero-valued positions; this biconditional is part of T4's axiom and is therefore inherited directly through the T4 dependency, with no additional precondition required.
- *Depends:* T0 (CarrierSetDefinition) — invoked at the role-assignment step "T0's carrier ℕ ensures that every non-zero component is strictly positive, so no value other than zero can play the separator role and no separator can be anything but zero"; without T0 the exclusivity of zero as separator marker (used to license the zeros-counts-separators step) would have no source, and the side remark on `[1, 0, 0, 3]` ("excluded not by a positivity clause — which would be vacuous over ℕ") would similarly be unsourced. T4 (HierarchicalParsing) — invoked at the opening to fix `zeros(t) ∈ {0, 1, 2, 3}` ("By T4, valid address tumblers have at most three zero-valued components, so `zeros(t) ∈ {0, 1, 2, 3}`"), at the role-assignment biconditional "a position `i` of `t` is a field separator iff `tᵢ = 0`" — T4's axiom carries both directions of this biconditional as part of its Definitional role-assignment clause, supplying the equality between `zeros(t)` and the separator count that is the central step of the proof — and at the no-adjacent-zeros clause used to exclude `[1, 0, 0, 3]`; these jointly establish the four-value range, the separator semantics, and the parsability constraint on which the bijection rests. No precondition on T4b (UniqueParse) is required: the zero↔separator biconditional and the bound `zeros(t) ≤ 3` are both clauses of T4's own axiom, so the equality between `zeros(t)` and the separator count follows from T4 alone, without passing through T4b's `fields(t)` decomposition.
- *Postconditions:* The claim is universally quantified over the T4-valid subset of `T` — tumblers `t ∈ T` satisfying T4's positional conditions (`zeros(t) ≤ 3`, no two zeros adjacent, `t₁ ≠ 0`, `t_{#t} ≠ 0`) — and no value is assigned to the level outside this subdomain. On that subdomain, `zeros(t)` counts exactly the number of field separators in `t`, and the number of fields present equals `zeros(t) + 1`. The level-assignment biconditionals hold as `(A t ∈ T : t is T4-valid :: zeros(t) = 0 ↔ t is a node address ∧ zeros(t) = 1 ↔ t is a user address ∧ zeros(t) = 2 ↔ t is a document address ∧ zeros(t) = 3 ↔ t is an element address)`, so the mapping `zeros(t) → hierarchical level` restricted to the T4-valid subdomain is a bijection `{0, 1, 2, 3} → {node, user, document, element}`: distinct zero counts among T4-valid tumblers imply distinct hierarchical levels (injectivity), and every level in {node, user, document, element} is realized by exactly one zero count (surjectivity). On tumblers `t ∈ T` that are not T4-valid — those with `zeros(t) ≥ 4` (outside the four-value enumeration) or with `zeros(t) ≤ 3` but violating the field-segment constraint (e.g., `[1, 0, 0, 2]` with adjacent zeros, or `[0, 1, 0, 2]` with a leading zero) — this corollary assigns no hierarchical level, so consumers must discharge T4-validity before reading a level off `zeros(t)`; citations of T4c at an arbitrary `t ∈ T` are licensed only after T4-validity has been established.
