**TA6 (ZeroTumblers).** No zero tumbler is a valid address — no all-zero tumbler designates content. Every zero tumbler is less than every positive tumbler under T1.

  `(A t ∈ T : Zero(t) ⟹ t is not a valid address)`

  `(A s, t ∈ T : Zero(s) ∧ (E j : 1 ≤ j ≤ #t : tⱼ > 0) ⟹ s < t)`

*Proof (from T0, T4, TA-Pos, TA-PosDom, NAT-zero).* We prove each conjunct separately. Recall that T4's field-segment constraint includes `t₁ ≠ 0`, and since `t₁ ∈ ℕ` by T0, this gives `t₁ > 0` — the first component belongs to the node field. Recall also TA-PosDom's postcondition `(A t ∈ T, z ∈ T : Pos(t) ∧ Zero(z) :: z < t)` — every positive tumbler is T1-greater than every zero tumbler of any length — established in the TA-PosDom section by a case analysis over T1 cases (i) and (ii), routed through ActionPoint. We designate that postcondition as the *canonical* statement of the zero-tumbler-below-positive-tumbler relation and cite it in Conjunct 2 rather than reproving it, so that any future tightening of the statement happens at one site. TA-Pos (now the definition-only split of the former combined property) supplies both paired predicate symbols: `Pos(t)` via its existential `(E i : 1 ≤ i ≤ #t : tᵢ ≠ 0)`, and `Zero(t)` via its universal `(A i : 1 ≤ i ≤ #t : tᵢ = 0)` — the latter binding the zero-tumbler guard as a single symbol that TA6's conjuncts bind in their hypotheses rather than spelling the universal out inline, matching TA-PosDom's uniform binding of the same predicate.

**Conjunct 1** (invalidity): Let `t ∈ T` with `Zero(t)`; unpacking `Zero(t)` via its TA-Pos defining clause gives `tᵢ = 0` for all `1 ≤ i ≤ #t`. Since `t ∈ T`, T0 guarantees `#t ≥ 1`, so `t₁` is defined and `t₁ = 0` (the unpacked clause at `i = 1`). This violates T4's requirement `t₁ ≠ 0`, so `t` is not a valid address.

**Conjunct 2** (ordering): Let `s ∈ T` with `Zero(s)` — unpacking via its TA-Pos defining clause gives `(A i : 1 ≤ i ≤ #s : sᵢ = 0)` pointwise — and let `t ∈ T` satisfy the conjunct's other hypothesis, `(E j : 1 ≤ j ≤ #t : tⱼ > 0)`. By T0, every tumbler component lies in ℕ, and NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` therefore furnishes the lower bound `0 ≤ tⱼ` on each component; with that lower bound, the conditions `tⱼ > 0` and `tⱼ ≠ 0` coincide on ℕ-valued components — the forward direction `tⱼ > 0 ⟹ tⱼ ≠ 0` excludes equality because `0 < tⱼ` is strict, and the converse `tⱼ ≠ 0 ⟹ tⱼ > 0` combines `0 ≤ tⱼ` with `tⱼ ≠ 0` to yield `0 < tⱼ`. The hypothesis is therefore literally `(E j : 1 ≤ j ≤ #t : tⱼ ≠ 0)`, which is the definition of `Pos(t)` recorded in TA-Pos. We may therefore apply TA-PosDom's postcondition, instantiated with the current `t` (playing the role of TA-PosDom's `t`, since `Pos(t)` holds) and with `z := s` (playing the role of TA-PosDom's `z`, since `Zero(s)` is exactly the postcondition's guard `Zero(z)` at the substitution `z := s`), to obtain `s < t` — which is the conclusion of the conjunct. ∎

*Formal Contract:*
- *Depends:* T0 (CarrierSetDefinition) — Conjunct 1 uses T0's guarantee `#t ≥ 1` for all `t ∈ T` to establish that `t₁` is defined, and Conjunct 2 uses T0's constraint `aᵢ ∈ ℕ` to place every tumbler component in ℕ, which is the precondition under which NAT-zero's lower bound supplies the equivalence `tⱼ > 0 ⟺ tⱼ ≠ 0` required to bridge the hypothesis form to the TA-Pos definition of `Pos(t)`; T0 alone is insufficient, since it supplies no ordering fact about `0`. T1 (LexicographicOrder) — invoked transitively through TA-PosDom's postcondition for Conjunct 2; TA-PosDom's proof uses T1 case (i) at the action point of `t` and T1 case (ii) when `s` is a proper prefix of `t`, and Conjunct 2 inherits this case analysis without repeating it. T3 (CanonicalRepresentation) — invoked transitively through TA-PosDom's preamble for Conjunct 2; TA-PosDom's preamble notes that under T3 the tumblers `[0]`, `[0, 0]`, `[0, 0, 0]`, etc., are distinct elements of T (their lengths differ), which is motivational commentary for the postcondition's universal quantification over `z ∈ T` with `Zero(z)` of arbitrary length — the very quantifier that Conjunct 2 instantiates with its arbitrary-length zero tumbler `s`, and Conjunct 2 inherits the motivational framing without repeating it. T4 (HierarchicalParsing) — Conjunct 1 relies on T4's boundary clause `t₁ ≠ 0` to reject zero tumblers as valid addresses. TA-Pos (PositiveTumbler) — Conjunct 2 uses both paired predicate symbols TA-Pos introduces: `Pos(t)` (its existential `(E i : 1 ≤ i ≤ #t : tᵢ ≠ 0)`) and `Zero(t)` (its universal `(A i : 1 ≤ i ≤ #t : tᵢ = 0)`). The `Pos(t)` definition is what the bridge from the conjunct's hypothesis form `(E j : 1 ≤ j ≤ #t : tⱼ > 0)` lands in (via NAT-zero's `tⱼ > 0 ⟺ tⱼ ≠ 0` equivalence on ℕ-valued components), and the `Zero(t)` definition is what the conjunct's first hypothesis `Zero(s)` (and Conjunct 1's `Zero(t)`) binds as a single symbol — unpacked pointwise via its TA-Pos defining clause at the indices where components are needed, rather than spelling the universal out inline. The paired-predicate pattern is what lets Conjunct 2 cite TA-PosDom's postcondition directly without re-wrapping its guard form. TA-Pos is now the definition-only split; the ordering theorem TA-PosDom cites it for both the `Pos` and `Zero` predicate symbols rather than carrying the theorem inside TA-Pos's contract. TA-PosDom (PositiveDominatesZero) — Conjunct 2 cites TA-PosDom's postcondition `(A t ∈ T, z ∈ T : Pos(t) ∧ Zero(z) :: z < t)` as the canonical statement of the zero-tumbler-below-positive-tumbler relation; the conjunct is now a restatement of that postcondition, with operand names swapped (TA6's `s` plays the role of TA-PosDom's `z` — both binding TA-Pos's `Zero` predicate — and TA6's `t` coincides with TA-PosDom's `t`), rather than an independent reproof, which prevents drift between the two statements. TA-PosDom is the theorem-bearing property split off from the former combined TA-Pos, so that the Depends edges ActionPoint → TA-Pos (definition consumption: ActionPoint depends on TA-Pos for the `Pos(w)` predicate) and TA-PosDom → ActionPoint (theorem consumption: TA-PosDom depends on ActionPoint for the `actionPoint(t)` witness) are acyclic, under the "X → Y means X depends on Y" convention that matches the `depends` lists in each property's `.yaml`; Conjunct 2 cites both halves of the split — TA-Pos for the `Pos(t)` predicate and TA-PosDom for the postcondition that consumes it. NAT-zero (NatZeroMinimum) — Conjunct 2 invokes NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` to license the equivalence `tⱼ > 0 ⟺ tⱼ ≠ 0` on ℕ-valued components required to bridge the TA6 hypothesis form to the TA-Pos `Pos(t)` definition: the lower bound `0 ≤ tⱼ` combines with `tⱼ ≠ 0` to yield `0 < tⱼ` (the converse direction), while strictness of `<` supplies the forward direction. Without NAT-zero, T0's placement of components in ℕ would not rule out `tⱼ < 0`, and the equivalence between `> 0` and `≠ 0` would fail; this matches the citation discipline of the sibling property TA-PosDom, whose Case `#z ≥ k` invokes NAT-zero transitively through ActionPoint for an analogous step. NAT-order (NatStrictTotalOrder) — invoked at two distinct sites for two distinct roles in Conjunct 2. *Direct use at Conjunct 2's `≠ 0 ⟺ > 0` bridge (component pair `(0, tⱼ)`)*: before TA-PosDom is invoked, Conjunct 2 bridges the hypothesis form `(E j : 1 ≤ j ≤ #t : tⱼ > 0)` to the TA-Pos `Pos(t)` definition `(E j : 1 ≤ j ≤ #t : tⱼ ≠ 0)` pointwise at the named `j` by combining two NAT-order clauses with NAT-zero's lower bound `0 ≤ tⱼ` (instantiated at `n = tⱼ`, with `tⱼ ∈ ℕ` supplied by T0's component-typing). (a) The converse direction `tⱼ ≠ 0 ⟹ tⱼ > 0` uses NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` at `m = 0, n = tⱼ` to unfold `0 ≤ tⱼ` into `0 < tⱼ ∨ 0 = tⱼ`, whose equality disjunct the hypothesis `tⱼ ≠ 0` excludes to leave `0 < tⱼ`. (b) The forward direction `tⱼ > 0 ⟹ tⱼ ≠ 0` uses NAT-order's irreflexivity `(A n ∈ ℕ :: ¬(n < n))` at `n = 0` — the strictness of `<` the prose names verbatim — since if `tⱼ = 0` then `0 < tⱼ` would become `0 < 0`, contradicting irreflexivity, whence the strict comparison `0 < tⱼ` excludes `tⱼ = 0`. This NAT-zero + NAT-order discharge matches the per-step convention TumblerSub's precondition-consequence case (ii), TA2's sub-case (ii), and ActionPoint's third postcondition all apply for the structurally identical `≠ 0 ⟹ > 0` inference; the bidirectional form Conjunct 2 consumes additionally requires the irreflexivity citation for the forward direction, mirroring the parallel NAT-zero entry in the same Depends list which already names "strictness of `<`" as the source of the forward-direction step. Without NAT-order's defining clause of `≤` from `<`, NAT-zero's bare lower bound `0 ≤ tⱼ` would not refine to `0 < tⱼ ∨ 0 = tⱼ` for the disjunct-exclusion argument of the converse direction; without NAT-order's irreflexivity, the strict comparison `0 < tⱼ` would not exclude the possibility `tⱼ = 0` for the forward direction. *Transitive use through TA-PosDom's postcondition*: after the bridge establishes `Pos(t)` and TA-PosDom's postcondition is instantiated with the current `t` and `z := s`, TA-PosDom's Case `#z ≥ k` uses NAT-order's definitional unfold `m ≤ n ⟺ m < n ∨ m = n` at `m = 1, n = tₖ` to refine `1 ≤ tₖ` (supplied by ActionPoint's postcondition (iii)) into the disjunction `1 < tₖ ∨ 1 = tₖ` composed with `0 < 1` to yield `0 < tₖ`, and TA-PosDom's Case `#z < k` uses NAT-order's trichotomy, irreflexivity, and `≤`-transitivity in the contrapositive argument that derives `#z + 1 ≤ #t`; Conjunct 2 inherits both steps without repeating them. The two roles — the direct use at the bridge step and the transitive use through TA-PosDom — are enumerated together under this single NAT-order citation, matching the per-site accounting convention this ASN applies elsewhere (TumblerSub enumerates five NAT-order sites, TA2 three). NAT-discrete (NatDiscreteness) — invoked transitively through TA-PosDom's postcondition for Conjunct 2; TA-PosDom's Case `#z < k` uses NAT-discrete in its contrapositive form `m < n ⟹ m + 1 ≤ n` (instantiated at `m = #z, n = #t`) to produce the T1 case (ii) witness `#z + 1 ≤ #t`, and Conjunct 2 inherits this derivation without repeating it. NAT-wellorder (NatWellOrdering) — invoked transitively through TA-PosDom's postcondition for Conjunct 2; TA-PosDom delegates the least-index construction to ActionPoint, which applies NAT-wellorder's well-ordering of ℕ to the nonempty subset `{i : 1 ≤ i ≤ #t ∧ tᵢ ≠ 0} ⊆ ℕ` to extract the least index `k` with `tₖ ≠ 0` (the pivot around which Cases `#z ≥ k` and `#z < k` are organised), and Conjunct 2 inherits this least-index construction without repeating it.
- *Postconditions:* (a) `(A t ∈ T : Zero(t) ⟹ t is not a valid address)`. (b) `(A s, t ∈ T : Zero(s) ∧ (E j : 1 ≤ j ≤ #t : tⱼ > 0) ⟹ s < t)` — a restatement of TA-PosDom's postcondition with operand names swapped; TA-PosDom is the canonical site, and any tightening of this statement must be reflected there.

Zero tumblers serve as *sentinels*: they mark uninitialized values, denote "unbounded" when used as span endpoints, and act as lower bounds.
