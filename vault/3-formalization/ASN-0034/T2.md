**T2 (IntrinsicComparison).** The order relation T1 is computable from the two tumblers alone, without consulting any external data structure. The comparison examines at most `min(#a, #b)` component pairs.

*Proof.* We must establish two claims: (1) the comparison requires no data beyond the components and lengths of the two tumblers, and (2) the number of component pairs examined is at most `min(#a, #b)`.

Let `a = a₁. ... .aₘ` and `b = b₁. ... .bₙ`. The definition of `<` in T1 asks for the existence of a witness position `k ≥ 1` satisfying two conditions: agreement at all positions before `k`, and either a component divergence at `k` or prefix exhaustion at `k`. We trace exactly what data this examination consults and how many comparisons it performs.

The comparison proceeds by scanning positions `i = 1, 2, ...` and comparing the pair `(aᵢ, bᵢ)` at each position. Two outcomes are possible.

*Case 1: divergence at some position `k ≤ min(m, n)`.* The scan finds `aₖ ≠ bₖ` after verifying `aᵢ = bᵢ` for all `1 ≤ i < k`. Exactly `k` component pairs are examined. Since `k ≤ min(m, n)`, the bound holds. NAT-order's trichotomy at the component pair `(aₖ, bₖ)` resolves `aₖ ≠ bₖ` into the two-way split `aₖ < bₖ ∨ bₖ < aₖ` — its remaining disjunct `aₖ = bₖ` is excluded by the case hypothesis, and the exactly-one clause guarantees one of the strict comparisons holds. Whichever holds decides the ordering via T1 case (i): `aₖ < bₖ` yields `a < b`, and `bₖ < aₖ` yields `b < a`. Both outcomes are comparisons of two natural numbers intrinsic to the tumblers.

*Case 2: no divergence within the shared range.* The scan exhausts all `min(m, n)` positions without finding `aᵢ ≠ bᵢ`. Exactly `min(m, n)` component pairs are examined. The result is then determined by NAT-order's trichotomy at the length pair `(m, n)`, which partitions the outcome exhaustively into three branches: if `m < n`, then `a` is a proper prefix of `b` and T1 case (ii) gives `a < b`; if `n < m`, the symmetric argument gives `b < a`; if `m = n`, then T3 gives `a = b`. The exhaustiveness of this three-way partition — the "exactly one" clause of NAT-order's trichotomy — is what underwrites postcondition (a) for this branch: some ordering among `a` and `b` is always decided, because no fourth outcome is possible for the length pair `(m, n)`.

In both cases, the number of component pairs examined is at most `min(m, n) = min(#a, #b)`, establishing claim (2).

For claim (1), we observe that every value consulted — the components `aᵢ` and `bᵢ` at each position, and the lengths `m` and `n` — belongs to the two tumblers themselves. The definition of T1 is expressed entirely in terms of these values. No tree structure, no index, no auxiliary mapping, and no external state participates in the decision. ∎

The importance of T2 is operational: span containment tests, link search, and index traversal all reduce to tumbler comparison. If comparison required a lookup, these operations would depend on auxiliary state, and the system's decentralization guarantee would collapse — one could not determine whether an address falls within a span without access to the index that manages that span.

Gregory's implementation confirms T2. The comparison function `tumblercmp` delegates to `abscmp`, which performs a purely positional comparison: exponent first (a proxy for the number of leading zeros), then lexicographic mantissa slot-by-slot. No tree structure, no index, no external state is consulted.

*Formal Contract:*
- *Preconditions:* `a, b ∈ T` — two well-formed tumblers (finite sequences over ℕ with `#a ≥ 1` and `#b ≥ 1`, per T0).
- *Depends:* T0 (CarrierSetDefinition) — the proof's `min(#a, #b)` bound on component pairs examined is expressed in terms of the length operator `#·` that T0 introduces, and the componentwise scan at positions `i = 1, 2, ...` consults the component-projection `·ᵢ` that T0 also introduces; postcondition (c) enumerates the consulted values as `{aᵢ : 1 ≤ i ≤ #a}`, `{bᵢ : 1 ≤ i ≤ #b}`, `#a`, and `#b`, each of which is a T0 operator applied to a T0 element. T1 (LexicographicOrder) — T2's claim is that the order relation defined by T1 is computable from the two tumblers alone, so T1 is the subject of the claim; the proof unfolds T1's witness structure (existence of `k ≥ 1` with `aᵢ = bᵢ` for all `1 ≤ i < k` and either case (i) component divergence or case (ii) prefix exhaustion) and dispatches the two outcomes of the scan to T1 case (i) — Case 1 (divergence at `k ≤ min(m, n)` with the ordering decided by `aₖ < bₖ` vs. `bₖ < aₖ`) — and T1 case (ii) — Case 2 with `m < n`, which gives `a < b` as a proper prefix (the symmetric `n < m` branch of Case 2 invokes T1 case (ii) with the roles swapped to conclude `b < a`). T3 (CanonicalRepresentation) — postcondition (a) requires T3 for the equality case: when all `min(m, n)` component pairs agree and `m = n`, T3 bridges component-level agreement to `a = b`. NAT-order (NatStrictTotalOrder) — the proof invokes trichotomy on ℕ at two distinct sites, each necessary for the case split to be exhaustive and to dispatch unambiguously into T1's clauses. Case 1 invokes trichotomy at the component pair `(aₖ, bₖ)` to resolve the case hypothesis `aₖ ≠ bₖ` into the two-way strict dichotomy `aₖ < bₖ ∨ bₖ < aₖ`: trichotomy's three disjuncts `aₖ < bₖ`, `aₖ = bₖ`, `bₖ < aₖ` are exhaustive, and the case hypothesis excludes the middle disjunct, while the exactly-one clause guarantees that one of the remaining two strict comparisons holds. Whichever holds dispatches to T1 case (i) — `aₖ < bₖ` witnessing `a < b` and `bₖ < aₖ` witnessing `b < a` — so postcondition (a) is decided in this case exactly because trichotomy supplies the strict comparison. Case 2 invokes trichotomy at the length pair `(m, n)` to partition the outcome of exhausting the shared range into three branches — `m < n`, `n < m`, `m = n` — which dispatch respectively to T1 case (ii) (`a < b` as a proper prefix), its symmetric instance (`b < a`), and T3 (`a = b` from componentwise agreement on the shared range extended to agreement everywhere by `m = n`); the exhaustiveness of this three-way partition — the claim that no fourth outcome is possible for the length pair — is what underwrites postcondition (a) in this branch, since without trichotomy's exactly-one clause at `(m, n)` the "no divergence within the shared range" branch could fail to produce any conclusion.
- *Postconditions:* (a) The ordering among `a` and `b` under T1 is determined. (b) At most `min(#a, #b)` component pairs are examined. (c) The only values consulted are `{aᵢ : 1 ≤ i ≤ #a}`, `{bᵢ : 1 ≤ i ≤ #b}`, `#a`, and `#b`.
- *Frame:* No external data structure is read or modified — the comparison is a pure function of the two tumblers.
