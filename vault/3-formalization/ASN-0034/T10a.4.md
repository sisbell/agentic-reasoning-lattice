**T10a.4 (T4Preservation).** The allocator discipline produces only T4-compliant addresses.

*Proof.* The discipline permits two operations: sibling production via `inc(·, 0)` and child-spawning via `inc(·, k')` with `k' ∈ {1, 2}`. By TA5a, `inc(t, 0)` unconditionally preserves T4 — no zeros are added and no new adjacencies are introduced, since sibling increment modifies only position `sig(t)`, advancing a positive value by one. For child-spawning, TA5a establishes that `inc(t, k)` preserves T4 if and only if `k = 1 ∧ zeros(t) ≤ 3` or `k = 2 ∧ zeros(t) ≤ 2` — these are biconditional: T4 is preserved exactly when these bounds hold, and violated otherwise. The T10a axiom restricts child-spawning to precisely these TA5a-preserving cases: `k' = 1` only when `zeros(t) ≤ 3`, `k' = 2` only when `zeros(t) ≤ 2`. The discipline therefore admits no child-spawning operation outside the T4-preserving envelope. Since both permitted operations preserve T4, every output of a conforming allocator satisfies T4. ∎

*Formal Contract:*
- *Preconditions:* Allocator conforming to T10a; parent address `t` satisfying T4.
- *Postconditions:* Every sibling and child output satisfies T4.
