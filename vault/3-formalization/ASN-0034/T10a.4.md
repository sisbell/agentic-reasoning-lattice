**T10a.4 (T4Preservation).** The allocator discipline produces only T4-compliant addresses.

*Proof.* The discipline permits two operations: sibling production via `inc(·, 0)` and child-spawning via `inc(·, k')` with `k' ∈ {1, 2}`. By TA5a, `inc(t, 0)` unconditionally preserves T4 — no zeros are added and no new adjacencies are introduced, since sibling increment modifies only position `sig(t)`, advancing a positive value by one. For child-spawning, TA5a establishes that `inc(t, k)` preserves T4 when `k = 1 ∧ zeros(t) ≤ 3` or `k = 2 ∧ zeros(t) ≤ 2`. The T10a axiom restricts `k'` to `{1, 2}` within these bounds. Since both permitted operations preserve T4, every output of a conforming allocator satisfies T4. ∎

*Formal Contract:*
- *Preconditions:* Allocator conforming to T10a; parent address `t` satisfying T4.
- *Postconditions:* Every sibling and child output satisfies T4.
