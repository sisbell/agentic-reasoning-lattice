**T10a.4 (T4Preservation).** The allocator discipline produces only T4-compliant addresses.

*Proof.* We proceed by induction on the depth of the allocator tree.

*Base case.* The root allocator's base address satisfies T4 by the initialization constraint of T10a. Without this premise, a conforming allocator starting from a non-T4 base — say `[1, 0, 0, 3]`, which contains adjacent zeros and therefore a degenerate parse — would produce only non-T4 outputs despite obeying every operational constraint.

*Inductive step.* Suppose an allocator at depth `d` has a base address `b` satisfying T4. The discipline permits two operations: sibling production via `inc(·, 0)` and child-spawning via `inc(·, k')` with `k' ∈ {1, 2}`. By TA5a, `inc(b, 0)` unconditionally preserves T4 — no zeros are added and no new adjacencies are introduced, since sibling increment modifies only position `sig(b)`, advancing a positive value by one. For child-spawning, TA5a establishes that `inc(t, k)` preserves T4 if and only if `k = 1` (unconditional on any T4-valid `t`) or `k = 2 ∧ zeros(t) ≤ 2` — biconditional: T4 is preserved exactly when these conditions hold, and violated otherwise. The T10a axiom restricts child-spawning to precisely these TA5a-preserving cases: `k' = 1` imposes no further runtime bound beyond the T4-validity of `t` that the inductive hypothesis supplies (every `t ∈ dom` of the depth-`d` allocator is T4-valid, since the base `b` is T4-valid and the unconditional `k = 0` branch of TA5a propagates T4 along the sibling chain `t₀ = b, t₁ = inc(b, 0), …`), and `k' = 2` is admitted only when `zeros(t) ≤ 2`. The child-spawning output `inc(b, k')` therefore satisfies T4, and this output serves as the child allocator's base address at depth `d + 1`, closing the induction.

Since the base is T4-valid and every permitted operation preserves T4, every output of a conforming allocator satisfies T4. ∎

*Formal Contract:*
- *Preconditions:* Allocator tree conforming to T10a (including initialization: root base address satisfies T4).
- *Postconditions:* Every output at every depth satisfies T4.
- *Proof structure:* Induction on allocator tree depth. Base: T10a initialization constraint. Step: TA5a preservation under `inc(·, 0)` and `inc(·, k')` with `k' ∈ {1, 2}`.
- *Depends:* T10a (allocator discipline and initialization constraint), T4 (HierarchicalParsing — the invariant preserved by induction), TA5a (IncrementPreservesT4 — the preservation mechanism).
