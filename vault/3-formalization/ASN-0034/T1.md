## The total order

We require a total order on T. Nelson describes the "tumbler line" as a single linear sequence: "In a sense the tumbler line is like the real line, i.e., the line of integers and all the numbers in between." The system maps a hierarchical tree — servers containing accounts containing documents containing elements — onto this flat line via depth-first traversal. The traversal inherently produces a total order: for any two nodes in a tree, depth-first traversal visits one before the other. The ordering rule is lexicographic:

**T1 (LexicographicOrder).** For tumblers `a = a₁. ... .aₘ` and `b = b₁. ... .bₙ`, define `a < b` iff there exists `k ≥ 1` such that `(A i : 1 ≤ i < k : aᵢ = bᵢ)` and either:

  (i) `k ≤ m ∧ k ≤ n` and `aₖ < bₖ`, or

  (ii) `k = m + 1 ≤ n` (that is, `a` is a proper prefix of `b`).

The prefix convention — a prefix is less than any proper extension — is what makes depth-first traversal work. The server address `2` is less than every address within server `2`'s subtree, because every such address extends the prefix `2` with further components. This means server `2`'s subtree begins immediately after `2` in the order and extends until some address whose first component exceeds `2`.

*Proof.* We show that `<` as defined is a strict total order on T by establishing irreflexivity, trichotomy, and transitivity. The argument relies on the corresponding properties of `<` on ℕ (NAT-order) and on T3 (CanonicalRepresentation): tumblers with the same length and identical components at every position are equal. T3 is a consequence of T0's extensional characterisation of T as a set of finite sequences — two finite sequences are equal exactly when they agree in length and at every position — so it is a derived theorem, not an independent postulate.

*(a) Irreflexivity.* We must show: `(A a ∈ T :: ¬(a < a))`.

Suppose `a < a` for some `a ∈ T` with `#a = m`. Then there exists `k ≥ 1` with `aᵢ = aᵢ` for all `1 ≤ i < k` (vacuously satisfied) and either (i) `k ≤ m` and `aₖ < aₖ`, or (ii) `k = m + 1 ≤ m`. Case (i) requires `aₖ < aₖ`, violating NAT-order's irreflexivity of `<` on ℕ. Case (ii) requires `m + 1 ≤ m`. NAT-addcompat's strict successor inequality gives `m < m + 1`; NAT-order's trichotomy on ℕ then excludes both `m + 1 < m` and `m + 1 = m`, so the disjunction `m + 1 ≤ m` (which abbreviates `m + 1 < m ∨ m + 1 = m`) is false. Both cases produce contradictions, so no witness `k` exists and `¬(a < a)`.

*(b) Trichotomy.* We must show: `(A a, b ∈ T :: (a < b ∨ a = b ∨ b < a) ∧ ¬(a < b ∧ a = b) ∧ ¬(a < b ∧ b < a) ∧ ¬(a = b ∧ b < a))`, formalising "exactly one of `a < b`, `a = b`, `b < a`" as the conjunction of *AtLeastOne* — the opening disjunction asserting that at least one of the three relations holds — and *AtMostOne* — the three negated conjunctions asserting pairwise mutual exclusion, so that no two of the three relations co-hold. The three cases below each produce a witness for exactly one disjunct of *AtLeastOne* and discharge the two mutual-exclusion clauses of *AtMostOne* that involve that disjunct, so the two components are established jointly per case rather than in two separate sweeps.

Let `a, b ∈ T` with `#a = m` and `#b = n`. Define the *first divergence position* `k` as the least positive integer at which `a` and `b` disagree — either because `aₖ ≠ bₖ` at some position present in both tumblers (`k ≤ m ∧ k ≤ n`), or because one tumbler is exhausted at position `k` while the other continues (`k = m + 1 ≤ n`, or `k = n + 1 ≤ m`). The three-way branch on the length pair `(m, n)` — `m < n` contributing `{m + 1}`, `n < m` contributing `{n + 1}`, and the remaining `else` branch contributing `∅` — is a single NAT-order trichotomy invocation at `(m, n)`; we name this the *opening length-pair trichotomy*, and every subsequent same-pair invocation within part (b) — Case 1's derivation of `m = n` (reading off the `else` branch), Case 3's opening resolution of `m ≠ n` into `m < n ∨ n < m` together with the mutual exclusivity of those branches, and Case 3's case-(ii) reverse-witness rebuttals in each branch — folds into it under the co-pair absorption convention declared for the Depends enumeration below, earning no fresh trichotomy sites. (This same concept is formalized downstream as the corollary Divergence, whose two cases correspond to the two disjuncts here; the definitions are identical by construction, and any tightening of either must be mirrored in the other. The least-element claim — that `k` is well-defined whenever `a ≠ b` — rests on the well-ordering of ℕ applied to the nonempty subset of positions at which the two sequences fail to match, and is discharged from NAT-wellorder.) Three cases are exhaustive.

*Case 1: no divergence exists.* The divergence set is empty iff both its shared-position component and its length-excess component are empty. Emptiness of the shared-position component gives `aᵢ = bᵢ` for all `i` with `1 ≤ i ∧ i ≤ m ∧ i ≤ n`. Emptiness of the length-excess component places us in the `else` branch of the opening length-pair trichotomy at `(m, n)` — the branch where neither `m < n` nor `n < m` holds, which that trichotomy's remaining disjunct fixes as `m = n` (a co-pair reuse of the opening invocation, not a fresh trichotomy site). With `m = n` in hand, the shared-position equalities range over all of `1 ≤ i ≤ m`, so `a = b` by T3. By part (a), `¬(a < a)` and `¬(a > a)`, giving equality as the unique outcome.

*Case 2: `k ≤ m ∧ k ≤ n` and `aₖ ≠ bₖ`.* Since `aᵢ = bᵢ` for all `i < k` but `aₖ ≠ bₖ`, we have `a ≠ b` by the reverse direction of T3 (tumblers that differ in any component are distinct). By NAT-order's trichotomy on ℕ, exactly one of `aₖ < bₖ` or `bₖ < aₖ` holds. If `aₖ < bₖ`, then `k` witnesses `a < b` via T1 case (i); if `bₖ < aₖ`, then `k` witnesses `b < a` via T1 case (i). We confirm that no witness establishes the reverse. Any witness `k'` for the opposite ordering requires `aᵢ = bᵢ` for all `i < k'`. If `k' > k`, this fails at position `k` where `aₖ ≠ bₖ`. If `k' = k`, case (i) requires the opposite inequality at position `k`, which the exactly-one clause of the same NAT-order trichotomy invocation at the component pair `(aₖ, bₖ)` used to fix the forward witness already forecloses — a co-pair reuse of the opening's invocation, not a fresh trichotomy site — and case (ii) requires `k = n + 1` (or `k = m + 1`), contradicting the case hypothesis's conjunct `k ≤ n` (respectively `k ≤ m`). If `k' < k`, the minimality of `k` gives `a_{k'} = b_{k'}`, so case (i) — which at position `k'` requires a strict inequality between `a_{k'}` and `b_{k'}` (either `a_{k'} < b_{k'}` or `b_{k'} < a_{k'}`, depending on sub-case) — is excluded by NAT-order's trichotomy at the component pair `(a_{k'}, b_{k'})`, whose exactly-one clause rules out both candidate strict inequalities once the equality is in hand; and case (ii) requires `k' = n + 1` (or `m + 1`), but `k' < k` combined with the case hypothesis's conjunct `k ≤ n` (respectively `k ≤ m`) gives `k' < n` (respectively `k' < m`), contradicting both. No witness exists; exactly one ordering holds.

*Case 3: all shared positions agree but `m ≠ n` — that is, `aᵢ = bᵢ` for all positions `i` present in both tumblers (`1 ≤ i ∧ i ≤ m ∧ i ≤ n`), and the first divergence arises because one tumbler is exhausted before the other.* Since `aᵢ = bᵢ` for all such `i` but `m ≠ n`, we have `a ≠ b` by T3 (distinct lengths). The opening length-pair trichotomy — already invoked at the divergence-set construction above — resolves the case hypothesis `m ≠ n` into the strict dichotomy `m < n ∨ n < m` as the two non-`else` branches of its three-way partition, and the same invocation's exactly-one clause makes those two branches mutually exclusive — eliminating any third concurrent outcome at the length level as a co-pair reuse of the opening rather than a fresh trichotomy site. If `m < n`, then `k = m + 1 ≤ n`, so `a` is a proper prefix of `b` and `k` witnesses `a < b` via T1 case (ii). No witness for `b < a` exists: case (i) would require `bⱼ < aⱼ` at some position `j` present in both tumblers (`j ≤ m ∧ j ≤ n`), but all such positions have equal components, so NAT-order's trichotomy at the component pair `(aⱼ, bⱼ)` — its exactly-one clause applied once the equality `aⱼ = bⱼ` is fixed — excludes the strict inequality `bⱼ < aⱼ`; case (ii) would require `b` to be a proper prefix of `a`, which by T1 case (ii)'s definition applied to the reverse witness for `b < a` expands to `#b + 1 ≤ #a`, i.e., `n + 1 ≤ m`. NAT-addcompat's strict successor inequality `n < n + 1` then composes with this bound — via NAT-order's transitivity in the strict disjunct `n + 1 < m` of the unpacked hypothesis, via substitution of the equality `n + 1 = m` into `n < n + 1` in the equality disjunct — to yield `n < m`, which together with the case hypothesis `m < n` is foreclosed by NAT-order's trichotomy at the length pair `(m, n)` (the mutual-exclusivity form of the same invocation that opened the case). If `n < m`, the mirrored argument gives `b < a` as the unique outcome. Case (i) of any reverse witness for `a < b` would require `aⱼ < bⱼ` at some position `j` present in both tumblers (`j ≤ m ∧ j ≤ n`), but all such positions have equal components, so NAT-order's trichotomy at the component pair `(aⱼ, bⱼ)` — its exactly-one clause applied once the equality `aⱼ = bⱼ` is fixed — excludes the strict inequality `aⱼ < bⱼ`. Case (ii) of any reverse witness for `a < b` requires `a` to be a proper prefix of `b`, i.e., `m + 1 ≤ n`, from which NAT-addcompat's `m < m + 1` composes (via NAT-order transitivity in the strict disjunct `m + 1 < n`, via substitution of `m + 1 = n` into `m < m + 1` in the equality disjunct) to `m < n`, contradicting `n < m` by the same length-pair trichotomy.

These three cases partition all pairs in `T × T`, and in each case exactly one of the three relations holds.

*(c) Transitivity.* We must show: `(A a, b, c ∈ T : a < b ∧ b < c : a < c)`.

Let `k₁` witness `a < b` and `k₂` witness `b < c`, with `#a = m`, `#b = n`, `#c = p`. For all `i` with `i < k₁ ∧ i < k₂`, the agreement conditions of the two hypotheses give `aᵢ = bᵢ` (since `i < k₁`) and `bᵢ = cᵢ` (since `i < k₂`), hence `aᵢ = cᵢ` by transitivity of equality. We produce a witness for `a < c` by case analysis on the relationship between `k₁` and `k₂`.

*Case k₁ < k₂.* Position `k₁` lies in the agreement range of `b < c`, so `bₖ₁ = cₖ₁`. If `a < b` via case (i): `aₖ₁ < bₖ₁` with `k₁ ≤ m ∧ k₁ ≤ n`, and since `bₖ₁ = cₖ₁` we have `aₖ₁ < cₖ₁`; the existence of `cₖ₁` gives `k₁ ≤ p`, so `k₁ ≤ m ∧ k₁ ≤ p`, and position `k₁` witnesses `a < c` via T1 case (i). If `a < b` via case (ii): `k₁ = m + 1 ≤ n`, and since `bₖ₁ = cₖ₁` the component `cₖ₁` exists, giving `p ≥ k₁ = m + 1`; thus `a` is a proper prefix of `c` and `k₁` witnesses `a < c` via T1 case (ii).

*Case k₂ < k₁.* Position `k₂` lies in the agreement range of `a < b`, so `aₖ₂ = bₖ₂`. We first show `b < c` must be via case (i). If `b < c` were via case (ii), then `k₂ = n + 1`. But `a < b` requires `k₁ ≤ n` — in case (i), the conjunct `k₁ ≤ n` of the case-(i) hypothesis `k₁ ≤ m ∧ k₁ ≤ n`; in case (ii), `k₁ = m + 1 ≤ n` — and NAT-addcompat's strict successor inequality gives `n + 1 > n`, so `k₂ = n + 1 > n ≥ k₁`, which by NAT-order's transitivity yields `k₂ > k₁`; NAT-order's trichotomy at the index pair `(k₁, k₂)` — specifically its exactly-one clause — then excludes the co-holding of `k₂ > k₁` (equivalently `k₁ < k₂`) and the case hypothesis `k₂ < k₁`, refuting the hypothetical case (ii) of `b < c`. Therefore `b < c` is via case (i): `bₖ₂ < cₖ₂` with `k₂ ≤ n ∧ k₂ ≤ p`. Since `k₂ < k₁` and `a` has components at all positions below `k₁`, we have `k₂ ≤ m`, giving `aₖ₂ = bₖ₂ < cₖ₂` with `k₂ ≤ m ∧ k₂ ≤ p`. Position `k₂` witnesses `a < c` via T1 case (i).

*Case k₁ = k₂ = k.* Both divergences occur at position `k`, and we have `aᵢ = cᵢ` for all `i < k`. The two-case structure of T1 gives four combinations for the pair of witnesses; we show that exactly two are realizable.

*Sub-case (i, i):* `aₖ < bₖ` with `k ≤ m ∧ k ≤ n` and `bₖ < cₖ` with `k ≤ n ∧ k ≤ p`. NAT-order's transitivity of `<` on ℕ gives `aₖ < cₖ`. The first hypothesis supplies `k ≤ m` and the second supplies `k ≤ p`, so `k ≤ m ∧ k ≤ p`, and position `k` witnesses `a < c` via T1 case (i).

*Sub-case (ii, i):* `k = m + 1 ≤ n` and `bₖ < cₖ` with `k ≤ n ∧ k ≤ p`. The second hypothesis's conjunct `k ≤ p` gives `m + 1 = k ≤ p`, so `a` is a proper prefix of `c` and position `k` witnesses `a < c` via T1 case (ii).

*Sub-case (i, ii):* `aₖ < bₖ` with `k ≤ m ∧ k ≤ n` supplies `k ≤ n` directly from the second conjunct, while `k = n + 1 ≤ p` requires `k > n`. The conjunction `k ≤ n` and `k > n` is a contradiction; this sub-case cannot arise.

*Sub-case (ii, ii):* `k = m + 1 ≤ n` and `k = n + 1 ≤ p`. Then `m + 1 = n + 1`, and NAT-cancel's right cancellation at the fixed summand `1` gives `m = n`. The first condition `m + 1 ≤ n` abbreviates `m + 1 < n ∨ m + 1 = n`; in the strict disjunct, NAT-order's transitivity composes NAT-addcompat's strict successor inequality `m < m + 1` with `m + 1 < n` to yield `m < n`, while in the equality disjunct, substituting `m + 1 = n` into `m < m + 1` likewise yields `m < n`. Either way `m < n`; NAT-order's trichotomy at the length pair `(m, n)` — specifically its exactly-one clause — then excludes the co-holding of this strict comparison and the equality `m = n` produced above by NAT-cancel's right cancellation. This sub-case cannot arise.

In every realizable combination, a witness for `a < c` under T1 is produced. ∎

The strict total order `<` admits the customary non-strict companions. For `a, b ∈ T`, we write `a ≤ b` to mean `a < b ∨ a = b`, and `a ≥ b` to mean `b ≤ a`. The well-definedness of these abbreviations is immediate from trichotomy (part (b)): exactly one of `a < b`, `a = b`, `b < a` holds for any pair in `T`, so each disjunction is decidable.

The foundation axiom NAT-wellorder (NatWellOrdering), consumed in part (b) Trichotomy's construction of the *first divergence position* `k` as the least element of the divergence set, is stated separately in this ASN and its formal content is reproduced here for reference at the citation site, so that the least-element step terminates at a stated source visible within this property's reading context. NAT-wellorder asserts the least-element principle for ℕ: `(A S : S ⊆ ℕ ∧ S ≠ ∅ : (E m ∈ S :: (A n ∈ S :: m ≤ n)))` — every nonempty subset of ℕ has a least element. The divergence set `{i ∈ ℕ : 1 ≤ i ∧ i ≤ m ∧ i ≤ n ∧ aᵢ ≠ bᵢ} ∪ ({m + 1} when m < n, {n + 1} when n < m, else ∅)` is a subset of ℕ, nonempty in Cases 2 and 3 by the construction's hypothesis that some divergence exists, and therefore has a least element under NAT-wellorder; that least element is the "first divergence position" `k` that Cases 2 and 3 operate on, and its minimality — that every `i < k` satisfies `aᵢ = bᵢ` — is the property Case 2's `k' < k` rebuttal and Case 3's "all shared positions agree" hypothesis both consume.

*Formal Contract:*
- *Definition:* `a < b` iff `∃ k ≥ 1` with `(A i : 1 ≤ i < k : aᵢ = bᵢ)` and either (i) `k ≤ #a ∧ k ≤ #b ∧ aₖ < bₖ`, or (ii) `k = #a+1 ≤ #b`.
- *Abbreviations:* `a ≤ b` abbreviates `a < b ∨ a = b`; `a ≥ b` abbreviates `b ≤ a`. These are introduction-rule definitions of the symbols `≤` and `≥`, not derived theorems — they extend the vocabulary of the formal contract rather than asserting a further obligation on `<`. Downstream properties that read `≥` or `≤` off T1 cite this abbreviation block as their definitional source; a reviser tightening the Definition of `<` must mirror the change here, since `≤` and `≥` unfold to `<` and `=` by these abbreviations and nowhere else.
- *Depends:* T0 (CarrierSetDefinition) — the definition uses length `#a` and component projection `aₖ` for `a ∈ T`, which T0 introduces. The NAT-* site counts that follow use a uniform per-instance convention: branches of a case that instantiate an axiom at distinct symbols count as separate sites. In particular, part (b) Trichotomy's Case 3 has two mirror branches (`m < n` and `n < m`) whose primary witnesses and reverse-witness rebuttals instantiate their NAT-* axioms at distinct symbols — `m + 1 ≤ n` vs `n + 1 ≤ m` for NAT-discrete's forward direction, `m < m + 1` vs `n < n + 1` for NAT-addcompat's strict successor inequality, and the correspondingly distinct transitivity compositions `m < m + 1 < n` vs `n < n + 1 < m` for NAT-order — so each branch contributes one site per axiom it invokes, and the two branches together contribute two sites per axiom rather than one combined "Case 3" site. Part (b) Trichotomy's Case 2 is packaged under the same convention: its two mirror sub-cases — the `aₖ < bₖ` sub-case (primary `a < b`, reverse witness hypothesising `b < a`) and the `bₖ < aₖ` sub-case (primary `b < a`, reverse witness hypothesising `a < b`) — instantiate the reverse-witness case-(ii) rebuttals' NAT-addcompat strict successor inequality and NAT-order trichotomy refutations at distinct symbols, `n < n + 1` refuting `n + 1 ≤ n` and `n + 1 < n` in the first sub-case versus `m < m + 1` refuting `m + 1 ≤ m` and `m + 1 < m` in the second, so each sub-case earns one site per axiom per rebuttal location rather than one combined site that folds both symbols via "(resp.)" notation. NAT-order (NatStrictTotalOrder) — the proof invokes the strict-total-order structure of `<` on ℕ at twenty-two distinct sites, each a clause of NAT-order's Axiom. - *Postconditions:* (a) Irreflexivity — `(A a ∈ T :: ¬(a < a))`. (b) Trichotomy — `(A a,b ∈ T :: (a < b ∨ a = b ∨ b < a) ∧ ¬(a < b ∧ a = b) ∧ ¬(a < b ∧ b < a) ∧ ¬(a = b ∧ b < a))`, conjoining *AtLeastOne* (the initial three-way disjunction) with *AtMostOne* (the three negated pairwise conjunctions); this is the formal expansion of "exactly one of `a < b`, `a = b`, `b < a`" into a predicate whose two components — existence of at least one relation and mutual exclusion of any pair — downstream consumers citing T1's trichotomy can invoke individually or jointly as each site requires. (c) Transitivity — `(A a,b,c ∈ T : a < b ∧ b < c : a < c)`.

Nelson's assertion that the tumbler line is total — that two addresses are never incomparable — is architecturally load-bearing. Spans are defined as contiguous regions on the tumbler line: "A span in the tumbler line, represented by two tumblers, refers to a subtree of the entire docuverse." If two addresses were incomparable, the interval between them would be undefined, and the entire machinery of span-sets, link endsets, and content reference would collapse.
