## The total order

We require a total order on T. Nelson describes the "tumbler line" as a single linear sequence: "In a sense the tumbler line is like the real line, i.e., the line of integers and all the numbers in between." The system maps a hierarchical tree — servers containing accounts containing documents containing elements — onto this flat line via depth-first traversal. The traversal inherently produces a total order: for any two nodes in a tree, depth-first traversal visits one before the other. The ordering rule is lexicographic:

**T1 (LexicographicOrder).** For tumblers `a = a₁. ... .aₘ` and `b = b₁. ... .bₙ`, define `a < b` iff there exists `k ≥ 1` such that `(A i : 1 ≤ i < k : aᵢ = bᵢ)` and either:

  (i) `k ≤ min(m, n)` and `aₖ < bₖ`, or

  (ii) `k = m + 1 ≤ n` (that is, `a` is a proper prefix of `b`).

The prefix convention — a prefix is less than any proper extension — is what makes depth-first traversal work. The server address `2` is less than every address within server `2`'s subtree, because every such address extends the prefix `2` with further components. This means server `2`'s subtree begins immediately after `2` in the order and extends until some address whose first component exceeds `2`.

*Proof.* We show that `<` as defined is a strict total order on T by establishing irreflexivity, trichotomy, and transitivity. The argument relies on the corresponding properties of `<` on ℕ (NAT-order) and on T3 (CanonicalRepresentation): tumblers with the same length and identical components at every position are equal. T3 is a consequence of T0's extensional characterisation of T as a set of finite sequences — two finite sequences are equal exactly when they agree in length and at every position — so it is a derived theorem, not an independent postulate.

*(a) Irreflexivity.* We must show: `(A a ∈ T :: ¬(a < a))`.

Suppose `a < a` for some `a ∈ T` with `#a = m`. Then there exists `k ≥ 1` with `aᵢ = aᵢ` for all `1 ≤ i < k` (vacuously satisfied) and either (i) `k ≤ m` and `aₖ < aₖ`, or (ii) `k = m + 1 ≤ m`. Case (i) requires `aₖ < aₖ`, violating NAT-order's irreflexivity of `<` on ℕ. Case (ii) requires `m + 1 ≤ m`. NAT-addcompat's strict successor inequality gives `m < m + 1`; NAT-order's trichotomy on ℕ then excludes both `m + 1 < m` and `m + 1 = m`, so the disjunction `m + 1 ≤ m` (which abbreviates `m + 1 < m ∨ m + 1 = m`) is false. Both cases produce contradictions, so no witness `k` exists and `¬(a < a)`.

*(b) Trichotomy.* We must show: `(A a, b ∈ T :: exactly one of a < b, a = b, b < a)`.

Let `a, b ∈ T` with `#a = m` and `#b = n`. Define the *first divergence position* `k` as the least positive integer at which `a` and `b` disagree — either because `aₖ ≠ bₖ` at some `k ≤ min(m, n)`, or because one tumbler is exhausted at position `k = min(m, n) + 1` while the other continues. (This same concept is formalized downstream as the corollary Divergence, whose two cases correspond to the two disjuncts here; the definitions are identical by construction, and any tightening of either must be mirrored in the other. The least-element claim — that `k` is well-defined whenever `a ≠ b` — rests on the well-ordering of ℕ applied to the nonempty subset of positions at which the two sequences fail to match, and is discharged from NAT-wellorder.) Three cases are exhaustive.

*Case 1: no divergence exists.* Then `m = n` and `aᵢ = bᵢ` for all `1 ≤ i ≤ m`, so `a = b` by T3. By part (a), `¬(a < a)` and `¬(a > a)`, giving equality as the unique outcome.

*Case 2: `k ≤ min(m, n)` and `aₖ ≠ bₖ`.* Since `aᵢ = bᵢ` for all `i < k` but `aₖ ≠ bₖ`, we have `a ≠ b` by the reverse direction of T3 (tumblers that differ in any component are distinct). By NAT-order's trichotomy on ℕ, exactly one of `aₖ < bₖ` or `bₖ < aₖ` holds. If `aₖ < bₖ`, then `k` witnesses `a < b` via T1 case (i); if `bₖ < aₖ`, then `k` witnesses `b < a` via T1 case (i). We confirm that no witness establishes the reverse. Any witness `k'` for the opposite ordering requires `aᵢ = bᵢ` for all `i < k'`. If `k' > k`, this fails at position `k` where `aₖ ≠ bₖ`. If `k' = k`, case (i) requires the opposite inequality at position `k`, contradicting NAT-order's trichotomy, and case (ii) requires `k = n + 1` (or `k = m + 1`), contradicting `k ≤ min(m, n) ≤ n` (respectively `≤ m`). If `k' < k`, the minimality of `k` gives `a_{k'} = b_{k'}`, so case (i) fails on equal components and case (ii) requires `k' = n + 1` (or `m + 1`), but `k' < k ≤ min(m, n)` gives `k' < n` and `k' < m`, contradicting both. No witness exists; exactly one ordering holds.

*Case 3: `k = min(m, n) + 1` — all shared positions agree but `m ≠ n`.* Since `aᵢ = bᵢ` for all `1 ≤ i ≤ min(m, n)` but `m ≠ n`, we have `a ≠ b` by T3 (distinct lengths). NAT-order's trichotomy on ℕ applied to the length pair `(m, n)` resolves the case hypothesis `m ≠ n` into the strict dichotomy `m < n ∨ n < m`, and the same trichotomy invocation's exactly-one clause makes the two branches mutually exclusive — eliminating any third concurrent outcome at the length level without a second trichotomy use. If `m < n`, then `k = m + 1 ≤ n`, so `a` is a proper prefix of `b` and `k` witnesses `a < b` via T1 case (ii). No witness for `b < a` exists: case (i) would require `bⱼ < aⱼ` at some position `j ≤ min(m, n)`, but all such positions have equal components; case (ii) would require `b` to be a proper prefix of `a`, which by T1 case (ii)'s definition applied to the reverse witness for `b < a` expands to `#b + 1 ≤ #a`, i.e., `n + 1 ≤ m`. NAT-addcompat's strict successor inequality `n < n + 1` then composes with this bound — via NAT-order's transitivity in the strict disjunct `n + 1 < m` of the unpacked hypothesis, via substitution of the equality `n + 1 = m` into `n < n + 1` in the equality disjunct — to yield `n < m`, which together with the case hypothesis `m < n` is foreclosed by NAT-order's trichotomy at the length pair `(m, n)` (the mutual-exclusivity form of the same invocation that opened the case). If `n < m`, the mirrored argument gives `b < a` as the unique outcome — case (ii) of any reverse witness for `a < b` requires `a` to be a proper prefix of `b`, i.e., `m + 1 ≤ n`, from which NAT-addcompat's `m < m + 1` composes (via NAT-order transitivity in the strict disjunct `m + 1 < n`, via substitution of `m + 1 = n` into `m < m + 1` in the equality disjunct) to `m < n`, contradicting `n < m` by the same length-pair trichotomy.

These three cases partition all pairs in `T × T`, and in each case exactly one of the three relations holds.

*(c) Transitivity.* We must show: `(A a, b, c ∈ T : a < b ∧ b < c : a < c)`.

Let `k₁` witness `a < b` and `k₂` witness `b < c`, with `#a = m`, `#b = n`, `#c = p`. For all `i < min(k₁, k₂)`, the agreement conditions of the two hypotheses give `aᵢ = bᵢ` (since `i < k₁`) and `bᵢ = cᵢ` (since `i < k₂`), hence `aᵢ = cᵢ` by transitivity of equality. We produce a witness for `a < c` by case analysis on the relationship between `k₁` and `k₂`.

*Case k₁ < k₂.* Position `k₁` lies in the agreement range of `b < c`, so `bₖ₁ = cₖ₁`. If `a < b` via case (i): `aₖ₁ < bₖ₁` with `k₁ ≤ min(m, n)`, and since `bₖ₁ = cₖ₁` we have `aₖ₁ < cₖ₁`; the existence of `cₖ₁` gives `k₁ ≤ p`, so `k₁ ≤ min(m, p)`, and position `k₁` witnesses `a < c` via T1 case (i). If `a < b` via case (ii): `k₁ = m + 1 ≤ n`, and since `bₖ₁ = cₖ₁` the component `cₖ₁` exists, giving `p ≥ k₁ = m + 1`; thus `a` is a proper prefix of `c` and `k₁` witnesses `a < c` via T1 case (ii).

*Case k₂ < k₁.* Position `k₂` lies in the agreement range of `a < b`, so `aₖ₂ = bₖ₂`. We first show `b < c` must be via case (i). If `b < c` were via case (ii), then `k₂ = n + 1`. But `a < b` requires `k₁ ≤ n` — in case (i), `k₁ ≤ min(m, n) ≤ n`; in case (ii), `k₁ = m + 1 ≤ n` — and NAT-addcompat's strict successor inequality gives `n + 1 > n`, so `k₂ = n + 1 > n ≥ k₁`, which by NAT-order's transitivity yields `k₂ > k₁`, contradicting `k₂ < k₁`. Therefore `b < c` is via case (i): `bₖ₂ < cₖ₂` with `k₂ ≤ min(n, p)`. Since `k₂ < k₁` and `a` has components at all positions below `k₁`, we have `k₂ ≤ m`, giving `aₖ₂ = bₖ₂ < cₖ₂` with `k₂ ≤ min(m, p)`. Position `k₂` witnesses `a < c` via T1 case (i).

*Case k₁ = k₂ = k.* Both divergences occur at position `k`, and we have `aᵢ = cᵢ` for all `i < k`. The two-case structure of T1 gives four combinations for the pair of witnesses; we show that exactly two are realizable.

*Sub-case (i, i):* `aₖ < bₖ` with `k ≤ min(m, n)` and `bₖ < cₖ` with `k ≤ min(n, p)`. NAT-order's transitivity of `<` on ℕ gives `aₖ < cₖ`. Since `k ≤ m` and `k ≤ p`, we have `k ≤ min(m, p)`, and position `k` witnesses `a < c` via T1 case (i).

*Sub-case (ii, i):* `k = m + 1 ≤ n` and `bₖ < cₖ` with `k ≤ min(n, p)`. The bound `k ≤ p` gives `m + 1 = k ≤ p`, so `a` is a proper prefix of `c` and position `k` witnesses `a < c` via T1 case (ii).

*Sub-case (i, ii):* `aₖ < bₖ` with `k ≤ min(m, n)` requires `k ≤ n`, while `k = n + 1 ≤ p` requires `k > n`. The conjunction `k ≤ n` and `k > n` is a contradiction; this sub-case cannot arise.

*Sub-case (ii, ii):* `k = m + 1 ≤ n` and `k = n + 1 ≤ p`. Then `m + 1 = n + 1`, and NAT-cancel's right cancellation at the fixed summand `1` gives `m = n`. The first condition `m + 1 ≤ n` abbreviates `m + 1 < n ∨ m + 1 = n`; in the strict disjunct, NAT-order's transitivity composes NAT-addcompat's strict successor inequality `m < m + 1` with `m + 1 < n` to yield `m < n`, while in the equality disjunct, substituting `m + 1 = n` into `m < m + 1` likewise yields `m < n`. Either way `m < n`, contradicting `m = n`. This sub-case cannot arise.

In every realizable combination, a witness for `a < c` under T1 is produced. ∎

The strict total order `<` admits the customary non-strict companions. For `a, b ∈ T`, we write `a ≤ b` to mean `a < b ∨ a = b`, and `a ≥ b` to mean `b ≤ a`. The well-definedness of these abbreviations is immediate from trichotomy (part (b)): exactly one of `a < b`, `a = b`, `b < a` holds for any pair in `T`, so each disjunction is decidable.

*Formal Contract:*
- *Definition:* `a < b` iff `∃ k ≥ 1` with `(A i : 1 ≤ i < k : aᵢ = bᵢ)` and either (i) `k ≤ min(#a,#b) ∧ aₖ < bₖ`, or (ii) `k = #a+1 ≤ #b`.
- *Depends:* T0 (CarrierSetDefinition) — the definition uses length `#a` and component projection `aₖ` for `a ∈ T`, which T0 introduces. The NAT-* site counts that follow use a uniform per-instance convention: branches of a case that instantiate an axiom at distinct symbols count as separate sites. In particular, part (b) Trichotomy's Case 3 has two mirror branches (`m < n` and `n < m`) whose primary witnesses and reverse-witness rebuttals instantiate their NAT-* axioms at distinct symbols — `m + 1 ≤ n` vs `n + 1 ≤ m` for NAT-discrete's forward direction, `m < m + 1` vs `n < n + 1` for NAT-addcompat's strict successor inequality, and the correspondingly distinct transitivity compositions `m < m + 1 < n` vs `n < n + 1 < m` for NAT-order — so each branch contributes one site per axiom it invokes, and the two branches together contribute two sites per axiom rather than one combined "Case 3" site. NAT-order (NatStrictTotalOrder) — the proof invokes the strict-total-order structure of `<` on ℕ at thirteen distinct sites, each a clause of NAT-order's Axiom. Irreflexivity is invoked once, at part (a) Case (i), to contradict `aₖ < aₖ`. Trichotomy is invoked seven times: part (a) Case (ii), to exclude `m + 1 < m` and `m + 1 = m` once `m < m + 1` is in hand; part (b) Case 2's opening step, to resolve the ordering of disagreeing components `aₖ` and `bₖ` into `aₖ < bₖ` or `bₖ < aₖ`; part (b) Case 2's `k' = k` reverse-witness rebuttal via the reverse witness's case (i), to exclude the opposite inequality at position `k` against the forward witness already fixed by the opening step; part (b) Case 2's `k' = k` reverse-witness rebuttal via the reverse witness's case (ii), to exclude `n + 1 < n` and `n + 1 = n` (respectively `m + 1 < m` and `m + 1 = m`) once NAT-addcompat supplies `n < n + 1` (resp. `m < m + 1`), thereby refuting the sub-case's `k = n + 1 ≤ n` (resp. `k = m + 1 ≤ m`); part (b) Case 2's `k' < k` reverse-witness rebuttal via the reverse witness's case (ii), to exclude `n + 1 < n` (resp. `m + 1 < m`) once NAT-addcompat supplies `n < n + 1` (resp. `m < m + 1`), thereby refuting the sub-case's `k' = n + 1 < n` (resp. `k' = m + 1 < m`) under the agreement bound `k' < k ≤ min(m, n)`; part (b) Case 3's opening step, to resolve the case hypothesis `m ≠ n` into the strict dichotomy `m < n ∨ n < m` (paralleling Case 2's opening step but applied at the length pair `(m, n)` rather than at the component pair `(aₖ, bₖ)`), with the same invocation's exactly-one clause additionally underwriting both the mutual exclusivity of the `m < n` and `n < m` branches and the case-(ii) reverse-witness rebuttals within each branch (where, after NAT-addcompat's strict successor inequality composes with the rebuttal's `n + 1 ≤ m` resp. `m + 1 ≤ n` to yield `n < m` resp. `m < n`, the conjunction with the case hypothesis is foreclosed by mutual exclusivity at the same length pair) — so no second trichotomy use at the length level is required to eliminate `m = n` past the case hypothesis or to refute the case-(ii) rebuttals' derived strict comparisons; and part (c) Transitivity sub-case (i, ii), to refute the conjunction `k ≤ n ∧ k > n` once NAT-addcompat supplies `n < n + 1` and the substitution `k = n + 1` delivers the strict inequality `n < k`. Transitivity is invoked six times: part (c) Transitivity, Case `k₂ < k₁` under the hypothetical case-(ii) branch of `b < c`, to chain `k₂ > n ≥ k₁` into `k₂ > k₁`; part (c) Transitivity, Case `k₂ < k₁` under the case-(i) branch of `a < b`, to compose the strict comparison `k₂ < k₁` with the strict disjunct `k₁ < m` of the unpacked bound `k₁ ≤ m` (the bound itself discharged from the case-(i) hypothesis `k₁ ≤ min(m, n)` via the standard `min(m, n) ≤ m`) into `k₂ < m`, whence `k₂ ≤ m` — the bound required for position `k₂`'s T1 case-(i) witness for `a < c` to satisfy `k₂ ≤ min(m, p)` (the equality disjunct `k₁ = m` of the unpacked bound resolves by substitution of `k₁ = m` into `k₂ < k₁` to yield `k₂ < m` directly, so it does not contribute a separate transitivity site; the sibling case-(ii) branch of `a < b`, where `k₁ = m + 1`, routes `k₂ ≤ m` through NAT-discrete's contrapositive `k₂ < m + 1 ⟹ k₂ ≤ m` rather than NAT-order's transitivity, and so does not contribute a transitivity site at this step either); sub-case (i, i), to compose `aₖ < bₖ` and `bₖ < cₖ` into `aₖ < cₖ`; sub-case (ii, ii), to compose NAT-addcompat's strict successor inequality `m < m + 1` with the strict disjunct `m + 1 < n` of the unpacked hypothesis `m + 1 ≤ n` into `m < n` — the strict comparison that contradicts the `m = n` produced by NAT-cancel's right cancellation in the same sub-case (the equality disjunct `m + 1 = n` uses substitution of `m + 1 = n` into `m < m + 1` rather than transitivity, so it is not a separate transitivity site); part (b) Trichotomy, Case 3's `m < n` branch reverse-witness rebuttal under the reverse witness's case (ii), to compose NAT-addcompat's strict successor inequality `n < n + 1` with the strict disjunct `n + 1 < m` of the unpacked rebuttal hypothesis `n + 1 ≤ m` into `n < m` — the strict comparison that, against the case hypothesis `m < n`, is foreclosed by NAT-order's trichotomy at the length pair `(m, n)` (the equality disjunct `n + 1 = m` resolves by substitution of `n + 1 = m` into `n < n + 1` rather than transitivity, so it does not contribute a separate site); and part (b) Trichotomy, Case 3's `n < m` branch reverse-witness rebuttal under the reverse witness's case (ii), to compose NAT-addcompat's strict successor inequality `m < m + 1` with the strict disjunct `m + 1 < n` of the unpacked rebuttal hypothesis `m + 1 ≤ n` into `m < n` — the strict comparison that, against the case hypothesis `n < m`, is foreclosed by the same length-pair trichotomy (the equality disjunct `m + 1 = n` again resolves by substitution rather than transitivity). NAT-addcompat (NatAdditionOrderCompatibility) — the proof invokes the strict successor inequality `n < n + 1` at eight distinct sites: part (a) Irreflexivity, Case (ii), to supply `m < m + 1` so that NAT-order's trichotomy can refute `m + 1 ≤ m`; part (b) Trichotomy, Case 2's `k' = k` reverse-witness rebuttal under the reverse witness's case (ii), to supply `n < n + 1` (resp. `m < m + 1`) so that NAT-order's trichotomy can refute `n + 1 ≤ n` (resp. `m + 1 ≤ m`) — the inequality produced by substituting the sub-case's `k = n + 1` (resp. `k = m + 1`) into the bound `k ≤ min(m, n) ≤ n` (resp. `k ≤ m`); part (b) Trichotomy, Case 2's `k' < k` reverse-witness rebuttal under the reverse witness's case (ii), to supply `n < n + 1` (resp. `m < m + 1`) so that NAT-order's trichotomy can refute `n + 1 < n` (resp. `m + 1 < m`) — the strictly sharper inequality produced by substituting the sub-case's `k' = n + 1` (resp. `k' = m + 1`) into the agreement bound `k' < k ≤ min(m, n)`, which yields `k' < n` (resp. `k' < m`); part (b) Trichotomy, Case 3's `m < n` branch reverse-witness rebuttal under the reverse witness's case (ii), to supply `n < n + 1` so that — via NAT-order's transitivity in the strict disjunct `n + 1 < m`, or via substitution of `n + 1 = m` into `n < n + 1` in the equality disjunct, of the unpacked rebuttal hypothesis `n + 1 ≤ m` (the case-(ii) requirement that `b` be a proper prefix of `a`) — the strict comparison `n < m` is derived; against the case hypothesis `m < n` this is then foreclosed by NAT-order's trichotomy at the length pair `(m, n)` (the mutual-exclusivity form of the same invocation that opened Case 3); part (b) Trichotomy, Case 3's `n < m` branch reverse-witness rebuttal under the reverse witness's case (ii), to supply `m < m + 1` so that — via NAT-order's transitivity in the strict disjunct `m + 1 < n`, or via substitution of `m + 1 = n` into `m < m + 1` in the equality disjunct, of the unpacked rebuttal hypothesis `m + 1 ≤ n` (the case-(ii) requirement that `a` be a proper prefix of `b`) — the strict comparison `m < n` is derived; against the case hypothesis `n < m` this is then foreclosed by the same length-pair trichotomy; part (c) Transitivity, Case `k₂ < k₁` under the hypothetical Case (ii) of `b < c`, to derive `k₂ = n + 1 > n` and chain (via NAT-order's transitivity) to a contradiction with `k₂ < k₁`; part (c) Transitivity, sub-case (i, ii), to supply `n < n + 1` so that substituting the sub-case's `k = n + 1` yields `n < k` (equivalently `k > n`), which together with the case (i) requirement `k ≤ min(m, n) ≤ n` produces the conjunction `k ≤ n ∧ k > n` that NAT-order's trichotomy then forecloses; and part (c) Transitivity, sub-case (ii, ii), to supply `m < m + 1` which — via NAT-order's transitivity in the strict disjunct `m + 1 < n` of the unpacked hypothesis `m + 1 ≤ n`, or via substitution of `m + 1 = n` into `m < m + 1` in the equality disjunct — yields the strict comparison `m < n` that contradicts the `m = n` produced by NAT-cancel's right cancellation in the same sub-case. NAT-cancel (NatAdditionCancellation) — part (c) Transitivity, sub-case (ii, ii), invokes right cancellation at the fixed summand `1` to pass from `m + 1 = n + 1` to `m = n`; this successor-injectivity step is what produces the equality used to contradict the strict comparison derived in the same sub-case. NAT-discrete (NatDiscreteness) — the proof invokes the forward direction `m < n ⟹ m + 1 ≤ n` (the contrapositive form of discreteness, that no natural number lies strictly between `m` and `m + 1`) at three distinct sites: part (b) Trichotomy, Case 3's `m < n` branch, uses the forward direction `m < n ⟹ m + 1 ≤ n` to pass from the case hypothesis `m < n` to `k = m + 1 ≤ n` — the successor-comparison step that underwrites "`a` is a proper prefix of `b`" and therefore T1 case (ii)'s witness for `a < b`; part (b) Trichotomy, Case 3's `n < m` branch, uses the mirrored instance `n < m ⟹ n + 1 ≤ m` to pass from the case hypothesis `n < m` to `k = n + 1 ≤ m` — the symmetric successor-comparison step that underwrites "`b` is a proper prefix of `a`" and therefore T1 case (ii)'s witness for `b < a` (the two branches instantiate the forward direction at distinct symbols, which under the uniform per-instance convention declared above earns them separate sites, paralleling the per-branch counting of the reverse-witness rebuttals under NAT-addcompat and NAT-order-transitivity); and part (c) Transitivity, Case `k₂ < k₁` under the case-(ii) branch of `a < b`, uses the forward direction in its contrapositive form `k₂ < m + 1 ⟹ k₂ ≤ m` to translate the narrative step "`a` has components at all positions below `k₁`" — which reduces in this branch to `k₂ < k₁ = m + 1 ⟹ k₂ ≤ m` since `k₁ = m + 1` by case (ii) of `a < b` — into the bound `k₂ ≤ m` required for position `k₂`'s T1 case-(i) witness for `a < c` to satisfy `k₂ ≤ min(m, p)` (the sibling case-(i) branch of `a < b`, where `k₁ ≤ min(m, n) ≤ m`, yields `k₂ ≤ m` from `k₂ < k₁ ≤ m` via NAT-order alone, so the discreteness appeal is specific to the case-(ii) branch). The backward direction `m + 1 ≤ n ⟹ m < n` is derivable from NAT-addcompat's strict successor inequality together with NAT-order's transitivity (in the strict disjunct `m + 1 < n` of the unpacked hypothesis) or substitution (in the equality disjunct `m + 1 = n`), and is not itself a NAT-discrete invocation; sub-case (ii, ii)'s derivation of `m < n` from `m + 1 ≤ n` is therefore routed through NAT-addcompat + NAT-order rather than NAT-discrete. NAT-wellorder (NatWellOrdering) — part (b) Trichotomy defines the *first divergence position* `k` as the least positive integer at which `a` and `b` disagree, invoking the well-ordering of ℕ applied to the subset `{i ∈ ℕ : 1 ≤ i ≤ min(m, n) ∧ aᵢ ≠ bᵢ} ∪ ({min(m, n) + 1} when m ≠ n, else ∅) ⊆ ℕ` — nonempty in Cases 2 and 3 by construction (Case 1 covers the empty-set situation, where `m = n` and `aᵢ = bᵢ` for all `i`, and the proof reduces to equality by T3 without invoking a divergence position) — to supply the least element that the "first divergence position" names; Case 2's rebuttal "if `k' < k`, the minimality of `k` gives `a_{k'} = b_{k'}`" and Case 3's hypothesis that `aᵢ = bᵢ` for all `1 ≤ i ≤ min(m, n)` (which holds precisely because `k = min(m, n) + 1` was chosen minimal, so no smaller position belongs to the divergence set) both rest on this minimality, so the least-element claim is discharged from NAT-wellorder rather than left implicit. T3 (CanonicalRepresentation) — postcondition (b) requires the bridge from component-level agreement to tumbler equality: Case 1 invokes T3 to conclude `a = b` from `m = n` and `aᵢ = bᵢ` for all `i`; Case 2 invokes its reverse direction (tumblers that differ in any component are distinct) to conclude `a ≠ b` from `aₖ ≠ bₖ`, which is load-bearing because trichotomy's "exactly one" outcome requires ruling out `a = b` as a third concurrent option alongside the produced witness for `a < b` or `b < a`; Case 3 invokes its contrapositive to conclude `a ≠ b` from `m ≠ n`.
- *Postconditions:* (a) Irreflexivity — `(A a ∈ T :: ¬(a < a))`. (b) Trichotomy — `(A a,b ∈ T :: exactly one of a < b, a = b, b < a)`. (c) Transitivity — `(A a,b,c ∈ T : a < b ∧ b < c : a < c)`. (d) Non-strict order — `a ≤ b ⟺ a < b ∨ a = b`; `a ≥ b ⟺ b ≤ a`.

Nelson's assertion that the tumbler line is total — that two addresses are never incomparable — is architecturally load-bearing. Spans are defined as contiguous regions on the tumbler line: "A span in the tumbler line, represented by two tumblers, refers to a subtree of the entire docuverse." If two addresses were incomparable, the interval between them would be undefined, and the entire machinery of span-sets, link endsets, and content reference would collapse.
