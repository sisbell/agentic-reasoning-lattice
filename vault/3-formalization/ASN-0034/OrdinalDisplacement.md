### Ordinal displacement and shift

**OrdinalDisplacement (OrdinalDisplacement).** For natural number n ≥ 1 and depth m ≥ 1, the *ordinal displacement* δ(n, m) is the tumbler [0, 0, ..., 0, n] of length m — zero at positions 1 through m − 1, and n at position m. Its action point is m.

Since δ(n, m) is a finite sequence of length m ≥ 1 over ℕ, it satisfies the carrier set criterion, so δ(n, m) ∈ T by T0; T0's "each component ∈ ℕ" clause is discharged at the m-th position by the precondition `n ≥ 1` placing `n ∈ ℕ`, and at the leading positions 1..m−1 by NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)`, which quantifies over ℕ and thereby presupposes `0 ∈ ℕ` — the routing T0 itself prescribes for the `0 ∈ ℕ` *membership* of padding positions, and the same routing the sister proof TA-Pos applies at its analogous `0 ≤ tₖ` site.

The precondition `n ≥ 1` promotes to `n ≠ 0` by a strict-from-`≥` chain, the same pattern the sister proofs TumblerAdd and TA-Pos discharge at their analogous sites. NAT-addcompat's strict successor inequality `(A n ∈ ℕ :: n < n + 1)` instantiated at `n = 0` supplies `0 < 0 + 1`, i.e., `0 < 1`; NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = 1, n = n` unfolds `n ≥ 1` (i.e., `1 ≤ n`) into the disjunction `1 < n ∨ 1 = n`; in the first disjunct NAT-order's transitivity of `<` composes `0 < 1` with `1 < n` to yield `0 < n`, and in the second disjunct substitution of `n = 1` into `0 < 1` yields `0 < n` directly; either way `0 < n`, whence `n ≠ 0` by NAT-order's irreflexivity — the contrary assumption `n = 0` would specialise `0 < n` to `0 < 0`, contradicting `¬(n < n)`. The promotion is required in two places below: TA-Pos's positivity predicate `(E i : tᵢ ≠ 0)` requires δₘ ≠ 0 as its witness, and ActionPoint's minimum-position set `{i : δᵢ ≠ 0}` requires δₘ ≠ 0 to contain m.

Since δ(n, m)ₘ = n and `n ≠ 0` as just established, the m-th component of δ(n, m) is nonzero, whence Pos(δ(n, m)) by TA-Pos. By ActionPoint, actionPoint(δ(n, m)) = min({i : 1 ≤ i ≤ m ∧ δ(n, m)ᵢ ≠ 0}); since δ(n, m)ᵢ = 0 for 1 ≤ i < m and δ(n, m)ₘ = n ≠ 0, the set `{i : 1 ≤ i ≤ m ∧ δ(n, m)ᵢ ≠ 0}` equals the singleton `{m}`, whose minimum is m. ∎

When the depth is determined by context (typically m = #v for the tumbler being shifted), we write δₙ.

*Formal Contract:*
- *Preconditions:* n ≥ 1, m ≥ 1
- *Definition:* δ(n, m) = [0, 0, …, 0, n] of length m
- *Depends:* T0 (CarrierSetDefinition) — invoked to discharge `δ(n, m) ∈ T` ("Since δ(n, m) is a finite sequence of length m ≥ 1 over ℕ, it satisfies the carrier set criterion, so δ(n, m) ∈ T by T0"), where T0's three-part membership characterisation of T as the set of finite sequences over ℕ with length ≥ 1 supplies the carrier-set criterion. NAT-zero (NatZeroMinimum) — invoked to supply `0 ∈ ℕ` for the m − 1 leading zero components of δ(n, m) at positions 1..m−1, required by T0's "each component ∈ ℕ" clause within the carrier-set criterion. NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` quantifies `0` as a lower bound of ℕ and thereby presupposes `0 ∈ ℕ`, which is the routing T0 itself explicitly prescribes for the `0 ∈ ℕ` *membership* of padding positions (as distinct from the algebraic identity `0 + n = n`, which T0 reserves to NAT-closure). No other listed NAT-* axiom supplies this membership: NAT-closure's additive identity clause is an equation over ℕ that presupposes, rather than supplies, `0 ∈ ℕ`; NAT-order's trichotomy compares naturals without fixing `0 ∈ ℕ`; and NAT-wellorder names a least element of ℕ without identifying it as `0`. The m-th component's membership `n ∈ ℕ` is supplied directly by the precondition `n ≥ 1` — no NAT-zero citation is required there — so this citation discharges only the m − 1 padding-position zeros, matching the per-step citation discipline the sister proof TA-Pos enforces at its analogous `0 ≤ tₖ` site. NAT-order (NatStrictTotalOrder) — invoked in the `n ≥ 1 ⟹ n ≠ 0` promotion that both the TA-Pos discharge and the ActionPoint discharge consume as their premise: TA-Pos's positivity predicate `(E i : tᵢ ≠ 0)` requires the witness `δₘ = n ≠ 0`, and ActionPoint's minimum-position set `{i : δᵢ ≠ 0}` requires `δₘ ≠ 0` to contain m. Three NAT-order clauses participate in the chain: (i) the defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = 1, n = n` unfolds the precondition `n ≥ 1` (i.e., `1 ≤ n`) into the disjunction `1 < n ∨ 1 = n` that the case analysis consumes; (ii) the transitivity of `<` composes `0 < 1` (supplied by NAT-addcompat) with `1 < n` into `0 < n` in the first disjunct, while the second disjunct closes by substituting `n = 1` into `0 < 1`; (iii) the irreflexivity `¬(n < n)` excludes `n = 0` from `0 < n`, since the contrary assumption would specialise `0 < n` to `0 < 0`. No other listed NAT-* axiom discharges any of these steps: NAT-zero's lower bound `0 ≤ n` holds at every `n ∈ ℕ` including `n = 0` and so cannot exclude `n = 0`; NAT-discrete's `m ≤ n < m + 1 ⟹ n = m` concerns gaplessness rather than the `≤`/`<`/`≠` interface; NAT-wellorder names ℕ's least element without naming it `0`. This matches the per-step discipline the sister proofs TumblerAdd (at its strict-advancement chain `rₖ = aₖ + wₖ ≥ aₖ + 1 > aₖ`) and TA-Pos (at its Case `#z ≥ k` step `0 ≤ tₖ ∧ tₖ ≠ 0 ⟹ 0 < tₖ`) enforce at their structurally identical strict-from-`≥` promotion sites. NAT-addcompat (NatAdditionOrderAndSuccessor) — invoked in the same `n ≥ 1 ⟹ n ≠ 0` promotion to supply the anchor inequality `0 < 1`. The strict successor inequality clause `(A n ∈ ℕ :: n < n + 1)` instantiated at `n = 0` yields `0 < 0 + 1`, i.e., `0 < 1`; NAT-order's transitivity then composes this with `1 < n` in the `1 < n` disjunct (and substitution handles the `1 = n` disjunct) to conclude `0 < n`, from which NAT-order's irreflexivity delivers `n ≠ 0`. Without this anchor the chain has no starting inequality: NAT-order's `≤` unfolding reduces `n ≥ 1` to `1 < n ∨ 1 = n` but supplies no fact about the comparison of `0` and `1` themselves; NAT-zero's `0 ≤ n` instantiated at `n = 1` yields only the non-strict `0 ≤ 1`, whose strict promotion to `0 < 1` would itself require ruling out the equality `0 = 1`, which is the same `≠ 0`-flavoured content the chain aims to establish — a circularity. The strict successor inequality is the only listed axiom asserting a strict comparison between `0` and a positive natural at a fixed point, matching the routing T0(a) applies at `M + 1 > M`, T1 applies at `m < m + 1`, and TumblerAdd applies at the `aₖ + 1 > aₖ` step of its strict-advancement chain. TA-Pos (PositiveTumbler) — invoked to discharge `Pos(δ(n, m))` ("Since δ(n, m)ₘ = n and `n ≠ 0` as just established, the m-th component of δ(n, m) is nonzero, whence Pos(δ(n, m)) by TA-Pos"), where TA-Pos's positivity predicate `(E i : tᵢ ≠ 0)` is witnessed at i = m by the nonzero m-th component δₘ = n ≠ 0 — the non-zeroness supplied by the `n ≥ 1 ⟹ n ≠ 0` promotion discharged above from NAT-addcompat (anchor `0 < 1`) and NAT-order (`≤` unfolding, transitivity of `<`, irreflexivity). ActionPoint (ActionPoint) — invoked to discharge `actionPoint(δ(n, m)) = m` ("By ActionPoint, actionPoint(δ(n, m)) = min({i : 1 ≤ i ≤ m ∧ δ(n, m)ᵢ ≠ 0}); since δ(n, m)ᵢ = 0 for 1 ≤ i < m and δ(n, m)ₘ = n ≠ 0, the set `{i : 1 ≤ i ≤ m ∧ δ(n, m)ᵢ ≠ 0}` equals the singleton `{m}`, whose minimum is m"), where ActionPoint's defining minimum-position formula is computed against the explicit component pattern of δ — membership of m in `{i : δ(n, m)ᵢ ≠ 0}` supplied by the `n ≥ 1 ⟹ n ≠ 0` promotion discharged above from NAT-addcompat and NAT-order, and exclusion of positions 1..m−1 supplied by δ's leading-zero construction.
- *Postconditions:* δ(n, m) ∈ T (by T0), Pos(δ(n, m)) (by TA-Pos), actionPoint(δ(n, m)) = m (by ActionPoint)
