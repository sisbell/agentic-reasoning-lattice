**TA5a (IncrementPreservesT4).** Let `zeros(t) = #{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` denote the count of zero-valued components of `t` (defined in T4). The operation `inc(t, k)` on a valid address `t` preserves T4 if and only if `k = 0`, or `k = 1` with `zeros(t) ≤ 3`, or `k = 2` with `zeros(t) ≤ 2`. For all `k ≥ 3`, T4 is violated regardless of `zeros(t)`.

*Proof.* Let `t` be a valid address satisfying T4, and let `t' = inc(t, k)`. Write `zeros(t) = #{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` for the count of zero-valued components (T4). We determine when `t'` satisfies T4. Recall that T4 requires four positional conditions: (i) `zeros(t) ≤ 3`, (ii) no two zeros adjacent, (iii) `t₁ ≠ 0`, and (iv) `t_{#t} ≠ 0` — with (ii), (iii), and (iv) together forming the field-segment constraint, equivalent to the requirement that every field segment of `t` is non-empty (T4a). Positivity of non-zero components is not a separate T4 clause; it is supplied by T0's carrier ℕ together with NAT-zero (`(A n ∈ ℕ :: 0 ≤ n)`, zero as lower bound of ℕ) and NAT-discrete (`(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)`, instantiated at `m = 0`): T0 fixes the carrier so each `tᵢ ∈ ℕ`, NAT-zero supplies the lower bound `0 ≤ tᵢ`, and NAT-discrete rules out the interval `0 ≤ tᵢ < 1` under `tᵢ ≠ 0`, so a non-zero component is strictly positive on ℕ — the per-step sourcing of "non-zero ⇒ strictly positive on ℕ" already applied in T4a's opening, T4c's exhaustion step, TA5-SigValid, and TA5, not a reliance on T0's carrier alone. Two groups of constraints must hold simultaneously: the zero-count bound and the field-segment constraint. We examine each value of `k`.

*Case `k = 0` (sibling increment).* By TA5(c), `t'` has the same length as `t` and differs only at position `sig(t)`, where `t'_{sig(t)} = t_{sig(t)} + 1`. For `zeros(t') = zeros(t)` to hold, the pre-increment value `t_{sig(t)}` must itself be non-zero — otherwise the increment would replace a zero at position `sig(t)` with `0 + 1 = 1`, removing a zero and giving `zeros(t') = zeros(t) - 1`. On the unrestricted carrier this failure mode is real: for `t = (0, 0, 0)`, TA5-SIG's definition gives `sig(t) = #t = 3`, `t_{sig(t)} = 0`, so `inc(t, 0) = (0, 0, 1)` and `zeros(t') = 2 < 3 = zeros(t)`. Under the T4-validity precondition, `t_{sig(t)} ≠ 0` is sourced by the chain TA5-SigValid → T4: by TA5-SigValid, `sig(t) = #t`; by T4's boundary clause, `t_{#t} ≠ 0`; hence `t_{sig(t)} = t_{#t} ≠ 0`. The post-increment value `t_{sig(t)} + 1` is then non-zero as well: NAT-closure's closure of ℕ under successor places `t_{sig(t)} + 1 ∈ ℕ`, NAT-zero supplies the lower bound `0 ≤ t_{sig(t)}`, and NAT-addcompat's strict successor inequality `n < n + 1` for every `n ∈ ℕ` gives `t_{sig(t)} < t_{sig(t)} + 1`, so `t_{sig(t)} + 1 > t_{sig(t)} ≥ 0` and hence `t_{sig(t)} + 1 ≠ 0` — the per-step sourcing TA5 already applies for the structurally identical `t'_j = t_j + 1 > t_j` step, not an implicit appeal to ℕ-successor behaviour. Hence no zero components are added or removed at position `sig(t)`, and since all other positions are unchanged, `zeros(t') = zeros(t)`. No new adjacencies between components are created, since only one position changes and its new value is strictly greater than the old. Since `t` satisfies T4, so does `t'`. T4 is preserved unconditionally.

*Case `k = 1` (immediate child).* By TA5(d), `t'` extends `t` by one position: `#t' = #t + 1`, with `t'_{#t+1} = 1`. The range of separator positions `#t + 1` through `#t + k - 1 = #t` is empty, so no zero separators are introduced: `zeros(t') = zeros(t)`. The appended component `t'_{#t+1} = 1` is non-zero, flanked only by `t_{#t}` on the left, which is non-zero by T4's boundary clause `t_{#t} ≠ 0`; no adjacent zeros are created. The new last component `t'_{#t'} = 1` is non-zero, so the boundary condition `t'_{#t'} ≠ 0` holds; the field-segment constraint is preserved. T4 is preserved when the existing zero count is within bound: `zeros(t) ≤ 3`.

*Case `k = 2` (grandchild with one separator).* By TA5(d), `t'` extends `t` by two positions: `t'_{#t+1} = 0` (one field separator) and `t'_{#t+2} = 1` (the first child). Thus `zeros(t') = zeros(t) + 1`. The appended sequence is `[0, 1]`: the zero at position `#t + 1` is flanked by `t_{#t}` (non-zero, by T4's boundary clause `t_{#t} ≠ 0`) on the left and by `1` on the right, so no adjacent zeros are created. The new last component `t'_{#t+2} = 1` is non-zero, so the boundary condition `t'_{#t'} ≠ 0` holds; the field-segment constraint is preserved. T4 is preserved when `zeros(t') = zeros(t) + 1 ≤ 3`, i.e., when `zeros(t) ≤ 2`.

*Case `k ≥ 3`.* By TA5(d), the appended sequence has `k - 1` zero components at positions `#t + 1` through `#t + k - 1`, followed by `1` at position `#t + k`. Since `k ≥ 3 ≥ 1`, NAT-sub's conditional closure (`m ≥ n ⟹ m − n ∈ ℕ`) places `k - 1 ∈ ℕ`, and NAT-sub's monotonicity at subtrahend `p = 1` sharpens the hypothesis `3 ≤ k` to `2 = 3 - 1 ≤ k - 1` — the parallel ℕ-level sharpening to TA5's `k > 0 ⟹ k ≥ 1` step, there cited jointly as NAT-zero + NAT-discrete. Since `k - 1 ≥ 2`, at least two of these zeros are adjacent — specifically, positions `#t + 1` and `#t + 2` are both zero. Instantiating T4's no-adjacent-zeros clause `(A i : 1 ≤ i < #t' : ¬(t'ᵢ = 0 ∧ t'ᵢ₊₁ = 0))` at `i = #t + 1` therefore yields `¬(0 = 0 ∧ 0 = 0)`, which is false; T4 fails at this positional clause alone, with no recourse to field decomposition or level labelling. As a concrete witness: `inc([1], 3)` produces `[1, 0, 0, 1]`, and instantiating the same clause at `i = 2` gives `¬(t'₂ = 0 ∧ t'₃ = 0)`, which fails because `t'₂ = t'₃ = 0` — a purely positional violation read directly off the component sequence, exhibited without appealing to `fields(t')` (T4b's decomposition is restricted to the T4-valid subdomain and so does not apply to `[1, 0, 0, 1]`) and without reading a hierarchical level off `zeros(t')` (T4c's level assignment is likewise restricted to T4-valid tumblers and assigns no level here). The zero count of `[1, 0, 0, 1]` is 2 (within the bound of 3), yet the adjacent-zero failure is independent of the zero-count clause. T4 is violated for all `k ≥ 3`.

The effective constraints are therefore: `k = 0` (always valid), `k = 1` (when `zeros(t) ≤ 3`), `k = 2` (when `zeros(t) ≤ 2`). The hierarchy enforces this naturally: each `inc(·, k)` with `k > 0` introduces one new hierarchical level, and the address format has exactly four fields with three separators, so at most three child-creation steps can be applied from a node address — three `inc(·, 2)` steps, with `zeros(t) = 0, 1, 2` respectively before each step, each satisfying `zeros(t) ≤ 2`. ∎

*Formal Contract:*
- *Precondition:* `t` satisfies T4 (valid address tumbler), `k ≥ 0`. `zeros(t) = #{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` (T4).
- *Depends:* T4 (HierarchicalParsing) — invoked at the case-analysis preface to fix the four positional clauses being checked ("T4 requires four positional conditions: (i) `zeros(t) ≤ 3`, (ii) no two zeros adjacent, (iii) `t₁ ≠ 0`, and (iv) `t_{#t} ≠ 0`"), at every case to verify each clause survives the increment, at case `k = 0` for the boundary clause `t_{#t} ≠ 0` ("by T4's boundary clause, `t_{#t} ≠ 0`") composed via TA5-SigValid's `sig(t) = #t` to yield `t_{sig(t)} = t_{#t} ≠ 0` — the non-zeroness of the pre-increment value at position `sig(t)` that licenses the step "no zero components are added or removed at position `sig(t)`" and hence `zeros(t') = zeros(t)`, without which the sibling increment on the unrestricted carrier could strip a zero (witnessed by `t = (0, 0, 0)`: `sig(t) = #t = 3`, `inc(t, 0) = (0, 0, 1)`, `zeros(t') = 2 < 3 = zeros(t)`), at the closing of cases `k = 1` and `k = 2` for the boundary clauses `t_{#t} ≠ 0` ("by T4's boundary clause `t_{#t} ≠ 0`") and zero-count bound, and at case `k ≥ 3` as the sole source of the violation: the no-adjacent-zeros clause `(A i : 1 ≤ i < #t' : ¬(t'ᵢ = 0 ∧ t'ᵢ₊₁ = 0))` is the specific positional clause whose instantiation at `i = #t + 1` fails once TA5(d) places two zeros at positions `#t + 1` and `#t + 2`, and the concrete witness `inc([1], 3) = [1, 0, 0, 1]` exhibits the same failure at `i = 2`. T4a (SyntacticEquivalence) — invoked at the case-analysis preface to license the equivalence between the no-adjacent-zeros / boundary-non-zero clauses and the non-empty-field-segment formulation ("(ii), (iii), and (iv) together forming the field-segment constraint, equivalent to the requirement that every field segment of `t` is non-empty (T4a)"), and at the closing of case `k = 2` where the phrase "the field-segment constraint is preserved" summarises the per-clause checks under the equivalence supplied by T4a; without T4a the "field-segment constraint" name in these sites would be unsourced. Case `k ≥ 3` does not cite T4a: the failure there is exhibited against T4's positional no-adjacent-zeros clause directly, without passing through the field-segment reading. T0 (CarrierSetDefinition) — invoked at the case-analysis preface to fix the carrier as ℕ so that every component `tᵢ ∈ ℕ` ("T0 fixes the carrier so each `tᵢ ∈ ℕ`"); T0 supplies the underlying set on which the NAT-* axioms speak but does not on its own license the step from non-zero to strictly positive. NAT-zero (NatZeroMinimum) — invoked at the case-analysis preface to supply the lower bound `0 ≤ tᵢ` required to instantiate NAT-discrete at `m = 0` for the "non-zero ⇒ strictly positive on ℕ" inference ("NAT-zero supplies the lower bound `0 ≤ tᵢ`"), and at case `k = 0` to supply `0 ≤ t_{sig(t)}` ("NAT-zero supplies the lower bound `0 ≤ t_{sig(t)}`") which, combined with NAT-addcompat's strict successor inequality, yields `t_{sig(t)} + 1 > t_{sig(t)} ≥ 0` and hence `t_{sig(t)} + 1 ≠ 0` — the non-zeroness of the post-increment value that sustains `zeros(t') = zeros(t)`; without NAT-zero the preface's interval `0 ≤ tᵢ < 1` could not be formed and case `k = 0` would have no sourced lower bound for the chained-inequality conclusion `t_{sig(t)} + 1 > 0`. Matches the per-step citation convention already applied in T4a's opening, T4c's exhaustion step, TA5-SigValid, and TA5. NAT-discrete (NatDiscreteness) — invoked at the case-analysis preface jointly with NAT-zero to convert the lower bound `0 ≤ tᵢ` together with `tᵢ ≠ 0` into strict positivity `tᵢ ≥ 1` ("NAT-discrete rules out the interval `0 ≤ tᵢ < 1` under `tᵢ ≠ 0`"); without NAT-discrete the equivalence "non-zero ⇔ strictly positive" on ℕ would fail (the reals ℝ admit `0 < r < 1` and satisfy NAT-zero but not NAT-discrete), and the preface's positivity attribution would revert to the T0-only form the ASN has moved away from. Matches the per-step citation convention established in T4a, T4c, TA5-SigValid, and TA5 for this same inference. NAT-closure (NatArithmeticClosure) — invoked at case `k = 0` to confirm that the post-increment value `t_{sig(t)} + 1` lies in ℕ via closure of ℕ under successor ("NAT-closure's closure of ℕ under successor places `t_{sig(t)} + 1 ∈ ℕ`"); without NAT-closure the post-increment component would not be secured as a natural, and the subsequent non-zero claim would speak about a quantity outside T0's carrier. Matches TA5's citation of NAT-closure for the structurally identical sibling-construction step `t'_{sig(t)} = t_{sig(t)} + 1`. NAT-addcompat (NatAdditionOrderCompatibility) — invoked at case `k = 0` to supply the strict successor inequality `n < n + 1` for every `n ∈ ℕ` at the step `t_{sig(t)} < t_{sig(t)} + 1` ("NAT-addcompat's strict successor inequality `n < n + 1`"); combined with NAT-zero's lower bound `0 ≤ t_{sig(t)}` this yields `t_{sig(t)} + 1 > 0`, hence non-zero. Matches TA5's citation of NAT-addcompat for the structurally identical sibling step `t'_j = t_j + 1 > t_j`, so TA5a's "successor of non-zero is non-zero" inference is discharged per-step rather than left as an implicit appeal to ℕ-successor behaviour. NAT-sub (NatPartialSubtraction) — invoked at case `k ≥ 3` to place `k - 1 ∈ ℕ` via conditional closure (since `k ≥ 3 ≥ 1`, so `k ≥ 1`) and to sharpen the hypothesis `3 ≤ k` to `2 ≤ k - 1` via monotonicity at subtrahend `p = 1` ("NAT-sub's monotonicity at subtrahend `p = 1` sharpens the hypothesis `3 ≤ k` to `2 = 3 - 1 ≤ k - 1`"); without NAT-sub the `k - 1` quantity appearing in TA5(d)'s appended-zero range would not be secured as a natural, and the ℕ-level sharpening needed to witness two adjacent zeros at positions `#t + 1` and `#t + 2` would consume unsourced arithmetic. Parallels the ℕ-level sharpening in TA5's Case `k > 0` (there cited as NAT-zero + NAT-discrete for the `k > 0 ⟹ k ≥ 1` step). TA5 (HierarchicalIncrement) — invoked at every case to extract the structural form of `inc(t, k)`: TA5(c) at case `k = 0` ("By TA5(c), `t'` has the same length as `t` and differs only at position `sig(t)`"), TA5(d) at case `k = 1` ("By TA5(d), `t'` extends `t` by one position"), at case `k = 2` ("By TA5(d), `t'` extends `t` by two positions") and at case `k ≥ 3` ("By TA5(d), the appended sequence has `k - 1` zero components at positions `#t + 1` through `#t + k - 1`"). TA5-SigValid (SigOnValidAddresses) — invoked at case `k = 0` to supply the equality `sig(t) = #t` ("by TA5-SigValid, `sig(t) = #t`") under the T4-validity precondition, which composed with T4's boundary clause `t_{#t} ≠ 0` yields `t_{sig(t)} = t_{#t} ≠ 0` — the non-zeroness of the pre-increment value at position `sig(t)` without which the sibling increment could remove a zero and give `zeros(t') = zeros(t) − 1` (witnessed on the unrestricted carrier by `t = (0, 0, 0)`: TA5-SIG gives `sig(t) = #t = 3`, `inc(t, 0) = (0, 0, 1)`, `zeros(t') = 2 < 3 = zeros(t)`), so the step "no zero components are added or removed at position `sig(t)`" is discharged from this chain rather than left as an implicit appeal to the T4-validity precondition, matching the per-step citation convention the ASN follows elsewhere for the structurally analogous "non-zero ⇒ strictly positive" inference.
- *Guarantee:* `inc(t, k)` satisfies T4 iff `k = 0`, or `k = 1 ∧ zeros(t) ≤ 3`, or `k = 2 ∧ zeros(t) ≤ 2`.
- *Failure:* For `k ≥ 3`, `inc(t, k)` violates T4 (adjacent zeros create an empty field).
