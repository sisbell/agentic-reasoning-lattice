**ActionPoint (ActionPoint).** For w ∈ T with Pos(w), the *action point* of w is actionPoint(w) = min({i : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0}). Since Pos(w), TA-Pos guarantees at least one component of w is nonzero, so the set {i : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0} is a nonempty subset of ℕ — nonempty by Pos(w)'s existential, a subset of ℕ because its elements are indices in the range 1 ≤ i ≤ #w — and NAT-wellorder's least-element principle supplies the least element that min names. The bound 1 ≤ actionPoint(w) ≤ #w follows because actionPoint(w) is a member of that set, and wᵢ = 0 for all 1 ≤ i < actionPoint(w) follows from the minimality of actionPoint(w): any index i with 1 ≤ i < actionPoint(w) and wᵢ ≠ 0 would be a smaller member of the set {i : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0}, contradicting that actionPoint(w) is the minimum — the lower bound 1 ≤ i is required so that wᵢ lies in T0's defined component-projection domain (T0 defines wᵢ ∈ ℕ only for 1 ≤ i ≤ #w), matching TA-Pos's analogous postcondition `(A i : 1 ≤ i ≤ #z : …)` which restricts the index explicitly. For the third postcondition w_{actionPoint(w)} ≥ 1: the definition of actionPoint(w) as the least index with a nonzero component supplies w_{actionPoint(w)} ≠ 0; NAT-zero furnishes 0 ≤ w_{actionPoint(w)} since w_{actionPoint(w)} ∈ ℕ, which NAT-order's definition m ≤ n ⟺ m < n ∨ m = n at m = 0, n = w_{actionPoint(w)} unfolds to 0 < w_{actionPoint(w)} ∨ 0 = w_{actionPoint(w)}; the construction's w_{actionPoint(w)} ≠ 0 excludes the equality disjunct, leaving 0 < w_{actionPoint(w)}; NAT-discrete's axiom m ≤ n < m + 1 ⟹ n = m in its contrapositive form m < n ⟹ m + 1 ≤ n — since the contrary assumption ¬(m + 1 ≤ n) gives n < m + 1 by NAT-order's trichotomy, combined with m < n yields m ≤ n < m + 1, whence NAT-discrete forces n = m, contradicting m < n by NAT-order's irreflexivity — instantiated at m = 0, n = w_{actionPoint(w)} yields 0 + 1 ≤ w_{actionPoint(w)}; NAT-closure's additive identity `0 + n = n` instantiated at `n = 1` rewrites the left-hand side `0 + 1` to `1`, yielding 1 ≤ w_{actionPoint(w)} — the same per-step rewrite TA-PosDom's Case `#z ≥ k` applies when NAT-closure's additive identity rewrites NAT-addcompat's strict-successor shape `0 < 0 + 1` to `0 < 1`, and TumblerAdd's dominance sub-case `aₖ > 0` applies when NAT-closure's additive identity `0 + wₖ = wₖ` rewrites the right-hand side of `aₖ + wₖ ≥ 0 + wₖ` to `wₖ`, rather than elided as a syntactic identity. ∎
