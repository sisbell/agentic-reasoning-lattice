## Tumbler subtraction

**TumblerSub (TumblerSub).** Given two tumblers `a` (minuend) and `w` (subtrahend), compute their component-wise difference at the first point of zero-padded divergence. The primary downstream use is computing displacements between positions — given positions `a` and `b` with `b ≥ a`, the subtraction `b ⊖ a` yields the component-wise difference from `a` to `b`. This difference is a broader object than a displacement consumable by TumblerAdd: when `a` is a proper sequence-prefix of `b`, the action point of `b ⊖ a` satisfies `actionPoint(b ⊖ a) = zpd(b, a) ≥ #a + 1 > #a`, violating TumblerAdd's precondition `actionPoint(w) ≤ #a`. A second obstruction is independent of the prefix case: when `#a > #b`, the Definition's NAT-order trichotomy dispatch (below) — applied to the length pair `(#b, #a)` of the operand pair of `b ⊖ a` — places the pair in its `#b < #a` sub-case, whose longer length `L = #a` furnishes the result length, so the subtraction produces `#(b ⊖ a) = #a`; TumblerAdd's result-length identity gives `#(a ⊕ (b ⊖ a)) = #a > #b`, so `a ⊕ (b ⊖ a) ≠ b` by T3 — the round-trip fails on length alone even when the zpd constraint is satisfied. The roundtrip `a ⊕ (b ⊖ a) = b` therefore requires two independently necessary constraints: `divergence(a, b) ≤ #a` (ensuring TumblerAdd's precondition) and `#a ≤ #b` (ensuring the result length equals `#b`). D0 (DisplacementWellDefined) takes the first as a precondition and proves the necessity of the second via its negative postcondition (`#a > #b → a ⊕ (b ⊖ a) ≠ b`); the positive round-trip identity under the full conjunction is established by D1 (DisplacementRoundTrip). When the operands have different lengths, we conceptually zero-pad the shorter to the length of the longer — writing `aᵢ = 0` for `i > #a` and `wᵢ = 0` for `i > #w`, with the literal `0` supplied as an element of ℕ by NAT-zero so that the padded values extend T0's ℕ-valued component-typing uniformly onto the padding region — and use these padded values throughout the procedure. Reproducing ZPD's length-pair dispatch for the same operand pair, NAT-order's trichotomy on `(#a, #w)` selects exactly one of the sub-cases (α) `#a = #w` with `L = #a` (equivalently `#w`), (β) `#a < #w` with `L = #w`, or (γ) `#w < #a` with `L = #a`; `L` — the longer of `#a` and `#w` — is furnished by the selected sub-case rather than by any primitive binary-maximum operator on ℕ, matching the convention that every operator in the Definition must be discharged by a Depends entry. When the zero-padded sequences agree at every position (no divergence exists), the result is the zero tumbler of length `L`: `a ⊖ w = [0, ..., 0]`. Otherwise, `zpd(a, w)` is defined (ZPD) — write `k = zpd(a, w)` for the first position at which the padded sequences disagree. The result is:

```
         ⎧ 0             if i < k        (these levels matched — zero them)
rᵢ   =  ⎨ aₖ - wₖ      if i = k        (subtract at the divergence point)
         ⎩ aᵢ           if i > k        (copy from minuend, zero-padded)
```

The result has length `L`.

**Precondition:** `a ≥ w` (T1). We prove that when `zpd(a, w)` is defined, this entails `aₖ > wₖ` at `k = zpd(a, w)`. Since zpd is defined, `a` and `w` are not zero-padded-equal (ZPD), so in particular `a ≠ w`; combined with `a ≥ w`, this yields `w < a` (T1). Two Divergence cases arise for the pair `(w, a)` with `w ≠ a`:

  (i) Component divergence at position `k` with `k ≤ #w ∧ k ≤ #a` (the conjunction replacing `k ≤ min(#w, #a)` so that no primitive binary-minimum operator on ℕ is invoked, matching Divergence's case (i)) and `wₖ ≠ aₖ`. The ZPD–Divergence relationship (ZPD) gives `zpd(a, w) = divergence(a, w) = k`. Since `w < a` via T1 case (i), `wₖ < aₖ`, whence `aₖ > wₖ`.

  (ii) Prefix divergence — `w` is a proper prefix of `a` (the direction forced by `w < a` via T1 case (ii)), so `#w < #a` and `wᵢ = aᵢ` for all `1 ≤ i ≤ #w`. The padded extension sets `wₖ = 0` for `k > #w`, with `0 ∈ ℕ` supplied by NAT-zero so that the equality is a well-formed ℕ-valued comparison. Since zpd is defined, the longer operand `a` has some nonzero component beyond `#w`; at `k = zpd(a, w)`, `aₖ ≠ 0 = wₖ`. The step `aₖ ≠ 0 ⟹ aₖ > 0` is discharged by NAT-zero's lower bound `0 ≤ aₖ` (instantiated at `n = aₖ`, with `aₖ ∈ ℕ` supplied by T0's component-typing), NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` at `m = 0, n = aₖ` unfolding `0 ≤ aₖ` to `0 < aₖ ∨ 0 = aₖ`, and the divergence observation `aₖ ≠ 0` excluding the equality disjunct to leave `0 < aₖ`; hence `aₖ > 0 = wₖ`.

In both cases `aₖ > wₖ` at `k = zpd(a, w)`. When zpd is undefined, the precondition `a ≥ w` continues to hold by hypothesis; the consequence `aₖ > wₖ` is vacuous because there is no `k = zpd(a, w)` at which to evaluate it, and the constructive definition produces the zero tumbler with no subtraction at a divergence point.  ∎

Each component of the result is a natural number: for `i < k`, `rᵢ = 0 ∈ ℕ` by NAT-zero; at the divergence point, `rₖ = aₖ − wₖ ∈ ℕ` by NAT-sub (conditional closure), whose precondition `aₖ ≥ wₖ` is discharged by the precondition consequence `aₖ > wₖ` at the zpd point (proved above) combined with NAT-order; for `i > k`, `rᵢ` is the zero-padded value of `a`, which is either `aᵢ ∈ ℕ` (when `i ≤ #a`) or `0 ∈ ℕ` by NAT-zero (when `i > #a`). In the equal case (no divergence), every component is `0 ∈ ℕ` by NAT-zero. The result is therefore a finite sequence over ℕ with length `L ≥ 1` — since `a, w ∈ T` requires both `#a ≥ 1` and `#w ≥ 1`, and `L` is named by the trichotomy dispatch as one of `#a` or `#w`, each of which is at least 1 — hence **`a ⊖ w ∈ T`** by T0.

When `zpd(a, w)` is defined — write `k = zpd(a, w)` — the result is positive. Components before position `k` are zero by construction, and `rₖ = aₖ − wₖ ≥ 1` by NAT-sub (strict positivity), whose precondition `aₖ > wₖ` is the precondition consequence proved above. The result is therefore not the zero tumbler, whence **`Pos(a ⊖ w)`** (TA-Pos). By ActionPoint, `actionPoint(a ⊖ w) = min({i : 1 ≤ i ≤ #(a ⊖ w) ∧ rᵢ ≠ 0}) = k`, since `rᵢ = 0` for `i < k` and `rₖ ≥ 1`: **`actionPoint(a ⊖ w) = zpd(a, w)`**. When `zpd(a, w)` is undefined, the result is the zero tumbler and neither conclusion holds.

*Formal Contract:*
- *Preconditions:* a ∈ T, w ∈ T, a ≥ w (T1). Consequence: when zpd(a, w) is defined, aₖ > wₖ at k = zpd(a, w).
- *Depends:* T0 (CarrierSetDefinition) — membership `a ⊖ w ∈ T` is concluded via T0's characterisation of T as finite sequences over ℕ with length ≥ 1. T1 (LexicographicOrder) — the precondition `a ≥ w` is a T1 ordering; the precondition consequence derives `w < a` from `a ≥ w ∧ a ≠ w` via T1 trichotomy. T3 (CanonicalRepresentation) — invoked at two distinct sites for two distinct roles, each consuming a different direction of T3's biconditional characterisation of tumbler equality as componentwise sequence equality. (i) *Precondition-consequence's `not-padded-equal ⟹ a ≠ w` step*: the proof concludes `a ≠ w` from the premise that `a` and `w` are not zero-padded-equal (derived from zpd being defined, per ZPD) before feeding the inequality to T1 trichotomy; the contrapositive of T3's forward direction supplies this inference — if `a = w` as tumblers, then by T3 they share both length and every component, so their zero-padded extensions (formed by appending the literal `0` at positions beyond each operand's native length, a construction determined entirely by length and components) coincide at every position, contradicting the "not zero-padded-equal" fact. Without T3 there is no basis for transporting the tumbler-level equality `a = w` to componentwise identity of the padded sequences, matching the per-step convention TA2 applies for the structurally identical `a ≠ w` derivation. (ii) *Motivation's length-difference-to-`≠` step*: the result-length identity `#(a ⊕ (b ⊖ a)) = #a > #b` (from TumblerAdd under `#a > #b`) witnesses `#(a ⊕ (b ⊖ a)) ≠ #b`; T3's reverse direction — equal tumblers have equal lengths (Leibniz's law applied to the length function), so distinct lengths imply distinct tumblers — concludes `a ⊕ (b ⊖ a) ≠ b` from this length discrepancy without recourse to componentwise comparison, discharging the `by T3` citation the motivation already makes by name. Divergence (Divergence) — the precondition consequence proceeds by case analysis on Divergence's two cases for the pair `(w, a)`. ZPD (ZPD) — defines `zpd(a, w)` and supplies the ZPD–Divergence relationship identifying `zpd(a, w) = divergence(a, w) = k` in case (i). TA-Pos (PositiveTumbler) — defines `Pos` for the conditional postcondition `Pos(a ⊖ w)`. ActionPoint (ActionPoint) — supplies the action-point identification for the conditional postcondition `actionPoint(a ⊖ w) = zpd(a, w)`. NAT-sub (NatPartialSubtraction) — conditional closure discharges `rₖ = aₖ − wₖ ∈ ℕ` at the divergence point (under `aₖ ≥ wₖ`, supplied by the precondition consequence), and strict positivity discharges `rₖ ≥ 1` under the strict form `aₖ > wₖ`; without NAT-sub the definition's divergence-point component would use partial subtraction on ℕ as an unsourced primitive. NAT-zero (NatZeroMinimum) — the literal `0` appears at four ASN-internal sites in TumblerSub, each requiring `0 ∈ ℕ` so that the construction respects T0's component-typing requirement — projected components are ℕ-valued — and the ℕ-valued (in)equalities the proofs invoke are well-formed. (i) The Definition's operand zero-padding stipulation `aᵢ = 0` for `i > #a` and `wᵢ = 0` for `i > #w` extends T0's component projection from its native domains `{1, ..., #a}` and `{1, ..., #w}` onto the extended index range `{1, ..., L}` consumed by the piecewise formula; this stipulation names the operands `a`, `w` directly and is therefore a TumblerSub-internal construction separate from ZPD's padded projections `â`, `ŵ` (whose NAT-zero citation is scoped to ZPD's own Depends and does not propagate into TumblerSub through ZPD's postcondition consumption). (ii) The Definition's `rᵢ = 0` clause for `i < k` assigns the literal `0` to every result component strictly below the divergence point, each requiring membership in ℕ so that the membership proof's claim `rᵢ = 0 ∈ ℕ` is well-formed and the result is ℕ-valued throughout positions `1, ..., k - 1`. (iii) The no-divergence branch `a ⊖ w = [0, …, 0]` constructs the zero tumbler of length `L` entirely from the literal `0`; the membership proof's closing observation that every component is `0 ∈ ℕ` in this branch requires `0 ∈ ℕ` at every position. (iv) The precondition-consequence's case (ii) padded extension sets `wₖ = 0` at `k > #w` and underwrites the equality `aₖ ≠ 0 = wₖ` and the strict inequality `aₖ > 0 = wₖ` — both are ℕ-valued comparisons, well-formed only when `0 ∈ ℕ`; the same case (ii) step additionally invokes NAT-zero's lower-bound axiom `(A n ∈ ℕ :: 0 ≤ n)` at `n = aₖ` to supply the premise `0 ≤ aₖ` consumed by NAT-order's defining-clause unfolding (see NAT-order's Depends entry) that derives `aₖ > 0` from `aₖ ≠ 0`, matching ActionPoint's third postcondition's use of the same lower-bound premise at `n = w_{actionPoint(w)}` in the structurally identical `≠ 0 ⟹ > 0` step before ActionPoint additionally composes with NAT-discrete to reach `≥ 1`; the TA-Pos postcondition proof's observation that components before position `k` are zero by construction is similarly grounded in ℕ-valued zeros. T0 supplies component-membership in ℕ only for *existing* positions `1 ≤ i ≤ #a`, `1 ≤ i ≤ #w`, so neither the padded values at positions `i > #a`, `i > #w` nor the literal-`0` components of the result are among T0's assertions; among the remaining NAT-* axioms in T0's exhaustive enumeration (NAT-closure, NAT-order, NAT-discrete, NAT-addcompat, NAT-wellorder, NAT-sub, NAT-cancel, NAT-addassoc) none identifies which elements belong to ℕ — each quantifies over ℕ presupposing its extension rather than fixing it. NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` quantifies `0` as a lower bound of ℕ and thereby presupposes `0 ∈ ℕ` as a member of its quantification domain, supplying exactly the missing membership premise at all four sites — this citation discharges the literal `0` from NAT-zero rather than leaving `0 ∈ ℕ` implicit, matching the per-step convention ZPD, TA-Pos, and ActionPoint already follow for discharging identical `0`-related premises in ℕ from NAT-zero rather than T0. NAT-order (NatStrictTotalOrder) — invoked at three distinct ℕ-pairs for three distinct roles: one trichotomy instance on a length pair, one defining-clause unfolding of `≤` as `< ∨ =` in the precondition-consequence's case (ii), and one defining-clause conversion of `>` to `≥` at the NAT-sub discharge. *Length pair `(#a, #w)` (internal to the Definition)*: trichotomy's exactly-one clause discharges the common-length selection, leaving exactly one of sub-case (α) `#a = #w` (`L = #a`), sub-case (β) `#a < #w` (`L = #w`), or sub-case (γ) `#w < #a` (`L = #a`); `L` — the longer of `#a` and `#w`, which furnishes the result length `#(a ⊖ w) = L` in the Definition, the Postconditions, and the `L ≥ 1` membership check that concludes `a ⊖ w ∈ T` — is named by the selected sub-case rather than by a primitive binary-maximum operator on ℕ, reproducing ZPD's analogous length-pair dispatch for the same operand pair `(a, w)` at which both definitions evaluate and matching the convention that every operator in the Definition must be discharged by a Depends entry. The same length-pair dispatch also names the longer length `#a` in the motivational observation `#(b ⊖ a) = #a` (under `#a > #b`) applied to the reversed operand pair of `b ⊖ a`. *Component pair `(0, aₖ)` (precondition-consequence's case (ii))*: the defining clause `m ≤ n ⟺ m < n ∨ m = n` at `m = 0, n = aₖ` unfolds NAT-zero's lower bound `0 ≤ aₖ` to `0 < aₖ ∨ 0 = aₖ`, after which the divergence observation's `aₖ ≠ 0` (from `aₖ ≠ 0 = wₖ` at `k = zpd(a, w)` where `wₖ = 0` by the prefix zero-padding) excludes the equality disjunct to leave `0 < aₖ`, i.e., `aₖ > 0`; this NAT-zero + NAT-order discharge matches the per-step convention ActionPoint's third postcondition applies for the structurally identical `≠ 0 ⟹ > 0` inference — ActionPoint then additionally composes with NAT-discrete to reach the stronger `≥ 1`, a step TumblerSub does not require at this site because the `≥ 1` result needed for the Pos postcondition at the divergence point is supplied instead by NAT-sub's strict-positivity clause (discharging `rₖ = aₖ − wₖ ≥ 1` directly from `aₖ > wₖ`). *Component pair `(aₖ, wₖ)` (internal to the NAT-sub discharge)*: converts the strict inequality `aₖ > wₖ` of the precondition consequence into the weak `aₖ ≥ wₖ` required by NAT-sub's conditional-closure clause, via the definition of `≤` from `<`. All three roles are enumerated together under this single NAT-order citation, matching the per-step convention ZPD and Divergence apply when a single axiom is invoked at multiple ℕ-pairs.
- *Definition:* NAT-order's trichotomy on `(#a, #w)` selects exactly one of: (α) `#a = #w`, `L = #a`; (β) `#a < #w`, `L = #w`; (γ) `#w < #a`, `L = #a`. a ⊖ w is computed by case analysis on k = zpd(a, w) (ZPD), all component references using zero-padded values (aᵢ = 0 for i > #a, wᵢ = 0 for i > #w); rᵢ = 0 for i < k, rₖ = aₖ − wₖ, rᵢ = aᵢ (zero-padded) for i > k; when zpd(a, w) is undefined, a ⊖ w = [0, …, 0]; #(a ⊖ w) = L (the longer length named by the trichotomy dispatch above, replacing `max(#a, #w)`).
- *Postconditions:* a ⊖ w ∈ T, #(a ⊖ w) = L (the longer of `#a` and `#w`, named by NAT-order trichotomy on `(#a, #w)` per the Definition: `L = #a` in sub-cases (α) and (γ), `L = #w` in sub-case (β)); when zpd(a, w) is defined: Pos(a ⊖ w) (TA-Pos), actionPoint(a ⊖ w) = zpd(a, w) (ActionPoint)
