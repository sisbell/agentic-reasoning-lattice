# Cone Review — ASN-0034/TA6 (cycle 7)

*2026-04-17 15:43*

### TA6 Depends discipline is inconsistent: T1 cited transitively through TA-Pos, but other transitive dependencies of TA-Pos are omitted
**Foundation**: TA6 (ZeroTumblers), Formal Contract *Depends*.
**ASN**: TA6's Depends entry for T1 explicitly flags the citation as transitive — "T1 (LexicographicOrder) — **invoked transitively through TA-Pos's postcondition for Conjunct 2**; TA-Pos's proof uses T1 case (i) at the first positive position of `t` and T1 case (ii) when `s` is a proper prefix of `t`, and Conjunct 2 inherits this case analysis without repeating it." TA6's Depends list otherwise enumerates T0, T4, TA-Pos, NAT-zero. It does not list T3, NAT-wellorder, NAT-discrete, or NAT-order.
**Issue**: If "invoked transitively through TA-Pos's postcondition" is the rule that puts T1 on TA6's Depends list, the same rule must also put T3 (TA-Pos's preamble invokes it for distinctness of zero tumblers of differing lengths — load-bearing for the postcondition's universal quantification over zero tumblers `z ∈ T` of arbitrary length, which TA6 instantiates with `s`), NAT-wellorder (TA-Pos invokes it to produce the least index `k` with `tₖ ≠ 0`), NAT-discrete (TA-Pos's Case `#z < k` uses the forward direction to derive `#z + 1 ≤ #t`), and NAT-order (TA-Pos's Case `#z ≥ k` uses `≤`'s definitional unfold and trichotomy) on that list too. Conversely, if TA6 cites only direct dependencies, then T1 itself does not belong — TA6's proof never unpacks T1's case definition; it treats `<` as a black-box relation returned by TA-Pos and uses T1 only to name the relation symbol in the conclusion `s < t`. Either discipline is internally coherent; mixing the two — listing T1 with a "transitive" justification while omitting the other transitive hops from the same sibling — leaves a downstream reader unable to tell which dependencies are actually load-bearing for TA6 and breaks the parity with TA-Pos, which lists all of T3/NAT-wellorder/NAT-discrete/NAT-order/NAT-zero itself.
**What needs resolving**: Adopt one discipline and apply it uniformly. Either (a) extend TA6's Depends to include the remaining transitive dependencies inherited from TA-Pos (T3, NAT-wellorder, NAT-discrete, NAT-order) with parallel "invoked transitively through TA-Pos" justifications, or (b) restrict TA6's Depends to direct dependencies only — remove T1 (whose sole role is to supply the relation symbol in TA-Pos's postcondition) and document that TA-Pos is the sole TA6 → (foundation graph) hop beyond T0, T4, and NAT-zero, with TA-Pos's own Depends list carrying the onward citations.
