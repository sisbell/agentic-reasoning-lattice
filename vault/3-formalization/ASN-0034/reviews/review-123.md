### TA5-SIG (LastSignificantPosition) lacks a Depends clause
**Foundation**: (foundation ASN — internal consistency; per-step citation convention)
**ASN**: TA5-SIG's Formal Contract reads only *"Preconditions: `t ∈ T` (any tumbler with `#t ≥ 1`). Definition: `sig(t) = max({i : 1 ≤ i ≤ #t ∧ tᵢ ≠ 0})` when `(E i : 1 ≤ i ≤ #t : tᵢ ≠ 0)`; `sig(t) = #t` when `(A i : 1 ≤ i ≤ #t : tᵢ = 0)`. Postconditions: `1 ≤ sig(t) ≤ #t`."* No Depends line appears. The definition, however, rests on T0 (carrier set T, length `#·`, component projection, ℕ-valued components underwriting the `tᵢ ≠ 0` test), NAT-wellorder (to assert `max(·)` exists as an element of a finite nonempty subset of ℕ — the dual of NAT-wellorder's least-element principle, which the ASN otherwise cites explicitly whenever `min(·)` is named, as in ActionPoint and Divergence), and T0 again for the `#t ≥ 1` lower bound that guarantees the fallback branch's `#t` is a well-defined index. Sister definitional properties carry Depends for exactly these appeals: ActionPoint cites T0 and NAT-wellorder; Prefix cites T0 and NAT-order; Divergence cites T3 and NAT-wellorder; TA5-SigValid cites T4, T0, and TA5-SIG.
**Issue**: The per-step citation convention that other definitional properties in this ASN uphold is not applied to TA5-SIG. A reviser tightening NAT-wellorder or T0 has no Depends-backed signal that TA5-SIG consumes those axioms, and a reader reconstructing the dependency graph has no entry point into `sig(·)`'s foundation. The missing Depends also leaves the `max` construction's well-definedness unsourced: `max({i : 1 ≤ i ≤ #t ∧ tᵢ ≠ 0})` is named without appeal to the axiom that guarantees it exists.
**What needs resolving**: TA5-SIG's Formal Contract needs a Depends clause matching the granularity used in ActionPoint, Prefix, and Divergence — naming the T0 operators it references and the NAT-wellorder step that underwrites `max`'s well-definedness — so the definition's foundation is visible to downstream audits.

### Definition (Span) lacks a Depends clause
**Foundation**: (foundation ASN — internal consistency)
**ASN**: Definition (Span)'s Formal Contract reads only *"Preconditions: `s ∈ T`, `ℓ ∈ T`, `Pos(ℓ)`, `actionPoint(ℓ) ≤ #s`. Definition: `span(s, ℓ) = {t ∈ T : s ≤ t < s ⊕ ℓ}`."* No Depends line appears. The preconditions invoke TA-Pos (the predicate `Pos(·)`) and ActionPoint (the action-point function). The definition uses `s ⊕ ℓ`, which is TumblerAdd's operator and whose well-definedness under the stated preconditions is exactly TA0's postcondition, and uses the relations `s ≤ t` and `t < s ⊕ ℓ`, which are T1 and T1(d)'s derived non-strict companion. Sister properties that reference the same operators carry Depends for each: T12 cites Definition (Span), T1, TA0, and TA-strict; TA0 cites T0, TA-Pos, TumblerAdd, and ActionPoint.
**Issue**: Definition (Span) names five other properties in its preconditions and definition body — TA-Pos, ActionPoint, TumblerAdd, TA0, T1 — without a Depends clause to record the dependence. The ASN's citation convention, applied uniformly elsewhere, makes the Depends line the mechanism by which a reviser tightening (say) TA0's preconditions can enumerate downstream consumers; Definition (Span) is invisible to that audit, and the span preconditions `Pos(ℓ)` and `actionPoint(ℓ) ≤ #s` — which Definition (Span) presents as its load-bearing conditions — cannot be traced back to the properties that supply them.
**What needs resolving**: Definition (Span) needs a Depends clause naming TA-Pos, ActionPoint, TumblerAdd, TA0, and T1, matching the granularity that T12 uses when citing Definition (Span). As written, the reader cannot mechanically resolve which property furnishes each symbol the definition consumes.

### TS5's Depends omits NAT-sub despite the proof computing subtraction on ℕ
**Foundation**: (foundation ASN — internal consistency; per-step NAT-* citation convention)
**ASN**: TS5 (ShiftAmountMonotonicity)'s proof opens: *"Define d = n₂ − n₁. Since n₂ > n₁ and both are natural numbers, d ≥ 1. And since n₁ ≥ 1, we have n₂ = n₁ + d with both n₁ ≥ 1 and d ≥ 1."* Two NAT-sub clauses are consumed. First, conditional closure (`m ≥ n ⟹ m − n ∈ ℕ`) licenses `n₂ − n₁ ∈ ℕ` under `n₂ ≥ n₁` (from `n₂ > n₁` via NAT-order). Second, strict positivity (`m > n ⟹ m − n ≥ 1`) licenses `d ≥ 1` from the hypothesis `n₂ > n₁`. The step `n₂ = n₁ + d` additionally uses NAT-sub's left-inverse characterisation `n + (m − n) = m`. TS5's Depends lists only *"TS3 (ShiftComposition) ... OrdinalShift (OrdinalShift) ... TS4 (ShiftStrictIncrease)"* — no NAT-sub citation, and neither TS3, TS4, nor OrdinalShift invokes partial subtraction, so the fact is not transitively supplied.
**Issue**: The ASN's stated convention (T0) is that each proof cites only the ℕ facts it actually uses. Sister proofs that write `m − n` at a divergence point cite NAT-sub by name — TumblerSub, D0, D1, D2, TA3, TA4, TA3-strict, TA-LC, TA7a, ReverseInverse, TA5a, among others. TS5 uses NAT-sub's conditional closure, strict positivity, and left-inverse characterisation all in the opening two sentences of its proof, yet no NAT-* axiom appears in its Depends. A reviser who tightens NAT-sub has Depends-backed visibility into every downstream consumer except TS5, leaving TS5's subtraction step tacitly reliant on background arithmetic.
**What needs resolving**: TS5's Depends must cite NAT-sub (and NAT-order for the strict-to-weak inequality step that supplies NAT-sub's precondition), matching the per-step convention the ASN applies to every other proof that writes `m − n` for `m, n ∈ ℕ`.
