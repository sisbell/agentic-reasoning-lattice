### T6 preamble quantifies over all `a, b ∈ T` while Preconditions restrict to T4-valid (and the proof only works on T4-valid inputs)
**Foundation**: T4b (UniqueParse), Postconditions — "`fields : T ⇀ Seq(ℕ⁺) × …` is a partial function whose domain is exactly the T4-valid subset of `T` … On tumblers `t ∈ T` that are not T4-valid … this corollary assigns no value to `fields(t)`, so consumers must carry T4-validity as a precondition before reading any projection."
**ASN**: T6 preamble — "For any two tumblers `a, b ∈ T`, the following are decidable from the addresses alone: (a) Whether `a` and `b` share the same node field. (b) …" T6 Formal Contract, Preconditions — "`a, b ∈ T` are valid tumblers satisfying T4 (at most three zeros, no adjacent zeros, no leading or trailing zero) and satisfying T3 (CanonicalRepresentation) …"
**Issue**: T6's informal statement asserts decidability "for any two tumblers `a, b ∈ T`" — the full carrier — but its Formal Contract restricts the claim to T4-valid inputs, and the proof body relies on that restriction at every step (Ingredient 1 extracts via `fields`, which is partial on the T4-valid subdomain; Ingredient 2 reads presence off `zeros(t)` thresholds that are only tied to presence within that subdomain; each case assumes `fields(a)`, `fields(b)` are values, not undefined). The gap between the preamble's universal "any two tumblers `a, b ∈ T`" and the contract's narrower "T4-valid" is not bridged anywhere: for a tumbler in `T` that fails T4's positional conditions (e.g., `[0, 1]` or `[1, 0, 0, 2]`), the procedure has no defined input to `fields`, so the preamble's decidability claim is not established for such tumblers. A formalization reading the preamble will infer universal decidability on `T`; reading the Preconditions it will infer decidability only on T4-valid pairs. Cross-cutting, these two scopes disagree about what T6 actually establishes.
**What needs resolving**: Either narrow the preamble so the universal scope matches the Precondition (e.g., "For any two T4-valid tumblers `a, b ∈ T` …"), or extend the proof so decidability genuinely holds on all of `T` — for example by observing that T4-validity is itself decidable from the component sequence and prefixing each procedure with a T4-validity check that returns NO (or a designated "undefined" result) on non-T4-valid inputs, with the extended postcondition stated accordingly.
