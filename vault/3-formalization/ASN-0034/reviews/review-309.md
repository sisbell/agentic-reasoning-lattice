### TA3-strict omits NAT-zero though the proof writes literal `0` in ℕ-valued comparisons and consumes zero-padded values

**Foundation**: N/A — internal consistency. The relevant anchors are the convention stated verbatim in TumblerSub's, ZPD's, TA2's, and ActionPoint's NAT-zero Depends entries ("the literal `0` appears at … sites, each requiring `0 ∈ ℕ` so that the componentwise membership claim … respects T0's ℕ-valued component-typing requirement and the ℕ-valued (in)equalities … are well-formed … T0 supplies component-membership in ℕ only for *existing* positions … so the literal `0` components … must be supplied separately"), and TumblerSub's Definition, which introduces the zero-padded operand values `aᵢ = 0` for `i > #a` and `wᵢ = 0` for `i > #w`.

**ASN**: TA3-strict, Case A: "`(a ⊖ w)_j = 0 < b_j - w_j = (b ⊖ w)_j`"; Case 3: "`(a ⊖ w)_{d_b} = 0` … The first disagreement is at `d_b` with `0 < b_{d_b} - w_{d_b}`"; Case 1 subcase `j > d`: "both results are … zero before position `j`" and "`(a ⊖ w)_i = 0` and `(b ⊖ w)_i = 0` for all `i < d`". Case A additionally consumes zero-padded operand values when `#w < #a` or `#w > #a`: "`a_i = w_i` (zero-padded equality)" and "`w_j = a_j` (zero-padded equality)" at positions where one of `a_i`, `w_i` is a padded `0` rather than a native component, and at the divergence point "`(b ⊖ w)_j = b_j - w_j`" reads `w_j` as the padded `0` when `j > #w`.

TA3-strict's Depends lists only T1, TumblerSub, ZPD, TA2, NAT-sub, and NAT-order.

**Issue**: Every strict comparison of the form `0 < …` in Cases A, 1, and 3 is a ℕ-valued inequality whose left operand is the literal `0`; for the comparison to be well-formed and to feed T1 case (i)'s component-disagreement witness, `0 ∈ ℕ` is required. The zero-padded-equality reasoning in Case A and the subtraction `b_j − w_j` at positions `j > #w` further consume the padded operand value `w_j = 0`, which is the same `0 ∈ ℕ` premise. Upstream ASNs (TumblerSub, ZPD, TA2, ActionPoint, TA-Pos) discharge this by citing NAT-zero — whose axiom `(A n ∈ ℕ :: 0 ≤ n)` quantifies over a domain containing `0` — and explicitly note that T0's component-typing covers only *existing* positions and that no other NAT-* axiom fixes membership. TA3-strict inherits the literal-`0` construction through TumblerSub but performs its own ℕ-valued comparisons with those zeros (the T1 case (i) witness comparisons at every case's first disagreement), yet does not cite NAT-zero. Under the ASN's stated convention that "each proof cites only the ℕ facts it actually uses", the NAT-zero premise TA3-strict directly consumes is unsourced.

**What needs resolving**: Either add NAT-zero to TA3-strict's Depends with a per-site enumeration of the literal-`0` usages (Case A's `0 < b_j - w_j` T1 witness and its zero-padded-equality consumption of `w_j = 0` at `j > #w`; Case 1 subcase `j > d`'s pre-divergence zero agreement; Case 3's `0 < b_{d_b} - w_{d_b}` T1 witness) matching the per-site convention TumblerSub, ZPD, TA2, and ActionPoint follow, or restructure the proof so every literal `0` is reached through a cited consumer (e.g., route every `0` claim through TumblerSub's postcondition without introducing a fresh ℕ-valued comparison in TA3-strict).

### TA3-strict omits T0 though the proof consumes component projection and the length operator directly

**Foundation**: N/A — internal consistency. T0 (CarrierSetDefinition) is the source for the length operator `#·` and for the component projection `xᵢ ∈ ℕ` on native domain `{1, ..., #x}`, as explicitly cited by TA2, TumblerSub, and ZPD, each of which consumes the same operators and treats T0 as a required Depends entry rather than inheriting it through downstream citations.

**ASN**: TA3-strict's preconditions read `#a = #b`; Case A argues at position `j ≤ #a` using component values `a_i`, `b_i`, `w_i`; Cases 1–3 read `d_a, d_b` against `#a`, `#b`, `#w` bounds and use native component values `a_{d_a}`, `b_{d_b}`, `w_{d_b}` in strict inequalities on ℕ; TA2 is invoked for the membership `a ⊖ w, b ⊖ w ∈ T`.

TA3-strict's Depends lists only T1, TumblerSub, ZPD, TA2, NAT-sub, and NAT-order.

**Issue**: The length operator `#·` and the native-domain component projection `xᵢ ∈ ℕ` are T0 operators — `#a = #b` in the preconditions is a T0 operator comparison, and every native component value consumed in the ℕ-valued strict inequalities (for example `a_{d_b} = w_{d_b}` in Case 3, `a_j < b_j` in Case A, `a_d < b_d` in Case 1 subcase `j = d`) is typed ℕ by T0's component-typing clause. TA1 through TA3 and the TumblerSub/ZPD/TA2 chain uniformly cite T0 directly when consuming these operators; the convention is the same "every operator in the Definition must be discharged by a Depends entry" stated in TumblerSub's Depends. TA3-strict inherits many derived facts through TA2, TumblerSub, and ZPD, but the length-comparison `#a = #b` in its own precondition and the native-component ℕ-valued comparisons it performs (not routed through a downstream consumer's postcondition) are not themselves discharged by any cited entry: TA2/TumblerSub/ZPD export facts *about* differences and divergence indices, not the component-typing of the original operands' native positions that TA3-strict reads off directly.

**What needs resolving**: Either add T0 to TA3-strict's Depends with a citation covering the length operator (used in the precondition `#a = #b` and in the bound `j ≤ #a`) and the native-domain component projection (used in Case A's `a_j`, `b_j`, and in every case's native-domain component comparison), matching the per-operator pattern TA2 and TumblerSub follow, or reformulate the proof so that every length and native-component consumption is routed through a cited postcondition of T1/TumblerSub/ZPD/TA2 rather than read off the original operands directly.
