### TumblerAdd's length derivation uses NAT-sub without citation
**Foundation**: (foundation ASN — internal consistency; per-step NAT-* citation convention)
**ASN**: TumblerAdd's body, establishing the result-length identity, reads: *"The result `a ⊕ w = [r₁, ..., rₚ]` has length `p = (k - 1) + 1 + (n - k) = n = #w`."* The identity `(k - 1) + 1 + (n - k) = n` is arithmetic on ℕ that consumes NAT-sub at three sites. (1) `k - 1 ∈ ℕ` rests on NAT-sub's conditional closure instantiated at `k ≥ 1` (from ActionPoint). (2) `n - k ∈ ℕ` rests on NAT-sub's conditional closure instantiated at `n ≥ k` (from ActionPoint's `k ≤ #w = n`). (3) Both collapses `(k - 1) + 1 = k` and `k + (n - k) = n` rest on NAT-sub's left-inverse characterisation `n + (m - n) = m`. This identity is load-bearing: it discharges the *result-length identity* `#(a ⊕ w) = #w`, which TumblerAdd promulgates as a postcondition and which TA0, Definition (Span), and every downstream consumer of `a ⊕ w`'s length relies on. TumblerAdd's Depends lists only *"T0 … NAT-closure … NAT-addcompat … NAT-wellorder … ActionPoint … TA-Pos … T1 … T3"* — no NAT-sub citation, and no cited dependency invokes partial subtraction, so the fact is not transitively supplied.
**Issue**: The ASN's per-step convention — applied in TumblerSub, D0, D1, D2, TA3, TA4, TA3-strict, TA-LC, TA7a, ReverseInverse, TA5a, and every other proof that writes `m − n` for `m, n ∈ ℕ` — requires each proof that invokes NAT-sub to cite it by name. TumblerAdd writes the subtraction identity `(k - 1) + 1 + (n - k) = n` explicitly in the body, consuming three distinct NAT-sub clauses (conditional closure twice, left-inverse twice), yet the Depends clause does not acknowledge NAT-sub. A reviser tightening NAT-sub has Depends-backed visibility into every downstream consumer except TumblerAdd, leaving the result-length identity's arithmetic — and transitively TA0's `#(a ⊕ w) = #w` postcondition and Span's upper-bound length — tacitly reliant on background arithmetic.
**What needs resolving**: TumblerAdd's Depends must cite NAT-sub (and enumerate the three consumption sites at the granularity the ASN applies to sister properties), or the body's length derivation must be restated so that it does not write `k - 1` or `n - k` — e.g., by reading `p = n` directly off the piecewise construction's enumeration of positions `1, ..., n` without a subtraction-based counting identity.
