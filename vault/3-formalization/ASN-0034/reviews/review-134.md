### Implicit zero-count uniformity across sibling streams
**Foundation**: N/A (foundation ASN — internal consistency)
**ASN**: GlobalUniqueness, Case 5's length-collision argument (subcase `(k'₁, k'₂) = (2, 1)`), and the exhaustiveness routing for `p₁ = p₂`. The text concludes: *"k'₁ = 2 introduces one field separator (TA5(d)), giving zeros(A₁ output) = zeros(p₁) + 1; k'₂ = 1 introduces no separator (TA5(d)), giving zeros(A₂ output) = zeros(p₂) = zeros(p₁). The zero counts differ by one, routing this pair to Case 4."* The exhaustiveness `p₁ = p₂` routing echoes this: *"one output carries an additional field separator zero (by the TA5(d) argument of Case 5), yielding different zero counts and routing the pair to Case 4."*
**Issue**: The phrase "zeros(A₁ output)" denotes the zero count of an **arbitrary** allocated address in `dom(A₁)`, not only the base `c₀ = inc(p₁, 2)`. TA5(d) cited here only fixes the zero count of the **base address** — it says nothing about siblings produced later by `inc(·, 0)`. Case 4 needs `zeros(a) ≠ zeros(b)` for the **specific** `a ∈ dom(A₁), b ∈ dom(A₂)` whose distinctness is at issue, so the Case 4 reduction silently requires a within-allocator zero-count uniformity: every `inc(·, 0)` step preserves the zero count. This fact is derivable (TA5(b) preserves positions other than `sig`; TA5(c) advances position `sig` by `+1`; TA5-SigValid + T10a.4 give `sig(t) = #t` with `t_{#t} > 0`, so the advanced position stays strictly positive and no zero is added or removed), but it is not stated or cited anywhere in the ASN. T10a's consequences list `T10a.1 (UniformSiblingLength)` for length uniformity but has no parallel zero-count uniformity consequence, and the justification parenthetical "(TA5(d))" at this site does not cover the sibling-stream extension from base to general output.
**What needs resolving**: Either add an explicit consequence of T10a asserting that within a single allocator's sibling stream, every output has the same zero count as its base (with the TA5(b)/TA5(c)/TA5-SigValid/T10a.4 chain as its Depends), or extend the Case 5 and exhaustiveness text to justify the base-to-stream extension at each use site so that the Case 4 routing relies on a stated fact rather than an implicit inductive argument.
