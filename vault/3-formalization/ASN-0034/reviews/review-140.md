### NAT-discrete over-attribution at Transitivity sub-case (ii, ii): backward direction doesn't require discreteness

**Foundation**: NAT-discrete (NatDiscreteness), NAT-addcompat (NatAdditionOrderCompatibility), NAT-order (NatStrictTotalOrder).

**ASN**: T1 (LexicographicOrder), part (c) Transitivity, sub-case (ii, ii): *"But the first condition requires `m + 1 ≤ n`, which by NAT-discrete's equivalence between strict and non-strict successor comparisons (`m + 1 ≤ n ⟺ m < n`) means `m < n`, contradicting `m = n`."* And T1's Depends for NAT-discrete, which enumerates this site as one of three: *"part (c) Transitivity, sub-case (ii, ii), uses the backward direction `m + 1 ≤ n ⟹ m < n`..."*

**Issue**: NAT-discrete's axiomatic content is `m ≤ n < m + 1 ⟹ n = m` — a forbidden-strict-between statement. The sub-case (ii, ii) step, however, goes in the backward direction `m + 1 ≤ n ⟹ m < n`. Unpacking `m + 1 ≤ n` via NAT-order's definition of `≤` gives `m + 1 < n ∨ m + 1 = n`. Either disjunct, combined with NAT-addcompat's `m < m + 1` through NAT-order's transitivity (or through substitution in the equality case), yields `m < n` directly. No forbidden-strict-between reasoning enters. The backward direction is derivable from NAT-addcompat + NAT-order without NAT-discrete at all. The "equivalence `m + 1 ≤ n ⟺ m < n`" framing conflates two directions with different dependency structures: the forward direction `m < n ⟹ m + 1 ≤ n` genuinely consumes NAT-discrete (it rules out the forbidden strict-between position between `m` and `m + 1`), but the backward direction does not. The ASN's citation convention (T0: "each proof cites only the ℕ facts it actually uses") is therefore violated at this site — NAT-discrete is cited where NAT-addcompat + NAT-order alone suffice — and the NAT-addcompat enumeration misses a corresponding strict-successor-inequality use at sub-case (ii, ii). The parallel concern raised by cycle 3 for NAT-addcompat undercount continues here under the complementary lens of NAT-discrete over-attribution.

**What needs resolving**: Either (a) replace the citation at sub-case (ii, ii) with NAT-addcompat's strict successor inequality + NAT-order's transitivity (deriving `m < n` directly from `m + 1 ≤ n` and `m < m + 1`), which would add a sixth NAT-addcompat site to the enumeration and reduce NAT-discrete's sites from three to two; or (b) restructure the sub-case to use the simpler contradiction route — from `m = n` and `m + 1 ≤ n`, substitute to obtain `n + 1 ≤ n`, and refute via NAT-addcompat's `n < n + 1` against NAT-order's irreflexivity/trichotomy — which avoids NAT-discrete entirely at this site; or (c) correct the framing so that NAT-discrete is cited only for the forward direction `m < n ⟹ m + 1 ≤ n` and the backward direction is understood as a consequence of NAT-addcompat + NAT-order, with the Depends enumeration adjusted accordingly.
