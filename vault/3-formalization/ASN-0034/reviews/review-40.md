### GlobalUniqueness domain taxonomy defines "owning allocator" using the conclusion it proves
**Foundation**: (internal consistency — proof structure / circular definition)
**ASN**: GlobalUniqueness proof, taxonomy section: *"The owning allocator of an address a is the unique A satisfying a ∈ dom(A)."* Case analysis: *"The argument partitions all pairs of distinct allocation events into five exhaustive cases based on the relationship between the owning allocators."*
**Issue**: The definition of "owning allocator" is value-based: it requires that each address value belongs to exactly one allocator domain — i.e., that `dom(A₁) ∩ dom(A₂) = ∅` for `A₁ ≠ A₂`. This domain-disjointness is precisely what GlobalUniqueness establishes (distinct events produce distinct values, so no value appears in two domains). The definition therefore presupposes the theorem's conclusion. The case analysis routes pairs "based on the relationship between the owning allocators," invoking a term whose well-definedness depends on the result being proved. The circularity is formally real but logically non-load-bearing: every case in the proof actually uses only the *event-based* assignment — which allocator's `inc` operation produced each address — not value-based domain membership. The event-based assignment is trivially well-defined (each `inc` invocation is assigned to exactly one allocator by the taxonomy rule already present in the text: "sibling outputs (k = 0) to the producing allocator's domain, child-spawning outputs (k > 0) to the newly created child's domain"). The exhaustiveness paragraph compounds the issue: *"every address in the system has a unique owning allocator, so every pair of distinct addresses has well-defined owning allocators"* — asserting domain-disjointness as a premise for the case routing, when it is the conclusion. For TLA+ formalization, this structure cannot be mechanized: a `CONSTANT`-level definition cannot reference an unproved `THEOREM`.
**What needs resolving**: Define "producing allocator" (or "owning allocator") in terms of the allocation event that generated the address, not in terms of domain membership. The case analysis should be framed as partitioning pairs of *distinct allocation events* by their producing allocators (which are well-defined by the event-assignment rule). Domain-disjointness — each address value appearing in at most one `dom(A)` — should be stated as a corollary of the theorem, not assumed in the definition that the proof depends on.
