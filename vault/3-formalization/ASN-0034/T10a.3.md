**T10a.3 (LengthSeparation).** Child allocator outputs have strictly greater length than any parent sibling output, with additive separation across nesting levels.

Let a parent allocator have base address `t₀` with sibling length `γ = #t₀`. When the parent spawns a child via `inc(t, k')` with `k' ∈ {1, 2}` (per T10a) — where `t` is one of the parent's siblings — the child's base address `c₀` has length `#c₀ = #t + k'` by TA5(d). Since `t` is a parent sibling, `#t = γ` by T10a.1, so `#c₀ = γ + k'`.

The child allocator produces its own siblings by `inc(·, 0)`. By T10a.1 applied to the child, all child outputs have uniform length `γ + k'`. Since `k' ≥ 1`, every child output has length at least `γ + 1 > γ` — strictly longer than any parent sibling. By T3, tumblers that differ in length are distinct: no child output can equal any parent sibling.

The separation is additive across nesting levels. Consider a lineage of allocators where at each depth `i` the child is spawned by `inc(·, k'_i)` with `k'_i ∈ {1, 2}`. The depth-1 child has sibling length `γ + k'₁` by the single-step argument above. The depth-2 child, spawned from a depth-1 sibling of length `γ + k'₁`, has sibling length `γ + k'₁ + k'₂` by the same reasoning. By induction, the descendant at depth `d` has sibling length exactly `γ + k'₁ + k'₂ + … + k'_d`. Since each `k'_i ≥ 1`, the cumulative sum `k'₁ + … + k'_d` is strictly increasing with `d` along any lineage, so the exact lengths form a strictly increasing sequence. By T3, outputs at different depths along a lineage never collide.

A local monotonicity consequence follows: for any ancestor allocator A at depth `d_A` and descendant allocator B at depth `d_B > d_A` on the same lineage, every output of B is strictly longer than every output of A. By T10a.1, outputs of A have length `γ + k'₁ + … + k'_{d_A}` and outputs of B have length `γ + k'₁ + … + k'_{d_B}`. The difference `k'_{d_A+1} + … + k'_{d_B}` is at least `d_B − d_A ≥ 1` (since each `k'_i ≥ 1`), so `#output(B) > #output(A)`. ∎

*Formal Contract:*
- *Precondition:* Parent allocator with sibling length `γ`; `t` is a parent sibling (so `#t = γ` by T10a.1); child spawned via `inc(t, k')` with `k' ∈ {1, 2}` (per T10a).
- *Postcondition:* All child outputs have length `γ + k' > γ`. No child output equals any parent sibling (by T3, tumblers of different lengths are distinct). Descendant at depth `d` along a lineage with child-spawning parameters `k'₁, …, k'_d` (each `k'_i ∈ {1, 2}`) has output length exactly `γ + k'₁ + … + k'_d ≥ γ + d`; along any lineage the cumulative length is strictly increasing with depth, so outputs at different nesting depths never collide (by T3). Local monotonicity: for any ancestor allocator A at depth `d_A` and descendant allocator B at depth `d_B > d_A` on the same lineage, `#output(B) − #output(A) = k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A ≥ 1`.
- *Depends:* T10a (AllocatorDiscipline) — restricts child-spawning to `inc(·, k')` with `k' ∈ {1, 2}`, supplying the `k' ∈ {1, 2}` premise on which the length-increment and additive-lineage counts rest. T10a.1 (UniformSiblingLength) — supplies `#t = γ` for the parent sibling and the uniform length `γ + k'` for child outputs, applied at every depth along a lineage to sustain the additive count `γ + k'₁ + … + k'_d`. TA5 (HierarchicalIncrement), postcondition (c) — `#inc(t, 0) = #t` — the length-preservation postcondition discharged transitively through T10a.1 at each `inc(·, 0)` stream within the lineage; postcondition (d) — `#inc(t, k') = #t + k'` for `k' > 0` — supplies the per-step length increment `#c₀ = #t + k'` at each child-spawning event, underlying both the single-step claim and the additive cross-lineage sum. T3 (TumblerDistinguishability) — tumblers of different lengths are distinct — excludes collisions between parent siblings and child outputs and between outputs at different nesting depths along any lineage.
