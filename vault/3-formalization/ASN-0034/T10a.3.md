**T10a.3 (LengthSeparation).** Child allocator outputs have strictly greater length than any parent sibling output, with additive separation across nesting levels.

Let a parent allocator have base address `t₀` with sibling length `γ = #t₀`. When the parent spawns a child via `inc(t, k')` with `k' ∈ {1, 2}` (per T10a) — where `t` is one of the parent's siblings — the child's base address `c₀` has length `#c₀ = #t + k'` by TA5(d). Since `t` is a parent sibling, `#t = γ` by T10a.1, so `#c₀ = γ + k'`.

The child allocator produces its own siblings by `inc(·, 0)`. By T10a.1 applied to the child, all child outputs have uniform length `γ + k'`. Since `k' ∈ {1, 2}` we have `1 ≤ k'`; NAT-addcompat's strict successor inequality `n < n + 1` for every `n ∈ ℕ`, instantiated at `n = γ`, gives `γ < γ + 1`, and NAT-addcompat's left order-compatibility (`p ≤ n ⟹ m + p ≤ m + n`), instantiated at `m = γ`, `p = 1`, `n = k'`, lifts `1 ≤ k'` to `γ + 1 ≤ γ + k'`. Chaining, every child output has length `γ + k' ≥ γ + 1 > γ` — strictly longer than any parent sibling. By T3, tumblers that differ in length are distinct: no child output can equal any parent sibling.

The separation is additive across nesting levels. Consider a lineage of allocators where at each depth `i` the child is spawned by `inc(·, k'_i)` with `k'_i ∈ {1, 2}`. The depth-1 child has sibling length `γ + k'₁` by the single-step argument above. The depth-2 child, spawned from a depth-1 sibling of length `γ + k'₁`, has sibling length `γ + k'₁ + k'₂` by the same reasoning. By induction, the descendant at depth `d` has sibling length exactly `γ + k'₁ + k'₂ + … + k'_d`. That this cumulative length strictly increases with `d` is the same strict-successor-plus-order-compatibility inference applied at each depth boundary: for any `d ≥ 0`, NAT-addcompat's strict successor inequality at `n = γ + k'₁ + … + k'_d` gives `γ + k'₁ + … + k'_d < γ + k'₁ + … + k'_d + 1`, and NAT-addcompat's left order-compatibility at `m = γ + k'₁ + … + k'_d`, `p = 1`, `n = k'_{d+1}` lifts `1 ≤ k'_{d+1}` to `γ + k'₁ + … + k'_d + 1 ≤ γ + k'₁ + … + k'_{d+1}`; chaining yields `γ + k'₁ + … + k'_d < γ + k'₁ + … + k'_{d+1}`. The exact lengths form a strictly increasing sequence along any lineage, so by T3, outputs at different depths along a lineage never collide. The running-sum bound `k'₁ + … + k'_d ≥ d` — underlying the postcondition's `γ + k'₁ + … + k'_d ≥ γ + d` clause — arises by induction on `d`. *Base case* `d = 1`: `k'₁ ≥ 1` by `k'₁ ∈ {1, 2}`. *Inductive step*: assume `k'₁ + … + k'_i ≥ i`; NAT-addcompat's right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at `n = k'₁ + … + k'_i`, `p = i`, `m = k'_{i+1}`) lifts the hypothesis to `k'₁ + … + k'_{i+1} ≥ i + k'_{i+1}`, and NAT-addcompat's left order-compatibility (at `m = i`, `p = 1`, `n = k'_{i+1}`) lifts `1 ≤ k'_{i+1}` to `i + 1 ≤ i + k'_{i+1}`; chaining delivers `k'₁ + … + k'_{i+1} ≥ i + 1`. Hence `k'₁ + … + k'_d ≥ d` for all `d ≥ 1`. One final NAT-addcompat left order-compatibility at `m = γ`, `p = d`, `n = k'₁ + … + k'_d` yields `γ + d ≤ γ + k'₁ + … + k'_d`.

A local monotonicity consequence follows: for any ancestor allocator A at depth `d_A` and descendant allocator B at depth `d_B > d_A` on the same lineage, every output of B is strictly longer than every output of A. By T10a.1, outputs of A have length `γ + k'₁ + … + k'_{d_A}` and outputs of B have length `γ + k'₁ + … + k'_{d_B}`. The difference `k'_{d_A+1} + … + k'_{d_B}` is a sum of `d_B − d_A` terms, each `k'_i ≥ 1`; iterated NAT-addcompat order-compatibility applied term-by-term as in the running-sum induction above (right order-compatibility at the running partial sum, left order-compatibility to absorb each per-term `1 ≤ k'_i`) lifts these per-term bounds to `k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A`, and since `d_B > d_A` this gives `d_B − d_A ≥ 1`, hence `1 ≤ k'_{d_A+1} + … + k'_{d_B}`. NAT-addcompat's strict successor inequality at `n = γ + k'₁ + … + k'_{d_A}` gives `γ + k'₁ + … + k'_{d_A} < γ + k'₁ + … + k'_{d_A} + 1`, and NAT-addcompat's left order-compatibility at `m = γ + k'₁ + … + k'_{d_A}`, `p = 1`, `n = k'_{d_A+1} + … + k'_{d_B}` lifts `1 ≤ k'_{d_A+1} + … + k'_{d_B}` to `γ + k'₁ + … + k'_{d_A} + 1 ≤ γ + k'₁ + … + k'_{d_A} + (k'_{d_A+1} + … + k'_{d_B}) = γ + k'₁ + … + k'_{d_B}`; chaining yields `#output(B) = γ + k'₁ + … + k'_{d_B} > γ + k'₁ + … + k'_{d_A} = #output(A)`. ∎

*Formal Contract:*
- *Precondition:* Parent allocator with sibling length `γ`; `t` is a parent sibling (so `#t = γ` by T10a.1); child spawned via `inc(t, k')` with `k' ∈ {1, 2}` (per T10a).
- *Postcondition:* All child outputs have length `γ + k' > γ`. No child output equals any parent sibling (by T3, tumblers of different lengths are distinct). Descendant at depth `d` along a lineage with child-spawning parameters `k'₁, …, k'_d` (each `k'_i ∈ {1, 2}`) has output length exactly `γ + k'₁ + … + k'_d ≥ γ + d`; along any lineage the cumulative length is strictly increasing with depth, so outputs at different nesting depths never collide (by T3). Local monotonicity: for any ancestor allocator A at depth `d_A` and descendant allocator B at depth `d_B > d_A` on the same lineage, `#output(B) − #output(A) = k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A ≥ 1`.
- *Depends:* T10a (AllocatorDiscipline) — restricts child-spawning to `inc(·, k')` with `k' ∈ {1, 2}`, supplying the `k' ∈ {1, 2}` premise on which the length-increment and additive-lineage counts rest. T10a.1 (UniformSiblingLength) — supplies `#t = γ` for the parent sibling and the uniform length `γ + k'` for child outputs, applied at every depth along a lineage to sustain the additive count `γ + k'₁ + … + k'_d`. TA5 (HierarchicalIncrement), postcondition (c) — `#inc(t, 0) = #t` — the length-preservation postcondition discharged transitively through T10a.1 at each `inc(·, 0)` stream within the lineage; postcondition (d) — `#inc(t, k') = #t + k'` for `k' > 0` — supplies the per-step length increment `#c₀ = #t + k'` at each child-spawning event, underlying both the single-step claim and the additive cross-lineage sum. NAT-addcompat (NatAdditionOrderAndSuccessor) — invoked at four sites, matching the per-step citation convention TA5's Case `k > 0` applies and Previous Findings for T10a.5 and T10a.8 established for the structurally identical strict-successor and order-compatibility inferences; TA5(c)/(d) deliver the length equations `#inc(t, 0) = #t` and `#inc(t, k') = #t + k'` but do not on their own supply the order inferences `#t + k' > #t` or `γ + k'₁ + … + k'_d ≥ γ + d`, so NAT-addcompat is cited in parallel with TA5 at every step that invokes strict successor `n < n + 1` or order-compatibility of addition. First, the single-step claim `γ + 1 > γ` (paragraph 2): the strict successor inequality `n < n + 1` for every `n ∈ ℕ`, instantiated at `n = γ`, supplies `γ < γ + 1`, and left order-compatibility (`p ≤ n ⟹ m + p ≤ m + n`), instantiated at `m = γ`, `p = 1`, `n = k'`, lifts `1 ≤ k'` to `γ + 1 ≤ γ + k'`; chaining yields `γ + k' > γ`. Second, the additive-lineage strict-increase claim (paragraph 3): at each depth boundary `d → d + 1`, strict successor at `n = γ + k'₁ + … + k'_d` combined with left order-compatibility at `m = γ + k'₁ + … + k'_d`, `p = 1`, `n = k'_{d+1}` (lifting `1 ≤ k'_{d+1}`) yields `γ + k'₁ + … + k'_d < γ + k'₁ + … + k'_{d+1}`, establishing strict increase with depth along any lineage. Third, the local-monotonicity difference-≥-depth-gap claim (paragraph 4): iterated order-compatibility term-by-term (right order-compatibility at the running partial sum, left order-compatibility to absorb each per-term `1 ≤ k'_i`) lifts the per-term bounds across `d_B − d_A` summands to `k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A`, and strict successor at `n = γ + k'₁ + … + k'_{d_A}` combined with left order-compatibility at `m = γ + k'₁ + … + k'_{d_A}`, `p = 1`, `n = k'_{d_A+1} + … + k'_{d_B}` yields `#output(B) > #output(A)`. Fourth, the postcondition's `γ + k'₁ + … + k'_d ≥ γ + d` clause (paragraph 3): the running-sum induction uses right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, at `n = k'₁ + … + k'_i`, `p = i`, `m = k'_{i+1}`) to lift `k'₁ + … + k'_i ≥ i` to `k'₁ + … + k'_{i+1} ≥ i + k'_{i+1}`, and left order-compatibility (at `m = i`, `p = 1`, `n = k'_{i+1}`) to lift `1 ≤ k'_{i+1}` to `i + 1 ≤ i + k'_{i+1}`; chaining delivers `k'₁ + … + k'_d ≥ d`, and one more left order-compatibility at `m = γ`, `p = d`, `n = k'₁ + … + k'_d` delivers `γ + d ≤ γ + k'₁ + … + k'_d`. Without NAT-addcompat each of these `n < n + 1` and order-compatibility steps would revert to an implicit appeal to ℕ-arithmetic that the ASN has moved away from. T3 (CanonicalRepresentation) — tumblers of different lengths are distinct — excludes collisions between parent siblings and child outputs and between outputs at different nesting depths along any lineage.
