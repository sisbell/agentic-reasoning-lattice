**T10a.3 (LengthSeparation).** Child allocator outputs have strictly greater length than any parent sibling output, with additive separation across nesting levels.

Let a parent allocator have base address `t₀` with sibling length `γ = #t₀`. When the parent spawns a child via `inc(t, k')` with `k' ∈ {1, 2}` (per T10a) — where `t` is one of the parent's siblings — the child's base address `c₀` has length `#c₀ = #t + k'` by TA5(d). Since `t` is a parent sibling, `#t = γ` by T10a.1, so `#c₀ = γ + k'`.

The child allocator produces its own siblings by `inc(·, 0)`. By T10a.1 applied to the child, all child outputs have uniform length `γ + k'`. Since `k' ∈ {1, 2}` we have `1 ≤ k'`; NAT-addcompat's strict successor inequality `n < n + 1` for every `n ∈ ℕ`, instantiated at `n = γ`, gives `γ < γ + 1`, and NAT-addcompat's left order-compatibility (`p ≤ n ⟹ m + p ≤ m + n`), instantiated at `m = γ`, `p = 1`, `n = k'`, lifts `1 ≤ k'` to `γ + 1 ≤ γ + k'`. Chaining, every child output has length `γ + k' ≥ γ + 1 > γ` — strictly longer than any parent sibling. By T3, tumblers that differ in length are distinct: no child output can equal any parent sibling.

The separation is additive across nesting levels. Consider a lineage of allocators where at each depth `i` the child is spawned by `inc(·, k'_i)` with `k'_i ∈ {1, 2}`. The depth-1 child has sibling length `γ + k'₁` by the single-step argument above. The depth-2 child, spawned from a depth-1 sibling of length `γ + k'₁`, has sibling length `γ + k'₁ + k'₂` by the same reasoning. By induction, the descendant at depth `d` has sibling length exactly `γ + k'₁ + k'₂ + … + k'_d`. That this cumulative length strictly increases with `d` is the same strict-successor-plus-order-compatibility inference applied at each depth boundary: for any `d ≥ 0`, NAT-addcompat's strict successor inequality at `n = γ + k'₁ + … + k'_d` gives `γ + k'₁ + … + k'_d < γ + k'₁ + … + k'_d + 1`, and NAT-addcompat's left order-compatibility at `m = γ + k'₁ + … + k'_d`, `p = 1`, `n = k'_{d+1}` lifts `1 ≤ k'_{d+1}` to `γ + k'₁ + … + k'_d + 1 ≤ γ + k'₁ + … + k'_{d+1}`; chaining yields `γ + k'₁ + … + k'_d < γ + k'₁ + … + k'_{d+1}`. The exact lengths form a strictly increasing sequence along any lineage, so by T3, outputs at different depths along a lineage never collide. The running-sum bound `k'₁ + … + k'_d ≥ d` — underlying the postcondition's `γ + k'₁ + … + k'_d ≥ γ + d` clause — arises by induction on `d`. *Base case* `d = 1`: `k'₁ ≥ 1` by `k'₁ ∈ {1, 2}`. *Inductive step*: assume `k'₁ + … + k'_i ≥ i`; NAT-addcompat's right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at `n = k'₁ + … + k'_i`, `p = i`, `m = k'_{i+1}`) lifts the hypothesis to `k'₁ + … + k'_{i+1} ≥ i + k'_{i+1}`, and NAT-addcompat's left order-compatibility (at `m = i`, `p = 1`, `n = k'_{i+1}`) lifts `1 ≤ k'_{i+1}` to `i + 1 ≤ i + k'_{i+1}`; chaining delivers `k'₁ + … + k'_{i+1} ≥ i + 1`. Hence `k'₁ + … + k'_d ≥ d` for all `d ≥ 1`. One final NAT-addcompat left order-compatibility at `m = γ`, `p = d`, `n = k'₁ + … + k'_d` yields `γ + d ≤ γ + k'₁ + … + k'_d`.

A local monotonicity consequence follows: for any ancestor allocator A at depth `d_A` and descendant allocator B at depth `d_B > d_A` on the same lineage, every output of B is strictly longer than every output of A. By T10a.1, outputs of A have length `γ + k'₁ + … + k'_{d_A}` and outputs of B have length `γ + k'₁ + … + k'_{d_B}`. The hypothesis `d_B > d_A` on naturals sharpens to `d_B − d_A ≥ 1` (equivalently `d_A + 1 ≤ d_B`) per the per-step convention TA5's Case `k > 0` applies for the structurally identical "strict order in ℕ ⟹ successor-bounded difference" inference: NAT-sub's conditional-closure clause (`(A m, n ∈ ℕ : m ≥ n : m − n ∈ ℕ)`, instantiated at `m = d_B`, `n = d_A` under the non-strict weakening `d_B ≥ d_A` of the hypothesis `d_B > d_A`) places `d_B − d_A` in ℕ — the membership premise NAT-zero's universal quantifier requires to apply at `n = d_B − d_A`; NAT-zero (`(A n ∈ ℕ :: 0 ≤ n)`, zero as lower bound of ℕ) then supplies `0 ≤ d_B − d_A`, which is the premise NAT-discrete requires at `m = 0`, and NAT-discrete (`(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)`, instantiated at `m = 0`: no natural lies strictly between `0` and `0 + 1`) rules out the interval `0 ≤ d_B − d_A < 1` under `d_B > d_A`; together NAT-sub, NAT-zero, and NAT-discrete sharpen `d_B > d_A` to `d_B − d_A ≥ 1`. The index range `d_A + 1, …, d_B` is therefore non-empty (the same sharpening reread as `d_A + 1 ≤ d_B`), so the difference `k'_{d_A+1} + … + k'_{d_B}` is a well-formed sum of `d_B − d_A` terms, each `k'_i ≥ 1`; iterated NAT-addcompat order-compatibility applied term-by-term as in the running-sum induction above (right order-compatibility at the running partial sum, left order-compatibility to absorb each per-term `1 ≤ k'_i`) lifts these per-term bounds to `k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A`, and chaining with `d_B − d_A ≥ 1` yields `1 ≤ k'_{d_A+1} + … + k'_{d_B}`. NAT-addcompat's strict successor inequality at `n = γ + k'₁ + … + k'_{d_A}` gives `γ + k'₁ + … + k'_{d_A} < γ + k'₁ + … + k'_{d_A} + 1`, and NAT-addcompat's left order-compatibility at `m = γ + k'₁ + … + k'_{d_A}`, `p = 1`, `n = k'_{d_A+1} + … + k'_{d_B}` lifts `1 ≤ k'_{d_A+1} + … + k'_{d_B}` to `γ + k'₁ + … + k'_{d_A} + 1 ≤ γ + k'₁ + … + k'_{d_A} + (k'_{d_A+1} + … + k'_{d_B}) = γ + k'₁ + … + k'_{d_B}`; chaining yields `#output(B) = γ + k'₁ + … + k'_{d_B} > γ + k'₁ + … + k'_{d_A} = #output(A)`. ∎

*Formal Contract:*
- *Precondition:* Parent allocator with sibling length `γ`; `t` is a parent sibling (so `#t = γ` by T10a.1); child spawned via `inc(t, k')` with `k' ∈ {1, 2}` (per T10a).
- *Postcondition:* All child outputs have length `γ + k' > γ`. No child output equals any parent sibling (by T3, tumblers of different lengths are distinct). Descendant at depth `d` along a lineage with child-spawning parameters `k'₁, …, k'_d` (each `k'_i ∈ {1, 2}`) has output length exactly `γ + k'₁ + … + k'_d ≥ γ + d`; along any lineage the cumulative length is strictly increasing with depth, so outputs at different nesting depths never collide (by T3). Local monotonicity: for any ancestor allocator A at depth `d_A` and descendant allocator B at depth `d_B > d_A` on the same lineage, `#output(B) − #output(A) = k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A ≥ 1`.
- *Depends:* T10a (AllocatorDiscipline) — restricts child-spawning to `inc(·, k')` with `k' ∈ {1, 2}`, supplying the `k' ∈ {1, 2}` premise on which the length-increment and additive-lineage counts rest. T10a.1 (UniformSiblingLength) — supplies `#t = γ` for the parent sibling and the uniform length `γ + k'` for child outputs, applied at every depth along a lineage to sustain the additive count `γ + k'₁ + … + k'_d`. TA5 (HierarchicalIncrement), postcondition (c) — `#inc(t, 0) = #t` — the length-preservation postcondition discharged transitively through T10a.1 at each `inc(·, 0)` stream within the lineage; postcondition (d) — `#inc(t, k') = #t + k'` for `k' > 0` — supplies the per-step length increment `#c₀ = #t + k'` at each child-spawning event, underlying both the single-step claim and the additive cross-lineage sum. NAT-addcompat (NatAdditionOrderAndSuccessor) — invoked at four sites, matching the per-step citation convention TA5's Case `k > 0` applies and Previous Findings for T10a.5 and T10a.8 established for the structurally identical strict-successor and order-compatibility inferences; TA5(c)/(d) deliver the length equations `#inc(t, 0) = #t` and `#inc(t, k') = #t + k'` but do not on their own supply the order inferences `#t + k' > #t` or `γ + k'₁ + … + k'_d ≥ γ + d`, so NAT-addcompat is cited in parallel with TA5 at every step that invokes strict successor `n < n + 1` or order-compatibility of addition. First, the single-step claim `γ + 1 > γ` (paragraph 2): the strict successor inequality `n < n + 1` for every `n ∈ ℕ`, instantiated at `n = γ`, supplies `γ < γ + 1`, and left order-compatibility (`p ≤ n ⟹ m + p ≤ m + n`), instantiated at `m = γ`, `p = 1`, `n = k'`, lifts `1 ≤ k'` to `γ + 1 ≤ γ + k'`; chaining yields `γ + k' > γ`. Second, the additive-lineage strict-increase claim (paragraph 3): at each depth boundary `d → d + 1`, strict successor at `n = γ + k'₁ + … + k'_d` combined with left order-compatibility at `m = γ + k'₁ + … + k'_d`, `p = 1`, `n = k'_{d+1}` (lifting `1 ≤ k'_{d+1}`) yields `γ + k'₁ + … + k'_d < γ + k'₁ + … + k'_{d+1}`, establishing strict increase with depth along any lineage. Third, the local-monotonicity difference-≥-depth-gap claim (paragraph 4): iterated order-compatibility term-by-term (right order-compatibility at the running partial sum, left order-compatibility to absorb each per-term `1 ≤ k'_i`) lifts the per-term bounds across `d_B − d_A` summands to `k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A`, and strict successor at `n = γ + k'₁ + … + k'_{d_A}` combined with left order-compatibility at `m = γ + k'₁ + … + k'_{d_A}`, `p = 1`, `n = k'_{d_A+1} + … + k'_{d_B}` yields `#output(B) > #output(A)`. Fourth, the postcondition's `γ + k'₁ + … + k'_d ≥ γ + d` clause (paragraph 3): the running-sum induction uses right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, at `n = k'₁ + … + k'_i`, `p = i`, `m = k'_{i+1}`) to lift `k'₁ + … + k'_i ≥ i` to `k'₁ + … + k'_{i+1} ≥ i + k'_{i+1}`, and left order-compatibility (at `m = i`, `p = 1`, `n = k'_{i+1}`) to lift `1 ≤ k'_{i+1}` to `i + 1 ≤ i + k'_{i+1}`; chaining delivers `k'₁ + … + k'_d ≥ d`, and one more left order-compatibility at `m = γ`, `p = d`, `n = k'₁ + … + k'_d` delivers `γ + d ≤ γ + k'₁ + … + k'_d`. Without NAT-addcompat each of these `n < n + 1` and order-compatibility steps would revert to an implicit appeal to ℕ-arithmetic that the ASN has moved away from. NAT-sub (NatPartialSubtraction) — invoked at the local-monotonicity step (paragraph 4) and in the postcondition's subtraction-form statement. The conditional-closure clause `(A m, n ∈ ℕ : m ≥ n : m − n ∈ ℕ)`, instantiated at `m = d_B`, `n = d_A` under the non-strict weakening `d_B ≥ d_A` of the hypothesis `d_B > d_A`, places `d_B − d_A ∈ ℕ` — the membership premise NAT-zero's universal quantifier requires to apply at `n = d_B − d_A`; without this ℕ-membership, NAT-zero's `(A n ∈ ℕ :: 0 ≤ n)` has no instance to supply to NAT-discrete at `n = d_B − d_A`, and the hypothesis-sharpening chain `d_B > d_A ⟹ d_B − d_A ≥ 1` would consume an unsourced ℕ-membership for `d_B − d_A`. The same conditional-closure clause, instantiated at `m = #output(B)`, `n = #output(A)` under `#output(B) ≥ #output(A)` (the non-strict weakening of `#output(B) > #output(A)` established at the end of paragraph 4), places `#output(B) − #output(A) ∈ ℕ`, and the right-telescoping clause `(A m, n ∈ ℕ :: (m + n) − n = m)`, instantiated at `m = k'_{d_A+1} + … + k'_{d_B}` and `n = γ + k'₁ + … + k'_{d_A}` (so that `m + n = γ + k'₁ + … + k'_{d_B} = #output(B)` and `n = #output(A)`), supplies the equation `#output(B) − #output(A) = k'_{d_A+1} + … + k'_{d_B}` stated in the postcondition's subtraction form; together these NAT-sub clauses license the postcondition's `#output(B) − #output(A) = k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A ≥ 1` chain as a well-typed composition within ℕ rather than an implicit appeal to background arithmetic. Matches the per-step citation convention TA3, TA3-strict, TA4, TA5a, TA7a, TumblerSub, and D0/D1/D2 already follow for subtraction-form expressions, where NAT-sub is cited alongside the order and discreteness axioms it interacts with rather than left implicit. NAT-zero (NatZeroMinimum) — invoked at the local-monotonicity step (paragraph 4) to supply the premise `0 ≤ d_B − d_A` required to instantiate NAT-discrete at `m = 0` for the sharpening of the hypothesis `d_B > d_A` in ℕ to `d_B − d_A ≥ 1` (equivalently `d_A + 1 ≤ d_B`); without NAT-zero the NAT-discrete interval `0 ≤ d_B − d_A < 1` could not be formed, and the sharpening would consume an unsourced lower bound. This matches the per-step citation convention TA5's Case `k > 0`, T10a.5, T10a.8, TA5-SigValid, T4a, and T4c already follow for the structurally identical "strict order in ℕ ⟹ successor-bounded difference" inference, where T0 + NAT-zero + NAT-discrete are cited jointly rather than collapsing the lower-bound step into an implicit appeal to NAT-discrete alone. NAT-discrete (NatDiscreteness) — invoked at the local-monotonicity step (paragraph 4) jointly with NAT-zero's lower bound `0 ≤ d_B − d_A` to convert the hypothesis `d_B > d_A` in ℕ into the sharpened lower bound `d_B − d_A ≥ 1` — no natural lies strictly between `0` and `0 + 1`. This NAT-discrete sharpening is consumed in two places within the same step: first, framing the sum `k'_{d_A+1} + … + k'_{d_B}` as `d_B − d_A` terms with non-empty index range `d_A + 1, …, d_B` presupposes `d_A + 1 ≤ d_B`, which is the same sharpening reread; second, the `d_B − d_A ≥ 1` bound that feeds the transitivity step `k'_{d_A+1} + … + k'_{d_B} ≥ d_B − d_A ≥ 1` is the NAT-discrete conclusion stated directly. NAT-addcompat alone supplies the term-by-term order-compatibility that lifts per-term `1 ≤ k'_i` across the sum, but does not convert the strict order `d_B > d_A` between naturals into the lower bound `d_B − d_A ≥ 1` on their difference — that content is NAT-discrete's, and NAT-zero provides the `0 ≤ d_B − d_A` lower bound the NAT-discrete instantiation requires. Matches the per-step citation convention established in T1, TumblerAdd, ActionPoint, TA-Pos, TA5's Case `k > 0`, and T10a-N for this same inference. T3 (CanonicalRepresentation) — tumblers of different lengths are distinct — excludes collisions between parent siblings and child outputs and between outputs at different nesting depths along any lineage.
