**AllocatedSet (AllocatedSet).** The properties T8 (AllocationPermanence — stated later in this ASN: for every state transition `s → s'`, `allocated(s) ⊆ allocated(s')`, so the allocated set is monotonically non-decreasing) and NoDeallocation (stated later in this ASN: the axiom that no operation in the system's transition vocabulary Σ removes any element from `allocated(s)`) constrain `allocated(s)` — the set of addresses allocated in state s — but the term itself requires a definition grounded in the allocation mechanism; both T8 and NoDeallocation are sibling statements of this ASN, so the chain from AllocatedSet's vocabulary to T8's precise content is internal and can be traced by reading the present document. The direction of logical dependence runs from T8 and NoDeallocation to AllocatedSet (both cite AllocatedSet in their respective *Depends*), and AllocatedSet does not reciprocally depend on T8 — the mentions of T8 in this section are forward references to a consumer, not load-bearing uses of T8's content, which is why T8 is not listed in AllocatedSet's *Depends* below and no circular dependence arises. T10a governs the allocator tree and defines the domain `dom(A) = {tₙ : n ≥ 0}` of each allocator A, where `tₙ₊₁ = inc(tₙ, 0)`. We derive `allocated(s)` from this, and we establish the explicit bridge that relates T10a's unindexed chain to the state-indexed realized domain used by T8, so that per-allocator properties proven over `dom(A)` (notably T9's forward-ordering conclusion) transfer to realized allocations in any reachable state.

A *state* `s` of the allocation system is the configuration of the allocator tree at a point in execution: which allocators have been activated and, for each allocator A, how many sibling increments it has performed. Write `nₛ(A) ≥ 0` for this count. The *realized domain* of A at state s is the finite set

  domₛ(A) = {t₀, t₁, …, t_{nₛ(A)}}

where t₀ is A's base address and tᵢ₊₁ = inc(tᵢ, 0), a finite prefix of dom(A) as defined in T10a. The *allocated set* at state s is the union across all activated allocators:

  allocated(s) = ⋃ { domₛ(A) : A activated in s }

The initial state s₀ activates exactly the root allocator with nₛ₀(root) = 0, so allocated(s₀) = {t₀} where t₀ is the root's base address, which satisfies T4 by T10a's initialization constraint. A *state transition* s → s' is the application of one operation from the transition vocabulary Σ (introduced by NoDeallocation — forward reference, stated after this section — with the frame assumption that Σ is closed: every reachable state transition is the application of some op ∈ Σ). Each allocation-affecting transition does exactly one of two things: it advances some allocator's frontier by one inc(·, 0) step, adding t_{nₛ(A)+1} to the realized domain; or it spawns a child allocator via inc(t, k') with k' ∈ {1, 2} per T10a, activating a new allocator whose base address enters the allocated set. Non-allocating transitions — reads, comparisons, pure arithmetic — leave every realized domain unchanged.

*Domain embedding.* The two notations `dom(A)` and `domₛ(A)` are not independent concepts but stand in an explicit relationship that we now spell out. Since both are generated by the same chain `tᵢ₊₁ = inc(tᵢ, 0)` starting from A's base address t₀, the realized domain at any reachable state s is precisely the initial segment of length `nₛ(A) + 1` in T10a's enumeration of `dom(A)`. Three facts follow, which we assert as an explicit bridge between the state-indexed and unindexed formulations:

  (i) *Inclusion:* for every reachable state s and every activated allocator A, `domₛ(A) ⊆ dom(A)`.

  (ii) *Initial-segment structure:* `domₛ(A) = {tᵢ : 0 ≤ i ≤ nₛ(A)}` where the tᵢ are exactly the `inc(·, 0)` iterates of T10a's definition; consequently, the enumeration index of any element of `domₛ(A)` agrees with its enumeration index in `dom(A)`.

  (iii) *Reachable-state containment:* `dom(A) ⊇ ⋃ { domₛ(A) : s reachable from s₀ }`, so T10a's theoretical chain contains every address realized by A across reachable states, while each `domₛ(A)` is a finite snapshot at one point in execution. The reverse inclusion is not claimed: it would assert that every `tₙ ∈ dom(A)` is realized by some reachable state, a liveness statement that this ASN does not furnish. T10a defines `dom(A)` as the chain of all `inc(·, 0)` iterates without committing Σ to any advancement schedule; NoDeallocation forbids shrinkage but not stagnation; the unbounded-carrier properties T0, T0(a), T0(b) size the ambient tumbler space without constraining which elements reachable states produce. Downstream uses that need the enumeration-index correspondence rest on facts (i) and (ii) alone.

These facts are direct consequences of T10a and the allocator-step semantics given above: T10a restricts sibling production to `inc(·, 0)`, so the only way `domₛ(A)` can grow is by appending `t_{nₛ(A)+1}` — it cannot skip an index, produce an out-of-order element, or insert a gap. Each step of the allocator therefore extends the realized initial segment by one, so every element of `domₛ(A)` lies, at its fixed enumeration index, in T10a's enumeration of `dom(A)`; whether every theoretical element is eventually realized depends on operational properties of Σ that this ASN does not stipulate, which is precisely the asymmetry fact (iii) records.

*Transfer of T9 to realized allocations.* Predicates and claims defined over `dom(A)` that depend only on the `tᵢ`-indexed chain structure carry over unchanged to realized pairs. In particular, if `a, b ∈ domₛ(A)` with `a = tᵢ, b = tⱼ` and `i < j ≤ nₛ(A)`, then by (i) both `a, b ∈ dom(A)`, and by (ii) the indices i, j coincide with those in T10a's enumeration, so `same_allocator(a, b)` (T10a) and `allocated_before(a, b)` (T9) hold for the same pair. T9's conclusion `a < b` therefore applies. Hence T8's use of `allocated(s) = ⋃{domₛ(A) : A activated in s}` is compatible with T9's per-allocator ordering claim: any two addresses co-realized by a single allocator in state s are ordered in the tumbler order according to the index at which they entered the chain, and this ordering is the one T9 establishes.

The unbounded-carrier properties T0, T0(a), T0(b) consumed in fact (iii)'s asymmetry discussion are stated separately in this ASN and their formal content is reproduced here for reference at the citation site, so that the sizing claim "T0, T0(a), T0(b) size the ambient tumbler space without constraining which elements reachable states produce" terminates at stated sources visible within this property's reading context. T0 (CarrierSetDefinition) fixes the carrier `T = {d₁.d₂. ... .dₙ : each dᵢ ∈ ℕ, n ≥ 1}` with length `#· : T → ℕ` satisfying `#a ≥ 1` and component projection `·ᵢ` yielding `aᵢ ∈ ℕ` for `1 ≤ i ≤ #a`. T0(a) (UnboundedComponentValues) asserts `(A t ∈ T, i : 1 ≤ i ≤ #t : (A M ∈ ℕ :: (E t' ∈ T :: #t' = #t ∧ t' agrees with t except t'.dᵢ > M)))` — for every tumbler, every component position, and every bound `M`, a same-length tumbler exists whose value at that position exceeds `M`. T0(b) (UnboundedLength) asserts `(A n ∈ ℕ : n ≥ 1 : (E t ∈ T :: #t ≥ n))` — for every length bound, a tumbler of at least that length exists. Together T0(a) and T0(b) give T an inexhaustible component space at every depth and an inexhaustible range of depths, so the state space `𝒮` admits allocator-tree configurations of arbitrary size; what fact (iii) leaves open is only whether every theoretically admissible address is realized by some reachable state, a liveness question this ASN does not resolve.

*Formal Contract:*
- *Definitions:*
  - *State:* s ∈ 𝒮 is a configuration of the allocator tree — the set of activated allocators and, for each, the count nₛ(A) of sibling increments performed. Here 𝒮 denotes the state space of the allocation system.
  - *Realized domain:* domₛ(A) = {t₀, t₁, …, t_{nₛ(A)}} where tᵢ₊₁ = inc(tᵢ, 0), a finite prefix of dom(A) (T10a).
  - *Allocated set:* allocated(s) = ⋃ { domₛ(A) : A activated in s }.
  - *Initial state:* allocated(s₀) = {t₀} where t₀ is the root allocator's T4-valid base address.
  - *State transition:* s → s' is the application of one operation from Σ — the system's transition vocabulary, introduced by NoDeallocation (forward reference, stated after this section), which supplies the closure frame assumption that every reachable state transition is governed by some op ∈ Σ. Allocation-affecting transitions either advance an allocator's frontier (adding one sibling output) or spawn a child allocator (activating a new allocator whose base address enters the allocated set).
  - *Domain embedding:* For every reachable state s and every activated allocator A: (i) `domₛ(A) ⊆ dom(A)`; (ii) `domₛ(A) = {tᵢ : 0 ≤ i ≤ nₛ(A)}` is the initial segment of T10a's `inc(·, 0)` enumeration of `dom(A)`, with enumeration indices preserved; (iii) `dom(A) ⊇ ⋃ { domₛ(A) : s reachable from s₀ }` — the theoretical chain contains every realized address across reachable states, but the reverse inclusion is not asserted (it would require a liveness/progress axiom, driving `nₛ(A)` arbitrarily high for every activated A, that no axiom of this ASN furnishes). Consequently, any predicate or claim defined over `dom(A)` in terms of the `tᵢ`-indexed chain (notably `same_allocator` (T10a), `allocated_before` (T9), and T9's forward-ordering conclusion) applies unchanged to pairs `a, b ∈ domₛ(A)` — the transfer rests on facts (i) and (ii) alone, independent of (iii).
- *Depends:* T10a (AllocatorDiscipline) — governs the allocator tree structure (sibling production by inc(·, 0), child-spawning by inc(·, k') with k' ∈ {1, 2}, root's T4-valid base address, and at-most-once child-spawning constraint) and supplies the per-allocator domain definition `dom(A) = {tₙ : n ≥ 0}` of which domₛ(A) is a finite prefix; justifies the initial-segment structure of the embedding by restricting sibling production to in-order `inc(·, 0)` steps. T9 (ForwardAllocation) — supplies the enumeration-based `allocated_before` ordering and the per-allocator forward-ordering conclusion that transfers to realized allocations. NoDeallocation (NoDeallocation) [forward reference — NoDeallocation is stated after this section] — introduces the transition vocabulary Σ and its closure frame assumption (every reachable state transition is the application of some op ∈ Σ); AllocatedSet's state-transition clause, both in the prose of this section and in the Formal Contract, cites Σ to name the operations that advance an allocator's frontier or spawn a child allocator, and the asymmetry discussion in fact (iii) explicitly leans on NoDeallocation's "forbids shrinkage but not stagnation" reading of Σ's semantics.
