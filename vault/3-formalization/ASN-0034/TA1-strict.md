**TA1-strict (StrictOrderPreservation).** `(A a, b, w ‚ąą T : a < b ‚ąß Pos(w) ‚ąß actionPoint(w) ‚Č§ #a ‚ąß actionPoint(w) ‚Č§ #b ‚ąß actionPoint(w) ‚Č• divergence(a, b) : a ‚äē w < b ‚äē w)`.

When the action point falls before the divergence ‚ÄĒ `k < divergence(a, b)` ‚ÄĒ both operands agree at position `k`, both get the same `w‚āĖ` added, and both copy the same tail from `w` afterward. The original divergence is erased and the results are equal. For example, `a = [1, 3]`, `b = [1, 5]` (diverge at position 2), `w = [2]` (action point at position 1): `a ‚äē w = [3] = b ‚äē w`. Order degrades to equality, never reversal.

*Proof.* We show that tumbler addition by `w` preserves the strict inequality `a < b` whenever the action point of `w` falls at or beyond the first disagreement between `a` and `b`.

Let `j = divergence(a, b)` and let `k` be the action point of `w`. The preconditions give `k ‚Č• j`, `k ‚Č§ #a`, and `k ‚Č§ #b`. To rule out Divergence case (ii) without invoking a primitive binary-minimum operator on ‚Ąē, we apply NAT-order's trichotomy at the length pair `(#a, #b)` ‚ÄĒ the same split Divergence's own case (ii) performs. The `#a = #b` branch makes case (ii) inapplicable, since case (ii) requires `#a ‚Č† #b`. Under the strict branch `#a < #b` (which selects sub-case (ii-a) with `j = #a + 1`), combining `k ‚Č• j = #a + 1` with the precondition `k ‚Č§ #a` gives `#a + 1 ‚Č§ #a`; NAT-addcompat's strict successor inequality (`n < n + 1`, instantiated at `n = #a`) gives `#a < #a + 1`; chaining these produces `#a + 1 ‚Č§ #a < #a + 1`, ruled out by NAT-order's irreflexivity (`¬¨(n < n)`, instantiated at `n = #a + 1`). Under the symmetric strict branch `#b < #a` (which selects sub-case (ii-b) with `j = #b + 1`), the same chain at `n = #b` ‚ÄĒ `k ‚Č• #b + 1` with `k ‚Č§ #b` and NAT-addcompat's strict successor inequality at `n = #b` ‚ÄĒ yields `#b + 1 ‚Č§ #b < #b + 1`, again ruled out by NAT-order's irreflexivity at `n = #b + 1`. Both sub-cases fail, so case (ii) cannot apply, and we are in case (i): position `j` is shared by both tumblers, `a‚Īľ ‚Č† b‚Īľ`, and `aŠĶĘ = bŠĶĘ` for all `i < j`. Since `a < b`, the T1 case (i) direction gives `a‚Īľ < b‚Īľ`.

Recall TumblerAdd's constructive definition: for any tumbler `x` and positive displacement `w` with action point `k ‚Č§ #x`, the result `x ‚äē w` is built component-wise as `(x ‚äē w)ŠĶĘ = xŠĶĘ` for `i < k` (prefix copy), `(x ‚äē w)‚āĖ = x‚āĖ + w‚āĖ` (single-component advance), and `(x ‚äē w)ŠĶĘ = wŠĶĘ` for `i > k` (tail from displacement). By TA0, both `a ‚äē w` and `b ‚äē w` are well-defined members of T, since `k ‚Č§ #a` and `k ‚Č§ #b` each ensure the action point falls within its respective operand. Two cases arise from the relationship between `k` and `j`.

*Case 1: `k = j`.* For `i < k`: since `i < j`, the Divergence agreement property gives `aŠĶĘ = bŠĶĘ`, and TumblerAdd's prefix-copy rule gives `(a ‚äē w)ŠĶĘ = aŠĶĘ = bŠĶĘ = (b ‚äē w)ŠĶĘ`. At position `k = j`: TumblerAdd gives `(a ‚äē w)‚āĖ = a‚āĖ + w‚āĖ` and `(b ‚äē w)‚āĖ = b‚āĖ + w‚āĖ`. The divergence inequality gives `a‚āĖ < b‚āĖ` on ‚Ąē, and we must promote this strict inequality through the shared summand `w‚āĖ` to `a‚āĖ + w‚āĖ < b‚āĖ + w‚āĖ`; since NAT-addcompat axiomatises only non-strict order-compatibility, the promotion is a four-step chain. First, NAT-order's defining clause `m ‚Č§ n ‚üļ m < n ‚ą® m = n` weakens `a‚āĖ < b‚āĖ` to the non-strict `a‚āĖ ‚Č§ b‚āĖ`. Second, NAT-addcompat's right order-compatibility (`n ‚Č• p ‚üĻ n + m ‚Č• p + m`, instantiated at `m = w‚āĖ`, `n = b‚āĖ`, `p = a‚āĖ`) lifts `a‚āĖ ‚Č§ b‚āĖ` to `a‚āĖ + w‚āĖ ‚Č§ b‚āĖ + w‚āĖ`. Third, to sharpen this non-strict ‚Č§ back to strict we rule out the equality `a‚āĖ + w‚āĖ = b‚āĖ + w‚āĖ`: NAT-cancel's right cancellation (`n + m = p + m ‚üĻ n = p`, instantiated at `m = w‚āĖ`, `n = a‚āĖ`, `p = b‚āĖ`) would extract `a‚āĖ = b‚āĖ` from that equality, contradicting `a‚āĖ < b‚āĖ` via NAT-order's irreflexivity (`¬¨(a‚āĖ < a‚āĖ)`). Fourth, NAT-order's defining clause, applied to the non-strict `a‚āĖ + w‚āĖ ‚Č§ b‚āĖ + w‚āĖ` together with the non-equality just discharged, yields the strict `a‚āĖ + w‚āĖ < b‚āĖ + w‚āĖ`. The results agree on all positions before `k` and diverge strictly at `k`. T1 case (i) additionally requires the length bounds `k ‚Č§ #(a ‚äē w)` and `k ‚Č§ #(b ‚äē w)`: ActionPoint's defining postcondition `1 ‚Č§ actionPoint(w) ‚Č§ #w` supplies its upper clause `k ‚Č§ #w`, and TA0's result-length identity `#(a ‚äē w) = #w` ‚ÄĒ engaged once under the precondition `actionPoint(w) ‚Č§ #a`, symmetrically once under `actionPoint(w) ‚Č§ #b` to obtain `#(b ‚äē w) = #w` ‚ÄĒ rewrites `k ‚Č§ #w` into the required `k ‚Č§ #(a ‚äē w)` and `k ‚Č§ #(b ‚äē w)`. By T1 case (i), `a ‚äē w < b ‚äē w`.

*Case 2: `k > j`.* For `i < k`: TumblerAdd's prefix-copy rule gives `(a ‚äē w)ŠĶĘ = aŠĶĘ` and `(b ‚äē w)ŠĶĘ = bŠĶĘ`. Since `j < k`, position `j` lies in this prefix-copy region: `(a ‚äē w)‚Īľ = a‚Īľ < b‚Īľ = (b ‚äē w)‚Īľ` (the divergence inequality is preserved). For `i < j`: the Divergence agreement property gives `aŠĶĘ = bŠĶĘ`, so `(a ‚äē w)ŠĶĘ = aŠĶĘ = bŠĶĘ = (b ‚äē w)ŠĶĘ`. The original divergence at position `j` survives intact in the results ‚ÄĒ the action point, being deeper, does not touch positions at or above `j`. T1 case (i) additionally requires the length bounds `j ‚Č§ #(a ‚äē w)` and `j ‚Č§ #(b ‚äē w)`: the case assumption `k > j` composed with ActionPoint's upper clause `k ‚Č§ #w` through NAT-order's transitivity of `<` on ‚Ąē ‚ÄĒ applied to the strict disjunct `k < #w` of the unpacked bound, with the equality disjunct `k = #w` resolving by substitution of `k = #w` into `k > j` to yield `j < #w` directly rather than through a second transitivity step ‚ÄĒ produces `j < #w`; NAT-order's defining clause `m ‚Č§ n ‚üļ m < n ‚ą® m = n` instantiated at `m = j, n = #w` weakens this to `j ‚Č§ #w`; and TA0's result-length identity `#(a ‚äē w) = #w` ‚ÄĒ engaged once under the precondition `actionPoint(w) ‚Č§ #a`, symmetrically once under `actionPoint(w) ‚Č§ #b` to obtain `#(b ‚äē w) = #w` ‚ÄĒ rewrites `j ‚Č§ #w` into the required `j ‚Č§ #(a ‚äē w)` and `j ‚Č§ #(b ‚äē w)`. By T1 case (i), `a ‚äē w < b ‚äē w`.

In both cases, `a ‚äē w < b ‚äē w`. ‚ąé
