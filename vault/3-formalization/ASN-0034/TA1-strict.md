**TA1-strict (StrictOrderPreservation).** `(A a, b, w ∈ T : a < b ∧ Pos(w) ∧ actionPoint(w) ≤ #a ∧ actionPoint(w) ≤ #b ∧ actionPoint(w) ≥ divergence(a, b) : a ⊕ w < b ⊕ w)`.

When the action point falls before the divergence — `k < divergence(a, b)` — both operands agree at position `k`, both get the same `wₖ` added, and both copy the same tail from `w` afterward. The original divergence is erased and the results are equal. For example, `a = [1, 3]`, `b = [1, 5]` (diverge at position 2), `w = [2]` (action point at position 1): `a ⊕ w = [3] = b ⊕ w`. Order degrades to equality, never reversal.

*Proof.* We show that tumbler addition by `w` preserves the strict inequality `a < b` whenever the action point of `w` falls at or beyond the first disagreement between `a` and `b`.

The invocation `divergence(a, b)` — appearing both in the precondition `actionPoint(w) ≥ divergence(a, b)` and in the opening of this proof — requires `a ≠ b` at Divergence's Definition. This follows from the precondition `a < b` via T1's irreflexivity: the hypothetical equality `a = b`, substituted into `a < b`, would yield `a < a`, contradicting `¬(a < a)` from T1 part (a). Thus `a ≠ b`, and Divergence's Definition-level precondition is discharged at the caller. Let `j = divergence(a, b)` and let `k` be the action point of `w`. The preconditions give `k ≥ j`, `k ≤ #a`, and `k ≤ #b`. To rule out Divergence case (ii) without invoking a primitive binary-minimum operator on ℕ, we apply NAT-order's trichotomy at the length pair `(#a, #b)` — the same split Divergence's own case (ii) performs. The `#a = #b` branch makes case (ii) inapplicable, since case (ii) requires `#a ≠ #b`. Under the strict branch `#a < #b` (which selects sub-case (ii-a) with `j = #a + 1`), combining `k ≥ j = #a + 1` with the precondition `k ≤ #a` gives `#a + 1 ≤ #a`; NAT-addcompat's strict successor inequality (`n < n + 1`, instantiated at `n = #a`) gives `#a < #a + 1`; chaining these produces `#a + 1 ≤ #a < #a + 1`, ruled out by NAT-order's irreflexivity (`¬(n < n)`, instantiated at `n = #a + 1`). Under the symmetric strict branch `#b < #a` (which selects sub-case (ii-b) with `j = #b + 1`), the same chain at `n = #b` — `k ≥ #b + 1` with `k ≤ #b` and NAT-addcompat's strict successor inequality at `n = #b` — yields `#b + 1 ≤ #b < #b + 1`, again ruled out by NAT-order's irreflexivity at `n = #b + 1`. Both sub-cases fail, so case (ii) cannot apply, and we are in Divergence case (i): position `j` is shared by both tumblers (`1 ≤ j ∧ j ≤ #a ∧ j ≤ #b`), `aⱼ ≠ bⱼ`, and `aᵢ = bᵢ` for all `1 ≤ i < j`. The precondition `a < b` supplies some T1 witness, which we must align with `j` before reading off `aⱼ < bⱼ`. We first rule out T1 case (ii) for `a < b`: that disjunct would require `a` to be a proper prefix of `b` — a witness `k' = #a + 1 ≤ #b` with `aᵢ = bᵢ` for all `1 ≤ i < k'`, equivalently `aᵢ = bᵢ` for all `1 ≤ i ≤ #a`. Divergence case (i)'s bound `j ≤ #a` places position `j` in this range, so T1 case (ii)'s agreement requirement instantiated at `i = j` would force `aⱼ = bⱼ`, contradicting Divergence case (i)'s `aⱼ ≠ bⱼ`. T1 case (ii) is thereby foreclosed, and `a < b` holds via T1 case (i): there exists `k'` with `1 ≤ k' ∧ k' ≤ #a ∧ k' ≤ #b`, `aₖ' < bₖ'`, and `aᵢ = bᵢ` for all `1 ≤ i < k'`. NAT-order's trichotomy at the index pair `(k', j)` supplies exactly one of `k' < j`, `k' = j`, or `k' > j`. In the `k' < j` branch, Divergence case (i)'s prior-position agreement `aᵢ = bᵢ` for all `1 ≤ i < j` instantiated at `i = k'` (with `1 ≤ k'` from T1 case (i)'s witness bound and `k' < j` from the trichotomy branch) gives `aₖ' = bₖ'`; substituting this equality into T1 case (i)'s strict inequality `aₖ' < bₖ'` yields `aₖ' < aₖ'`, contradicted by NAT-order's irreflexivity instantiated at `n = aₖ'`. In the `k' > j` branch, T1 case (i)'s agreement requirement `aᵢ = bᵢ` for all `1 ≤ i < k'` instantiated at `i = j` (with `1 ≤ j` from Divergence case (i) and `j < k'` from the trichotomy branch) gives `aⱼ = bⱼ`, contradicting Divergence case (i)'s `aⱼ ≠ bⱼ`. Only the `k' = j` branch survives, at which T1 case (i)'s strict inequality at position `k' = j` delivers `aⱼ < bⱼ`.

Recall TumblerAdd's constructive definition: for any tumbler `x` and positive displacement `w` with action point `k ≤ #x`, the result `x ⊕ w` is built component-wise as `(x ⊕ w)ᵢ = xᵢ` for `i < k` (prefix copy), `(x ⊕ w)ₖ = xₖ + wₖ` (single-component advance), and `(x ⊕ w)ᵢ = wᵢ` for `i > k` (tail from displacement). By TA0, both `a ⊕ w` and `b ⊕ w` are well-defined members of T, since `k ≤ #a` and `k ≤ #b` each ensure the action point falls within its respective operand. Two cases arise from the relationship between `k` and `j`.

*Case 1: `k = j`.* For `i < k`: since `i < j`, the Divergence agreement property gives `aᵢ = bᵢ`, and TumblerAdd's prefix-copy rule gives `(a ⊕ w)ᵢ = aᵢ = bᵢ = (b ⊕ w)ᵢ`. At position `k = j`: TumblerAdd gives `(a ⊕ w)ₖ = aₖ + wₖ` and `(b ⊕ w)ₖ = bₖ + wₖ`. The divergence inequality gives `aₖ < bₖ` on ℕ, and we must promote this strict inequality through the shared summand `wₖ` to `aₖ + wₖ < bₖ + wₖ`; since NAT-addcompat axiomatises only non-strict order-compatibility, the promotion is a four-step chain. First, NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` weakens `aₖ < bₖ` to the non-strict `aₖ ≤ bₖ`. Second, NAT-addcompat's right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at `m = wₖ`, `n = bₖ`, `p = aₖ`) lifts `aₖ ≤ bₖ` to `aₖ + wₖ ≤ bₖ + wₖ`. Third, to sharpen this non-strict ≤ back to strict we rule out the equality `aₖ + wₖ = bₖ + wₖ`: NAT-cancel's right cancellation (`n + m = p + m ⟹ n = p`, instantiated at `m = wₖ`, `n = aₖ`, `p = bₖ`) would extract `aₖ = bₖ` from that equality, contradicting `aₖ < bₖ` via NAT-order's irreflexivity (`¬(aₖ < aₖ)`). Fourth, NAT-order's defining clause, applied to the non-strict `aₖ + wₖ ≤ bₖ + wₖ` together with the non-equality just discharged, yields the strict `aₖ + wₖ < bₖ + wₖ`. The results agree on all positions before `k` and diverge strictly at `k`. T1 case (i) additionally requires the length bounds `k ≤ #(a ⊕ w)` and `k ≤ #(b ⊕ w)`: ActionPoint's defining postcondition `1 ≤ actionPoint(w) ≤ #w` supplies its upper clause `k ≤ #w`, and TA0's result-length identity `#(a ⊕ w) = #w` — engaged once under the precondition `actionPoint(w) ≤ #a`, symmetrically once under `actionPoint(w) ≤ #b` to obtain `#(b ⊕ w) = #w` — rewrites `k ≤ #w` into the required `k ≤ #(a ⊕ w)` and `k ≤ #(b ⊕ w)`. By T1 case (i), `a ⊕ w < b ⊕ w`.

*Case 2: `k > j`.* For `i < k`: TumblerAdd's prefix-copy rule gives `(a ⊕ w)ᵢ = aᵢ` and `(b ⊕ w)ᵢ = bᵢ`. Since `j < k`, position `j` lies in this prefix-copy region: `(a ⊕ w)ⱼ = aⱼ < bⱼ = (b ⊕ w)ⱼ` (the divergence inequality is preserved). For `i < j`: the Divergence agreement property gives `aᵢ = bᵢ`, so `(a ⊕ w)ᵢ = aᵢ = bᵢ = (b ⊕ w)ᵢ`. The original divergence at position `j` survives intact in the results — the action point, being deeper, does not touch positions at or above `j`. T1 case (i) additionally requires the length bounds `j ≤ #(a ⊕ w)` and `j ≤ #(b ⊕ w)`: the case assumption `k > j` composed with ActionPoint's upper clause `k ≤ #w` through NAT-order's transitivity of `<` on ℕ — applied to the strict disjunct `k < #w` of the unpacked bound, with the equality disjunct `k = #w` resolving by substitution of `k = #w` into `k > j` to yield `j < #w` directly rather than through a second transitivity step — produces `j < #w`; NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = j, n = #w` weakens this to `j ≤ #w`; and TA0's result-length identity `#(a ⊕ w) = #w` — engaged once under the precondition `actionPoint(w) ≤ #a`, symmetrically once under `actionPoint(w) ≤ #b` to obtain `#(b ⊕ w) = #w` — rewrites `j ≤ #w` into the required `j ≤ #(a ⊕ w)` and `j ≤ #(b ⊕ w)`. By T1 case (i), `a ⊕ w < b ⊕ w`.

In both cases, `a ⊕ w < b ⊕ w`. ∎
