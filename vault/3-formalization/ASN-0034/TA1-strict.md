**TA1-strict (StrictOrderPreservation).** `(A a, b, w : a < b ∧ Pos(w) ∧ actionPoint(w) ≤ #a ∧ actionPoint(w) ≤ #b ∧ actionPoint(w) ≥ divergence(a, b) : a ⊕ w < b ⊕ w)`.

When the action point falls before the divergence — `k < divergence(a, b)` — both operands agree at position `k`, both get the same `wₖ` added, and both copy the same tail from `w` afterward. The original divergence is erased and the results are equal. For example, `a = [1, 3]`, `b = [1, 5]` (diverge at position 2), `w = [2]` (action point at position 1): `a ⊕ w = [3] = b ⊕ w`. Order degrades to equality, never reversal.

*Proof.* We show that tumbler addition by `w` preserves the strict inequality `a < b` whenever the action point of `w` falls at or beyond the first disagreement between `a` and `b`.

Let `j = divergence(a, b)` and let `k` be the action point of `w`. The preconditions give `k ≥ j`, `k ≤ #a`, and `k ≤ #b`. To rule out Divergence case (ii) without invoking a primitive binary-minimum operator on ℕ, we apply NAT-order's trichotomy at the length pair `(#a, #b)` — the same split Divergence's own case (ii) performs. The `#a = #b` branch makes case (ii) inapplicable, since case (ii) requires `#a ≠ #b`. Under the strict branch `#a < #b` (which selects sub-case (ii-a) with `j = #a + 1`), combining `k ≥ j = #a + 1` with the precondition `k ≤ #a` gives `#a + 1 ≤ #a`; NAT-addcompat's strict successor inequality (`n < n + 1`, instantiated at `n = #a`) gives `#a < #a + 1`; chaining these produces `#a + 1 ≤ #a < #a + 1`, ruled out by NAT-order's irreflexivity (`¬(n < n)`, instantiated at `n = #a + 1`). Under the symmetric strict branch `#b < #a` (which selects sub-case (ii-b) with `j = #b + 1`), the same chain at `n = #b` — `k ≥ #b + 1` with `k ≤ #b` and NAT-addcompat's strict successor inequality at `n = #b` — yields `#b + 1 ≤ #b < #b + 1`, again ruled out by NAT-order's irreflexivity at `n = #b + 1`. Both sub-cases fail, so case (ii) cannot apply, and we are in case (i): position `j` is shared by both tumblers, `aⱼ ≠ bⱼ`, and `aᵢ = bᵢ` for all `i < j`. Since `a < b`, the T1 case (i) direction gives `aⱼ < bⱼ`.

Recall TumblerAdd's constructive definition: for any tumbler `x` and positive displacement `w` with action point `k ≤ #x`, the result `x ⊕ w` is built component-wise as `(x ⊕ w)ᵢ = xᵢ` for `i < k` (prefix copy), `(x ⊕ w)ₖ = xₖ + wₖ` (single-component advance), and `(x ⊕ w)ᵢ = wᵢ` for `i > k` (tail from displacement). By TA0, both `a ⊕ w` and `b ⊕ w` are well-defined members of T, since `k ≤ #a` and `k ≤ #b` each ensure the action point falls within its respective operand. Two cases arise from the relationship between `k` and `j`.

*Case 1: `k = j`.* For `i < k`: since `i < j`, the Divergence agreement property gives `aᵢ = bᵢ`, and TumblerAdd's prefix-copy rule gives `(a ⊕ w)ᵢ = aᵢ = bᵢ = (b ⊕ w)ᵢ`. At position `k = j`: TumblerAdd gives `(a ⊕ w)ₖ = aₖ + wₖ` and `(b ⊕ w)ₖ = bₖ + wₖ`. The divergence inequality gives `aₖ < bₖ` on ℕ, and we must promote this strict inequality through the shared summand `wₖ` to `aₖ + wₖ < bₖ + wₖ`; since NAT-addcompat axiomatises only non-strict order-compatibility, the promotion is a four-step chain. First, NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` weakens `aₖ < bₖ` to the non-strict `aₖ ≤ bₖ`. Second, NAT-addcompat's right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at `m = wₖ`, `n = bₖ`, `p = aₖ`) lifts `aₖ ≤ bₖ` to `aₖ + wₖ ≤ bₖ + wₖ`. Third, to sharpen this non-strict ≤ back to strict we rule out the equality `aₖ + wₖ = bₖ + wₖ`: NAT-cancel's right cancellation (`n + m = p + m ⟹ n = p`, instantiated at `m = wₖ`, `n = aₖ`, `p = bₖ`) would extract `aₖ = bₖ` from that equality, contradicting `aₖ < bₖ` via NAT-order's irreflexivity (`¬(aₖ < aₖ)`). Fourth, NAT-order's defining clause, applied to the non-strict `aₖ + wₖ ≤ bₖ + wₖ` together with the non-equality just discharged, yields the strict `aₖ + wₖ < bₖ + wₖ`. The results agree on all positions before `k` and diverge strictly at `k`. By T1 case (i), `a ⊕ w < b ⊕ w`.

*Case 2: `k > j`.* For `i < k`: TumblerAdd's prefix-copy rule gives `(a ⊕ w)ᵢ = aᵢ` and `(b ⊕ w)ᵢ = bᵢ`. Since `j < k`, position `j` lies in this prefix-copy region: `(a ⊕ w)ⱼ = aⱼ < bⱼ = (b ⊕ w)ⱼ` (the divergence inequality is preserved). For `i < j`: the Divergence agreement property gives `aᵢ = bᵢ`, so `(a ⊕ w)ᵢ = aᵢ = bᵢ = (b ⊕ w)ᵢ`. The original divergence at position `j` survives intact in the results — the action point, being deeper, does not touch positions at or above `j`. By T1 case (i), `a ⊕ w < b ⊕ w`.

In both cases, `a ⊕ w < b ⊕ w`. ∎

NAT-addcompat (NatAdditionOrderAndSuccessor) — invoked at two distinct sites. (a) At the opening of the proof, in the case-(ii) rule-out, NAT-addcompat's strict successor inequality (`n < n + 1`) is instantiated at `n = #a` (sub-case (ii-a)) and `n = #b` (sub-case (ii-b)) to form the contradictions `#a + 1 ≤ #a < #a + 1` and `#b + 1 ≤ #b < #b + 1` that NAT-order's irreflexivity discharges, ruling out the sub-cases in which `j` would equal `#a + 1` or `#b + 1`. (b) In Case 1 at the position `k = j` step as one link of the four-step strict-promotion chain: the right order-compatibility clause (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at `m = wₖ`, `n = bₖ`, `p = aₖ`) lifts the weakened `aₖ ≤ bₖ` into `aₖ + wₖ ≤ bₖ + wₖ`. The right form is the form used because the shared summand `wₖ` is added on the right of the variable summands `aₖ`, `bₖ`; NAT-addcompat axiomatises both left and right forms precisely so this step need not tacitly assume commutativity of `+` on ℕ. Since NAT-addcompat axiomatises only non-strict order-compatibility, this lift alone does not promote `aₖ < bₖ` directly — the strict result is reached via the subsequent NAT-cancel and NAT-order steps. NAT-order (NatStrictTotalOrder) — invoked at two distinct sites. *Opening (case-(ii) rule-out)*: (i*) trichotomy at the length pair `(#a, #b)` — the same instance Divergence's own case (ii) consumes — supplies exactly one of `#a = #b`, `#a < #b`, or `#b < #a`; the `#a = #b` branch makes Divergence case (ii) inapplicable, and the two strict branches select sub-case (ii-a) and sub-case (ii-b) respectively. This replaces any appeal to an un-axiomatized `min(#a, #b)`: the shorter length is named by the selected sub-case rather than by a primitive binary-minimum operator, matching the convention Divergence establishes. (ii*) Irreflexivity (`¬(n < n)`) instantiated at `n = #a + 1` (sub-case (ii-a)) and `n = #b + 1` (sub-case (ii-b)) discharges the contradictions `#a + 1 ≤ #a < #a + 1` and `#b + 1 ≤ #b < #b + 1` formed from the precondition `k ≤ #a` (resp. `k ≤ #b`), `k ≥ j = #a + 1` (resp. `j = #b + 1`), and NAT-addcompat's strict successor inequality at `n = #a` (resp. `n = #b`), thereby ruling out both sub-cases of case (ii). *Case 1 at position `k = j` (strict-promotion chain)*: three sub-roles. (i) Its defining clause `m ≤ n ⟺ m < n ∨ m = n` weakens the divergence inequality `aₖ < bₖ` into the non-strict `aₖ ≤ bₖ` that NAT-addcompat consumes. (ii) Irreflexivity (`¬(n < n)` for every `n ∈ ℕ`) discharges the contradiction when the hypothetical equality `aₖ + wₖ = bₖ + wₖ` produces `aₖ = bₖ`, ruling that equality out. (iii) The same defining clause, applied to `aₖ + wₖ ≤ bₖ + wₖ` together with the non-equality just discharged, converts non-strict with non-equality back into the strict `aₖ + wₖ < bₖ + wₖ` that T1 case (i) demands. NAT-cancel (NatAdditionCancellation) — invoked in Case 1 at the position `k = j` step for the equality-ruling-out sub-step: right cancellation (`n + m = p + m ⟹ n = p`, instantiated at `m = wₖ`, `n = aₖ`, `p = bₖ`) shows that the hypothetical equality `aₖ + wₖ = bₖ + wₖ` would yield `aₖ = bₖ`, which contradicts the divergence inequality `aₖ < bₖ` via NAT-order's irreflexivity; this is the step that allows the non-strict ≤ supplied by NAT-addcompat to sharpen to the strict < in the fourth link of the chain. The same four-step chain is discharged explicitly in GlobalUniqueness Case 5's length-collision argument, and the per-step citation convention established by T0 requires each link to be sourced to the specific NAT-* axiom that licenses it.
