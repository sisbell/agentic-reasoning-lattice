**TA1-strict (StrictOrderPreservation).** `(A a, b, w ‚ąą T : a < b ‚ąß Pos(w) ‚ąß actionPoint(w) ‚Č§ #a ‚ąß actionPoint(w) ‚Č§ #b ‚ąß actionPoint(w) ‚Č• divergence(a, b) : a ‚äē w < b ‚äē w)`.

When the action point falls before the divergence ‚ÄĒ `k < divergence(a, b)` ‚ÄĒ both operands agree at position `k`, both get the same `w‚āĖ` added, and both copy the same tail from `w` afterward. The original divergence is erased and the results are equal. For example, `a = [1, 3]`, `b = [1, 5]` (diverge at position 2), `w = [2]` (action point at position 1): `a ‚äē w = [3] = b ‚äē w`. Order degrades to equality, never reversal.

*Proof.* We show that tumbler addition by `w` preserves the strict inequality `a < b` whenever the action point of `w` falls at or beyond the first disagreement between `a` and `b`.

The invocation `divergence(a, b)` ‚ÄĒ appearing both in the precondition `actionPoint(w) ‚Č• divergence(a, b)` and in the opening of this proof ‚ÄĒ requires `a ‚Č† b` as Divergence's precondition. This follows from the precondition `a < b` via T1's irreflexivity: the hypothetical equality `a = b`, substituted into `a < b`, would yield `a < a`, contradicting `¬¨(a < a)` from T1 part (a). Thus `a ‚Č† b`, and Divergence's precondition is discharged at the caller. Let `j = divergence(a, b)` and let `k` be the action point of `w`. The preconditions give `k ‚Č• j`, `k ‚Č§ #a`, and `k ‚Č§ #b`. To rule out Divergence case (ii) without invoking a primitive binary-minimum operator on ‚Ąē, we apply NAT-order's trichotomy at the length pair `(#a, #b)` ‚ÄĒ the same split Divergence's own case (ii) performs. The `#a = #b` branch makes case (ii) inapplicable, since case (ii) requires `#a ‚Č† #b`. Under the strict branch `#a < #b` (which selects sub-case (ii-a) with `j = #a + 1`), combining `k ‚Č• j = #a + 1` with the precondition `k ‚Č§ #a` gives the non-strict `#a + 1 ‚Č§ #a`, and NAT-addcompat's strict successor inequality (`n < n + 1`, instantiated at `n = #a`) gives the strict `#a < #a + 1`; composing these into the strict `#a + 1 < #a + 1` that irreflexivity consumes is a three-step promotion through NAT-order. First, NAT-order's defining clause `m ‚Č§ n ‚üļ m < n ‚ą® m = n` instantiated at `m = #a + 1, n = #a` unfolds the non-strict `#a + 1 ‚Č§ #a` into the disjunction `#a + 1 < #a ‚ą® #a + 1 = #a`. Second, the first disjunct `#a + 1 < #a` chains with NAT-addcompat's strict successor inequality `#a < #a + 1` through NAT-order's transitivity of `<` to yield the strict `#a + 1 < #a + 1`; the second disjunct `#a + 1 = #a` substitutes into `#a < #a + 1` to yield `#a + 1 < #a + 1` directly. Third, NAT-order's irreflexivity (`¬¨(n < n)`, instantiated at `n = #a + 1`) rules out `#a + 1 < #a + 1`, foreclosing the sub-case. Under the symmetric strict branch `#b < #a` (which selects sub-case (ii-b) with `j = #b + 1`), the same three-step promotion at `n = #b` applies: `k ‚Č• #b + 1` with the precondition `k ‚Č§ #b` gives `#b + 1 ‚Č§ #b`, and NAT-addcompat's strict successor inequality at `n = #b` gives `#b < #b + 1`; NAT-order's defining clause instantiated at `m = #b + 1, n = #b` unfolds `#b + 1 ‚Č§ #b` into `#b + 1 < #b ‚ą® #b + 1 = #b`, with the first disjunct chaining with `#b < #b + 1` through NAT-order's transitivity of `<` into the strict `#b + 1 < #b + 1` and the second substituting `#b + 1 = #b` into `#b < #b + 1` to the same conclusion; NAT-order's irreflexivity at `n = #b + 1` rules out `#b + 1 < #b + 1`. Both sub-cases fail, so case (ii) cannot apply, and we are in Divergence case (i): position `j` is shared by both tumblers (`1 ‚Č§ j ‚ąß j ‚Č§ #a ‚ąß j ‚Č§ #b`), `a‚Īľ ‚Č† b‚Īľ`, and `aŠĶĘ = bŠĶĘ` for all `1 ‚Č§ i < j`. The precondition `a < b` supplies some T1 witness, which we must align with `j` before reading off `a‚Īľ < b‚Īľ`. We first rule out T1 case (ii) for `a < b`: that disjunct would require `a` to be a proper prefix of `b` ‚ÄĒ a witness `k' = #a + 1 ‚Č§ #b` with `aŠĶĘ = bŠĶĘ` for all `1 ‚Č§ i < k'`, equivalently `aŠĶĘ = bŠĶĘ` for all `1 ‚Č§ i ‚Č§ #a`. Divergence case (i)'s bound `j ‚Č§ #a` places position `j` in this range, so T1 case (ii)'s agreement requirement instantiated at `i = j` would force `a‚Īľ = b‚Īľ`, contradicting Divergence case (i)'s `a‚Īľ ‚Č† b‚Īľ`. T1 case (ii) is thereby foreclosed, and `a < b` holds via T1 case (i): there exists `k'` with `1 ‚Č§ k' ‚ąß k' ‚Č§ #a ‚ąß k' ‚Č§ #b`, `a‚āĖ' < b‚āĖ'`, and `aŠĶĘ = bŠĶĘ` for all `1 ‚Č§ i < k'`. NAT-order's trichotomy at the index pair `(k', j)` supplies exactly one of `k' < j`, `k' = j`, or `k' > j`. In the `k' < j` branch, Divergence case (i)'s prior-position agreement `aŠĶĘ = bŠĶĘ` for all `1 ‚Č§ i < j` instantiated at `i = k'` (with `1 ‚Č§ k'` from T1 case (i)'s witness bound and `k' < j` from the trichotomy branch) gives `a‚āĖ' = b‚āĖ'`; substituting this equality into T1 case (i)'s strict inequality `a‚āĖ' < b‚āĖ'` yields `a‚āĖ' < a‚āĖ'`, contradicted by NAT-order's irreflexivity instantiated at `n = a‚āĖ'`. In the `k' > j` branch, T1 case (i)'s agreement requirement `aŠĶĘ = bŠĶĘ` for all `1 ‚Č§ i < k'` instantiated at `i = j` (with `1 ‚Č§ j` from Divergence case (i) and `j < k'` from the trichotomy branch) gives `a‚Īľ = b‚Īľ`, contradicting Divergence case (i)'s `a‚Īľ ‚Č† b‚Īľ`. Only the `k' = j` branch survives, at which T1 case (i)'s strict inequality at position `k' = j` delivers `a‚Īľ < b‚Īľ`.

Recall TumblerAdd's constructive definition: for any tumbler `x` and positive displacement `w` with action point `k ‚Č§ #x`, the result `x ‚äē w` is built component-wise as `(x ‚äē w)ŠĶĘ = xŠĶĘ` for `i < k` (prefix copy), `(x ‚äē w)‚āĖ = x‚āĖ + w‚āĖ` (single-component advance), and `(x ‚äē w)ŠĶĘ = wŠĶĘ` for `i > k` (tail from displacement). Two cases arise from the relationship between `k` and `j`.

*Case 1: `k = j`.* For `i < k`: since `i < j`, the Divergence agreement property gives `aŠĶĘ = bŠĶĘ`, and TumblerAdd's prefix-copy rule gives `(a ‚äē w)ŠĶĘ = aŠĶĘ = bŠĶĘ = (b ‚äē w)ŠĶĘ`. At position `k = j`: TumblerAdd gives `(a ‚äē w)‚āĖ = a‚āĖ + w‚āĖ` and `(b ‚äē w)‚āĖ = b‚āĖ + w‚āĖ`. First, NAT-order's defining clause `m ‚Č§ n ‚üļ m < n ‚ą® m = n` weakens `a‚āĖ < b‚āĖ` to the non-strict `a‚āĖ ‚Č§ b‚āĖ`. Second, NAT-addcompat's right order-compatibility (`n ‚Č• p ‚üĻ n + m ‚Č• p + m`, instantiated at `m = w‚āĖ`, `n = b‚āĖ`, `p = a‚āĖ`) lifts `a‚āĖ ‚Č§ b‚āĖ` to `a‚āĖ + w‚āĖ ‚Č§ b‚āĖ + w‚āĖ`. Third, to sharpen this non-strict ‚Č§ back to strict we rule out the equality `a‚āĖ + w‚āĖ = b‚āĖ + w‚āĖ`: NAT-cancel's right cancellation (`n + m = p + m ‚üĻ n = p`, instantiated at `m = w‚āĖ`, `n = a‚āĖ`, `p = b‚āĖ`) would extract `a‚āĖ = b‚āĖ` from that equality, contradicting `a‚āĖ < b‚āĖ` via NAT-order's irreflexivity (`¬¨(a‚āĖ < a‚āĖ)`). Fourth, NAT-order's defining clause, applied to the non-strict `a‚āĖ + w‚āĖ ‚Č§ b‚āĖ + w‚āĖ` together with the non-equality just discharged, yields the strict `a‚āĖ + w‚āĖ < b‚āĖ + w‚āĖ`. The results agree on all positions before `k` and diverge strictly at `k`. T1 case (i) additionally requires the length bounds `k ‚Č§ #(a ‚äē w)` and `k ‚Č§ #(b ‚äē w)`: ActionPoint's defining postcondition `1 ‚Č§ actionPoint(w) ‚Č§ #w` supplies its upper clause `k ‚Č§ #w`, and TA0's result-length identity `#(a ‚äē w) = #w` ‚ÄĒ engaged once under the precondition `actionPoint(w) ‚Č§ #a`, symmetrically once under `actionPoint(w) ‚Č§ #b` to obtain `#(b ‚äē w) = #w` ‚ÄĒ rewrites `k ‚Č§ #w` into the required `k ‚Č§ #(a ‚äē w)` and `k ‚Č§ #(b ‚äē w)`. By T1 case (i), `a ‚äē w < b ‚äē w`.

*Case 2: `k > j`.* For `i < k`: TumblerAdd's prefix-copy rule gives `(a ‚äē w)ŠĶĘ = aŠĶĘ` and `(b ‚äē w)ŠĶĘ = bŠĶĘ`. T1 case (i) additionally requires the length bounds `j ‚Č§ #(a ‚äē w)` and `j ‚Č§ #(b ‚äē w)`: the case assumption `k > j` composed with ActionPoint's upper clause `k ‚Č§ #w` through NAT-order's transitivity of `<` on ‚Ąē ‚ÄĒ applied to the strict disjunct `k < #w` of the unpacked bound, with the equality disjunct `k = #w` resolving by substitution of `k = #w` into `k > j` to yield `j < #w` directly rather than through a second transitivity step ‚ÄĒ produces `j < #w`; NAT-order's defining clause `m ‚Č§ n ‚üļ m < n ‚ą® m = n` instantiated at `m = j, n = #w` weakens this to `j ‚Č§ #w`; and TA0's result-length identity `#(a ‚äē w) = #w` ‚ÄĒ engaged once under the precondition `actionPoint(w) ‚Č§ #a`, symmetrically once under `actionPoint(w) ‚Č§ #b` to obtain `#(b ‚äē w) = #w` ‚ÄĒ rewrites `j ‚Č§ #w` into the required `j ‚Č§ #(a ‚äē w)` and `j ‚Č§ #(b ‚äē w)`.
