**Definition (ZeroPaddedDivergence).** For tumblers `a, w ∈ T`, the *zero-padded divergence* `zpd(a, w)` is defined over the zero-padded extensions of both operands to a common length `L`, selected by NAT-order's trichotomy applied to the length pair `(#a, #w)`: in sub-case (α) `#a = #w`, `L = #a` (equivalently `#w`) and no padding is required; in sub-case (β) `#a < #w`, `L = #w` and `a` is extended by setting `aᵢ = 0` for `#a < i ≤ L`; in sub-case (γ) `#w < #a`, `L = #a` and `w` is extended by setting `wᵢ = 0` for `#w < i ≤ L`. Trichotomy's exactly-one clause ensures precisely one sub-case applies, so `L` — the longer of `#a` and `#w` — is named by the selected sub-case without appeal to any primitive binary-maximum operator on ℕ, matching Divergence's analogous length-pair dispatch. If the (possibly padded) sequences agree at every position — `(A i : 1 ≤ i ≤ L : aᵢ = wᵢ)` — then `zpd(a, w)` is *undefined*; we say `a` and `w` are *zero-padded-equal*. Otherwise, `zpd(a, w)` is the least `k` with `1 ≤ k ≤ L` such that `aₖ ≠ wₖ` and `(A i : 1 ≤ i < k : aᵢ = wᵢ)` — the first position at which the (possibly padded) sequences disagree.

The function is partial. It is undefined precisely when the padded sequences agree everywhere — as occurs when one operand is a proper prefix of the other with all trailing components zero (e.g., `a = [3, 0]` and `w = [3]` pad to the identical sequence `[3, 0]`). Two equal tumblers are trivially zero-padded-equal, so `zpd(a, a)` is always undefined.

**Relationship to Divergence.** When `a ≠ w`, the formal Divergence and `zpd` may disagree. NAT-order's trichotomy on `(#a, #w)` partitions the analysis into the same three sub-cases (α), (β), (γ) as the Definition. In Divergence case (i) — component divergence at a shared position `k` with `k ≤ #a ∧ k ≤ #w` — no padding is consulted at positions `1, ..., k`, since both operands have actual values there; hence `zpd(a, w) = divergence(a, w) = k`. The shared-position bound is stated as the conjunction `k ≤ #a ∧ k ≤ #w` rather than `k ≤ min(#a, #w)` so that no primitive binary-minimum operator on ℕ is invoked, and the conclusion holds uniformly across sub-cases (α), (β), and (γ) since the conjunction is independent of which of `#a`, `#w` is smaller. In Divergence case (ii) — one tumbler is a proper prefix of the other, which requires `#a ≠ #w` and therefore falls in sub-case (β) or (γ), excluding (α). In sub-case (β), where `#a < #w`, Divergence reports the boundary `#a + 1` (replacing `min(#a, #w) + 1` for this sub-case), whereas `zpd` scans through the zero-padded values `aᵢ = 0` for `#a < i ≤ #w`: if `w` has a nonzero component at some position `i` with `#a < i ≤ #w`, then `zpd(a, w)` equals the least such `i`, which is at least `#a + 1 = divergence(a, w)`; if all components of `w` at positions `#a < i ≤ #w` are zero, the padded sequences agree everywhere and `zpd(a, w)` is undefined. In sub-case (γ), where `#w < #a`, the situation is symmetric: Divergence reports the boundary `#w + 1`, `w` is zero-padded beyond position `#w`, and the same dichotomy applies to the components of `a` at positions `#w < i ≤ #a`. In either sub-case the distinction reduces to whether the longer operand carries any nonzero component past the shorter operand's last position.

*Formal Contract:*
- *Domain:* a ∈ T, w ∈ T
- *Definition:* NAT-order's trichotomy on `(#a, #w)` selects exactly one of: (α) `#a = #w`, `L = #a`, no padding; (β) `#a < #w`, `L = #w`, `aᵢ = 0` for `#a < i ≤ L`; (γ) `#w < #a`, `L = #a`, `wᵢ = 0` for `#w < i ≤ L`. If `(A i : 1 ≤ i ≤ L : aᵢ = wᵢ)`, `zpd(a, w)` is undefined. Otherwise, `zpd(a, w) = min {k : 1 ≤ k ≤ L ∧ aₖ ≠ wₖ}`.
- *Depends:* T0 (CarrierSetDefinition) — the membership `a, w ∈ T` in the Domain, together with the lengths `#a`, `#w` (used in the trichotomy case-split that selects the common length `L` and in the index conditions `#a < i ≤ L`, `#w < i ≤ L` that delimit the padding region) and the component projections `aᵢ`, `wᵢ`, `aₖ`, `wₖ` appearing in the padding equations and the disagreement predicate, all come from T0's characterisation of T as finite sequences over ℕ with length ≥ 1; the disagreement predicate `aₖ ≠ wₖ` (and the agreement predicate `aᵢ = wᵢ`) rests on T0's stipulation that each component is ℕ-valued, making the (in)equality a well-formed comparison in ℕ rather than an uninterpreted symbol — this citation discharges the carrier-set, length, and component-projection operators from T0 rather than leaving them implicit, matching the per-step convention established in ActionPoint, Divergence, and TA-Pos. NAT-zero (NatZeroMinimum) — the padding values assigned by `aᵢ = 0` for `#a < i ≤ L` (sub-case (β)) and `wᵢ = 0` for `#w < i ≤ L` (sub-case (γ)) must themselves lie in ℕ so that T0's component-typing requirement — projected components are ℕ-valued — extends uniformly to the padded sequences on which the agreement predicate `(A i : 1 ≤ i ≤ L : aᵢ = wᵢ)`, the disagreement predicate `aₖ ≠ wₖ`, and the `min{k : 1 ≤ k ≤ L ∧ aₖ ≠ wₖ}` selection all operate. T0 supplies component-membership in ℕ only for *existing* positions `1 ≤ i ≤ #a`, `1 ≤ i ≤ #w`, and its companion notation `ℕ⁺ = {n ∈ ℕ : n > 0}` expressly excludes zero from the strictly-positive subcarrier, so `0 ∈ ℕ` is not among T0's assertions; among the remaining NAT-* axioms named in T0's exhaustive list (NAT-closure, NAT-order, NAT-discrete, NAT-addcompat, NAT-wellorder, NAT-sub, NAT-cancel, NAT-addassoc) none identifies which elements belong to ℕ — each quantifies over ℕ presupposing its extension rather than fixing it. NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` quantifies `0` as a lower bound of ℕ and thereby presupposes `0 ∈ ℕ` as a member of its quantification domain, supplying exactly the missing membership premise: the literal `0` assigned in each padding equation is an element of ℕ by NAT-zero, and the padded sequences consequently respect T0's component-typing — this citation discharges the padding-value operator from NAT-zero rather than leaving `0 ∈ ℕ` implicit, matching the per-step convention ActionPoint and TA-Pos already follow for discharging `0`-related premises in ℕ from NAT-zero rather than T0. NAT-order (NatStrictTotalOrder) — invoked at the single length pair `(#a, #w)` in two coupled roles, both replacing uses of primitive binary operators on ℕ that are not declared in the Depends list. *Internal to the Definition*: trichotomy's exactly-one clause discharges the common-length selection, leaving exactly one of `#a = #w` (sub-case (α), `L = #a`, no padding), `#a < #w` (sub-case (β), `L = #w`, with `a` zero-padded from position `#a + 1` to `L`), or `#w < #a` (sub-case (γ), `L = #a`, with `w` zero-padded from position `#w + 1` to `L`); `L` — the longer of `#a` and `#w` — is named by the selected sub-case rather than by an un-axiomatized `max(#a, #w)`, matching Divergence's analogous length-pair dispatch for the shorter length and the convention that every operator in the Definition must be discharged by a Depends entry. *Internal to the Relationship-to-Divergence postcondition*: the same trichotomy apparatus dispatches both references to the shorter length — Divergence case (i)'s shared-position bound is stated as the conjunction `k ≤ #a ∧ k ≤ #w` (replacing `k ≤ min(#a, #w)`), and Divergence case (ii)'s boundary takes the value `#a + 1` under sub-case (β) or `#w + 1` under sub-case (γ) (replacing `min(#a, #w) + 1`), with sub-case (α) excluded because prefix divergence requires `#a ≠ #w`. Both roles are instances of the same NAT-order axiom applied at the single length-pair `(#a, #w)` and are enumerated together under this one citation. NAT-wellorder (NatWellOrdering) — the Definition's `min{k : 1 ≤ k ≤ L ∧ aₖ ≠ wₖ}` step selects `k` as the least index at which the padded sequences disagree; the least-element principle applied to the nonempty subset `{k ∈ ℕ : 1 ≤ k ≤ L ∧ aₖ ≠ wₖ} ⊆ ℕ` (nonempty precisely on the `Otherwise` branch, where the universal agreement `(A i : 1 ≤ i ≤ L : aᵢ = wᵢ)` fails and so some `k` in the range falsifies it) supplies that minimum, making `zpd(a, w)` well-defined as a function (rather than a relation) on the defined branch — matching the per-step convention ActionPoint and Divergence apply to structurally identical `min(·)` constructions over nonempty subsets of ℕ. Divergence (Divergence) — the Relationship to Divergence postconditions consume Divergence's two-case structure (case (i): component divergence at `k` with `k ≤ #a ∧ k ≤ #b`, case (ii): prefix divergence at `#a + 1` in sub-case (ii-a) or `#b + 1` in sub-case (ii-b) per NAT-order trichotomy at `(#a, #b)`) and its domain restriction `a ≠ b`, which gates the postcondition's guard `a ≠ w`.
- *Codomain:* When defined, `zpd(a, w) ∈ {1, ..., L}`, where `L` is the common length selected by NAT-order trichotomy on `(#a, #w)`: `L = #a` in sub-cases (α) and (γ), `L = #w` in sub-case (β).
- *Partiality:* zpd(a, w) is undefined iff a and w are zero-padded-equal.
- *Postconditions (Symmetry):* `zpd(a, w)` is defined iff `zpd(w, a)` is defined, and when defined, `zpd(a, w) = zpd(w, a)`. Padding both operands to the common length `L` selected by NAT-order trichotomy on `(#a, #w)` is symmetric under exchange — sub-case (α) is self-symmetric, while sub-cases (β) and (γ) swap under exchange of operands and both yield the same `L` (the longer of `#a`, `#w`) — and the disagreement predicate is symmetric: `aₖ ≠ wₖ` iff `wₖ ≠ aₖ`.
- *Postconditions (Relationship to Divergence):* For `a ≠ w`: in Divergence case (i) — component divergence at `k` with `k ≤ #a ∧ k ≤ #w` (the conjunction replacing `k ≤ min(#a, #w)`) — both operands possess actual values at positions `1, ..., k`, so no padding is consulted and `zpd(a, w) = divergence(a, w)`; in Divergence case (ii) — one operand is a proper prefix of the other, occurring under NAT-order trichotomy sub-case (β) (`#a < #w`) or (γ) (`#w < #a`) — if the longer operand has a nonzero component at some position beyond the shorter operand's last (position `#a` under (β) or position `#w` under (γ)), then `zpd(a, w)` is defined and `zpd(a, w) ≥ divergence(a, w)` (≥ `#a + 1` under (β), ≥ `#w + 1` under (γ), each replacing `min(#a, #w) + 1` for its sub-case); if all such trailing components are zero, then `zpd(a, w)` is undefined (the operands are zero-padded-equal despite being distinct).
