**TA1 (OrderPreservationUnderAddition).** `(A a, b, w : a < b ‚àß Pos(w) ‚àß actionPoint(w) ‚â§ min(#a, #b) : a ‚äï w ‚â§ b ‚äï w)`.

TA1 guarantees weak (`‚â§`) order preservation universally ‚Äî if two positions were in order before advancement, they remain in non-reversed order after. The precondition `actionPoint(w) ‚â§ min(#a, #b)` inherits from TA0: both operations must be well-defined.

*Proof.* We show that for all `a, b, w ‚àà T` with `a < b`, `Pos(w)`, and `actionPoint(w) ‚â§ min(#a, #b)`, the advanced positions satisfy `a ‚äï w ‚â§ b ‚äï w`.

Let `k` be the action point of `w`. By TumblerAdd, the operation `‚äï` builds the result in three regions: for `i < k`, `(a ‚äï w)·µ¢ = a·µ¢` (copy from start); at `i = k`, `(a ‚äï w)‚Çñ = a‚Çñ + w‚Çñ` (advance); for `i > k`, `(a ‚äï w)·µ¢ = w·µ¢` (copy from displacement). By TA0, both `a ‚äï w` and `b ‚äï w` are well-defined members of `T` with length `#w`, since `k ‚â§ min(#a, #b)` ensures the action point falls within both operands. The same three rules apply to `b ‚äï w`.

Since `a < b`, T1 provides exactly two cases: either (i) there exists a least position `j` with `j ‚â§ min(#a, #b)` where `a‚±º < b‚±º` and `a·µ¢ = b·µ¢` for all `i < j`, or (ii) `a` is a proper prefix of `b`, that is, `#a < #b` and `a·µ¢ = b·µ¢` for all `1 ‚â§ i ‚â§ #a`.

*Case (ii): `a` is a proper prefix of `b`.* Here `min(#a, #b) = #a`, so `k ‚â§ #a`. Since `a·µ¢ = b·µ¢` for all `1 ‚â§ i ‚â§ #a` and `k ‚â§ #a`, the two start positions agree at every position that TumblerAdd consults: for `i < k`, `(a ‚äï w)·µ¢ = a·µ¢ = b·µ¢ = (b ‚äï w)·µ¢`; at `i = k`, `(a ‚äï w)‚Çñ = a‚Çñ + w‚Çñ = b‚Çñ + w‚Çñ = (b ‚äï w)‚Çñ` since `a‚Çñ = b‚Çñ`; for `i > k`, `(a ‚äï w)·µ¢ = w·µ¢ = (b ‚äï w)·µ¢`. Both results have length `#w` by TA0. Every component agrees, so `a ‚äï w = b ‚äï w` by T3, satisfying `‚â§`.

*Case (i): component divergence at position `j`.* Here `j ‚â§ min(#a, #b)`, `a‚±º < b‚±º`, and `a·µ¢ = b·µ¢` for all `i < j`. Three sub-cases arise from the relationship between the first divergence `j` and the action point `k`.

*Sub-case j < k:* Position `j` falls in the prefix-copy region of TumblerAdd, so `(a ‚äï w)‚±º = a‚±º` and `(b ‚äï w)‚±º = b‚±º`, giving `(a ‚äï w)‚±º = a‚±º < b‚±º = (b ‚äï w)‚±º`. For all `i < j`, we have `i < j < k`, so both results are in the prefix-copy region and `(a ‚äï w)·µ¢ = a·µ¢ = b·µ¢ = (b ‚äï w)·µ¢` by the agreement-before-divergence property. Position `j` witnesses T1 case (i): `a ‚äï w < b ‚äï w`.

*Sub-case j = k:* For all `i < k = j`, both results are in the prefix-copy region and `(a ‚äï w)·µ¢ = a·µ¢ = b·µ¢ = (b ‚äï w)·µ¢` by agreement before the divergence. At position `k`, `(a ‚äï w)‚Çñ = a‚Çñ + w‚Çñ` and `(b ‚äï w)‚Çñ = b‚Çñ + w‚Çñ`. Since `a‚Çñ < b‚Çñ` (the divergence at `j = k`), NAT-addcompat's order-compatibility of addition with NAT-order's trichotomy promotes the strict ‚Ñï-inequality through the shared increment `w‚Çñ`, yielding `a‚Çñ + w‚Çñ < b‚Çñ + w‚Çñ`. Position `k` witnesses T1 case (i): `a ‚äï w < b ‚äï w`.

*Sub-case j > k:* Since `k < j` and `a·µ¢ = b·µ¢` for all `i < j`, in particular `a‚Çñ = b‚Çñ` (because `k < j`). For `i < k`: both results are in the prefix-copy region, and `i < k < j` gives `a·µ¢ = b·µ¢`, so `(a ‚äï w)·µ¢ = a·µ¢ = b·µ¢ = (b ‚äï w)·µ¢`. At position `k`: `(a ‚äï w)‚Çñ = a‚Çñ + w‚Çñ = b‚Çñ + w‚Çñ = (b ‚äï w)‚Çñ` since `a‚Çñ = b‚Çñ`. For `i > k`: both results copy from the displacement, so `(a ‚äï w)·µ¢ = w·µ¢ = (b ‚äï w)·µ¢`. Both results have length `#w` by TA0. Every component agrees, so `a ‚äï w = b ‚äï w` by T3, satisfying `‚â§`.

In every case and sub-case, `a ‚äï w ‚â§ b ‚äï w`. ‚àé
