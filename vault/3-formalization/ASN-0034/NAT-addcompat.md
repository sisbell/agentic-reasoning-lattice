**NAT-addcompat (NatAdditionOrderAndSuccessor).** Addition on ‚Ąē is compatible with the order on either side, and `n < n + 1` for every `n ‚ąą ‚Ąē`.

Three related facts are stated together as one axiom because they all concern the interaction between addition and `<` on ‚Ąē:

- Left order compatibility: `n ‚Č• p ‚üĻ m + n ‚Č• m + p` for every `m, n, p ‚ąą ‚Ąē`. Adding the same natural number on the left preserves the order.
- Right order compatibility: `n ‚Č• p ‚üĻ n + m ‚Č• p + m` for every `m, n, p ‚ąą ‚Ąē`. Adding the same natural number on the right preserves the order.
- Strict successor inequality: `n < n + 1` for every `n ‚ąą ‚Ąē`. The successor is strictly greater.

Both order-compatibility forms are stated explicitly because downstream proofs supply the fixed summand on either side: TumblerAdd, TA5, T10a-N, and GlobalUniqueness Case 5 add the fixed term on the left of the variable summands to lift non-strict inequalities (for example, TA5 and T10a-N lift `1 ‚Č§ k` to the length-level `m + 1 ‚Č§ m + k`), while TA1's sub-case `j = k`, TA1-strict's Case 1, and GlobalUniqueness Case 5 add the fixed increment on the right of the variable summands, also to lift non-strict inequalities (for example, TA1 lifts the weakened `a‚āĖ ‚Č§ b‚āĖ` to `a‚āĖ + w‚āĖ ‚Č§ b‚āĖ + w‚āĖ`). Both order-compatibility clauses deliver *only non-strict* conclusions: the hypothesis `n ‚Č• p` admits the equality `n = p` and the conclusion `m + n ‚Č• m + p` (resp. `n + m ‚Č• p + m`) inherits that possibility symmetrically, so neither clause by itself lifts the strict `a‚āĖ < b‚āĖ` into the strict `a‚āĖ + w‚āĖ < b‚āĖ + w‚āĖ`. The strict promotion through addition that TA1's sub-case `j = k` requires is therefore a four-step chain consuming NAT-order, NAT-addcompat, and NAT-cancel together: NAT-order's defining clause `m ‚Č§ n ‚üļ m < n ‚ą® m = n` weakens `a‚āĖ < b‚āĖ` to the non-strict `a‚āĖ ‚Č§ b‚āĖ`; NAT-addcompat's right compatibility lifts that non-strict `‚Č§` to `a‚āĖ + w‚āĖ ‚Č§ b‚āĖ + w‚āĖ`; NAT-cancel's right cancellation (`n + m = p + m ‚üĻ n = p`, instantiated at `m = w‚āĖ`, `n = a‚āĖ`, `p = b‚āĖ`) rules out the equality `a‚āĖ + w‚āĖ = b‚āĖ + w‚āĖ`, because that equality would yield `a‚āĖ = b‚āĖ` and contradict `a‚āĖ < b‚āĖ` via NAT-order's irreflexivity; and NAT-order's defining clause, applied to the non-strict `a‚āĖ + w‚āĖ ‚Č§ b‚āĖ + w‚āĖ` together with the just-discharged non-equality, returns the strict `a‚āĖ + w‚āĖ < b‚āĖ + w‚āĖ`. TA1, TA1-strict, and GlobalUniqueness Case 5 therefore cite NAT-addcompat, NAT-cancel, and NAT-order together at their strict-promotion sites ‚ÄĒ no downstream Depends list attributes a strict-inequality-through-addition step to NAT-addcompat alone. The ASN's convention (T0) is that each proof cites only the ‚Ąē facts it actually uses; without the right-addition clause stated as an axiom, TA1's right-added non-strict lift would tacitly assume commutativity of addition on ‚Ąē, breaking the citation policy that the other NAT-* axioms enforce. T0(a) and T1 use only the strict successor inequality clause directly ‚ÄĒ at `M + 1 > M` and `m < m + 1` respectively ‚ÄĒ without invoking either order-compatibility form. Downstream proofs use the order-compatibility facts to lift non-strict inequalities through arithmetic and the strict successor inequality to bound lengths or indices under incrementing operations.

*Formal Contract:*
- *Axiom:* `(A m, n, p ‚ąą ‚Ąē : n ‚Č• p : m + n ‚Č• m + p)` (left order compatibility); `(A m, n, p ‚ąą ‚Ąē : n ‚Č• p : n + m ‚Č• p + m)` (right order compatibility); `(A n ‚ąą ‚Ąē :: n < n + 1)` (strict successor inequality).
