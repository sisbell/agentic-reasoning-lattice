**OrdinalShift (OrdinalShift).** For a tumbler v ∈ T of length m = #v and natural number n ≥ 1:

`shift(v, n) = v ⊕ δ(n, m)`

We verify the four preconditions of TA0 for this application. First, v ∈ T by assumption. Before invoking OrdinalDisplacement at `(n, m)` we discharge its four preconditions one-for-one: `n ∈ ℕ` transfers directly from OrdinalShift's own precondition `n ∈ ℕ`; `m ∈ ℕ` follows from T0's length operator typing `#·: T → ℕ` instantiated at `v ∈ T`, placing `m = #v ∈ ℕ`; `n ≥ 1` transfers directly from OrdinalShift's own precondition `n ≥ 1`; and `m ≥ 1` follows from T0's length axiom `#a ≥ 1 for all a ∈ T` instantiated at `a = v ∈ T`, yielding `m = #v ≥ 1`. Second, by OrdinalDisplacement, δ(n, m) = [0, …, 0, n] is a finite sequence of length m ≥ 1 over ℕ, so by T0, δ(n, m) ∈ T. Third, since n ≥ 1 the m-th component of δ(n, m) is nonzero, whence by TA-Pos Pos(δ(n, m)). Fourth, the action point of δ(n, m) is m = #v by OrdinalDisplacement, so actionPoint(δ(n, m)) ≤ #v holds trivially.

With all TA0 preconditions satisfied, shift(v, n) = v ⊕ δ(n, m) ∈ T. By TumblerAdd: shift(v, n)ᵢ = vᵢ for i < m, shift(v, n)ₘ = vₘ + n, and #shift(v, n) = #δ(n, m) = m = #v. The shift advances the deepest component by exactly n, leaving all higher-level components unchanged.

Additionally, shift preserves structural properties. When m ≥ 2, the action point of δₙ leaves position 1 unchanged — shift(v, n)₁ = v₁. When m = 1, letting c denote the sole component so that v = [c] with c ∈ ℕ, shift([c], n) = [c + n] changes the first component. The component-positivity postcondition `shift(v, n)ₘ = vₘ + n ≥ 1` is an arithmetic lower bound on ℕ — TumblerAdd supplies only the equation `shift(v, n)ₘ = vₘ + n`, not any bound on the sum — and is discharged per step. T0's carrier characterisation applied to v ∈ T places `vₘ ∈ ℕ`; NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` instantiated at `n = vₘ` supplies `0 ≤ vₘ`; NAT-addcompat's right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, with axiom-`n = vₘ`, axiom-`p = 0`, and axiom-`m = n` — the shift amount filling the axiom's fixed right summand) lifts `0 ≤ vₘ` into the non-strict `vₘ + n ≥ 0 + n`; NAT-closure's additive identity `0 + n = n` rewrites the right-hand side, yielding `vₘ + n ≥ n`; and NAT-order composes this non-strict inequality with the precondition `n ≥ 1` into `vₘ + n ≥ 1` — NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = n, n = vₘ + n` unfolds `vₘ + n ≥ n` into `vₘ + n > n ∨ vₘ + n = n`, the same clause at `m = 1, n = n` unfolds `n ≥ 1` into `n > 1 ∨ n = 1`, the three cases with at least one strict disjunct (both strict chaining through NAT-order's transitivity of `<`, and each mixed case substituting the equality into the strict) yield the strict `vₘ + n > 1`, which the defining clause at `m = 1, n = vₘ + n` folds back into the non-strict `vₘ + n ≥ 1` via its strict disjunct, and the all-equal case substitutes both equalities to yield `vₘ + n = 1`, which the same defining clause folds into the non-strict via its equality disjunct. NAT-addcompat's order-compatibility clauses deliver *only* non-strict conclusions — as NAT-addcompat's own contract articulates — so the composition of the two non-strict inequalities `vₘ + n ≥ n` and `n ≥ 1` into the non-strict `vₘ + n ≥ 1` is routed through NAT-order rather than NAT-addcompat alone, matching the per-step citation discipline TumblerAdd applies at its analogous non-strict-arithmetic composition sites. ∎
