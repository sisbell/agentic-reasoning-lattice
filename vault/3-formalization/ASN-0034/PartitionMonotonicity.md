**PartitionMonotonicity (PartitionMonotonicity).** Within any prefix-delimited partition of the address space, the set of allocated addresses is totally ordered by T1, and this order is consistent with the allocation order of any single allocator within that partition. Moreover, for any two sibling sub-partitions with non-nesting prefixes `p‚āĀ < p‚āā`, every address extending `p‚āĀ` precedes every address extending `p‚āā` under T1 ‚ÄĒ the per-allocator ordering extends to a cross-allocator ordering determined by the prefix structure.

*Proof.* We must show that within a prefix-delimited partition, allocated addresses are totally ordered by T1 consistently with allocation order, and that for sibling sub-partition prefixes `p‚āĀ < p‚āā` satisfying the non-nesting condition, every address extending `p‚āĀ` precedes every address extending `p‚āā`.

**Partition structure.** Consider a partition with prefix `p`. Every allocated address `a` in this partition satisfies `p ‚Čľ a`, placing it in the set `{t ‚ąą T : p ‚Čľ t}`. By T5 (prefix convexity), this set forms a contiguous interval under T1: if `p ‚Čľ a`, `p ‚Čľ c`, and `a ‚Č§ b ‚Č§ c`, then `p ‚Čľ b`. No address from outside the partition can interleave between two addresses inside it.

Within the partition, the parent allocator spawns a child allocator according to T10a (allocator discipline). The child prefix `t‚āÄ` is produced by the parent's single `inc(s, k)` with `k > 0`, where `s` is a parent sibling extending `p`; by TA5(d), `#t‚āÄ = #s + k`. The parent's own sibling stream resumes independently at the parent level with `inc(s, 0)` (T10a). The child-level allocator, starting from `t‚āÄ`, produces its sibling stream via `inc(¬∑, 0)`: `t‚āĀ = inc(t‚āÄ, 0)`, `t‚āā = inc(t‚āĀ, 0)`, and so on, each serving as the prefix for a distinct sub-partition within the child's domain.

**Sibling prefixes are non-nesting.** We establish that for distinct sibling prefixes `tŠĶĘ` and `t‚Īľ` with `i ‚Č† j`: `tŠĶĘ ‚č† t‚Īľ ‚ąß t‚Īľ ‚č† tŠĶĘ`.

*Uniform length.* By TA5(c), `inc(t, 0)` preserves length: `#inc(t, 0) = #t`. Applying this inductively from `t‚āÄ` ‚ÄĒ `#t‚āĀ = #inc(t‚āÄ, 0) = #t‚āÄ`, and for each `n ‚Č• 0`, `#t‚āô‚āä‚āĀ = #inc(t‚āô, 0) = #t‚āô` ‚ÄĒ we obtain `#t‚āô = #t‚āÄ` for all `n ‚Č• 0`. Every sibling prefix has the same length.

*Distinctness.* By TA5(a), each application of `inc(¬∑, 0)` produces a strictly greater tumbler under T1, so the sibling prefix sequence is strictly increasing: `t‚āÄ < t‚āĀ < t‚āā < ...`. In particular, `tŠĶĘ ‚Č† t‚Īľ` for all `i ‚Č† j`.

*Non-nesting.* A proper prefix relationship `q ‚Čļ r` requires `#q < #r`, since T1 case (ii) defines `q < r` when `q` is a proper prefix of `r`, which demands `#q = m < n = #r`. Since `#tŠĶĘ = #t‚Īľ` (uniform length), neither can be a proper prefix of the other. The prefix relation `tŠĶĘ ‚Čľ t‚Īľ` means either `tŠĶĘ = t‚Īľ` or `tŠĶĘ ‚Čļ t‚Īľ`; we have excluded both (`tŠĶĘ ‚Č† t‚Īľ` from distinctness, `tŠĶĘ ‚Čļ t‚Īľ` from equal length). So `tŠĶĘ ‚č† t‚Īľ`, and by the symmetric argument `t‚Īľ ‚č† tŠĶĘ`.

**Cross-partition ordering.** Take two sibling sub-partition prefixes `tŠĶĘ` and `t‚Īľ` with `i < j`. From the strict monotonicity of the sibling sequence we have `tŠĶĘ < t‚Īľ`, and we have just established `tŠĶĘ ‚č† t‚Īľ ‚ąß t‚Īľ ‚č† tŠĶĘ`. These are precisely the preconditions of PrefixOrderingExtension: for every address `a` with `tŠĶĘ ‚Čľ a` and every address `b` with `t‚Īľ ‚Čľ b`, we conclude `a < b`. The prefix ordering of sub-partitions determines the address ordering across them.

**Intra-partition ordering (by structural induction).** A sub-partition with prefix `tŠĶĘ` need not contain addresses from a single allocator alone: a child allocator spawned via `inc(tŠĶĘ, k')` with `k' > 0` (T10a) produces addresses extending `tŠĶĘ`, and its own children may do likewise. *Termination.* Each child-spawning increment `inc(s, k)` with `k > 0` yields a prefix of length `#s + k ‚Č• #s + 1` (TA5(d)). Since every allocated tumbler has finite length, the nesting depth within any sub-partition is bounded.

*Base case (nesting depth 0).* A sub-partition in which no child allocator is spawned contains only the root address `tŠĶĘ` ‚ÄĒ a single element, trivially ordered.

*Inductive step.* Assume the result holds for every sub-partition of nesting depth less than `d`. Consider a sub-partition with prefix `tŠĶĘ` at nesting depth `d ‚Č• 1`. The child allocator spawned from `tŠĶĘ` produces a sibling stream `u‚āÄ, u‚āĀ, u‚āā, ...` via `inc(¬∑, 0)`, where `u‚āÄ = inc(tŠĶĘ, k')` for some `k' > 0`. Within each sub-sub-partition, the nesting depth is less than `d`, so by the induction hypothesis it is totally ordered by T1 consistently with allocation order.

The root address `tŠĶĘ` precedes every other address in the sub-partition: for any `a ‚Č† tŠĶĘ` with `tŠĶĘ ‚Čľ a`, the inequality `#a > #tŠĶĘ` (since `a` extends some `u‚Īľ` with `#u‚Īľ > #tŠĶĘ`) gives `tŠĶĘ ‚Čļ a`, whence `tŠĶĘ < a` by T1 case (ii). Combining root ordering, cross-sub-sub-partition ordering (PrefixOrderingExtension), and within-sub-sub-partition ordering (induction hypothesis), every pair of distinct addresses in the sub-partition is comparable under T1. Allocation order is consistent throughout: `tŠĶĘ` was allocated before any of its descendants (T10a temporal ordering), PrefixOrderingExtension preserves the sibling stream's allocation sequence, and the induction hypothesis secures consistency within each `{a : u‚Īľ ‚Čľ a}`.

**Total ordering.** Every address in the partition belongs to exactly one sub-partition ‚ÄĒ the unique sub-partition whose prefix it extends (uniqueness follows from non-nesting). For any two distinct allocated addresses `a` and `b` within the partition: if `a` belongs to sub-partition `tŠĶĘ` and `b` to sub-partition `t‚Īľ` with `i < j`, then `a < b` by PrefixOrderingExtension; if `i > j`, then `b < a`; if both belong to the same sub-partition `tŠĶĘ`, the structural induction above gives their ordering. In every case `a` and `b` are comparable under T1, and the ordering is consistent with allocation order ‚ÄĒ within each allocator's sibling stream by T9, across sub-partitions by PrefixOrderingExtension, and within nested sub-partitions by structural induction. ‚ąé
