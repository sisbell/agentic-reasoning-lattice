**PartitionMonotonicity (PartitionMonotonicity).** Within any prefix-delimited partition of the address space, the set of allocated addresses is totally ordered by T1, and this order is consistent with the allocation order of any single allocator within that partition. Moreover, for any two sibling sub-partitions with non-nesting prefixes `p‚āĀ < p‚āā`, every address extending `p‚āĀ` precedes every address extending `p‚āā` under T1 ‚ÄĒ the per-allocator ordering extends to a cross-allocator ordering determined by the prefix structure.

*Proof.* We must show that within a prefix-delimited partition, allocated addresses are totally ordered by T1 consistently with per-allocator allocation order, and that for sibling sub-partition prefixes `p‚āĀ < p‚āā` satisfying the non-nesting condition, every address extending `p‚āĀ` precedes every address extending `p‚āā`.

*Notation.* For any address `c ‚ąą T`, write `subtree(c) = {t ‚ąą T : c ‚Čľ t}` for the *prefix closure* of `c` ‚ÄĒ `c` itself together with every address of which `c` is a proper prefix. T10a's formal `dom(c) = {c‚āô : n ‚Č• 0}` (with `c‚āÄ = c` and `c‚āô‚āä‚āĀ = inc(c‚āô, 0)`) is a different slice of the address space: TA5(c) preserves length, so every `c‚āô` has the same length as `c`; and TA5(b)/(c) make `c‚āô` for `n ‚Č• 1` differ from `c` at position `sig(c) = #c` (TA5-SigValid, with T4 supplied by T10a.4), so `c ‚č† c‚āô` ‚ÄĒ the Prefix definition would require `#c < #c‚āô` for a proper prefix, which fails at equal length. The two sets meet only at `c` itself: `dom(c) ‚ą© subtree(c) = {c}`. Where the proof speaks of a sub-partition rooted at `c`, the prefix-closed `subtree(c)` is meant; the sibling stream `dom(c)` is invoked separately when same-length comparisons among siblings are at issue. Define the *allocator reach* of a child allocator with base `c` as `reach(c) = ‚čÉ_{s ‚ąą dom(c)} subtree(s)` ‚ÄĒ the union of prefix-closures taken over every sibling base produced by `c`'s allocator. The partition rooted at `p` (= the set of allocated addresses with `p` as prefix) is therefore *not* simply `{p} ‚ą™ subtree(c‚āĀ) ‚ą™ subtree(c‚āā)` for the children spawned from `p`: `c‚āĀ`'s siblings via `inc(¬∑, 0)` ‚ÄĒ namely `inc(c‚āĀ, 0), inc(inc(c‚āĀ, 0), 0), ‚Ä¶`, with components `2, 3, ‚Ä¶` at position `#p + 1` ‚ÄĒ lie outside `subtree(c‚āĀ)` (whose members all carry value `1` at that position) yet still satisfy `p ‚Čľ ¬∑` (TA5(b) preserves positions `1..#p` through every increment). The partition decomposes at the sibling level as `{p} ‚ą™ reach(c‚āĀ) ‚ą™ reach(c‚āā)` (when both children spawn): every allocated address with `p` as prefix is either `p` itself, or extends some sibling produced by one of the two child allocators. The proof below uses this *reach*-based decomposition wherever it needs to cover the partition exhaustively, reserving the strictly narrower `subtree(c‚āĀ)` and `subtree(c‚āā)` for arguments about the child bases themselves.

**Partition structure.** Consider a partition with prefix `p`. Every allocated address `a` in this partition satisfies `p ‚Čľ a`, placing it in `subtree(p) = {t ‚ąą T : p ‚Čľ t}`. By T5 (prefix convexity), this set forms a contiguous interval under T1: if `p ‚Čľ a`, `p ‚Čľ c`, and `a ‚Č§ b ‚Č§ c`, then `p ‚Čľ b`. No address from outside the partition can interleave between two addresses inside it.

Within the partition, the parent allocator may spawn child allocators from `p` according to T10a (allocator discipline). Since T10a constrains each `(p, k')` pair to at most one child-spawning event with `k' ‚ąą {1, 2}`, the parent spawns at most two children from `p`: a *depth-1 child* with prefix `c‚āĀ = inc(p, 1)` at length `#p + 1` (TA5(d)), and a *depth-2 child* with prefix `c‚āā = inc(p, 2)` at length `#p + 2` (TA5(d)), the latter carrying a field separator `0` at position `#p + 1`. Either or both may be absent. Each child allocator produces its sibling stream via repeated `inc(¬∑, 0)` (T10a), and the parent's own sibling stream resumes independently at the parent level.

**Sibling prefixes are non-nesting.** Within any single child allocator's sibling stream, we establish that for distinct sibling prefixes `tŠĶĘ` and `t‚Īľ` with `i ‚Č† j`: `tŠĶĘ ‚č† t‚Īľ ‚ąß t‚Īľ ‚č† tŠĶĘ`.

*Uniform length.* By TA5(c), `inc(t, 0)` preserves length: `#inc(t, 0) = #t`. Applying this inductively from `t‚āÄ` ‚ÄĒ `#t‚āĀ = #inc(t‚āÄ, 0) = #t‚āÄ`, and for each `n ‚Č• 0`, `#t‚āô‚āä‚āĀ = #inc(t‚āô, 0) = #t‚āô` ‚ÄĒ we obtain `#t‚āô = #t‚āÄ` for all `n ‚Č• 0`. Every sibling prefix has the same length.

*Distinctness.* By TA5(a), each application of `inc(¬∑, 0)` produces a strictly greater tumbler under T1, so the sibling prefix sequence is strictly increasing: `t‚āÄ < t‚āĀ < t‚āā < ...`. In particular, `tŠĶĘ ‚Č† t‚Īľ` for all `i ‚Č† j`.

*Non-nesting.* A proper prefix relationship `q ‚Čļ r` requires `#q < #r`, since T1 case (ii) defines `q < r` when `q` is a proper prefix of `r`, which demands `#q = m < n = #r`. Since `#tŠĶĘ = #t‚Īľ` (uniform length), neither can be a proper prefix of the other. By Prefix, `tŠĶĘ ‚Čľ t‚Īľ` holds iff `tŠĶĘ = t‚Īľ` or `tŠĶĘ ‚Čļ t‚Īľ`; we have excluded both (`tŠĶĘ ‚Č† t‚Īľ` from distinctness, `tŠĶĘ ‚Čļ t‚Īľ` from equal length). So `tŠĶĘ ‚č† t‚Īľ`, and by the symmetric argument `t‚Īľ ‚č† tŠĶĘ`.

**Cross-partition ordering.** Take two sibling sub-partition prefixes `tŠĶĘ` and `t‚Īľ` with `i < j`. From the strict monotonicity of the sibling sequence we have `tŠĶĘ < t‚Īľ`, and we have just established `tŠĶĘ ‚č† t‚Īľ ‚ąß t‚Īľ ‚č† tŠĶĘ`. These are precisely the preconditions of PrefixOrderingExtension: for every address `a` with `tŠĶĘ ‚Čľ a` and every address `b` with `t‚Īľ ‚Čľ b`, we conclude `a < b`. The prefix ordering of sub-partitions determines the address ordering across them.

**Cross-depth ordering.** When both a depth-1 and a depth-2 child are spawned from the same prefix `p`, every address in `reach(c‚āā)` precedes every address in `reach(c‚āĀ)` under T1. The depth-1 child's base `c‚āĀ = inc(p, 1)` has `(c‚āĀ)_{#p+1} = 1` (TA5(d) with `k = 1` sets position `#p + 1` to `1`) and length `#p + 1`; each subsequent sibling `inc(¬∑, 0)` in `c‚āĀ`'s stream modifies only position `sig(t) = #t = #p + 1` (TA5-SigValid with T10a.4 supplying T4-validity, plus TA5(c) preserving length), and TA5(c) increments the value there, yielding components `2, 3, ...` at position `#p + 1`. Every sibling `u ‚ąą dom(c‚āĀ)` therefore has `u_{#p+1} ‚Č• 1`. For a general `a ‚ąą subtree(u)` with `u ‚ąą dom(c‚āĀ)`: if `a = u`, then `a_{#p+1} = u_{#p+1} ‚Č• 1` directly; if `u ‚Čļ a`, then `a` is reached from `u` through a chain of child-spawning increments and sibling increments in descendant allocators. Each child-spawning increment `inc(s, k')` with `k' > 0` preserves positions `1..#s` (TA5(b)), and since every `s` along this chain has `#s ‚Č• #u = #p + 1`, position `#p + 1` carries through unchanged into every descendant base; each sibling increment in a descendant allocator's chain acts at position `sig ‚Č• #base > #p + 1` (TA5(c), with the descendant's base length exceeding `#p + 1`), leaving position `#p + 1` fixed at its inherited value. Hence `a_{#p+1} = u_{#p+1} ‚Č• 1`. Every address in `reach(c‚āĀ) = ‚čÉ_{u ‚ąą dom(c‚āĀ)} subtree(u)` therefore has component `‚Č• 1` at position `#p + 1`. The depth-2 child's base `c‚āā = inc(p, 2)` has `(c‚āā)_{#p+1} = 0` (the field separator of TA5(d) with `k = 2`) and length `#p + 2`; each subsequent sibling `inc(¬∑, 0)` in `c‚āā`'s stream modifies position `sig(t) = #t = #p + 2 ‚Č† #p + 1` (TA5-SigValid, TA5(c)), leaving position `#p + 1` at `0`. Every sibling `v ‚ąą dom(c‚āā)` therefore has `v_{#p+1} = 0`. For a general `b ‚ąą subtree(v)` with `v ‚ąą dom(c‚āā)`: if `b = v`, then `b_{#p+1} = 0` directly; if `v ‚Čļ b`, the same inheritance argument applies ‚ÄĒ TA5(b) copies `v_{#p+1} = 0` unchanged into every descendant base (since `#v = #p + 2 > #p + 1`, position `#p + 1` is preserved by every child-spawning increment), and subsequent sibling increments in descendant allocators act at positions strictly beyond `#p + 1`. Hence `b_{#p+1} = 0`. Every address in `reach(c‚āā) = ‚čÉ_{v ‚ąą dom(c‚āā)} subtree(v)` therefore has component `0` at position `#p + 1`. Both reaches extend `p`, agreeing with `p` on positions `1, ..., #p` (TA5(b) propagates these through every increment). At position `#p + 1`: for any `a ‚ąą reach(c‚āĀ)` and `b ‚ąą reach(c‚āā)`, `b_{#p+1} = 0 < 1 ‚Č§ a_{#p+1}`. By T1 case (i), `b < a` ‚ÄĒ every address in `reach(c‚āā)` precedes every address in `reach(c‚āĀ)`.

**Intra-partition ordering (by structural induction).** A sub-partition with prefix `tŠĶĘ` may contain addresses from multiple allocators: the parent allocator may spawn up to two child allocators via `inc(tŠĶĘ, k')` with `k' ‚ąą {1, 2}` (T10a), each producing addresses extending `tŠĶĘ`, and their own children may do likewise. We prove total ordering within each sub-partition by induction on the depth of allocator nesting.

*Termination.* Each child-spawning increment `inc(s, k)` with `k > 0` yields a prefix of length `#s + k ‚Č• #s + 1` (TA5(d)). Since every allocated tumbler has finite length, the nesting depth within any sub-partition is bounded.

*Base case (nesting depth 0).* A sub-partition in which no child allocator is spawned contains only the root address `tŠĶĘ` ‚ÄĒ a single element, trivially ordered.

*Inductive step.* Assume the result holds for every sub-partition of nesting depth less than `d`. Consider a sub-partition with prefix `tŠĶĘ` at nesting depth `d ‚Č• 1`. The parent allocator spawns one or two child allocators from `tŠĶĘ`: a depth-1 child via `inc(tŠĶĘ, 1)` and a depth-2 child via `inc(tŠĶĘ, 2)`, one or both present as permitted by T10a. Each child allocator present produces a sibling stream via `inc(¬∑, 0)`, and each sibling `u‚Īľ` in such a stream heads a sub-sub-partition `subtree(u‚Īľ) = {a ‚ąą T : u‚Īľ ‚Čľ a}`. Within each child's stream, the prefixes have uniform length (TA5(c)) and are strictly increasing (TA5(a)), hence non-nesting (as established above); by PrefixOrderingExtension, for `j‚āĀ < j‚āā`, every address extending `u‚Īľ‚āĀ` precedes every address extending `u‚Īľ‚āā`. When two child allocators are present, the cross-depth ordering (applied with `tŠĶĘ` in place of `p`) establishes that every address in the depth-2 child's reach precedes every address in the depth-1 child's reach ‚ÄĒ that is, every address descended from any sibling of the depth-2 child precedes every address descended from any sibling of the depth-1 child. Within each sub-sub-partition, the nesting depth is less than `d`, so by the induction hypothesis it is totally ordered by T1 consistently with per-allocator allocation order.

The root address `tŠĶĘ` precedes every other address in the sub-partition: for any `a ‚Č† tŠĶĘ` with `tŠĶĘ ‚Čľ a`, the inequality `#a > #tŠĶĘ` (since `a` extends some child prefix with length exceeding `#tŠĶĘ`) gives `tŠĶĘ ‚Čļ a`, whence `tŠĶĘ < a` by T1 case (ii). Combining root ordering, cross-depth ordering (when two children exist), within-depth cross-sub-sub-partition ordering (PrefixOrderingExtension), and within-sub-sub-partition ordering (induction hypothesis), every pair of distinct addresses in the sub-partition is comparable under T1. Per-allocator allocation order is consistent throughout: `tŠĶĘ` was allocated before any of its descendants (T10a temporal ordering), PrefixOrderingExtension preserves each sibling stream's allocation sequence, and the induction hypothesis secures consistency within each `subtree(u‚Īľ)`.

**Total ordering.** The partition root `p` is itself an allocated address in `subtree(p)`, yet it belongs to no child's reach: every sibling base `s ‚ąą dom(c‚āĀ) ‚ą™ dom(c‚āā)` satisfies `#s > #p` (TA5(d) gives child-base lengths `#p + 1` or `#p + 2`, and T10a.1 ‚ÄĒ re-derived from TA5(c) ‚ÄĒ preserves this length under sibling increments), so every `s` has length exceeding `#p`, and no such `s` can be a prefix of `p`; consequently `p ‚ąČ subtree(s)` for any sibling `s`, hence `p ‚ąČ reach(c‚āĀ) ‚ą™ reach(c‚āā)`. We handle `p` directly: for any allocated `a ‚Č† p` with `p ‚Čľ a`, the relation is proper ‚ÄĒ `p ‚Čļ a`, hence `#p < #a` ‚ÄĒ and T1 case (ii) gives `p < a`. Thus `p` precedes every other allocated address in the partition.

For the remaining pairs, every allocated address `a ‚Č† p` belongs to exactly one of `reach(c‚āĀ)` and `reach(c‚āā)` (when both children exist; otherwise to whichever single child's reach is present), and within that, to the prefix-closure `subtree(u)` of exactly one sibling `u` in the corresponding child's stream. *Existence* follows from T10a: every such `a` was produced by an allocator descended from `p`, and the lineage from `p` to `a`'s producer first leaves `p` through one of the two child-spawning increments, landing in some sibling stream `dom(cŠĶĘ)`; within that stream, `a` either equals some sibling `u ‚ąą dom(cŠĶĘ)` (when `a` itself is a domain element of the child's allocator) or descends further from such a `u` via subsequent child spawnings (when `a` lies deeper in the lineage), so in either case `a ‚ąą subtree(u) ‚äÜ reach(cŠĶĘ)`. *Uniqueness across reaches* (when both children exist): the Cross-depth ordering above showed that `reach(c‚āĀ)` carries component `‚Č• 1` at position `#p + 1` while `reach(c‚āā)` carries component `0` there, so `reach(c‚āĀ) ‚ą© reach(c‚āā) = ‚ąÖ` and the component `a_{#p+1}` decides which reach contains `a` ‚ÄĒ `0` for `reach(c‚āā)`, `‚Č• 1` for `reach(c‚āĀ)`. *Uniqueness within a reach*: distinct siblings `u, u' ‚ąą dom(cŠĶĘ)` have `#u = #u'` (T10a.1, uniform sibling length) and `u ‚Č† u'` (TA5(a) gives the strictly increasing sibling stream `c_{i,0} < c_{i,1} < c_{i,2} < ...`); if some `s ‚ąą subtree(u) ‚ą© subtree(u')` existed, then `u ‚Čľ s` and `u' ‚Čľ s` together with `#u = #u'` would force `u = u'` (a tumbler is determined by its prefix at any given length, since two distinct tumblers of equal length cannot both equal `s` on positions `1..#u`), a contradiction. Hence `subtree(u) ‚ą© subtree(u') = ‚ąÖ` for distinct siblings of the same stream, and `a` lies in the prefix-closure of a uniquely determined sibling. Within that prefix-closure `subtree(u)`, the structural induction (applied to the sub-partition rooted at `u`) places `a` in exactly one sub-sub-partition.

For any two distinct addresses `a ‚Č† p` and `b ‚Č† p` within the partition: if they belong to different children's reaches, the Cross-depth ordering determines their comparison (the `reach(c‚āā)`-address precedes the `reach(c‚āĀ)`-address); if they belong to the same child's reach but to prefix-closures of different siblings `uŠĶĘ, u‚Īľ ‚ąą dom(c‚āĖ)` with `i < j`, then `uŠĶĘ < u‚Īľ` (TA5(a)) and `uŠĶĘ ‚č† u‚Īľ ‚ąß u‚Īľ ‚č† uŠĶĘ` (uniform length plus distinctness, as in *Sibling prefixes are non-nesting*), so PrefixOrderingExtension gives that the `subtree(uŠĶĘ)`-address precedes the `subtree(u‚Īľ)`-address; if both belong to `subtree(u)` for the same sibling `u`, the structural induction (applied to the sub-partition rooted at `u`) gives their ordering. In every case the pair is comparable under T1, and the ordering is consistent with per-allocator allocation order ‚ÄĒ `p` precedes all its descendants by T10a temporal ordering, within each allocator's sibling stream by T9, across depths by T1 case (i), across sibling prefix-closures within a single reach by PrefixOrderingExtension, and within nested sub-partitions by structural induction. ‚ąé
