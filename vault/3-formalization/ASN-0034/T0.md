**T0 (CarrierSetDefinition).** `T = {d₁.d₂. ... .dₙ : each dᵢ ∈ ℕ, n ≥ 1}`. This is an axiom: we posit the carrier set by definition, not by derivation. For each `a ∈ T`, write `#a` for the number of components in `a` — the length of the sequence — so that `#a ≥ 1`, and write `aᵢ` for the value of the `i`-th component, defined for `1 ≤ i ≤ #a` and satisfying `aᵢ ∈ ℕ`. Write `ℕ⁺ = {n ∈ ℕ : n > 0}` for the subcarrier of strictly positive naturals — downstream properties (notably T4b and T6) use this notation as the value domain of non-zero components, distinct from the full carrier ℕ which also admits zero in the role of field separator.

The carrier set and its operators are stated here. The standard properties of ℕ that downstream proofs cite — closure under successor and addition together with `0` as additive identity (the clause `0 + n = n`, bundled with closure because both statements concern how the two closure operations — successor and addition — act on ℕ, and in particular how addition acts on the distinguished element `0`), strict total order, discreteness, order compatibility of addition on either side together with the strict successor inequality `n < n + 1` (the strict-successor clause bundled with order compatibility because all three clauses concern the interaction between addition and `<` on ℕ — the two compatibility clauses cover the non-strict case where a fixed summand is added to both sides of a non-strict inequality, and the strict-successor clause covers the distinct fact that the successor operation strictly advances under `<`), well-ordering, zero as lower bound of ℕ, partial subtraction (with its conditional closure, inverse characterisations, strict positivity, right telescoping, and strict monotonicity clauses), additive cancellation (left, right, and summand absorption), and associativity of addition — are stated as separate axioms, each established as its own foundation object under both a hyphenated short label and a PascalCase canonical name: NAT-closure (NatArithmeticClosureAndIdentity), NAT-order (NatStrictTotalOrder), NAT-discrete (NatDiscreteness), NAT-addcompat (NatAdditionOrderAndSuccessor), NAT-wellorder (NatWellOrdering), NAT-zero (NatZeroMinimum), NAT-sub (NatPartialSubtraction), NAT-cancel (NatAdditionCancellation), NAT-addassoc (NatAdditionAssociative). The two name forms denote the same axiom in each case — the hyphenated form is the file-local label, the PascalCase form is the canonical identifier — so a downstream Depends entry that cites either form (or both, as in the conventional pairing `NAT-zero (NatZeroMinimum)`) resolves unambiguously to the corresponding foundation object. The canonical names for NAT-closure and NAT-addcompat are chosen to reflect the full scope each axiom carries: `NatArithmeticClosureAndIdentity` signals that NAT-closure bundles successor closure, addition closure, and the additive identity `0 + n = n`, and `NatAdditionOrderAndSuccessor` signals that NAT-addcompat bundles left and right order compatibility with the strict successor inequality `n < n + 1`. This list is exhaustive: the ASN states no NAT-* axiom outside it, and each clause listed here for each axiom is in that axiom's stated scope — downstream Depends entries that attribute the additive identity `0 + n = n` to NAT-closure (as TumblerAdd's dominance and strict-advancement proofs do at `0 + wₖ = wₖ` and `0 + 1 = 1`, as D0's reading-off derivation does in combining the identity with NAT-addcompat to yield `aₖ + wₖ ≥ aₖ`, and as TA7a's Case `k ≥ 2` closing does at `0 + o₁ = o₁`) or the strict successor inequality `n < n + 1` to NAT-addcompat (as TumblerAdd's strict-advancement chain does at `n = aₖ`, as ActionPoint's `w_{actionPoint(w)} ≥ 1` derivation does at `n = 0`, and as T1, TA5, TA5a, T0a, T10a.8, GlobalUniqueness, T10a-N, TA1, TA1-strict do at their various successor-chain sites) route through content this enumeration explicitly endorses. The `0 ∈ ℕ` *membership* of padding or separator positions — a distinct step that supplies the *set-membership* of `0`, not the algebraic identity `0 + n = n` — is routed separately through NAT-zero, whose axiom `(A n ∈ ℕ :: 0 ≤ n)` quantifies `0` as a lower bound of ℕ and thereby presupposes `0 ∈ ℕ`; the additive identity clause of NAT-closure is an equation over ℕ that presupposes, rather than supplies, `0 ∈ ℕ`, so the two clauses cover disjoint downstream uses. Each proof cites only the ℕ facts it actually uses.

*Formal Contract:*
- *Axiom:* T is the set of all finite sequences over ℕ with length ≥ 1, equipped with length `#· : T → ℕ` satisfying `#a ≥ 1` for all `a ∈ T`, and component projection `·ᵢ` yielding `aᵢ ∈ ℕ` for each `1 ≤ i ≤ #a`. The subcarrier `ℕ⁺ = {n ∈ ℕ : n > 0}` denotes the strictly positive naturals, supplying the value domain of non-zero components that downstream properties cite.
