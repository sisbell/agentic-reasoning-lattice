**S7d (HierarchicalAllocationDiscipline).** Every address at every level of the tumbler hierarchy is the product of a T10a-conforming allocation event. For every `a ∈ dom(Σ.C)`: (i) the document-level prefix `origin(a)` (DocumentLevelPrefix) was produced by a distinct allocation event within a system conforming to T10a (ASN-0034); and (ii) the element-level address `a` itself — with `zeros(a) = 3` (entailed by S7b) — was produced by a distinct allocation event within a system conforming to T10a.

This is a design requirement paralleling S7a. Where S7a constrains element-level allocation to be document-scoped, S7d constrains allocation at both the document level and the element level to fall under T10a's discipline. Nelson's baptism principle operates at every level of the hierarchy: node operators allocate user prefixes, users allocate document prefixes, documents allocate element addresses. Each level is governed by T10a's allocator discipline — the mechanism by which a parent spawns a child allocator via `inc(·, k')` with `k' ∈ {1, 2}` applies uniformly across levels. Without clause (i), GlobalUniqueness — whose precondition demands that the addresses in question be "produced by distinct allocation events within a system conforming to T10a" — cannot be discharged for document-level tumblers, and S7's uniqueness-across-documents argument has an unjustified step. Without clause (ii), S4's application to element-level addresses in S7's permanence argument is equally unjustified: S4 requires its arguments to be products of T10a allocation events, and no other axiom establishes this for `a` as opposed to `origin(a)`.

*Formal Contract:*
- *Axiom:* For every `a ∈ dom(Σ.C)`: (i) the document-level prefix `origin(a)` (DocumentLevelPrefix) was produced by a T10a-conforming allocation event; (ii) the element-level address `a` was itself produced by a T10a-conforming allocation event.
- *Preconditions:* `a ∈ dom(Σ.C)` with `zeros(a) = 3` (entailed by S7b). `origin(a)` is well-defined with `zeros(origin(a)) = 2` (origin(a), DocumentLevelPrefix).
