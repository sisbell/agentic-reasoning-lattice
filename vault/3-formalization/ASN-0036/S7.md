**S7 (Structural attribution).** For every `a ∈ dom(Σ.C)`, the *origin* `origin(a)` (DocumentLevelPrefix) is the document-level prefix — the tumbler `N.0.U.0.D` obtained by truncating the element field from `a`'s T4 decomposition.

This is the full document tumbler `N.0.U.0.D` — uniquely identifying the allocating document across the system. A document, formally, is an abstract entity created by a single document-level allocation event (D-DOC); two documents are distinct precisely when their creating events are distinct. Since every such event is T10a-conforming (S7d), GlobalUniqueness (ASN-0034) guarantees that distinct creating events produce distinct tumblers, and therefore distinct document-level prefixes. It is not metadata that can be stripped or forged — it IS the address. To retrieve the content, the system must know its I-address; to know its I-address is to know its origin.

S7 follows from origin(a) (DocumentLevelPrefix, definition of the origin function), D-DOC (formal definition of document and document distinctness), S7a (document-scoped allocation ensures the document-level prefix identifies the allocating document), S7b (element-level restriction ensures all three identifying fields are present), S7d (hierarchical allocation discipline establishes allocation at every level as T10a-conforming), T4 (HierarchicalParsing, ASN-0034), and GlobalUniqueness (ASN-0034) (distinct allocation events produce distinct addresses). Since I-addresses are permanent (S0) and unique (S4, whose T10a precondition is discharged by S7d), this attribution is permanent and unseverable.

We note a subtlety. S7 identifies the document that ALLOCATED the I-address — the document where the content was first created. This is distinct from the document where the content currently appears. When content is transcluded from document B into document A, the reader viewing A sees the content, but S7 traces it to B. The distinction between "where I am reading" (Vstream context, document A) and "where this came from" (Istream structure, document B) is precisely the two-stream separation made visible.

Gregory's implementation reveals two mechanisms for origin lookup. The I-address prefix itself encodes the originating document (used during address allocation to scope the search range). Separately, each arrangement entry carries an explicit `homedoc` field recording the allocating document (used during retrieval). At the abstract level, S7 says only that the information is present in the address — it does not prescribe how an implementation extracts it.

*Proof.* We wish to show that for every `a ∈ dom(Σ.C)`, the function `origin(a)` (DocumentLevelPrefix) uniquely identifies the document that allocated `a`, and that this identification is permanent and unseverable.

**Well-definedness.** By origin(a) (DocumentLevelPrefix), for every `a ∈ dom(Σ.C)` with `zeros(a) = 3` (S7b), `origin(a)` is a well-defined tumbler satisfying `zeros(origin(a)) = 2`, placing it at the document level in T4's hierarchy.

**Identification.** By S7a (document-scoped allocation), every I-address is allocated under the tumbler prefix of the document that created it. The document-level prefix of `a` — precisely `origin(a)`, the tumbler `N.0.U.0.D` obtained by truncating the element field — identifies the document whose owner performed the allocation that placed `a` into `dom(C)`. This is not a lookup or annotation: the address structurally encodes its provenance. S7a ensures that `origin(a)` IS the allocating document's tumbler.

**Uniqueness across documents.** Let `d₁` and `d₂` be distinct documents; by D-DOC, this means they were created by distinct allocation events. By S7d (hierarchical allocation discipline), each of these creating events is T10a-conforming. GlobalUniqueness (ASN-0034) — whose precondition requires precisely that the addresses be produced by distinct allocation events within a T10a-conforming system — therefore applies: the document-level tumblers of `d₁` and `d₂` are distinct. Therefore, for any `a₁, a₂ ∈ dom(Σ.C)` allocated under distinct documents (D-DOC): `origin(a₁) ≠ origin(a₂)`. The origin function discriminates allocating documents without ambiguity.

**Permanence.** By S0 (content immutability), once `a ∈ dom(Σ.C)`, then `a ∈ dom(Σ'.C)` for all successor states `Σ'` — the address persists. Since `a` is a tumbler — a fixed sequence of components, not a mutable reference — and `origin(a)` is computed from the components of `a` alone via T4's deterministic field decomposition, `origin(a)` yields the same result in every state in which `a` exists. By S7d (hierarchical allocation discipline), `a` was itself produced by a T10a-conforming allocation event; S4 (origin-based identity) therefore applies, and distinct allocation events produce distinct addresses, so `a` is never reassigned or reused. The attribution cannot be severed because it is not a separate datum attached to the content — it is a structural property of the address itself. To retrieve content at `a`, a system must know `a`; to know `a` is to know `origin(a)`. ∎

*Remark.* The distinctness of document-level tumblers established in the uniqueness section is moreover decidable by component-wise comparison (T3, CanonicalRepresentation, ASN-0034). This is a property of the computational character of tumbler equality, not a precondition of the proof.

*Formal Contract:*
- *Preconditions:* `a ∈ dom(Σ.C)` in a system conforming to S0 (content immutability), S4 (origin-based identity), D-DOC (document definition), S7a (document-scoped allocation), S7b (element-level I-addresses), S7d (hierarchical allocation discipline), origin(a) (DocumentLevelPrefix), T4 (HierarchicalParsing, ASN-0034), and GlobalUniqueness (ASN-0034).
- *Postconditions:* (a) `origin(a)` is well-defined and is a document-level tumbler with `zeros(origin(a)) = 2`. (b) `origin(a)` is the identifier (D-DOC) of the document that allocated `a`. (c) For `a₁, a₂` allocated under distinct documents (D-DOC), `origin(a₁) ≠ origin(a₂)`.
- *Invariant:* `origin(a)` is invariant across all state transitions: for every `Σ → Σ'` with `a ∈ dom(Σ.C)`, the address `a` persists in `dom(Σ'.C)` (by S0) and `origin(a)` — computed from the components of `a` alone — yields the same document tumbler in every state.
- *Frame:* The content values `Σ.C(a)` and arrangement functions `Σ.M(d)` play no role — attribution is a property of the addressing scheme alone.
