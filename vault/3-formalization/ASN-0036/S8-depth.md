**S8-depth (Fixed-depth V-positions).** Within a given subspace `s` of document `d`, all V-positions share the same tumbler depth:

`(A d, v‚āĀ, v‚āā : v‚āĀ ‚ąą dom(ő£.M(d)) ‚ąß v‚āā ‚ąą dom(ő£.M(d)) ‚ąß (v‚āĀ)‚āĀ = (v‚āā)‚āĀ : #v‚āĀ = #v‚āā)`

This is a design requirement, not a convention ‚ÄĒ parallel to S7a. Gregory's evidence supports it: V-addresses in the text subspace consistently use the form `s.x` ‚ÄĒ two tumbler digits, where `s` is the subspace identifier and `x` is the ordinal. The two-blade knife computation (which sets the second blade at `(N+1).1` for any insertion at `N.x`) works only if all positions within a subspace share the same depth. Any correct implementation must satisfy this constraint.

S8-depth allows us to define "consecutive V-positions" precisely. Within a subspace, consecutive positions differ only at the ordinal (last) component: position `s.x` is followed by `s.(x+1)`. A parallel uniformity holds for I-addresses within a correspondence run: all I-addresses in a run share the same tumbler depth and prefix, differing only at the element ordinal. This follows directly from TumblerAdd (PositionAdvance, ASN-0034): ordinal displacement `őī‚āĖ = [0, ..., 0, k]` has its action point at the last component, so TumblerAdd copies all earlier components from the start address unchanged (preserving the prefix) and produces a result of length `#őī‚āĖ` (preserving depth). The uniformity is definitional ‚ÄĒ a correspondence run `(v, a, n)` specifies `M(d)(v + k) = a + k`, and both `v + k` and `a + k` are ordinal displacements whose depth and prefix preservation follow from TumblerAdd's component-wise definition. Subspace preservation follows separately: for V-positions with `#v ‚Č• 2`, the subspace identifier `v‚āĀ` precedes the action point `#v` and is copied unchanged by TumblerAdd's prefix rule (at `#v = 1` the action point coincides with position 1, so `shift([s], k)‚āĀ = s + k ‚Č† s` for `k ‚Č• 1` ‚ÄĒ subspace preservation does not hold); for I-addresses, S7c guarantees element-field depth `őī ‚Č• 2`, so the subspace identifier `E‚āĀ` is structural context outside the ordinal, and the shift acts on `[E‚āā, ..., E_őī]` without altering `E‚āĀ`. We write `v + k` for ordinal displacement applied to V-positions, and `a + k` for the same applied to the element ordinal of I-addresses.

(Why non-trivial runs arise in practice is a separate question. Allocator discipline ‚ÄĒ T10a, ASN-0034 ‚ÄĒ establishes that each allocator produces sibling outputs exclusively by `inc(¬∑, 0)`, and TA5(c) guarantees the successor has the same depth as the predecessor. Consecutive allocations therefore produce consecutive I-addresses, which is why sequential content creation naturally yields correspondence runs of length greater than one. But this operational fact is motivation for the definition of correspondence runs, not a dependency of the decomposition proof.)

We extend the ordinal displacement notation to `k = 0`: define `v + 0 = v` (identity) and `v + k = shift(v, k)` for `k ‚Č• 1`. OrdinalShift (ASN-0034) has precondition `n ‚Č• 1`; the extension to `k = 0` is purely notational ‚ÄĒ no arithmetic is performed. For I-addresses, define `a + 0 = a` and `a + k = shift(a, k) = a ‚äē őī(k, #a)` for `k ‚Č• 1`. This is well-defined: the action point of `őī(k, #a)` is `#a`, which falls at the element field's last component ‚ÄĒ S7c guarantees element-field depth `őī ‚Č• 2`, so the last component of the full address *is* the element ordinal's deepest position ‚ÄĒ and TumblerAdd's prefix rule copies all earlier components (node, user, document fields, their separators, and the subspace identifier) unchanged, producing a result of length `#a`.

A *correspondence run* in document `d` is a triple `(v, a, n)` ‚ÄĒ a V-position, an I-address, and a natural number `n ‚Č• 1` ‚ÄĒ such that the arrangement preserves ordinal displacement within the run:

`(A k : 0 ‚Č§ k < n : ő£.M(d)(v + k) = a + k)`

At `k = 0` this is the base case `M(d)(v) = a`. Each subsequent `k` increments both the V-ordinal and the I-ordinal by the same amount. Within a correspondence run, each step forward in Vstream corresponds to the same step forward in Istream.

*Formal Contract:*
- *Axiom:* `(A d, v‚āĀ, v‚āā : v‚āĀ ‚ąą dom(ő£.M(d)) ‚ąß v‚āā ‚ąą dom(ő£.M(d)) ‚ąß (v‚āĀ)‚āĀ = (v‚āā)‚āĀ : #v‚āĀ = #v‚āā)`
- *Definition:* A correspondence run in document `d` is a triple `(v, a, n)` with `n ‚Č• 1` such that `(A k : 0 ‚Č§ k < n : ő£.M(d)(v + k) = a + k)`, where `v + 0 = v`, `a + 0 = a`, and for `k ‚Č• 1`, `v + k = shift(v, k)`, `a + k = shift(a, k)`.
- *Postconditions:* For any correspondence run `(v, a, n)` in document `d`:
  1. V-position subspace preservation (requires `#v ‚Č• 2`): `(A k : 0 ‚Č§ k < n : (v + k)‚āĀ = v‚āĀ)` ‚ÄĒ position 1 precedes the action point `#v` iff `1 < #v`, so OrdinalShift's prefix rule copies the subspace identifier unchanged. (At `#v = 1`, the action point coincides with position 1 and subspace preservation does not hold.)
  2. V-position depth preservation: `(A k : 0 ‚Č§ k < n : #(v + k) = #v)` ‚ÄĒ TumblerAdd produces a result of length `#v`.
  3. I-address subspace preservation: `(A k : 0 ‚Č§ k < n : E‚āĀ(a + k) = E‚āĀ(a))` ‚ÄĒ S7c guarantees element-field depth `őī ‚Č• 2`, placing `E‚āĀ` outside the action point; TumblerAdd's prefix rule copies it unchanged.
  4. I-address depth preservation: `(A k : 0 ‚Č§ k < n : #(a + k) = #a)` ‚ÄĒ TumblerAdd produces a result of length `#a`.
