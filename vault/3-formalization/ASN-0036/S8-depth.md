**S8-depth (Fixed-depth V-positions).** Within a given subspace `s` of document `d`, all V-positions share the same tumbler depth:

`(A d, v‚āĀ, v‚āā : v‚āĀ ‚ąą dom(ő£.M(d)) ‚ąß v‚āā ‚ąą dom(ő£.M(d)) ‚ąß (v‚āĀ)‚āĀ = (v‚āā)‚āĀ : #v‚āĀ = #v‚āā)`

This is a design requirement, not a convention ‚ÄĒ parallel to S7a. Gregory's evidence supports it: V-addresses in the text subspace consistently use the form `s.x` ‚ÄĒ two tumbler digits, where `s` is the subspace identifier and `x` is the ordinal. The two-blade knife computation (which sets the second blade at `(N+1).1` for any insertion at `N.x`) works only if all positions within a subspace share the same depth. Any correct implementation must satisfy this constraint.

S8-depth allows us to define "consecutive V-positions" precisely. Within a subspace, consecutive positions differ only at the ordinal (last) component: position `s.x` is followed by `s.(x+1)`. A parallel uniformity holds for I-addresses within a correspondence run: all I-addresses in a run share the same tumbler depth and prefix, differing only at the element ordinal. This follows directly from OrdinalShift (ShiftDefinition, ASN-0034): OrdinalShift's prefix rule (`shift(v, k)ŠĶĘ = vŠĶĘ` for `i < #v`) copies all earlier components from the start address unchanged (preserving the prefix), and its last-component increment modifies only position `#v`, producing a result of length `#v` (preserving depth). The uniformity is definitional ‚ÄĒ a correspondence run `(v, a, n)` specifies `M(d)(v + k) = a + k`, and both `v + k` and `a + k` are ordinal displacements whose depth and prefix preservation follow from OrdinalShift's postconditions. Subspace preservation follows separately: for V-positions, the subspace identifier `v‚āĀ` is at position 1; since `#v ‚Č• 2` (S8-vdepth), `1 < #v`, so OrdinalShift's prefix rule copies it unchanged; for I-addresses, S7c guarantees element-field depth `őī ‚Č• 2`, so the subspace identifier `E‚āĀ` is structural context outside the ordinal, and the shift acts on `[E‚āā, ..., E_őī]` without altering `E‚āĀ`. We write `v + k` for ordinal displacement applied to V-positions, and `a + k` for the same applied to the element ordinal of I-addresses.

(Why non-trivial runs arise in practice is a separate question. Allocator discipline ‚ÄĒ T10a, ASN-0034 ‚ÄĒ establishes that each allocator produces sibling outputs exclusively by `inc(¬∑, 0)`, and TA5(c) guarantees the successor has the same depth as the predecessor. Consecutive allocations therefore produce consecutive I-addresses, which is why sequential content creation naturally yields correspondence runs of length greater than one. But this operational fact is motivation for the definition of correspondence runs, not a dependency of the decomposition proof.)

We extend the ordinal displacement notation to `k = 0`: define `v + 0 = v` (identity) and `v + k = shift(v, k)` for `k ‚Č• 1`. OrdinalShift (ASN-0034) has precondition `n ‚Č• 1`; the extension to `k = 0` is purely notational ‚ÄĒ no arithmetic is performed. For I-addresses, define `a + 0 = a` and `a + k = shift(a, k)` for `k ‚Č• 1`. This is well-defined: S7c guarantees element-field depth `őī ‚Č• 2`, so the last component of the full address *is* the element ordinal's deepest position. OrdinalShift's prefix rule (`shift(a, k)ŠĶĘ = aŠĶĘ` for `i < #a`) copies all earlier components (node, user, document fields, their separators, and the subspace identifier) unchanged, and its last-component increment (`shift(a, k)_{#a} = a_{#a} + k`) shifts only the element ordinal, producing a result of length `#a`.

A *correspondence run* in document `d` is a triple `(v, a, n)` ‚ÄĒ a V-position, an I-address, and a natural number `n ‚Č• 1` ‚ÄĒ such that the arrangement preserves ordinal displacement within the run:

`(A k : 0 ‚Č§ k < n : ő£.M(d)(v + k) = a + k)`

At `k = 0` this is the base case `M(d)(v) = a`. Each subsequent `k` increments both the V-ordinal and the I-ordinal by the same amount. Within a correspondence run, each step forward in Vstream corresponds to the same step forward in Istream.

*Formal Contract:*
- *Axiom:* `(A d, v‚āĀ, v‚āā : v‚āĀ ‚ąą dom(ő£.M(d)) ‚ąß v‚āā ‚ąą dom(ő£.M(d)) ‚ąß (v‚āĀ)‚āĀ = (v‚āā)‚āĀ : #v‚āĀ = #v‚āā)`
- *Definition:* A correspondence run in document `d` is a triple `(v, a, n)` with `n ‚Č• 1` such that `(A k : 0 ‚Č§ k < n : ő£.M(d)(v + k) = a + k)`, where `v + 0 = v`, `a + 0 = a`, and for `k ‚Č• 1`, `v + k = shift(v, k)`, `a + k = shift(a, k)`.
- *Preconditions:* Postcondition 3 requires that each `a + k` in the run is an element-level address. For every 0 ‚Č§ k < n, the correspondence run definition gives `M(d)(v + k) = a + k`; S3 (ReferentialIntegrity) gives `a + k ‚ąą dom(ő£.C)`; S7b (Element-level I-addresses) gives `zeros(a + k) = 3`, confirming four-field structure and making `E‚āĀ(a + k)` well-defined. At `k = 0`, S7c then gives `#fields(a).element ‚Č• 2` (`őī ‚Č• 2`). At element-field depth `őī = 1`, position 1 would be the last component (`#a`), and OrdinalShift's last-component increment would alter `E‚āĀ` ‚ÄĒ subspace preservation would be false.
- *Postconditions:* For any correspondence run `(v, a, n)` in document `d`:
  1. V-position subspace preservation: `(A k : 0 ‚Č§ k < n : (v + k)‚āĀ = v‚āĀ)` ‚ÄĒ S8-vdepth guarantees `#v ‚Č• 2`, so `1 < #v` and OrdinalShift's prefix rule (`shift(v, k)ŠĶĘ = vŠĶĘ` for `i < #v`) copies the subspace identifier at position 1 unchanged.
  2. V-position depth preservation: `(A k : 0 ‚Č§ k < n : #(v + k) = #v)` ‚ÄĒ OrdinalShift produces a result of length `#v`.
  3. I-address subspace preservation (requires S3, S7b, S7c): `(A k : 0 ‚Č§ k < n : E‚āĀ(a + k) = E‚āĀ(a))` ‚ÄĒ We first establish that `E‚āĀ(a + k)` is well-defined. By the correspondence run definition, `M(d)(v + k) = a + k` for each 0 ‚Č§ k < n; S3 gives `a + k ‚ąą dom(ő£.C)`; S7b gives `zeros(a + k) = 3`, so `a + k` is a four-field element-level address and `E‚āĀ(a + k)` is well-defined. For the equality: at `k = 0`, S7c gives element-field depth `őī ‚Č• 2`, so `E‚āĀ(a)` occupies a position `p < #a`. For `k ‚Č• 1`, OrdinalShift's prefix rule preserves all positions before `#a` ‚ÄĒ including the three zero-valued field separators ‚ÄĒ and the last position satisfies `a_{#a} + k > 0`, so no zero is introduced; the separators of `a + k` therefore occupy the same positions as those of `a`, and `E‚āĀ(a + k)` refers to the same position `p`. The prefix rule gives `(a + k)_p = a_p`, hence `E‚āĀ(a + k) = E‚āĀ(a)`.
  4. I-address depth preservation: `(A k : 0 ‚Č§ k < n : #(a + k) = #a)` ‚ÄĒ OrdinalShift produces a result of length `#a`.
