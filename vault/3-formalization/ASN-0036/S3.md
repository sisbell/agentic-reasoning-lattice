**S3 (Referential integrity).** `(A d, v : v вҲҲ dom(ОЈ.M(d)) : ОЈ.M(d)(v) вҲҲ dom(ОЈ.C))`

Every V-reference resolves. If a document's arrangement says "at position `v`, display the content at I-address `a`," then `a` must be in `dom(C)`. There are no dangling references.

The maintenance of S3 across state transitions reveals a temporal ordering constraint. The weakest precondition for S3 under an operation that adds a V-mapping `M(d)(v) = a` is:

`wp(add-mapping, S3) вҹ№ a вҲҲ dom(ОЈ.C)`

For an operation that only adds a V-mapping without creating content, the target I-address must already be in `dom(C)`. An operation that atomically creates content at `a` and adds the mapping `M(d)(v) = a` satisfies S3 in the post-state without sequential precedence вҖ” `a вҲҲ dom(ОЈ'.C)` and `ОЈ'.M(d)(v) = a` are established simultaneously. The dependency is logical, not temporal: a reference presupposes the existence of its target, but existence need not precede reference in a prior transition. What matters for persistence is that S1 guarantees once `a` enters `dom(C)`, it remains вҖ” so a valid reference cannot become dangling through any subsequent state transition.

We observe a deliberate asymmetry. S3 says arrangement implies existence: `ran(M(d)) вҠҶ dom(C)`. It does NOT say existence implies arrangement. Content can exist in Istream without being arranged in any current document. Nelson calls such content "deleted bytes вҖ” not currently addressable, awaiting historical backtrack functions, may remain included in other versions." The asymmetry is the space in which persistence independence lives.

*Proof.* We establish the invariant by induction over the operation history.

*Base case.* By AX-1 (initial empty state), `dom(ОЈвӮҖ.M(d)) = вҲ…` for every document `d` in the initial state `ОЈвӮҖ`. The universal quantification `(A d, v : v вҲҲ dom(ОЈвӮҖ.M(d)) : ОЈвӮҖ.M(d)(v) вҲҲ dom(ОЈвӮҖ.C))` holds vacuously вҖ” there are no V-positions to quantify over.

*Inductive step.* Suppose S3 holds in state `ОЈ`. Let `ОЈ вҶ’ ОЈ'` be any state transition. We must show `(A d, v : v вҲҲ dom(ОЈ'.M(d)) : ОЈ'.M(d)(v) вҲҲ dom(ОЈ'.C))`. We partition the V-positions in `dom(ОЈ'.M(d))` into those inherited from `ОЈ` and those introduced or modified by the transition.

*Case 1: Inherited mappings.* Let `v вҲҲ dom(ОЈ'.M(d))` with `ОЈ'.M(d)(v) = ОЈ.M(d)(v)` вҖ” a V-mapping unchanged by the transition. By the inductive hypothesis, `ОЈ.M(d)(v) вҲҲ dom(ОЈ.C)`. By S1 (store monotonicity), `dom(ОЈ.C) вҠҶ dom(ОЈ'.C)`, so `ОЈ'.M(d)(v) = ОЈ.M(d)(v) вҲҲ dom(ОЈ'.C)`.

*Case 2: Added mappings.* Let `v вҲҲ dom(ОЈ'.M(d))` with `v вҲү dom(ОЈ.M(d))` вҖ” a new V-mapping introduced by the operation. The operation sets `ОЈ'.M(d)(v) = a` for some I-address `a`. For S3 to hold in `ОЈ'`, we require `a вҲҲ dom(ОЈ'.C)`. This is the per-operation post-state obligation stated in the formal contract: every operation that adds a V-mapping `M(d)(v) = a` must ensure `a вҲҲ dom(ОЈ'.C)`. Note that this obligation is strictly weaker than the narrative's weakest precondition `a вҲҲ dom(ОЈ.C)`, which is a pre-state predicate covering the mapping-only case. For operations that atomically create content at `a`, the pre-state condition `a вҲҲ dom(ОЈ.C)` need not hold вҖ” yet `a вҲҲ dom(ОЈ'.C)` is established by the operation itself. Whether `a` was already in `dom(ОЈ.C)` before the transition or is created atomically in the same transition, the post-state obligation `a вҲҲ dom(ОЈ'.C)` suffices.

*Case 3: Modified mappings.* Let `v вҲҲ dom(ОЈ'.M(d)) вҲ© dom(ОЈ.M(d))` with `ОЈ'.M(d)(v) вү  ОЈ.M(d)(v)` вҖ” an existing V-mapping repointed to a different I-address `a' = ОЈ'.M(d)(v)`. The same obligation applies: the operation must ensure `a' вҲҲ dom(ОЈ'.C)`. The prior target `ОЈ.M(d)(v)` is no longer relevant; the new target must resolve.

*Case 4: Removed mappings.* If `v вҲҲ dom(ОЈ.M(d)) \ dom(ОЈ'.M(d))` вҖ” a V-mapping removed by the operation вҖ” the position `v` leaves `dom(ОЈ'.M(d))` and imposes no obligation on S3 in the post-state. Removal of mappings preserves S3 trivially.

The content store plays a supporting role through S1: no operation removes content from `dom(C)`, so valid references cannot become dangling through content disappearance. The only threat to S3 is the introduction of a V-mapping whose target is not in `dom(ОЈ'.C)`, and this is excluded by the per-operation obligation identified in Cases 2 and 3.

This is a design requirement: every operation specification that adds or modifies V-mappings must individually discharge the obligation that target I-addresses lie in `dom(ОЈ'.C)`. The base case and the closure argument above reduce the global invariant to a per-operation verification condition. вҲҺ

*Formal Contract:*
- *Preconditions:* AX-1 (initial empty state); S1 (store monotonicity); every operation adding or modifying a V-mapping `M(d)(v) = a` ensures `a вҲҲ dom(ОЈ'.C)` in the post-state.
- *Invariant:* `(A d, v : v вҲҲ dom(ОЈ.M(d)) : ОЈ.M(d)(v) вҲҲ dom(ОЈ.C))` for every reachable state ОЈ.
