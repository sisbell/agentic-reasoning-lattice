**S5 (Unrestricted sharing).** The same I-address may appear in the ranges of multiple arrangements, and at multiple V-positions within a single arrangement. S0–S4 and S6 are consistent with any finite sharing multiplicity — they place no constraint on `|{(d, v) : v ∈ dom(Σ.M(d)) ∧ Σ.M(d)(v) = a}|`. Because S0 and S1 quantify over state transitions (S6 is a named restatement of S1 — see S6), we state this for states reachable through conforming execution traces — finite sequences `σ = ⟨Σ₀, Σ₁, …, Σₖ⟩` with `k ≥ 1`, beginning at the initial state (AX-1), in which every transition `Σᵢ → Σᵢ₊₁` satisfies S0 and S1, every state `Σᵢ` satisfies S2 and S3, S4 holds for all allocation events across the trace, and AX-2 (allocation-mediated entry) holds:

`(A N ∈ ℕ :: (E σ = ⟨Σ₀, …, Σₖ⟩ :: k ≥ 1 ∧ σ conforming ∧ (E a ∈ dom(Σₖ.C) :: |{(d, v) : v ∈ dom(Σₖ.M(d)) ∧ Σₖ.M(d)(v) = a}| > N)))`

To see this, fix any `N`. We construct an execution trace of `N + 2` transitions beginning at `Σ₀`: first a content-creation transition that introduces an I-address `a`, then `N + 1` arrangement-extension transitions that each place `a` in a new document. The resulting state has sharing multiplicity `N + 1 > N`. At each of the `N + 1` arrangement transitions, `a ∈ dom(Σᵢ.C)` and the content store is unchanged, so S0 and S1 hold non-vacuously — content immutability and store monotonicity are substantively exercised, not merely vacuously true for want of transitions.

In any particular state, the sharing multiplicity of each address is a definite finite number — possibly zero for orphaned content (S6). The property is an architectural anti-constraint: the invariants place no finite cap on how many references may accumulate.

Nelson: "The virtual byte stream of a document may include bytes from any other document." And: "A document may have a window to another document, and that one to yet another, indefinitely. Thus A contains part of B, and so on. One document can be built upon another, and yet another document can be built upon that one, indefinitely." Transclusion is recursive and unlimited — a claim not merely about the existence of high-sharing states but about their reachability through the system's operations.

Gregory confirms the unbounded nature at the implementation level. The global index that records which documents reference which I-addresses accumulates entries without cap — "no counter, cap, MAX_TRANSCLUSIONS constant, or any other limiting mechanism anywhere in the code path." Each referential inclusion adds one entry. The only constraints are physical resources (memory and disk), not architectural limits.

The combination of S4 and S5 gives the system its distinctive character. S4 says identity is structural — determined by I-address, not by value. S5 says sharing is unlimited — any number of documents can reference the same content. Together they establish a regime in which quotation is a first-class structural relationship: any number of documents can quote the same passage, and the system knows they are all quoting — not independently writing — because they share I-addresses.

We observe that the state `Σ = (C, M)` makes the sharing relation computable: given any `a ∈ dom(C)`, the set `{d : (E v :: M(d)(v) = a)}` is determined by the state. Nelson requires this to be queryable: "It must also be possible for the reader to ask to see whatever documents window to the current document. Both are available at any time." The state model supports this — the information is present; only the efficiency of its extraction is an implementation concern.

*Proof.* We wish to show that for every `N ∈ ℕ`, there exists a conforming execution trace whose final state contains an I-address with sharing multiplicity exceeding `N`. We give two constructions — one for cross-document sharing, one for within-document sharing — each succeeding for arbitrary `N`.

**Cross-document construction.** Fix `N ∈ ℕ`. We construct an execution trace `σ_N = ⟨Σ₀, Σ₁, Σ₂, …, Σ_{N+2}⟩` comprising `N + 2` transitions:

- `Σ₀`: The initial state. By AX-1, `dom(Σ₀.M(d)) = ∅` for all documents `d`, and `dom(Σ₀.C) = ∅`.
- `Σ₀ → Σ₁` (content creation): Perform a T10a-conforming allocation event producing a fresh I-address `a` — by AX-2 (allocation-mediated entry), this is the exclusive mechanism by which addresses enter `dom(C)` — and store `Σ₁.C = {a ↦ w}` for some value `w ∈ Val`. Arrangements are unchanged: `Σ₁.M(d) = Σ₀.M(d)` for all `d`.
- `Σᵢ → Σᵢ₊₁` for `i = 1, …, N + 1` (arrangement extension): Introduce a V-mapping `Σᵢ₊₁.M(dᵢ)(vᵢ) = a` for a document `dᵢ` at a fresh V-position `vᵢ`, with the `dᵢ` pairwise distinct — such elements exist by AX-3, which guarantees `|Doc| = ℵ₀`. Content is unchanged: `Σᵢ₊₁.C = Σᵢ.C`. All other arrangements are unchanged.

The final state `Σ_{N+2}` has `C = {a ↦ w}` and `N + 1` documents `d₁, …, d_{N+1}`, each with `M(dᵢ) = {vᵢ ↦ a}`.

We verify each property across the entire trace.

*S0 (content immutability).* At `Σ₀ → Σ₁`: `dom(Σ₀.C) = ∅`, so the universal quantification `(A a ∈ dom(Σ₀.C) : …)` holds vacuously — no content yet exists to protect, and this is necessarily so for the trace's initial content-creation event. At each subsequent transition `Σᵢ → Σᵢ₊₁` for `i = 1, …, N + 1`: the content store is unchanged, `Σᵢ₊₁.C = Σᵢ.C = {a ↦ w}`, so `a ∈ dom(Σᵢ.C)` and `Σᵢ₊₁.C(a) = w = Σᵢ.C(a)`. S0 holds non-vacuously at these `N + 1` transitions: the existing address `a` is verified to persist with its value intact.

*S1 (store monotonicity).* At `Σ₀ → Σ₁`: `dom(Σ₀.C) = ∅ ⊆ {a} = dom(Σ₁.C)`. At each `Σᵢ → Σᵢ₊₁` for `i = 1, …, N + 1`: `dom(Σᵢ.C) = {a} = dom(Σᵢ₊₁.C)`, so `{a} ⊆ {a}`. In particular, `a ∈ dom(Σᵢ.C)` witnesses the non-vacuity: the domain has an element whose membership persists.

*S2 (arrangement functionality).* In `Σ₀` and `Σ₁`, all arrangements are empty — trivially functions. In each `Σᵢ` for `i ≥ 2`, each newly introduced `M(dⱼ) = {vⱼ ↦ a}` contains a single entry — a function with domain `{vⱼ}`. All other arrangements remain empty.

*S3 (referential integrity).* In `Σ₀` and `Σ₁`, no V-mappings exist — holds vacuously. In each `Σᵢ` for `i ≥ 2`, every V-mapping targets `a`, and `a ∈ dom(Σᵢ.C)` because `a` was created at `Σ₁` and S0 guarantees its persistence in every subsequent state.

*S4 (origin-based identity).* The trace contains a single allocation event (at `Σ₀ → Σ₁`), producing `a`. No pair of distinct allocation events exists, so `(A a₁, a₂ from distinct events :: a₁ ≠ a₂)` holds vacuously.

*S6 (persistence independence).* Formally identical to S1; holds by S1's verification above.

The sharing multiplicity of `a` in `Σ_{N+2}` is `|{(d, v) : v ∈ dom(Σ_{N+2}.M(d)) ∧ Σ_{N+2}.M(d)(v) = a}| = N + 1`, since each of the `N + 1` documents contributes exactly one pair `(dᵢ, vᵢ)`. Thus the multiplicity exceeds `N`.

**Within-document construction.** Fix `N ∈ ℕ`. We construct an execution trace `σ'_N = ⟨Σ₀, Σ₁, Σ₂, …, Σ_{N+2}⟩`:

- `Σ₀`: The initial state (AX-1). `dom(Σ₀.M(d)) = ∅` for all `d`, and `dom(Σ₀.C) = ∅`.
- `Σ₀ → Σ₁` (content creation): Perform a T10a-conforming allocation event producing a fresh I-address `a` (AX-2) and store `Σ₁.C = {a ↦ w}` for some `w ∈ Val`. Arrangements unchanged.
- `Σᵢ → Σᵢ₊₁` for `i = 1, …, N + 1` (arrangement extension): Extend the arrangement of a single document `d` by adding `vᵢ ↦ a` for a fresh V-position `vᵢ`, so that `Σᵢ₊₁.M(d) = Σᵢ.M(d) ∪ {vᵢ ↦ a}` with the `vᵢ` pairwise distinct — such elements exist by AX-3, which guarantees `|VPos(S)| = ℵ₀` for the subspace `S` of `d`. Content is unchanged: `Σᵢ₊₁.C = Σᵢ.C`.

The final state `Σ_{N+2}` has one document `d` with `M(d) = {v₁ ↦ a, v₂ ↦ a, …, v_{N+1} ↦ a}`.

We verify each property. *S0 (content immutability):* at `Σ₀ → Σ₁`, `dom(Σ₀.C) = ∅` — vacuous; at each `Σᵢ → Σᵢ₊₁` for `i = 1, …, N + 1`, `Σᵢ₊₁.C = Σᵢ.C = {a ↦ w}`, so `a ∈ dom(Σᵢ.C)` and `Σᵢ₊₁.C(a) = w = Σᵢ.C(a)` — non-vacuous at `N + 1` transitions. *S1 (store monotonicity):* at `Σ₀ → Σ₁`, `∅ ⊆ {a}`; at each `Σᵢ → Σᵢ₊₁` for `i ≥ 1`, `{a} ⊆ {a}` with `a` witnessing the non-vacuity. *S2 (arrangement functionality):* in each state after `Σ₁`, the V-positions `v₁, …, vⱼ` accumulated in `M(d)` are pairwise distinct by hypothesis, so each maps to exactly one I-address; `M(d)` is a well-defined function at every stage. *S3 (referential integrity):* in `Σ₀` and `Σ₁`, no V-mappings exist — vacuous; in each `Σᵢ` for `i ≥ 2`, every V-mapping targets `a`, and `a ∈ dom(Σᵢ.C)` because `a` was created at `Σ₁` and preserved by S0. *S4 (origin-based identity):* a single allocation event produces `a`; no pair of distinct events exists — vacuous. *S6 (persistence independence):* formally identical to S1; holds by S1's verification above.

The within-document sharing multiplicity is `|{(d, v) : v ∈ dom(Σ_{N+2}.M(d)) ∧ Σ_{N+2}.M(d)(v) = a}| = N + 1`, since the single document `d` contributes `N + 1` pairs `(d, v₁), …, (d, v_{N+1})`. Thus the multiplicity exceeds `N`.

**Conclusion.** Since both constructions produce conforming execution traces for arbitrary `N ∈ ℕ`, the conjunction S0 ∧ S1 ∧ S2 ∧ S3 ∧ S4 ∧ S6 ∧ AX-2 is consistent with sharing multiplicity exceeding any given finite bound — not merely in isolated states but in states reachable through sequences of transitions that substantively enforce content immutability and store monotonicity. No finite cap on `|{(d, v) : v ∈ dom(Σ.M(d)) ∧ Σ.M(d)(v) = a}|` is entailed by these properties — neither across documents nor within a single document. The remaining properties of the ASN (S7 onwards — document-level allocation, ordered arrangement structure, operations) are not checked: the constructions use minimal traces that make no claims about V-position ordering or operational behaviour, and extending the consistency result to those properties would require constructions of substantially different character. ∎

*Formal Contract:*
- *Preconditions:* `N ∈ ℕ` arbitrary; AX-1 (initial empty state); AX-2 (allocation-mediated entry, ensuring content-creation transitions are T10a-conforming allocation events); AX-3 (infinite type domains — the document identifier space and V-position spaces are countably infinite, ensuring the constructions can produce `N + 1` pairwise-distinct elements for arbitrary `N`).
- *Postconditions:* There exists a conforming execution trace `σ = ⟨Σ₀, …, Σₖ⟩` with `k ≥ 1` — one in which every transition satisfies S0 (content immutability) and S1 (store monotonicity) (and therefore S6, which is formally identical to S1), every state satisfies S2 (arrangement functionality) and S3 (referential integrity), S4 (origin-based identity) holds for all allocation events, and AX-2 (allocation-mediated entry) holds — such that for some `a ∈ dom(Σₖ.C)`, `|{(d, v) : v ∈ dom(Σₖ.M(d)) ∧ Σₖ.M(d)(v) = a}| > N`. The construction works both across documents (multiplicity `N + 1` over `N + 1` documents) and within a single document (multiplicity `N + 1` at `N + 1` distinct V-positions). In both constructions, S0 and S1 are non-vacuously satisfied at `N + 1` of the `N + 2` transitions — the transitions where the content store already contains the address `a`.
- *Frame:* S0–S4 and AX-2 are the properties checked (S6 is formally identical to S1). AX-2 holds non-vacuously: the content-creation transition at `Σ₀ → Σ₁` introduces `a` into `dom(C)` via a T10a-conforming allocation event. S4 holds vacuously in both constructions — a single allocation event provides no pair to violate. Properties S7 onwards (document-level allocation, ordered arrangement structure, operations) are not checked; the constructions are minimal — single I-address, trivial arrangements — and make no claims about V-position ordering or operational behaviour.
