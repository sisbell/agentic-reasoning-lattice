# Cone Review — ASN-0036/S7 (cycle 2)

*2026-04-14 12:06*

### S7 uniqueness proof treats "distinct documents" as implying "distinct allocation events" without defining either "document" or the implication

**Foundation**: GlobalUniqueness (ASN-0034) — precondition: "a, b ∈ T produced by distinct allocation events … within a system conforming to T10a"; S7d — axiom: "origin(a) was produced by a T10a-conforming allocation event"
**ASN**: S7 proof, Uniqueness across documents — "For documents d₁ ≠ d₂, their document-level tumblers are products of distinct allocation events within a T10a-conforming system; GlobalUniqueness (ASN-0034) therefore guarantees their distinctness"
**Issue**: GlobalUniqueness's precondition requires *distinct allocation events*, not *distinct documents*. The proof asserts that "d₁ ≠ d₂" implies their tumblers are "products of distinct allocation events," but this step is the entire logical work and it is assumed, not derived. The ASN never formally defines what a "document" is or what "d₁ ≠ d₂" means. S7d provides that each `origin(a)` was produced by *some* T10a event, but says nothing about whether two different documents' prefixes come from *different* events. If "document" is identified with its tumbler, then d₁ ≠ d₂ already means the tumblers differ and postcondition (c) is tautological — GlobalUniqueness is unnecessary. If "document" is an abstract entity distinct from its tumbler, the implication "distinct documents → distinct allocation events" needs a formal bridge that is nowhere stated. Either way, the current proof has a gap: it uses GlobalUniqueness to establish something that is either trivially true without it (if documents = tumblers) or requires an unstated definition and lemma (if documents ≠ tumblers).
**What needs resolving**: The ASN must formally define "document" and "distinct documents." If a document IS its tumbler, the GlobalUniqueness step should be removed from postcondition (c)'s proof and replaced with a note that the conclusion is definitional. If a document is an abstract entity created by an allocation event, the definition must be stated and the implication "distinct documents → distinct allocation events → distinct tumblers" must be made explicit with both links justified.

### S4 lists T3 as a precondition for a decidability claim that is absent from the postconditions

**Foundation**: T3 (CanonicalRepresentation, ASN-0034) — "Tumbler equality is sequence equality: a = b ⟺ #a = #b ∧ (A i : 1 ≤ i ≤ #a : aᵢ = bᵢ)"
**ASN**: S4 formal contract — "Preconditions: … T3 (CanonicalRepresentation, ASN-0034)"; S4 postcondition — "a₁ ≠ a₂, regardless of whether Σ.C(a₁) = Σ.C(a₂)"; S4 proof final paragraph — "the distinctness a₁ ≠ a₂ is decidable from the addresses alone by T3 … computable in time proportional to the shorter address"
**Issue**: S4's postcondition is a bare inequality: `a₁ ≠ a₂`. The proof establishes this in its first paragraph using GlobalUniqueness alone — "GlobalUniqueness yields a₁ ≠ a₂ directly." T3 appears only in the final paragraph, which makes two additional claims not captured in the postcondition: (1) the inequality is decidable from component sequences, and (2) it is computable in linear time. These are commentary about the computational character of the result, not part of the formal theorem. A formalizer translating S4 into TLA+ would put GlobalUniqueness and T10a in the ASSUME clause; T3 would have no role in the PROVE clause. Listing T3 as a precondition creates a spurious dependency in the formal DAG — any property citing S4 inherits a T3 dependency it does not need. The same pattern recurs in S7's uniqueness proof, where T3 again appears for decidability ("this distinctness is decidable by component-wise comparison") without contributing to the truth of postcondition (c).
**What needs resolving**: Either add a decidability postcondition to S4 (e.g., "the inequality a₁ ≠ a₂ is decidable from the component sequences alone") so T3 has a postcondition to support, or remove T3 from S4's precondition list and acknowledge the decidability discussion as commentary. The same resolution should be applied to S7's use of T3 in the uniqueness section.

### origin(a) is used in S7d's axiom but formally defined only in S7, which depends on S7d

**Foundation**: T4 (HierarchicalParsing, ASN-0034) — field decomposition into four named fields
**ASN**: S7d axiom — "the document-level prefix origin(a) — the tumbler N.0.U.0.D with zeros(origin(a)) = 2 — was produced by a T10a-conforming allocation event"; S7 formal contract — "*Definition:* origin(a) = (fields(a).node).0.(fields(a).user).0.(fields(a).document)"; S7 preconditions include S7d
**Issue**: The function `origin(a)` first appears in S7d's axiom, which provides an inline description ("the tumbler N.0.U.0.D") but not a formal definition — S7d's preconditions (S7b, T4) do not define `origin`, and S7d's formal contract has no definition section. The canonical definition is in S7's formal contract. But S7 lists S7d as a precondition, creating a forward reference: the formalizer must formalize S7d before S7 (since S7 depends on S7d's axiom), yet S7d uses `origin(a)` whose definition is housed in S7. The two characterizations — S7d's "N.0.U.0.D" and S7's `(fields(a).node).0.(fields(a).user).0.(fields(a).document)` — are intensionally the same but are stated independently, so their equivalence is assumed rather than established.
**What needs resolving**: Extract the definition of `origin(a)` into a standalone definition that precedes both S7d and S7 in the property ordering. Both S7d's axiom and S7's formal contract should reference this single definition rather than each providing their own characterization. This eliminates the forward reference and ensures there is exactly one canonical definition of `origin`.
