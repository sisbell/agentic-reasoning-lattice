# Cone Review — ASN-0036/ValidInsertionPosition (cycle 2)

*2026-04-15 01:00*

### ValidInsertionPosition's sequential enumeration of V_S(d) requires an unstated intermediate result

**Foundation**: D-CTG (VContiguity), S8-depth (Fixed-depth V-positions), S8-fin (FiniteArrangement), D-MIN (VMinimumPosition), S8a (VPositionWellFormedness)
**ASN**: ValidInsertionPosition, body text: "In the non-empty case, there are exactly N + 1 valid insertion positions: the N positions coinciding with existing V-positions v₀ through v_{N−1}, plus the append position shift(min(V_S(d)), N)."
**Issue**: The formal definition parameterizes valid positions as `shift(min(V_S(d)), j)` for `0 ≤ j ≤ N` where `N = |V_S(d)|`. The body text claims these positions "coincide with existing V-positions" for `j < N` and that `j = N` is the "append position" (first position beyond the existing range). Both claims depend on V_S(d) having the sequential form `{shift(min, 0), shift(min, 1), ..., shift(min, N−1)}` — i.e., that existing positions differ only in their last component. At depth 2 this follows from D-CTG + D-MIN + S8-fin directly. At depth ≥ 3, it additionally requires D-CTG-depth's shared-prefix reduction (that all positions in V_S(d) must agree on components 2 through m−1), which is a non-trivial derivation from D-CTG + S8-fin + S8a (positions with differing intermediate components force infinitely many intermediates via D-CTG, contradicting S8-fin). The raw preconditions for this derivation are all cited (D-CTG, S8-fin, S8a), but the derivation itself is absent and D-CTG-depth is not cited. Without this step: (a) the coincidence claim is unproven at depth ≥ 3, (b) the semantic interpretation of `j` as indexing existing positions is unsubstantiated, (c) the characterization of `shift(min, N)` as beyond all existing positions is unverified — it could coincide with an existing V-position whose intermediate components differ from min's, and (d) the choice of `N = |V_S(d)|` as the upper bound for `j` lacks its semantic justification.
**What needs resolving**: ValidInsertionPosition must either cite D-CTG-depth (or D-SEQ) as a precondition and invoke its conclusion, or include the derivation showing that D-CTG + S8-fin + S8a forces the sequential form at all depths. The formal definition is self-consistent without this, but the body text's interpretive claims — which are load-bearing for any operation that uses this predicate — depend on it.

### ValidInsertionPosition's empty-case example overclaims S8-depth's permanence

**Foundation**: S8-depth (Fixed-depth V-positions)
**ASN**: ValidInsertionPosition, examples section: "For m = 3, the unique valid position is [1, 1, 1]; by T3, this is a different tumbler — once placed, S8-depth locks the subspace to depth 3 for all future positions."
**Issue**: S8-depth's formal invariant is `(A d ∈ D, p, q : p ∈ dom(Σ.M(d)) ∧ q ∈ dom(Σ.M(d)) ∧ p₁ = q₁ : #p = #q)` — it constrains positions that coexist in a single reachable state. If an operation atomically empties V_S(d) and repopulates it at a different depth m', the resulting state satisfies S8-depth (all positions in the new state share depth m'), D-MIN (the new minimum is `[S, 1, ..., 1]_{m'}`), and D-CTG (contiguity rebuilt from the new population). No invariant in the property set establishes that a subspace's depth, once determined, is permanent across states. The phrase "locks the subspace to depth 3 for all future positions" asserts a temporal permanence that S8-depth's state-level invariant does not provide. An implementer reading this guidance would incorrectly reject a depth change through an empty-and-refill operation that the formal invariants permit.
**What needs resolving**: Either weaken the claim to "locks the subspace to depth 3 as long as it remains non-empty" (matching what S8-depth actually guarantees), or introduce an explicit permanence axiom if the specification intends depth to be irrevocable once chosen.
