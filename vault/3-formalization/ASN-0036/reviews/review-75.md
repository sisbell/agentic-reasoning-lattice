# Cone Review — ASN-0036/D-CTG-depth (cycle 6)

*2026-04-14 22:20*

### D-CTG-depth's last-component reduction requires T-membership at specific values that no foundation property guarantees
**Foundation**: T0(a) (UnboundedComponentValues, ASN-0034): "for every bound `M ∈ ℕ`, there exists `t' ∈ T` … where `t'.dᵢ > M`" — guarantees existence of tumblers *exceeding* any bound, not tumblers at *specific* component values
**ASN**: D-CTG-depth Reduction: "D-CTG contiguity reduces to `(A n : (v₁)ₘ < n < (v₂)ₘ : π.n ∈ V_S(d))`"
**Issue**: The reduction rewrites D-CTG's quantifier — which ranges over `v ∈ T` — into one ranging over `n ∈ ℕ`. For this rewriting to be an equivalence, the map `v ↦ vₘ` must be surjective over the integer range `((v₁)ₘ, (v₂)ₘ)`, which requires `π.n ∈ T` for every `n` in that range. T0(a) guarantees tumblers in T with component values *exceeding* any bound but not at every intermediate value — the axioms are consistent with T being a strict subset of all finite positive-integer sequences (e.g., sequences whose components are all powers of 2 would satisfy T0(a), T1, T3, and T4). If some `π.n ∉ T`, then D-CTG's quantifier doesn't range over it, but the reduced form asserts `π.n ∈ V_S(d)` — an impossible membership since `V_S(d) ⊆ T`. The reduced form would then be strictly stronger than D-CTG, not equivalent to it. The Proof section (shared-prefix result) is unaffected — it uses T0(a) correctly, producing intermediates that are guaranteed to be in T. The gap is localized to the Reduction's rewriting step.
**What needs resolving**: Either establish that T contains all finite sequences of positive integers (making π.n ∈ T trivial for n ≥ 1), or state the reduction with the T-membership guard: `(A n : (v₁)ₘ < n < (v₂)ₘ ∧ π.n ∈ T : π.n ∈ V_S(d))`, which follows from D-CTG and postcondition (2) without any density assumption.

### The document parameter d is universally quantified in every formal contract without a carrier set
**Foundation**: T4 (HierarchicalParsing, ASN-0034): defines a four-level hierarchy where documents sit at level 3 (three separating zeros)
**ASN**: D-CTG invariant: `(A d, S, u, q : u ∈ V_S(d) ∧ …)`; S8-depth invariant: `(A d, p, q : p ∈ dom(Σ.M(d)) ∧ …)`; S8-fin: "dom(Σ.M(d)) is finite for every document d"; Σ.M(d) axiom: `Σ.M(d) : T ⇀ T`
**Issue**: Every property quantifies `∀ d` but no property specifies the set from which `d` is drawn. Σ.M(d) declares the arrangement for "document `d`" without constraining what values `d` can take — `M(d)` is defined for arbitrary `d` of unspecified type. T4's hierarchy implies documents are tumblers with exactly three separating zeros, which would give a concrete type: `d ∈ {t ∈ T : zeros(t) = 3}`. But this connection is never made. S8-fin uses the prose phrase "every document `d`" as if "document" denotes a set, while D-CTG and S8-depth quantify `d` alongside tumbler-valued variables without distinguishing its type. For TLA+ formalization, each `∀ d` requires a domain: `\A d \in D : ...`. No formal contract in the ASN provides `D`.
**What needs resolving**: A definition — either in Σ.M(d) or as a standalone definition property — that specifies the carrier set for `d` (e.g., `D = {t ∈ T : zeros(t) = 3}` if documents are T4 level-3 tumblers), so that the universal quantification in every formal contract has a defined domain.

### Formal contracts alternate between Σ.M(d) and M(d) for the same state component, with inconsistent state binding across invariants
**Foundation**: Σ.M(d) (Arrangement): introduces the notation `Σ.M(d) : T ⇀ T`
**ASN**: S8-depth invariant: `dom(Σ.M(d))`; S8-depth preconditions: `dom(M(d)) ⊆ T (Σ.M(d))`. D-CTG preconditions: `dom(M(d)) ⊆ T (Σ.M(d))`. S8-fin invariant: "`dom(Σ.M(d))` is finite for every document `d` in every reachable state `Σ`". D-CTG invariant: quantifies over `V_S(d)` without binding Σ. Base cases throughout: `dom(Σ₀.M(d)) = ∅`.
**Issue**: The same mathematical object — the domain of the arrangement of `d` — appears in three notational variants: `Σ.M(d)` (state explicit), `M(d)` (state suppressed), and `Σ₀.M(d)` (specific state). These coexist within single properties: S8-depth's invariant uses `dom(Σ.M(d))` while its preconditions use `dom(M(d))`. More substantively, state binding is inconsistent across the three peer invariants. S8-fin explicitly quantifies over "every reachable state Σ" — making Σ a bound variable in the invariant statement. D-CTG and S8-depth state their formulas without binding Σ at all, leaving it unclear whether the invariant is parameterized by a specific state (which one?) or implicitly universally quantified over reachable states. For TLA+ formalization, an invariant is a state formula evaluated in each state of a behavior — the state is always implicit. The ASN gives Σ three distinct roles (bound variable in S8-fin, notation prefix in S8-depth, absent in D-CTG) for what TLA+ treats as a single concept.
**What needs resolving**: Adopt a uniform convention — either Σ is always explicit (with consistent binding: "for every reachable state Σ" in each invariant) or always implicit (using `M(d)` throughout, with a single framing statement that all invariants hold in every reachable state). The current hybrid makes it ambiguous whether properties that suppress Σ are intentionally state-independent or merely abbreviating.
