# Cone Review — ASN-0036/S7 (cycle 1)

*2026-04-14 11:44*

### S7 permanence proof applies S4/GlobalUniqueness to element-level addresses without a formal bridge establishing they are T10a allocation outputs

**Foundation**: GlobalUniqueness (ASN-0034) — precondition: "a, b ∈ T produced by distinct allocation events — where an allocation event is either root initialization or an invocation of inc(t, k) — within a system conforming to T10a"; S4 — precondition: "a₁, a₂ ∈ dom(Σ.C) produced by distinct allocation events within a system conforming to T10a"
**ASN**: S7d axiom — "For every a ∈ dom(Σ.C), the document-level prefix origin(a) was produced by a T10a-conforming allocation event"; S7d body — "Nelson's baptism principle operates at every level of the hierarchy: node operators allocate user prefixes, users allocate document prefixes, documents allocate element addresses. Each level is governed by T10a's allocator discipline"; S7 permanence proof — "By S4 (origin-based identity), distinct allocation events produce distinct addresses, so the address a itself is never reassigned or reused"
**Issue**: S7d's own body text explains exactly why the allocation-event bridge is needed: "Without this axiom, GlobalUniqueness — whose precondition demands that the addresses in question be 'produced by distinct allocation events within a system conforming to T10a' — cannot be discharged for document-level tumblers." S7d then provides this bridge for document-level prefixes. The body text further acknowledges that element-level allocation is also "governed by T10a's allocator discipline" — but the formal axiom covers only `origin(a)`, not `a` itself. S7's permanence proof then applies S4/GlobalUniqueness to the element-level address `a` (not `origin(a)`) to conclude "the address a itself is never reassigned or reused." For S4 to apply, `a` must be a product of a T10a allocation event — exactly the bridge S7d provides for `origin(a)` but not for `a`. S7a asserts that "the allocation that placed a into dom(C)" occurred, but uses informal language rather than T10a's formal vocabulary ("allocation event: root initialization or invocation of inc(t, k)"), so the link from S7a's "allocation" to GlobalUniqueness's "allocation event" is assumed, not established.
**What needs resolving**: An element-level analog of S7d — asserting that every `a ∈ dom(Σ.C)` was itself produced by a T10a-conforming allocation event (not just that `origin(a)` was) — must be stated, either as a standalone axiom or by strengthening S7d's scope to cover both levels. Alternatively, S7a's axiom language must be tightened to use T10a's formal allocation-event vocabulary so it can discharge S4/GlobalUniqueness's precondition directly.

### T4 names the first hierarchical field "network"; S7 and S7a call it "node" throughout

**Foundation**: T4 (HierarchicalParsing, ASN-0034) — "Address tumblers encode a four-level containment hierarchy (network, user, document, element)"
**ASN**: S7 definition — "origin(a) = (fields(a).node).0.(fields(a).user).0.(fields(a).document)"; S7 proof — "fields(a) decomposes a into four fields: node, user, document, and element"; S7a — "the subtumbler N.0.U.0.D consisting of the node, user, and document fields with their separators"
**Issue**: T4 defines the first hierarchical level as "network." ASN-0036 uniformly calls it "node" — in S7a's body text, in S7's formal definition of `origin`, and in S7's proof. Since `fields(a)` is defined by T4's decomposition, the accessor `fields(a).node` references a field that T4 calls "network." A formalizer implementing `fields` from T4 would define a `.network` accessor; every use of `.node` in ASN-0036 would fail to resolve. The inconsistency is systematic (every reference in ASN-0036 says "node") rather than a one-off typo, suggesting the two ASNs were written against different naming conventions.
**What needs resolving**: Either ASN-0036 must adopt T4's "network" terminology, or T4 must be updated to use "node," or the alias must be explicitly declared (e.g., "the network field (hereafter 'node')") at the point where ASN-0036 first references T4's field names. The definition of `origin(a)` and all subsequent field references must use whichever name T4 canonically defines.
