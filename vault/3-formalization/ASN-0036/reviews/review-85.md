### AX-5 body text claims "exactly one" operation per transition, but the formal contract uses a plain existential quantifier

**Foundation**: AX-5 (ClosedWorldTransition) — formal axiom: `(A Σ, Σ' : Σ' is a successor of Σ in a system trace : (E op : op ∈ Op : Σ' = op(Σ)))`
**ASN**: AX-5 body text: "For every pair of consecutive reachable states Σ, Σ' in a system trace, there exists exactly one op ∈ Op such that Σ' is the result of applying op to Σ." TLA+ gloss: `Next ≡ INSERT ∨ DELETE ∨ COPY ∨ MOVE ∨ REARRANGE ∨ APPEND`.
**Issue**: The body text asserts unique existence — "exactly one op" — a claim that for any state pair (Σ, Σ'), the producing operation is uniquely determined. The formal axiom uses the standard existential `(E op : op ∈ Op : ...)`, which guarantees only that some operation produced the transition, not that only one could. The TLA+ translation uses inclusive disjunction (∨), which is consistent with the plain existential — multiple disjuncts could be satisfiable for the same (Σ, Σ') pair. Unique existence is strictly stronger: it would mean the operation is identifiable from the state pair alone, which requires that no two distinct operations ever produce the same state change. The inductive proofs throughout ASN-0036 (S8-fin, S8-depth, D-CTG) cite AX-5 only for exhaustiveness — they need "some op did it" to case-split, not "only that op could have done it." So the formal contract is sufficient for all current downstream consumers. But the body text overclaims relative to the formal contract and the TLA+ gloss, and a reader taking the body text at face value would believe the system provides a stronger determinism guarantee than the formal contract establishes.
**What needs resolving**: Either the body text should be weakened to match the formal contract ("there exists an op ∈ Op" rather than "exactly one op"), or the formal contract should be strengthened to a unique existential `(E! op : op ∈ Op : Σ' = op(Σ))` if the uniqueness claim is architecturally intended. The TLA+ gloss should also be reconciled — unique existence would require the disjuncts to be mutually exclusive for any given (Σ, Σ').

### D-CTG body text claims `subspace` is not defined on T; SubspaceIdentifier explicitly says it is — and names D-CTG as an example

**Foundation**: SubspaceIdentifier — definition: `subspace(v) = v₁` for `v ∈ T` with `#v ≥ 1`; preconditions: `v ∈ T`, `#v ≥ 1`; body text: "Since v₁ is defined for any tumbler with at least one component, subspace applies throughout T wherever #v ≥ 1 — including contexts where V-position membership has not yet been established (D-CTG's inner quantifier, OrdAddHom's result tumbler, D-CTG-depth's constructed intermediates)."
**ASN**: D-CTG (VContiguity) body text: "The guard `v₁ = S` is first-component extraction — for V-positions this coincides with `subspace(v)` (SubspaceIdentifier), but the invariant quantifies over all `v ∈ T`, where `subspace` is not defined, so the component notation is used directly." D-CTG formal contract annotation: "The guard `v₁ = S` is first-component extraction (equivalent to `subspace(v)` on V-positions; see SubspaceIdentifier)."
**Issue**: T0 (CarrierSetDefinition, ASN-0034) defines T as all finite sequences over ℕ with length ≥ 1, so `#v ≥ 1` for every `v ∈ T`. SubspaceIdentifier's preconditions are `v ∈ T, #v ≥ 1` — both satisfied by every element of T. SubspaceIdentifier's body text explicitly names D-CTG's inner quantifier as an example context where `subspace` applies to non-V-position tumblers. D-CTG's body text asserts the opposite: "the invariant quantifies over all `v ∈ T`, where `subspace` is not defined." These two properties directly contradict each other on the scope of a shared definition. D-CTG's formal contract annotation narrows the equivalence to V-positions ("equivalent to `subspace(v)` on V-positions"), which is unnecessarily restrictive but not false. The body text goes further by asserting non-definition, which is false. The contradiction matters for formalization: SubspaceIdentifier exists precisely to provide a single reusable accessor for the first component across all of T; D-CTG's body text tells the formalizer that this accessor doesn't apply in D-CTG's inner quantifier, forcing a duplicate mechanism for what should be one definition.
**What needs resolving**: D-CTG's body text must acknowledge that `subspace(v)` is defined on all of T (matching SubspaceIdentifier's stated scope and T0's guarantee that `#v ≥ 1`). The notation choice (`v₁ = S` vs `subspace(v) = S`) is a stylistic preference, not a scope limitation — the body text should not justify the choice by claiming `subspace` is undefined where SubspaceIdentifier explicitly says it is.
