### S8 table label "Span decomposition" introduces an undefined relationship with the foundation's formal Span concept

**Foundation**: Span (ContiguousAddressRange, ASN-0034) — `span(s, ℓ) = {t ∈ T : s ≤ t < s ⊕ ℓ}`, a pair `(s, ℓ)` denoting a set of tumblers at *all* depths within the T1 interval. T12 (SpanWellDefinedness) establishes well-formedness conditions for these objects.
**ASN**: S8 table entry: "Span decomposition: `dom(M(d))` decomposes into finitely many correspondence runs `(vⱼ, aⱼ, nⱼ)` with `M(d)(vⱼ + k) = aⱼ + k` for `0 ≤ k < nⱼ`". S8's formal content defines and works entirely with *correspondence runs* — triples `(v, a, n)` pairing V-position ranges with I-address ranges at a fixed depth.
**Issue**: The foundation defines Span as a formal mathematical object: a `(start, displacement)` pair whose denotation `{t ∈ T : s ≤ t < s ⊕ ℓ}` includes tumblers at every depth within the T1 interval. A correspondence run's V-range — the positions `v, shift(v, 1), ..., shift(v, n-1)` — is a *depth-restricted* subset of a Span, containing only the `n` tumblers at the common depth `m`. These are structurally different objects: a Span is a pair in `T × T`, a correspondence run is a triple in `T × T × ℕ`; a Span denotes a (generally infinite) set of tumblers across all depths, a correspondence run identifies `n` specific positions at one depth. The ASN never cites T12, never constructs a Span object, and establishes no formal bridge between Spans and correspondence runs. Yet the table labels S8 with the foundation's terminology. A downstream ASN needing to express "the V-range of a correspondence run as a Span" (e.g., for span arithmetic or span overlap detection) would find no formal connection to build on — and might incorrectly import T12's well-formedness machinery, whose precondition `actionPoint(ℓ) ≤ #s` governs a different object than the run-length `n`.
**What needs resolving**: Either S8's table label must use terminology that does not collide with the formal Span definition (e.g., "correspondence-run decomposition"), or the ASN must formally define the relationship between a correspondence run `(v, a, n)` and the Span `(v, δ(n, m))` — establishing that `{shift(v, k) : 0 ≤ k < n} = span(v, δ(n, m)) ∩ {t ∈ T : #t = m}` under the depth-uniformity constraint S8-depth.

### S5 constructions claim "no transitions" but S0 is universally quantified over all reachable successor states, not just transitions from the constructed state

**Foundation**: N/A — internal structural issue in S5's proof method.
**ASN**: S5 proof (cross-document construction): "S0 (content immutability) and S1 (store monotonicity) are universally quantified over state transitions `Σ → Σ'`; the constructed state `Σ_N` admits no transitions, so both predicates hold vacuously." S5 proof (within-document construction): identical claim.
**Issue**: S0's formal statement is an *invariant*: "For every state transition `Σ → Σ'`: `a ∈ dom(Σ.C) ⟹ a ∈ dom(Σ'.C) ∧ Σ'.C(a) = Σ.C(a)`." The invariant quantifies over all transitions in the system, not just transitions originating from the constructed state. The proof's claim that `Σ_N` "admits no transitions" is a model-theoretic move: it constructs a standalone model containing only `Σ_N`, in which the transition quantifier ranges over an empty set. This is valid for demonstrating *logical consistency* (S0–S3 don't entail a bound), but the proof doesn't explicitly acknowledge that the consistency it demonstrates is model-theoretic — that the constructed states need not be *reachable* from the initial state `Σ₀` via any finite operation sequence. The same constructions do happen to be reachable (e.g., create one document with `N+1` transclusions of the same content), and they satisfy all additional invariants (S8-fin, D-CTG, D-MIN, S8-depth, S8a), but neither fact is verified. The proof's "admits no transitions" framing obscures the distinction between model-theoretic consistency and operational reachability that the rest of the ASN carefully maintains (e.g., S3's inductive proof over operation traces, S8-fin's induction on trace length).
**What needs resolving**: The S5 proof should either (a) explicitly state that the consistency claim is model-theoretic and that reachability is not asserted, or (b) verify the constructions against reachability — showing each `Σ_N` is reachable from `Σ₀` by a finite operation sequence, which would strengthen the result from "S0–S3 are consistent with unbounded sharing" to "unbounded sharing is achievable in the system."

### D-CTG's universal inner quantifier ranges over all tumblers in T but the proof of S8's partition implicitly restricts to dom(M(d)) without noting the gap

**Foundation**: T1 (LexicographicOrder, ASN-0034) — total order on T; S8-depth — common depth within a subspace; S8a — all V-position components positive.
**ASN**: D-CTG axiom: `(A v : subspace(v) = S ∧ #v = #u ∧ u < v < q : v ∈ V_S(d))`. S8 partition proof, "uniqueness within a subspace": "Suppose for contradiction that `t ≠ v` satisfies `#t = m` and `v ≤ t < v + 1`." The proof proceeds by case analysis on same-depth tumblers only.
**Issue**: D-CTG quantifies over *all* `v ∈ T` satisfying `subspace(v) = S ∧ #v = #u ∧ u < v < q`, forcing every such tumbler into `V_S(d) ⊆ dom(M(d))`. In particular, D-CTG can force tumblers into `dom(M(d))` that were not placed there by any operation — it is a *constraint on valid states*, not a construction. The S8 partition proof's uniqueness argument, however, considers only tumblers of depth `m` (the common V-position depth from S8-depth). It does not explicitly argue that D-CTG cannot force tumblers of *other* depths into `V_S(d)`. The resolution is that `V_S(d) = {v ∈ dom(M(d)) : v₁ = S}` and S8-depth constrains all members to depth `m`, so D-CTG's quantifier `#v = #u = m` restricts to depth-`m` tumblers by construction. But this argument — that D-CTG's `#v = #u` condition, combined with S8-depth's uniformity, means D-CTG only forces same-depth tumblers — is never stated in S8's proof. The proof jumps from D-CTG's axiom to the case analysis on depth-`m` tumblers without bridging the two constraints.
**What needs resolving**: S8's partition proof should explicitly note that D-CTG's inner quantifier includes `#v = #u`, which by S8-depth equals the common depth `m`, so D-CTG can only force depth-`m` tumblers into `V_S(d)` — closing the gap between D-CTG's universal quantification over T and the proof's restriction to depth-`m` tumblers.
