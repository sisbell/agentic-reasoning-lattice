# Sub-Questions — Concurrency and Global Indexes

**Inquiry:** What consistency must the system guarantee when multiple users concurrently modify different documents that share I-space addresses? What coordination do the global indexes (span index, link index) require? How does single-writer ownership interact with shared-read global state?

1. [nelson] When two users simultaneously insert content into their own separate documents, must the system guarantee that neither insertion can observe a partially completed state from the other?
2. [nelson] If the same original content appears in multiple documents through transclusion, must an edit to one document's structure leave the shared content byte-identical for all other documents that reference it?
3. [nelson] Must the global index that tracks where every piece of content currently resides be updated atomically with respect to each document operation, or may it lag behind?
4. [nelson] When a new link is created in one document while another user is retrieving links for a different document, must the link index present a consistent snapshot, or may it reflect a partial update?
5. [nelson] Does single-writer ownership of a document guarantee that no coordination is ever needed between writers of different documents, even when those documents share content through transclusion?
6. [nelson] If a reader queries "find all documents containing this content" while a writer is mid-operation on one of those documents, what must the reader see — the state before the operation, after, or is either acceptable?
7. [nelson] Must the system enforce a total ordering of all operations across all documents, or is it sufficient that operations within each document are ordered and operations on different documents are independent?
8. [nelson] When a user creates a new version of a document, must the global content index and the global link index both reflect the new version at the same instant, or may they update separately?
9. [nelson] Can the owner of a document ever be blocked or delayed by operations another user performs on a completely separate document, or must document independence be absolute?
10. [nelson] If two documents both contain links whose endpoints reference content in each other, must any operation on either document preserve the validity of all four link endpoints simultaneously?

12. [gregory] If two users each own different documents that share I-addresses via transclusion, and both perform INSERT simultaneously into their respective documents, does `findisatoinsertmolecule` ever return the same I-address for both allocations — given that each document has an independent I-address range under its own document orgl?
13. [gregory] When session A is executing a `findlinksfromtothreesp` spanfilade traversal and session B completes an INSERT that adds a new DOCISPAN entry to the same spanfilade, can session A see a partially-written spanfilade node — or does the run-to-completion guarantee mean session A's traversal either fully precedes or fully follows session B's write?
14. [gregory] Does the grim reaper's cache eviction (`grimlyreap`) hold any lock or guard that prevents a second session's operation from reading a half-written disk block if eviction is triggered mid-operation — or is this impossible because the event loop is single-threaded and eviction completes synchronously within the triggering operation?
15. [gregory] When session A has document D open with WRITEBERT and session B calls `find_links` with a specset that resolves to I-addresses shared with document D, does session B's spanfilade search require any BERT check on document D, or does the global spanfilade bypass per-document access control entirely?

[gregory] When two sessions each CREATELINK in different documents but both links reference the same I-address range as an endset, do both `insertendsetsinspanf` calls write to the same spanfilade region — and if so, can the second write's `splitcrumupwards` corrupt the tree structure left by the first write, or does single-threaded execution prevent this?
18. [gregory] After session A performs DELETE on document D₁ removing content at I-addresses that session B's document D₂ also references via transclusion, does session B's subsequent `find_links` on D₂ still discover links whose endsets overlap those I-addresses — confirming that DELETE in D₁ has zero effect on D₂'s link discoverability?
19. [gregory] If a session crashes (TCP connection drops) mid-INSERT — after `createenf` has appended to the granfilade but before `insertpm` has updated the POOM — does the orphaned granfilade entry affect any other session's I-address allocation via `findisatoinsertmolecule`, causing the next allocation to skip past the orphaned range?
20. [gregory] When multiple sessions are connected and the `writeenfilades` flush occurs at session end, does it flush ALL modified nodes across ALL sessions' operations, or only nodes dirtied by the exiting session — and if the former, could a still-active session's partially-built tree state be written to disk prematurely?
