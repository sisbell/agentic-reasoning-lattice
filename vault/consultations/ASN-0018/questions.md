# Sub-Questions — Concurrent Operation Ordering

**Inquiry:** What must the system guarantee about the result when multiple operations modify the same or overlapping content simultaneously? What determines the final state and what invariants must the resolution satisfy?

1. [nelson] When two users simultaneously insert content at the same position within a shared document, must the system guarantee a single deterministic ordering of those insertions visible to all readers?
2. [nelson] If one operation deletes a span of content while another operation concurrently creates a link to that same span, must the link survive and continue to reference the original content?
3. [nelson] Must every modification to a document produce a distinct, permanently retrievable version, even when multiple modifications arrive simultaneously?
4. [nelson] When overlapping edits target the same content within a document, must the system preserve every user's contributed content without silent loss, or may one edit supersede another?
5. [nelson] Does the system guarantee that the identity of any content byte remains stable regardless of how many concurrent operations attempt to rearrange or copy the surrounding material?
6. [nelson] If two users simultaneously transclude the same passage into different documents, must both transclusions independently reflect any concurrent edits to the original passage?
7. [nelson] Must the system assign unambiguous authorship and ownership to each fragment of content that results from resolving simultaneous modifications by different users?
8. [nelson] When concurrent operations produce a conflict over document structure, must the resolution satisfy the invariant that no content ever added to the system becomes unreachable or unaddressable?
9. [nelson] Must the system guarantee that the sequence of versions for a document forms a single linear history, or may concurrent modifications produce branching version histories that later require explicit merging?
10. [nelson] If simultaneous operations each modify content that is shared across multiple documents through transclusion, must every document referencing that content converge to a consistent view of the resolved state?
11. [gregory] When two FEBE sessions issue INSERT at the same V-position in the same document, does the run-to-completion scheduling in `bed.c`'s `select()` loop guarantee a deterministic winner, and does the second INSERT see the shifted V-addresses produced by the first?
12. [gregory] If session A issues DELETE on V-span [1.3,1.5] and session B issues INSERT at V:1.4 in the same document, does the final POOM state depend entirely on which operation the event loop processes first, with no merging or conflict detection?
13. [gregory] When two sessions each COPY overlapping I-address ranges into the same target document at the same V-position, does `isanextensionnd` coalesce the second copy's POOM entries with the first's if their I-addresses happen to be contiguous?
14. [gregory] Does `makegappm` in `insertnd.c` shift ALL entries at or beyond the insertion point in a single tree walk, or could a concurrent cache eviction via `grimlyreap` leave the POOM in a partially-shifted state mid-operation?
15. [gregory] If session A performs REARRANGE (pivot) on V:[1.1,1.3,1.5] while session B simultaneously issues DELETE on V:[1.2,1.4], does the event loop's atomic processing of each operation guarantee that the second operation sees a fully consistent POOM from the first, with no interleaved crum modifications?
16. [gregory] When two CREATELINK operations from different sessions target the same document, does `findnextlinkvsa` always return distinct link V-addresses because the second call sees the first link's POOM entry, or is there a race in the query-and-increment allocation?
17. [gregory] If session A deletes content at I-address range [.0.1.3,.0.1.5] from doc1 while session B simultaneously copies that same I-address range from doc1 into doc2, does `vspanset2sporglset` in session B's COPY read doc1's pre-delete or post-delete POOM depending solely on event loop ordering?
18. [gregory] After two overlapping DELETE operations on the same document — first deleting V:[1.2,1.4] then V:[1.1,1.3] — can `tumblersub` shifting from the second DELETE produce negative V-position tumblers for entries that were already shifted left by the first DELETE?
19. [gregory] When session A performs INSERT at V:1.3 and session B performs CREATENEWVERSION on the same document, does the version capture the pre-insert or post-insert text depending entirely on event loop order, and does `docopyinternal` take a consistent snapshot of the text subspace POOM?
20. [gregory] If two sessions issue APPEND and INSERT to the same document in rapid succession, does the I-address allocation via `findisatoinsertmolecule` guarantee non-overlapping I-addresses because the second operation's `findpreviousisagr` query sees the first operation's granfilade entry?
