**T4b (UniqueParse).** Under the constraints of T4 — at most three zero-valued components as field separators, every field component strictly positive, every present field non-empty — the function `fields(t)` that decomposes a tumbler into its node, user, document, and element sub-sequences is well-defined and uniquely determined by `t` alone.

The argument turns on a single observation: the positive-component constraint makes the separator positions exactly recoverable from the raw tumbler. We claim that a position `i` satisfies `tᵢ = 0` if and only if `i` is a field separator. The forward direction holds by construction: every separator has value 0 in the T4 address format `N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`. For the reverse direction: if `tᵢ = 0`, then `i` cannot be a field component, because the positive-component constraint requires every field component to be strictly positive. Therefore `i` must be a separator. The set `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` is exactly the set of separator positions — computable by a single scan of `t`.

Given the separator positions, the field boundaries follow deterministically. The node field occupies positions `1` through `s₁ - 1` where `s₁` is the first separator (or positions `1` through `#t` when no separator exists). The user field occupies positions `s₁ + 1` through `s₂ - 1` where `s₂` is the second separator, and so on. By T4a (SyntacticEquivalence), the T4 constraints guarantee that no two separators are adjacent and that the tumbler neither begins nor ends with zero, so each field segment is non-empty. The canonical representation of T3 ensures that the components of `t` are fixed — there is no alternative encoding of the same tumbler with different component values. The separator positions are therefore uniquely determined, the field boundaries are uniquely determined, and the sub-sequences assigned to each field are uniquely determined.

Two distinct decompositions of `t` would require two distinct sets of separator positions. But there is exactly one such set. Therefore `fields(t)` is well-defined and unique. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints (at most three zero-valued components, positive-component constraint, non-empty field constraint).
- *Postconditions:* `fields(t)` — the decomposition into node, user, document, and element sub-sequences — is well-defined and uniquely determined by `t`.
