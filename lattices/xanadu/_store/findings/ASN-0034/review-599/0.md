### Failure clause covers only one of two failure modes
**Class**: REVISE
**Foundation**: T4 (HierarchicalParsing) — the Guarantee's iff implies two distinct failure regions
**ASN**: TA5a *Formal Contract*, *Failure* slot: "For `k ≥ 3`, `inc(t, k)` violates T4: by TA5(d) and the established `k − 1 ≥ 2`, the appended separator zeros at positions `#t + 1` and `#t + 2` are adjacent indices of `t'` carrying zero, so T4(ii) instantiated at `i = #t + 1` directly fails on `t'`."
**Issue**: The Guarantee `T4(t') iff k ∈ {0,1} or (k = 2 ∧ zeros(t) ≤ 2)` has *two* failure regions on the precondition domain (`t` satisfies T4, so `zeros(t) ∈ {0,1,2,3}`; `k ≥ 0`): (a) `k ≥ 3`, where T4(ii) on `t'` fails by adjacent appended zeros; (b) `k = 2 ∧ zeros(t) = 3`, where T4(i) on `t'` fails because `zeros(t') = zeros(t) + 1 = 4 > 3`. The Failure slot describes only (a). Mode (b) — established mid-proof in Case `k = 2` ("T4 preserved iff `zeros(t) + 1 ≤ 3`, i.e., `zeros(t) ≤ 2`") and structurally distinct from (a) (it violates T4(i), not T4(ii); it occurs at `k = 2`, not `k ≥ 3`; it depends on a precondition fact `zeros(t) = 3`, not on TA5(d)'s appended zeros) — has no description in the Failure slot. A downstream consumer asking "under what circumstance and via which T4 conjunct does `inc(t, 2)` break T4 on a zero-count-3 input?" cannot answer from the contract.
**What needs resolving**: Either extend the Failure slot to describe both failure modes — naming the violating T4 conjunct and the witnessing mechanism in each case (`k ≥ 3` → T4(ii) at `i = #t + 1` via TA5(d)'s adjacent zeros; `k = 2 ∧ zeros(t) = 3` → T4(i) via `zeros(t') = zeros(t) + 1 = 4`) — or restate the Failure slot's intent so its scope (one failure mode, the directly-violated adjacency mode) is explicit and the other failure region is acknowledged to live in the Guarantee's negation.

VERDICT: REVISE
