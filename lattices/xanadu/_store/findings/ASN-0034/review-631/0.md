### NAT-discrete depends-prose for NAT-order buries the strict order `<`
**Class**: OBSERVE
**Foundation**: NAT-discrete (NatDiscreteness)
**ASN**: NAT-discrete, *Depends:* "NAT-order (NatStrictTotalOrder) — supplies the non-strict companion `≤` (defined by `m ≤ n ⟺ m < n ∨ m = n`) used in the axiom's consequent `m + 1 ≤ n` and in the Consequence derivation's case split on `m ≤ n`, the exactly-one-trichotomy clause `¬(a < b ∧ b < a)` instantiated at `(m + 1, n)` in the derivation, and irreflexivity `¬(n < n)` used to discharge the rewritten `n < n`."
**Issue**: The Axiom's hypothesis `m < n` and the no-interval Consequence's second hypothesis `n < m + 1` use the strict order `<` directly — not as the derived `≤`. NAT-order's headline primitive is `< ⊆ ℕ × ℕ` (the very thing the foundation's name advertises). The depends-prose leads with "supplies the non-strict companion `≤`" without a parallel call-out that NAT-order also supplies `<` for these direct hypothesis sites. The reader who scans only the depends list learns that `≤`, the trichotomy clause, and irreflexivity flow from NAT-order, but not that the bare `<` relation does. This is the same understated-surface-area pattern flagged for T0's depends-prose in a prior cycle.

VERDICT: OBSERVE
