### ActionPoint omits NAT-carrier from declared depends
**Class**: REVISE
**Foundation**: ActionPoint
**ASN**: ActionPoint, *Depends:* lists T0, TA-Pos, NAT-wellorder, NAT-zero, NAT-order, NAT-discrete, NAT-closure — but not NAT-carrier. The Definition reads "S = {i ∈ ℕ : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0}" and the Postcondition reads "`(A i ∈ ℕ : 1 ≤ i < actionPoint(w) : wᵢ = 0)`". The Derivation also writes "the carrier `i ∈ ℕ` holds by the universal's domain restriction" and "Instantiating NAT-zero's disjunction axiom `(A n ∈ ℕ :: 0 < n ∨ 0 = n)` at n = w_{actionPoint(w)}".
**Issue**: ℕ is referenced directly in ActionPoint's own Definition and Postcondition, not just transitively via T0/NAT-zero/etc. Every other foundation in this ASN that mentions ℕ explicitly (TA-Pos, T0, NAT-zero, NAT-closure, NAT-order) lists NAT-carrier in its *Depends:* with prose anchored on the role ℕ plays in its clauses. The same standard the previous cycle applied to NAT-wellorder applies here: the symbol `ℕ` appears in ActionPoint's contract but is not credited to any declared dependency. Relying on transitive availability through T0 (whose depends-prose mentions an "index domain" subset of ℕ) is the silent-dependency pattern the previous finding rejected.
**What needs resolving**: ActionPoint's *Depends:* list must explicitly cite NAT-carrier, with prose identifying that NAT-carrier supplies `ℕ` for the comprehension `{i ∈ ℕ : 1 ≤ i ≤ #w ∧ wᵢ ≠ 0}` defining S, for the bounded universal `(A i ∈ ℕ : 1 ≤ i < actionPoint(w) : wᵢ = 0)` in the Postcondition, and as the ambient set whose elements `m, n` (the bound variables of NAT-wellorder's least-element principle as instantiated on S) inhabit.
