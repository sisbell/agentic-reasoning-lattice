### Component projection used as notation without an operator signature
**Class**: OBSERVE
**ASN**: T0 introduces length as `#· : T → ℕ`. The component projection has no analogous signature: the *Axiom* asserts only the typing `(A a ∈ T, i ∈ {j ∈ ℕ : 1 ≤ j ≤ #a} :: aᵢ ∈ ℕ)` and the extensionality and comprehension clauses use `aᵢ`, `bᵢ`, `tᵢ` as bare notation. TA-Pos and NAT-zero downstream then write `tᵢ = 0` against this notation.
**Issue**: Asymmetric presentation. The typing claim and extensionality together pin down projection's behavior at valid indices, so soundness is preserved, but a downstream consumer wanting to formalize the operator (e.g. `proj : (a ∈ T) × {j ∈ ℕ : 1 ≤ j ≤ #a} → ℕ`, or a partial-function variant) has to reconstruct the signature from the typing claim. Mirroring `#·`'s style at projection would let later ASNs cite the operator by signature rather than by notation.
