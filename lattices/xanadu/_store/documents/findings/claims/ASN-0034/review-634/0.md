### "unfolds" mischaracterises the direction of the ≤-step in strict monotonicity
**Class**: OBSERVE
**Foundation**: NAT-sub (NatPartialSubtraction)
**ASN**: Strict-monotonicity prose, in the `b < a` case: "The b < a case unfolds via NAT-order's ≤-definition to b ≤ a; NAT-addcompat's right order compatibility applied at antecedent b ≤ a then delivers b + p ≤ a + p, which the ≤-definition splits into b + p < a + p ∨ b + p = a + p."
**Issue**: "Unfolding" the ≤-definition `m ≤ n ⟺ m < n ∨ m = n` rewrites `m ≤ n` into the disjunction (the second use in the same sentence — "the ≤-definition splits..." — is exactly that direction, and is the correct usage). The first use, however, goes the *other* way: from `b < a` to `b ≤ a`. That step weakens `b < a` into the ≤-disjunction (or folds the definition right-to-left), not unfolds it. The inference is sound — `b < a ⟹ b ≤ a` is immediate from the disjunction — but the wording within one paragraph applies "unfolds" to two opposite directions, which a precise reader has to mentally reconcile.

VERDICT: OBSERVE
