### T0 depends-prose for NAT-order undercounts uses of ≤ and omits ≥
**Class**: OBSERVE
**Foundation**: T0
**ASN**: T0, *Depends:* "NAT-order (NatStrictTotalOrder) — supplies the non-strict relation `≤` on ℕ appearing in the nonemptiness clause `1 ≤ #a`."
**Issue**: T0's axioms use `≤` in many places besides the nonemptiness clause: in the index-domain comprehensions `{j ∈ ℕ : 1 ≤ j ≤ #a}` and `{j ∈ ℕ : 1 ≤ j ≤ p}`, in the extensionality clause `(A i : 1 ≤ i ≤ #a : aᵢ = bᵢ)`, and in the comprehension axiom's `(A i : 1 ≤ i ≤ p : tᵢ = r(i))`. The comprehension axiom's outer guard `p ≥ 1` additionally relies on `≥`, defined by NAT-order. The depends-prose understates the surface area on which T0 leans on NAT-order — a downstream consumer reading only the depends list would not learn that `≥` is in play.
