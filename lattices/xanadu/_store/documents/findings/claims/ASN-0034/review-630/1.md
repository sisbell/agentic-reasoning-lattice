### NAT-discrete omits NAT-carrier from declared depends
**Class**: REVISE
**Foundation**: NAT-discrete (NatDiscreteness)
**ASN**: NAT-discrete, *Depends:* lists only NAT-order and NAT-closure. The Axiom reads `(A m, n ∈ ℕ :: m < n ⟹ m + 1 ≤ n)` and the Consequence reads `(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)`.
**Issue**: NAT-discrete's Axiom and Consequence both directly reference `ℕ` as the range of the bounded universal `(A m, n ∈ ℕ :: ...)`. Every other foundation in this ASN that mentions `ℕ` explicitly (TA-Pos aside, which has the analogous omission) declares NAT-carrier as a dependency with prose anchored on the role `ℕ` plays. The same standard the previous cycle applied to NAT-wellorder and ActionPoint applies here. Relying on transitive availability through NAT-order or NAT-closure is the silent-dependency pattern the prior cycle rejected — a downstream consumer cannot read the supplier of `ℕ` off NAT-discrete's contract.
**What needs resolving**: NAT-discrete's *Depends:* list must explicitly cite NAT-carrier, with prose identifying that NAT-carrier supplies `ℕ` as the range of the bounded universal `(A m, n ∈ ℕ :: ...)` in both the Axiom and the no-interval Consequence.
