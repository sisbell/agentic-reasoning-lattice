### NAT-wellorder omits NAT-carrier from declared depends
**Class**: REVISE
**Foundation**: NAT-wellorder (NatWellOrdering)
**ASN**: NAT-wellorder, *Depends:* lists only `NAT-order (NatStrictTotalOrder) — supplies the non-strict companion ≤ ...`. The axiom is `(A S : S ⊆ ℕ ∧ S ≠ ∅ : (E m ∈ S :: (A n ∈ S :: m ≤ n)))`.
**Issue**: The axiom explicitly references `ℕ` in the carrier-side clause `S ⊆ ℕ` (and the bound variables `m, n` range over elements of S, hence over ℕ). Every other foundation in this ASN that mentions `ℕ` explicitly declares `NAT-carrier (NatCarrierSet)` as a dependency — TA-Pos, T0, NAT-zero, NAT-closure, and NAT-order all do so, with depends-prose anchored on the role `ℕ` plays in their clauses. NAT-wellorder breaks the pattern: the symbol `ℕ` appears in the axiom but is not credited to any declared dependency. Relying on transitive availability through NAT-order is a silent dependency — the kind a downstream consumer cannot read off the contract.
**What needs resolving**: NAT-wellorder's *Depends:* list must explicitly cite NAT-carrier, with prose identifying that NAT-carrier supplies `ℕ` for the carrier-side clause `S ⊆ ℕ` and as the ambient set over which the comprehension's variable `m, n` range.

VERDICT: REVISE
