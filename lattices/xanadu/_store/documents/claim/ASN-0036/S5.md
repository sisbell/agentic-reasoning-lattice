**S5 (Unrestricted sharing).** The same I-address may appear in the ranges of multiple arrangements, and at multiple V-positions within a single arrangement. S0вҖ“S4 and S6 are consistent with any finite sharing multiplicity вҖ” they place no constraint on `|{(d, v) : v вҲҲ dom(ОЈ.M(d)) вҲ§ ОЈ.M(d)(v) = a}|`. Because S0 and S1 quantify over state transitions (S6 is a named restatement of S1 вҖ” see S6), we state this for states reachable through conforming execution traces вҖ” finite sequences `Пғ = вҹЁОЈвӮҖ, ОЈвӮҒ, вҖҰ, ОЈвӮ–вҹ©` with `k вүҘ 1`, beginning at the initial state (AX-1), in which every transition `ОЈбөў вҶ’ ОЈбөўвӮҠвӮҒ` satisfies S0 and S1, every state `ОЈбөў` satisfies S2 and S3, S4 holds for all allocation events across the trace, and AX-2 (allocation-mediated entry) holds:

`(A N вҲҲ в„• :: (E Пғ = вҹЁОЈвӮҖ, вҖҰ, ОЈвӮ–вҹ© :: k вүҘ 1 вҲ§ Пғ conforming вҲ§ (E a вҲҲ dom(ОЈвӮ–.C) :: |{(d, v) : v вҲҲ dom(ОЈвӮ–.M(d)) вҲ§ ОЈвӮ–.M(d)(v) = a}| > N)))`

To see this, fix any `N`. We construct an execution trace of `N + 2` transitions beginning at `ОЈвӮҖ`: first a content-creation transition that introduces an I-address `a`, then `N + 1` arrangement-extension transitions that each place `a` in a new document. The resulting state has sharing multiplicity `N + 1 > N`. At each of the `N + 1` arrangement transitions, `a вҲҲ dom(ОЈбөў.C)` and the content store is unchanged, so S0 and S1 hold non-vacuously вҖ” content immutability and store monotonicity are substantively exercised, not merely vacuously true for want of transitions.

In any particular state, the sharing multiplicity of each address is a definite finite number вҖ” possibly zero for orphaned content (S6). The property is an architectural anti-constraint: the invariants place no finite cap on how many references may accumulate.

Nelson: "The virtual byte stream of a document may include bytes from any other document." And: "A document may have a window to another document, and that one to yet another, indefinitely. Thus A contains part of B, and so on. One document can be built upon another, and yet another document can be built upon that one, indefinitely." Transclusion is recursive and unlimited вҖ” a claim not merely about the existence of high-sharing states but about their reachability through the system's operations.

Gregory confirms the unbounded nature at the implementation level. The global index that records which documents reference which I-addresses accumulates entries without cap вҖ” "no counter, cap, MAX_TRANSCLUSIONS constant, or any other limiting mechanism anywhere in the code path." Each referential inclusion adds one entry. The only constraints are physical resources (memory and disk), not architectural limits.

The combination of S4 and S5 gives the system its distinctive character. S4 says identity is structural вҖ” determined by I-address, not by value. S5 says sharing is unlimited вҖ” any number of documents can reference the same content. Together they establish a regime in which quotation is a first-class structural relationship: any number of documents can quote the same passage, and the system knows they are all quoting вҖ” not independently writing вҖ” because they share I-addresses.

We observe that the state `ОЈ = (C, M)` makes the sharing relation computable: given any `a вҲҲ dom(C)`, the set `{d : (E v :: M(d)(v) = a)}` is determined by the state. Nelson requires this to be queryable: "It must also be possible for the reader to ask to see whatever documents window to the current document. Both are available at any time." The state model supports this вҖ” the information is present; only the efficiency of its extraction is an implementation concern.

*Proof.* We wish to show that for every `N вҲҲ в„•`, there exists a conforming execution trace whose final state contains an I-address with sharing multiplicity exceeding `N`. We give two constructions вҖ” one for cross-document sharing, one for within-document sharing вҖ” each succeeding for arbitrary `N`.

**Cross-document construction.** Fix `N вҲҲ в„•`. We construct an execution trace `Пғ_N = вҹЁОЈвӮҖ, ОЈвӮҒ, ОЈвӮӮ, вҖҰ, ОЈ_{N+2}вҹ©` comprising `N + 2` transitions:

- `ОЈвӮҖ`: The initial state. By AX-1, `dom(ОЈвӮҖ.M(d)) = вҲ…` for all documents `d`, and `dom(ОЈвӮҖ.C) = вҲ…`.
- `ОЈвӮҖ вҶ’ ОЈвӮҒ` (content creation): Perform a T10a-conforming allocation event producing a fresh I-address `a` вҖ” by AX-2 (allocation-mediated entry), every transition that adds an address to `dom(C)` is a T10a-conforming allocation event вҖ” and store `ОЈвӮҒ.C = {a вҶҰ w}` for some value `w вҲҲ Val`. Arrangements are unchanged: `ОЈвӮҒ.M(d) = ОЈвӮҖ.M(d)` for all `d`.
- `ОЈбөў вҶ’ ОЈбөўвӮҠвӮҒ` for `i = 1, вҖҰ, N + 1` (arrangement extension): Introduce a V-mapping `ОЈбөўвӮҠвӮҒ.M(dбөў)(vбөў) = a` for a document `dбөў` at a fresh V-position `vбөў`, with the `dбөў` pairwise distinct вҖ” such elements exist by AX-3, which guarantees `|Doc| = в„өвӮҖ`. Content is unchanged: `ОЈбөўвӮҠвӮҒ.C = ОЈбөў.C`. All other arrangements are unchanged.

The final state `ОЈ_{N+2}` has `C = {a вҶҰ w}` and `N + 1` documents `dвӮҒ, вҖҰ, d_{N+1}`, each with `M(dбөў) = {vбөў вҶҰ a}`.

We verify each property across the entire trace.

*S0 (content immutability).* At `ОЈвӮҖ вҶ’ ОЈвӮҒ`: `dom(ОЈвӮҖ.C) = вҲ…`, so the universal quantification `(A a вҲҲ dom(ОЈвӮҖ.C) : вҖҰ)` holds vacuously вҖ” no content yet exists to protect, and this is necessarily so for the trace's initial content-creation event. At each subsequent transition `ОЈбөў вҶ’ ОЈбөўвӮҠвӮҒ` for `i = 1, вҖҰ, N + 1`: the content store is unchanged, `ОЈбөўвӮҠвӮҒ.C = ОЈбөў.C = {a вҶҰ w}`, so `a вҲҲ dom(ОЈбөў.C)` and `ОЈбөўвӮҠвӮҒ.C(a) = w = ОЈбөў.C(a)`. S0 holds non-vacuously at these `N + 1` transitions: the existing address `a` is verified to persist with its value intact.

*S1 (store monotonicity).* At `ОЈвӮҖ вҶ’ ОЈвӮҒ`: `dom(ОЈвӮҖ.C) = вҲ… вҠҶ {a} = dom(ОЈвӮҒ.C)`. At each `ОЈбөў вҶ’ ОЈбөўвӮҠвӮҒ` for `i = 1, вҖҰ, N + 1`: `dom(ОЈбөў.C) = {a} = dom(ОЈбөўвӮҠвӮҒ.C)`, so `{a} вҠҶ {a}`. In particular, `a вҲҲ dom(ОЈбөў.C)` witnesses the non-vacuity: the domain has an element whose membership persists.

*S2 (arrangement functionality).* In `ОЈвӮҖ` and `ОЈвӮҒ`, all arrangements are empty вҖ” trivially functions. In each `ОЈбөў` for `i вүҘ 2`, each newly introduced `M(dвұј) = {vвұј вҶҰ a}` contains a single entry вҖ” a function with domain `{vвұј}`. All other arrangements remain empty.

*S3 (referential integrity).* In `ОЈвӮҖ` and `ОЈвӮҒ`, no V-mappings exist вҖ” holds vacuously. In each `ОЈбөў` for `i вүҘ 2`, every V-mapping targets `a`, and `a вҲҲ dom(ОЈбөў.C)` because `a` was created at `ОЈвӮҒ` and S0 guarantees its persistence in every subsequent state.

*S4 (origin-based identity).* The trace contains a single allocation event (at `ОЈвӮҖ вҶ’ ОЈвӮҒ`), producing `a`. No pair of distinct allocation events exists, so `(A aвӮҒ, aвӮӮ from distinct events :: aвӮҒ вү  aвӮӮ)` holds vacuously.

*S6 (persistence independence).* Formally identical to S1; holds by S1's verification above.

The sharing multiplicity of `a` in `ОЈ_{N+2}` is `|{(d, v) : v вҲҲ dom(ОЈ_{N+2}.M(d)) вҲ§ ОЈ_{N+2}.M(d)(v) = a}| = N + 1`, since each of the `N + 1` documents contributes exactly one pair `(dбөў, vбөў)`. Thus the multiplicity exceeds `N`.

**Within-document construction.** Fix `N вҲҲ в„•`. We construct an execution trace `Пғ'_N = вҹЁОЈвӮҖ, ОЈвӮҒ, ОЈвӮӮ, вҖҰ, ОЈ_{N+2}вҹ©`:

- `ОЈвӮҖ`: The initial state (AX-1). `dom(ОЈвӮҖ.M(d)) = вҲ…` for all `d`, and `dom(ОЈвӮҖ.C) = вҲ…`.
- `ОЈвӮҖ вҶ’ ОЈвӮҒ` (content creation): Perform a T10a-conforming allocation event producing a fresh I-address `a` (AX-2) and store `ОЈвӮҒ.C = {a вҶҰ w}` for some `w вҲҲ Val`. Arrangements unchanged.
- `ОЈбөў вҶ’ ОЈбөўвӮҠвӮҒ` for `i = 1, вҖҰ, N + 1` (arrangement extension): Extend the arrangement of a single document `d` by adding `vбөў вҶҰ a` for a fresh V-position `vбөў`, so that `ОЈбөўвӮҠвӮҒ.M(d) = ОЈбөў.M(d) вҲӘ {vбөў вҶҰ a}` with the `vбөў` pairwise distinct вҖ” such elements exist by AX-3, which guarantees `|VPos(S)| = в„өвӮҖ` for the subspace `S` of `d`. Content is unchanged: `ОЈбөўвӮҠвӮҒ.C = ОЈбөў.C`.

The final state `ОЈ_{N+2}` has one document `d` with `M(d) = {vвӮҒ вҶҰ a, vвӮӮ вҶҰ a, вҖҰ, v_{N+1} вҶҰ a}`.

We verify each property. *S0 (content immutability):* at `ОЈвӮҖ вҶ’ ОЈвӮҒ`, `dom(ОЈвӮҖ.C) = вҲ…` вҖ” vacuous; at each `ОЈбөў вҶ’ ОЈбөўвӮҠвӮҒ` for `i = 1, вҖҰ, N + 1`, `ОЈбөўвӮҠвӮҒ.C = ОЈбөў.C = {a вҶҰ w}`, so `a вҲҲ dom(ОЈбөў.C)` and `ОЈбөўвӮҠвӮҒ.C(a) = w = ОЈбөў.C(a)` вҖ” non-vacuous at `N + 1` transitions. *S1 (store monotonicity):* at `ОЈвӮҖ вҶ’ ОЈвӮҒ`, `вҲ… вҠҶ {a}`; at each `ОЈбөў вҶ’ ОЈбөўвӮҠвӮҒ` for `i вүҘ 1`, `{a} вҠҶ {a}` with `a` witnessing the non-vacuity. *S2 (arrangement functionality):* in each state after `ОЈвӮҒ`, the V-positions `vвӮҒ, вҖҰ, vвұј` accumulated in `M(d)` are pairwise distinct by hypothesis, so each maps to exactly one I-address; `M(d)` is a well-defined function at every stage. *S3 (referential integrity):* in `ОЈвӮҖ` and `ОЈвӮҒ`, no V-mappings exist вҖ” vacuous; in each `ОЈбөў` for `i вүҘ 2`, every V-mapping targets `a`, and `a вҲҲ dom(ОЈбөў.C)` because `a` was created at `ОЈвӮҒ` and preserved by S0. *S4 (origin-based identity):* a single allocation event produces `a`; no pair of distinct events exists вҖ” vacuous. *S6 (persistence independence):* formally identical to S1; holds by S1's verification above.

The within-document sharing multiplicity is `|{(d, v) : v вҲҲ dom(ОЈ_{N+2}.M(d)) вҲ§ ОЈ_{N+2}.M(d)(v) = a}| = N + 1`, since the single document `d` contributes `N + 1` pairs `(d, vвӮҒ), вҖҰ, (d, v_{N+1})`. Thus the multiplicity exceeds `N`.

**Conclusion.** Since both constructions produce conforming execution traces for arbitrary `N вҲҲ в„•`, the conjunction S0 вҲ§ S1 вҲ§ S2 вҲ§ S3 вҲ§ S4 вҲ§ S6 вҲ§ AX-2 is consistent with sharing multiplicity exceeding any given finite bound вҖ” not merely in isolated states but in states reachable through sequences of transitions that substantively enforce content immutability and store monotonicity. No finite cap on `|{(d, v) : v вҲҲ dom(ОЈ.M(d)) вҲ§ ОЈ.M(d)(v) = a}|` is entailed by these properties вҖ” neither across documents nor within a single document. The remaining properties of the ASN (S7 onwards вҖ” document-level allocation, ordered arrangement structure, operations) are not checked: the constructions use minimal traces that make no claims about V-position ordering or operational behaviour, and extending the consistency result to those properties would require constructions of substantially different character. вҲҺ
