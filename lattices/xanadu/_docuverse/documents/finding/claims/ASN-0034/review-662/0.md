### T4a interior segment — implicit associativity of addition
**Class**: REVISE
**Foundation**: NAT-closure posits `+ : ℕ × ℕ → ℕ` with identity axioms only; no associativity axiom appears in any NAT-* ASN
**ASN**: T4a (SyntacticEquivalence), Reverse proof, Interior segments paragraph — "since the zero positions are strictly increasing we already have `s_{i+1} ≥ s_i + 1`; together these give `s_{i+1} ≥ s_i + 2`, so the segment has at least one index"
**Issue**: NAT-discrete applied at `(s_i + 1, s_{i+1})` delivers `(s_i + 1) + 1 ≤ s_{i+1}`. The text rewrites this as `s_i + 2 ≤ s_{i+1}`, where `2 := 1 + 1` by definition, so `s_i + 2 = s_i + (1 + 1)`. The rewrite requires `(s_i + 1) + 1 = s_i + (1 + 1)`, i.e., associativity of `+` at `(s_i, 1, 1)`. Associativity is not in the axiom inventory of any NAT-* ASN. The conclusion "at least one index" is provable (from `s_i + 1 < s_{i+1}` via NAT-sub strict monotonicity at `p = 1` followed by right-telescoping and NAT-discrete), but the derivation as written contains an unjustified equational step.
**What needs resolving**: Either add an associativity axiom to an appropriate NAT-* ASN and cite it, or restate the interior segment argument in terms of `(s_i + 1) + 1` throughout, replacing the `s_i + 2` notation with a derivation path that applies only right-telescoping and NAT-discrete without conflating `s_i + (1 + 1)` with `(s_i + 1) + 1`.
