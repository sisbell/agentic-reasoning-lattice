### T0 inner index quantifiers omit domain declaration
**Class**: REVISE
**Foundation**: T0 (CarrierSetDefinition) — *Formal Contract* Axiom bullets
**ASN**: T0 extensionality axiom: `(A a, b ∈ T : #a = #b ∧ (A i : 1 ≤ i ≤ #a : aᵢ = bᵢ) : a = b)`; and comprehension axiom inner clause: `(A i : 1 ≤ i ≤ p : tᵢ = r(i))`
**Issue**: Both axioms quantify over an index variable `i` with range condition `1 ≤ i ≤ ...` but state no domain for `i`. The `≤` relation is defined on `ℕ × ℕ` by NAT-order; without declaring `i ∈ ℕ`, the domain of the inner universal is formally unspecified. In an untyped setting this makes the axioms weaker than intended — the condition `1 ≤ i` is false for non-ℕ values of `i`, so the inner quantifier is vacuously satisfied by anything outside ℕ. TA-Pos, the only other claim that uses the same bounded index quantifier pattern, writes `(E i ∈ ℕ : 1 ≤ i ≤ #t : ...)` and `(A i ∈ ℕ : 1 ≤ i ≤ #t : ...)` — explicitly declaring `i ∈ ℕ` both times. The inconsistency is not a style choice: every outer quantifier in every formal contract in this ASN states its domain, and a downstream formal verifier (Dafny) requires explicit type/domain declarations. The two inner quantifiers in T0 are the sole exceptions.
**What needs resolving**: The extensionality axiom's inner universal must declare `i ∈ ℕ` (matching NAT-carrier's role as the carrier from which the index variable is drawn, as already annotated for TA-Pos's analogous quantifiers). The comprehension axiom's inner universal must do the same. Neither axiom changes meaning for a reader who already knows `≤` is defined on ℕ, but the formal contract must not rely on that background inference — the domain must be stated explicitly in the axiom text.
