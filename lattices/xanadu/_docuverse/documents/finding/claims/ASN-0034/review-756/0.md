### T3 headline claim uses `:` where `::` is needed for the biconditional body
**Class**: OBSERVE
**Foundation**: (none — notation issue internal to this foundation ASN)
**ASN**: T3 (CanonicalRepresentation) — opening claim `(A a, b ∈ T : #a = #b ∧ (A i : 1 ≤ i ≤ #a : aᵢ = bᵢ) ≡ a = b)`
**Issue**: Throughout the ASN, `(A vars : range : body)` places the range between the two colons and the body after the second colon (e.g., T0's extensionality `(A a, b ∈ T : #a = #b ∧ ... : a = b)`), while `(A vars :: body)` marks the no-range form. T3's headline uses a single `:` where `::` is needed: as written, `#a = #b ∧ (A i : 1 ≤ i ≤ #a : aᵢ = bᵢ) ≡ a = b` is parsed as the range with an empty body, giving a vacuously true claim. The intended reading — a biconditional body with no range restriction — requires `::`. The Formal Contract postcondition states the biconditional correctly as `a = b ⟺ #a = #b ∧ (A i : 1 ≤ i ≤ #a : aᵢ = bᵢ)`, so the mathematical content is sound; the ambiguity is confined to the headline.
