### T5 formal contract missing T0 as a direct dependency
**Class**: REVISE
**Foundation**: T0 (CarrierSetDefinition) — length operator `#· : T → ℕ`, component projection `i ↦ aᵢ`
**ASN**: T5 (ContiguousSubtrees), Formal Contract *Depends* section
**Issue**: The formal contract lists Prefix, T1, T3, NAT-order, NAT-addcompat, and NAT-wellorder — but not T0. The proof directly invokes T0's operators on tumbler `b`, which is not constrained by any prefix relation and whose length and components are not supplied by any listed dependency. Concretely: Case 1 constructs the set `{k ∈ {1,...,#p} : bₖ ≠ pₖ}`, which requires T0's component projection to be defined on `b` for indices up to `#p`; Case 2 directly compares `#b < #p`, requiring T0's length operator on `b`. Prefix unfolds `p ≼ a` and `p ≼ c` — it does not supply `#b` or `bₖ`. T3 speaks to equality from component agreement — it does not supply the projection operator itself. T0 is the sole source of these operators for arbitrary `b ∈ T`.
**What needs resolving**: T0 (CarrierSetDefinition) must appear in T5's *Depends* section, with an annotation identifying its role: supplying `#b` for the Case 1/Case 2 length comparisons and the component projection `bₖ` used to construct the divergence-index set in Case 1.
