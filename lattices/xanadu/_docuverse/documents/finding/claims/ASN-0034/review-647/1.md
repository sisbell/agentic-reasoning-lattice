### Hierarchical-structure preamble forward-references T4c with a per-k label inventory
**Class**: REVISE
**Foundation**: T4 (HierarchicalParsing) — *Hierarchical structure* preamble, final paragraph
**ASN**:
> "The count of zero-valued components determines the specificity level — for each `k ∈ {0, 1, 2, 3}` at which a T4-valid tumbler has `zeros(t) = k`, T4c is the single definitional site, assigning the label node address (at `k = 0`), user address (at `k = 1`), document address (at `k = 2`), or element address (at `k = 3`). The names appear here only as a forward reference to that definition; they are not defined by the present paragraph."

**Issue**: This is the forward-reference-with-inventory pattern flagged in the prior cycle, now relocated from the end of T4's body to the preamble before T4. The paragraph enumerates what T4c does at each `k`, walking the four cases — content that belongs in T4c. The closing self-disclaimer ("not defined by the present paragraph") is itself defensive prose, the same self-cancellation pattern flagged previously ("the present paragraph is a forward reference, not a restatement"). T4's correctness does not depend on T4c's labelling; reproducing T4c's per-k assignments here is content relocated rather than removed.
**What needs resolving**: Remove the per-k label inventory and the self-disclaimer. If the preamble needs to flag that level labels exist downstream, one declarative sentence without the four-case enumeration suffices.
