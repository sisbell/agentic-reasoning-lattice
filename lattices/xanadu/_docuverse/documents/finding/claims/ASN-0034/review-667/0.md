### Redundant axiom in NAT-closure
**Class**: OBSERVE
**Foundation**: NAT-closure (NatArithmeticClosureAndIdentity)
**ASN**: Formal Contract, Axiom list — `0 < 1` (distinctness of the two named constants)
**Issue**: The prose explicitly notes that `0 < 1` is recoverable as the case `n := 0` of the universal `(A n ∈ ℕ :: 0 < n + 1)` together with the left-identity rewrite `0 + 1 = 1`. The axiom is therefore redundant given the other axioms already in the formal contract. NAT-card's depends entry cites `0 < 1` from NAT-closure, so consumers of the axiom exist — but the derivability means the axiom is not load-bearing as a separate postulate.
