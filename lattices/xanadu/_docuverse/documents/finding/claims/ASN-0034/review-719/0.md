### Irreflexivity citation in NAT-discrete Consequence annotation creates a naming collision
**Class**: OBSERVE
**Foundation**: NAT-order (NatStrictTotalOrder) — irreflexivity `(A n ∈ ℕ :: ¬(n < n))`
**ASN**: NAT-discrete (NatDiscreteness) — Formal Contract *Consequence:* annotation: `"irreflexivity ¬(n < n) at n := m + 1 after rewriting"`
**Issue**: The outer consequence quantifies over a bound variable also named `n` (in `(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)`). The phrase "at `n := m + 1`" reads as substituting the outer `n` with `m + 1`, when the intent is to instantiate the irreflexivity schema's universally-quantified variable at the element whose value is simultaneously `n` (outer) and `m + 1` (after rewriting under `m + 1 = n`). The annotation is technically correct — since the outer `n` equals `m + 1` at the point of application — but the shared name obscures which `n` is being instantiated. A reader tracing the annotation against the irreflexivity schema must manually resolve the collision.
