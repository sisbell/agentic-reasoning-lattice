### Redundant axiom in NAT-closure
**Class**: OBSERVE
**Foundation**: NAT-closure (NatArithmeticClosureAndIdentity)
**ASN**: Formal Contract, Axiom bullet — "`0 < 1` (distinctness of the two named constants)"
**Issue**: The text explicitly acknowledges this clause is recoverable as "the case `n := 0` together with the left-identity rewrite `0 + 1 = 1`" — applying successor positivity `0 < n + 1` at `n := 0` gives `0 < 0 + 1`, and left-identity gives `0 + 1 = 1`, so `0 < 1`. The axiom is therefore a consequence of the other listed axioms, not an independent primitive. The documentation is honest about this, but listing a derivable statement as an axiom mixes two layers of the contract.
