### Component projection symbol declared by property, not by signature
**Class**: REVISE
**Foundation**: T0 (CarrierSetDefinition)
**ASN**: T0 Axiom — component projection clause: `(A a ∈ T, i ∈ {j ∈ ℕ : 1 ≤ j ≤ #a} :: aᵢ ∈ ℕ)`
**Issue**: The Axiom slot declares `T` as a set and `#· : T → ℕ` as a typed operator. The component-projection clause asserts the *property* `aᵢ ∈ ℕ` for valid (a, i), but never introduces the projection symbol with a signature. The notation `aᵢ` is established only in informal prose ("write `aᵢ` for the value of the `i`-th component"). Downstream uses treat `aᵢ` as a function-valued term: T0's own extensionality writes `aᵢ = bᵢ`, the comprehension writes `tᵢ = r(i)`, and TA-Pos's defining bodies write `tᵢ = 0`. Each of these requires `aᵢ` to be a deterministic, totally-defined-on-its-stated-domain function-valued expression — a commitment the Axiom slot does not actually make. A precise reader sees the asymmetry: `#·` has both signature and property; the projection has only property.
**What needs resolving**: T0's Axiom slot must introduce the component projection as a typed operator with explicit domain (a partial-on-T or dependently-typed signature whose definedness range is the index domain `{j ∈ ℕ : 1 ≤ j ≤ #a}`) before — or alongside — the property clause `aᵢ ∈ ℕ`. The depends-on-`≤`/`1 ∈ ℕ` for the index domain should follow the same convention as `#·`.

VERDICT: REVISE
