### T4c routes `0 < 1` through NAT-addcompat without citing the rewrite step
**Class**: REVISE
**Foundation**: NAT-addcompat's strict successor inequality `(A n ∈ ℕ :: n < n + 1)`; NAT-closure's `0 < 1` axiom and left additive identity `(A n ∈ ℕ :: 0 + n = n)`.
**ASN**: T4c (LevelDetermination), Injectivity step:
> "NAT-addcompat's strict successor inequality `n < n + 1`, instantiated at `n ∈ {0, 1, 2}` … gives `0 < 1`, `1 < 2`, and `2 < 3`"

**Issue**: At `n = 0`, NAT-addcompat yields `0 < 0 + 1`, not `0 < 1`. The reduction `0 + 1 = 1` requires NAT-closure's left additive identity at `n := 1` — a step the prose elides and the depends slot does not name (the NAT-closure depends entry mentions only the numerals `2 := 1 + 1` and `3 := 2 + 1`, which are definitional rewrites, not the left-identity rewrite). For `n = 1` and `n = 2`, the rewrites `1 + 1 = 2` and `2 + 1 = 3` are by numeral definition and need no extra cite, so the gap is specific to the `n = 0` link. T4 itself, in the parallel m=0 step of its Exhaustion derivation, explicitly cites left-identity to collapse `0 + 1 ≤ zeros(t)` to `1 ≤ zeros(t)`; T4c does not match that discipline.
**What needs resolving**: Ground the `0 < 1` link of the chain explicitly. Either cite NAT-closure's left additive identity (consistent with T4's own m=0 handling) and add it to the depends slot, or — simpler — drop the NAT-addcompat detour at `n = 0` and cite NAT-closure's axiom `0 < 1` directly, restricting the NAT-addcompat instantiation to `n ∈ {1, 2}`.

VERDICT: REVISE
