### T1 proof — `a < b ≤ c → a < c` used without case split at four sites
**Class**: REVISE
**Foundation**: NAT-order (NatStrictTotalOrder) — ≤-definition `m ≤ n ⟺ m < n ∨ m = n`; `<`-transitivity `m < n ∧ n < p ⟹ m < p`
**ASN**: T1 (LexicographicOrder) — trichotomy proof Case 3 and transitivity proof Case k₁ < k₂
**Issue**: The step from `a < a+1` and `a+1 ≤ b` to the strict conclusion `a < b` appears four times, each citing only the source of the strict premise:

- Trichotomy Case 3: "(β) gives m + 1 ≤ n, hence m < n via NAT-addcompat's m < m + 1"
- Trichotomy Case 3, m < n sub-case: "n + 1 ≤ m, which with NAT-addcompat's n < n + 1 gives n < m"
- Trichotomy Case 3, n < m sub-case: "m + 1 ≤ n, which with NAT-addcompat's m < m + 1 gives m < n"
- Transitivity Case k₁ < k₂: "Combined with k₁ < k₂, NAT-order's transitivity yields k₁ < p"

NAT-addcompat provides `m < m+1`; NAT-order's `<`-transitivity covers the `<`+`<` case. Neither delivers a strict conclusion from `<`+`≤`. Doing so requires expanding the `≤` premise via NAT-order's ≤-definition into `b < c ∨ b = c` and walking both cases: `<`-transitivity for the strict disjunct, Leibniz substitution for the equality disjunct. This case split is absent at all four sites.

The sub-case (ii, ii) of transitivity partially acknowledges the gap — "via NAT-order transitivity or substitution" — but the other three sites carry no such acknowledgment.
**What needs resolving**: Each site must expand the `≤` premise via NAT-order's ≤-definition and discharge both resulting cases — the strict case by `<`-transitivity, the equality case by substitution — before asserting the strict conclusion.
