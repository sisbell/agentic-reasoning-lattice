### T4a Reverse interior segment — precondition `s_i < #t` not established before invoking condition (i) at `j := s_i`
**Class**: REVISE
**Foundation**: T4 (HierarchicalParsing) — condition (i) `(A j : 1 ≤ j < #t : ¬(t_j = 0 ∧ t_{j+1} = 0))`
**ASN**: T4a (SyntacticEquivalence), Reverse direction, "Interior segments (`1 ≤ i < k`)" paragraph — "Condition (i) forbids `s_{i+1} = s_i + 1`"
**Issue**: Instantiating condition (i) at `j := s_i` requires `1 ≤ s_i < #t`. The lower bound `1 ≤ s_i` follows from `s_i` being an enumerated index in `{1, …, #t}`. The upper bound `s_i < #t` is not derived. At the point of invocation the proof has established `s_i + 1 ≤ s_{i+1}` (from NAT-discrete on `s_i < s_{i+1}`) and, from the enumeration domain, `s_{i+1} ≤ #t`. Combining: `s_i + 1 ≤ s_{i+1} ≤ #t`, giving `s_i + 1 ≤ #t` by NAT-order's ≤-transitivity, and hence `s_i < #t` by NAT-order's unfolding of `≤`. This two-step derivation is skipped. Without it, the condition-(i) instantiation has an ungrounded precondition.
**What needs resolving**: Before invoking condition (i) to exclude `s_{i+1} = s_i + 1`, the proof must establish `s_i < #t`. The derivation: `s_i + 1 ≤ s_{i+1}` (already present) and `s_{i+1} ≤ #t` (from the enumeration domain `{s_1, …, s_k} ⊆ {1, …, #t}`) yield `s_i + 1 ≤ #t` by NAT-order's ≤-transitivity, and NAT-order's unfolding then gives `s_i < #t`.
