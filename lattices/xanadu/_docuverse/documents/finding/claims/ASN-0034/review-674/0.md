### No-interval derivation mixes ⊥ with n = m
**Class**: OBSERVE
**Foundation**: NAT-discrete — no-interval consequence derivation
**ASN**: Prose derivation, closing sentence: *"Both branches of the split terminate in `n = m`, closing the case analysis."* (preceded by *"each of which we walk to `n = m`"*)
**Issue**: The m < n branch does not terminate in n = m — it terminates in ⊥ via two sub-case contradictions, then is discharged as impossible. Only the m = n branch terminates in n = m directly. The sentence conflates "discharged by contradiction" with "concludes n = m." While ⊥ → n = m is vacuously valid, stating that a branch "terminates in" a conclusion normally means the conclusion is reached directly, not vacuously. A precise reader following the derivation will find the final summary statement misleading relative to the actual two-step argument immediately above it.
