### GlobalUniqueness exhaustiveness routes through U(d) at the wrong depth
**Class**: REVISE
**Foundation**: (foundation ASN — internal consistency)
**ASN**: GlobalUniqueness, Exhaustiveness paragraph: *"Both non-root, `p₁ = p₂`: let `P₁`, `P₂` be parents at depth ≤ *d*. If `P₁ ≠ P₂`, the two spawning events are distinct yet yield the same output `p₁`, contradicting `U(d)`."*
**Issue**: The producing-allocator taxonomy (stated earlier in the same proof) assigns the output of `inc(t, k')` with `k' > 0` to the *newly created child*. So the spawning events that produce `p₁ = c₀(A₁)` and `p₂ = c₀(A₂)` have producing allocators `A₁` and `A₂` at depth `d + 1`, not `≤ d`. `U(d)`, defined as constraining "every pair of distinct allocation events whose producing allocators both have depth ≤ *d*", does not range over these events. The cite "contradicting `U(d)`" therefore does not refute the `P₁ ≠ P₂` configuration — what it actually wants is `U(d + 1)`, which is what's being established. The underlying claim (distinct depth-`(d-1)+` parents cannot spawn the same child base) is sound but needs a different argument: e.g. via T10a.5 or T10a.6 on `(P₁, P₂)` (non-ancestor-descendant at the same depth ⟹ `dom(P₁) ∩ dom(P₂) = ∅` and prefix-incomparable), then lifting to `c₀(A₁) ≠ c₀(A₂)` via TA5(b) agreement on the parent's siblings.
**What needs resolving**: Replace the appeal to `U(d)` in the `P₁ ≠ P₂` branch with an argument that uses depth-`≤d` consequences whose producing allocators are actually at depth `≤ d` — namely T10a.5/T10a.6 on the parent pair, or a recasting of `U(d)` that records the (parent, spawnPt, spawnParam) characterisation of spawning events at parent depth. Either path closes the gap; the current one-sentence cite does not.

VERDICT: REVISE
