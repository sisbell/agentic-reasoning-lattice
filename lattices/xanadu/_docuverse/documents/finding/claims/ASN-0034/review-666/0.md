### T4-valid predicate: `tᵢ₊₁` access ungrounded for `i ≥ 1`
**Class**: REVISE
**Foundation**: NAT-closure (`+`, `1`, identity laws), NAT-order (`<`, `≤`), NAT-zero (`0 < n ∨ 0 = n`), NAT-discrete (`m < n ⟹ m + 1 ≤ n`)
**ASN**: T4 (HierarchicalParsing), T4-valid predicate: `"(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))"`
**Issue**: The component access `tᵢ₊₁` requires `i + 1 ∈ {j ∈ ℕ : 1 ≤ j ≤ #t}` — both `i + 1 ≤ #t` and `1 ≤ i + 1`. The upper bound follows from NAT-discrete at `(i, #t)`. The lower bound `1 ≤ i + 1` (equivalently `0 < i + 1`) is not grounded for `i ≥ 1`. NAT-zero applied to `i + 1` yields only `0 < i + 1 ∨ 0 = i + 1`; nothing in the axiom system excludes `0 = i + 1` for `i ≥ 1`. The left-identity establishes the `i = 0` case (`0 + 1 = 1` and `0 < 1`), but the quantifier restricts to `i ≥ 1`, making that case irrelevant. A non-standard model witnessing the gap: ℕ = {0, 1, 2, 3, …} with the standard strict total order but with `m + 1 = 0` for some `m ≥ 1`. This satisfies all stated axioms — NAT-discrete holds because `0 ≤ n` for every `n` (NAT-zero), so `m + 1 = 0 ≤ n` trivially when `m < n`. Yet for a tumbler with `#t > m`, the index `i = m` is in the quantifier range, `i + 1 = 0 ∉ {j ∈ ℕ : 1 ≤ j ≤ #t}`, and `tᵢ₊₁` is undefined.
**What needs resolving**: The axiom system must assert `(A n ∈ ℕ :: 0 < n + 1)` — the Peano no-predecessor-of-zero clause for the addition-based successor. Without it, `1 ≤ i + 1` for `i ≥ 1` cannot be discharged within the stated axioms, and the T4-valid predicate's component access is formally ungrounded.
