### TA3-strict Depends missing NAT-addcompat; used to rule out T1(ii) under `#a = #b`
**Class**: REVISE
**Foundation**: NAT-addcompat (NatAdditionOrderAndSuccessor) — strict successor inequality `n < n + 1`
**ASN**: TA3-strict (OrderPreservationUnderSubtractionStrict), proof section "The form of `a < b`": "Since `#a = #b`, T1 case (ii) (which requires `#a < #b`) is impossible."
**Issue**: The parenthetical "(which requires `#a < #b`)" is a formal derivation step: T1(ii)'s condition k = #a + 1 ≤ #b, combined with #b = #a, yields #a + 1 ≤ #a. Ruling this out requires NAT-addcompat's `n < n + 1` at n := #a — from `#a < #a + 1` and `#a + 1 ≤ #a`, NAT-order's transitivity gives `#a < #a`, contradicting irreflexivity. NAT-addcompat is absent from TA3-strict's Depends. This is the same reasoning pattern used in T1's own irreflexivity and trichotomy proofs, both of which cite NAT-addcompat for exactly this role.
**What needs resolving**: NAT-addcompat must be added to TA3-strict's Depends with an annotation identifying its use in ruling out T1(ii) when `#a = #b`.
