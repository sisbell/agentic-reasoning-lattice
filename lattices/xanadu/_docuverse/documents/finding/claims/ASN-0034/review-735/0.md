### ZPD Relationship-to-Divergence postcondition requires undeclared discrete successor property
**Class**: REVISE
**Foundation**: NAT-closure (NatArithmeticClosureAndIdentity), NAT-order (NatStrictTotalOrder)
**ASN**: ZPD (ZeroPaddedDivergence) — Postconditions (Relationship to Divergence): "if the longer operand has a nonzero component beyond the shorter's last position, `zpd(a, w)` is defined and `zpd(a, w) ≥ divergence(a, w)`"
**Issue**: In Divergence sub-case (β), `divergence(a, w) = #a + 1` and `zpd(a, w) = k` for the minimum `k` satisfying `#a < k ≤ #w ∧ wₖ ≠ 0`. The postcondition asserts `k ≥ #a + 1`, i.e., `#a + 1 ≤ k`. The only fact about `k` established from the construction is `#a < k`. Getting from `#a < k` to `#a + 1 ≤ k` requires that no element of ℕ lies strictly between `#a` and `#a + 1` — the discrete successor property. This property is not stated as an axiom and is not derivable from the stated NAT-* axioms. Concretely: a model satisfying every stated axiom can place an element γ with `1 < γ < 1 + 1` (by defining `+` so that `n + 1` skips over γ; e.g. `ℕ = {0, 1, γ, β, 2, …}` with `1 < γ < β < 2`, `1 + 1 = β`, `γ + 1 = 2`, `0 + n = n`, `n + 0 = n`, `0 < n + 1` everywhere). In that model T0's index domain for length 2 is `{1, γ, β, 2}`, a tumbler b of length 2 can have `bγ ≠ 0`, the first padded disagreement lands at γ, `zpd = γ`, `divergence = β`, and `γ < β` — the postcondition fails. The same gap applies symmetrically in sub-case (γ) with `#b + 1`.
**What needs resolving**: The gap between `#a < k` and `#a + 1 ≤ k` must be grounded. Either the NAT-* foundation must supply the discrete successor property (e.g., that `n + 1` is the strict lower bound on elements above `n`), or the Relationship-to-Divergence postcondition must be restated in terms that are provable from the current axioms — specifically that `zpd(a, w)` lies in `{k ∈ ℕ : #a < k}` rather than `{k ∈ ℕ : #a + 1 ≤ k}`, unless those two are shown equivalent under a declared axiom.
