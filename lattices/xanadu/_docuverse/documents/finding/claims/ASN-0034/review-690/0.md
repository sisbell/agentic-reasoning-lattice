### Antisymmetry case labeling elides a symmetry-of-equality step
**Class**: OBSERVE
**Foundation**: NAT-order (NatStrictTotalOrder) — exactly-one trichotomy consequence `¬(m < n ∧ m = n)`
**ASN**: TA5-SigValid — "antisymmetry of ≤ — supplied by NAT-order via exactly-one trichotomy, which eliminates the three disjoint-pair cases `m < n ∧ n < m`, `m < n ∧ m = n`, and `m = n ∧ n < m` from the four-way distribution of the conjoined disjunctions"
**Issue**: The four-way distribution of `(m < n ∨ m = n) ∧ (n < m ∨ n = m)` yields the second case as `m < n ∧ n = m` (equality drawn from the second factor's `n = m` disjunct), not `m < n ∧ m = n`. The exactly-one trichotomy clause ¬(m < n ∧ m = n) does not directly eliminate `m < n ∧ n = m`; doing so requires the unstated step that `n = m ↔ m = n` (symmetry of equality). The proof lists the three trichotomy exclusion-clause labels rather than the actual expanded cases, silently invoking symmetry to bridge the mismatch on the second case. Cases one and three match their expansion counterparts exactly; only case two requires this elision.
