### TA-dom precondition introduces unexplained local name
**Class**: OBSERVE
**Foundation**: ActionPoint (ActionPoint) — postcondition `1 ≤ actionPoint(w) ≤ #w`; TumblerAdd (TumblerAdd) — precondition `actionPoint(w) ≤ #a`; TA0 (WellDefinedAddition) — precondition `actionPoint(w) ≤ #a`
**ASN**: TA-dom (DisplacementDominance) — Formal Contract *Preconditions:* `k ≤ #a` where `k` is the action point of `w`
**Issue**: Every other claim that imposes this bound — TumblerAdd and TA0 — writes `actionPoint(w) ≤ #a` directly. TA-dom introduces a fresh local name `k` not used in the postcondition or proof, creating a notational inconsistency with the rest of the ASN. The TA-dom *Depends:* section itself reverts to the consistent form ("licenses `actionPoint(w) ≤ #a` precondition"), making the divergence visible within the same claim's contract. A reader consulting only the Preconditions slot will see a convention mismatch.

VERDICT: OBSERVE
