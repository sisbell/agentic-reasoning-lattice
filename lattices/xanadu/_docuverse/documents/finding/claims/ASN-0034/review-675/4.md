### Implicit case dependency of `k ≤ m` and `k ≤ n` in the index-domain check
**Class**: OBSERVE
**Foundation**: —
**ASN**: T10, index-domain paragraph: "with `k ≤ m`…" and "with `k ≤ n`…"
**Issue**: After the cases are merged, the paragraph asserts both `k ≤ m` (for the a-side) and `k ≤ n` (for the b-side) as if they are directly available from `k ≤ ℓ`. Each bound requires a case-dependent transitivity step that the merged paragraph does not trace: in Case 2, `k ≤ m` needs `k ≤ ℓ = n ≤ m` (using the `n ≤ m` derived within Case 2); in Case 1, `k ≤ n` needs `k ≤ ℓ = m ≤ n` (using the `m ≤ n` hypothesis of Case 1). Both chains are available from the respective case derivations, but the merged argument treats them as given without citation.
