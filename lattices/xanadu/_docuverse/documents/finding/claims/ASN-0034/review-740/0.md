### T4 non-adjacency quantifier missing explicit `∈ ℕ` domain annotation
**Class**: OBSERVE
**Foundation**: T0 (CarrierSetDefinition) — component-projection signature `i ↦ aᵢ : {j ∈ ℕ : 1 ≤ j ≤ #a} → ℕ`
**ASN**: T4 (HierarchicalParsing) — Formal Contract, T4-valid predicate: `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`
**Issue**: The non-adjacency quantifier declares `i` with a numeric range but no explicit `∈ ℕ` annotation. For the component accesses `tᵢ` and `tᵢ₊₁` to be grounded via T0's projection signature — which is typed over `{j ∈ ℕ : 1 ≤ j ≤ #t}` — a downstream consumer formally needs `i ∈ ℕ`. T0's comparable inner quantifier uses the explicit form `(A i ∈ ℕ : 1 ≤ i ≤ p : tᵢ = r(i))`; T4's quantifier is inconsistent with that standard. The body text makes the ℕ domain clear, so soundness is not affected, and the arithmetic bounds `1 ≤ i < #t` with `1, #t ∈ ℕ` make the domain contextually unambiguous.
