### T10a.5: The sub-case `tₓ = tᵧ, k'ₓ = 2, k'ᵧ = 1` is not covered by the inductive step
**Class**: REVISE
**Foundation**: TA5 (HierarchicalIncrement) — TA5(d): for k=1, the final position `#t+1` is set to 1; for k=2, position `#t+1` is set to 0 (separator) and position `#t+2` to 1
**ASN**: T10a (AllocatorDiscipline), Consequence 5 proof, base case `tₓ = tᵧ, k'ₓ ≠ k'ᵧ` and the two inductive-step branches
**Issue**: The base case says "take `k'ₓ = 1`, `k'ᵧ = 2`" and the inductive step has two branches: branch 1 ("When `j < #bₓ`... symmetrically for Cᵧ") and branch 2 ("When `j = #bₓ` (the case `k'ₓ = 1`, `k'ᵧ = 2`)"). For the sub-case `k'ₓ = 1, k'ᵧ = 2`, branch 2 applies: j = m+1 = #bₓ < m+2 = #bᵧ, Cₓ carries positive values at j, Cᵧ has j interior with value 0. This is fully handled.

The sub-case `k'ₓ = 2, k'ᵧ = 1` is not handled. Here j = m+1, #bₓ = m+2, #bᵧ = m+1, and bₓ[j] = 0, bᵧ[j] = 1. Branch 1's condition `j < #bₓ` holds for the Cₓ side, but branch 1 then invokes "symmetrically for Cᵧ" — which requires j to be interior to Cᵧ's base. It is not: j = m+1 = #bᵧ. For Cᵧ, position j is the sig position, incremented by each inc(·,0) step, producing values 1, 2, 3, … The needed conclusion (Cᵧ's subtree carries positive values at j, Cₓ's subtree carries 0) is correct, but is not established by either branch. The "take" amounts to an implicit WLOG that is never justified, and branch 1's "symmetrically for Cᵧ" makes a false claim when applied to this sub-case.
**What needs resolving**: The proof must either (a) explicitly justify the WLOG relabeling — noting that X and Y can always be assigned labels so that the param-1 child is called Cₓ, given that the conclusion `x ⋠ y ∧ y ⋠ x` is symmetric under X ↔ Y — or (b) add a third branch to the inductive step covering the case `j < #bᵧ ∧ j = #bᵧ` (equivalently, treat both `j = #bₓ` and `j = #bᵧ` as the "trailing-position" scenario, applying the positive-value propagation argument to whichever side has j at its sig position, and the interior-zero argument to the other side).
