### Mixed strict/non-strict transitivity used without derivation
**Class**: OBSERVE
**Foundation**: NAT-order — supplies `<`-transitivity and the `≤`-definition `m ≤ n ⟺ m < n ∨ m = n`, but states no mixed form
**ASN**: T1 (LexicographicOrder) — trichotomy Case 3: "hence `m < n` via NAT-addcompat's `m < m + 1`"; transitivity Case k₂ < k₁: "combines with `n < k₂` by NAT-order's transitivity to yield `k₁ < k₂`"; T2 (IntrinsicComparison) — "the forward inclusion `i ≤ m ⟹ i < m + 1` uses NAT-addcompat's strict successor inequality `m < m + 1` composed with NAT-order transitivity"
**Issue**: Each of these steps uses `a < b ≤ c ⟹ a < c` (or the symmetric form `a ≤ b < c ⟹ a < c`) without a derivation. The inference is correct — unfold `b ≤ c` into `b < c ∨ b = c`, dispatch by `<`-transitivity or substitution — but it recurs five or more times across T1 and T2 without being named or cited. For mechanical verification each step would need to be discharged explicitly; the repeated elision is the kind of gap a proof assistant surfaces first.
