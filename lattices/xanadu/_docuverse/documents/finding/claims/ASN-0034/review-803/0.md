### T10a.5 postcondition asserts universal prefix-comparability for ancestor-descendant allocator pairs, which is false and unestablished
**Class**: REVISE
**Foundation**: Prefix (PrefixRelation) — `p ≼ q` iff `#p ≤ #q ∧ (∀i : 1 ≤ i ≤ #p : qᵢ = pᵢ)`
**ASN**: T10a (AllocatorDiscipline), Formal Contract *Postconditions:*, T10a.5 bullet: "Ancestor-descendant pairs inherently nest; non-ancestor-descendant pairs are prefix-incomparable."
**Issue**: The preceding formal claim in the same bullet — "For allocators X, Y not in an ancestor-descendant relationship, for all x ∈ domain(X), y ∈ domain(Y), x ⋠ y ∧ y ⋠ x" — is universally quantified over both domains. The sentence "Ancestor-descendant pairs inherently nest" sits in the same postcondition bullet as a direct contrast, and its parallel position induces the same reading: for all x ∈ dom(X), y ∈ dom(Y) with X ancestor of Y, x and y are prefix-comparable. That claim is false.

What the proof actually establishes is narrower: for each spawning event (tᵢ, k'), the *specific element* tᵢ ∈ dom(parent) is a proper prefix of every element in the spawned child's domain. TA5(b) copies positions 1..#tᵢ into the child's base, and since inc(·,0) modifies only the final position (TA5-SigValid + T10a.4), positions 1..#tᵢ remain fixed throughout the child's domain. But elements of dom(A) produced after the spawning event are not prefix-comparable to descendant elements. A concrete counterexample from the axiom's own structure: let A have base (1), so dom(A) = (1),(2),(3),…; let C be spawned from (2) with k′=1, giving dom(C) = (2,1),(2,2),…. Then (3) ∈ dom(A) and (2,1) ∈ dom(C) disagree at position 1 (3 ≠ 2), so neither is a prefix of the other. Yet A is an ancestor of C.

The false claim is in the formal postcondition, where downstream consumers read it as an exportable guarantee. T10a.6 avoids relying on it (using length separation instead), but any proof that cites T10a.5 for the nesting property and quantifies universally over ancestor-descendant domain pairs would derive a wrong result.

**What needs resolving**: Remove "Ancestor-descendant pairs inherently nest" from the postcondition or replace it with the precise statement actually established: for each spawning event (tᵢ, k'), the element tᵢ ∈ dom(parent) satisfies tᵢ ≺ c for every c in the spawned child's domain (and transitively, for every element in any descendant allocator's domain tracing through this event). Elements of dom(parent) other than the spawn point need not be prefix-comparable to descendant domain elements.
