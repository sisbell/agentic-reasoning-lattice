### T3 postcondition drops `i ∈ ℕ` from the component-equality quantifier, and the proof body mis-cites T0's extensionality clause to match
**Class**: REVISE
**Foundation**: T0 (CarrierSetDefinition) — extensionality axiom, stated in T0's Formal Contract as `(A a, b ∈ T : #a = #b ∧ (A i ∈ ℕ : 1 ≤ i ≤ #a : aᵢ = bᵢ) : a = b)`, with `i ∈ ℕ` explicit.
**ASN**: T3 (CanonicalRepresentation) — Formal Contract postcondition: `a = b ⟺ #a = #b ∧ (A i : 1 ≤ i ≤ #a : aᵢ = bᵢ)`. Proof body quotation: "T0's extensionality clause — `(A a, b ∈ T : #a = #b ∧ (A i : 1 ≤ i ≤ #a : aᵢ = bᵢ) : a = b)`".
**Issue**: Both the postcondition and the proof body's quotation of T0 drop `i ∈ ℕ` from the index quantifier. T0's component-projection signature `i ↦ aᵢ : {j ∈ ℕ : 1 ≤ j ≤ #a} → ℕ` defines `aᵢ` only for `i ∈ ℕ`; for `i` outside ℕ the term `aᵢ` is not well-formed. The untyped quantifier `(A i : 1 ≤ i ≤ #a : ...)` leaves the carrier of `i` implicit, making the expression containing `aᵢ` formally ill-typed. The proof body then mis-cites T0: the axiom it claims to instantiate is not T0's actual extensionality clause — the `i ∈ ℕ` carrier present in T0's stated axiom is absent in T3's quotation. Every other index quantifier in this ASN writes `(A i ∈ ℕ : ...)` explicitly; T3 is the sole exception and is inconsistent with its own source axiom.
**What needs resolving**: T3's Formal Contract postcondition must carry the carrier: `a = b ⟺ #a = #b ∧ (A i ∈ ℕ : 1 ≤ i ≤ #a : aᵢ = bᵢ)`. The quoted T0 extensionality clause in the proof body must match T0's actual axiom, restoring `i ∈ ℕ`.
