- `𝒯` — the carrier set of allocators; the tree induced by the spawning rule, containing the root and every allocator reachable by finite iteration of child-spawning
- `parent(·)` — function mapping each non-root allocator to its parent in `𝒯`
- `spawnPt(·)` — function mapping each non-root allocator to the tumbler in the parent's domain at which it was spawned
- `spawnParam(·)` — function mapping each non-root allocator to the spawn parameter `k' ∈ {1, 2}` used at its creation
- `dom(·)` — function mapping each allocator `A` to its output domain `{tₙ : n ≥ 0}` generated by iterated `inc(·, 0)` from `A`'s base
- `same_allocator(·, ·)` — binary predicate asserting that two tumblers lie in the domain of a common allocator
- `depth(·)` — function on `𝒯` satisfying `depth(root) = 0` and `depth(Y) = depth(parent(Y)) + 1`
