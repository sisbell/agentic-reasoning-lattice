**T10a.5 (CrossAllocatorIncomparability).** For any two allocators X and Y not in an ancestor-descendant relationship, every output of X is prefix-incomparable with every output of Y. This supplies T10's non-nesting precondition `p‚āĀ ‚č† p‚āā ‚ąß p‚āā ‚č† p‚āĀ` at the domain prefixes of every non-ancestor-descendant allocator pair. (The ancestor-descendant case carries a narrower nesting fact, not universal pairwise comparability between domains: at each spawning event `(tŠĶĘ, k')`, TA5(b) gives agreement on positions `1, ‚Ä¶, #tŠĶĘ` and TA5(d) gives `#inc(tŠĶĘ, k') = #tŠĶĘ + k'`; since `k' ‚ąą {1, 2}`, NAT-addcompat's strict successor `n < n + 1` and left order-compatibility sharpen this to `#inc(tŠĶĘ, k') > #tŠĶĘ`, so `tŠĶĘ ‚Čļ inc(tŠĶĘ, k')`. TA5(b) also keeps positions `1, ‚Ä¶, #tŠĶĘ` fixed throughout the spawned child's domain and every descendant allocator traced through `tŠĶĘ`, so `tŠĶĘ ‚Čļ c` for every such `c`; elements of `dom(parent)` other than `tŠĶĘ` need not be prefix-comparable to descendant domain elements.)

*Proof.* Two allocators not in an ancestor-descendant relationship share a lowest common ancestor C. Let C_X be the child of C on the path to X (with X = C_X when X is a direct child), and C_Y the child on the path to Y. Since C is the lowest common ancestor and X, Y are not ancestor-descendant related, C_X ‚Č† C_Y. Let s_X be the output of C from which C_X was spawned, and s_Y the output from which C_Y was spawned.

We require a preliminary fact.

**Positional inheritance.** Let allocator A have base b of length m, and let p be a position with 1 ‚Č§ p < m. Then every output of every allocator in the subtree rooted at A has value b_p at position p.

For A itself: every output has length m by T10a.1, and inc(¬∑, 0) modifies only position sig(¬∑) = m (by TA5-SigValid, since every output satisfies T4 by T10a.4). Since p < m, position p is unmodified: every output of A has value b_p at position p. For a child of A spawned from output u via inc(u, k'') with k'' ‚ąą {1, 2}: the child's base c has length m + k'' and agrees with u on positions 1 through m by TA5(b). Since u_p = b_p, c_p = b_p. Since p < m < m + k'', position p satisfies p < #c, and the argument applies to the child's subtree. By induction on subtree depth, every output of every descendant has value b_p at position p. ‚ąé (positional inheritance)

**Case 1: s_X ‚Č† s_Y.** Both are sibling outputs of C, so #s_X = #s_Y = L by T10a.1. Since s_X ‚Č† s_Y and both have length L, T3 yields a position j with 1 ‚Č§ j ‚Č§ L where (s_X)_j ‚Č† (s_Y)_j.

The base of C_X is inc(s_X, k'_X) for some k'_X ‚ąą {1, 2}, which has length L + k'_X and agrees with s_X on positions 1 through L by TA5(b). Since j ‚Č§ L < L + k'_X, positional inheritance gives: every output in C_X's subtree has value (s_X)_j at position j. The base of C_Y is inc(s_Y, k'_Y) for some k'_Y ‚ąą {1, 2}, and symmetrically every output in C_Y's subtree has value (s_Y)_j at position j.

For any output x of X and y of Y: x_j = (s_X)_j ‚Č† (s_Y)_j = y_j. Every output in C_X's subtree has length at least L + 1 (by T10a.1 for C_X's own outputs and T10a.3 for deeper descendants); the same for C_Y. So j ‚Č§ L < L + 1 ‚Č§ min(#x, #y). The disagreement at position j excludes both x ‚Čľ y and y ‚Čľ x by the Prefix definition. ‚ąé (Case 1)

**Case 2: s_X = s_Y = s.** The at-most-once child-spawning constraint of T10a requires that each (t, k') pair produces at most one child. Since C_X ‚Č† C_Y are both spawned from s, they must use distinct spawning parameters: k'_X ‚Č† k'_Y with {k'_X, k'_Y} ‚äÜ {1, 2}. The T10a.5 postcondition is invariant under the exchange (X, Y) ‚ÜĒ (Y, X), so assume k'_X = 1 and k'_Y = 2.

The base of C_X is inc(s, 1), which has length #s + 1 with (inc(s, 1))_{#s+1} = 1 by TA5(d). The base of C_Y is inc(s, 2), which has length #s + 2 with (inc(s, 2))_{#s+1} = 0 and (inc(s, 2))_{#s+2} = 1 by TA5(d).

We show that every output in C_X's subtree has value ‚Č• 1 at position #s + 1, while every output in C_Y's subtree has value 0 there.

*Outputs of C_X at position #s + 1.* Position #s + 1 is the last position of C_X's outputs (length #s + 1 by T10a.1). Enumerate C_X's outputs as `t‚āÄ, t‚āĀ, t‚āā, ‚Ä¶` with `t‚āÄ` the base and `t‚āô‚āä‚āĀ = inc(t‚āô, 0)`. We prove `(t‚āô)_{#s+1} ‚Č• 1` by induction on `n`. *Base case* `n = 0`: `(t‚āÄ)_{#s+1} = 1` by TA5(d). *Inductive step.* Assume `(t‚āô)_{#s+1} ‚Č• 1`. Every output of C_X satisfies T4 (T10a.4), so TA5-SigValid gives `sig(t‚āô) = #t‚āô = #s + 1`, and TA5(c) yields `(t‚āô‚āä‚āĀ)_{#s+1} = (t‚āô)_{#s+1} + 1`. T0 places `(t‚āô)_{#s+1} ‚ąą ‚Ąē`; NAT-closure places `(t‚āô)_{#s+1} + 1 ‚ąą ‚Ąē`; NAT-zero gives `0 ‚Č§ (t‚āô)_{#s+1}`; NAT-addcompat's strict successor gives `(t‚āô)_{#s+1} + 1 > (t‚āô)_{#s+1} ‚Č• 0`; NAT-discrete (at `m = 0`) with NAT-zero's `0 ‚Č§ (t‚āô)_{#s+1} + 1` rules out `0 ‚Č§ (t‚āô)_{#s+1} + 1 < 1`, forcing `(t‚āô)_{#s+1} + 1 ‚Č• 1`.

*Descendants of C_X at position #s + 1.* Any child of C_X is spawned from an output u with u_{#s+1} ‚Č• 1. The child's base has length #s + 1 + k'' ‚Č• #s + 2 for some k'' ‚ąą {1, 2}, and agrees with u on positions 1 through #s + 1 by TA5(b), inheriting value ‚Č• 1 at position #s + 1. Since #s + 1 < #s + 1 + k'', positional inheritance propagates this through the child's subtree. By induction on depth, every output in C_X's subtree has value ‚Č• 1 at position #s + 1.

*All outputs in C_Y's subtree at position #s + 1.* The base of C_Y has value 0 at position #s + 1, and #s + 1 < #s + 2. Positional inheritance gives: every output in C_Y's subtree has value 0 at position #s + 1.

For any output x of X and y of Y: x_{#s+1} ‚Č• 1 and y_{#s+1} = 0. Every output in C_X's subtree has length ‚Č• #s + 1, and every output in C_Y's subtree has length ‚Č• #s + 2, so position #s + 1 lies within the range of both x and y. The disagreement excludes both x ‚Čľ y and y ‚Čľ x by the Prefix definition. ‚ąé
