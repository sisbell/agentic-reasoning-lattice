## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this, writing *node* for Nelson's *server* and *element* for his *contents* and retaining *user* and *document* verbatim; the renaming is notational. An address tumbler has up to four fields separated by zero-valued components; which fields are present in a given tumbler is determined by its zero count, and T4's Definition below gives the per-`k` canonical written form explicitly. The maximal written form ‚ÄĒ the case with all four fields present, `zeros(t) = 3` ‚ÄĒ is:

`t = N‚āĀ. ... .N‚āź . 0 . U‚āĀ. ... .UŠĶ¶ . 0 . D‚āĀ. ... .DŠĶß . 0 . E‚āĀ. ... .Eőī`

where `0 < NŠĶĘ, 0 < U‚Īľ, 0 < D‚āĖ, 0 < E‚āó` at every position. The four fields ‚ÄĒ each of which may be absent at lower zero counts ‚ÄĒ are:

- **Node field** `N‚āĀ. ... .N‚āź`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U‚āĀ. ... .UŠĶ¶`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D‚āĀ. ... .DŠĶß`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked ‚ÄĒ "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E‚āĀ. ... .Eőī`: identifies the content element. T4 constrains only `0 < E‚āó` at every element-field position; typed interpretations of `E‚āĀ` are the concern of downstream ASNs.

The count of zero-valued components determines the specificity level ‚ÄĒ for each `k ‚ąą {0, 1, 2, 3}` at which a T4-valid tumbler has `zeros(t) = k`, T4c is the single definitional site, assigning the label node address (at `k = 0`), user address (at `k = 1`), document address (at `k = 2`), or element address (at `k = 3`). The names appear here only as a forward reference to that definition; they are not defined by the present paragraph.

**T4 (HierarchicalParsing).** NAT-card axiomatizes the cardinality operator `|¬∑|` as a well-defined total function on subsets of every initial segment `{1, 2, ‚Ä¶, n} ‚äÜ ‚Ąē` (`n ‚ąą ‚Ąē`), with codomain ‚Ąē and with `|S|` characterised as the length of the unique strictly increasing enumeration of `S`; here at T4, where we use both operators, we distinguish `|¬∑|` from T0's tumbler-length `#¬∑ : T ‚Üí ‚Ąē`, which acts on sequences rather than sets. Define `zeros(t) = |{i : 1 ‚Č§ i ‚Č§ #t ‚ąß tŠĶĘ = 0}|`; the argument `{i : 1 ‚Č§ i ‚Č§ #t ‚ąß tŠĶĘ = 0}` is a subset of the index domain `{1, ‚Ä¶, #t} ‚äÜ ‚Ąē` (T0), so NAT-card applies at `n = #t` and `zeros(t) ‚ąą ‚Ąē` with `zeros(t) ‚Č§ #t`. We say a position `i` of `t` is a *field separator* iff `tŠĶĘ = 0`, and a *field component* otherwise; the two roles are decidable by inspecting `tŠĶĘ`. Call `t ‚ąą T` *T4-valid* iff the four conditions `zeros(t) ‚Č§ 3`, `(A i : 1 ‚Č§ i < #t : ¬¨(tŠĶĘ = 0 ‚ąß tŠĶĘ‚āä‚āĀ = 0))`, `t‚āĀ ‚Č† 0`, `t_{#t} ‚Č† 0` all hold; we name the last three conjuncts collectively the *field-segment constraint*. T4 is purely definitional: it characterises T4-valid as a predicate on `T` (the four-conjunct conjunction above) without asserting which `t ‚ąą T` satisfy it; downstream consumers requiring T4-validity (T4b's projection domains `dom(N), dom(U), dom(D), dom(E)`; T4c's level subdomain) carry it as an explicit precondition rather than receiving it from T4. T0 fixes the carrier as ‚Ąē; NAT-order supplies the strict total order `<` on ‚Ąē and its companion `‚Č§` (defined by `m ‚Č§ n ‚üļ m < n ‚ą® m = n`), which support the zero-count bound `zeros(t) ‚Č§ 3` and the adjacency clause's index range `1 ‚Č§ i < #t`; NAT-closure supplies `1 ‚ąą ‚Ąē` together with closure of ‚Ąē under addition, grounding the numerals `2 := 1 + 1 ‚ąą ‚Ąē` and `3 := 2 + 1 ‚ąą ‚Ąē` so that the bound `zeros(t) ‚Č§ 3` compares two elements of ‚Ąē; and NAT-zero's disjunction `(A n ‚ąą ‚Ąē :: 0 < n ‚ą® 0 = n)` instantiated at each field-component position `tŠĶĘ` forces strict positivity ‚ÄĒ T0 places `tŠĶĘ ‚ąą ‚Ąē`, and excluding the equality branch `0 = tŠĶĘ` via `tŠĶĘ ‚Č† 0` leaves `0 < tŠĶĘ`, giving `0 < NŠĶĘ, 0 < U‚Īľ, 0 < D‚āĖ, 0 < E‚āó`; write `‚Ąē‚Āļ = {n ‚ąą ‚Ąē : 0 < n}` for the strictly positive naturals. The *field segments* of `t` are the maximal contiguous sub-sequences of field-component positions ‚ÄĒ `k+1` of them when `zeros(t) = k`, separated by the field separators.

*Exhaustion.* We derive as a Consequence ‚ÄĒ universally quantified by the bound alone ‚ÄĒ that `(A t ‚ąą T : zeros(t) ‚Č§ 3 : zeros(t) ‚ąą {0, 1, 2, 3})`, i.e. for every `t ‚ąą T` with `zeros(t) ‚Č§ 3`, `zeros(t) ‚ąą {0, 1, 2, 3}`. The derivation uses only `zeros(t) ‚Č§ 3`, NAT-zero, NAT-order's exactly-one trichotomy (Consequence), NAT-discrete, and NAT-closure's left additive identity (cited at the `m = 0` step below). NAT-card's codomain secures `zeros(t) ‚ąą ‚Ąē` so that trichotomy applies at `zeros(t)`. NAT-zero supplies `0 < zeros(t) ‚ą® 0 = zeros(t)` ‚ÄĒ equivalently `0 ‚Č§ zeros(t)` by NAT-order's definition of `‚Č§`. Trichotomy at `(zeros(t), m)` asserts that exactly one of `zeros(t) < m`, `zeros(t) = m`, `m < zeros(t)` holds. The mechanism for excluding the branch `zeros(t) < m` given a lower bound `m ‚Č§ zeros(t)` is uniform: unfolding `‚Č§` to `m < zeros(t) ‚ą® m = zeros(t)` shows that one of trichotomy's other two alternatives holds, and trichotomy's exactly-one clause ‚ÄĒ exported by NAT-order as a Consequence ‚ÄĒ then forbids the third. At `m = 0`: the case `zeros(t) < 0` is excluded by `0 ‚Č§ zeros(t)` via the exactly-one route just described, so either `zeros(t) = 0`, whereupon `zeros(t) ‚ąą {0, 1, 2, 3}`, or `0 < zeros(t)`, which NAT-discrete at `(m, n) := (0, zeros(t))` promotes to `0 + 1 ‚Č§ zeros(t)`, which NAT-closure's left additive identity `(A n ‚ąą ‚Ąē :: 0 + n = n)` at `n := 1` reduces to `1 ‚Č§ zeros(t)`. At `m = 1` (continuing the `0 < zeros(t)` branch): `zeros(t) < 1` is excluded by `1 ‚Č§ zeros(t)` by the same route, so either `zeros(t) = 1`, whereupon `zeros(t) ‚ąą {0, 1, 2, 3}`, or `1 < zeros(t)`, which NAT-discrete promotes to `2 ‚Č§ zeros(t)`. At `m = 2` (continuing the `1 < zeros(t)` branch): `zeros(t) < 2` is excluded by `2 ‚Č§ zeros(t)` by the same route, so either `zeros(t) = 2`, whereupon `zeros(t) ‚ąą {0, 1, 2, 3}`, or `2 < zeros(t)`, which NAT-discrete promotes to `3 ‚Č§ zeros(t)`. At the final step, trichotomy at `(zeros(t), 3)` is confronted with two bounds: `3 ‚Č§ zeros(t)` excludes `zeros(t) < 3` (unfolding `3 ‚Č§ zeros(t)` to `3 < zeros(t) ‚ą® 3 = zeros(t)` makes one of the other two trichotomy alternatives hold, forcing `zeros(t) < 3` out by exactly-one), and symmetrically `zeros(t) ‚Č§ 3` excludes `3 < zeros(t)` (unfolding to `zeros(t) < 3 ‚ą® zeros(t) = 3` makes one of the other two alternatives hold, forcing `3 < zeros(t)` out by exactly-one), leaving `zeros(t) = 3`. Every branch terminates with `zeros(t) ‚ąą {0, 1, 2, 3}`.

Three downstream claims ‚ÄĒ **T4a (SyntacticEquivalence)**, **T4b (UniqueParse)**, and **T4c (LevelDetermination)** ‚ÄĒ build on T4 with additional foundations. T4a relates T4's positional field-segment constraint to segment non-emptiness; T4b introduces the four partial field-projections `N, U, D, E : T ‚áÄ T` over the T4-valid subdomain; T4c assigns the four hierarchical level labels ‚ÄĒ node, user, document, and element address ‚ÄĒ by zero count. Each is stated, proved, and equipped with its own Formal Contract in its own claim section; the present paragraph is a forward reference, not a restatement.

*Formal Contract:*
- *Definition:*
  - *Zero-count.* `zeros(t) = |{i : 1 ‚Č§ i ‚Č§ #t ‚ąß tŠĶĘ = 0}|`; the index set `{i : 1 ‚Č§ i ‚Č§ #t ‚ąß tŠĶĘ = 0}` is a subset of `{1, ‚Ä¶, #t} ‚äÜ ‚Ąē` (T0's index domain), so NAT-card applies at `n = #t` and yields `zeros(t) ‚ąą ‚Ąē`.
  - *Field separator.* A position `i` of `t` is a *field separator* iff `tŠĶĘ = 0`; the remaining positions are *field components*.
  - *T4-valid predicate.* `t ‚ąą T` is *T4-valid* iff `zeros(t) ‚Č§ 3 ‚ąß (A i : 1 ‚Č§ i < #t : ¬¨(tŠĶĘ = 0 ‚ąß tŠĶĘ‚āä‚āĀ = 0)) ‚ąß t‚āĀ ‚Č† 0 ‚ąß t_{#t} ‚Č† 0`; the last three conjuncts are collectively the *field-segment constraint*.
  - *Canonical written form.* For each `k ‚ąą ‚Ąē` with `0 ‚Č§ k ‚Č§ 3` at which `zeros(t) = k`, the canonical written form of a T4-valid `t` is ‚ÄĒ `k = 0`: `t = N‚āĀ. ... .N‚āź`; `k = 1`: `t = N‚āĀ. ... .N‚āź . 0 . U‚āĀ. ... .UŠĶ¶`; `k = 2`: `t = N‚āĀ. ... .N‚āź . 0 . U‚āĀ. ... .UŠĶ¶ . 0 . D‚āĀ. ... .DŠĶß`; `k = 3`: `t = N‚āĀ. ... .N‚āź . 0 . U‚āĀ. ... .UŠĶ¶ . 0 . D‚āĀ. ... .DŠĶß . 0 . E‚āĀ. ... .Eőī`, with `0 < NŠĶĘ, 0 < U‚Īľ, 0 < D‚āĖ, 0 < E‚āó` at every position present.
- *Consequence:* `(A t ‚ąą T : zeros(t) ‚Č§ 3 : zeros(t) ‚ąą {0, 1, 2, 3})` ‚ÄĒ for every `t ‚ąą T` with `zeros(t) ‚Č§ 3`, `zeros(t) ‚ąą {0, 1, 2, 3}` (equivalently `zeros(t) = 0 ‚ą® zeros(t) = 1 ‚ą® zeros(t) = 2 ‚ą® zeros(t) = 3`). Derived below from the bound `zeros(t) ‚Č§ 3`, NAT-zero's `0 ‚Č§ zeros(t)`, NAT-order's trichotomy, NAT-discrete, and NAT-closure's left additive identity `(A n ‚ąą ‚Ąē :: 0 + n = n)` at `n := 1` (collapsing NAT-discrete's output `0 + 1 ‚Č§ zeros(t)` to `1 ‚Č§ zeros(t)` at the `m = 0` step); NAT-card's codomain secures `zeros(t) ‚ąą ‚Ąē` so that trichotomy applies.
- *Preconditions:* `t ‚ąą T`.
- *Depends:*
  - T0 (CarrierSetDefinition) ‚ÄĒ supplies the carrier ‚Ąē, the tumbler length `#¬∑`, and the index domain `{1, ‚Ä¶, #t}`.
  - NAT-zero (NatZeroMinimum) ‚ÄĒ supplies `(A n ‚ąą ‚Ąē :: 0 < n ‚ą® 0 = n)`.
  - NAT-discrete (NatDiscreteness) ‚ÄĒ supplies the strict-to-`+1` promotion `m < n ‚üĻ m + 1 ‚Č§ n`.
  - NAT-order (NatStrictTotalOrder) ‚ÄĒ supplies `<` on ‚Ąē with its companion `‚Č§` (`m ‚Č§ n ‚üļ m < n ‚ą® m = n`) and the exactly-one trichotomy Consequence.

  - NAT-card (NatFiniteSetCardinality) ‚ÄĒ axiomatizes `|¬∑|` on subsets of every initial segment `{1, ‚Ä¶, n} ‚äÜ ‚Ąē` with codomain ‚Ąē.
