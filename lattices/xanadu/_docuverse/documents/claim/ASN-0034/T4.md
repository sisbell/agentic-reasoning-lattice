## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this, writing *node* for Nelson's *server* and *element* for his *contents* and retaining *user* and *document* verbatim; the renaming is notational. An address tumbler has up to four fields separated by zero-valued components; which fields are present in a given tumbler is determined by its zero count. As an illustration of the structure that T4-validity admits, the maximal written form ‚Äî the case with all four fields present, `zeros(t) = 3` ‚Äî is:

`t = N‚ÇÅ. ... .N‚Çê . 0 . U‚ÇÅ. ... .U·µ¶ . 0 . D‚ÇÅ. ... .D·µß . 0 . E‚ÇÅ. ... .EŒ¥`

where `0 < N·µ¢, 0 < U‚±º, 0 < D‚Çñ, 0 < E‚Çó` at every position. The four fields ‚Äî each of which may be absent at lower zero counts ‚Äî are:

- **Node field** `N‚ÇÅ. ... .N‚Çê`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U‚ÇÅ. ... .U·µ¶`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D‚ÇÅ. ... .D·µß`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked ‚Äî "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E‚ÇÅ. ... .EŒ¥`: identifies the content element. T4 constrains only `0 < E‚Çó` at every element-field position; typed interpretations of `E‚ÇÅ` are the concern of downstream ASNs.

The count of zero-valued components determines the specificity level; T4c is the single definitional site for the resulting address-kind labels.

**T4 (HierarchicalParsing).** Define `zeros(t) = |{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}|`, with `|¬∑|` NAT-card's set-cardinality operator (distinct from T0's tumbler-length `#¬∑`, which acts on sequences); the argument `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}` is a subset of the index domain `{1, ‚Ä¶, #t} ‚äÜ ‚Ñï` (T0), so NAT-card applies at `n = #t` and `zeros(t) ‚àà ‚Ñï` with `zeros(t) ‚â§ #t`. We say a position `i` of `t` is a *field separator* iff `t·µ¢ = 0`, and a *field component* otherwise; the two roles are decidable by inspecting `t·µ¢`. The non-adjacency clause names two component accesses, `t·µ¢` and `t·µ¢‚Çä‚ÇÅ`, both of which T0's component-projection signature requires to be indexed in `{j ‚àà ‚Ñï : 1 ‚â§ j ‚â§ #t}`. For `i` in the quantifier's range `1 ‚â§ i < #t`, `t·µ¢` is licensed directly by the range; the neighbour `t·µ¢‚Çä‚ÇÅ` requires `1 ‚â§ i + 1 ‚â§ #t`, and we discharge each bound separately. The upper bound `i + 1 ‚â§ #t` is NAT-discrete at `(m, n) := (i, #t)` applied to the range hypothesis `i < #t`. The lower bound `1 ‚â§ i + 1` is NAT-closure's successor-positivity axiom `(A n ‚àà ‚Ñï :: 0 < n + 1)`, instantiated at `n := i`, which yields `0 < i + 1`; NAT-discrete at `(m, n) := (0, i + 1)` then promotes this to `0 + 1 ‚â§ i + 1`, and NAT-closure's left additive identity `(A n ‚àà ‚Ñï :: 0 + n = n)` at `n := 1` rewrites the left-hand side to `1`, leaving `1 ‚â§ i + 1`. Both bounds in hand, `i + 1 ‚àà {j ‚àà ‚Ñï : 1 ‚â§ j ‚â§ #t}`, so T0's projection signature gives `t·µ¢‚Çä‚ÇÅ ‚àà ‚Ñï` and the access is well-defined throughout the quantifier's range ‚Äî including the `i ‚â• 1` indices for which NAT-zero alone would leave `0 = i + 1` unrefuted. Call `t ‚àà T` *T4-valid* iff the four conditions `zeros(t) ‚â§ 3`, `(A i : 1 ‚â§ i < #t : ¬¨(t·µ¢ = 0 ‚àß t·µ¢‚Çä‚ÇÅ = 0))`, `t‚ÇÅ ‚âÝ 0`, `t_{#t} ‚âÝ 0` all hold; we name the last three conjuncts collectively the *field-segment constraint*. The *field segments* of `t` are the maximal contiguous sub-sequences of field-component positions, delimited by the field separators; the equivalence between every-segment-non-empty and the field-segment constraint is proved downstream in T4a rather than stipulated here.

*Exhaustion.* We derive as a Consequence ‚Äî universally quantified by the bound alone ‚Äî that `(A t ‚àà T : zeros(t) ‚â§ 3 : zeros(t) ‚àà {0, 1, 2, 3})`, i.e. for every `t ‚àà T` with `zeros(t) ‚â§ 3`, `zeros(t) ‚àà {0, 1, 2, 3}`. NAT-order's trichotomy at `(zeros(t), 0)` asserts that exactly one of `zeros(t) < 0`, `zeros(t) = 0`, `0 < zeros(t)` holds; NAT-zero supplies `0 < zeros(t) ‚à® 0 = zeros(t)`, two of those three alternatives, so by trichotomy's exactly-one clause `zeros(t) < 0` is forbidden. Either `zeros(t) = 0`, whereupon `zeros(t) ‚àà {0, 1, 2, 3}`, or `0 < zeros(t)`, which NAT-discrete at `(m, n) := (0, zeros(t))` promotes to `0 + 1 ‚â§ zeros(t)`, which NAT-closure's left additive identity `(A n ‚àà ‚Ñï :: 0 + n = n)` at `n := 1` reduces to `1 ‚â§ zeros(t)`. At `m = 1` (continuing the `0 < zeros(t)` branch): trichotomy at `(zeros(t), 1)` excludes `zeros(t) < 1` because `1 ‚â§ zeros(t)` unfolds via NAT-order's `‚â§` to `1 < zeros(t) ‚à® 1 = zeros(t)`, two of trichotomy's three alternatives, forcing the third out by exactly-one; so either `zeros(t) = 1`, whereupon `zeros(t) ‚àà {0, 1, 2, 3}`, or `1 < zeros(t)`, which NAT-discrete promotes to `2 ‚â§ zeros(t)`. At `m = 2` (continuing the `1 < zeros(t)` branch): trichotomy excludes `zeros(t) < 2` by the same route from `2 ‚â§ zeros(t)`, so either `zeros(t) = 2`, whereupon `zeros(t) ‚àà {0, 1, 2, 3}`, or `2 < zeros(t)`, which NAT-discrete promotes to `3 ‚â§ zeros(t)`. At the final step, trichotomy at `(zeros(t), 3)` is confronted with two bounds: `3 ‚â§ zeros(t)` excludes `zeros(t) < 3` (unfolding `3 ‚â§ zeros(t)` to `3 < zeros(t) ‚à® 3 = zeros(t)` makes one of the other two trichotomy alternatives hold, forcing `zeros(t) < 3` out by exactly-one), and symmetrically `zeros(t) ‚â§ 3` excludes `3 < zeros(t)` (unfolding to `zeros(t) < 3 ‚à® zeros(t) = 3` makes one of the other two alternatives hold, forcing `3 < zeros(t)` out by exactly-one), leaving `zeros(t) = 3`. Every branch terminates with `zeros(t) ‚àà {0, 1, 2, 3}`.

*Formal Contract:*
- *Definition:*
  - *Zero-count.* `zeros(t) = |{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}|`; the index set `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}` is a subset of `{1, ‚Ä¶, #t} ‚äÜ ‚Ñï` (T0's index domain), so NAT-card applies at `n = #t` and yields `zeros(t) ‚àà ‚Ñï`.
  - *Field separator.* A position `i` of `t` is a *field separator* iff `t·µ¢ = 0`; the remaining positions are *field components*.
  - *Field segment.* The *field segments* of `t` are the maximal contiguous sub-sequences of field-component positions, delimited by the field separators. The terminology names what a segment *is*; the equivalence between every-segment-non-empty and the field-segment constraint is proved in T4a, not stipulated here.
  - *T4-valid predicate.* `t ‚àà T` is *T4-valid* iff `zeros(t) ‚â§ 3 ‚àß (A i : 1 ‚â§ i < #t : ¬¨(t·µ¢ = 0 ‚àß t·µ¢‚Çä‚ÇÅ = 0)) ‚àß t‚ÇÅ ‚âÝ 0 ‚àß t_{#t} ‚âÝ 0`; the last three conjuncts are collectively the *field-segment constraint*.
- *Consequence:* `(A t ‚àà T : zeros(t) ‚â§ 3 : zeros(t) ‚àà {0, 1, 2, 3})` ‚Äî for every `t ‚àà T` with `zeros(t) ‚â§ 3`, `zeros(t) ‚àà {0, 1, 2, 3}` (equivalently `zeros(t) = 0 ‚à® zeros(t) = 1 ‚à® zeros(t) = 2 ‚à® zeros(t) = 3`). Derived in the *Exhaustion* paragraph above.
- *Preconditions:* `t ‚àà T`.
- *Depends:*
  - T0 (CarrierSetDefinition) ‚Äî supplies the carrier ‚Ñï, the tumbler length `#¬∑`, and the index domain `{1, ‚Ä¶, #t}`.
  - NAT-zero (NatZeroMinimum) ‚Äî supplies `(A n ‚àà ‚Ñï :: 0 < n ‚à® 0 = n)`.
  - NAT-discrete (NatDiscreteness) ‚Äî supplies the strict-to-`+1` promotion `m < n ‚üπ m + 1 ‚â§ n`, instantiated at `(i, #t)` for the upper bound `i + 1 ‚â§ #t` of the `t·µ¢‚Çä‚ÇÅ` well-definedness, at `(0, i + 1)` for the conversion `0 < i + 1 ‚üπ 0 + 1 ‚â§ i + 1` in that same derivation, and at `(0, zeros(t))`, `(1, zeros(t))`, `(2, zeros(t))` in the *Exhaustion* induction.
  - NAT-order (NatStrictTotalOrder) ‚Äî supplies `<` on ‚Ñï with its companion `‚â§` (`m ‚â§ n ‚ü∫ m < n ‚à® m = n`) and the exactly-one trichotomy Consequence.
  - NAT-closure (NatArithmeticClosureAndIdentity) ‚Äî supplies `1 ‚àà ‚Ñï`, closure of ‚Ñï under addition (grounding the numerals `2 := 1 + 1` and `3 := 2 + 1`), the left additive identity `(A n ‚àà ‚Ñï :: 0 + n = n)`, and the successor-positivity axiom `(A n ‚àà ‚Ñï :: 0 < n + 1)` consumed at `n := i` to discharge the lower bound `1 ‚â§ i + 1` of the `t·µ¢‚Çä‚ÇÅ` well-definedness derivation.
  - NAT-card (NatFiniteSetCardinality) ‚Äî axiomatizes `|¬∑|` on subsets of every initial segment `{1, ‚Ä¶, n} ‚äÜ ‚Ñï` with codomain ‚Ñï.
