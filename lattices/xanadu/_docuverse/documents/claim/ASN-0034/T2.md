**T2 (IntrinsicComparison).** The order relation T1 is computable from the two tumblers alone, without consulting any external data structure. The number of component pairs the comparison examines is bounded by both `#a` and `#b` вҖ” at most `#a` pairs and at most `#b` pairs.

*Proof.* Let `a = aвӮҒ. ... .aвӮҳ` and `b = bвӮҒ. ... .bвӮҷ`. The definition of `<` in T1 asks for a witness position `k вүҘ 1` with agreement at all positions before `k`, and either a component divergence at `k` or prefix exhaustion at `k`. The comparison scans positions `i = 1, 2, ...` and compares the pair `(aбөў, bбөў)` at each. Two outcomes are possible.

*Case 1: divergence at some position `k` with `k вүӨ m вҲ§ k вүӨ n`.* The scan finds `aвӮ– вү  bвӮ–` after verifying `aбөў = bбөў` for all `1 вүӨ i < k`. Then `k` is the least element of `{i вҲҲ в„• : 1 вүӨ i вҲ§ i вүӨ m вҲ§ i вүӨ n вҲ§ aбөў вү  bбөў}`, well-defined by NAT-wellorder. By minimality, the examined positions are precisely `{1, 2, ..., k}`, so exactly `k` component pairs are examined; `k вүӨ m = #a` and `k вүӨ n = #b` by the case hypothesis. NAT-order's trichotomy at `(aвӮ–, bвӮ–)` resolves `aвӮ– вү  bвӮ–` into `aвӮ– < bвӮ– вҲЁ bвӮ– < aвӮ–`. By T1 case (i), `aвӮ– < bвӮ–` yields `a < b` and `bвӮ– < aвӮ–` yields `b < a`.

*Case 2: no divergence within the shared range.* The scan exhausts every position `i` with `1 вүӨ i вҲ§ i вүӨ m вҲ§ i вүӨ n`. NAT-order's trichotomy at `(m, n)` partitions the outcome.

*Sub-case `m < n`.* By NAT-order transitivity `i вүӨ m вҲ§ m < n вҹ№ i вүӨ n`, the shared range equals `{i : 1 вүӨ i вүӨ m}`, so exactly `m` pairs are examined; `m вүӨ m` and `m вүӨ n`. T1 case (ii) requires a witness `k = m + 1 вүӨ n`; NAT-discrete's forward direction `m < n вҹ№ m + 1 вүӨ n` bridges the case hypothesis to that arithmetic witness. Taking `k = m + 1`, we identify the scan's agreement domain `{i : 1 вүӨ i вүӨ m}` with T1 case (ii)'s witness domain `{i : 1 вүӨ i < k}`: the forward inclusion `i вүӨ m вҹ№ i < m + 1` uses NAT-addcompat's strict successor inequality `m < m + 1` composed with NAT-order transitivity, and the reverse `i < m + 1 вҹ№ i вүӨ m` uses NAT-discrete's no-interval Consequence. T1 case (ii) then gives `a < b`.

*Sub-case `n < m`.* By the mirrored transitivity, the shared range equals `{i : 1 вүӨ i вүӨ n}`, so exactly `n` pairs are examined; `n вүӨ n` and `n вүӨ m`. NAT-discrete's forward direction `n < m вҹ№ n + 1 вүӨ m`, applied with roles swapped, supplies the witness `k = n + 1 вүӨ m` demanded by T1 case (ii); the domain identification `{i : 1 вүӨ i вүӨ n} = {i : 1 вүӨ i < n + 1}` is furnished symmetrically by NAT-addcompat's `n < n + 1` with NAT-order transitivity for the forward inclusion and NAT-discrete's no-interval Consequence for the reverse. T1 case (ii) with roles swapped gives `b < a`.

*Sub-case `m = n`.* The shared range covers every position of either tumbler, so exactly `m` pairs are examined; `m вүӨ m` and `m вүӨ n` by substitution. Componentwise agreement with `m = n` gives `a = b` by T3.

In every case, the count of pairs examined is at most `#a` and at most `#b`. Every value consulted вҖ” the components `aбөў`, `bбөў`, and the lengths `m`, `n` вҖ” belongs to the two tumblers themselves; no tree, index, or external state participates. вҲҺ

Span containment tests, link search, and index traversal all reduce to tumbler comparison. If comparison required a lookup, these operations would depend on auxiliary state and the decentralization guarantee would collapse.

Gregory's `tumblercmp` delegates to `abscmp`, which performs a purely positional comparison: exponent first, then lexicographic mantissa slot-by-slot. No external state is consulted.
