## Spans

By **Definition (Span)**, a span denotes a contiguous range of tumblers from a start address up to but not including the displaced endpoint. The form of `в„“` depends on the hierarchical level at which the span operates, because the action point of `в„“` must match the level of the start address `s`.

Nelson makes spans self-describing at every hierarchical level: "A digit of 'one' may be used to designate all of a given version, all versions of a given document, all works of a given author, all documents in a given project, all documents on a given server вҖ” or the entire docuverse." The "1-position convention" exploits T5: because subtrees are contiguous, a span whose start is a high-level prefix and whose length reaches to the next sibling captures exactly that subtree's content.

A span's *content* may be empty вҖ” the range populated by nothing at present вҖ” yet the span itself remains a valid address-set: "A span that contains nothing today may at a later time contain a million documents." Throughout this document, *span* denotes the address-set `span(s, в„“) = {t вҲҲ T : s вүӨ t < s вҠ• в„“}` fixed by Definition (Span), and *a span's content* denotes the documents populating that range at a given moment.

**T12 (SpanWellDefinedness).** For any `(s, в„“)` satisfying the preconditions of Definition (Span), the set `span(s, в„“)` has three properties: its upper bound `s вҠ• в„“` exists in `T`, the set is non-empty (with `s` always a member), and it is order-convex under T1.

*Proof.* Let `(s, в„“)` satisfy the preconditions of Definition (Span): `s вҲҲ T`, `в„“ вҲҲ T`, `Pos(в„“)`, and the action point `k` of `в„“` satisfies `k вүӨ #s`. Write `S = span(s, в„“) = {t вҲҲ T : s вүӨ t < s вҠ• в„“}`.

*(a) Endpoint existence.* The preconditions supply `s вҲҲ T`, `в„“ вҲҲ T`, `Pos(в„“)`, and `k вүӨ #s` вҖ” precisely the four preconditions of TA0 under `(a, w) := (s, в„“)` вҖ” so TA0 gives `s вҠ• в„“ вҲҲ T`. The set `S` is well-defined.

*(b) Non-emptiness.* We show `s вҲҲ S`. First, `s вүӨ s` by reflexivity of `вүӨ` (from T1: `a вүӨ b` iff `a < b вҲЁ a = b`). Second, the same preconditions match TA-strict under `(a, w) := (s, в„“)`, so `s вҠ• в„“ > s`, i.e., `s < s вҠ• в„“`. Both conditions hold, so `s вҲҲ S`.

*(c) Contiguity.* We show `S` is order-convex: for `a, c вҲҲ S` and `b вҲҲ T` with `a вүӨ b вүӨ c`, we have `b вҲҲ S`. By T1, `<` is a strict total order on `T` with transitivity from T1(c). Transitivity of `вүӨ` follows by case analysis on `< вҲЁ =`: the (<,<) case composes via T1(c); the (<,=) and (=,<) cases substitute equality; the (=,=) case chains equalities.

From `a вҲҲ S`, `s вүӨ a`; with `a вүӨ b`, transitivity of `вүӨ` gives `s вүӨ b`.

From `c вҲҲ S`, `c < s вҠ• в„“`. If `b = c`, then `b < s вҠ• в„“` immediately; if `b < c`, then T1(c) with `c < s вҠ• в„“` gives `b < s вҠ• в„“`.

Together, `s вүӨ b` and `b < s вҠ• в„“`, so `b вҲҲ S`. вҲҺ

We reserve T5 for the distinct claim that *prefix-defined* sets are contiguous вҖ” a non-trivial property of the lexicographic order.
