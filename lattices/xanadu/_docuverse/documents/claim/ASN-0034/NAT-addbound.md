**NAT-addbound (NatAdditionDominatesOperands).** For every `m, n ‚ąą ‚Ąē`, the sum `m + n` is bounded below by each of its operands: `m + n ‚Č• n` (right dominance) and `m + n ‚Č• m` (left dominance).

*Right dominance.* Fix `m, n ‚ąą ‚Ąē`. NAT-zero's minimality clause `(A k ‚ąą ‚Ąē :: 0 < k ‚ą® 0 = k)`, instantiated at `k := m`, delivers `0 < m ‚ą® 0 = m`; NAT-order's defining equivalence `a ‚Č§ b ‚üļ a < b ‚ą® a = b`, instantiated at `(a, b) := (0, m)`, rewrites this disjunction as `0 ‚Č§ m`. NAT-addcompat's right order compatibility `(A m', n', p ‚ąą ‚Ąē : p ‚Č§ n' : p + m' ‚Č§ n' + m')`, instantiated under the renaming `(p, n', m') := (0, m, n)`, then yields the implication `0 ‚Č§ m ‚üĻ 0 + n ‚Č§ m + n`; modus ponens with the just-established `0 ‚Č§ m` gives `0 + n ‚Č§ m + n`. NAT-closure's left additive identity `(A k ‚ąą ‚Ąē :: 0 + k = k)`, instantiated at `k := n`, rewrites the left-hand side to `n`, leaving `n ‚Č§ m + n`. NAT-order's defining equivalence `a ‚Č• b ‚üļ b ‚Č§ a`, instantiated at `(a, b) := (m + n, n)`, then converts this to the stated `m + n ‚Č• n`. The choice of `m, n` was arbitrary, so the conclusion holds universally. ‚ąé

*Left dominance.* Fix `m, n ‚ąą ‚Ąē`. NAT-zero's minimality clause, instantiated at `k := n`, delivers `0 < n ‚ą® 0 = n`; NAT-order's `‚Č§`-definition at `(a, b) := (0, n)` rewrites this disjunction as `0 ‚Č§ n`. NAT-addcompat's left order compatibility `(A m', n', p ‚ąą ‚Ąē : p ‚Č§ n' : m' + p ‚Č§ m' + n')`, instantiated under the renaming `(m', n', p) := (m, n, 0)`, then yields the implication `0 ‚Č§ n ‚üĻ m + 0 ‚Č§ m + n`; modus ponens with the just-established `0 ‚Č§ n` gives `m + 0 ‚Č§ m + n`. NAT-closure's right additive identity `(A k ‚ąą ‚Ąē :: k + 0 = k)`, instantiated at `k := m`, rewrites the left-hand side to `m`, leaving `m ‚Č§ m + n`. NAT-order's `‚Č•`-definition at `(a, b) := (m + n, m)` then converts this to the stated `m + n ‚Č• m`. The choice of `m, n` was arbitrary, so the conclusion holds universally. ‚ąé

*Formal Contract:*
- *Consequence:* `(A m, n ‚ąą ‚Ąē :: m + n ‚Č• n)` (the sum dominates its right operand) ‚ÄĒ derived from NAT-zero, NAT-addcompat (right order compatibility), NAT-closure (left additive identity), and NAT-order as shown in the preceding *Right dominance* prose.
- *Consequence:* `(A m, n ‚ąą ‚Ąē :: m + n ‚Č• m)` (the sum dominates its left operand) ‚ÄĒ derived from NAT-zero, NAT-addcompat (left order compatibility), NAT-closure (right additive identity), and NAT-order as shown in the preceding *Left dominance* prose.
