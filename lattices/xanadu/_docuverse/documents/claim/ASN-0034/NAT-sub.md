**NAT-sub (NatPartialSubtraction).** Subtraction on ‚Ñï is a partial binary operation `‚àí : {(m, n) ‚àà ‚Ñï √ó ‚Ñï : m ‚â• n} ‚Üí ‚Ñï`: whenever `m, n ‚àà ‚Ñï` satisfy `m ‚â• n`, the difference `m ‚àí n` is the unique natural number characterised by `(m ‚àí n) + n = m`.

The axiom slot introduces `‚àí` before constraining it: its first clause `‚àí : {(m, n) ‚àà ‚Ñï √ó ‚Ñï : m ‚â• n} ‚Üí ‚Ñï` posits the signature ‚Äî fixing arity (binary), domain of definition `{(m, n) ‚àà ‚Ñï √ó ‚Ñï : m ‚â• n}`, and codomain ‚Ñï. Positing `‚àí` as a function fixes single-valuedness on its domain: for each `(m, n)` with `m ‚â• n`, `m ‚àí n` denotes a unique element of ‚Ñï.

The following facts about partial subtraction and its interaction with addition and order on ‚Ñï are stated together:

- Conditional closure: `m ‚â• n ‚üπ m ‚àí n ‚àà ‚Ñï` for every `m, n ‚àà ‚Ñï`.
- Right-inverse characterisation: `m ‚â• n ‚üπ (m ‚àí n) + n = m` for every `m, n ‚àà ‚Ñï`.
- Left-inverse characterisation: `m ‚â• n ‚üπ n + (m ‚àí n) = m` for every `m, n ‚àà ‚Ñï`.
- Right telescoping: `(m + n) ‚àí n = m` for every `m, n ‚àà ‚Ñï`.
- Left telescoping: `(n + m) ‚àí n = m` for every `m, n ‚àà ‚Ñï`.

Strict monotonicity ‚Äî `m ‚â• p ‚àß n ‚â• p ‚àß m < n ‚üπ m ‚àí p < n ‚àí p` ‚Äî derives from the right-inverse together with NAT-addcompat's right order compatibility and NAT-order's at-least-one trichotomy with irreflexivity. Assume `m, n, p ‚àà ‚Ñï` with `m ‚â• p`, `n ‚â• p`, and `m < n`. Right-inverse at `(m, p)` and `(n, p)` delivers `(m ‚àí p) + p = m` and `(n ‚àí p) + p = n`; substituting both into `m < n` via indiscernibility of `=` rewrites the hypothesis to `(m ‚àí p) + p < (n ‚àí p) + p`. Abbreviate `a := m ‚àí p` and `b := n ‚àí p` ‚Äî conditional closure places both in ‚Ñï ‚Äî so the rewritten hypothesis reads `a + p < b + p`. NAT-order's at-least-one trichotomy on `(a, b)` presents three cases. The `a < b` case is the desired conclusion `m ‚àí p < n ‚àí p`. The `a = b` case substitutes into `a + p < b + p` via indiscernibility to yield `a + p < a + p`, contradicting NAT-order's irreflexivity at `n := a + p`. The `b < a` case unfolds via NAT-order's `‚â§`-definition to `b ‚â§ a`; NAT-addcompat's right order compatibility applied at antecedent `b ‚â§ a` then delivers `b + p ‚â§ a + p`, which the `‚â§`-definition splits into `b + p < a + p ‚à® b + p = a + p`. The `<` branch, conjoined with the already-established `a + p < b + p`, contradicts exactly-one trichotomy's `¬¨(x < y ‚àß y < x)` clause at `(x, y) := (a + p, b + p)`; the `=` branch substitutes into `a + p < b + p` via indiscernibility to yield `a + p < a + p`, again against irreflexivity. Both disjuncts of the `b < a` case collapse, leaving `a < b`, i.e., `m ‚àí p < n ‚àí p`.

Strict positivity ‚Äî `m > n ‚üπ m ‚àí n ‚â• 1` ‚Äî derives from the right-inverse together with NAT-discrete: lifting `m ‚àí n ‚âÝ 0` to `m ‚àí n ‚â• 1` requires the discreteness fact that no natural number lies strictly between `0` and `1`, which NAT-discrete names. Assume `m, n ‚àà ‚Ñï` with `m > n`. NAT-order's reverse companion `m > n ‚ü∫ n < m` and the `‚â§`-definition deliver `n ‚â§ m`, hence `m ‚â• n`; conditional closure then places `m ‚àí n ‚àà ‚Ñï`. Suppose, for contradiction, that `m ‚àí n = 0`; the right-inverse rewrites `(m ‚àí n) + n = m` to `0 + n = m`, which NAT-closure's left-identity collapses to `n = m`, contradicting NAT-order's exactly-one-trichotomy Consequence's `¬¨(m < n ‚àß m = n)` conjunct at `(m, n) := (n, m)` ‚Äî the conjunction `n < m ‚àß n = m` is ruled out, yet both sides are established (`n < m` from the hypothesis `m > n` unfolded by NAT-order's `m > n ‚ü∫ n < m`, `n = m` from the collapse just derived). Hence `m ‚àí n ‚âÝ 0`. NAT-zero's axiom clause `(A k ‚àà ‚Ñï :: 0 < k ‚à® 0 = k)` at `k := m ‚àí n` then leaves `0 < m ‚àí n` once the equality branch is discarded, and NAT-discrete at `(m, n) := (0, m ‚àí n)` delivers `0 + 1 ‚â§ m ‚àí n`, which NAT-closure's left-identity reduces to `1 ‚â§ m ‚àí n`, i.e., `m ‚àí n ‚â• 1`.

*Formal Contract:*
- *Axiom:* `‚àí : {(m, n) ‚àà ‚Ñï √ó ‚Ñï : m ‚â• n} ‚Üí ‚Ñï` (signature: `‚àí` is a partial binary operation on ‚Ñï, single-valued on its domain of definition); `(A m, n ‚àà ‚Ñï : m ‚â• n : m ‚àí n ‚àà ‚Ñï)` (conditional closure); `(A m, n ‚àà ‚Ñï : m ‚â• n : (m ‚àí n) + n = m)` (right-inverse characterisation); `(A m, n ‚àà ‚Ñï : m ‚â• n : n + (m ‚àí n) = m)` (left-inverse characterisation); `(A m, n ‚àà ‚Ñï :: (m + n) ‚àí n = m)` (right telescoping); `(A m, n ‚àà ‚Ñï :: (n + m) ‚àí n = m)` (left telescoping).
- *Consequence:* `(A m, n, p ‚àà ‚Ñï : m ‚â• p ‚àß n ‚â• p ‚àß m < n : m ‚àí p < n ‚àí p)` (strict monotonicity) ‚Äî derived from the right-inverse clause, NAT-addcompat (right order compatibility), and NAT-order (at-least-one trichotomy, irreflexivity, the `‚â§`-definition, and the exactly-one-trichotomy Consequence's `¬¨(x < y ‚àß y < x)` clause) as shown in the preceding strict-monotonicity prose.
- *Consequence:* `(A m, n ‚àà ‚Ñï : m > n : m ‚àí n ‚â• 1)` (strict positivity) ‚Äî derived from the right-inverse clause, NAT-closure (left additive identity and `1 ‚àà ‚Ñï`), NAT-order (the `>`/`‚â§`/`‚â•` definitions and the exactly-one-trichotomy Consequence's `¬¨(m < n ‚àß m = n)` conjunct at `(m, n) := (n, m)`, contrapositively `n < m ‚üπ n ‚âÝ m`), NAT-zero (`(A k ‚àà ‚Ñï :: 0 < k ‚à® 0 = k)`), and NAT-discrete (discreteness instantiated at `(0, m ‚àí n)`) as shown in the preceding strict-positivity prose.
- *Depends:*
  - NAT-order (NatStrictTotalOrder) ‚Äî supplies the strict order `<` and its companions `‚â§`, `‚â•`, `>` (defined by `m ‚â§ n ‚ü∫ m < n ‚à® m = n`, `m ‚â• n ‚ü∫ n ‚â§ m`, `m > n ‚ü∫ n < m`), used in the signature's domain condition `{(m, n) ‚àà ‚Ñï √ó ‚Ñï : m ‚â• n}` and in the antecedents `m ‚â• n` of the conditional-closure and inverse-characterisation clauses; supplies the at-least-one trichotomy axiom and irreflexivity axiom, together with two conjuncts of the exactly-one-trichotomy Consequence ‚Äî `¬¨(x < y ‚àß y < x)`, against which the strict-monotonicity derivation dispatches the `a = b`, `b < a`-and-`<`, and `b < a`-and-`=` subcases, and `¬¨(m < n ‚àß m = n)` at `(m, n) := (n, m)` (contrapositively `n < m ‚üπ n ‚âÝ m`), against which the strict-positivity derivation contradicts the `m ‚àí n = 0` case.
  - NAT-closure (NatArithmeticClosureAndIdentity) ‚Äî posits `1 ‚àà ‚Ñï` and closes ‚Ñï under addition, so every sum `(m ‚àí n) + n`, `n + (m ‚àí n)`, `m + n`, `n + m` appearing in the inverse-characterisation and telescoping clauses is an ‚Ñï-element; additionally supplies the left-identity `(A k ‚àà ‚Ñï :: 0 + k = k)`, which the strict-positivity derivation invokes twice ‚Äî once to collapse `0 + n = m` to `n = m`, once to collapse `0 + 1 ‚â§ m ‚àí n` to `1 ‚â§ m ‚àí n`.

  - NAT-addcompat (NatAdditionOrderAndSuccessor) ‚Äî supplies right order compatibility `(A m, n, p ‚àà ‚Ñï : p ‚â§ n : p + m ‚â§ n + m)`, instantiated at antecedent `b ‚â§ a` in the strict-monotonicity derivation's `b < a` case to lift `b ‚â§ a` through right-addition by `p` to `b + p ‚â§ a + p`.
