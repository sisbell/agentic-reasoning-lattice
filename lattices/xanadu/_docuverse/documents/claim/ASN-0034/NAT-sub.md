**NAT-sub (NatPartialSubtraction).** Subtraction on ‚Ąē is a partial binary operation `‚ąí : {(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n} ‚Üí ‚Ąē`: whenever `m, n ‚ąą ‚Ąē` satisfy `m ‚Č• n`, the difference `m ‚ąí n` is the unique natural number characterised by `(m ‚ąí n) + n = m`.

The axiom slot introduces `‚ąí` before constraining it: its first clause `‚ąí : {(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n} ‚Üí ‚Ąē` posits the signature ‚ÄĒ fixing arity (binary), domain of definition `{(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n}`, and codomain ‚Ąē. Positing `‚ąí` as a function fixes single-valuedness on its domain: for each `(m, n)` with `m ‚Č• n`, `m ‚ąí n` denotes a unique element of ‚Ąē.

The following facts about partial subtraction and its interaction with addition and order on ‚Ąē are stated together:

- Conditional closure: `m ‚Č• n ‚üĻ m ‚ąí n ‚ąą ‚Ąē` for every `m, n ‚ąą ‚Ąē`.
- Right-inverse characterisation: `m ‚Č• n ‚üĻ (m ‚ąí n) + n = m` for every `m, n ‚ąą ‚Ąē`.
- Left-inverse characterisation: `m ‚Č• n ‚üĻ n + (m ‚ąí n) = m` for every `m, n ‚ąą ‚Ąē`.
- Right telescoping: `(m + n) ‚ąí n = m` for every `m, n ‚ąą ‚Ąē`.
- Left telescoping: `(n + m) ‚ąí n = m` for every `m, n ‚ąą ‚Ąē`.

Strict monotonicity ‚ÄĒ `m ‚Č• p ‚ąß n ‚Č• p ‚ąß m < n ‚üĻ m ‚ąí p < n ‚ąí p` ‚ÄĒ derives from the right-inverse together with NAT-addcompat's right order compatibility and NAT-order's at-least-one trichotomy with irreflexivity. Assume `m, n, p ‚ąą ‚Ąē` with `m ‚Č• p`, `n ‚Č• p`, and `m < n`. Right-inverse at `(m, p)` and `(n, p)` delivers `(m ‚ąí p) + p = m` and `(n ‚ąí p) + p = n`; substituting both into `m < n` via indiscernibility of `=` rewrites the hypothesis to `(m ‚ąí p) + p < (n ‚ąí p) + p`. Abbreviate `a := m ‚ąí p` and `b := n ‚ąí p` ‚ÄĒ conditional closure places both in ‚Ąē ‚ÄĒ so the rewritten hypothesis reads `a + p < b + p`. NAT-order's at-least-one trichotomy on `(a, b)` presents three cases. The `a < b` case is the desired conclusion `m ‚ąí p < n ‚ąí p`. The `a = b` case substitutes into `a + p < b + p` via indiscernibility to yield `a + p < a + p`, contradicting NAT-order's irreflexivity at `n := a + p`. The `b < a` case unfolds via NAT-order's `‚Č§`-definition to `b ‚Č§ a`; NAT-addcompat's right order compatibility applied at antecedent `b ‚Č§ a` then delivers `b + p ‚Č§ a + p`, which the `‚Č§`-definition splits into `b + p < a + p ‚ą® b + p = a + p`. The `<` branch, conjoined with the already-established `a + p < b + p`, contradicts exactly-one trichotomy's `¬¨(x < y ‚ąß y < x)` clause at `(x, y) := (a + p, b + p)`; the `=` branch substitutes into `a + p < b + p` via indiscernibility to yield `a + p < a + p`, again against irreflexivity. Both disjuncts of the `b < a` case collapse, leaving `a < b`, i.e., `m ‚ąí p < n ‚ąí p`.

Strict positivity ‚ÄĒ `m > n ‚üĻ m ‚ąí n ‚Č• 1` ‚ÄĒ derives from the right-inverse together with NAT-discrete: lifting `m ‚ąí n ‚Č† 0` to `m ‚ąí n ‚Č• 1` requires the discreteness fact that no natural number lies strictly between `0` and `1`, which NAT-discrete names. Assume `m, n ‚ąą ‚Ąē` with `m > n`. NAT-order's reverse companion `m > n ‚üļ n < m` and the `‚Č§`-definition deliver `n ‚Č§ m`, hence `m ‚Č• n`; conditional closure then places `m ‚ąí n ‚ąą ‚Ąē`. Suppose, for contradiction, that `m ‚ąí n = 0`; the right-inverse rewrites `(m ‚ąí n) + n = m` to `0 + n = m`, which NAT-closure's left-identity collapses to `n = m`, contradicting NAT-order's exactly-one-trichotomy Consequence's `¬¨(m < n ‚ąß m = n)` conjunct at `(m, n) := (n, m)` ‚ÄĒ the conjunction `n < m ‚ąß n = m` is ruled out, yet both sides are established (`n < m` from the hypothesis `m > n` unfolded by NAT-order's `m > n ‚üļ n < m`, `n = m` from the collapse just derived). Hence `m ‚ąí n ‚Č† 0`. NAT-zero's axiom clause `(A k ‚ąą ‚Ąē :: 0 < k ‚ą® 0 = k)` at `k := m ‚ąí n` then leaves `0 < m ‚ąí n` once the equality branch is discarded, and NAT-discrete at `(m, n) := (0, m ‚ąí n)` delivers `0 + 1 ‚Č§ m ‚ąí n`, which NAT-closure's left-identity reduces to `1 ‚Č§ m ‚ąí n`, i.e., `m ‚ąí n ‚Č• 1`.

*Formal Contract:*
- *Axiom:* `‚ąí : {(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n} ‚Üí ‚Ąē` (signature: `‚ąí` is a partial binary operation on ‚Ąē, single-valued on its domain of definition); `(A m, n ‚ąą ‚Ąē : m ‚Č• n : m ‚ąí n ‚ąą ‚Ąē)` (conditional closure); `(A m, n ‚ąą ‚Ąē : m ‚Č• n : (m ‚ąí n) + n = m)` (right-inverse characterisation); `(A m, n ‚ąą ‚Ąē : m ‚Č• n : n + (m ‚ąí n) = m)` (left-inverse characterisation); `(A m, n ‚ąą ‚Ąē :: (m + n) ‚ąí n = m)` (right telescoping); `(A m, n ‚ąą ‚Ąē :: (n + m) ‚ąí n = m)` (left telescoping).
- *Consequence:* `(A m, n, p ‚ąą ‚Ąē : m ‚Č• p ‚ąß n ‚Č• p ‚ąß m < n : m ‚ąí p < n ‚ąí p)` (strict monotonicity) ‚ÄĒ derived from the right-inverse clause, NAT-addcompat (right order compatibility), and NAT-order (at-least-one trichotomy, irreflexivity, the `‚Č§`-definition, and the exactly-one-trichotomy Consequence's `¬¨(x < y ‚ąß y < x)` clause) as shown in the preceding strict-monotonicity prose.
- *Consequence:* `(A m, n ‚ąą ‚Ąē : m > n : m ‚ąí n ‚Č• 1)` (strict positivity) ‚ÄĒ derived from the right-inverse clause, NAT-closure (left additive identity and `1 ‚ąą ‚Ąē`), NAT-order (the `>`/`‚Č§`/`‚Č•` definitions and the exactly-one-trichotomy Consequence's `¬¨(m < n ‚ąß m = n)` conjunct at `(m, n) := (n, m)`, contrapositively `n < m ‚üĻ n ‚Č† m`), NAT-zero (`(A k ‚ąą ‚Ąē :: 0 < k ‚ą® 0 = k)`), and NAT-discrete (discreteness instantiated at `(0, m ‚ąí n)`) as shown in the preceding strict-positivity prose.
- *Depends:*

  - NAT-order (NatStrictTotalOrder) ‚ÄĒ supplies the strict order `<` and its companions `‚Č§`, `‚Č•`, `>` (defined by `m ‚Č§ n ‚üļ m < n ‚ą® m = n`, `m ‚Č• n ‚üļ n ‚Č§ m`, `m > n ‚üļ n < m`), used in the signature's domain condition `{(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n}` and in the antecedents `m ‚Č• n` of the conditional-closure and inverse-characterisation clauses; supplies the at-least-one trichotomy axiom and irreflexivity axiom, together with two conjuncts of the exactly-one-trichotomy Consequence ‚ÄĒ `¬¨(x < y ‚ąß y < x)`, against which the strict-monotonicity derivation dispatches the `a = b`, `b < a`-and-`<`, and `b < a`-and-`=` subcases, and `¬¨(m < n ‚ąß m = n)` at `(m, n) := (n, m)` (contrapositively `n < m ‚üĻ n ‚Č† m`), against which the strict-positivity derivation contradicts the `m ‚ąí n = 0` case.

  - NAT-addcompat (NatAdditionOrderAndSuccessor) ‚ÄĒ supplies right order compatibility `(A m, n, p ‚ąą ‚Ąē : p ‚Č§ n : p + m ‚Č§ n + m)`, instantiated at antecedent `b ‚Č§ a` in the strict-monotonicity derivation's `b < a` case to lift `b ‚Č§ a` through right-addition by `p` to `b + p ‚Č§ a + p`.
