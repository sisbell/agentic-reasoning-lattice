### Enumeration notation `s‚āĀ < s‚āā < ‚Ä¶ < s_k` is not well-formed at `k = 0`
**Class**: REVISE
**Foundation**: ‚ÄĒ
**ASN**: NAT-card Axiom: "`|S|` is the unique `k ‚ąą ‚Ąē` such that `S` admits a strictly increasing enumeration `s‚āĀ < s‚āā < ‚Ä¶ < s_k` under `<` on ‚Ąē with `S = {s‚āĀ, s‚āā, ‚Ä¶, s_k})` (empty enumeration when `k = 0`, giving `|‚ąÖ| = 0`)". **Issue**: The quantifier ranges over `S ‚äÜ {1, ‚Ä¶, n}` for every `n ‚ąą ‚Ąē`, which includes the case `S = ‚ąÖ` (forced when `n = 0`, permitted when `n ‚Č• 1`). In that case `k = 0`, and the enumeration notation `s‚āĀ < s‚āā < ‚Ä¶ < s_k` does not parse: there is no `s‚āĀ` to anchor the list, no chain of strict inequalities to form, and `{s‚āĀ, ‚Ä¶, s_k}` has no canonical reading as `‚ąÖ` from the notation alone. The parenthetical "(empty enumeration when `k = 0`, giving `|‚ąÖ| = 0`)" states the intended value by convention, but the quantified clause inside the axiom still asserts a predicate whose surface syntax does not resolve at `k = 0`. A downstream consumer reading the axiom alone cannot determine whether the empty-list convention or a case split is in force.
**What needs resolving**: Formalize the enumeration in a shape that is well-defined at `k = 0`. Two standard options: (i) phrase the enumeration as a strictly-increasing function `f : {j ‚ąą ‚Ąē : 1 ‚Č§ j ‚Č§ k} ‚Üí ‚Ąē` with image `S` ‚ÄĒ at `k = 0` the domain is empty, `f` is the empty function, vacuously strictly increasing with image `‚ąÖ`; (ii) split the axiom into the `k = 0` case (`S = ‚ąÖ`, `|S| = 0`) and the `k ‚Č• 1` case (list-notation well-formed). The goal is to make the predicate well-formed without relying on an extra-axiomatic convention for empty lists.
