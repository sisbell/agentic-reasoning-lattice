### T1 case (i) length bounds not discharged at TA1-strict's closing steps
**Foundation**: T1 (LexicographicOrder) case (i) requires, as one of its defining conjuncts, `k ≤ #a ∧ k ≤ #b` — the divergence position must be a shared index of both operands. Per-step citation convention (set by TumblerAdd's own strict-advancement paragraph: "The precondition `k ≤ m` gives `k ≤ #a`, and the result-length identity gives `k ≤ n = #(a ⊕ w)`, so `k ≤ #a ∧ k ≤ #(a ⊕ w)` and both tumblers have a component at position `k`") requires each T1 case (i) invocation to discharge these length bounds explicitly; TumblerAdd's dominance proof also enumerates this explicitly ("*Bound `j ≤ #w`*: ... *Bound `j ≤ #r`*: the result-length identity ...").
**ASN**: TA1-strict's proof closes both cases with bare T1 case (i) invocations. Case 1: "The results agree on all positions before `k` and diverge strictly at `k`. By T1 case (i), `a ⊕ w < b ⊕ w`." Case 2: "The original divergence at position `j` survives intact in the results — the action point, being deeper, does not touch positions at or above `j`. By T1 case (i), `a ⊕ w < b ⊕ w`."
**Issue**: T1 case (i) requires `k ≤ #(a ⊕ w) ∧ k ≤ #(b ⊕ w)` (Case 1, divergence position `k`) and `j ≤ #(a ⊕ w) ∧ j ≤ #(b ⊕ w)` (Case 2, divergence position `j`), but neither is discharged. The bounds do hold — Case 1 via TA0's result-length identity `#(a ⊕ w) = #w` combined with ActionPoint's `k ≤ #w` (and symmetrically for `b ⊕ w`); Case 2 via `j < k ≤ #w = #(a ⊕ w) = #(b ⊕ w)` through NAT-order's transitivity/defining-clause — but the proof does not source them per step, and TA0 is not cited in the Depends list for the length-identity role. The ambient TumblerAdd citation supplies the constructive recall, not the postcondition handle `#(a ⊕ w) = #w` that TA0 exists to label; downstream formalizers mirroring the per-step discipline will find the load-bearing length bounds of the final T1 case (i) step uncited.
**What needs resolving**: Either add explicit length-bound discharges at each T1 case (i) invocation in TA1-strict (Case 1 sourcing `k ≤ #(a ⊕ w) ∧ k ≤ #(b ⊕ w)` through TA0 + ActionPoint; Case 2 sourcing `j ≤ #(a ⊕ w) ∧ j ≤ #(b ⊕ w)` through `j < k ≤ #w` and the result-length identity via NAT-order), extend the Depends list accordingly (adding TA0 for the length-identity role, and NAT-order for the Case 2 transitivity/substitution step if not already covered), or document a convention waiving per-step discharge of T1 case (i) length bounds at this caller and amend TumblerAdd and Divergence's analogous sites to match.
