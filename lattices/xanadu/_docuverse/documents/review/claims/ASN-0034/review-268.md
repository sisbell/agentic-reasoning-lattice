### D0 proof and contract reintroduce primitive `min`/`max` operators the ASN otherwise avoids
**Foundation**: (none — internal consistency.) Divergence's Definition explicitly states case (ii) sub-case values as `#a + 1` (sub-case ii-a, under `#a < #b`) and `#b + 1` (sub-case ii-b, under `#b < #a`), and its Depends narrative emphasizes "stated without invoking any primitive binary-minimum operator on ℕ". TumblerSub's Definition parallels this by naming the result length as `L` via NAT-order trichotomy on `(#a, #w)`, explicitly "rather than by an un-axiomatized `max(#a, #w)`".
**ASN**: D0 (DisplacementWellDefined). *Proof:* "That case requires `#a ≠ #b` with all shared components agreeing, giving `k = min(#a, #b) + 1`. Two sub-cases arise. If `#a < #b`, then `min(#a, #b) = #a` … If `#b < #a`, then `min(#a, #b) = #b` …". *Round-trip boundary:* "The displacement has length `#w = max(#a, #b)`." *Depends:* "TA2 (WellDefinedSubtraction) — establishes `b ⊖ a ∈ T` and `#(b ⊖ a) = max(#a, #b)`". *Postconditions:* "`#(b ⊖ a) = max(#a, #b)`".
**Issue**: D0 is the only place in the ASN that reaches for primitive `min`/`max` on ℕ. Every other property — Divergence, ZPD, TumblerSub, TumblerAdd, D1 — routes these through NAT-order trichotomy sub-cases (α/β/γ or ii-a/ii-b), and the Depends lists make a point of declaring that no binary-min/max operator is consumed. D0 both (a) invokes `min(#a, #b) + 1` instead of Divergence's native sub-case values `#a + 1` / `#b + 1`, and (b) asserts `#(b ⊖ a) = max(#a, #b)` as a result-length claim, conflicting with TumblerSub's postcondition which gives `#(a ⊖ w) = L` via the trichotomy dispatch and lists no `max` operator in its Depends. D1 handles the same subtraction correctly — "NAT-order's trichotomy at `(#b, #a)` selecting sub-case (α) … (β) … (γ)" — so D0 is out of step with its sibling consumer of TumblerSub.
**What needs resolving**: D0 must state its case (ii) elimination in terms of Divergence's sub-cases (ii-a) and (ii-b) at their native sub-case values `#a + 1` and `#b + 1`, and must express the length of `b ⊖ a` via TumblerSub's trichotomy-dispatched `L` at the operand pair `(#b, #a)` rather than as `max(#a, #b)`. The Depends entry that currently cites "TA2 … `#(b ⊖ a) = max(#a, #b)`" must be reconciled with TumblerSub's actual postcondition form, and the Depends list must declare any NAT-order invocations used for the length-pair dispatch it now consumes.
