# Regional Review — ASN-0034/TA-Pos (cycle 7)

*2026-04-22 17:06*

### T0 formal contract conflates "T is the set of finite sequences" with a nonemptiness restriction
**Foundation**: T0 (CarrierSetDefinition)
**ASN**: T0 formal contract: "*Axiom:* T is the set of finite sequences over ℕ, equipped with length `#· : T → ℕ` and component projection `·ᵢ` whose index domain for each `a ∈ T` is `{1, …, #a} ⊆ ℕ`, with `aᵢ ∈ ℕ` at each `i ∈ {1, …, #a}`; and `(A a ∈ T :: 1 ≤ #a)`."
**Issue**: The axiom asserts two things joined by semicolon/"and": (i) *T is* the set of finite sequences over ℕ (definite article, definitional equality — i.e., T equals the set of *all* finite sequences), and (ii) every `a ∈ T` has `1 ≤ #a`. Read literally these clauses are inconsistent: the set of all finite sequences includes the empty sequence `ε` with `#ε = 0`, contradicting the universal. The intended reading is restriction-style ("T is the set of finite sequences satisfying `1 ≤ #a`"), which matches the prose "nonempty finite sequences." But a precise reader cannot tell whether clause (i) is a definition that (ii) then contradicts, or a classification that (ii) restricts. The prior finding asked for `(A a ∈ T :: #a ≥ 1)` to be added; adding it alongside the "set of finite sequences" clause without rewording the first creates a new ambiguity.
**What needs resolving**: State T with a single unambiguous clause — e.g., "T is the set of finite sequences `a` over ℕ with `1 ≤ #a`", or "T is the set of nonempty finite sequences over ℕ" (matching the prose) — so the formal contract no longer reads as two jointly-inconsistent assertions.

### NAT-order's formal-contract Axiom slot never states `<`'s signature
**Foundation**: NAT-order (NatStrictTotalOrder)
**ASN**: NAT-order formal contract: "*Axiom:* `(A n ∈ ℕ :: ¬(n < n))` (irreflexivity); `(A m, n, p ∈ ℕ : m < n ∧ n < p : m < p)` (transitivity); `(A m, n ∈ ℕ :: …)` (trichotomy). The non-strict relation `≤` on ℕ is defined by `m ≤ n ⟺ m < n ∨ m = n`." NAT-order prose: "The binary relation `<` on ℕ is a strict total order…"
**Issue**: The Axiom slot uses `<` in all three constraint clauses but never axiomatizes `<`'s signature (e.g., `< : ℕ × ℕ → {⊤, ⊥}` or "`<` is a binary relation on ℕ"). The signature lives only in prose above and in a Depends-slot note. The prior `+`-signature finding explicitly invoked "the same register NAT-order uses to posit `<`" as the cleaner benchmark, but NAT-order itself does not meet that benchmark — its primitive is introduced by first use in axioms, the same pattern the prior finding faulted. The asymmetry a precise reader sees is that *neither* root primitive is actually introduced at the axiom level.
**What needs resolving**: Either add a signature/introduction clause for `<` to NAT-order's Axiom slot (so the primitive is posited before being constrained), or explicitly state once — in the register the ASN adopts consistently — that primitives are introduced by their first use in axioms and update NAT-closure's treatment of `+` to match. The two roots should use the same rule.

### TA-Pos formal contract mixes formal and prose register across structural slots
**Foundation**: TA-Pos (PositiveTumbler)
**ASN**: TA-Pos formal contract: "*Complementarity:* `(A t ∈ T :: Pos(t) ⟺ ¬Zero(t))`. *Nonvacuity of the partition:* the index range `1 ≤ i ≤ #t` in the Pos/Zero clauses is nonempty for every `t ∈ T`, by T0's `(A a ∈ T :: 1 ≤ #a)`."
**Issue**: The Complementarity bullet is a closed symbolic formula. The Nonvacuity bullet, sitting in the same contract, is an English sentence with embedded justification ("by T0's …"). The formal contract is the slot where the claim's exported content lives; mixing a symbolic assertion with a prose consequence-plus-rationale inside the same slot makes it hard for a mechanical reader (or TLA+-style translator) to extract the claim cleanly. Further, the Nonvacuity statement as written is not a new claim — it is a restatement of T0's nonemptiness axiom applied to the Pos/Zero quantifier ranges — so it either needs to be lifted to a symbolic form parallel to Complementarity or moved out of the formal-contract slot entirely.
**What needs resolving**: Render Nonvacuity in the same symbolic register as Complementarity (e.g., `(A t ∈ T :: {i ∈ ℕ : 1 ≤ i ≤ #t} ≠ ∅)` or equivalent), or remove it from the formal contract and leave the consequence in prose where the justification "by T0's clause" naturally belongs. Every bullet in the formal contract should be a stand-alone claim, stated without embedded derivation.
