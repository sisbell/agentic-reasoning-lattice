### T1 Case 1 uses NAT-discrete's contrapositive direction but Depends restricts its use to Case 3 and part (c)
**Foundation**: NAT-discrete (NatDiscreteness) — T0 announces the axiom content as "`m < n ⟹ m + 1 ≤ n`." T1's Depends entry records the scope precisely: "forward direction `m < n ⟹ m + 1 ≤ n`, used in Case 3 to supply the successor-comparison step and in part (c) Case `k₂ < k₁` case-(ii) branch to obtain `k₂ ≤ m` from `k₂ < m + 1`."
**ASN**: T1 trichotomy, part (b), Case 1: "Then `aᵢ = bᵢ` for all `i` with `1 ≤ i ≤ m ∧ i ≤ n`, and neither `m < n` nor `n < m` holds; by NAT-order's trichotomy at `(m, n)`, `m = n`."
**Issue**: The "no divergence exists" premise of Case 1 directly delivers `¬(m + 1 ≤ n) ∧ ¬(n + 1 ≤ m)` — the divergence definition explicitly lists `k = m + 1 ≤ n` and `k = n + 1 ≤ m` as two of the three divergence shapes being ruled out. To reach "neither `m < n` nor `n < m` holds," the proof must move from `¬(m + 1 ≤ n)` to `¬(m < n)` (and symmetrically for `n`). That move is the contrapositive of NAT-discrete's forward direction: `¬(m + 1 ≤ n) ⟹ ¬(m < n)`. Without this step, trichotomy at `(m, n)` cannot extract `m = n`, because trichotomy only eliminates the two strict disjuncts when each has been independently ruled out. T1's Depends entry for NAT-discrete lists only Case 3 and part (c) as use-sites, so the Case 1 use-site is unaccounted for and the scope clause under-reports how NAT-discrete is consumed.
**What needs resolving**: Either extend T1's Depends entry for NAT-discrete to name the Case 1 use (the `¬(m + 1 ≤ n) ⟹ ¬(m < n)` step that bridges "no exhaustion-shape divergence" with "neither strict inequality holds"), or rewrite Case 1 so that `m = n` is reached without invoking the forward-direction contrapositive — e.g., by discharging `¬(m < n)` and `¬(n < m)` through content already listed for Case 1 (NAT-order trichotomy plus the divergence-definition disjunction structure).
