# Sub-Questions — Link Survivability

**Inquiry:** What guarantees does the system provide about links across state changes? What survives, what changes, and what can a link holder rely on?

1. [nelson] When content that a link points to is edited — words inserted before or after the linked passage — must the link continue to address exactly the same content it originally addressed?
2. [nelson] If a linked passage is deleted from a document, what must the system guarantee about the link itself — does it persist, become invalid, or enter some other state?
3. [nelson] Must a link created against one version of a document remain meaningful when the document reaches a later version, or is a link bound to the version in which it was made?
4. [nelson] Can the creator of a link ever destroy it, or does the system guarantee that once a link exists it is permanent in the same way that content is permanent?
5. [nelson] When a document is reorganized — passages reordered or moved to different positions — must links to those passages track the content rather than the location?
6. [nelson] If two independent links point to the same passage and that passage is later split across two documents, must both links survive and continue to reach their original content?
7. [nelson] What can a link holder rely on about the identity of the endpoints — must the system guarantee that the content at each end is exactly what was there when the link was created, or only that the addresses remain valid?
8. [nelson] Must the system preserve the directionality and endpoint ordering of a link across all state changes, or can editing operations ever alter which end is the source and which is the target?
9. [nelson] When new content is inserted at the exact boundary of a linked passage, must the system have a definite rule about whether the link expands to include the insertion or excludes it?
10. [nelson] If a link connects passages owned by two different users and one user's document undergoes extensive revision, what guarantee does the other user have that their end of the link remains intact and reachable?
11. [gregory] When a link's source endset spans I-addresses `[i, i+5)` and a DELETE removes the content at `[i+2, i+4)` from the only document containing it, does `RETRIEVEENDSETS` for that link still return the full original I-span `[i, i+5)`, or does it return only the portions `[i, i+2)` and `[i+4, i+5)` that remain reachable through some POOM?
12. [gregory] If a link L references content at I-addresses `[i, i+3)`, that content is deleted from all documents (making L orphaned), and then a COPY transcludes just `[i+1, i+2)` (a subset of the original endset) into a new document, does `find_links` on that new document rediscover L — and if so, does the returned endset show only the single transcluded byte or the full original three-byte span?
13. [gregory] After REARRANGE with a 3-cut pivot splits a contiguous endset span across two non-adjacent V-regions in the same document, does `RETRIEVEENDSETS` followed by I-to-V conversion via `span2spanset` return a two-VSpec specset reflecting both V-positions, or does the conversion lose one of the fragments?
14. [gregory] When CREATENEWVERSION copies only text subspace `1.x` and not links from subspace `2.x`, is the link's own I-address (its orgl at `docISA.0.2.N`) excluded from the version's POOM — and if so, does `RETRIEVEDOCVSPANSET` on the version show zero entries in the link subspace even though `find_links` discovers the original's links through shared text I-addresses?
15. [gregory] In a multi-session scenario where session A creates a link L targeting content in document D, and session B simultaneously deletes that content from D before A's operation completes — given run-to-completion atomicity, is there any window where L's spanfilade entries could reference I-addresses that were never actually present in D's POOM, or does the serialized execution guarantee consistency between link creation and POOM state?
16. [gregory] When a link's endset references I-addresses shared by three documents (original, version, and transclusion target), and the original document deletes that content, does `find_links` still discover the link from both the version and the transclusion target independently — confirming that link discovery is purely I-address intersection against the spanfilade with no provenance tracking back to the creating document?
17. [gregory] If DELETEVSPAN is applied to a link's V-position in subspace `2.x` (creating a "reverse orphaned" link), and then new content is inserted whose I-addresses happen to overlap with the deleted link's endset I-spans, does `find_links` on that new content still discover the reverse-orphaned link through the spanfilade — even though the link has no POOM entry in any document?
18. [gregory] When INSERT at position V:1.3 in a document shifts text content rightward via `makegappm`, and a link L has source endset referencing I-addresses of that shifted content, does `FOLLOWLINK` on L resolve the endset to the new V-positions (post-shift) by performing a fresh I-to-V lookup through the POOM, or does it cache any V-position from link creation time?
19. [gregory] Does the spanfilade entry for a link's endset record the document ID where the link was created, and if the link's home document is itself deleted from the POOM (all content removed), do the spanfilade endset entries for that link remain fully intact — meaning the link is discoverable from any other document sharing those I-addresses regardless of the creating document's state?
20. [gregory] When a link has a source endset spanning I-addresses `[i, i+10)` and a COPY transcludes only `[i+3, i+7)` into a target document, does `find_links` on the target document match L because the partial I-address overlap is sufficient, and does the returned endset for L show all three endsets (source, target, type) in full or only the overlapping portion of the queried endset?
