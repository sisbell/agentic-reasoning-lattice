### Subtraction on ℕ used as a primitive without NAT-* attribution

**Foundation**: (foundation ASN — internal consistency; the ASN's per-step citation policy for ℕ facts)
**ASN**: TumblerSub's definition writes `rₖ = aₖ − wₖ` at the divergence point, and the Depends for TumblerSub does not cite any NAT-* axiom; downstream properties inherit this. TumblerSub's precondition consequence argues *"rₖ = aₖ − wₖ ∈ ℕ since a ≥ w entails aₖ > wₖ at the zpd point"*, using conditional subtraction on ℕ as if it were a primitive operation. D0 writes *"wₖ = bₖ − aₖ (well-defined since bₖ > aₖ)"*, D1 and D2 repeat the same, TA3 uses it in every sub-case (e.g., *"(a ⊖ w)_d = a_d − w_d > 0"*), and TA4 uses `rₖ - wₖ` in Case 1. NAT-closure's axiom lists only successor closure, addition closure, and additive identity — not a subtraction clause stating `m ≥ n ⟹ m − n ∈ ℕ` or giving a constructive definition of `−` on ℕ.
**Issue**: The ASN axiomatizes ℕ carefully — closure under successor and addition (NAT-closure), strict total order (NAT-order), discreteness (NAT-discrete), order-compatibility (NAT-addcompat), well-ordering (NAT-wellorder), associativity (NAT-addassoc), cancellation (NAT-cancel) — but never states the partial subtraction operation that TumblerSub and its downstream consumers all invoke. The convention set by T0 ("each proof cites only the ℕ facts it actually uses") is applied scrupulously to every other ℕ fact, yet the subtraction step that appears in roughly a dozen proofs has no axiomatic source. A reader reconstructing these proofs from the NAT-* suite as stated cannot discharge `aₖ − wₖ ∈ ℕ`.
**What needs resolving**: Either add a NAT-* axiom supplying conditional subtraction on ℕ (and have TumblerSub, TA3, TA4, D0, D1, D2, and others cite it in their Depends), or construct `−` inside the proofs that use it via existing axioms and show the step explicitly. As written, the citation policy for ℕ facts is broken for the subtraction primitive.

### GlobalUniqueness Case 5 uses implicit ℕ-inequality cancellation without attribution

**Foundation**: (foundation ASN — internal consistency; per-step NAT-* citation convention)
**ASN**: GlobalUniqueness Case 5's length-collision argument: *"Suppose for contradiction that `#p₁ + k'₁ = #p₂ + k'₂`. Since p₁ is a proper prefix of p₂, #p₁ < #p₂, so k'₁ > k'₂; with both values in {1, 2}, the only possibility is k'₁ = 2 and k'₂ = 1..."* The inference "from `a + x = b + y ∧ a < b` conclude `x > y`" is a non-trivial composition on ℕ: it requires either right-cancellation combined with order-compatibility of `<`, or a direct cancellation-of-inequalities axiom. GlobalUniqueness's Depends clause lists AllocatedSet, T9, T1, T10, T10a, T10a.1, T10a.3, T10a.4, T3, T4, T4a, TA5, TA5-SigValid, Prefix — no NAT-* axioms are cited, despite the step being a pure ℕ-arithmetic inference.
**Issue**: The ASN's stated convention (T0) is that each proof cites only the ℕ facts it actually uses. Sister proofs that use ℕ steps of comparable complexity — T10a-N's `#t₁ < #t₂`, TA1's `aₖ + wₖ < bₖ + wₖ`, TS3's `(vₘ + n₁) + n₂ = vₘ + (n₁ + n₂)` — cite the appropriate NAT-addcompat, NAT-addassoc, or NAT-order axioms by name. GlobalUniqueness's Case 5 uses a composed ℕ step of the same flavor but omits every citation. A reviser who tightens NAT-addcompat or NAT-cancel has no Depends-backed signal that GlobalUniqueness's Case 5 consumes those facts.
**What needs resolving**: GlobalUniqueness's Depends must be extended to cite the NAT-* axioms that Case 5's length-collision step rests on (NAT-addcompat, NAT-cancel, NAT-order, or whatever combination underwrites the `a + x = b + y ∧ a < b ⟹ x > y` inference), or the step must be reworked to avoid implicit ℕ-arithmetic. As written, the convention established for T1, TumblerAdd, TA5, and T10a-N is not applied here.

### Symbol collision: `S` as set of positive ordinals vs. `S` as component-value placeholder in OrdinalShift

**Foundation**: (internal consistency)
**ASN**: Vocabulary: *"**S** — set of ordinals with all positive components: S = {o ∈ T : #o ≥ 1 ∧ (A i : 1 ≤ i ≤ #o : oᵢ > 0)}"*. TA7a repeats this definition formally: *"Define **S** = {o ∈ T : #o ≥ 1 ∧ ...}"* and uses S throughout its postconditions and proof. AllocatedSet and NoDeallocation even explicitly flag the name collision between this set `S` and the state space `𝒮`, going so far as to introduce script form specifically to keep them distinct. OrdinalShift's proof then writes: *"When m = 1, shift([S], n) = [S + n] changes the first component."* Here `[S]` is meant as a length-1 tumbler whose sole component is some arbitrary natural number, with `S` functioning as a placeholder meta-variable — the same symbol that the Vocabulary and TA7a define as the specific set of positive-component ordinals.
**Issue**: A reader encountering `shift([S], n) = [S + n]` has no cue that `S` here differs from the Vocabulary's `S`. Since the Vocabulary entry for `S` defines it as a set (not a component value), `[S]` under the Vocabulary reading is a tumbler whose component is a set — type-incoherent — and `[S + n]` is then nonsensical. The ASN has already demonstrated awareness of this collision (adopting `𝒮` to distinguish the state space from `S`), so the reuse inside OrdinalShift is a silent reintroduction of the same category error at a different call site.
**What needs resolving**: OrdinalShift's proof must use a different symbol for the arbitrary component value (conventional choices: `c`, `x`, or lowercase `s`), matching the care already applied to the state-space symbol. Reusing `S` inside an expression where the Vocabulary reads it as a set undermines the uniform-meaning discipline established elsewhere in the ASN.
