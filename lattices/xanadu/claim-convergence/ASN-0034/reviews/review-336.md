### ActionPoint labels a direct NAT-discrete implication as its "contrapositive"
**Foundation**: T0 enumerates NAT-discrete's content as "discreteness, **including** `m < n ⟹ m + 1 ≤ n`" — presenting this implication as direct content of the axiom, not as a derived contrapositive.
**ASN**: ActionPoint proof body: "NAT-discrete's contrapositive `m < n ⟹ m + 1 ≤ n` at `m = 0, n = w_{actionPoint(w)}` yields `0 + 1 ≤ w_{actionPoint(w)}`." ActionPoint's Depends repeats the label: "NAT-discrete (NatDiscreteness) — contrapositive `m < n ⟹ m + 1 ≤ n`." T1's Depends, by contrast, calls exactly the same implication the "forward direction `m < n ⟹ m + 1 ≤ n`."
**Issue**: "Contrapositive" has a fixed logical meaning: the contrapositive of `m < n ⟹ m + 1 ≤ n` is `¬(m + 1 ≤ n) ⟹ ¬(m < n)`, i.e., `m + 1 > n ⟹ m ≥ n` — not the original implication. T0 presents `m < n ⟹ m + 1 ≤ n` as direct NAT-discrete content, and T1 cites it as "forward direction." ActionPoint's "contrapositive" label contradicts both T0's framing and T1's citation of the same implication, and leaves a reader uncertain whether ActionPoint believes NAT-discrete's primary statement is something else (from which `m < n ⟹ m + 1 ≤ n` would be derived by contraposition). Without a NAT-discrete claim body in the ASN (see prior finding), the reader cannot resolve the disagreement by reading the axiom directly.
**What needs resolving**: Reconcile the citation label — either ActionPoint should call the implication by the same name T0 and T1 use (direct/forward), or NAT-discrete's stated form must be written as the converse implication `m + 1 > n ⟹ m ≥ n` so that "contrapositive" accurately names the rewrite ActionPoint performs.
