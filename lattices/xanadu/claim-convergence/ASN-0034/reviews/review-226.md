# Cone Review — ASN-0034/TA1-strict (cycle 6)

*2026-04-18 02:45*

### Divergence's contract block lacks an explicit *Preconditions* field
**Foundation**: TumblerAdd, TA0, ActionPoint, TA1-strict all factor their hypotheses into a labelled `*Preconditions:*` field separate from the Definition (e.g., TumblerAdd: "*Preconditions:* a ∈ T, w ∈ T, Pos(w) (TA-Pos), actionPoint(w) ≤ #a (ActionPoint)"; TA0 likewise; ActionPoint's "*Preconditions:* w ∈ T, Pos(w)").
**ASN**: Divergence's Formal Contract has only `*Definition:*`, `*Depends:*`, `*Postconditions:*`. The hypotheses `a, b ∈ T` and `a ≠ b` appear inside the Definition string ("For `a, b ∈ T` with `a ≠ b`: …") but never as a labelled precondition list. Callers — including TA1-strict, which goes to some length to discharge Divergence's `a ≠ b` requirement at the call site — must inspect the Definition prose to recover what TumblerAdd/TA0/ActionPoint expose in a uniform slot.
**Issue**: The structural slot a caller (or mechanical contract-translator) reads to discover what to prove before invoking `divergence(a, b)` is missing for Divergence alone among the operator-introducing properties of this ASN. TA1-strict's Depends now enumerates the irreflexivity-mediated discharge of `a ≠ b` "for Divergence's Definition-level precondition" — naming the field "Definition-level precondition" precisely because Divergence has no Preconditions field for it to live in. This works around the gap rather than closing it; downstream consumers writing their own discharge text must each rediscover that the precondition lives in the Definition body.
**What needs resolving**: Either lift Divergence's hypotheses (`a ∈ T`, `b ∈ T`, `a ≠ b`) into a `*Preconditions:*` field matching the TumblerAdd / TA0 / ActionPoint / TA1-strict structure, or document a convention (and amend the call-side phrasing in TA1-strict and any other consumer of `divergence(·,·)`) under which function-introducing Definitions may omit the Preconditions field and have the typing/non-equality conditions read off the Definition string.

### TA1-strict's case-(ii) rule-out leaves the `≤` + `<` → `<` composition uncited
**Foundation**: NAT-order axiomatises irreflexivity, transitivity of `<`, and trichotomy; the non-strict `≤` is *defined* as `m < n ∨ m = n`. Composing a non-strict premise (`a ≤ b`) with a strict one (`b < c`) into a strict conclusion (`a < c`) is therefore a derived step that must unpack `≤` via NAT-order's defining clause and then route each disjunct through transitivity (strict disjunct) or substitution (equality disjunct) — a pattern the ASN cites in detail elsewhere (e.g., TumblerAdd's strict-advancement chain "NAT-order's defining clause … unfolds the `≥` into … the first disjunct chains with … through NAT-order's transitivity of `<` … the second disjunct substitutes …").
**ASN**: TA1-strict, opening, case-(ii) rule-out: "combining `k ≥ j = #a + 1` with the precondition `k ≤ #a` gives `#a + 1 ≤ #a`; NAT-addcompat's strict successor inequality (`n < n + 1`, instantiated at `n = #a`) gives `#a < #a + 1`; **chaining these produces `#a + 1 ≤ #a < #a + 1`**, ruled out by NAT-order's irreflexivity (`¬(n < n)`, instantiated at `n = #a + 1`)." TA1-strict's Depends sub-role (ii*) for NAT-order says only "Irreflexivity (`¬(n < n)`) instantiated at `n = #a + 1` … discharges the contradictions … formed from the precondition `k ≤ #a` … and NAT-addcompat's strict successor inequality at `n = #a`."
**Issue**: Irreflexivity at `n = #a + 1` is `¬(#a + 1 < #a + 1)`. The chain as written delivers `#a + 1 ≤ #a < #a + 1`, not `#a + 1 < #a + 1` — the bridge from the former to the latter is the uncited step. Under the per-step discipline this ASN enforces (and that NAT-order's defining clause is named for at three other sites in the same Depends entry: the Case 1 strict-promotion sub-roles, Case 2's `j ≤ #w` weakening, etc.), the case-(ii) rule-out's composition needs either (a) NAT-order's defining clause (to unfold `#a + 1 ≤ #a` into the disjunction) plus NAT-order's transitivity of `<` (for the `#a + 1 < #a` disjunct) or trichotomy at `(#a, #a + 1)` (to exclude `#a + 1 < #a` once `#a < #a + 1` is fixed), and (b) substitution of `#a + 1 = #a` into `#a < #a + 1` (for the equality disjunct). Sub-role (ii*) names none of these. The mirror branch at `n = #b` inherits the same gap.
**What needs resolving**: Extend TA1-strict's NAT-order Depends sub-role (ii*) to enumerate the defining-clause unfolding and transitivity/substitution sub-steps that compose `#a + 1 ≤ #a` with `#a < #a + 1` into the strict `#a + 1 < #a + 1` consumed by irreflexivity (and the symmetric pair at `n = #b`) — or rephrase the rule-out's chain so that the contradiction is read directly off named axiom instances rather than off an implicit `≤`-with-`<` composition.
