### TS1's quantifier binding lacks type annotations present in sister properties
**Foundation**: TA1-strict's property statement types its bound variables inline: `(A a, b, w ∈ T : a < b ∧ Pos(w) ∧ actionPoint(w) ≤ #a ∧ actionPoint(w) ≤ #b ∧ actionPoint(w) ≥ divergence(a, b) : a ⊕ w < b ⊕ w)`. Every universally quantified variable receives its domain (`∈ T`) at the quantifier header, so the body's symbols `#a`, `aᵢ`, `a < b`, `Pos(w)` are well-formed as soon as the quantifier is read.
**ASN**: TS1's property statement: `(A v₁, v₂, n : n ≥ 1 ∧ #v₁ = #v₂ ∧ v₁ < v₂ : shift(v₁, n) < shift(v₂, n))`. No variable is typed. The body uses `#v₁`, `v₁ < v₂`, and `shift(v₁, n)` — each presupposing `v₁, v₂ ∈ T` for well-formedness, and `n ∈ ℕ` for the precondition `n ≥ 1` to be meaningful — but neither the quantifier header nor a preceding clause places these variables in their domains. The Formal Contract's Preconditions list adds `v₁ ∈ T, v₂ ∈ T, n ≥ 1`, but the property statement itself stands without this.
**Issue**: Under strict reading, the operators `#·`, `<`, and `shift(·, ·)` in the property body are only defined when their arguments lie in `T` (and `n` in ℕ); the property as stated does not establish this, so the body's symbols are ill-formed at the point the quantifier is read. Every other property and definition in this ASN that binds tumbler-valued variables at a quantifier — TA1-strict, Divergence's case (i) inner quantifier `(A i : 1 ≤ i < k : aᵢ = bᵢ)`, ActionPoint's inner minimum-set quantifier — carries either an `∈ T` type or a range-bound guard that makes the body well-formed. TS1 breaks this pattern at the ASN's top-level property statement.
**What needs resolving**: TS1's property statement must type its bound variables — either `(A v₁, v₂ ∈ T, n ∈ ℕ : ...)` matching TA1-strict's `∈ T` inline typing, or a clearly equivalent mechanism that places `v₁, v₂` in `T` and `n` in ℕ before the body symbols are used.

### TS1 Depends claims NAT-order trichotomy forecloses length-pair branches, but the proof step is pure equality refutation
**Foundation**: Per-step citation discipline: Depends entries must accurately describe the axiom consumed at each proof step. NAT-order's trichotomy asserts that exactly one of `m < n`, `m = n`, `n < m` holds; it is consumed when a proof performs a three-way case split or uses exclusivity to foreclose one side given another.
**ASN**: TS1 proof at precondition check (viii) reads: "since #v₁ = #v₂ by hypothesis, Divergence case (ii) (prefix divergence) is excluded: it requires #v₁ ≠ #v₂." TS1 Depends' NAT-order entry, sub-role (c), describes this step as: "NAT-order's trichotomy at the length pair `(#v₁, #v₂)` — its exactly-one clause — forecloses the strict branches `#v₁ < #v₂` and `#v₂ < #v₁` against the hypothesis `#v₁ = #v₂`, leaving only the `#v₁ = #v₂` branch; since Divergence case (ii) requires `#v₁ ≠ #v₂` at its precondition, this foreclosure renders case (ii) inapplicable and commits the proof to Divergence case (i)."
**Issue**: The proof step shown performs no trichotomy: it substitutes the hypothesis `#v₁ = #v₂` into case (ii)'s precondition `#v₁ ≠ #v₂` and observes direct logical contradiction. Foreclosing `#v₁ < #v₂` and `#v₂ < #v₁` branches is not needed — case (ii) is ruled out by the equality directly, without appeal to NAT-order's exactly-one clause. The Depends entry describes an inference route that the proof does not execute, violating the per-step citation discipline this ASN enforces ("every operator consumed at a step be sourced at that step"). A reviser reading NAT-order (c) expects to find a trichotomy case-split at (viii), but the proof text has none.
**What needs resolving**: Either the TS1 proof at (viii) must actually perform the trichotomy case-split that Depends describes (so the sub-role (c) citation reflects an executed step), or Depends' sub-role (c) must be rewritten to describe what the proof actually does — a direct refutation of case (ii)'s precondition `#v₁ ≠ #v₂` from the hypothesis `#v₁ = #v₂`, sourced to whichever foundation (if any) licenses `≠`-refutation from equality.

### OrdinalShift's Formal Contract Postconditions reference `m` outside the Definition's binding
**Foundation**: Precise specification requires every symbol in a contract's Postconditions to be bound, either by the Preconditions, by a constant declared in the contract, or by the Definition in a way that carries scope to the Postconditions. TS1's own fix of the earlier free-`m` issue (per previous finding #9) removed `m` from TS1's Preconditions and property statement for precisely this reason.
**ASN**: OrdinalShift's *Formal Contract*:
- *Preconditions:* `v ∈ T, n ≥ 1`
- *Definition:* `shift(v, n) = v ⊕ δ(n, m) where m = #v`
- *Postconditions:* `shift(v, n) ∈ T, #shift(v, n) = #v, shift(v, n)ᵢ = vᵢ for i < m, shift(v, n)ₘ = vₘ + n ≥ 1`

The last two postconditions — `shift(v, n)ᵢ = vᵢ for i < m` and `shift(v, n)ₘ = vₘ + n ≥ 1` — mention `m`. The Preconditions do not bind `m`. The Definition clause introduces `m = #v` with the `where` scope conventionally limited to the Definition's right-hand side; nothing in the contract extends that binding to the Postconditions.
**Issue**: A reader of OrdinalShift's contract encounters two postconditions in which `m` is a free identifier. The intended reading is "`m = #v`, inherited from the Definition," but this inheritance is not made explicit. Under the strict-reading standard TS1's free-`m` finding applied to TS1, OrdinalShift has the same issue at its own level: `m` is used in the Postconditions without a contract-scope binding. The Postconditions cannot be consumed at a downstream call site (as TS1 would if it needed them) without the reader reconstructing `m = #v` from the Definition.
**What needs resolving**: OrdinalShift must either rewrite its Postconditions to eliminate `m` (e.g., `shift(v, n)ᵢ = vᵢ for i < #v`, `shift(v, n)_{#v} = v_{#v} + n ≥ 1`), or explicitly extend `m`'s binding to the Postcondition scope (e.g., introducing `m = #v` as a contract-level abbreviation rather than a Definition-local one), matching the scoping rigor TS1 adopted when it removed the free `m`.
