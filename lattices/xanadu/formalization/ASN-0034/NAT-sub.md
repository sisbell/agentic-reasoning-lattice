**NAT-sub (NatPartialSubtraction).** Subtraction on ℕ is a partial binary operation `− : {(m, n) ∈ ℕ × ℕ : m ≥ n} → ℕ`: whenever `m, n ∈ ℕ` satisfy `m ≥ n`, the difference `m − n` is the unique natural number characterised by `(m − n) + n = m`.

The axiom slot introduces `−` before constraining it: its first clause `− : {(m, n) ∈ ℕ × ℕ : m ≥ n} → ℕ` posits the signature — fixing arity (binary), domain of definition `{(m, n) ∈ ℕ × ℕ : m ≥ n}`, and codomain ℕ. The operation is posited directly as a partial function on ℕ rather than derived from an earlier axiom — the same register NAT-closure uses to posit `+ : ℕ × ℕ → ℕ` and NAT-order uses to posit `< ⊆ ℕ × ℕ`. Single-valuedness on the domain of definition is carried by the function signature itself, so the uniqueness claimed above — that `m − n` is *the* unique natural number characterised by `(m − n) + n = m` — is established by positing `−` as a function rather than by derivation from right-cancellation of `+`, which no NAT foundation declares. The conditional-closure clause `m ≥ n ⟹ m − n ∈ ℕ` enumerated below is subsumed by the signature's codomain commitment; it is retained as a standalone bullet so that downstream claims can cite "NAT-sub conditional closure" by name when they need codomain membership without invoking the full inverse characterisation.

The following facts about partial subtraction and its interaction with addition and order on ℕ are stated together:

- Conditional closure: `m ≥ n ⟹ m − n ∈ ℕ` for every `m, n ∈ ℕ`.
- Right-inverse characterisation: `m ≥ n ⟹ (m − n) + n = m` for every `m, n ∈ ℕ`.
- Left-inverse characterisation: `m ≥ n ⟹ n + (m − n) = m` for every `m, n ∈ ℕ`.
- Strict positivity: `m > n ⟹ m − n ≥ 1` for every `m, n ∈ ℕ`.
- Right telescoping: `(m + n) − n = m` for every `m, n ∈ ℕ`.
- Strict monotonicity: `m ≥ p ∧ n ≥ p ∧ m < n ⟹ m − p < n − p` for every `m, n, p ∈ ℕ`.

Both inverse forms are stated because citing either one without commutativity of addition would otherwise be tacit.

The right-telescoping clause `(m + n) − n = m` is stated unconditionally — without an explicit `m + n ≥ n` antecedent — because that bound holds for every `m, n ∈ ℕ`, so the conditional-closure precondition `m + n ≥ n` (required by this same axiom for `(m + n) − n` to denote an ℕ-element) is universally satisfied. The universal `(A m, n ∈ ℕ :: m + n ≥ n)` is established by NAT-addbound (NatAdditionDominatesRightOperand) and is therefore declared in the Depends slot; without it, the right-telescoping equality would assert an equation whose left-hand side is not guaranteed to be ℕ-valued.

The axiom body invokes symbols beyond ℕ's primitive membership. The strict order `<` together with its non-strict companion `≤` and reverse companions `≥`, `>` — all defined in NAT-order by `m ≤ n ⟺ m < n ∨ m = n`, `m ≥ n ⟺ n ≤ m`, `m > n ⟺ n < m` — appear in the signature's domain condition `{(m, n) ∈ ℕ × ℕ : m ≥ n}`, in the antecedents `m ≥ n`, `m > n`, `m ≥ p ∧ n ≥ p ∧ m < n` of the conditional-closure, inverse-characterisation, strict-positivity, and strict-monotonicity clauses, and in the consequents `m − n ≥ 1` and `m − p < n − p`. The binary addition `+` closed over ℕ by NAT-closure appears in the sums `(m − n) + n`, `n + (m − n)`, and `m + n`, and the numeral `1 ∈ ℕ` posited by NAT-closure grounds the lower bound of `m − n ≥ 1`. The right-telescoping clause's well-definedness is supplied by NAT-addbound's universal `m + n ≥ n`, which discharges the conditional-closure precondition implicit in the unguarded equation `(m + n) − n = m`. All three foundations are declared in the Depends slot so that the axiom body can be read without silently importing them.

*Formal Contract:*
- *Axiom:* `− : {(m, n) ∈ ℕ × ℕ : m ≥ n} → ℕ` (signature: `−` is a partial binary operation on ℕ, posited as a function — hence single-valued — on its domain of definition); `(A m, n ∈ ℕ : m ≥ n : m − n ∈ ℕ)` (conditional closure, subsumed by the signature's codomain commitment and retained for downstream citation); `(A m, n ∈ ℕ : m ≥ n : (m − n) + n = m)` (right-inverse characterisation); `(A m, n ∈ ℕ : m ≥ n : n + (m − n) = m)` (left-inverse characterisation); `(A m, n ∈ ℕ : m > n : m − n ≥ 1)` (strict positivity); `(A m, n ∈ ℕ :: (m + n) − n = m)` (right telescoping); `(A m, n, p ∈ ℕ : m ≥ p ∧ n ≥ p ∧ m < n : m − p < n − p)` (strict monotonicity).
- *Depends:*
  - NAT-order (NatStrictTotalOrder) — supplies the strict order `<` and its companions `≤`, `≥`, `>` (defined by `m ≤ n ⟺ m < n ∨ m = n`, `m ≥ n ⟺ n ≤ m`, `m > n ⟺ n < m`), used in the signature's domain condition `{(m, n) ∈ ℕ × ℕ : m ≥ n}`, in the antecedents `m ≥ n`, `m > n`, `m ≥ p ∧ n ≥ p ∧ m < n`, and in the consequents `m − n ≥ 1` and `m − p < n − p`.
  - NAT-closure (NatArithmeticClosureAndIdentity) — posits `1 ∈ ℕ` and closes ℕ under addition, so the literal `1` appearing in the strict-positivity consequent `m − n ≥ 1` is grounded and every sum `(m − n) + n`, `n + (m − n)`, `m + n` appearing in the inverse-characterisation and right-telescoping clauses is an ℕ-element.
  - NAT-addbound (NatAdditionDominatesRightOperand) — supplies the universal `(A m, n ∈ ℕ :: m + n ≥ n)`, which discharges the conditional-closure precondition `m + n ≥ n` implicit in the right-telescoping clause `(m + n) − n = m`; without this lift, `(m + n) − n` would not be guaranteed to denote an ℕ-element via the conditional-closure clause and the equality would be vacuous on its left-hand side.
