**NAT-sub (NatPartialSubtraction).** Subtraction on ‚Ąē is a partial binary operation `‚ąí : {(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n} ‚Üí ‚Ąē`: whenever `m, n ‚ąą ‚Ąē` satisfy `m ‚Č• n`, the difference `m ‚ąí n` is the unique natural number characterised by `(m ‚ąí n) + n = m`.

The axiom slot introduces `‚ąí` before constraining it: its first clause `‚ąí : {(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n} ‚Üí ‚Ąē` posits the signature ‚ÄĒ fixing arity (binary), domain of definition `{(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n}`, and codomain ‚Ąē. Positing `‚ąí` as a function fixes single-valuedness on its domain: for each `(m, n)` with `m ‚Č• n`, `m ‚ąí n` denotes a unique element of ‚Ąē.

The following facts about partial subtraction and its interaction with addition and order on ‚Ąē are stated together:

- Conditional closure: `m ‚Č• n ‚üĻ m ‚ąí n ‚ąą ‚Ąē` for every `m, n ‚ąą ‚Ąē`.
- Right-inverse characterisation: `m ‚Č• n ‚üĻ (m ‚ąí n) + n = m` for every `m, n ‚ąą ‚Ąē`.
- Left-inverse characterisation: `m ‚Č• n ‚üĻ n + (m ‚ąí n) = m` for every `m, n ‚ąą ‚Ąē`.
- Right telescoping: `(m + n) ‚ąí n = m` for every `m, n ‚ąą ‚Ąē`.

The right-inverse and left-inverse characterisations are stated as independent clauses because the NAT-* axioms of this ASN do not declare commutativity of addition on ‚Ąē ‚ÄĒ without `m + n = n + m`, neither form is derivable from the other. Both-sided coverage is the foundation's deliberate substitute for the absent commutativity axiom: downstream consumers that need to collapse `n + (m ‚ąí n)` to `m` cite the left-inverse directly rather than routing through the right-inverse, matching the same design choice NAT-cancel makes for its left/right cancellation pair and NAT-addcompat makes for its left/right compatibility pair.

Strict monotonicity ‚ÄĒ `m ‚Č• p ‚ąß n ‚Č• p ‚ąß m < n ‚üĻ m ‚ąí p < n ‚ąí p` ‚ÄĒ is exported as a *Consequence:* rather than an additional axiom clause, because its content derives from the right-inverse together with NAT-addcompat's right order compatibility and NAT-order's at-least-one trichotomy with irreflexivity. Retaining it as an axiom clause would launder that derivation through a non-minimal clause, the same concern that relocated NAT-order's disjointness form `m < n ‚üĻ m ‚Č† n` to a Consequence. Assume `m, n, p ‚ąą ‚Ąē` with `m ‚Č• p`, `n ‚Č• p`, and `m < n`. Right-inverse at `(m, p)` and `(n, p)` delivers `(m ‚ąí p) + p = m` and `(n ‚ąí p) + p = n`; substituting both into `m < n` via indiscernibility of `=` rewrites the hypothesis to `(m ‚ąí p) + p < (n ‚ąí p) + p`. Abbreviate `a := m ‚ąí p` and `b := n ‚ąí p` ‚ÄĒ conditional closure places both in ‚Ąē ‚ÄĒ so the rewritten hypothesis reads `a + p < b + p`. NAT-order's at-least-one trichotomy on `(a, b)` presents three cases. The `a < b` case is the desired conclusion `m ‚ąí p < n ‚ąí p`. The `a = b` case substitutes into `a + p < b + p` via indiscernibility to yield `a + p < a + p`, contradicting NAT-order's irreflexivity at `n := a + p`. The `b < a` case unfolds via NAT-order's `‚Č§`-definition to `b ‚Č§ a`; NAT-addcompat's right order compatibility applied at antecedent `b ‚Č§ a` then delivers `b + p ‚Č§ a + p`, which the `‚Č§`-definition splits into `b + p < a + p ‚ą® b + p = a + p`. The `<` branch, conjoined with the already-established `a + p < b + p`, contradicts exactly-one trichotomy's `¬¨(x < y ‚ąß y < x)` clause at `(x, y) := (a + p, b + p)`; the `=` branch substitutes into `a + p < b + p` via indiscernibility to yield `a + p < a + p`, again against irreflexivity. Both disjuncts of the `b < a` case collapse, leaving `a < b`, i.e., `m ‚ąí p < n ‚ąí p`.

Strict positivity ‚ÄĒ `m > n ‚üĻ m ‚ąí n ‚Č• 1` ‚ÄĒ is exported as a *Consequence:* rather than an additional axiom clause, because its content is not purely subtractive: lifting `m ‚ąí n ‚Č† 0` to `m ‚ąí n ‚Č• 1` requires the discreteness fact that no natural number lies strictly between `0` and `1`, which NAT-discrete names. Assume `m, n ‚ąą ‚Ąē` with `m > n`. NAT-order's reverse companion `m > n ‚üļ n < m` and the `‚Č§`-definition deliver `n ‚Č§ m`, hence `m ‚Č• n`; conditional closure then places `m ‚ąí n ‚ąą ‚Ąē`. Suppose, for contradiction, that `m ‚ąí n = 0`; the right-inverse rewrites `(m ‚ąí n) + n = m` to `0 + n = m`, which NAT-closure's left-identity collapses to `n = m`, contradicting NAT-order's disjointness Consequence `n < m ‚üĻ n ‚Č† m`. Hence `m ‚ąí n ‚Č† 0`. NAT-zero's axiom clause `(A k ‚ąą ‚Ąē :: 0 < k ‚ą® 0 = k)` at `k := m ‚ąí n` then leaves `0 < m ‚ąí n` once the equality branch is discarded, and NAT-discrete at `(m, n) := (0, m ‚ąí n)` delivers `0 + 1 ‚Č§ m ‚ąí n`, which NAT-closure's left-identity reduces to `1 ‚Č§ m ‚ąí n`, i.e., `m ‚ąí n ‚Č• 1`.

The axiom body invokes symbols beyond ‚Ąē's primitive membership. The strict order `<` together with its non-strict companion `‚Č§` and reverse companions `‚Č•`, `>` ‚ÄĒ all defined in NAT-order by `m ‚Č§ n ‚üļ m < n ‚ą® m = n`, `m ‚Č• n ‚üļ n ‚Č§ m`, `m > n ‚üļ n < m` ‚ÄĒ appear in the signature's domain condition `{(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n}` and in the antecedents `m ‚Č• n` of the conditional-closure and inverse-characterisation clauses. The binary addition `+` closed over ‚Ąē by NAT-closure appears in the sums `(m ‚ąí n) + n`, `n + (m ‚ąí n)`, and `m + n`. NAT-addbound's universal `(A m, n ‚ąą ‚Ąē :: m + n ‚Č• n)` discharges the conditional-closure precondition implicit in the unguarded equation `(m + n) ‚ąí n = m`. The strict-monotonicity *Consequence:* additionally cites NAT-addcompat's right order compatibility `(A m, n, p ‚ąą ‚Ąē : p ‚Č§ n : p + m ‚Č§ n + m)` (instantiated at antecedent `b ‚Č§ a` to lift it through right-addition to `b + p ‚Č§ a + p`) together with NAT-order's at-least-one trichotomy, irreflexivity, and the exactly-one-trichotomy Consequence's `¬¨(x < y ‚ąß y < x)` clause; NAT-addcompat is declared in the Depends slot accordingly. The strict-positivity *Consequence:* additionally cites NAT-closure's left-identity `0 + n = n` and literal `1 ‚ąą ‚Ąē`, NAT-order's disjointness Consequence `n < m ‚üĻ n ‚Č† m`, NAT-zero's `(A k ‚ąą ‚Ąē :: 0 < k ‚ą® 0 = k)`, and NAT-discrete's `m < n ‚üĻ m + 1 ‚Č§ n` at `(0, m ‚ąí n)`; NAT-zero and NAT-discrete are declared in the Depends slot accordingly.

*Formal Contract:*
- *Axiom:* `‚ąí : {(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n} ‚Üí ‚Ąē` (signature: `‚ąí` is a partial binary operation on ‚Ąē, single-valued on its domain of definition); `(A m, n ‚ąą ‚Ąē : m ‚Č• n : m ‚ąí n ‚ąą ‚Ąē)` (conditional closure); `(A m, n ‚ąą ‚Ąē : m ‚Č• n : (m ‚ąí n) + n = m)` (right-inverse characterisation); `(A m, n ‚ąą ‚Ąē : m ‚Č• n : n + (m ‚ąí n) = m)` (left-inverse characterisation); `(A m, n ‚ąą ‚Ąē :: (m + n) ‚ąí n = m)` (right telescoping).
- *Consequence:* `(A m, n, p ‚ąą ‚Ąē : m ‚Č• p ‚ąß n ‚Č• p ‚ąß m < n : m ‚ąí p < n ‚ąí p)` (strict monotonicity). Right-inverse at `(m, p)` and `(n, p)` rewrites `m < n` via indiscernibility of `=` to `(m ‚ąí p) + p < (n ‚ąí p) + p`; abbreviating `a := m ‚ąí p`, `b := n ‚ąí p` (both in ‚Ąē by conditional closure), the hypothesis reads `a + p < b + p`. At-least-one trichotomy on `(a, b)` presents three cases: `a < b` is the conclusion; `a = b` rewrites `a + p < b + p` to `a + p < a + p` against irreflexivity; `b < a` unfolds via `‚Č§`-definition to `b ‚Č§ a`, and NAT-addcompat's right order compatibility delivers `b + p ‚Č§ a + p`, whose `<` branch ‚ÄĒ paired with `a + p < b + p` ‚ÄĒ collides with exactly-one trichotomy's `¬¨(x < y ‚ąß y < x)` clause at `(a + p, b + p)`, and whose `=` branch rewrites `a + p < b + p` to `a + p < a + p` against irreflexivity. So `a < b`, i.e., `m ‚ąí p < n ‚ąí p`.
- *Consequence:* `(A m, n ‚ąą ‚Ąē : m > n : m ‚ąí n ‚Č• 1)` (strict positivity). Assuming `m ‚ąí n = 0` forces `0 + n = m` via the right-inverse, which NAT-closure's left-identity collapses to `n = m`, contradicting NAT-order's disjointness Consequence at `(n, m)`; so `m ‚ąí n ‚Č† 0`. NAT-zero's `(A k ‚ąą ‚Ąē :: 0 < k ‚ą® 0 = k)` at `k := m ‚ąí n` then leaves `0 < m ‚ąí n`, and NAT-discrete at `(m, n) := (0, m ‚ąí n)` delivers `0 + 1 ‚Č§ m ‚ąí n`, which left-identity reduces to `1 ‚Č§ m ‚ąí n`, i.e., `m ‚ąí n ‚Č• 1`.
- *Depends:*
  - NAT-order (NatStrictTotalOrder) ‚ÄĒ supplies the strict order `<` and its companions `‚Č§`, `‚Č•`, `>` (defined by `m ‚Č§ n ‚üļ m < n ‚ą® m = n`, `m ‚Č• n ‚üļ n ‚Č§ m`, `m > n ‚üļ n < m`), used in the signature's domain condition `{(m, n) ‚ąą ‚Ąē √ó ‚Ąē : m ‚Č• n}` and in the antecedents `m ‚Č• n` of the conditional-closure and inverse-characterisation clauses; supplies the at-least-one trichotomy axiom and irreflexivity axiom, together with the exactly-one-trichotomy Consequence's `¬¨(x < y ‚ąß y < x)` clause, against which the strict-monotonicity derivation dispatches the `a = b`, `b < a`-and-`<`, and `b < a`-and-`=` subcases; and supplies the disjointness Consequence `n < m ‚üĻ n ‚Č† m`, against which the strict-positivity derivation contradicts the `m ‚ąí n = 0` case.
  - NAT-closure (NatArithmeticClosureAndIdentity) ‚ÄĒ posits `1 ‚ąą ‚Ąē` and closes ‚Ąē under addition, so every sum `(m ‚ąí n) + n`, `n + (m ‚ąí n)`, `m + n` appearing in the inverse-characterisation and right-telescoping clauses is an ‚Ąē-element; additionally supplies the left-identity `(A k ‚ąą ‚Ąē :: 0 + k = k)`, which the strict-positivity derivation invokes twice ‚ÄĒ once to collapse `0 + n = m` to `n = m`, once to collapse `0 + 1 ‚Č§ m ‚ąí n` to `1 ‚Č§ m ‚ąí n`.
  - NAT-addbound (NatAdditionDominatesRightOperand) ‚ÄĒ supplies the universal `(A m, n ‚ąą ‚Ąē :: m + n ‚Č• n)`, which discharges the conditional-closure precondition `m + n ‚Č• n` implicit in the right-telescoping clause `(m + n) ‚ąí n = m`.
  - NAT-addcompat (NatAdditionOrderAndSuccessor) ‚ÄĒ supplies right order compatibility `(A m, n, p ‚ąą ‚Ąē : p ‚Č§ n : p + m ‚Č§ n + m)`, instantiated at antecedent `b ‚Č§ a` in the strict-monotonicity derivation's `b < a` case to lift `b ‚Č§ a` through right-addition by `p` to `b + p ‚Č§ a + p`.
  - NAT-zero (NatZeroMinimum) ‚ÄĒ supplies the axiom clause `(A k ‚ąą ‚Ąē :: 0 < k ‚ą® 0 = k)`, instantiated at `k := m ‚ąí n` in the strict-positivity derivation to lift `m ‚ąí n ‚Č† 0` to `0 < m ‚ąí n` before NAT-discrete fires.
  - NAT-discrete (NatDiscreteness) ‚ÄĒ supplies `(A m, n ‚ąą ‚Ąē :: m < n ‚üĻ m + 1 ‚Č§ n)`, instantiated at `(m, n) := (0, m ‚ąí n)` in the strict-positivity derivation to yield `0 + 1 ‚Č§ m ‚ąí n`, which NAT-closure's left-identity reduces to `m ‚ąí n ‚Č• 1`.
