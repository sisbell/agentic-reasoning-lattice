**NAT-zero (NatZeroMinimum).** `0` is the minimum of ℕ: `0 ∈ ℕ` and `(A n ∈ ℕ :: 0 < n ∨ 0 = n)`.

In words: `0` is itself a natural number, and every natural number is either strictly above `0` or equal to it. Combined with NAT-order's irreflexivity `¬(n < n)`, transitivity `m < n ∧ n < p ⟹ m < p`, and disjointness of `<` and `=` — `(A m, n ∈ ℕ : m < n : m ≠ n)` — these clauses identify `0` as the minimum: `(A n ∈ ℕ :: ¬(n < 0))`. Suppose some `n ∈ ℕ` did satisfy `n < 0`; the second clause forces `0 < n ∨ 0 = n`. In the first case, `0 < n` and `n < 0` together yield `0 < 0` by transitivity, contradicting irreflexivity. In the second case, disjointness applied to `n < 0` at `(n, 0)` yields `n ≠ 0`, which by symmetry of `=` contradicts `0 = n`; this route avoids substituting equals under `<`, a step not among NAT-order's stated properties. The minimum predicate `(A n ∈ ℕ :: ¬(n < 0))` is therefore exported as a *Consequence:* of the formal contract, lifted from the axiom's disjunction by NAT-order's irreflexivity, transitivity, and disjointness; NAT-order is declared in the Depends slot accordingly.

*Formal Contract:*
- *Axiom:* `0 ∈ ℕ` (zero is a natural number); `(A n ∈ ℕ :: 0 < n ∨ 0 = n)` (every natural number is strictly above or equal to zero).
- *Consequence:* `(A n ∈ ℕ :: ¬(n < 0))` (no natural number is strictly below zero — the minimum reading).
- *Depends:*
  - NAT-order (NatStrictTotalOrder) — supplies the strict-order primitive `<` used in the second axiom clause; its transitivity `m < n ∧ n < p ⟹ m < p` against irreflexivity `¬(n < n)` excludes the `0 < n` branch of the disjunction, and its disjointness of `<` and `=` — `(A m, n ∈ ℕ : m < n : m ≠ n)` — excludes the `0 = n` branch (from `n < 0` disjointness yields `n ≠ 0`, contradicting `0 = n` by symmetry of `=`), jointly lifting the disjunction `0 < n ∨ 0 = n` to the *Consequence:* bullet `¬(n < 0)`.
