**TA5-SigValid (SigOnValidAddresses).** For every valid address `t` satisfying T4, `sig(t) = #t`.

*Proof.* Let `t` be a valid address satisfying T4. T4's field-segment constraint requires `t_{#t} ≠ 0`, so the last component is non-zero. T0 fixes the carrier as ℕ, so `t_{#t} ∈ ℕ`; NAT-zero (`(A n ∈ ℕ :: 0 ≤ n)`, zero as lower bound of ℕ) supplies the lower bound `0 ≤ t_{#t}`; and NAT-discrete (`(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)`), instantiated at `m = 0`, rules out the interval `0 ≤ t_{#t} < 1` under `t_{#t} ≠ 0`. Together these three citations force `t_{#t} > 0` — the same per-step sourcing of the "non-zero ⇒ strictly positive" inference used in T4, T4a, and T4c, not a reliance on T0's carrier alone.

Since `t_{#t} > 0`, the tumbler `t` has at least one nonzero component, and by TA5-SIG, `sig(t) = max({i : 1 ≤ i ≤ #t ∧ tᵢ ≠ 0})`. The index `#t` belongs to this set, so `sig(t) ≥ #t`. But `sig(t) ≤ #t` by the range guarantee of TA5-SIG. Therefore `sig(t) = #t`. ∎

*Formal Contract:*
- *Precondition:* `t` satisfies T4 (valid address tumbler: at most three zero-valued field separators, field-segment constraint — no two zeros adjacent, `t₁ ≠ 0`, `t_{#t} ≠ 0`). Positivity of non-zero components is supplied by the combination of T0's carrier ℕ, NAT-zero (`0 ≤ n` on ℕ), and NAT-discrete (instantiated at `m = 0`, ruling out `0 ≤ n < 1` under `n ≠ 0`); T0 alone fixes only the carrier and does not license the discreteness step.
- *Depends:* T4 (HierarchicalParsing) — invoked at the proof opening to extract the field-segment constraint `t_{#t} ≠ 0` ("T4's field-segment constraint requires `t_{#t} ≠ 0`, so the last component is non-zero"); without T4 the rightmost component could be zero and the equality `sig(t) = #t` would fail. T0 (CarrierSetDefinition) — invoked immediately after T4 to fix the carrier as ℕ so that `t_{#t} ∈ ℕ` ("T0 fixes the carrier as ℕ, so `t_{#t} ∈ ℕ`"); T0 supplies the underlying set on which the NAT-* axioms speak but does not on its own license the step from non-zero to strictly positive. NAT-zero (NatZeroMinimum) — invoked at the same step ("NAT-zero supplies the lower bound `0 ≤ t_{#t}`") to furnish the premise `0 ≤ t_{#t}` required to instantiate NAT-discrete at `m = 0`; without NAT-zero the interval `0 ≤ t_{#t} < 1` could not be formed. NAT-discrete (NatDiscreteness) — invoked at the same step ("NAT-discrete, instantiated at `m = 0`, rules out the interval `0 ≤ t_{#t} < 1` under `t_{#t} ≠ 0`") to convert the lower bound `0 ≤ t_{#t}` together with `t_{#t} ≠ 0` into strict positivity `t_{#t} > 0`; without NAT-discrete the equivalence "non-zero ⇔ strictly positive" on ℕ would fail (ℝ admits `0 < r < 1` and satisfies NAT-zero but not NAT-discrete). The trio T0 + NAT-zero + NAT-discrete together — not T0 alone — licenses the `> 0` step, matching the per-step citation convention already used for this same inference in T4, T4a, and T4c. TA5-SIG (LastSignificantPosition) — invoked twice: first to unfold `sig(t)` as the maximum-position formula ("by TA5-SIG, `sig(t) = max({i : 1 ≤ i ≤ #t ∧ tᵢ ≠ 0})`") which gives `sig(t) ≥ #t` from `t_{#t} > 0`'s membership, and second for the range guarantee `sig(t) ≤ #t` ("`sig(t) ≤ #t` by the range guarantee of TA5-SIG"); the two bounds combine to `sig(t) = #t`.
- *Guarantee:* `sig(t) = #t`.
