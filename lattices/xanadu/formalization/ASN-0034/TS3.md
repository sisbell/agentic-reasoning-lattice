**TS3 (ShiftComposition).**

`(A v, n₁, n₂, m : v ∈ T ∧ n₁ ∈ ℕ ∧ n₂ ∈ ℕ ∧ n₁ ≥ 1 ∧ n₂ ≥ 1 ∧ #v = m : shift(shift(v, n₁), n₂) = shift(v, n₁ + n₂))`

The dummy `m` in the binder is a local abbreviation for `#v`: for any fixed `v ∈ T`, the range predicate `#v = m` determines `m` uniquely as the length of `v` (with `m ∈ ℕ` following by T0's length operator typing `#·: T → ℕ` applied to `v ∈ T`, and `m ≥ 1` following by T0's length axiom `#a ≥ 1 for all a ∈ T`), so the universal over `m` collapses to the single witness `m = #v` — an abbreviation pattern matching the one OrdinalShift's own Definition clause `shift(v, n) = v ⊕ δ(n, m) where m = #v` uses at its point of introduction.

*Proof.* We show that shifting by n₁ then n₂ is the same as shifting by their sum: composing two ordinal shifts reduces to a single shift whose amount is the sum of the individual amounts.

Fix v ∈ T, and let m abbreviate #v so that the range predicate `#v = m` binds m as the length of v. Fix n₁, n₂ ∈ ℕ with n₁ ≥ 1, n₂ ≥ 1. We must prove shift(shift(v, n₁), n₂) = shift(v, n₁ + n₂). We compute each side by expanding OrdinalShift and applying TumblerAdd's constructive definition (Definition TumblerAdd), then show the results agree at every component.

**Left side.** Before invoking OrdinalShift at `(v, n₁)` we discharge its `n ∈ ℕ` precondition from TS3's own precondition `n₁ ∈ ℕ` under the substitution `n = n₁` — an order statement like `n₁ ≥ 1` presupposes rather than establishes carrier membership (the convention OrdinalDisplacement's own proof articulates at its m-th-position-typing site), so the carrier-typing must be routed from the separately named `n₁ ∈ ℕ` precondition rather than from the order precondition `n₁ ≥ 1` alone. By OrdinalShift, shift(v, n₁) = v ⊕ δ(n₁, m), where δ(n₁, m) = [0, ..., 0, n₁] has action point m. Since m = #v, the precondition of TA0 is satisfied (k = m ≤ m = #v). Let u = v ⊕ δ(n₁, m). By TumblerAdd's constructive definition with action point k = m:

- For 1 ≤ i < m: uᵢ = vᵢ (prefix copy — all m − 1 higher-level components are preserved).
- At i = m: uₘ = vₘ + n₁ (single-component advance).
- No positions i > m exist, since #u = #δ(n₁, m) = m by the result-length identity.

Now we apply the second shift. Before invoking OrdinalShift at `(u, n₂)` we discharge two of its preconditions to named sources. First, the carrier-typing `u ∈ T` — required as OrdinalShift's first-argument precondition at the substitution `v = u` — comes from TA0's first exported postcondition `a ⊕ w ∈ T` instantiated at `a = v, w = δ(n₁, m)`: TA0's four preconditions at this instantiation are already in place from the first-shift construction of `u = v ⊕ δ(n₁, m)` above — `v ∈ T` transfers directly from TS3's own precondition, `δ(n₁, m) ∈ T` and `Pos(δ(n₁, m))` transfer directly from OrdinalDisplacement's exported postconditions (the same postconditions consumed at the first-shift site to build δ(n₁, m)), and `actionPoint(δ(n₁, m)) ≤ #v` is exactly the `k = m ≤ m = #v` bound discharged at the first shift via OrdinalDisplacement's exported `actionPoint(δ(n₁, m)) = m` combined with TS3's binding `m = #v`. Without this step the downstream action-point-precondition discharge `k = m ≤ m = #u` could not even type, since `#u` presupposes `u ∈ T` as the domain of T0's length operator `#·: T → ℕ`, so the gap is not merely cosmetic. Second, OrdinalShift's `n ∈ ℕ` precondition is discharged from TS3's own precondition `n₂ ∈ ℕ` under the substitution `n = n₂`, again routing the carrier-typing through the separately named `n₂ ∈ ℕ` precondition rather than through the order precondition `n₂ ≥ 1`. By OrdinalShift, shift(u, n₂) = u ⊕ δ(n₂, m), where δ(n₂, m) has action point m. The precondition of TA0 is satisfied: k = m ≤ m = #u. Let L = u ⊕ δ(n₂, m) denote the left side. By TumblerAdd with action point k = m:

- For 1 ≤ i < m: Lᵢ = uᵢ = vᵢ (prefix copy, substituting the values of u established above).
- At i = m: Lₘ = uₘ + n₂ = (vₘ + n₁) + n₂ (advance, substituting uₘ = vₘ + n₁).
- Length: #L = #δ(n₂, m) = m by the result-length identity.

**Right side.** We first source two sub-facts about `n₁ + n₂` required by OrdinalDisplacement's preconditions at the argument `n = n₁ + n₂`: the ℕ-typing `n₁ + n₂ ∈ ℕ` and the order fact `n₁ + n₂ ≥ 1`. Each sub-fact is routed through named NAT-* axioms under the per-step citation discipline this ASN enforces at sister strict-from-`≥` sites (TumblerAdd's strict-advancement chain, OrdinalDisplacement's `n ≥ 1 ⟹ n ≠ 0` promotion, TA-Pos's Case `#z ≥ k` step).

The ℕ-typing `n₁ + n₂ ∈ ℕ` is supplied by NAT-closure's addition-closure axiom `(A m, n ∈ ℕ :: m + n ∈ ℕ)` instantiated at `m = n₁, n = n₂` — the axiom's two preconditions transfer directly from TS3's own preconditions `n₁ ∈ ℕ, n₂ ∈ ℕ`.

The order fact `n₁ + n₂ ≥ 1` is routed through the chain `n₁ + n₂ ≥ 1 + n₂ ≥ 1 + 1 ≥ 1`, each link sourced from NAT-addcompat and each composition from NAT-order. NAT-addcompat's right order compatibility `(A m, n, p ∈ ℕ : n ≥ p : n + m ≥ p + m)` instantiated at `m = n₂, n = n₁, p = 1`, with premise `n₁ ≥ 1` supplied by TS3's precondition, yields `n₁ + n₂ ≥ 1 + n₂`. NAT-addcompat's left order compatibility `(A m, n, p ∈ ℕ : n ≥ p : m + n ≥ m + p)` instantiated at `m = 1, n = n₂, p = 1`, with premise `n₂ ≥ 1` supplied by TS3's precondition, yields `1 + n₂ ≥ 1 + 1`, i.e., `1 + n₂ ≥ 2`. NAT-addcompat's strict successor inequality `(A n ∈ ℕ :: n < n + 1)` instantiated at `n = 1` supplies `1 < 1 + 1`, i.e., `1 < 2`, from which NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` — reading the disjunction right-to-left at the `1 < 2` disjunct — returns `1 ≤ 2`, i.e., `2 ≥ 1`.

Composing the three non-strict links `n₁ + n₂ ≥ 1 + n₂`, `1 + n₂ ≥ 2`, and `2 ≥ 1` into the single fact `n₁ + n₂ ≥ 1` consumes NAT-order's defining clause paired with NAT-order's transitivity of `<`: unfolding each non-strict inequality `a ≥ b` into the disjunction `a > b ∨ a = b` via the defining clause, each pairwise composition closes by NAT-order's transitivity of `<` in the strict-strict case and by substitution in the three mixed/equal cases, refolding via the defining clause to deliver the composed non-strict inequality in every case. Two such pairwise compositions — first of `n₁ + n₂ ≥ 1 + n₂` with `1 + n₂ ≥ 2` to yield `n₁ + n₂ ≥ 2`, then of `n₁ + n₂ ≥ 2` with `2 ≥ 1` to yield `n₁ + n₂ ≥ 1` — discharge the chain.

So δ(n₁ + n₂, m) is a well-formed ordinal displacement with action point m. Before invoking OrdinalShift at `(v, n₁ + n₂)` we discharge its `n ∈ ℕ` precondition at the argument `n = n₁ + n₂` from the just-established `n₁ + n₂ ∈ ℕ`: the routing here is NAT-closure applied to TS3's two `n_i ∈ ℕ` preconditions rather than a direct transfer, since TS3 names `n₁ ∈ ℕ` and `n₂ ∈ ℕ` separately but not their sum's ℕ-typing, and the convention this ASN applies forecloses routing the typing through the order chain `n₁ + n₂ ≥ 1 + n₂ ≥ 1 + 1 ≥ 1` alone. By OrdinalShift, shift(v, n₁ + n₂) = v ⊕ δ(n₁ + n₂, m). The precondition of TA0 is satisfied: k = m ≤ m = #v. Let R = v ⊕ δ(n₁ + n₂, m) denote the right side. By TumblerAdd with action point k = m:

- For 1 ≤ i < m: Rᵢ = vᵢ (prefix copy).
- At i = m: Rₘ = vₘ + (n₁ + n₂) (single-component advance).
- Length: #R = #δ(n₁ + n₂, m) = m by the result-length identity.

**Comparison.** Both sides have length m: #L = m = #R. We verify component-wise equality:

- For 1 ≤ i < m: Lᵢ = vᵢ = Rᵢ — both sides copied the prefix from v.
- At i = m: Lₘ = (vₘ + n₁) + n₂ and Rₘ = vₘ + (n₁ + n₂). Before invoking NAT-addassoc at the instantiation `m = vₘ, n = n₁, p = n₂` we discharge its three ℕ-typing preconditions one-for-one: `vₘ ∈ ℕ` follows from T0's carrier characterisation of `T` as finite sequences over ℕ applied to `v ∈ T` at position `m` (with the binder's `#v = m` placing position `m` within the range `1 ≤ i ≤ #v` of valid component indices, so `vₘ` is a well-typed projection of `v`); `n₁ ∈ ℕ` and `n₂ ∈ ℕ` transfer directly from TS3's own preconditions. These are equal by NAT-addassoc (NatAdditionAssociative), which states `(m + n) + p = m + (n + p)` for every `m, n, p ∈ ℕ`: instantiated with `vₘ, n₁, n₂`, it yields `(vₘ + n₁) + n₂ = vₘ + (n₁ + n₂)`.

Every component agrees and both tumblers have the same length. By T3 (CanonicalRepresentation): L = R, that is, shift(shift(v, n₁), n₂) = shift(v, n₁ + n₂). ∎
