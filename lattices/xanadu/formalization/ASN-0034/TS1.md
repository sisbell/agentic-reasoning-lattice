**TS1 (ShiftOrderPreservation).**

`(A v₁, v₂ ∈ T, n ∈ ℕ : n ≥ 1 ∧ #v₁ = #v₂ ∧ v₁ < v₂ : shift(v₁, n) < shift(v₂, n))`

*Proof.* We show that shifting two equal-length tumblers by the same amount preserves their strict ordering.

Fix v₁, v₂ ∈ T with #v₁ = #v₂, v₁ < v₂, and n ≥ 1, and let m abbreviate #v₁ (so that #v₂ = m as well by the hypothesis #v₁ = #v₂). By OrdinalShift, shift(v₁, n) = v₁ ⊕ δ(n, m) and shift(v₂, n) = v₂ ⊕ δ(n, m), so we must show v₁ ⊕ δ(n, m) < v₂ ⊕ δ(n, m). We verify the eight preconditions of TA1-strict with a = v₁, b = v₂, w = δ(n, m), enumerated one-for-one in the order TA1-strict's contract states them — `a ∈ T, b ∈ T, w ∈ T, a < b, Pos(w), actionPoint(w) ≤ #a, actionPoint(w) ≤ #b, actionPoint(w) ≥ divergence(a, b)` — with each of the three membership preconditions and each of the two length bounds discharged separately rather than fused into a composite item:

(i) v₁ ∈ T — given by hypothesis (this is one of TS1's own preconditions).

(ii) v₂ ∈ T — given by hypothesis (this is one of TS1's own preconditions).

(iii) δ(n, m) ∈ T — by OrdinalDisplacement's postcondition `δ(n, m) ∈ T`. The OrdinalDisplacement invocation at `(n, m)` is engaged through OrdinalShift's two applications at the proof opening (one per v_i), each of which discharges OrdinalDisplacement's four preconditions `n ∈ ℕ, m ∈ ℕ, n ≥ 1, m ≥ 1` at its own call site, so the membership `δ(n, m) ∈ T` is sourced here at the TA1-strict call site through OrdinalDisplacement's exported postcondition rather than left to flow tacitly through OrdinalShift's `v ⊕ δ(n, m)` notation. The three membership preconditions of TA1-strict — its first three conjuncts `a ∈ T, b ∈ T, w ∈ T` — are thereby each discharged to a named source (hypothesis, hypothesis, OrdinalDisplacement) rather than fused into a single "all operands well-typed" item.

(iv) v₁ < v₂ — given by hypothesis (this is one of TS1's own preconditions).

(v) Pos(δ(n, m)) — by OrdinalDisplacement's postcondition `Pos(δ(n, m))`. The OrdinalDisplacement invocation at `(n, m)` is the same invocation cited at (iii), engaged through OrdinalShift's two applications at the proof opening (one per v_i), each of which discharges OrdinalDisplacement's four preconditions `n ∈ ℕ, m ∈ ℕ, n ≥ 1, m ≥ 1` at its own call site, so the positivity `Pos(δ(n, m))` is sourced here at the TA1-strict call site through OrdinalDisplacement's exported postcondition rather than re-derived from δ's component pattern inline. TA1-strict's fifth precondition `Pos(w)` is discharged one-for-one under its actual name rather than under an order comparison `δ(n, m) > 0`: the notation `>` in this ASN denotes T1's lexicographic ordering on equal-typed tumblers, not a scalar comparison, and no distinguished zero-tumbler is in scope to stand on the right-hand side (T0 stipulates length ≥ 1 with components in ℕ but names no such tumbler). The predicate `Pos(·)` is TA-Pos's defining existential `(E i : tᵢ ≠ 0)`, which OrdinalDisplacement's own proof witnesses at `i = m` (δ's m-th component equals n, and `n ≥ 1` promotes to `n ≠ 0`) and exports as a named postcondition, so TS1 cites the exported predicate directly rather than unpacking its witness.

(vi) actionPoint(δ(n, m)) ≤ #v₁ — the action point of δ(n, m) is m (OrdinalDisplacement), and #v₁ = m by the abbreviation m = #v₁ introduced at the proof opening, so this conjunct reduces to m ≤ m, which holds.

(vii) actionPoint(δ(n, m)) ≤ #v₂ — the action point of δ(n, m) is m (OrdinalDisplacement), and #v₂ = m by the hypothesis #v₁ = #v₂ composed with the abbreviation m = #v₁, so this second conjunct likewise reduces to m ≤ m, which holds. The two length bounds are discharged separately against #v₁ and #v₂ rather than folded into a binary-minimum operator `min(#v₁, #v₂)`, matching TA1-strict's contract — which demands the conjunction `actionPoint(w) ≤ #a ∧ actionPoint(w) ≤ #b` — and matching the ASN-wide convention (established in Divergence's Definition prose: "stated without invoking any primitive binary-minimum operator on ℕ") that no listed NAT-* axiom supplies such a `min`.

(viii) actionPoint(δ(n, m)) ≥ divergence(v₁, v₂) — since #v₁ = #v₂ by hypothesis, Divergence case (ii) (prefix divergence) is excluded: it requires #v₁ ≠ #v₂. Since v₁ < v₂ implies v₁ ≠ v₂, Divergence case (i) applies and supplies an index `k` with `1 ≤ k ∧ k ≤ #v₁ ∧ k ≤ #v₂` satisfying `(v₁)ₖ ≠ (v₂)ₖ ∧ (A i : 1 ≤ i < k : (v₁)ᵢ = (v₂)ᵢ)`, for which `divergence(v₁, v₂) = k`. The bound `divergence(v₁, v₂) ≤ m` is then read off case (i)'s own defining conjunct `k ≤ #v₁` against the abbreviation `m = #v₁`: `divergence(v₁, v₂) = k ≤ #v₁ = m` — discharged through Divergence's contract rather than by re-defining the function with a primitive binary-minimum operator `min` that no listed NAT-* axiom supplies, matching the ASN-wide convention (established in Divergence's Definition prose: "stated without invoking any primitive binary-minimum operator on ℕ") and preserving the prior-position-agreement conjunct `(A i : 1 ≤ i < k : (v₁)ᵢ = (v₂)ᵢ)` that Divergence's contract requires for `k` to qualify. The action point m ≥ divergence(v₁, v₂) follows.

All eight preconditions hold. By TA1-strict: v₁ ⊕ δ(n, m) < v₂ ⊕ δ(n, m), that is, shift(v₁, n) < shift(v₂, n). ∎
