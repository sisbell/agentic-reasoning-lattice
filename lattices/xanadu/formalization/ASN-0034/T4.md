## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this, writing *node* for Nelson's *server* and *element* for his *contents* and retaining *user* and *document* verbatim; the renaming is notational. Define a *field separator* as a position `i` of `t` with `tŠĶĘ = 0`. An address tumbler has the form:

`t = N‚āĀ. ... .N‚āź . 0 . U‚āĀ. ... .UŠĶ¶ . 0 . D‚āĀ. ... .DŠĶß . 0 . E‚āĀ. ... .Eőī`

where `0 < NŠĶĘ, 0 < U‚Īľ, 0 < D‚āĖ, 0 < E‚āó` at every position. The four fields are:

- **Node field** `N‚āĀ. ... .N‚āź`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U‚āĀ. ... .UŠĶ¶`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D‚āĀ. ... .DŠĶß`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked ‚ÄĒ "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E‚āĀ. ... .Eőī`: identifies the content element. T4 constrains only `0 < E‚āó` at every element-field position; typed interpretations of `E‚āĀ` are the concern of downstream ASNs.

A tumbler with zero zeros addresses a node. One zero: a user account. Two zeros: a document. Three zeros: an element. The count of zero-valued components determines the specificity level.

**T4 (HierarchicalParsing).** Write `|S|` for the cardinality of a finite subset `S ‚äÜ ‚Ąē`; `|¬∑|` has codomain ‚Ąē and is a set-level operator, distinct from T0's tumbler-length `#¬∑ : T ‚Üí ‚Ąē` (which acts on sequences, not sets). Define `zeros(t) = |{i : 1 ‚Č§ i ‚Č§ #t ‚ąß tŠĶĘ = 0}|`; the argument `{i : 1 ‚Č§ i ‚Č§ #t ‚ąß tŠĶĘ = 0}` is a subset of the index domain `{1, ‚Ä¶, #t} ‚äÜ ‚Ąē` (T0), hence a finite subset of ‚Ąē, so `zeros(t) ‚ąą ‚Ąē`. Every tumbler `t ‚ąą T` used as an address satisfies: `zeros(t) ‚Č§ 3`; `(A i : 1 ‚Č§ i < #t : ¬¨(tŠĶĘ = 0 ‚ąß tŠĶĘ‚āä‚āĀ = 0))`; `t‚āĀ ‚Č† 0`; `t_{#t} ‚Č† 0`. We call the last three conditions collectively the *field-segment constraint*. T0 fixes the carrier as ‚Ąē; NAT-order supplies the strict total order `<` on ‚Ąē and its companion `‚Č§` (defined by `m ‚Č§ n ‚üļ m < n ‚ą® m = n`), which support the zero-count bound `zeros(t) ‚Č§ 3` and the adjacency clause's index range `1 ‚Č§ i < #t`; NAT-closure supplies `1 ‚ąą ‚Ąē` together with closure of ‚Ąē under addition, grounding the numerals `2 := 1 + 1 ‚ąą ‚Ąē` and `3 := 2 + 1 ‚ąą ‚Ąē` so that the bound `zeros(t) ‚Č§ 3` compares two elements of ‚Ąē; and NAT-zero together with NAT-discrete (at `m = 0`) force every non-zero component to be strictly positive; write `‚Ąē‚Āļ = {n ‚ąą ‚Ąē : 0 < n}` for the strictly positive naturals. The *field segments* of `t` are the maximal contiguous sub-sequences of non-zero positions ‚ÄĒ `k+1` of them when `zeros(t) = k`, separated by the zeros. T4 stipulates that a position `i` is a *field separator* iff `tŠĶĘ = 0`, and that every other position carries a *field component* ‚ÄĒ strictly positive by the preceding argument; the two roles are decidable by inspecting `tŠĶĘ`.

A direct consequence of T4, the first-class numbered theorem **T4a (SyntacticEquivalence)**, proves that the field-segment constraint is equivalent to the condition that every field segment of `t` is non-empty.

Two further first-class theorems, not direct postconditions of T4 but downstream combinations, round out the parsing picture. **T4b (UniqueParse)** is built on T4a: it introduces the partial function `fields(t)` with domain the T4-valid subset of `T`, splitting `t` at its zero positions into four field sub-sequences (with absent fields represented by the empty sequence `őĶ`), and proves it well-defined and uniquely determined by `t` ‚ÄĒ using T4a's reverse direction to supply the conclusion that every field segment is non-empty, which T4b locally re-expresses as the segment-length inequalities that discharge non-emptiness of each extracted sub-sequence. The codomain is given in T4b's section, where the notation `Seq(X)` for finite, possibly empty sequences over `X` is introduced and contrasted with T0's nonempty `T`. **T4c (LevelDetermination)** combines T4 with NAT-addcompat's strict successor inequality `n < n + 1` to assign the four hierarchical level labels definitionally by zero count ‚ÄĒ `(A t ‚ąą T : t is T4-valid :: (zeros(t) = 0 ‚ÜĒ t is a node address) ‚ąß (zeros(t) = 1 ‚ÜĒ t is a user address) ‚ąß (zeros(t) = 2 ‚ÜĒ t is a document address) ‚ąß (zeros(t) = 3 ‚ÜĒ t is an element address))` ‚ÄĒ and discharges exhaustion of the four zero-count values on the T4-valid subdomain together with pairwise distinctness of the labels. Realisation ‚ÄĒ existence of a T4-valid tumbler at each zero count ‚ÄĒ is not claimed: the foundations in scope underwrite neither a strictly positive numeral in ‚Ąē nor tumblers of length ‚Č• 7, so T4c stands as a pure definition on whatever T4-valid tumblers exist. Outside the T4-valid subdomain T4c assigns no level.

*Formal Contract:*
- *Axiom:* Valid address tumblers satisfy: `zeros(t) ‚Č§ 3`; `(A i : 1 ‚Č§ i < #t : ¬¨(tŠĶĘ = 0 ‚ąß tŠĶĘ‚āä‚āĀ = 0))`; `t‚āĀ ‚Č† 0`; `t_{#t} ‚Č† 0`. T4 stipulates that a position `i` of `t` is a *field separator* iff `tŠĶĘ = 0`. The canonical written form of a T4-valid address tumbler is keyed to `zeros(t) ‚ąą {0, 1, 2, 3}`, with one form per zero count: `zeros(t) = 0` gives `t = N‚āĀ. ... .N‚āź`; `zeros(t) = 1` gives `t = N‚āĀ. ... .N‚āź . 0 . U‚āĀ. ... .UŠĶ¶`; `zeros(t) = 2` gives `t = N‚āĀ. ... .N‚āź . 0 . U‚āĀ. ... .UŠĶ¶ . 0 . D‚āĀ. ... .DŠĶß`; `zeros(t) = 3` gives `t = N‚āĀ. ... .N‚āź . 0 . U‚āĀ. ... .UŠĶ¶ . 0 . D‚āĀ. ... .DŠĶß . 0 . E‚āĀ. ... .Eőī`. In every case, `0 < NŠĶĘ, 0 < U‚Īľ, 0 < D‚āĖ, 0 < E‚āó` at every position present.
- *Preconditions:* `t ‚ąą T`.
- *Depends:*
  - T0 (CarrierSetDefinition) ‚ÄĒ supplies the carrier ‚Ąē, the tumbler length `#¬∑`, and the index domain `{1, ‚Ä¶, #t}` in which the separator positions and field-segment clauses are stated.
  - NAT-zero (NatZeroMinimum) ‚ÄĒ supplies the lower bound `0 ‚Č§ tŠĶĘ` at every position.
  - NAT-discrete (NatDiscreteness) ‚ÄĒ at `m = 0`, rules out `0 ‚Č§ tŠĶĘ < 1` under `tŠĶĘ ‚Č† 0`, so the component `tŠĶĘ` at every non-separator position is strictly positive (`0 < NŠĶĘ, 0 < U‚Īľ, 0 < D‚āĖ, 0 < E‚āó`).
  - NAT-order (NatStrictTotalOrder) ‚ÄĒ supplies `<` on ‚Ąē and its companion `‚Č§` (with `m ‚Č§ n ‚üļ m < n ‚ą® m = n`), which underwrite the zero-count bound `zeros(t) ‚Č§ 3` and the adjacency clause `(A i : 1 ‚Č§ i < #t : ¬¨(tŠĶĘ = 0 ‚ąß tŠĶĘ‚āä‚āĀ = 0))`.
  - NAT-closure (NatArithmeticClosureAndIdentity) ‚ÄĒ supplies `1 ‚ąą ‚Ąē` and closure of ‚Ąē under addition, grounding the numerals `2 := 1 + 1` and `3 := 2 + 1` in ‚Ąē so that `zeros(t) ‚Č§ 3` is a comparison of two ‚Ąē-elements.
- *Postconditions:* T4a (SyntacticEquivalence), a direct consequence: the field-segment constraint is equivalent to every field segment of `t` being non-empty.

**T4b (UniqueParse).** Write `Seq(X)` for the set of finite, possibly empty sequences over `X`, and `őĶ` for its unique zero-length element. T0 fixes `T` as the *nonempty* finite sequences over `‚Ąē` ‚ÄĒ by the stipulation `#a ‚Č• 1` for every `a ‚ąą T` ‚ÄĒ so `T` strictly excludes `őĶ`; `Seq(¬∑)` differs from T's construction only by adjoining `őĶ`, and we use `Seq(‚Ąē‚Āļ)` (with `‚Ąē‚Āļ = {n ‚ąą ‚Ąē : 0 < n}`) precisely because field absence demands a value outside `T`. Under the T4 constraints, the partial function

 `fields : T ‚áÄ Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ)`

that decomposes a tumbler into its node, user, document, and element sub-sequences is well-defined and uniquely determined by `t` on the T4-valid subdomain, with absent fields represented by `őĶ`. Outside that subdomain `fields(t)` is not assigned a value.

We read `fields(t)` as `(N(t), U(t), D(t), E(t))` and fix the projections `N, U, D, E : T ‚áÄ Seq(‚Ąē‚Āļ)` accordingly. Each projection returns a finite sequence over `‚Ąē‚Āļ` when defined: NAT-zero supplies `0 ‚Č§ tŠĶĘ`, NAT-discrete at `m = 0` rules out `0 ‚Č§ tŠĶĘ < 1` when `tŠĶĘ ‚Č† 0`. A projection may be `őĶ` ‚ÄĒ the representation for an *absent* field. *Field absence* is defined independently of the projections as a direct predicate on `zeros(t)`: `N` is never absent on the T4-valid subdomain; `U` is *absent in `t`* iff `zeros(t) = 0`; `D` is *absent in `t`* iff `zeros(t) ‚Č§ 1`; `E` is *absent in `t`* iff `zeros(t) ‚Č§ 2`. The biconditional `X(t) = őĶ` iff field `X` is absent in `t` follows by inspection of the case construction below: in each case the projections set to `őĶ` are exactly those the predicate marks absent.

Absence pattern by case on `zeros(t) ‚ąą {0, 1, 2, 3}`:

  - `zeros(t) = 0`: `N(t) = (t‚āĀ, ..., t_{#t})`, `U(t) = D(t) = E(t) = őĶ`.
  - `zeros(t) = 1`: `N(t)` and `U(t)` non-empty, `D(t) = E(t) = őĶ`.
  - `zeros(t) = 2`: `N(t)`, `U(t)`, `D(t)` non-empty, `E(t) = őĶ`.
  - `zeros(t) = 3`: all four projections non-empty.

To access individual components of a field we introduce the notation `t.X‚āĀ`, grounded in T0's projection as follows. Since `‚Ąē‚Āļ = {n ‚ąą ‚Ąē : 0 < n} ‚äÜ ‚Ąē`, a finite sequence over `‚Ąē‚Āļ` is a finite sequence over `‚Ąē`; if nonempty, it is therefore an element of `T` by T0's stipulation that `T` is the set of nonempty finite sequences over `‚Ąē`. Whenever `X(t) ‚Č† őĶ` the value `X(t) ‚ąą Seq(‚Ąē‚Āļ)` is such a nonempty sequence, so `X(t) ‚ąą T` with `#(X(t)) ‚Č• 1`, and T0's component projection `¬∑ŠĶĘ` is defined at every `i ‚ąą {1, ‚Ä¶, #(X(t))}` ‚ÄĒ in particular at `i = 1`. We set `t.X‚āĀ := (X(t))‚āĀ`, so the dot-accessor is notational shorthand for T0's subscript applied to `X(t)`, and `t.X‚āĀ` is defined iff `X(t) ‚Č† őĶ`: `t.N‚āĀ` on every T4-valid `t`; `t.U‚āĀ` iff `zeros(t) ‚Č• 1`; `t.D‚āĀ` iff `zeros(t) ‚Č• 2`; `t.E‚āĀ` iff `zeros(t) = 3`.

By T4's stipulation, the separator positions of `t` are exactly `{i : 1 ‚Č§ i ‚Č§ #t ‚ąß tŠĶĘ = 0}`, and every non-separator position carries a field component (strictly positive by NAT-zero and NAT-discrete at `m = 0` on T0's carrier ‚Ąē).

Let the zero positions in increasing order be `s‚āĀ < s‚āā < ... < s_k` (NAT-order licenses the enumeration), `k = zeros(t) ‚ąą {0, 1, 2, 3}`. Since `t` is T4-valid, T4a's reverse direction applied to the field-segment constraint yields its exported conclusion: every field segment of `t` is non-empty. We unpack this locally into the position-inequalities used below. The arithmetic in what follows ‚ÄĒ the numeral `2`, additions `s_i + 1` and `s_i + 2`, and the partial subtraction `#t ‚ąí 1` ‚ÄĒ is grounded thus: NAT-closure posits `1 ‚ąą ‚Ąē` and closes ‚Ąē under addition, so `2 := 1 + 1 ‚ąą ‚Ąē` and, since every `s_i` and `#t` lies in ‚Ąē, each `s_i + 1`, `s_i + 2`, and `s_k + 1 ‚ąą ‚Ąē`; NAT-sub's conditional-closure clause applied at the hypothesis `#t ‚Č• 1` (T0) and `1 ‚ąą ‚Ąē` gives `#t ‚ąí 1 ‚ąą ‚Ąē`. T4 identifies the field segments as the `k + 1` maximal contiguous non-zero runs delimited by the zero positions and the sequence boundaries. The first segment occupies the indices `1, ‚Ä¶, s‚āĀ - 1` (when `k ‚Č• 1`), so its non-emptiness requires `s‚āĀ ‚Č• 2`; the last segment occupies the indices `s_k + 1, ‚Ä¶, #t` (when `k ‚Č• 1`), so its non-emptiness requires `s_k ‚Č§ #t - 1`; each interior segment (`1 ‚Č§ j < k`) occupies the indices `s_j + 1, ‚Ä¶, s_{j+1} - 1`, so its non-emptiness requires `s_{j+1} ‚Č• s_j + 2`. Compute `fields(t)` case by case, citing the relevant inequality for each segment:

  - *Case k = 0.* `N(t) = (t‚āĀ, ..., t_{#t})`, others `őĶ`. The sole segment spans indices 1 through `#t`, non-empty because `#t ‚Č• 1` (T0).
  - *Case k = 1.* `N(t) = (t‚āĀ, ..., t_{s‚āĀ - 1})`, `U(t) = (t_{s‚āĀ + 1}, ..., t_{#t})`. `s‚āĀ ‚Č• 2` makes `N(t)` non-empty; `s‚āĀ ‚Č§ #t - 1` makes `U(t)` non-empty.
  - *Case k = 2.* `N(t) = (t‚āĀ, ..., t_{s‚āĀ - 1})`, `U(t) = (t_{s‚āĀ + 1}, ..., t_{s‚āā - 1})`, `D(t) = (t_{s‚āā + 1}, ..., t_{#t})`. `s‚āĀ ‚Č• 2` makes `N(t)` non-empty; `s‚āā ‚Č• s‚āĀ + 2` makes `U(t)` non-empty; `s‚āā ‚Č§ #t - 1` makes `D(t)` non-empty.
  - *Case k = 3.* `N(t) = (t‚āĀ, ..., t_{s‚āĀ - 1})`, `U(t) = (t_{s‚āĀ + 1}, ..., t_{s‚āā - 1})`, `D(t) = (t_{s‚āā + 1}, ..., t_{s‚āÉ - 1})`, `E(t) = (t_{s‚āÉ + 1}, ..., t_{#t})`. `s‚āĀ ‚Č• 2` makes `N(t)` non-empty; `s‚āā ‚Č• s‚āĀ + 2` makes `U(t)` non-empty; `s‚āÉ ‚Č• s‚āā + 2` makes `D(t)` non-empty; `s‚āÉ ‚Č§ #t - 1` makes `E(t)` non-empty.

By T0, each `tŠĶĘ` is the value of the `i`-th component of `t` ‚ÄĒ a function of `t` and `i` ‚ÄĒ so the set `{i : 1 ‚Č§ i ‚Č§ #t ‚ąß tŠĶĘ = 0}` of separator positions is determined by `t`, and with it the field boundaries and the sub-sequences. Therefore `fields(t)` is well-defined and unique. ‚ąé

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints.
- *Definition:* `fields : T ‚áÄ Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ)` is the partial function whose domain is the T4-valid subset of `T` and whose action splits the component sequence of `t` at its zero positions. - *Postconditions:* `fields : T ‚áÄ Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ) √ó Seq(‚Ąē‚Āļ)` is a partial function whose domain is the T4-valid subset of `T`.
