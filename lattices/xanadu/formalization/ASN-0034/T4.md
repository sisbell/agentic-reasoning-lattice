## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this. Define a *field separator* as a position `i` of `t` with `tᵢ = 0`. An address tumbler has the form:

`t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`

where `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every position. The four fields are:

- **Node field** `N₁. ... .Nₐ`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U₁. ... .Uᵦ`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D₁. ... .Dᵧ`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked — "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E₁. ... .Eδ`: identifies the content element. T4 constrains only `0 < Eₗ` at every element-field position; typed interpretations of `E₁` are the concern of downstream ASNs.

A tumbler with zero zeros addresses a node. One zero: a user account. Two zeros: a document. Three zeros: an element. The count of zero-valued components determines the specificity level.

**T4 (HierarchicalParsing).** Write `|S|` for the cardinality of a finite subset `S ⊆ ℕ`; `|·|` has codomain ℕ and is a set-level operator, distinct from T0's tumbler-length `#· : T → ℕ` (which acts on sequences, not sets). Define `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|`; the argument `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` is a subset of the index domain `{1, …, #t} ⊆ ℕ` (T0), hence a finite subset of ℕ, so `zeros(t) ∈ ℕ`. Every tumbler `t ∈ T` used as an address satisfies: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. We call the last three conditions collectively the *field-segment constraint*. T0 fixes the carrier as ℕ, and NAT-zero together with NAT-discrete (at `m = 0`) force every non-zero component to be strictly positive; write `ℕ⁺ = {n ∈ ℕ : 0 < n}` for the strictly positive naturals. The *field segments* of `t` are the maximal contiguous sub-sequences of non-zero positions — `k+1` of them when `zeros(t) = k`, separated by the zeros — and the field-segment constraint is exactly the requirement that all of them are non-empty. T4 stipulates that a position `i` is a *field separator* iff `tᵢ = 0`, and that every other position carries a *field component* — strictly positive by the preceding argument; the two roles are decidable by inspecting `tᵢ`.

Two direct consequences of T4 round out the parsing picture, each a first-class numbered theorem in this ASN:

  - **T4a (SyntacticEquivalence)** proves that the field-segment constraint is equivalent to the condition that every field segment of `t` is non-empty.
  - **T4c (LevelDetermination)** assigns the four hierarchical level labels definitionally by zero count: `(A t ∈ T : t is T4-valid :: (zeros(t) = 0 ↔ t is a node address) ∧ (zeros(t) = 1 ↔ t is a user address) ∧ (zeros(t) = 2 ↔ t is a document address) ∧ (zeros(t) = 3 ↔ t is an element address))`. The four labels are pairwise distinct, and each is realised by some T4-valid tumbler. Outside the T4-valid subdomain no level is assigned.

A further first-class theorem, **T4b (UniqueParse)**, is not a direct postcondition of T4 but a downstream combination built on T4a: it introduces the partial function `fields(t)` with domain the T4-valid subset of `T`, splitting `t` at its zero positions into four field sub-sequences (with absent fields represented by the empty sequence `ε`), and proves it well-defined and uniquely determined by `t` — using T4a's reverse direction to supply the segment-length inequalities that discharge non-emptiness of each extracted sub-sequence. The codomain is given in T4b's section, where the notation `Seq(X)` for finite, possibly empty sequences over `X` is introduced and contrasted with T0's nonempty `T`.

A subtlety deserves emphasis: the hierarchy is *convention layered over flat arithmetic*, not enforcement by the algebra. The comparison, addition, subtraction, and increment operations treat every mantissa slot identically. There is no `isparent`, `isancestor`, or `ischild` primitive in the arithmetic layer. The hierarchical interpretation is projected onto flat sequences by the allocation machinery and by the field-parsing function `fields(t)`.

Hierarchy is constructed by the allocation machinery. The `.0.` separator is produced when the allocation `depth` parameter equals 2 — creating a child at a *different hierarchical type* than its parent. When creating a same-type child (`depth = 1`), no zero separator is introduced. Gregory confirms: there was even a bug where the first document under an account failed to receive its `.0.` separator — the convention had to be explicitly constructed by the allocator, not enforced by any algebraic invariant. ∎

*Formal Contract:*
- *Axiom:* Valid address tumblers satisfy: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. T4 stipulates that a position `i` of `t` is a *field separator* iff `tᵢ = 0`. The canonical written form of a T4-valid address tumbler is keyed to `zeros(t) ∈ {0, 1, 2, 3}`, with one form per zero count: `zeros(t) = 0` gives `t = N₁. ... .Nₐ`; `zeros(t) = 1` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ`; `zeros(t) = 2` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ`; `zeros(t) = 3` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`. In every case, `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every position present.
- *Preconditions:* `t ∈ T`.
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies the carrier ℕ, the tumbler length `#·`, and the index domain `{1, …, #t}` in which the separator positions and field-segment clauses are stated.
  - NAT-zero (NatZeroMinimum) — supplies the lower bound `0 ≤ tᵢ` at every position.
  - NAT-discrete (NatDiscreteness) — at `m = 0`, rules out `0 ≤ tᵢ < 1` under `tᵢ ≠ 0`, so the component `tᵢ` at every non-separator position is strictly positive (`0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ`).
- *Postconditions:* T4a (SyntacticEquivalence): the field-segment constraint is equivalent to every field segment of `t` being non-empty. T4c (LevelDetermination): on the T4-valid subset, the four biconditionals define *node address*, *user address*, *document address*, *element address* by zero count; the four labels are pairwise distinct, and each is realised by some T4-valid tumbler. T4b (UniqueParse) is not listed here as a direct postcondition: it combines T4 with T4a's reverse direction to construct and uniquely determine `fields(t)`, and is stated as a downstream theorem in its own section.

**T4b (UniqueParse).** Write `Seq(X)` for the set of finite, possibly empty sequences over `X`, and `ε` for its unique zero-length element. T0 fixes `T` as the *nonempty* finite sequences over `ℕ` — by the stipulation `#a ≥ 1` for every `a ∈ T` — so `T` strictly excludes `ε`; `Seq(·)` differs from T's construction only by adjoining `ε`, and we use `Seq(ℕ⁺)` (with `ℕ⁺ = {n ∈ ℕ : 0 < n}`) precisely because field absence demands a value outside `T`. Under the T4 constraints, the partial function

 `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)`

that decomposes a tumbler into its node, user, document, and element sub-sequences is well-defined and uniquely determined by `t` on the T4-valid subdomain, with absent fields represented by `ε`. Outside that subdomain `fields(t)` is not assigned a value.

We read `fields(t)` as `(N(t), U(t), D(t), E(t))` and fix the projections `N, U, D, E : T ⇀ Seq(ℕ⁺)` accordingly. Each projection returns a finite sequence over `ℕ⁺` when defined: NAT-zero supplies `0 ≤ tᵢ`, NAT-discrete at `m = 0` rules out `0 ≤ tᵢ < 1` when `tᵢ ≠ 0`. A projection may be `ε` — the representation for an *absent* field. *Field absence* is defined independently of the projections as a direct predicate on `zeros(t)`: `N` is never absent on the T4-valid subdomain; `U` is *absent in `t`* iff `zeros(t) = 0`; `D` is *absent in `t`* iff `zeros(t) ≤ 1`; `E` is *absent in `t`* iff `zeros(t) ≤ 2`. The biconditional `X(t) = ε` iff field `X` is absent in `t` follows by inspection of the case construction below: in each case the projections set to `ε` are exactly those the predicate marks absent.

Absence pattern by case on `zeros(t) ∈ {0, 1, 2, 3}`:

  - `zeros(t) = 0`: `N(t) = (t₁, ..., t_{#t})`, `U(t) = D(t) = E(t) = ε`.
  - `zeros(t) = 1`: `N(t)` and `U(t)` non-empty, `D(t) = E(t) = ε`.
  - `zeros(t) = 2`: `N(t)`, `U(t)`, `D(t)` non-empty, `E(t) = ε`.
  - `zeros(t) = 3`: all four projections non-empty.

The component-access notation `t.X₁` denotes the first component of `X(t)` and is defined iff `X(t) ≠ ε`: `t.N₁` always defined on T4-valid `t`; `t.U₁` iff `zeros(t) ≥ 1`; `t.D₁` iff `zeros(t) ≥ 2`; `t.E₁` iff `zeros(t) = 3`.

The well-definedness argument: under T4's role-assignment, separator positions are exactly recoverable from the raw tumbler. Forward: every separator has value 0. Reverse: if `tᵢ = 0` then `i` is a separator; non-zero positions carry field components. So `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` is exactly the separator set.

Let the zero positions in increasing order be `s₁ < s₂ < ... < s_k` (NAT-order licenses the enumeration), `k = zeros(t) ∈ {0, 1, 2, 3}`. Since `t` is T4-valid, T4a's reverse direction applied to the field-segment constraint makes every field segment of `t` non-empty; re-expressed in terms of zero positions, `s₁ ≥ 2` and `s_k ≤ #t - 1` (when `k ≥ 1`), and `s_{j+1} ≥ s_j + 2` for every interior `j`. Compute `fields(t)` case by case, citing the relevant inequality for each segment:

  - *Case k = 0.* `N(t) = (t₁, ..., t_{#t})`, others `ε`. The sole segment spans indices 1 through `#t`, non-empty because `#t ≥ 1` (T0).
  - *Case k = 1.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{#t})`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₁ ≤ #t - 1` makes `U(t)` non-empty.
  - *Case k = 2.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{#t})`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₂ ≥ s₁ + 2` makes `U(t)` non-empty; `s₂ ≤ #t - 1` makes `D(t)` non-empty.
  - *Case k = 3.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{s₃ - 1})`, `E(t) = (t_{s₃ + 1}, ..., t_{#t})`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₂ ≥ s₁ + 2` makes `U(t)` non-empty; `s₃ ≥ s₂ + 2` makes `D(t)` non-empty; `s₃ ≤ #t - 1` makes `E(t)` non-empty.

By T0, each `tᵢ` is the value of the `i`-th component of `t` — a function of `t` and `i` — so the set `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` of separator positions is determined by `t`, and with it the field boundaries and the sub-sequences. Therefore `fields(t)` is well-defined and unique. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints.
- *Definition:* `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)` is the partial function whose domain is the T4-valid subset of `T` and whose action splits the component sequence of `t` at its zero positions. Let `s₁ < s₂ < ... < s_k` enumerate the zero positions, `k = zeros(t) ∈ {0, 1, 2, 3}`, with sentinels `s₀ = 0` and `s_{k+1} = #t + 1`; the `(i+1)`-th field segment (for `0 ≤ i ≤ k`) is `(t_{s_i + 1}, ..., t_{s_{i+1} - 1})`. `fields(t) = (N(t), U(t), D(t), E(t))` assigns these `k + 1` field segments to its first `k + 1` components and `ε` to the remaining `3 - k`. Outside the T4-valid subdomain no value is assigned.
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies ℕ, and `#t ≥ 1` in case `k = 0`.
  - NAT-zero (NatZeroMinimum) — lower bound `0 ≤ tᵢ`.
  - NAT-discrete (NatDiscreteness) — at `m = 0`, promotes "non-zero" to "strictly positive".
  - NAT-order (NatStrictTotalOrder) — licenses the strictly increasing enumeration of separator positions and the inequality manipulations on them.
  - T4 (HierarchicalParsing) — zero-count bound, field-segment clauses, role-assignment of zeros as separators.
  - T4a (SyntacticEquivalence) — field-segment non-emptiness (reverse direction of T4's field-segment constraint) yielding `s₁ ≥ 2`, `s_k ≤ #t - 1`, `s_{j+1} ≥ s_j + 2`.
- *Postconditions:* `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)` is a partial function whose domain is the T4-valid subset of `T`. On that subdomain `fields(t) = (N(t), U(t), D(t), E(t))` is well-defined and uniquely determined by `t`. Each projection `N, U, D, E : T ⇀ Seq(ℕ⁺)` shares the T4-valid subdomain and returns a finite sequence over `ℕ⁺`; absence is represented by `ε`, with absence defined directly in terms of `zeros(t)` (`N` never absent on the T4-valid subdomain; `U` absent iff `zeros(t) = 0`; `D` absent iff `zeros(t) ≤ 1`; `E` absent iff `zeros(t) ≤ 2`), and the biconditional `X(t) = ε` iff `X` is absent in `t` discharged by the case construction on `zeros(t)`. Absence pattern: `zeros(t) = 0` → only `N(t)` non-empty; `zeros(t) = 1` → `N(t), U(t)` non-empty; `zeros(t) = 2` → `N(t), U(t), D(t)` non-empty; `zeros(t) = 3` → all four non-empty. `t.X₁` is defined iff `X(t) ≠ ε`: `t.N₁` always; `t.U₁` iff `zeros(t) ≥ 1`; `t.D₁` iff `zeros(t) ≥ 2`; `t.E₁` iff `zeros(t) = 3`. No value is assigned outside the T4-valid subdomain.

**T4c (LevelDetermination).** With `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|` as in T4 (the cardinality `|·|` of a finite subset of ℕ, distinct from T0's tumbler-length `#·`), on the T4-valid subset of `T`, T4c assigns the four hierarchical level labels definitionally by zero count: `t` is a *node address* iff `zeros(t) = 0`, a *user address* iff `zeros(t) = 1`, a *document address* iff `zeros(t) = 2`, an *element address* iff `zeros(t) = 3`. Outside this subdomain no level is assigned.

The proof obligation reduces to three elementary facts: exhaustion, injectivity, and surjectivity.

*Exhaustion.* By T4, `zeros(t) ≤ 3`; NAT-zero supplies `0 ≤ zeros(t)`; so `zeros(t) ∈ {0, 1, 2, 3}` on the T4-valid subdomain.

*Injectivity.* The values `0, 1, 2, 3` are pairwise distinct in ℕ: NAT-addcompat's strict successor inequality `n < n + 1`, instantiated at `n ∈ {0, 1, 2}`, yields `0 < 1`, `1 < 2`, and `2 < 3`; NAT-order transitivity chains these to `0 < 1 < 2 < 3`, so `m < n` holds for every pair with `m` preceding `n` in the chain; NAT-order irreflexivity then excludes `m = n` for each such pair. Since `zeros(t)` is single-valued, distinct zero counts yield distinct labels.

*Surjectivity.* Each value is realised by a concrete T4-valid witness:

  - `(1)`: `zeros = 0`, *node address*.
  - `(1, 0, 1)`: `zeros = 1`, *user address*.
  - `(1, 0, 1, 0, 1)`: `zeros = 2`, *document address*.
  - `(1, 0, 1, 0, 1, 0, 1)`: `zeros = 3`, *element address*.

T4c is definitional: the four biconditionals name the zero counts. A field-segment characterisation — e.g., *user address* as "`N(t)` and `U(t)` only non-empty in T4b's `fields(t)`" — is consistent with T4c on the T4-valid subdomain, but the equivalence is a separate downstream theorem. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints.
- *Depends:*
  - T0 (CarrierSetDefinition) — carrier ℕ for `zeros(t)` and for the witnesses.
  - NAT-zero (NatZeroMinimum) — lower bound `0 ≤ zeros(t)` at exhaustion.
  - NAT-order (NatStrictTotalOrder) — transitivity to chain `0 < 1 < 2 < 3` and irreflexivity to exclude equality within the chain, giving pairwise distinctness of `0, 1, 2, 3` at injectivity.
  - NAT-addcompat (NatAdditionOrderAndSuccessor) — strict successor inequality `n < n + 1`, instantiated at `n ∈ {0, 1, 2}` to supply the base links `0 < 1`, `1 < 2`, `2 < 3` in injectivity's chain.
  - T4 (HierarchicalParsing) — `zeros(t) ≤ 3` at exhaustion; field-segment clauses to validate the witnesses.
- *Postconditions:* Universally quantified over the T4-valid subset of `T`. The biconditionals `(A t ∈ T : t is T4-valid :: (zeros(t) = 0 ↔ t is a node address) ∧ (zeros(t) = 1 ↔ t is a user address) ∧ (zeros(t) = 2 ↔ t is a document address) ∧ (zeros(t) = 3 ↔ t is an element address))` define the four labels. The four labels are pairwise distinct, and each is realised by some T4-valid tumbler. No level is assigned outside the T4-valid subdomain.
