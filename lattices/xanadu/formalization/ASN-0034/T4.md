## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this, writing *node* for Nelson's *server* and *element* for his *contents* and retaining *user* and *document* verbatim; the renaming is notational. Define a *field separator* as a position `i` of `t` with `tᵢ = 0`. An address tumbler has the form:

`t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`

where `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every position. The four fields are:

- **Node field** `N₁. ... .Nₐ`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U₁. ... .Uᵦ`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D₁. ... .Dᵧ`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked — "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E₁. ... .Eδ`: identifies the content element. T4 constrains only `0 < Eₗ` at every element-field position; typed interpretations of `E₁` are the concern of downstream ASNs.

A tumbler with zero zeros addresses a node. One zero: a user account. Two zeros: a document. Three zeros: an element. The count of zero-valued components determines the specificity level.

**T4 (HierarchicalParsing).** NAT-card axiomatizes the cardinality operator `|·|` as a well-defined total function on subsets of every initial segment `{1, 2, …, n} ⊆ ℕ` (`n ∈ ℕ`), with codomain ℕ and with `|S|` characterised as the length of the unique strictly increasing enumeration of `S`; NAT-card further separates `|·|` from T0's tumbler-length `#· : T → ℕ`, which acts on sequences rather than sets. Define `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|`; the argument `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` is a subset of the index domain `{1, …, #t} ⊆ ℕ` (T0), so NAT-card applies at `n = #t` and `zeros(t) ∈ ℕ` with `zeros(t) ≤ #t`. Every tumbler `t ∈ T` used as an address satisfies: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. We call the last three conditions collectively the *field-segment constraint*. T0 fixes the carrier as ℕ; NAT-order supplies the strict total order `<` on ℕ and its companion `≤` (defined by `m ≤ n ⟺ m < n ∨ m = n`), which support the zero-count bound `zeros(t) ≤ 3` and the adjacency clause's index range `1 ≤ i < #t`; NAT-closure supplies `1 ∈ ℕ` together with closure of ℕ under addition, grounding the numerals `2 := 1 + 1 ∈ ℕ` and `3 := 2 + 1 ∈ ℕ` so that the bound `zeros(t) ≤ 3` compares two elements of ℕ; and NAT-zero together with NAT-discrete (at `m = 0`) force every non-zero component to be strictly positive; write `ℕ⁺ = {n ∈ ℕ : 0 < n}` for the strictly positive naturals. The *field segments* of `t` are the maximal contiguous sub-sequences of non-zero positions — `k+1` of them when `zeros(t) = k`, separated by the zeros. T4 stipulates that a position `i` is a *field separator* iff `tᵢ = 0`, and that every other position carries a *field component* — strictly positive by the preceding argument; the two roles are decidable by inspecting `tᵢ`.

A direct consequence of T4, the first-class numbered theorem **T4a (SyntacticEquivalence)**, proves that the field-segment constraint is equivalent to the condition that every field segment of `t` is non-empty.

Two further first-class theorems, not direct postconditions of T4 but downstream combinations, round out the parsing picture. **T4b (UniqueParse)** is built on T4a: it introduces the partial function `fields(t)` with domain the T4-valid subset of `T`, splitting `t` at its zero positions into four field sub-sequences (with absent fields represented by the empty sequence `ε`), and proves it well-defined and uniquely determined by `t` — using T4a's reverse direction to supply the conclusion that every field segment is non-empty, which T4b locally re-expresses as the segment-length inequalities that discharge non-emptiness of each extracted sub-sequence. The codomain is given in T4b's section, where the notation `Seq(X)` for finite, possibly empty sequences over `X` is introduced and contrasted with T0's nonempty `T`. **T4c (LevelDetermination)** combines T4 with NAT-addcompat's strict successor inequality `n < n + 1` to assign the four hierarchical level labels definitionally by zero count — `(A t ∈ T : t is T4-valid :: (zeros(t) = 0 ↔ t is a node address) ∧ (zeros(t) = 1 ↔ t is a user address) ∧ (zeros(t) = 2 ↔ t is a document address) ∧ (zeros(t) = 3 ↔ t is an element address))` — and discharges exhaustion of the four zero-count values on the T4-valid subdomain together with pairwise distinctness of the labels. Realisation — existence of a T4-valid tumbler at each zero count — is not claimed: NAT-closure's `1 ∈ ℕ` with NAT-addcompat's `0 < 1` already places `1` in `ℕ⁺`, but the foundations in scope supply no constructor producing tumblers of the required lengths, so T4c stands as a pure definition on whatever T4-valid tumblers exist. Outside the T4-valid subdomain T4c assigns no level.

*Formal Contract:*
- *Axiom:* Valid address tumblers satisfy: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. T4 stipulates that a position `i` of `t` is a *field separator* iff `tᵢ = 0`. The canonical written form of a T4-valid address tumbler is keyed to `zeros(t) ∈ {0, 1, 2, 3}`, with one form per zero count: `zeros(t) = 0` gives `t = N₁. ... .Nₐ`; `zeros(t) = 1` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ`; `zeros(t) = 2` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ`; `zeros(t) = 3` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`. In every case, `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every position present.
- *Preconditions:* `t ∈ T`.
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies the carrier ℕ, the tumbler length `#·`, and the index domain `{1, …, #t}` in which the separator positions and field-segment clauses are stated.
  - NAT-zero (NatZeroMinimum) — supplies the lower bound `0 ≤ tᵢ` at every position.
  - NAT-discrete (NatDiscreteness) — at `m = 0`, rules out `0 ≤ tᵢ < 1` under `tᵢ ≠ 0`, so the component `tᵢ` at every non-separator position is strictly positive (`0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ`).
  - NAT-order (NatStrictTotalOrder) — supplies `<` on ℕ and its companion `≤` (with `m ≤ n ⟺ m < n ∨ m = n`), which underwrite the zero-count bound `zeros(t) ≤ 3` and the adjacency clause `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`.
  - NAT-closure (NatArithmeticClosureAndIdentity) — supplies `1 ∈ ℕ` and closure of ℕ under addition, grounding the numerals `2 := 1 + 1` and `3 := 2 + 1` in ℕ so that `zeros(t) ≤ 3` is a comparison of two ℕ-elements.
  - NAT-card (NatFiniteSetCardinality) — axiomatizes `|·|` on subsets of every initial segment `{1, …, n} ⊆ ℕ` with codomain ℕ, grounding the type `zeros(t) ∈ ℕ` for `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|` and fixing `|·|` as distinct from T0's tumbler-length `#·`.
- *Postconditions:* T4a (SyntacticEquivalence), a direct consequence: the field-segment constraint is equivalent to every field segment of `t` being non-empty.

**T4a (SyntacticEquivalence).** T4's field-segment constraint — stated positionally as (i) no two zeros are adjacent, (ii) `t₁ ≠ 0`, (iii) `t_{#t} ≠ 0` — is equivalent to the condition that every *field segment* of `t` is non-empty, where the field segments are the `zeros(t) + 1` maximal contiguous sub-sequences of non-zero positions delimited by the zeros.

Let `t ∈ T` be a tumbler with `zeros(t) ≤ 3`. Set `k = zeros(t) ∈ {0, 1, 2, 3}` with zeros at positions `s₁ < s₂ < … < s_k` (strict increase by NAT-order; the length `k` of this enumeration equals `zeros(t)` by NAT-card's enumeration characterisation of `|·|` applied at the zero-index subset `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0} ⊆ {1, …, #t}`, whose cardinality T4 identifies with `zeros(t)`). Set `s₀ = 0` and `s_{k+1} = #t + 1` as sentinels. The arithmetic in what follows — the numerals `2` and `3`, the sums `s_i + 1`, `s_i + 2`, and `#t + 1`, and the partial subtraction `#t − 1` — is grounded thus: NAT-closure posits `1 ∈ ℕ` and closes ℕ under addition, so `2 := 1 + 1 ∈ ℕ`, `3 := 2 + 1 ∈ ℕ`, and each of `s_i + 1`, `s_i + 2`, `#t + 1` lies in ℕ; NAT-sub's conditional-closure clause applied at the hypothesis `#t ≥ 1` (T0) and `1 ∈ ℕ` gives `#t − 1 ∈ ℕ`. The `k + 1` field segments of `t` are the index ranges `(s_i, s_{i+1})` for `i = 0, 1, …, k` — i.e. the component sub-sequences `t[s_i + 1 .. s_{i+1} - 1]`. A segment is non-empty precisely when `s_{i+1} ≥ s_i + 2`. Every index not in `{s₁, …, s_k}` is a non-zero position; by T0 the carrier is ℕ, NAT-zero gives `0 ≤ tᵢ`, and NAT-discrete at `m = 0` rules out `0 ≤ tᵢ < 1` under `tᵢ ≠ 0`, so such a position carries a strictly positive value.

*Forward.* Assume every field segment is non-empty. We derive each positional condition.

*Condition (ii): `t₁ ≠ 0`.* The first field segment is `t[1 .. s₁ - 1]` when `k ≥ 1`, or `t[1 .. #t]` when `k = 0`; in either case its first index is 1. Segment non-emptiness gives `s₁ ≥ 2` (when `k ≥ 1`) or `#t ≥ 1` (when `k = 0`), so index 1 lies in the segment and `t₁` is a non-zero component. Hence `t₁ ≠ 0`.

*Condition (iii): `t_{#t} ≠ 0`.* The last field segment is `t[s_k + 1 .. #t]` when `k ≥ 1`, or `t[1 .. #t]` when `k = 0`; in either case its last index is `#t`. Segment non-emptiness gives `s_k ≤ #t - 1` (when `k ≥ 1`) or `#t ≥ 1` (when `k = 0`), so index `#t` lies in the segment and `t_{#t}` is a non-zero component. Hence `t_{#t} ≠ 0`.

*Condition (i): no adjacent zeros.* Suppose for contradiction that `tᵢ = 0 ∧ tᵢ₊₁ = 0` for some `i` with `1 ≤ i < #t`. Then `i` and `i + 1` are consecutive zero positions — say `s_j = i` and `s_{j+1} = i + 1` — and the interior field segment `t[s_j + 1 .. s_{j+1} - 1] = t[i + 1 .. i]` has no indices, so it is empty. This contradicts segment non-emptiness. Hence no two zeros are adjacent.

*Reverse.* Assume conditions (i), (ii), and (iii) hold. We show every field segment is non-empty.

*First segment (`i = 0`).* The segment occupies indices 1 through `s₁ - 1` (or 1 through `#t` when `k = 0`). If `k = 0` the segment equals `t` itself, which has `#t ≥ 1` indices. If `k ≥ 1`, condition (ii) forces `t₁ ≠ 0`, so index 1 is not a zero position, and therefore `s₁ ≥ 2` — the segment has at least one index.

*Last segment (`i = k`).* The segment occupies indices `s_k + 1` through `#t` (when `k ≥ 1`). Condition (iii) forces `t_{#t} ≠ 0`, so index `#t` is not a zero position, and therefore `s_k ≤ #t - 1` — the segment has at least one index.

*Interior segments (`1 ≤ i < k`).* Each such segment occupies indices `s_i + 1` through `s_{i+1} - 1`. Condition (i) forbids `s_{i+1} = s_i + 1`, and since the zero positions are strictly increasing we already have `s_{i+1} ≥ s_i + 1`; together these give `s_{i+1} ≥ s_i + 2`, so the segment has at least one index.

All segments — first, interior, and last — are non-empty. ∎

*Formal Contract:*
- *Preconditions:* `t ∈ T` with `zeros(t) ≤ 3`. The field-segment constraint of full T4-validity is *not* assumed; it appears as the conclusion of the Forward direction and the local hypothesis of the Reverse direction.
- *Depends:*
  - T0 (CarrierSetDefinition) — fixes the carrier as ℕ; supplies `#t ≥ 1` as the precondition for `#t − 1 ∈ ℕ`.
  - NAT-zero (NatZeroMinimum) — supplies the lower bound `0 ≤ tᵢ`.
  - NAT-discrete (NatDiscreteness) — converts `0 ≤ tᵢ` and `tᵢ ≠ 0` into `tᵢ ≥ 1`.
  - NAT-order (NatStrictTotalOrder) — licenses the strictly increasing enumeration `s₁ < s₂ < … < s_k` and the position inequalities.
  - NAT-closure (NatArithmeticClosureAndIdentity) — supplies `1 ∈ ℕ` and closure of ℕ under addition, grounding the numerals `2 := 1 + 1 ∈ ℕ` and `3 := 2 + 1 ∈ ℕ` used in `s₁ ≥ 2` and `zeros(t) ≤ 3`, and the sums `s_i + 1`, `s_i + 2`, `#t + 1` used to name the sentinels and the interior-segment inequality.
  - NAT-sub (NatPartialSubtraction) — conditional-closure clause applied at `#t ≥ 1` (T0) and `1 ∈ ℕ` (NAT-closure) gives `#t − 1 ∈ ℕ`, making `s_k ≤ #t − 1` a comparison of two ℕ-elements.
  - NAT-card (NatFiniteSetCardinality) — via the enumeration characterisation of `|·|`, identifies the length `k` of the strictly increasing enumeration `s₁ < s₂ < … < s_k` of the zero-index subset `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0} ⊆ {1, …, #t}` with `zeros(t)`, so that `k = zeros(t)`.
  - T4 (HierarchicalParsing) — supplies the positional conditions (i), (ii), (iii), the field-segment terminology, and the zero-count bound `zeros(t) ≤ 3`.
- *Postconditions:* The three positional conditions (i), (ii), (iii) hold if and only if every field segment of `t` is non-empty.

**T4b (UniqueParse).** Write `Seq(X)` for the set of finite, possibly empty sequences over `X`, and `ε` for its unique zero-length element. T0 fixes `T` as the *nonempty* finite sequences over `ℕ` — by the stipulation `#a ≥ 1` for every `a ∈ T` — so `T` strictly excludes `ε`; `Seq(·)` differs from T's construction only by adjoining `ε`, and we use `Seq(ℕ⁺)` (with `ℕ⁺ = {n ∈ ℕ : 0 < n}`) precisely because field absence demands a value outside `T`. Under the T4 constraints, the partial function

 `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)`

that decomposes a tumbler into its node, user, document, and element sub-sequences is well-defined and uniquely determined by `t` on the T4-valid subdomain, with absent fields represented by `ε`. Outside that subdomain `fields(t)` is not assigned a value.

We read `fields(t)` as `(N(t), U(t), D(t), E(t))` and fix the projections `N, U, D, E : T ⇀ Seq(ℕ⁺)` accordingly. Each projection returns a finite sequence over `ℕ⁺` when defined: NAT-zero supplies `0 ≤ tᵢ`, NAT-discrete at `m = 0` rules out `0 ≤ tᵢ < 1` when `tᵢ ≠ 0`. A projection may be `ε` — the representation for an *absent* field. *Field absence* is defined independently of the projections as a direct predicate on `zeros(t)`: `N` is never absent on the T4-valid subdomain; `U` is *absent in `t`* iff `zeros(t) = 0`; `D` is *absent in `t`* iff `zeros(t) ≤ 1`; `E` is *absent in `t`* iff `zeros(t) ≤ 2`. The biconditional `X(t) = ε` iff field `X` is absent in `t` follows by inspection of the case construction below: in each case the projections set to `ε` are exactly those the predicate marks absent.

Absence pattern by case on `zeros(t) ∈ {0, 1, 2, 3}`:

  - `zeros(t) = 0`: `N(t) = (t₁, ..., t_{#t})`, `U(t) = D(t) = E(t) = ε`.
  - `zeros(t) = 1`: `N(t)` and `U(t)` non-empty, `D(t) = E(t) = ε`.
  - `zeros(t) = 2`: `N(t)`, `U(t)`, `D(t)` non-empty, `E(t) = ε`.
  - `zeros(t) = 3`: all four projections non-empty.

To access individual components of a field we introduce the notation `t.X₁`, grounded in T0's projection as follows. Since `ℕ⁺ = {n ∈ ℕ : 0 < n} ⊆ ℕ`, a finite sequence over `ℕ⁺` is a finite sequence over `ℕ`; if nonempty, it is therefore an element of `T` by T0's stipulation that `T` is the set of nonempty finite sequences over `ℕ`. Whenever `X(t) ≠ ε` the value `X(t) ∈ Seq(ℕ⁺)` is such a nonempty sequence, so `X(t) ∈ T` with `#(X(t)) ≥ 1`, and T0's component projection `·ᵢ` is defined at every `i ∈ {1, …, #(X(t))}` — in particular at `i = 1`. We set `t.X₁ := (X(t))₁`, so the dot-accessor is notational shorthand for T0's subscript applied to `X(t)`, and `t.X₁` is defined iff `X(t) ≠ ε`: `t.N₁` on every T4-valid `t`; `t.U₁` iff `zeros(t) ≥ 1`; `t.D₁` iff `zeros(t) ≥ 2`; `t.E₁` iff `zeros(t) = 3`.

By T4's stipulation, the separator positions of `t` are exactly `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}`, and every non-separator position carries a field component (strictly positive by NAT-zero and NAT-discrete at `m = 0` on T0's carrier ℕ).

Let the zero positions in increasing order be `s₁ < s₂ < ... < s_k` (NAT-order licenses the strictly increasing enumeration; NAT-card's enumeration characterisation of `|·|`, applied at the zero-index subset `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0} ⊆ {1, …, #t}` whose cardinality T4 identifies with `zeros(t)`, identifies the enumeration length with `zeros(t)`), `k = zeros(t) ∈ {0, 1, 2, 3}`. Since `t` is T4-valid, T4a's reverse direction applied to the field-segment constraint yields its exported conclusion: every field segment of `t` is non-empty. We unpack this locally into the position-inequalities used below. The arithmetic in what follows — the numeral `2`, additions `s_i + 1` and `s_i + 2`, and the partial subtraction `#t − 1` — is grounded thus: NAT-closure posits `1 ∈ ℕ` and closes ℕ under addition, so `2 := 1 + 1 ∈ ℕ` and, since every `s_i` and `#t` lies in ℕ, each `s_i + 1`, `s_i + 2`, and `s_k + 1 ∈ ℕ`; NAT-sub's conditional-closure clause applied at the hypothesis `#t ≥ 1` (T0) and `1 ∈ ℕ` gives `#t − 1 ∈ ℕ`. T4 identifies the field segments as the `k + 1` maximal contiguous non-zero runs delimited by the zero positions and the sequence boundaries. The first segment occupies the indices `1, …, s₁ - 1` (when `k ≥ 1`), so its non-emptiness requires `s₁ ≥ 2`; the last segment occupies the indices `s_k + 1, …, #t` (when `k ≥ 1`), so its non-emptiness requires `s_k ≤ #t - 1`; each interior segment (`1 ≤ j < k`) occupies the indices `s_j + 1, …, s_{j+1} - 1`, so its non-emptiness requires `s_{j+1} ≥ s_j + 2`. Compute `fields(t)` case by case, citing the relevant inequality for each segment:

  - *Case k = 0.* `N(t) = (t₁, ..., t_{#t})`, others `ε`. The sole segment spans indices 1 through `#t`, non-empty because `#t ≥ 1` (T0).
  - *Case k = 1.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{#t})`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₁ ≤ #t - 1` makes `U(t)` non-empty.
  - *Case k = 2.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{#t})`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₂ ≥ s₁ + 2` makes `U(t)` non-empty; `s₂ ≤ #t - 1` makes `D(t)` non-empty.
  - *Case k = 3.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{s₃ - 1})`, `E(t) = (t_{s₃ + 1}, ..., t_{#t})`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₂ ≥ s₁ + 2` makes `U(t)` non-empty; `s₃ ≥ s₂ + 2` makes `D(t)` non-empty; `s₃ ≤ #t - 1` makes `E(t)` non-empty.

By T0, each `tᵢ` is the value of the `i`-th component of `t` — a function of `t` and `i` — so the set `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` of separator positions is determined by `t`, and with it the field boundaries and the sub-sequences. Therefore `fields(t)` is well-defined and unique. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints.
- *Definition:* `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)` is the partial function whose domain is the T4-valid subset of `T` and whose action splits the component sequence of `t` at its zero positions. Let `s₁ < s₂ < ... < s_k` enumerate the zero positions, with `k = zeros(t) ∈ {0, 1, 2, 3}`. The value `fields(t) = (N(t), U(t), D(t), E(t))` is fixed by case on `k`: for `k = 0`, `N(t) = (t₁, ..., t_{#t})` and `U(t) = D(t) = E(t) = ε`; for `k = 1`, `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{#t})`, and `D(t) = E(t) = ε`; for `k = 2`, `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{#t})`, and `E(t) = ε`; for `k = 3`, `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{s₃ - 1})`, and `E(t) = (t_{s₃ + 1}, ..., t_{#t})`. Each index `s_i + 1` lies in ℕ by NAT-closure's successor closure, and each `s_i − 1` lies in ℕ by NAT-sub's conditional-closure clause applied at `s_i ≥ 1` (T0's index domain) and `1 ∈ ℕ` (NAT-closure). The segment-length inequalities `s₁ ≥ 2`, `s_k ≤ #t - 1` (when `k ≥ 1`), and `s_{j+1} ≥ s_j + 2` — obtained above by unpacking T4a's conclusion that every field segment is non-empty, with numeral `2 := 1 + 1 ∈ ℕ` by NAT-closure and `#t − 1 ∈ ℕ` by NAT-sub under `#t ≥ 1` (T0) — keep every index `s_i ± 1` appearing above within `{1, …, #t}`. Outside the T4-valid subdomain no value is assigned.
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies ℕ, and `#t ≥ 1` in case `k = 0`; the index domain `{1, …, #t}` gives `s_i ≥ 1`, discharging the precondition of NAT-sub at `s_i − 1`.
  - NAT-zero (NatZeroMinimum) — lower bound `0 ≤ tᵢ`.
  - NAT-discrete (NatDiscreteness) — at `m = 0`, promotes "non-zero" to "strictly positive".
  - NAT-order (NatStrictTotalOrder) — licenses the strictly increasing enumeration of separator positions and the inequality manipulations on them.
  - NAT-closure (NatArithmeticClosureAndIdentity) — supplies `1 ∈ ℕ` together with closure of ℕ under addition, grounding the numeral `2 := 1 + 1 ∈ ℕ` (used in `s₁ ≥ 2` and `s_{j+1} ≥ s_j + 2`) and the sums `s_i + 1`, `s_i + 2` appearing in the case construction.
  - NAT-sub (NatPartialSubtraction) — conditional-closure clause at the hypothesis `#t ≥ 1` (T0) gives `#t − 1 ∈ ℕ` for the inequality `s_k ≤ #t − 1`, and at `s_i ≥ 1` gives `s_i − 1 ∈ ℕ` for the sub-sequence upper indices `t_{s_i − 1}`.
  - NAT-card (NatFiniteSetCardinality) — via the enumeration characterisation of `|·|`, identifies the length `k` of the strictly increasing enumeration `s₁ < s₂ < … < s_k` of the zero-index subset `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0} ⊆ {1, …, #t}` with `zeros(t)`, so that `k = zeros(t)` in the case analysis on `k`.
  - T4 (HierarchicalParsing) — zero-count bound, field-segment clauses, role-assignment of zeros as separators.
  - T4a (SyntacticEquivalence) — supplies, via its reverse direction applied to T4's field-segment constraint, the conclusion that every field segment of `t` is non-empty; the position-inequalities `s₁ ≥ 2`, `s_k ≤ #t - 1`, and `s_{j+1} ≥ s_j + 2` that discharge non-emptiness of each extracted sub-sequence are a local re-expression of this conclusion.
- *Postconditions:* `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)` is a partial function whose domain is the T4-valid subset of `T`. On that subdomain `fields(t) = (N(t), U(t), D(t), E(t))` is well-defined and uniquely determined by `t`. Each projection `N, U, D, E : T ⇀ Seq(ℕ⁺)` shares the T4-valid subdomain and returns a finite sequence over `ℕ⁺`; absence is represented by `ε`, with absence defined directly in terms of `zeros(t)` (`N` never absent on the T4-valid subdomain; `U` absent iff `zeros(t) = 0`; `D` absent iff `zeros(t) ≤ 1`; `E` absent iff `zeros(t) ≤ 2`), and the biconditional `X(t) = ε` iff `X` is absent in `t` discharged by the case construction on `zeros(t)`. Absence pattern: `zeros(t) = 0` → only `N(t)` non-empty; `zeros(t) = 1` → `N(t), U(t)` non-empty; `zeros(t) = 2` → `N(t), U(t), D(t)` non-empty; `zeros(t) = 3` → all four non-empty. The component-access notation `t.X₁ := (X(t))₁` — T0's component projection at index 1 applied to `X(t)`, which belongs to `T` whenever `X(t) ≠ ε` because `ℕ⁺ ⊆ ℕ` embeds every nonempty element of `Seq(ℕ⁺)` into `T` — is defined iff `X(t) ≠ ε`: `t.N₁` always; `t.U₁` iff `zeros(t) ≥ 1`; `t.D₁` iff `zeros(t) ≥ 2`; `t.E₁` iff `zeros(t) = 3`. No value is assigned outside the T4-valid subdomain.
