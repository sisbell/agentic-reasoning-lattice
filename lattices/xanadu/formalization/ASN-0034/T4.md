## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this. Define a *field separator* as a component with value zero. An address tumbler has the form:

`t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`

where each `Nᵢ, Uⱼ, Dₖ, Eₗ > 0`. The four fields are:

- **Node field** `N₁. ... .Nₐ`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U₁. ... .Uᵦ`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D₁. ... .Dᵧ`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked — "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E₁. ... .Eδ`: identifies the content element. T4 constrains only `Eₗ > 0` at every element-field position; typed interpretations of `E₁` are the concern of downstream ASNs.

A tumbler with zero zeros addresses a node. One zero: a user account. Two zeros: a document. Three zeros: an element. The count of zero-valued components determines the specificity level.

**T4 (HierarchicalParsing).** Define `zeros(t) = #{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}`. Every tumbler `t ∈ T` used as an address satisfies: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. We call the last three conditions collectively the *field-segment constraint*. T0 fixes the carrier as ℕ, and NAT-zero together with NAT-discrete (at `m = 0`) force every non-zero component to be strictly positive; write `ℕ⁺ = {n ∈ ℕ : n > 0}` for the strictly positive naturals. The *field segments* of `t` are the maximal contiguous sub-sequences of non-zero positions — `k+1` of them when `zeros(t) = k`, separated by the zeros — and the field-segment constraint is exactly the requirement that all of them are non-empty. T4 stipulates that zero components are field separators and non-zero components are field components; the two roles are positionally decidable by inspecting `tᵢ`.

Three consequences round out the parsing picture, each a first-class numbered theorem in this ASN:

  - **T4a (SyntacticEquivalence)** proves that the field-segment constraint is equivalent to the condition that every field segment of `t` is non-empty. Proved in `T4a.md` by case analysis on `k = zeros(t) ∈ {0, 1, 2, 3}`.
  - **T4b (UniqueParse)** introduces the partial function `fields(t)` with domain the T4-valid subset of `T` and codomain `Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)`, splitting `t` at its zero positions, and proves it well-defined and uniquely determined by `t`.
  - **T4c (LevelDetermination)** assigns the four hierarchical level labels definitionally by zero count: `(A t ∈ T : t is T4-valid :: (zeros(t) = 0 ↔ t is a node address) ∧ (zeros(t) = 1 ↔ t is a user address) ∧ (zeros(t) = 2 ↔ t is a document address) ∧ (zeros(t) = 3 ↔ t is an element address))`. The induced mapping `{0, 1, 2, 3} → {node, user, document, element}` is a bijection. Outside the T4-valid subdomain no level is assigned.

A subtlety deserves emphasis: the hierarchy is *convention layered over flat arithmetic*, not enforcement by the algebra. The comparison, addition, subtraction, and increment operations treat every mantissa slot identically. There is no `isparent`, `isancestor`, or `ischild` primitive in the arithmetic layer. The hierarchical interpretation is projected onto flat sequences by the allocation machinery and by the field-parsing function `fields(t)`.

Hierarchy is constructed by the allocation machinery. The `.0.` separator is produced when the allocation `depth` parameter equals 2 — creating a child at a *different hierarchical type* than its parent. When creating a same-type child (`depth = 1`), no zero separator is introduced. Gregory confirms: there was even a bug where the first document under an account failed to receive its `.0.` separator — the convention had to be explicitly constructed by the allocator, not enforced by any algebraic invariant. ∎

*Formal Contract:*
- *Axiom:* Valid address tumblers satisfy: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. Positivity of non-zero components follows from T0, NAT-zero, and NAT-discrete. T4 stipulates that a position `i` of `t` is a *field separator* iff `tᵢ = 0`. A T4-valid address tumbler has the canonical written form `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ` where each `Nᵢ, Uⱼ, Dₖ, Eₗ > 0`, with only the first `zeros(t) + 1` field segments present.
- *Preconditions:* T4b requires T3 (CanonicalRepresentation) for fixed component sequences, and T4a (SyntacticEquivalence) for the absence-marker unambiguity clause. T4c requires no inter-consequence precondition.
- *Postconditions:* T4a (SyntacticEquivalence): the field-segment constraint is equivalent to every field segment of `t` being non-empty. T4b (UniqueParse): the partial function `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)` with domain the T4-valid subset is well-defined and uniquely computable from `t`. T4c (LevelDetermination): on the T4-valid subset, the four biconditionals define *node address*, *user address*, *document address*, *element address* by zero count, inducing a bijection `{0, 1, 2, 3} → {node, user, document, element}`.

**T4b (UniqueParse).** Under the T4 constraints, the partial function

 `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)`

that decomposes a tumbler into its node, user, document, and element sub-sequences is well-defined and uniquely determined by `t` on the T4-valid subdomain, with absent fields represented by `ε`. Outside that subdomain `fields(t)` is not assigned a value.

We read `fields(t)` as `(N(t), U(t), D(t), E(t))` and fix the projections `N, U, D, E : T ⇀ Seq(ℕ⁺)` accordingly. Write `ℕ⁺ = {n ∈ ℕ : n > 0}` for the strictly positive naturals. Each projection returns a finite sequence over `ℕ⁺` when defined: NAT-zero supplies `0 ≤ tᵢ`, NAT-discrete at `m = 0` rules out `0 ≤ tᵢ < 1` when `tᵢ ≠ 0`. A projection may be `ε` — the representation for an *absent* field. By T4a, for each `X ∈ {N, U, D, E}` and every T4-valid `t`, `X(t) = ε` iff field `X` is absent in `t`.

Absence pattern by case on `zeros(t) ∈ {0, 1, 2, 3}`:

  - `zeros(t) = 0`: `N(t) = (t₁, ..., t_{#t})`, `U(t) = D(t) = E(t) = ε`.
  - `zeros(t) = 1`: `N(t)` and `U(t)` non-empty, `D(t) = E(t) = ε`.
  - `zeros(t) = 2`: `N(t)`, `U(t)`, `D(t)` non-empty, `E(t) = ε`.
  - `zeros(t) = 3`: all four projections non-empty.

The component-access notation `t.X₁` denotes the first component of `X(t)` and is defined iff `X(t) ≠ ε`: `t.N₁` always defined on T4-valid `t`; `t.U₁` iff `zeros(t) ≥ 1`; `t.D₁` iff `zeros(t) ≥ 2`; `t.E₁` iff `zeros(t) = 3`.

The well-definedness argument: under T4's role-assignment, separator positions are exactly recoverable from the raw tumbler. Forward: every separator has value 0. Reverse: if `tᵢ = 0` then `i` is a separator; non-zero positions carry field components. So `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` is exactly the separator set.

Let the zero positions in increasing order be `s₁ < s₂ < ... < s_k` (NAT-order licenses the enumeration), `k = zeros(t) ∈ {0, 1, 2, 3}`. Compute `fields(t)` case by case:

  - *Case k = 0.* `N(t) = (t₁, ..., t_{#t})`, others `ε`. Non-emptiness from `#t ≥ 1` (T0).
  - *Case k = 1.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{#t})`. `t₁ ≠ 0` forces `s₁ ≥ 2`; `t_{#t} ≠ 0` forces `s₁ ≤ #t - 1`.
  - *Case k = 2.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{#t})`. Interior non-emptiness from no-adjacent-zeros (`s₂ ≥ s₁ + 2`).
  - *Case k = 3.* `N(t), U(t), D(t), E(t)` each non-empty via the full field-segment constraint; `s₂ ≥ s₁ + 2`, `s₃ ≥ s₂ + 2`.

T3 fixes the components of `t`, so the separator positions are uniquely determined, the field boundaries are uniquely determined, and the sub-sequences are uniquely determined. Therefore `fields(t)` is well-defined and unique. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies T3 (CanonicalRepresentation) and the T4 constraints.
- *Definition:* `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)` is the partial function whose domain is the T4-valid subset of `T` and whose action splits the component sequence of `t` at its zero positions. Let `s₁ < s₂ < ... < s_k` enumerate the zero positions, `k = zeros(t) ∈ {0, 1, 2, 3}`, with sentinels `s₀ = 0` and `s_{k+1} = #t + 1`; the `(i+1)`-th field segment (for `0 ≤ i ≤ k`) is `(t_{s_i + 1}, ..., t_{s_{i+1} - 1})`. `fields(t) = (N(t), U(t), D(t), E(t))` assigns these `k + 1` field segments to its first `k + 1` components and `ε` to the remaining `3 - k`. Outside the T4-valid subdomain no value is assigned.
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies ℕ, and `#t ≥ 1` in case `k = 0`.
  - NAT-zero (NatZeroMinimum) — lower bound `0 ≤ tᵢ`.
  - NAT-discrete (NatDiscreteness) — at `m = 0`, promotes "non-zero" to "strictly positive".
  - NAT-order (NatStrictTotalOrder) — strictly increasing enumeration of separator positions and inequalities `s_{j+1} ≥ s_j + 2`, `s₁ ≥ 2`, `s_k ≤ #t - 1`.
  - T3 (CanonicalRepresentation) — fixed component sequences license uniqueness of the parse.
  - T4 (HierarchicalParsing) — zero-count bound, field-segment clauses, role-assignment of zeros as separators.
  - T4a (SyntacticEquivalence) — absence-marker unambiguity: `X(t) = ε` iff field `X` is absent.
- *Postconditions:* `fields : T ⇀ Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺) × Seq(ℕ⁺)` is a partial function whose domain is the T4-valid subset of `T`. On that subdomain `fields(t) = (N(t), U(t), D(t), E(t))` is well-defined and uniquely determined by `t`. Each projection `N, U, D, E : T ⇀ Seq(ℕ⁺)` shares the T4-valid subdomain and returns a finite sequence over `ℕ⁺`; absence is represented by `ε`, unambiguously by T4a. Absence pattern: `zeros(t) = 0` → only `N(t)` non-empty; `zeros(t) = 1` → `N(t), U(t)` non-empty; `zeros(t) = 2` → `N(t), U(t), D(t)` non-empty; `zeros(t) = 3` → all four non-empty. `t.X₁` is defined iff `X(t) ≠ ε`: `t.N₁` always; `t.U₁` iff `zeros(t) ≥ 1`; `t.D₁` iff `zeros(t) ≥ 2`; `t.E₁` iff `zeros(t) = 3`. No value is assigned outside the T4-valid subdomain.

**T4c (LevelDetermination).** Define `zeros(t) = #{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}`. On the T4-valid subset of `T`, T4c assigns the four hierarchical level labels definitionally by zero count: `t` is a *node address* iff `zeros(t) = 0`, a *user address* iff `zeros(t) = 1`, a *document address* iff `zeros(t) = 2`, an *element address* iff `zeros(t) = 3`. Outside this subdomain no level is assigned.

The proof obligation reduces to three elementary facts: exhaustion, injectivity, and surjectivity.

*Exhaustion.* By T4, `zeros(t) ≤ 3`; NAT-zero supplies `0 ≤ zeros(t)`; so `zeros(t) ∈ {0, 1, 2, 3}` on the T4-valid subdomain.

*Injectivity.* The values `0, 1, 2, 3` are pairwise distinct in ℕ (NAT-order trichotomy), and `zeros(t)` is single-valued. So distinct zero counts yield distinct labels.

*Surjectivity.* Each value is realised by a concrete T4-valid witness:

  - `(1)`: `zeros = 0`, *node address*.
  - `(1, 0, 1)`: `zeros = 1`, *user address*.
  - `(1, 0, 1, 0, 1)`: `zeros = 2`, *document address*.
  - `(1, 0, 1, 0, 1, 0, 1)`: `zeros = 3`, *element address*.

T4c is definitional: the four biconditionals name the zero counts. A field-segment characterisation — e.g., *user address* as "`N(t)` and `U(t)` only non-empty in T4b's `fields(t)`" — is consistent with T4c on the T4-valid subdomain, but the equivalence is a separate downstream theorem. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints.
- *Depends:*
  - T0 (CarrierSetDefinition) — carrier ℕ for `zeros(t)` and for the witnesses.
  - NAT-zero (NatZeroMinimum) — lower bound `0 ≤ zeros(t)` at exhaustion.
  - NAT-order (NatStrictTotalOrder) — pairwise distinctness of `0, 1, 2, 3` at injectivity.
  - T4 (HierarchicalParsing) — `zeros(t) ≤ 3` at exhaustion; field-segment clauses to validate the witnesses.
- *Postconditions:* Universally quantified over the T4-valid subset of `T`. The biconditionals `(A t ∈ T : t is T4-valid :: (zeros(t) = 0 ↔ t is a node address) ∧ (zeros(t) = 1 ↔ t is a user address) ∧ (zeros(t) = 2 ↔ t is a document address) ∧ (zeros(t) = 3 ↔ t is an element address))` define the four labels. The induced mapping restricted to the T4-valid subdomain is a bijection `{0, 1, 2, 3} → {node, user, document, element}`. No level is assigned outside the T4-valid subdomain.
