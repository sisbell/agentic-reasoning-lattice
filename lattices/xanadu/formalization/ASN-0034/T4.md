## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this, writing *node* for Nelson's *server* and *element* for his *contents* and retaining *user* and *document* verbatim; the renaming is notational. An address tumbler has up to four fields separated by zero-valued components; which fields are present in a given tumbler is determined by its zero count, and T4's Axiom below gives the per-`k` schema explicitly. The maximal written form — the case with all four fields present, `zeros(t) = 3` — is:

`t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`

where `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every position. The four fields — each of which may be absent at lower zero counts — are:

- **Node field** `N₁. ... .Nₐ`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U₁. ... .Uᵦ`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D₁. ... .Dᵧ`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked — "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E₁. ... .Eδ`: identifies the content element. T4 constrains only `0 < Eₗ` at every element-field position; typed interpretations of `E₁` are the concern of downstream ASNs.

The count of zero-valued components determines the specificity level — for each `k ∈ {0, 1, 2, 3}` at which a T4-valid tumbler has `zeros(t) = k`, T4c is the single definitional site, assigning the label node address (at `k = 0`), user address (at `k = 1`), document address (at `k = 2`), or element address (at `k = 3`). The names appear here only as a forward reference to that definition; they are not defined by the present paragraph.

**T4 (HierarchicalParsing).** NAT-card axiomatizes the cardinality operator `|·|` as a well-defined total function on subsets of every initial segment `{1, 2, …, n} ⊆ ℕ` (`n ∈ ℕ`), with codomain ℕ and with `|S|` characterised as the length of the unique strictly increasing enumeration of `S`; NAT-card further separates `|·|` from T0's tumbler-length `#· : T → ℕ`, which acts on sequences rather than sets. Define `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|`; the argument `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` is a subset of the index domain `{1, …, #t} ⊆ ℕ` (T0), so NAT-card applies at `n = #t` and `zeros(t) ∈ ℕ` with `zeros(t) ≤ #t`. Every tumbler `t ∈ T` used as an address satisfies: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. We call the last three conditions collectively the *field-segment constraint*. T0 fixes the carrier as ℕ; NAT-order supplies the strict total order `<` on ℕ and its companion `≤` (defined by `m ≤ n ⟺ m < n ∨ m = n`), which support the zero-count bound `zeros(t) ≤ 3` and the adjacency clause's index range `1 ≤ i < #t`; NAT-closure supplies `1 ∈ ℕ` together with closure of ℕ under addition, grounding the numerals `2 := 1 + 1 ∈ ℕ` and `3 := 2 + 1 ∈ ℕ` so that the bound `zeros(t) ≤ 3` compares two elements of ℕ; and NAT-zero's disjunction `(A n ∈ ℕ :: 0 < n ∨ 0 = n)` instantiated at each non-separator component `tᵢ` forces strict positivity — T0 places `tᵢ ∈ ℕ`, and excluding the equality branch `0 = tᵢ` via `tᵢ ≠ 0` leaves `0 < tᵢ`, giving `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ`; write `ℕ⁺ = {n ∈ ℕ : 0 < n}` for the strictly positive naturals. The *field segments* of `t` are the maximal contiguous sub-sequences of non-zero positions — `k+1` of them when `zeros(t) = k`, separated by the zeros. By the Axiom below, the zero-valued positions serve as *field separators* and the remaining positions carry *field components* — strictly positive by the preceding argument; the two roles are decidable by inspecting `tᵢ`.

*Exhaustion.* We derive as a Consequence that `zeros(t) ∈ {0, 1, 2, 3}` for every T4-valid tumbler, using the Axiom's bound `zeros(t) ≤ 3`, NAT-zero, NAT-order's exactly-one trichotomy (Consequence), and NAT-discrete; NAT-card's codomain secures `zeros(t) ∈ ℕ` so that trichotomy applies at `zeros(t)`. NAT-zero supplies `0 < zeros(t) ∨ 0 = zeros(t)` — equivalently `0 ≤ zeros(t)` by NAT-order's definition of `≤`. Trichotomy at `(zeros(t), m)` asserts that exactly one of `zeros(t) < m`, `zeros(t) = m`, `m < zeros(t)` holds. The mechanism for excluding the branch `zeros(t) < m` given a lower bound `m ≤ zeros(t)` is uniform: unfolding `≤` to `m < zeros(t) ∨ m = zeros(t)` shows that one of trichotomy's other two alternatives holds, and trichotomy's exactly-one clause — exported by NAT-order as a Consequence of the at-least-one axiom, the disjointness axiom `m < n ⟹ m ≠ n`, and irreflexivity+transitivity — then forbids the third. At `m = 0`: the case `zeros(t) < 0` is excluded by `0 ≤ zeros(t)` via the exactly-one route just described, so either `zeros(t) = 0`, whereupon `zeros(t) ∈ {0, 1, 2, 3}`, or `0 < zeros(t)`, which NAT-discrete promotes to `1 ≤ zeros(t)`. At `m = 1` (continuing the `0 < zeros(t)` branch): `zeros(t) < 1` is excluded by `1 ≤ zeros(t)` by the same route, so either `zeros(t) = 1`, whereupon `zeros(t) ∈ {0, 1, 2, 3}`, or `1 < zeros(t)`, which NAT-discrete promotes to `2 ≤ zeros(t)`. At `m = 2` (continuing the `1 < zeros(t)` branch): `zeros(t) < 2` is excluded by `2 ≤ zeros(t)` by the same route, so either `zeros(t) = 2`, whereupon `zeros(t) ∈ {0, 1, 2, 3}`, or `2 < zeros(t)`, which NAT-discrete promotes to `3 ≤ zeros(t)`. At the final step, trichotomy at `(zeros(t), 3)` is confronted with two bounds: `3 ≤ zeros(t)` excludes `zeros(t) < 3` (unfolding `3 ≤ zeros(t)` to `3 < zeros(t) ∨ 3 = zeros(t)` makes one of the other two trichotomy alternatives hold, forcing `zeros(t) < 3` out by exactly-one), and symmetrically `zeros(t) ≤ 3` excludes `3 < zeros(t)` (unfolding to `zeros(t) < 3 ∨ zeros(t) = 3` makes one of the other two alternatives hold, forcing `3 < zeros(t)` out by exactly-one), leaving `zeros(t) = 3`. Every branch terminates with `zeros(t) ∈ {0, 1, 2, 3}`.

Three downstream claims — **T4a (SyntacticEquivalence)**, **T4b (UniqueParse)**, and **T4c (LevelDetermination)** — build on T4 with additional foundations. T4a relates T4's positional field-segment constraint to segment non-emptiness; T4b introduces the four partial field-projections `N, U, D, E : T ⇀ T` over the T4-valid subdomain; T4c assigns the four hierarchical level labels — node, user, document, and element address — by zero count. Each is stated, proved, and equipped with its own Formal Contract in its own claim section; the present paragraph is a forward reference, not a restatement.

*Formal Contract:*
- *Axiom:* Valid address tumblers satisfy: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. T4 stipulates that a position `i` of `t` is a *field separator* iff `tᵢ = 0`. The canonical written form of a T4-valid address tumbler is given by the following schema, quantified per-`k`: for each `k ∈ ℕ` with `0 ≤ k ≤ 3` at which `zeros(t) = k`, the form is — `k = 0`: `t = N₁. ... .Nₐ`; `k = 1`: `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ`; `k = 2`: `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ`; `k = 3`: `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`. In every case, `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every position present.
- *Consequence:* `zeros(t) ∈ {0, 1, 2, 3}` for every T4-valid tumbler `t` — equivalently, `zeros(t) = 0 ∨ zeros(t) = 1 ∨ zeros(t) = 2 ∨ zeros(t) = 3` — so the Axiom's per-`k` schema collectively covers every T4-valid tumbler. Derived below from the bound `zeros(t) ≤ 3`, NAT-zero's `0 ≤ zeros(t)`, NAT-order's trichotomy, and NAT-discrete; NAT-card's codomain secures `zeros(t) ∈ ℕ` so that trichotomy applies.
- *Preconditions:* `t ∈ T`.
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies the carrier ℕ, the tumbler length `#·`, and the index domain `{1, …, #t}` in which the separator positions and field-segment clauses are stated.
  - NAT-zero (NatZeroMinimum) — supplies the disjunction `(A n ∈ ℕ :: 0 < n ∨ 0 = n)`; instantiated at each non-separator component `tᵢ`, the equality branch `0 = tᵢ` is excluded by `tᵢ ≠ 0`, leaving `0 < tᵢ` (hence `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every non-separator position); instantiated at `zeros(t)`, also supplies `0 < zeros(t) ∨ 0 = zeros(t)` (equivalently `0 ≤ zeros(t)`) as the base step of the Exhaustion Consequence.
  - NAT-discrete (NatDiscreteness) — at each `m ∈ {0, 1, 2}`, promotes `m < zeros(t)` to `m + 1 ≤ zeros(t)` in the Exhaustion Consequence's iterated case analysis so that `0 ≤ zeros(t) ≤ 3` collapses to `{0, 1, 2, 3}`.
  - NAT-order (NatStrictTotalOrder) — supplies `<` on ℕ and its companion `≤` (with `m ≤ n ⟺ m < n ∨ m = n`), which underwrite the zero-count bound `zeros(t) ≤ 3` and the adjacency clause `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; supplies trichotomy (exactly one of `m < n`, `m = n`, `n < m` for `m, n ∈ ℕ`) used at `(zeros(t), m)` for each `m ∈ {0, 1, 2, 3}` in the Exhaustion Consequence's iterated case analysis.
  - NAT-closure (NatArithmeticClosureAndIdentity, this ASN) — supplies `1 ∈ ℕ` and closure of ℕ under addition, grounding the numerals `2 := 1 + 1` and `3 := 2 + 1` in ℕ so that `zeros(t) ≤ 3` is a comparison of two ℕ-elements.
  - NAT-card (NatFiniteSetCardinality, this ASN) — axiomatizes `|·|` on subsets of every initial segment `{1, …, n} ⊆ ℕ` with codomain ℕ, grounding the type `zeros(t) ∈ ℕ` for `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|` and fixing `|·|` as distinct from T0's tumbler-length `#·`.
