## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this, writing *node* for Nelson's *server* and *element* for his *contents* and retaining *user* and *document* verbatim; the renaming is notational. An address tumbler has the form:

`t = N‚ÇÅ. ... .N‚Çê . 0 . U‚ÇÅ. ... .U·µ¶ . 0 . D‚ÇÅ. ... .D·µß . 0 . E‚ÇÅ. ... .EŒ¥`

where `0 < N·µ¢, 0 < U‚±º, 0 < D‚Çñ, 0 < E‚Çó` at every position. The four fields are:

- **Node field** `N‚ÇÅ. ... .N‚Çê`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U‚ÇÅ. ... .U·µ¶`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D‚ÇÅ. ... .D·µß`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked ‚Äî "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E‚ÇÅ. ... .EŒ¥`: identifies the content element. T4 constrains only `0 < E‚Çó` at every element-field position; typed interpretations of `E‚ÇÅ` are the concern of downstream ASNs.

The count of zero-valued components determines the specificity level ‚Äî the zero counts 0, 1, 2, 3 correspond to four levels, for which T4c is the single definitional site, assigning the labels node address, user address, document address, and element address respectively. The names appear here only as a forward reference to that definition; they are not defined by the present paragraph.

**T4 (HierarchicalParsing).** NAT-card axiomatizes the cardinality operator `|¬∑|` as a well-defined total function on subsets of every initial segment `{1, 2, ‚Ä¶, n} ‚äÜ ‚Ñï` (`n ‚àà ‚Ñï`), with codomain ‚Ñï and with `|S|` characterised as the length of the unique strictly increasing enumeration of `S`; NAT-card further separates `|¬∑|` from T0's tumbler-length `#¬∑ : T ‚Üí ‚Ñï`, which acts on sequences rather than sets. Define `zeros(t) = |{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}|`; the argument `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}` is a subset of the index domain `{1, ‚Ä¶, #t} ‚äÜ ‚Ñï` (T0), so NAT-card applies at `n = #t` and `zeros(t) ‚àà ‚Ñï` with `zeros(t) ‚â§ #t`. Every tumbler `t ‚àà T` used as an address satisfies: `zeros(t) ‚â§ 3`; `(A i : 1 ‚â§ i < #t : ¬¨(t·µ¢ = 0 ‚àß t·µ¢‚Çä‚ÇÅ = 0))`; `t‚ÇÅ ‚âÝ 0`; `t_{#t} ‚âÝ 0`. We call the last three conditions collectively the *field-segment constraint*. T0 fixes the carrier as ‚Ñï; NAT-order supplies the strict total order `<` on ‚Ñï and its companion `‚â§` (defined by `m ‚â§ n ‚ü∫ m < n ‚à® m = n`), which support the zero-count bound `zeros(t) ‚â§ 3` and the adjacency clause's index range `1 ‚â§ i < #t`; NAT-closure supplies `1 ‚àà ‚Ñï` together with closure of ‚Ñï under addition, grounding the numerals `2 := 1 + 1 ‚àà ‚Ñï` and `3 := 2 + 1 ‚àà ‚Ñï` so that the bound `zeros(t) ‚â§ 3` compares two elements of ‚Ñï; and NAT-zero together with NAT-discrete (at `m = 0`) force every non-zero component to be strictly positive; write `‚Ñï‚Å∫ = {n ‚àà ‚Ñï : 0 < n}` for the strictly positive naturals. The *field segments* of `t` are the maximal contiguous sub-sequences of non-zero positions ‚Äî `k+1` of them when `zeros(t) = k`, separated by the zeros. By the Axiom below, the zero-valued positions serve as *field separators* and the remaining positions carry *field components* ‚Äî strictly positive by the preceding argument; the two roles are decidable by inspecting `t·µ¢`.

Three first-class numbered theorems, each a downstream combination of T4 with additional foundations rather than a consequence of T4 alone, round out the parsing picture. **T4a (SyntacticEquivalence)** adds NAT-sub ‚Äî whose conditional-closure clause gives `#t ‚àí 1 ‚àà ‚Ñï` under the locally unpacked `#t ‚â• 1` from T0's nonempty qualification of `t ‚àà T` ‚Äî to prove that the field-segment constraint is equivalent to every field segment of `t` being non-empty. **T4b (UniqueParse)** is built on T4a: it introduces four partial functions `N, U, D, E : T ‚áÄ T`, each with domain a subset of the T4-valid tumblers determined by `zeros(t)`, splitting `t` at its zero positions into the corresponding field sub-sequence (each in the all-`‚Ñï‚Å∫`-component subset of `T`), and proves each well-defined and uniquely determined by `t` ‚Äî using T4a's reverse direction to supply the conclusion that every field segment is non-empty, which T4b locally re-expresses as the segment-length inequalities that discharge non-emptiness of each extracted sub-sequence. Field absence is encoded by partiality of the corresponding projection, not by a distinguished empty-sequence marker. **T4c (LevelDetermination)** combines T4 with NAT-addcompat's strict successor inequality `n < n + 1` to assign the four hierarchical level labels definitionally by zero count ‚Äî `(A t ‚àà T : t is T4-valid :: (zeros(t) = 0 ‚Üî t is a node address) ‚àß (zeros(t) = 1 ‚Üî t is a user address) ‚àß (zeros(t) = 2 ‚Üî t is a document address) ‚àß (zeros(t) = 3 ‚Üî t is an element address))` ‚Äî and discharges exhaustion of the four zero-count values on the T4-valid subdomain together with pairwise distinctness of the labels. Realisation ‚Äî existence of a T4-valid tumbler at each zero count ‚Äî is not claimed; T4c stands as a pure definition on whatever T4-valid tumblers exist. Outside the T4-valid subdomain T4c assigns no level.

*Formal Contract:*
- *Axiom:* Valid address tumblers satisfy: `zeros(t) ‚â§ 3`; `(A i : 1 ‚â§ i < #t : ¬¨(t·µ¢ = 0 ‚àß t·µ¢‚Çä‚ÇÅ = 0))`; `t‚ÇÅ ‚âÝ 0`; `t_{#t} ‚âÝ 0`. T4 stipulates that a position `i` of `t` is a *field separator* iff `t·µ¢ = 0`. The canonical written form of a T4-valid address tumbler is given by the following schema, quantified per-`k` and not pre-committing to exhaustion: for each `k ‚àà ‚Ñï` with `0 ‚â§ k ‚â§ 3` at which `zeros(t) = k`, the form is ‚Äî `k = 0`: `t = N‚ÇÅ. ... .N‚Çê`; `k = 1`: `t = N‚ÇÅ. ... .N‚Çê . 0 . U‚ÇÅ. ... .U·µ¶`; `k = 2`: `t = N‚ÇÅ. ... .N‚Çê . 0 . U‚ÇÅ. ... .U·µ¶ . 0 . D‚ÇÅ. ... .D·µß`; `k = 3`: `t = N‚ÇÅ. ... .N‚Çê . 0 . U‚ÇÅ. ... .U·µ¶ . 0 . D‚ÇÅ. ... .D·µß . 0 . E‚ÇÅ. ... .EŒ¥`. In every case, `0 < N·µ¢, 0 < U‚±º, 0 < D‚Çñ, 0 < E‚Çó` at every position present. Exhaustion of `zeros(t) ‚àà {0, 1, 2, 3}` is not asserted here; T4c's Exhaustion paragraph derives it from the bound `zeros(t) ‚â§ 3` using iterated NAT-order trichotomy and NAT-discrete.
- *Preconditions:* `t ‚àà T`.
- *Depends:*
  - T0 (CarrierSetDefinition) ‚Äî supplies the carrier ‚Ñï, the tumbler length `#¬∑`, and the index domain `{1, ‚Ä¶, #t}` in which the separator positions and field-segment clauses are stated.
  - NAT-zero (NatZeroMinimum) ‚Äî supplies the lower bound `0 ‚â§ t·µ¢` at every position.
  - NAT-discrete (NatDiscreteness) ‚Äî at `m = 0`, rules out `0 ‚â§ t·µ¢ < 1` under `t·µ¢ ‚âÝ 0`, so the component `t·µ¢` at every non-separator position is strictly positive (`0 < N·µ¢, 0 < U‚±º, 0 < D‚Çñ, 0 < E‚Çó`).
  - NAT-order (NatStrictTotalOrder) ‚Äî supplies `<` on ‚Ñï and its companion `‚â§` (with `m ‚â§ n ‚ü∫ m < n ‚à® m = n`), which underwrite the zero-count bound `zeros(t) ‚â§ 3` and the adjacency clause `(A i : 1 ‚â§ i < #t : ¬¨(t·µ¢ = 0 ‚àß t·µ¢‚Çä‚ÇÅ = 0))`.
  - NAT-closure (NatArithmeticClosureAndIdentity) ‚Äî supplies `1 ‚àà ‚Ñï` and closure of ‚Ñï under addition, grounding the numerals `2 := 1 + 1` and `3 := 2 + 1` in ‚Ñï so that `zeros(t) ‚â§ 3` is a comparison of two ‚Ñï-elements.
  - NAT-card (NatFiniteSetCardinality) ‚Äî axiomatizes `|¬∑|` on subsets of every initial segment `{1, ‚Ä¶, n} ‚äÜ ‚Ñï` with codomain ‚Ñï, grounding the type `zeros(t) ‚àà ‚Ñï` for `zeros(t) = |{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}|` and fixing `|¬∑|` as distinct from T0's tumbler-length `#¬∑`.

**T4a (SyntacticEquivalence).** T4's field-segment constraint ‚Äî stated positionally as (i) no two zeros are adjacent, (ii) `t‚ÇÅ ‚âÝ 0`, (iii) `t_{#t} ‚âÝ 0` ‚Äî is equivalent to the condition that every *field segment* of `t` is non-empty, where the field segments are the `zeros(t) + 1` maximal contiguous sub-sequences of non-zero positions delimited by the zeros.

Let `t ‚àà T` be a tumbler with `zeros(t) ‚â§ 3`. Set `k = zeros(t) ‚àà {0, 1, 2, 3}` with zeros at positions `s‚ÇÅ < s‚ÇÇ < ‚Ä¶ < s_k` (strict increase by NAT-order; the length `k` of this enumeration equals `zeros(t)` by NAT-card's enumeration characterisation of `|¬∑|` applied at the zero-index subset `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0} ‚äÜ {1, ‚Ä¶, #t}`, whose cardinality T4 identifies with `zeros(t)`). Set `s‚ÇÄ = 0` and `s_{k+1} = #t + 1` as sentinels. The arithmetic in what follows ‚Äî the numerals `2` and `3`, the sums `s_i + 1`, `s_i + 2`, and `#t + 1`, and the partial subtraction `#t ‚àí 1` ‚Äî is grounded thus: NAT-closure posits `1 ‚àà ‚Ñï` and closes ‚Ñï under addition, so `2 := 1 + 1 ‚àà ‚Ñï`, `3 := 2 + 1 ‚àà ‚Ñï`, and each of `s_i + 1`, `s_i + 2`, `#t + 1` lies in ‚Ñï. T0 declares every `t ‚àà T` to be a nonempty finite sequence over ‚Ñï; a nonempty sequence has at least one component, so by the definition of length `#t ‚â• 1` ‚Äî this is a local unpacking performed here, not a postcondition cited from T0. NAT-sub's conditional-closure clause applied at this `#t ‚â• 1` and `1 ‚àà ‚Ñï` gives `#t ‚àí 1 ‚àà ‚Ñï`. The `k + 1` field segments of `t` are the index ranges `(s_i, s_{i+1})` for `i = 0, 1, ‚Ä¶, k` ‚Äî i.e. the component sub-sequences `t[s_i + 1 .. s_{i+1} - 1]`. A segment is non-empty precisely when `s_{i+1} ‚â• s_i + 2`. Every index not in `{s‚ÇÅ, ‚Ä¶, s_k}` is a non-zero position; by T0 the carrier is ‚Ñï, NAT-zero gives `0 ‚â§ t·µ¢`, and NAT-discrete at `m = 0` rules out `0 ‚â§ t·µ¢ < 1` under `t·µ¢ ‚âÝ 0`, so such a position carries a strictly positive value.

*Forward.* Assume every field segment is non-empty. We derive each positional condition.

*Condition (ii): `t‚ÇÅ ‚âÝ 0`.* The first field segment is `t[1 .. s‚ÇÅ - 1]` when `k ‚â• 1`, or `t[1 .. #t]` when `k = 0`; in either case its first index is 1. Segment non-emptiness gives `s‚ÇÅ ‚â• 2` (when `k ‚â• 1`) or `#t ‚â• 1` (when `k = 0`), so index 1 lies in the segment and `t‚ÇÅ` is a non-zero component. Hence `t‚ÇÅ ‚âÝ 0`.

*Condition (iii): `t_{#t} ‚âÝ 0`.* The last field segment is `t[s_k + 1 .. #t]` when `k ‚â• 1`, or `t[1 .. #t]` when `k = 0`; in either case its last index is `#t`. Segment non-emptiness gives `s_k ‚â§ #t - 1` (when `k ‚â• 1`) or `#t ‚â• 1` (when `k = 0`), so index `#t` lies in the segment and `t_{#t}` is a non-zero component. Hence `t_{#t} ‚âÝ 0`.

*Condition (i): no adjacent zeros.* Suppose for contradiction that `t·µ¢ = 0 ‚àß t·µ¢‚Çä‚ÇÅ = 0` for some `i` with `1 ‚â§ i < #t`. Then `i` and `i + 1` are consecutive zero positions ‚Äî say `s_j = i` and `s_{j+1} = i + 1` ‚Äî and the interior field segment `t[s_j + 1 .. s_{j+1} - 1] = t[i + 1 .. i]` has no indices, so it is empty. This contradicts segment non-emptiness. Hence no two zeros are adjacent.

*Reverse.* Assume conditions (i), (ii), and (iii) hold. We show every field segment is non-empty.

*First segment (`i = 0`).* The segment occupies indices 1 through `s‚ÇÅ - 1` (or 1 through `#t` when `k = 0`). If `k = 0` the segment equals `t` itself, which has `#t ‚â• 1` indices. If `k ‚â• 1`, condition (ii) forces `t‚ÇÅ ‚âÝ 0`, so index 1 is not a zero position, and therefore `s‚ÇÅ ‚â• 2` ‚Äî the segment has at least one index.

*Last segment (`i = k`).* The segment occupies indices `s_k + 1` through `#t` (when `k ‚â• 1`). Condition (iii) forces `t_{#t} ‚âÝ 0`, so index `#t` is not a zero position, and therefore `s_k ‚â§ #t - 1` ‚Äî the segment has at least one index.

*Interior segments (`1 ‚â§ i < k`).* Each such segment occupies indices `s_i + 1` through `s_{i+1} - 1`. Condition (i) forbids `s_{i+1} = s_i + 1`, and since the zero positions are strictly increasing we already have `s_{i+1} ‚â• s_i + 1`; together these give `s_{i+1} ‚â• s_i + 2`, so the segment has at least one index.

All segments ‚Äî first, interior, and last ‚Äî are non-empty. ‚àé

*Formal Contract:*
- *Preconditions:* `t ‚àà T` with `zeros(t) ‚â§ 3`. The field-segment constraint of full T4-validity is *not* assumed; it appears as the conclusion of the Forward direction and the local hypothesis of the Reverse direction.
- *Depends:*
  - T0 (CarrierSetDefinition) ‚Äî fixes the carrier as ‚Ñï and qualifies every `t ‚àà T` as a nonempty finite sequence over ‚Ñï, from which T4a locally unpacks `#t ‚â• 1` (the length of a nonempty sequence is at least 1) as the precondition for `#t ‚àí 1 ‚àà ‚Ñï`.
  - NAT-zero (NatZeroMinimum) ‚Äî supplies the lower bound `0 ‚â§ t·µ¢`.
  - NAT-discrete (NatDiscreteness) ‚Äî converts `0 ‚â§ t·µ¢` and `t·µ¢ ‚âÝ 0` into `t·µ¢ ‚â• 1`.

  - NAT-sub (NatPartialSubtraction) ‚Äî conditional-closure clause applied at the locally unpacked `#t ‚â• 1` (from T0's nonempty qualification of `t ‚àà T`) and `1 ‚àà ‚Ñï` (NAT-closure) gives `#t ‚àí 1 ‚àà ‚Ñï`, making `s_k ‚â§ #t ‚àí 1` a comparison of two ‚Ñï-elements.
  - NAT-card (NatFiniteSetCardinality) ‚Äî via the enumeration characterisation of `|¬∑|`, identifies the length `k` of the strictly increasing enumeration `s‚ÇÅ < s‚ÇÇ < ‚Ä¶ < s_k` of the zero-index subset `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0} ‚äÜ {1, ‚Ä¶, #t}` with `zeros(t)`, so that `k = zeros(t)`.

**T4b (UniqueParse).** Under the constraints of T4, the four partial functions

 `N, U, D, E : T ‚áÄ T`

that extract the node, user, document, and element sub-sequences of `t` are well-defined and uniquely determined by `t`. Writing `‚Ñï‚Å∫ = {n ‚àà ‚Ñï : 0 < n}`, each projection's image lies in the subset of `T` whose every component is in `‚Ñï‚Å∫`: a nonempty finite sequence over `‚Ñï` (by T0) with every component strictly positive by NAT-zero and NAT-discrete at `m = 0`. The four projections share the T4-valid subset of `T` as the outer domain from which absence is carved; field *absence* is encoded by partiality of the projection, and is fixed by `zeros(t)`: `N` is defined on the whole T4-valid subdomain (`N` is never absent); `U` is defined iff `t` is T4-valid and `zeros(t) ‚â• 1`; `D` is defined iff `t` is T4-valid and `zeros(t) ‚â• 2`; `E` is defined iff `t` is T4-valid and `zeros(t) = 3`. A field `X` is *absent in `t`* iff `t ‚àâ dom(X)`.

Presence pattern by case on `zeros(t) ‚àà {0, 1, 2, 3}`:

  - `zeros(t) = 0`: only `N(t)` defined, with `N(t) = (t‚ÇÅ, ..., t_{#t})`.
  - `zeros(t) = 1`: `N(t)` and `U(t)` defined.
  - `zeros(t) = 2`: `N(t)`, `U(t)`, `D(t)` defined.
  - `zeros(t) = 3`: all four projections defined.

To access individual components of a field we introduce the notation `t.X‚ÇÅ`, grounded in T0's projection as follows. Whenever `X(t)` is defined, `X(t)` is a nonempty finite sequence over `‚Ñï‚Å∫ ‚äÜ ‚Ñï`, so `X(t) ‚àà T` with `#(X(t)) ‚â• 1` by T0, and T0's component projection `¬∑·µ¢` is defined at every `i ‚àà {1, ‚Ä¶, #(X(t))}` ‚Äî in particular at `i = 1`. We set `t.X‚ÇÅ := (X(t))‚ÇÅ`, so the dot-accessor is notational shorthand for T0's subscript applied to `X(t)`, and `t.X‚ÇÅ` is defined iff `X(t)` is defined: `t.N‚ÇÅ` on every T4-valid `t`; `t.U‚ÇÅ` iff `zeros(t) ‚â• 1`; `t.D‚ÇÅ` iff `zeros(t) ‚â• 2`; `t.E‚ÇÅ` iff `zeros(t) = 3`.

By T4's stipulation, the separator positions of `t` are exactly `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}`, and every non-separator position carries a field component (strictly positive by NAT-zero and NAT-discrete at `m = 0` on T0's carrier ‚Ñï).

Let the zero positions in increasing order be `s‚ÇÅ < s‚ÇÇ < ... < s_k` (NAT-order licenses the strictly increasing enumeration; NAT-card's enumeration characterisation of `|¬∑|`, applied at the zero-index subset `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0} ‚äÜ {1, ‚Ä¶, #t}` whose cardinality T4 identifies with `zeros(t)`, identifies the enumeration length with `zeros(t)`), `k = zeros(t) ‚àà {0, 1, 2, 3}`. Since `t` is T4-valid, T4a's reverse direction applied to the field-segment constraint yields its exported conclusion: every field segment of `t` is non-empty. We unpack this locally into the position-inequalities used below. The arithmetic in what follows ‚Äî the numeral `2`, additions `s_i + 1` and `s_i + 2`, and the partial subtraction `#t ‚àí 1` ‚Äî is grounded thus: NAT-closure posits `1 ‚àà ‚Ñï` and closes ‚Ñï under addition, so `2 := 1 + 1 ‚àà ‚Ñï` and, since every `s_i` and `#t` lies in ‚Ñï, each `s_i + 1`, `s_i + 2`, and `s_k + 1 ‚àà ‚Ñï`. T0 declares every `t ‚àà T` to be a nonempty finite sequence over ‚Ñï; a nonempty sequence has at least one component, so by the definition of length `#t ‚â• 1` ‚Äî this is a local unpacking performed here, not a postcondition cited from T0. NAT-sub's conditional-closure clause applied at this `#t ‚â• 1` and `1 ‚àà ‚Ñï` gives `#t ‚àí 1 ‚àà ‚Ñï`. T4 identifies the field segments as the `k + 1` maximal contiguous non-zero runs delimited by the zero positions and the sequence boundaries. The first segment occupies the indices `1, ‚Ä¶, s‚ÇÅ - 1` (when `k ‚â• 1`), so its non-emptiness requires `s‚ÇÅ ‚â• 2`; the last segment occupies the indices `s_k + 1, ‚Ä¶, #t` (when `k ‚â• 1`), so its non-emptiness requires `s_k ‚â§ #t - 1`; each interior segment (`1 ‚â§ j < k`) occupies the indices `s_j + 1, ‚Ä¶, s_{j+1} - 1`, so its non-emptiness requires `s_{j+1} ‚â• s_j + 2`. Compute the projections case by case, citing the relevant inequality for each segment:

  - *Case k = 0.* `N(t) = (t‚ÇÅ, ..., t_{#t})`; `U, D, E` undefined at `t`. The sole segment spans indices 1 through `#t`, non-empty because `#t ‚â• 1` ‚Äî the local unpacking above, drawn from T0's nonempty qualification of `t ‚àà T`.
  - *Case k = 1.* `N(t) = (t‚ÇÅ, ..., t_{s‚ÇÅ - 1})`, `U(t) = (t_{s‚ÇÅ + 1}, ..., t_{#t})`; `D, E` undefined at `t`. `s‚ÇÅ ‚â• 2` makes `N(t)` non-empty; `s‚ÇÅ ‚â§ #t - 1` makes `U(t)` non-empty.
  - *Case k = 2.* `N(t) = (t‚ÇÅ, ..., t_{s‚ÇÅ - 1})`, `U(t) = (t_{s‚ÇÅ + 1}, ..., t_{s‚ÇÇ - 1})`, `D(t) = (t_{s‚ÇÇ + 1}, ..., t_{#t})`; `E` undefined at `t`. `s‚ÇÅ ‚â• 2` makes `N(t)` non-empty; `s‚ÇÇ ‚â• s‚ÇÅ + 2` makes `U(t)` non-empty; `s‚ÇÇ ‚â§ #t - 1` makes `D(t)` non-empty.
  - *Case k = 3.* `N(t) = (t‚ÇÅ, ..., t_{s‚ÇÅ - 1})`, `U(t) = (t_{s‚ÇÅ + 1}, ..., t_{s‚ÇÇ - 1})`, `D(t) = (t_{s‚ÇÇ + 1}, ..., t_{s‚ÇÉ - 1})`, `E(t) = (t_{s‚ÇÉ + 1}, ..., t_{#t})`. `s‚ÇÅ ‚â• 2` makes `N(t)` non-empty; `s‚ÇÇ ‚â• s‚ÇÅ + 2` makes `U(t)` non-empty; `s‚ÇÉ ‚â• s‚ÇÇ + 2` makes `D(t)` non-empty; `s‚ÇÉ ‚â§ #t - 1` makes `E(t)` non-empty.

By T0, each `t·µ¢` is the value of the `i`-th component of `t` ‚Äî a function of `t` and `i` ‚Äî so the set `{i : 1 ‚â§ i ‚â§ #t ‚àß t·µ¢ = 0}` of separator positions is determined by `t`, and with it the field boundaries and the sub-sequences. Therefore each projection is well-defined and unique on its stated domain. ‚àé
