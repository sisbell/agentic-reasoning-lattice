## Hierarchical structure

Tumblers encode a containment hierarchy. Nelson uses zero-valued components as structural delimiters:

> "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents."

> "A tumbler address may have at most three zero digits... These are interpreted as the major dividers, and have lexical significance as punctuation."

We formalize this, writing *node* for Nelson's *server* and *element* for his *contents* and retaining *user* and *document* verbatim; the renaming is notational. An address tumbler has the form:

`t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`

where `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every position. The four fields are:

- **Node field** `N₁. ... .Nₐ`: identifies the server. "The server address always begins with the digit 1, since all other servers are descended from it."
- **User field** `U₁. ... .Uᵦ`: identifies the account. "Typically, the user will have no control over the node address he, she or it is assigned; but once assigned a User account, the user will have full control over its subdivision forevermore."
- **Document field** `D₁. ... .Dᵧ`: identifies the document and version. Nelson notes the boundary between base document and version is not syntactically marked — "the version, or subdocument number is only an accidental extension of the document number, and strictly implies no specific relationship of derivation."
- **Element field** `E₁. ... .Eδ`: identifies the content element. T4 constrains only `0 < Eₗ` at every element-field position; typed interpretations of `E₁` are the concern of downstream ASNs.

A tumbler with zero zeros is a *node address*. One zero: a *user address*. Two zeros: a *document address*. Three zeros: an *element address*. The count of zero-valued components determines the specificity level, and these are the level labels T4c fixes formally.

**T4 (HierarchicalParsing).** NAT-card axiomatizes the cardinality operator `|·|` as a well-defined total function on subsets of every initial segment `{1, 2, …, n} ⊆ ℕ` (`n ∈ ℕ`), with codomain ℕ and with `|S|` characterised as the length of the unique strictly increasing enumeration of `S`; NAT-card further separates `|·|` from T0's tumbler-length `#· : T → ℕ`, which acts on sequences rather than sets. Define `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|`; the argument `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` is a subset of the index domain `{1, …, #t} ⊆ ℕ` (T0), so NAT-card applies at `n = #t` and `zeros(t) ∈ ℕ` with `zeros(t) ≤ #t`. Every tumbler `t ∈ T` used as an address satisfies: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. We call the last three conditions collectively the *field-segment constraint*. T0 fixes the carrier as ℕ; NAT-order supplies the strict total order `<` on ℕ and its companion `≤` (defined by `m ≤ n ⟺ m < n ∨ m = n`), which support the zero-count bound `zeros(t) ≤ 3` and the adjacency clause's index range `1 ≤ i < #t`; NAT-closure supplies `1 ∈ ℕ` together with closure of ℕ under addition, grounding the numerals `2 := 1 + 1 ∈ ℕ` and `3 := 2 + 1 ∈ ℕ` so that the bound `zeros(t) ≤ 3` compares two elements of ℕ; and NAT-zero together with NAT-discrete (at `m = 0`) force every non-zero component to be strictly positive; write `ℕ⁺ = {n ∈ ℕ : 0 < n}` for the strictly positive naturals. The *field segments* of `t` are the maximal contiguous sub-sequences of non-zero positions — `k+1` of them when `zeros(t) = k`, separated by the zeros. By the Axiom below, the zero-valued positions serve as *field separators* and the remaining positions carry *field components* — strictly positive by the preceding argument; the two roles are decidable by inspecting `tᵢ`.

Three first-class numbered theorems, each a downstream combination of T4 with additional foundations rather than a consequence of T4 alone, round out the parsing picture. **T4a (SyntacticEquivalence)** adds NAT-sub — whose conditional-closure clause gives `#t − 1 ∈ ℕ` under the locally unpacked `#t ≥ 1` from T0's nonempty qualification of `t ∈ T` — to prove that the field-segment constraint is equivalent to every field segment of `t` being non-empty. **T4b (UniqueParse)** is built on T4a: it introduces four partial functions `N, U, D, E : T ⇀ T`, each with domain a subset of the T4-valid tumblers determined by `zeros(t)`, splitting `t` at its zero positions into the corresponding field sub-sequence (each in the all-`ℕ⁺`-component subset of `T`), and proves each well-defined and uniquely determined by `t` — using T4a's reverse direction to supply the conclusion that every field segment is non-empty, which T4b locally re-expresses as the segment-length inequalities that discharge non-emptiness of each extracted sub-sequence. Field absence is encoded by partiality of the corresponding projection, not by a distinguished empty-sequence marker. **T4c (LevelDetermination)** combines T4 with NAT-addcompat's strict successor inequality `n < n + 1` to assign the four hierarchical level labels definitionally by zero count — `(A t ∈ T : t is T4-valid :: (zeros(t) = 0 ↔ t is a node address) ∧ (zeros(t) = 1 ↔ t is a user address) ∧ (zeros(t) = 2 ↔ t is a document address) ∧ (zeros(t) = 3 ↔ t is an element address))` — and discharges exhaustion of the four zero-count values on the T4-valid subdomain together with pairwise distinctness of the labels. Realisation — existence of a T4-valid tumbler at each zero count — is not claimed: NAT-closure's `1 ∈ ℕ` with NAT-addcompat's `0 < 1` already places `1` in `ℕ⁺`, but the foundations in scope supply no constructor producing tumblers of the required lengths, so T4c stands as a pure definition on whatever T4-valid tumblers exist. Outside the T4-valid subdomain T4c assigns no level.

*Formal Contract:*
- *Axiom:* Valid address tumblers satisfy: `zeros(t) ≤ 3`; `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`; `t₁ ≠ 0`; `t_{#t} ≠ 0`. T4 stipulates that a position `i` of `t` is a *field separator* iff `tᵢ = 0`. The canonical written form of a T4-valid address tumbler is keyed to `zeros(t) ∈ {0, 1, 2, 3}`, with one form per zero count: `zeros(t) = 0` gives `t = N₁. ... .Nₐ`; `zeros(t) = 1` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ`; `zeros(t) = 2` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ`; `zeros(t) = 3` gives `t = N₁. ... .Nₐ . 0 . U₁. ... .Uᵦ . 0 . D₁. ... .Dᵧ . 0 . E₁. ... .Eδ`. In every case, `0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ` at every position present.
- *Preconditions:* `t ∈ T`.
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies the carrier ℕ, the tumbler length `#·`, and the index domain `{1, …, #t}` in which the separator positions and field-segment clauses are stated.
  - NAT-zero (NatZeroMinimum) — supplies the lower bound `0 ≤ tᵢ` at every position.
  - NAT-discrete (NatDiscreteness) — at `m = 0`, rules out `0 ≤ tᵢ < 1` under `tᵢ ≠ 0`, so the component `tᵢ` at every non-separator position is strictly positive (`0 < Nᵢ, 0 < Uⱼ, 0 < Dₖ, 0 < Eₗ`).
  - NAT-order (NatStrictTotalOrder) — supplies `<` on ℕ and its companion `≤` (with `m ≤ n ⟺ m < n ∨ m = n`), which underwrite the zero-count bound `zeros(t) ≤ 3` and the adjacency clause `(A i : 1 ≤ i < #t : ¬(tᵢ = 0 ∧ tᵢ₊₁ = 0))`.
  - NAT-closure (NatArithmeticClosureAndIdentity) — supplies `1 ∈ ℕ` and closure of ℕ under addition, grounding the numerals `2 := 1 + 1` and `3 := 2 + 1` in ℕ so that `zeros(t) ≤ 3` is a comparison of two ℕ-elements.
  - NAT-card (NatFiniteSetCardinality) — axiomatizes `|·|` on subsets of every initial segment `{1, …, n} ⊆ ℕ` with codomain ℕ, grounding the type `zeros(t) ∈ ℕ` for `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|` and fixing `|·|` as distinct from T0's tumbler-length `#·`.

**T4a (SyntacticEquivalence).** T4's field-segment constraint — stated positionally as (i) no two zeros are adjacent, (ii) `t₁ ≠ 0`, (iii) `t_{#t} ≠ 0` — is equivalent to the condition that every *field segment* of `t` is non-empty, where the field segments are the `zeros(t) + 1` maximal contiguous sub-sequences of non-zero positions delimited by the zeros.

Let `t ∈ T` be a tumbler with `zeros(t) ≤ 3`. Set `k = zeros(t) ∈ {0, 1, 2, 3}` with zeros at positions `s₁ < s₂ < … < s_k` (strict increase by NAT-order; the length `k` of this enumeration equals `zeros(t)` by NAT-card's enumeration characterisation of `|·|` applied at the zero-index subset `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0} ⊆ {1, …, #t}`, whose cardinality T4 identifies with `zeros(t)`). Set `s₀ = 0` and `s_{k+1} = #t + 1` as sentinels. The arithmetic in what follows — the numerals `2` and `3`, the sums `s_i + 1`, `s_i + 2`, and `#t + 1`, and the partial subtraction `#t − 1` — is grounded thus: NAT-closure posits `1 ∈ ℕ` and closes ℕ under addition, so `2 := 1 + 1 ∈ ℕ`, `3 := 2 + 1 ∈ ℕ`, and each of `s_i + 1`, `s_i + 2`, `#t + 1` lies in ℕ. T0 declares every `t ∈ T` to be a nonempty finite sequence over ℕ; a nonempty sequence has at least one component, so by the definition of length `#t ≥ 1` — this is a local unpacking performed here, not a postcondition cited from T0. NAT-sub's conditional-closure clause applied at this `#t ≥ 1` and `1 ∈ ℕ` gives `#t − 1 ∈ ℕ`. The `k + 1` field segments of `t` are the index ranges `(s_i, s_{i+1})` for `i = 0, 1, …, k` — i.e. the component sub-sequences `t[s_i + 1 .. s_{i+1} - 1]`. A segment is non-empty precisely when `s_{i+1} ≥ s_i + 2`. Every index not in `{s₁, …, s_k}` is a non-zero position; by T0 the carrier is ℕ, NAT-zero gives `0 ≤ tᵢ`, and NAT-discrete at `m = 0` rules out `0 ≤ tᵢ < 1` under `tᵢ ≠ 0`, so such a position carries a strictly positive value.

*Forward.* Assume every field segment is non-empty. We derive each positional condition.

*Condition (ii): `t₁ ≠ 0`.* The first field segment is `t[1 .. s₁ - 1]` when `k ≥ 1`, or `t[1 .. #t]` when `k = 0`; in either case its first index is 1. Segment non-emptiness gives `s₁ ≥ 2` (when `k ≥ 1`) or `#t ≥ 1` (when `k = 0`), so index 1 lies in the segment and `t₁` is a non-zero component. Hence `t₁ ≠ 0`.

*Condition (iii): `t_{#t} ≠ 0`.* The last field segment is `t[s_k + 1 .. #t]` when `k ≥ 1`, or `t[1 .. #t]` when `k = 0`; in either case its last index is `#t`. Segment non-emptiness gives `s_k ≤ #t - 1` (when `k ≥ 1`) or `#t ≥ 1` (when `k = 0`), so index `#t` lies in the segment and `t_{#t}` is a non-zero component. Hence `t_{#t} ≠ 0`.

*Condition (i): no adjacent zeros.* Suppose for contradiction that `tᵢ = 0 ∧ tᵢ₊₁ = 0` for some `i` with `1 ≤ i < #t`. Then `i` and `i + 1` are consecutive zero positions — say `s_j = i` and `s_{j+1} = i + 1` — and the interior field segment `t[s_j + 1 .. s_{j+1} - 1] = t[i + 1 .. i]` has no indices, so it is empty. This contradicts segment non-emptiness. Hence no two zeros are adjacent.

*Reverse.* Assume conditions (i), (ii), and (iii) hold. We show every field segment is non-empty.

*First segment (`i = 0`).* The segment occupies indices 1 through `s₁ - 1` (or 1 through `#t` when `k = 0`). If `k = 0` the segment equals `t` itself, which has `#t ≥ 1` indices. If `k ≥ 1`, condition (ii) forces `t₁ ≠ 0`, so index 1 is not a zero position, and therefore `s₁ ≥ 2` — the segment has at least one index.

*Last segment (`i = k`).* The segment occupies indices `s_k + 1` through `#t` (when `k ≥ 1`). Condition (iii) forces `t_{#t} ≠ 0`, so index `#t` is not a zero position, and therefore `s_k ≤ #t - 1` — the segment has at least one index.

*Interior segments (`1 ≤ i < k`).* Each such segment occupies indices `s_i + 1` through `s_{i+1} - 1`. Condition (i) forbids `s_{i+1} = s_i + 1`, and since the zero positions are strictly increasing we already have `s_{i+1} ≥ s_i + 1`; together these give `s_{i+1} ≥ s_i + 2`, so the segment has at least one index.

All segments — first, interior, and last — are non-empty. ∎

*Formal Contract:*
- *Preconditions:* `t ∈ T` with `zeros(t) ≤ 3`. The field-segment constraint of full T4-validity is *not* assumed; it appears as the conclusion of the Forward direction and the local hypothesis of the Reverse direction.
- *Depends:*
  - T0 (CarrierSetDefinition) — fixes the carrier as ℕ and qualifies every `t ∈ T` as a nonempty finite sequence over ℕ, from which T4a locally unpacks `#t ≥ 1` (the length of a nonempty sequence is at least 1) as the precondition for `#t − 1 ∈ ℕ`.
  - NAT-zero (NatZeroMinimum) — supplies the lower bound `0 ≤ tᵢ`.
  - NAT-discrete (NatDiscreteness) — converts `0 ≤ tᵢ` and `tᵢ ≠ 0` into `tᵢ ≥ 1`.
  - NAT-order (NatStrictTotalOrder) — licenses the strictly increasing enumeration `s₁ < s₂ < … < s_k` and the position inequalities.
  - NAT-closure (NatArithmeticClosureAndIdentity) — supplies `1 ∈ ℕ` and closure of ℕ under addition, grounding the numerals `2 := 1 + 1 ∈ ℕ` and `3 := 2 + 1 ∈ ℕ` used in `s₁ ≥ 2` and `zeros(t) ≤ 3`, and the sums `s_i + 1`, `s_i + 2`, `#t + 1` used to name the sentinels and the interior-segment inequality.
  - NAT-sub (NatPartialSubtraction) — conditional-closure clause applied at the locally unpacked `#t ≥ 1` (from T0's nonempty qualification of `t ∈ T`) and `1 ∈ ℕ` (NAT-closure) gives `#t − 1 ∈ ℕ`, making `s_k ≤ #t − 1` a comparison of two ℕ-elements.
  - NAT-card (NatFiniteSetCardinality) — via the enumeration characterisation of `|·|`, identifies the length `k` of the strictly increasing enumeration `s₁ < s₂ < … < s_k` of the zero-index subset `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0} ⊆ {1, …, #t}` with `zeros(t)`, so that `k = zeros(t)`.
  - T4 (HierarchicalParsing) — supplies the positional conditions (i), (ii), (iii), the field-segment terminology, and the zero-count bound `zeros(t) ≤ 3`.
- *Postconditions:* The three positional conditions (i), (ii), (iii) hold if and only if every field segment of `t` is non-empty.

**T4b (UniqueParse).** Under the constraints of T4, the four partial functions

 `N, U, D, E : T ⇀ T`

that extract the node, user, document, and element sub-sequences of `t` are well-defined and uniquely determined by `t`. Writing `ℕ⁺ = {n ∈ ℕ : 0 < n}`, each projection's image lies in the subset of `T` whose every component is in `ℕ⁺`: a nonempty finite sequence over `ℕ` (by T0) with every component strictly positive by NAT-zero and NAT-discrete at `m = 0`. The four projections share the T4-valid subset of `T` as the outer domain from which absence is carved; field *absence* is encoded by partiality of the projection, and is fixed by `zeros(t)`: `N` is defined on the whole T4-valid subdomain (`N` is never absent); `U` is defined iff `t` is T4-valid and `zeros(t) ≥ 1`; `D` is defined iff `t` is T4-valid and `zeros(t) ≥ 2`; `E` is defined iff `t` is T4-valid and `zeros(t) = 3`. A field `X` is *absent in `t`* iff `t ∉ dom(X)`.

Presence pattern by case on `zeros(t) ∈ {0, 1, 2, 3}`:

  - `zeros(t) = 0`: only `N(t)` defined, with `N(t) = (t₁, ..., t_{#t})`.
  - `zeros(t) = 1`: `N(t)` and `U(t)` defined.
  - `zeros(t) = 2`: `N(t)`, `U(t)`, `D(t)` defined.
  - `zeros(t) = 3`: all four projections defined.

To access individual components of a field we introduce the notation `t.X₁`, grounded in T0's projection as follows. Whenever `X(t)` is defined, `X(t)` is a nonempty finite sequence over `ℕ⁺ ⊆ ℕ`, so `X(t) ∈ T` with `#(X(t)) ≥ 1` by T0, and T0's component projection `·ᵢ` is defined at every `i ∈ {1, …, #(X(t))}` — in particular at `i = 1`. We set `t.X₁ := (X(t))₁`, so the dot-accessor is notational shorthand for T0's subscript applied to `X(t)`, and `t.X₁` is defined iff `X(t)` is defined: `t.N₁` on every T4-valid `t`; `t.U₁` iff `zeros(t) ≥ 1`; `t.D₁` iff `zeros(t) ≥ 2`; `t.E₁` iff `zeros(t) = 3`.

By T4's stipulation, the separator positions of `t` are exactly `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}`, and every non-separator position carries a field component (strictly positive by NAT-zero and NAT-discrete at `m = 0` on T0's carrier ℕ).

Let the zero positions in increasing order be `s₁ < s₂ < ... < s_k` (NAT-order licenses the strictly increasing enumeration; NAT-card's enumeration characterisation of `|·|`, applied at the zero-index subset `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0} ⊆ {1, …, #t}` whose cardinality T4 identifies with `zeros(t)`, identifies the enumeration length with `zeros(t)`), `k = zeros(t) ∈ {0, 1, 2, 3}`. Since `t` is T4-valid, T4a's reverse direction applied to the field-segment constraint yields its exported conclusion: every field segment of `t` is non-empty. We unpack this locally into the position-inequalities used below. The arithmetic in what follows — the numeral `2`, additions `s_i + 1` and `s_i + 2`, and the partial subtraction `#t − 1` — is grounded thus: NAT-closure posits `1 ∈ ℕ` and closes ℕ under addition, so `2 := 1 + 1 ∈ ℕ` and, since every `s_i` and `#t` lies in ℕ, each `s_i + 1`, `s_i + 2`, and `s_k + 1 ∈ ℕ`. T0 declares every `t ∈ T` to be a nonempty finite sequence over ℕ; a nonempty sequence has at least one component, so by the definition of length `#t ≥ 1` — this is a local unpacking performed here, not a postcondition cited from T0. NAT-sub's conditional-closure clause applied at this `#t ≥ 1` and `1 ∈ ℕ` gives `#t − 1 ∈ ℕ`. T4 identifies the field segments as the `k + 1` maximal contiguous non-zero runs delimited by the zero positions and the sequence boundaries. The first segment occupies the indices `1, …, s₁ - 1` (when `k ≥ 1`), so its non-emptiness requires `s₁ ≥ 2`; the last segment occupies the indices `s_k + 1, …, #t` (when `k ≥ 1`), so its non-emptiness requires `s_k ≤ #t - 1`; each interior segment (`1 ≤ j < k`) occupies the indices `s_j + 1, …, s_{j+1} - 1`, so its non-emptiness requires `s_{j+1} ≥ s_j + 2`. Compute the projections case by case, citing the relevant inequality for each segment:

  - *Case k = 0.* `N(t) = (t₁, ..., t_{#t})`; `U, D, E` undefined at `t`. The sole segment spans indices 1 through `#t`, non-empty because `#t ≥ 1` — the local unpacking above, drawn from T0's nonempty qualification of `t ∈ T`.
  - *Case k = 1.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{#t})`; `D, E` undefined at `t`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₁ ≤ #t - 1` makes `U(t)` non-empty.
  - *Case k = 2.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{#t})`; `E` undefined at `t`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₂ ≥ s₁ + 2` makes `U(t)` non-empty; `s₂ ≤ #t - 1` makes `D(t)` non-empty.
  - *Case k = 3.* `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{s₃ - 1})`, `E(t) = (t_{s₃ + 1}, ..., t_{#t})`. `s₁ ≥ 2` makes `N(t)` non-empty; `s₂ ≥ s₁ + 2` makes `U(t)` non-empty; `s₃ ≥ s₂ + 2` makes `D(t)` non-empty; `s₃ ≤ #t - 1` makes `E(t)` non-empty.

By T0, each `tᵢ` is the value of the `i`-th component of `t` — a function of `t` and `i` — so the set `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}` of separator positions is determined by `t`, and with it the field boundaries and the sub-sequences. Therefore each projection is well-defined and unique on its stated domain. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints.
- *Definition:* The four partial functions `N, U, D, E : T ⇀ T` are characterised as follows. `dom(N)` is the T4-valid subset of `T`; `dom(U) = {t ∈ dom(N) : zeros(t) ≥ 1}`; `dom(D) = {t ∈ dom(N) : zeros(t) ≥ 2}`; `dom(E) = {t ∈ dom(N) : zeros(t) = 3}`. Let `s₁ < s₂ < ... < s_k` enumerate the zero positions, with `k = zeros(t) ∈ {0, 1, 2, 3}`. The values are fixed by case on `k`: for `k = 0`, `N(t) = (t₁, ..., t_{#t})`; for `k = 1`, `N(t) = (t₁, ..., t_{s₁ - 1})` and `U(t) = (t_{s₁ + 1}, ..., t_{#t})`; for `k = 2`, `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{#t})`; for `k = 3`, `N(t) = (t₁, ..., t_{s₁ - 1})`, `U(t) = (t_{s₁ + 1}, ..., t_{s₂ - 1})`, `D(t) = (t_{s₂ + 1}, ..., t_{s₃ - 1})`, `E(t) = (t_{s₃ + 1}, ..., t_{#t})`. Each projection's image lies in the subset of `T` whose every component is in `ℕ⁺`; this subset is cut out by T0's carrier `ℕ` together with NAT-zero and NAT-discrete. Each index `s_i + 1` lies in ℕ by NAT-closure's successor closure, and each `s_i − 1` lies in ℕ by NAT-sub's conditional-closure clause applied at `s_i ≥ 1` (T0's index domain) and `1 ∈ ℕ` (NAT-closure). The segment-length inequalities `s₁ ≥ 2`, `s_k ≤ #t - 1` (when `k ≥ 1`), and `s_{j+1} ≥ s_j + 2` — obtained above by unpacking T4a's conclusion that every field segment is non-empty, with numeral `2 := 1 + 1 ∈ ℕ` by NAT-closure and `#t − 1 ∈ ℕ` by NAT-sub under the locally unpacked `#t ≥ 1` (from T0's nonempty qualification of `t ∈ T`) — keep every index `s_i ± 1` appearing above within `{1, …, #t}`. Outside the stated domains no projection is assigned a value.
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies ℕ as the carrier and fixes `T` as the set of nonempty finite sequences over ℕ, from which T4b locally unpacks `#t ≥ 1` (the length of a nonempty sequence is at least 1) for the case `k = 0` and for NAT-sub's precondition at `#t − 1`; the index domain `{1, …, #t}` gives `s_i ≥ 1`, discharging the precondition of NAT-sub at `s_i − 1`; each projection's all-`ℕ⁺`-component value embeds into `T`, so T0's subscript grounds `t.X₁ := (X(t))₁`.
  - NAT-zero (NatZeroMinimum) — lower bound `0 ≤ tᵢ`.
  - NAT-discrete (NatDiscreteness) — at `m = 0`, promotes "non-zero" to "strictly positive", placing the image of each projection in the all-`ℕ⁺`-component subset of `T`.
  - NAT-order (NatStrictTotalOrder) — licenses the strictly increasing enumeration of separator positions and the inequality manipulations on them.
  - NAT-closure (NatArithmeticClosureAndIdentity) — supplies `1 ∈ ℕ` together with closure of ℕ under addition, grounding the numeral `2 := 1 + 1 ∈ ℕ` (used in `s₁ ≥ 2` and `s_{j+1} ≥ s_j + 2`) and the sums `s_i + 1`, `s_i + 2` appearing in the case construction.
  - NAT-sub (NatPartialSubtraction) — conditional-closure clause applied at the locally unpacked `#t ≥ 1` (from T0's nonempty qualification of `t ∈ T`) gives `#t − 1 ∈ ℕ` for the inequality `s_k ≤ #t − 1`, and at `s_i ≥ 1` gives `s_i − 1 ∈ ℕ` for the sub-sequence upper indices `t_{s_i − 1}`.
  - NAT-card (NatFiniteSetCardinality) — via the enumeration characterisation of `|·|`, identifies the length `k` of the strictly increasing enumeration `s₁ < s₂ < … < s_k` of the zero-index subset `{i : 1 ≤ i ≤ #t ∧ tᵢ = 0} ⊆ {1, …, #t}` with `zeros(t)`, so that `k = zeros(t)` in the case analysis on `k`.
  - T4 (HierarchicalParsing) — zero-count bound, field-segment clauses, role-assignment of zeros as separators.
  - T4a (SyntacticEquivalence) — supplies, via its reverse direction applied to T4's field-segment constraint, the conclusion that every field segment of `t` is non-empty; the position-inequalities `s₁ ≥ 2`, `s_k ≤ #t - 1`, and `s_{j+1} ≥ s_j + 2` that discharge non-emptiness of each extracted sub-sequence are a local re-expression of this conclusion.
- *Postconditions:* `N, U, D, E : T ⇀ T` are partial functions. `dom(N)` is the T4-valid subset of `T`; `dom(U) ⊆ dom(N)` picks out `zeros(t) ≥ 1`; `dom(D) ⊆ dom(N)` picks out `zeros(t) ≥ 2`; `dom(E) ⊆ dom(N)` picks out `zeros(t) = 3`. On its domain each projection is well-defined, uniquely determined by `t`, and returns a nonempty finite sequence over `ℕ⁺` — an element of `T` whose every component is strictly positive. Field *absence* is encoded by partiality: `X` is *absent in `t`* iff `t ∉ dom(X)`. Presence pattern: `zeros(t) = 0` → only `N` defined; `zeros(t) = 1` → `N, U` defined; `zeros(t) = 2` → `N, U, D` defined; `zeros(t) = 3` → all four defined. The component-access notation `t.X₁ := (X(t))₁` — T0's component projection at index 1 applied to `X(t)`, which belongs to `T` whenever `X` is defined at `t` because `ℕ⁺ ⊆ ℕ` — is defined iff `X` is defined at `t`: `t.N₁` always on the T4-valid subset; `t.U₁` iff `zeros(t) ≥ 1`; `t.D₁` iff `zeros(t) ≥ 2`; `t.E₁` iff `zeros(t) = 3`. Outside the T4-valid subdomain, none of the projections is assigned a value; consumers must carry T4-validity as a precondition.
