**NAT-addcompat (NatAdditionOrderAndSuccessor).** Addition on ℕ is compatible with the order on either side, and `n < n + 1` for every `n ∈ ℕ`.

Three related facts are stated together as one axiom because they all concern the interaction between addition and `<` on ℕ:

- Left order compatibility: `n ≥ p ⟹ m + n ≥ m + p` for every `m, n, p ∈ ℕ`. Adding the same natural number on the left preserves the order.
- Right order compatibility: `n ≥ p ⟹ n + m ≥ p + m` for every `m, n, p ∈ ℕ`. Adding the same natural number on the right preserves the order.
- Strict successor inequality: `n < n + 1` for every `n ∈ ℕ`. The successor is strictly greater.

Both order-compatibility forms are stated explicitly because the proofs of this ASN supply the fixed summand on either side: TA5's Case `k > 0`, T10a.3's single-step length claim, T10a.5's ancestor-descendant nesting step, and GlobalUniqueness Case 5's sub-case `k'₁ < k'₂` add the fixed term on the left of the variable summands to lift non-strict inequalities (for example, TA5 and T10a.3 lift `1 ≤ k` — respectively `1 ≤ k'` — to the length-level `m + 1 ≤ m + k`, and GlobalUniqueness Case 5 lifts the weakened `k'₁ ≤ k'₂` to `#p₁ + k'₁ ≤ #p₁ + k'₂`), while T10a.3's running-sum induction step and GlobalUniqueness Case 5's sub-case `k'₁ < k'₂` add the fixed increment on the right of the variable summands, also to lift non-strict inequalities (for example, GlobalUniqueness Case 5 lifts the weakened `#p₁ ≤ #p₂` to `#p₁ + k'₂ ≤ #p₂ + k'₂`). Both order-compatibility clauses deliver *only non-strict* conclusions: the hypothesis `n ≥ p` admits the equality `n = p` and the conclusion `m + n ≥ m + p` (resp. `n + m ≥ p + m`) inherits that possibility symmetrically, so neither clause by itself lifts the strict `#p₁ < #p₂` into the strict `#p₁ + k'₂ < #p₂ + k'₂`. The strict promotion through addition that GlobalUniqueness Case 5's sub-case `k'₁ < k'₂` requires is therefore a four-step chain consuming NAT-order, NAT-addcompat, and NAT-cancel together: NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` weakens `#p₁ < #p₂` to the non-strict `#p₁ ≤ #p₂`; NAT-addcompat's right compatibility lifts that non-strict `≤` to `#p₁ + k'₂ ≤ #p₂ + k'₂`; NAT-cancel's right cancellation (`n + m = p + m ⟹ n = p`, instantiated at `m = k'₂`, `n = #p₁`, `p = #p₂`) rules out the equality `#p₁ + k'₂ = #p₂ + k'₂`, because that equality would yield `#p₁ = #p₂` and contradict `#p₁ < #p₂` via NAT-order's irreflexivity; and NAT-order's defining clause, applied to the non-strict `#p₁ + k'₂ ≤ #p₂ + k'₂` together with the just-discharged non-equality, returns the strict `#p₁ + k'₂ < #p₂ + k'₂`. GlobalUniqueness Case 5 therefore cites NAT-addcompat, NAT-cancel, and NAT-order together at its strict-promotion site — no proof in this ASN attributes a strict-inequality-through-addition step to NAT-addcompat alone. The ASN's convention (T0) is that each proof cites only the ℕ facts it actually uses; without the right-addition clause stated as an axiom, GlobalUniqueness Case 5's right-added non-strict lift would tacitly assume commutativity of addition on ℕ, breaking the citation policy that the other NAT-* axioms enforce. T0(a) and T1 use only the strict successor inequality clause directly — at `M + 1 > M` and `m < m + 1` respectively — without invoking either order-compatibility form. TA5 (Case `k = 0` at the divergence step `t'_j = t_j + 1 > t_j`), TA5a (Case `k = 0` at the post-increment step `t_{sig(t)} < t_{sig(t)} + 1`), T10a.3 (strict-increase step at each depth boundary), T10a.5 (Case 2 inductive step at `(tₙ)_{#s+1} < (tₙ)_{#s+1} + 1`, and the ancestor-descendant nesting step at `#tᵢ < #tᵢ + 1`), and T10a.8 (inductive step at `(tₙ)_{sig(tₙ)} < (tₙ)_{sig(tₙ)} + 1`) use the strict successor inequality to bound lengths or indices under incrementing operations, and the in-cone order-compatibility consumers enumerated above use the order-compatibility facts to lift non-strict inequalities through arithmetic.

*Formal Contract:*
- *Axiom:* `(A m, n, p ∈ ℕ : n ≥ p : m + n ≥ m + p)` (left order compatibility); `(A m, n, p ∈ ℕ : n ≥ p : n + m ≥ p + m)` (right order compatibility); `(A n ∈ ℕ :: n < n + 1)` (strict successor inequality).
