**T4c (LevelDetermination).** Let `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|` as in T4, where `|·|` is the cardinality operator on subsets of initial segments of ℕ axiomatized by NAT-card (codomain ℕ, distinct from T0's tumbler-length `#·` which acts on sequences); the indexed set is a subset of `{1, …, #t} ⊆ ℕ` by T0, so NAT-card applies at `n = #t` and `zeros(t) ∈ ℕ`. On the T4-valid subset of `T` (tumblers satisfying `zeros(t) ≤ 3`, no two zeros adjacent, `t₁ ≠ 0`, `t_{#t} ≠ 0`), T4c defines four hierarchical level labels by zero count: `t` is a *node address* iff `zeros(t) = 0`, a *user address* iff `zeros(t) = 1`, a *document address* iff `zeros(t) = 2`, and an *element address* iff `zeros(t) = 3`.

The four biconditionals are the definition of the labels. The proof obligation reduces to: the four zero-count values exhaust the T4-valid subdomain, and distinct values receive distinct labels. Realisation — existence of a T4-valid tumbler at each zero count — is not claimed here; NAT-closure's `1 ∈ ℕ` with NAT-addcompat's `0 < 1` (the strict successor inequality instantiated at `n = 0`) already places `1` in `ℕ⁺`, so strictly positive components are available, but the foundations in scope supply no constructor producing tumblers of the lengths these four cases require (from `1` up to `7`), so T4c stands as a pure definition on whatever T4-valid tumblers exist.

*Exhaustion.* By T4, every T4-valid tumbler satisfies `zeros(t) ≤ 3`, and by NAT-zero, `0 ≤ zeros(t)`. These bounds alone do not fix `zeros(t) ∈ {0, 1, 2, 3}` — without discreteness the segment could harbor intermediate values. Proceed by iterated case analysis using NAT-order's trichotomy, which asserts that for any `m, zeros(t) ∈ ℕ` exactly one of `zeros(t) < m`, `zeros(t) = m`, `m < zeros(t)` holds. At `m = 0`: the case `zeros(t) < 0` is excluded by NAT-zero's `0 ≤ zeros(t)` — unfolding ≤ to `0 < zeros(t) ∨ 0 = zeros(t)`, either disjunct together with `zeros(t) < 0` yields `0 < 0` via NAT-order transitivity, contradicting NAT-order irreflexivity — so either `zeros(t) = 0`, whereupon `zeros(t) ∈ {0, 1, 2, 3}`, or `0 < zeros(t)`, which NAT-discrete promotes to `1 ≤ zeros(t)`. At `m = 1` (continuing the `0 < zeros(t)` branch): trichotomy's case `zeros(t) < 1` is excluded by `1 ≤ zeros(t)` via the same transitivity-and-irreflexivity route, so either `zeros(t) = 1`, whereupon `zeros(t) ∈ {0, 1, 2, 3}`, or `1 < zeros(t)`, which NAT-discrete promotes to `2 ≤ zeros(t)`. At `m = 2` (continuing the `1 < zeros(t)` branch): trichotomy's case `zeros(t) < 2` is excluded by `2 ≤ zeros(t)` similarly, so either `zeros(t) = 2`, whereupon `zeros(t) ∈ {0, 1, 2, 3}`, or `2 < zeros(t)`, which NAT-discrete promotes to `3 ≤ zeros(t)`. Combining `3 ≤ zeros(t)` with T4's `zeros(t) ≤ 3`, trichotomy at `(zeros(t), 3)` excludes both `zeros(t) < 3` (which contradicts `3 ≤ zeros(t)` by the same route) and `3 < zeros(t)` (which contradicts `zeros(t) ≤ 3` symmetrically), forcing `zeros(t) = 3`. Every branch terminates with `zeros(t) ∈ {0, 1, 2, 3}`.

*Injectivity.* The values `0, 1, 2, 3` are pairwise distinct in ℕ. NAT-addcompat's strict successor inequality `n < n + 1`, instantiated at `n ∈ {0, 1, 2}`, gives `0 < 1`, `1 < 2`, and `2 < 3`; NAT-order transitivity chains these to `0 < 1 < 2 < 3`, so `m < n` holds for every pair with `m` preceding `n` in the chain; NAT-order irreflexivity then excludes `m = n` for each such pair. Since `zeros(t)` is single-valued, distinct zero counts induce distinct labels. ∎

*Formal Contract:*
- *Preconditions:* `t` satisfies the T4 constraints (`zeros(t) ≤ 3`, no two zeros adjacent, `t₁ ≠ 0`, `t_{#t} ≠ 0`).
- *Depends:*
  - T0 (CarrierSetDefinition) — supplies the index domain `{1, …, #t} ⊆ ℕ` that contains the zero-index subset, so NAT-card applies at `n = #t`.
  - NAT-card (NatFiniteSetCardinality) — axiomatizes `|·|` on subsets of every initial segment `{1, …, n} ⊆ ℕ` with codomain ℕ, grounding the type `zeros(t) ∈ ℕ` for `zeros(t) = |{i : 1 ≤ i ≤ #t ∧ tᵢ = 0}|`.
  - NAT-zero (NatZeroMinimum) — supplies `0 ≤ zeros(t)` for the exhaustion step.
  - NAT-discrete (NatDiscreteness) — supplies the absence of values strictly between consecutive integers, promoting each strict inequality `m < zeros(t)` (for `m ∈ {0, 1, 2}`) to `m + 1 ≤ zeros(t)` so that the segment `0 ≤ zeros(t) ≤ 3` collapses to `{0, 1, 2, 3}`.
  - NAT-order (NatStrictTotalOrder) — supplies trichotomy to case-split `zeros(t)` against each `m ∈ {0, 1, 2, 3}` in exhaustion (exactly one of `zeros(t) < m`, `zeros(t) = m`, `m < zeros(t)` holds), together with transitivity and irreflexivity to exclude the `zeros(t) < m` branch whenever `m ≤ zeros(t)` has already been derived (combining the two would yield `m < m`, against irreflexivity); and supplies the same transitivity to chain `0 < 1 < 2 < 3` and irreflexivity to exclude equality within that chain, so that `0, 1, 2, 3` are pairwise distinct for injectivity.
  - NAT-closure (NatArithmeticClosureAndIdentity) — posits `1 ∈ ℕ` and closes ℕ under addition, grounding the numerals `2 := 1 + 1 ∈ ℕ` and `3 := 2 + 1 ∈ ℕ` used in exhaustion's bound `zeros(t) ≤ 3` and injectivity's chain `0 < 1 < 2 < 3`, and the successors `m + 1` for `m ∈ {0, 1, 2}` used in the exhaustion step.
  - NAT-addcompat (NatAdditionOrderAndSuccessor) — supplies the strict successor inequality `n < n + 1`, instantiated at `n ∈ {0, 1, 2}` to obtain the base links `0 < 1`, `1 < 2`, `2 < 3` in injectivity's chain.
  - T4 (HierarchicalParsing) — supplies the T4-valid subdomain constraints, used in exhaustion.
- *Postconditions:* `(A t ∈ T : t is T4-valid :: (zeros(t) = 0 ↔ t is a node address) ∧ (zeros(t) = 1 ↔ t is a user address) ∧ (zeros(t) = 2 ↔ t is a document address) ∧ (zeros(t) = 3 ↔ t is an element address))`. The four labels are pairwise distinct. Outside this subdomain, T4c assigns no level; consumers must discharge T4-validity before reading a level off `zeros(t)`. No existence of a T4-valid tumbler at any given zero count is asserted — T4c is a pure definition of labels on whatever T4-valid tumblers exist.
