*Exhaustion.* By T4, every T4-valid tumbler satisfies `zeros(t) вүӨ 3`, and by NAT-zero, `0 < zeros(t) вҲЁ 0 = zeros(t)` вҖ” equivalently `0 вүӨ zeros(t)` via NAT-order's definition of `вүӨ`. These bounds alone do not fix `zeros(t) вҲҲ {0, 1, 2, 3}` вҖ” without discreteness the segment could harbor intermediate values. Proceed by iterated case analysis using NAT-order's trichotomy, which asserts that for any `m, zeros(t) вҲҲ в„•` exactly one of `zeros(t) < m`, `zeros(t) = m`, `m < zeros(t)` holds. The mechanism for excluding a trichotomy branch `zeros(t) < m` given a lower bound `m вүӨ zeros(t)` is uniform: unfolding вүӨ to `m < zeros(t) вҲЁ m = zeros(t)` shows that one of trichotomy's other two alternatives holds, and trichotomy's exactly-one clause then forbids the third. This uses trichotomy alone вҖ” neither transitivity, irreflexivity, nor substitution of equals under `<` is invoked, which matters because substitution of equals under `<` is not among NAT-order's stated properties. At `m = 0`: the case `zeros(t) < 0` is excluded by NAT-zero's `0 < zeros(t) вҲЁ 0 = zeros(t)` (equivalently `0 вүӨ zeros(t)` via NAT-order's definition of `вүӨ`) via the exactly-one route just described, so either `zeros(t) = 0`, whereupon `zeros(t) вҲҲ {0, 1, 2, 3}`, or `0 < zeros(t)`, which NAT-discrete promotes to `1 вүӨ zeros(t)`. At `m = 1` (continuing the `0 < zeros(t)` branch): trichotomy's case `zeros(t) < 1` is excluded by `1 вүӨ zeros(t)` via the same exactly-one route, so either `zeros(t) = 1`, whereupon `zeros(t) вҲҲ {0, 1, 2, 3}`, or `1 < zeros(t)`, which NAT-discrete promotes to `2 вүӨ zeros(t)`. At `m = 2` (continuing the `1 < zeros(t)` branch): trichotomy's case `zeros(t) < 2` is excluded by `2 вүӨ zeros(t)` via the same route, so either `zeros(t) = 2`, whereupon `zeros(t) вҲҲ {0, 1, 2, 3}`, or `2 < zeros(t)`, which NAT-discrete promotes to `3 вүӨ zeros(t)`. At the final step, trichotomy at `(zeros(t), 3)` is confronted with two bounds: `3 вүӨ zeros(t)` excludes `zeros(t) < 3` (unfolding `3 вүӨ zeros(t)` to `3 < zeros(t) вҲЁ 3 = zeros(t)` makes one of the other two trichotomy alternatives hold, forcing `zeros(t) < 3` out by exactly-one), and symmetrically `zeros(t) вүӨ 3` excludes `3 < zeros(t)` (unfolding to `zeros(t) < 3 вҲЁ zeros(t) = 3` makes one of the other two alternatives hold, forcing `3 < zeros(t)` out by exactly-one), leaving `zeros(t) = 3`. Every branch terminates with `zeros(t) вҲҲ {0, 1, 2, 3}`.

*Injectivity.* The values `0, 1, 2, 3` are pairwise distinct in в„•. NAT-addcompat's strict successor inequality `n < n + 1`, instantiated at `n вҲҲ {0, 1, 2}`, gives `0 < 1`, `1 < 2`, and `2 < 3`; NAT-order transitivity chains these to `0 < 1 < 2 < 3`, so `m < n` holds for every pair with `m` preceding `n` in the chain; NAT-order's disjointness of `<` and `=` вҖ” `(A m, n вҲҲ в„• : m < n : m вү  n)` вҖ” then yields `m вү  n` for each such pair directly from `m < n`, without appeal to substitution of equals under `<` (which Exhaustion already flagged as unavailable). Since `zeros(t)` is single-valued, distinct zero counts induce distinct labels. вҲҺ
