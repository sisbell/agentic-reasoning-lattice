**NAT-discrete (NatDiscreteness).** For every `m, n ∈ ℕ`, `m < n ⟹ m + 1 ≤ n`.

No natural number lies strictly between `n` and its successor `n + 1`: whenever `m < n`, the successor `m + 1` is already bounded above by `n`, so `m + 1 ≤ n`. This is the discreteness of ℕ. The axiom body invokes two symbols beyond ℕ's primitive `<` and `=`: the non-strict companion `≤`, *defined* in NAT-order by `m ≤ n ⟺ m < n ∨ m = n`, and the successor term `m + 1`, whose summand `1 ∈ ℕ` and closure `m + 1 ∈ ℕ` are posited by NAT-closure. Both are declared in the Depends slot so that the axiom body can be read without silently importing foundations.

The axiom has a familiar no-interval reformulation `(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)` — "nothing strictly between `m` and `m + 1`" — which we record as a *Consequence:* rather than a second axiom form, since its equivalence to the axiom body is not free from the body alone: each direction leans on NAT-order clauses beyond the `≤`-definition. Forward (axiom ⟹ no-interval): assume `m ≤ n < m + 1`; unfolding `m ≤ n` by the NAT-order definition `m ≤ n ⟺ m < n ∨ m = n` splits into two cases. In the case `m < n`, the axiom gives `m + 1 ≤ n`, which unfolds to `m + 1 < n ∨ m + 1 = n`; paired with the hypothesis `n < m + 1`, either disjunct is contradictory — `m + 1 < n` together with `n < m + 1` is excluded by NAT-order's exactly-one trichotomy (specifically its `¬(a < b ∧ b < a)` clause, instantiated at `(m + 1, n)`), while `m + 1 = n` rewrites `n < m + 1` to `n < n`, contradicting irreflexivity. The case `m < n` is therefore impossible, leaving `n = m`. Reverse (no-interval ⟹ axiom): assume `m < n` and suppose, for contradiction, that `¬(m + 1 ≤ n)`; by exactly-one trichotomy at `(m + 1, n)`, exactly one of `m + 1 < n`, `m + 1 = n`, `n < m + 1` holds, and the first two together constitute `m + 1 ≤ n` — which we have ruled out — so `n < m + 1`. But then `(m, n)` satisfies `m ≤ n < m + 1` (`m < n` gives `m ≤ n` by the NAT-order definition), and the no-interval form forces `n = m`, contradicting NAT-order's disjointness clause `m < n ⟹ m ≠ n`. Hence `m + 1 ≤ n`. ∎

*Formal Contract:*
- *Axiom:* `(A m, n ∈ ℕ :: m < n ⟹ m + 1 ≤ n)` (discreteness).
- *Consequence:* `(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)` (no-interval form). Forward: the `m < n` branch of `m ≤ n` yields `m + 1 ≤ n` via the axiom, whose disjuncts `m + 1 < n` and `m + 1 = n` both collide with `n < m + 1` — the first by NAT-order's exactly-one-trichotomy clause `¬(a < b ∧ b < a)` at `(m + 1, n)`, the second by irreflexivity `¬(n < n)` after rewriting. Reverse: if `m < n` yet `¬(m + 1 ≤ n)`, exactly-one trichotomy at `(m + 1, n)` leaves `n < m + 1`; the no-interval form at `(m, n)` then forces `n = m`, contradicting NAT-order's disjointness clause `m < n ⟹ m ≠ n`.
- *Depends:*
  - NAT-order (NatStrictTotalOrder) — supplies the non-strict companion `≤` (defined by `m ≤ n ⟺ m < n ∨ m = n`) used in the axiom's consequent `m + 1 ≤ n`, and three further clauses cited in the Consequence's bidirectional derivation: the exactly-one-trichotomy clause `¬(a < b ∧ b < a)` instantiated at `(m + 1, n)`, irreflexivity `¬(n < n)`, and the disjointness clause `m < n ⟹ m ≠ n`.
  - NAT-closure (NatArithmeticClosureAndIdentity, this ASN) — posits `1 ∈ ℕ` and closes ℕ under addition, so the successor `m + 1 ∈ ℕ` and the inequalities `m + 1 ≤ n` and `n < m + 1` are comparisons of two ℕ-elements.
