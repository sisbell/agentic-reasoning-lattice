**NAT-discrete (NatDiscreteness).** For every `m, n ‚àà ‚Ñï`, `m < n ‚üπ m + 1 ‚â§ n`.

No natural number lies strictly between `n` and its successor `n + 1`: whenever `m < n`, the successor `m + 1` is already bounded above by `n`, so `m + 1 ‚â§ n`. This is the discreteness of ‚Ñï. The axiom body invokes two symbols beyond ‚Ñï's primitive `<` and `=`: the non-strict companion `‚â§`, *defined* in NAT-order by `m ‚â§ n ‚ü∫ m < n ‚à® m = n`, and the successor term `m + 1`, whose summand `1 ‚àà ‚Ñï` and closure `m + 1 ‚àà ‚Ñï` are posited by NAT-closure. Both are declared in the Depends slot so that the axiom body can be read without silently importing foundations.

The axiom has a familiar no-interval reformulation `(A m, n ‚àà ‚Ñï :: m ‚â§ n < m + 1 ‚üπ n = m)` ‚Äî "nothing strictly between `m` and `m + 1`" ‚Äî which we record as a *Consequence:* rather than a second axiom form, because it is derivable from the axiom body together with NAT-order clauses, and we do not posit what we can derive. The derivation (axiom ‚üπ no-interval): assume `m ‚â§ n < m + 1`; unfolding `m ‚â§ n` by the NAT-order definition `m ‚â§ n ‚ü∫ m < n ‚à® m = n` splits into two cases, each of which we walk to `n = m`. In the case `m < n`, the axiom gives `m + 1 ‚â§ n`, which unfolds to `m + 1 < n ‚à® m + 1 = n`; paired with the hypothesis `n < m + 1`, either disjunct is contradictory ‚Äî `m + 1 < n` together with `n < m + 1` is excluded by NAT-order's exactly-one trichotomy (specifically its `¬¨(a < b ‚àß b < a)` clause, instantiated at `(m + 1, n)`), while `m + 1 = n` rewrites `n < m + 1` to `n < n`, contradicting irreflexivity. The case `m < n` is therefore impossible; it contributes nothing to the conclusion and is discharged by contradiction. In the case `m = n`, symmetry of `=` gives `n = m` directly. Both branches of the split terminate in `n = m`, closing the case analysis. ‚àé

*Equivalence note (separate from the Consequence derivation).* The converse direction ‚Äî from the no-interval form one can recover the axiom body `m < n ‚üπ m + 1 ‚â§ n` ‚Äî also holds, so the two formulations are interderivable over NAT-order. We record the observation only to justify the presentational choice of which form to posit; the Consequence itself is the forward conditional, established above, and is not strengthened by this note. The converse walk: assume `m < n` and suppose, for contradiction, that `¬¨(m + 1 ‚â§ n)`; by exactly-one trichotomy at `(m + 1, n)`, exactly one of `m + 1 < n`, `m + 1 = n`, `n < m + 1` holds, and the first two together constitute `m + 1 ‚â§ n` ‚Äî which we have ruled out ‚Äî so `n < m + 1`. But then `(m, n)` satisfies `m ‚â§ n < m + 1` (`m < n` gives `m ‚â§ n` by the NAT-order definition), and the no-interval form forces `n = m`, contradicting NAT-order's disjointness clause `m < n ‚üπ m ‚âÝ n`. Hence `m + 1 ‚â§ n`. We take the discreteness form as the axiom because it directly states the intended content ‚Äî `<` jumps by one step on ‚Ñï.

*Formal Contract:*
- *Axiom:* `(A m, n ‚àà ‚Ñï :: m < n ‚üπ m + 1 ‚â§ n)` (discreteness).
- *Consequence:* `(A m, n ‚àà ‚Ñï :: m ‚â§ n < m + 1 ‚üπ n = m)` (no-interval form) ‚Äî derived from the axiom together with NAT-order (the `‚â§`-definition used to split `m ‚â§ n`, the exactly-one-trichotomy clause `¬¨(a < b ‚àß b < a)` instantiated at `(m + 1, n)`, and irreflexivity `¬¨(n < n)` at `n := m + 1` after rewriting) via the forward walk in the preceding prose; recorded as a Consequence rather than a second axiom because it is derivable from the axiom and we do not posit what we can derive.
- *Depends:*
  - NAT-order (NatStrictTotalOrder) ‚Äî supplies the non-strict companion `‚â§` (defined by `m ‚â§ n ‚ü∫ m < n ‚à® m = n`) used in the axiom's consequent `m + 1 ‚â§ n` and in the Consequence derivation's case split on `m ‚â§ n`, the exactly-one-trichotomy clause `¬¨(a < b ‚àß b < a)` instantiated at `(m + 1, n)` in the derivation, and irreflexivity `¬¨(n < n)` used to discharge the rewritten `n < n`. The separate equivalence note additionally cites NAT-order's disjointness clause `m < n ‚üπ m ‚âÝ n` to derive the converse direction; that citation serves the equivalence observation only, not the Consequence.
  - NAT-closure (NatArithmeticClosureAndIdentity, this ASN) ‚Äî posits `1 ‚àà ‚Ñï` and closes ‚Ñï under addition, so the successor `m + 1 ‚àà ‚Ñï` and the inequalities `m + 1 ‚â§ n` and `n < m + 1` are comparisons of two ‚Ñï-elements.
