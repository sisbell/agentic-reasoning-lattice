**NAT-discrete (NatDiscreteness).** For every `m, n ∈ ℕ`, `m < n ⟹ m + 1 ≤ n`.

No natural number lies strictly between `n` and its successor `n + 1`: whenever `m < n`, the successor `m + 1` is already bounded above by `n`, so `m + 1 ≤ n`. This is the discreteness of ℕ. The axiom body invokes two symbols beyond ℕ's primitive `<` and `=`: the non-strict companion `≤`, *defined* in NAT-order by `m ≤ n ⟺ m < n ∨ m = n`, and the successor term `m + 1`, whose summand `1 ∈ ℕ` and closure `m + 1 ∈ ℕ` are posited by NAT-closure. Both are declared in the Depends slot so that the axiom body can be read without silently importing foundations.

The axiom has a familiar no-interval reformulation `(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)` — "nothing strictly between `m` and `m + 1`" — which we record as a *Consequence:* rather than a second axiom form, because it is derivable from the axiom body together with NAT-order clauses, and we do not posit what we can derive. The derivation (axiom ⟹ no-interval): assume `m ≤ n < m + 1`; unfolding `m ≤ n` by the NAT-order definition `m ≤ n ⟺ m < n ∨ m = n` splits into two cases, each of which we walk to `n = m`. In the case `m < n`, the axiom gives `m + 1 ≤ n`, which unfolds to `m + 1 < n ∨ m + 1 = n`; paired with the hypothesis `n < m + 1`, either disjunct is contradictory — `m + 1 < n` together with `n < m + 1` is excluded by NAT-order's exactly-one trichotomy (specifically its `¬(a < b ∧ b < a)` clause, instantiated at `(m + 1, n)`), while `m + 1 = n` rewrites `n < m + 1` to `n < n`, contradicting irreflexivity. The case `m < n` is therefore impossible; it contributes nothing to the conclusion and is discharged by contradiction. In the case `m = n`, symmetry of `=` gives `n = m` directly. Both branches of the split terminate in `n = m`, closing the case analysis. ∎

*Equivalence note (separate from the Consequence derivation).* The converse direction — from the no-interval form one can recover the axiom body `m < n ⟹ m + 1 ≤ n` — also holds, so the two formulations are interderivable over NAT-order. We record the observation only to justify the presentational choice of which form to posit; the Consequence itself is the forward conditional, established above, and is not strengthened by this note. The converse walk: assume `m < n` and suppose, for contradiction, that `¬(m + 1 ≤ n)`; by exactly-one trichotomy at `(m + 1, n)`, exactly one of `m + 1 < n`, `m + 1 = n`, `n < m + 1` holds, and the first two together constitute `m + 1 ≤ n` — which we have ruled out — so `n < m + 1`. But then `(m, n)` satisfies `m ≤ n < m + 1` (`m < n` gives `m ≤ n` by the NAT-order definition), and the no-interval form forces `n = m`, contradicting NAT-order's exactly-one-trichotomy Consequence's `¬(m < n ∧ m = n)` conjunct (equivalently its contrapositive `m < n ⟹ m ≠ n`): the assumed `m < n` paired with the just-derived `n = m` (symmetrised to `m = n`) realises the conjunction that conjunct rules out. Hence `m + 1 ≤ n`. We take the discreteness form as the axiom because it directly states the intended content — `<` jumps by one step on ℕ.

*Formal Contract:*
- *Axiom:* `(A m, n ∈ ℕ :: m < n ⟹ m + 1 ≤ n)` (discreteness).
- *Consequence:* `(A m, n ∈ ℕ :: m ≤ n < m + 1 ⟹ n = m)` (no-interval form) — derived from the axiom together with NAT-order (the `≤`-definition used to split `m ≤ n`, the exactly-one-trichotomy clause `¬(a < b ∧ b < a)` instantiated at `(m + 1, n)`, and irreflexivity `¬(n < n)` at `n := m + 1` after rewriting) via the forward walk in the preceding prose; recorded as a Consequence rather than a second axiom because it is derivable from the axiom and we do not posit what we can derive.
- *Depends:*
  - NAT-order (NatStrictTotalOrder) — supplies the non-strict companion `≤` (defined by `m ≤ n ⟺ m < n ∨ m = n`) used in the axiom's consequent `m + 1 ≤ n` and in the Consequence derivation's case split on `m ≤ n`, the exactly-one-trichotomy clause `¬(a < b ∧ b < a)` instantiated at `(m + 1, n)` in the derivation, and irreflexivity `¬(n < n)` used to discharge the rewritten `n < n`. The separate equivalence note additionally cites NAT-order's exactly-one-trichotomy Consequence's `¬(m < n ∧ m = n)` conjunct (equivalently its contrapositive `m < n ⟹ m ≠ n`) to derive the converse direction; that citation serves the equivalence observation only, not the Consequence.
  - NAT-closure (NatArithmeticClosureAndIdentity, this ASN) — posits `1 ∈ ℕ` and closes ℕ under addition, so the successor `m + 1 ∈ ℕ` and the inequalities `m + 1 ≤ n` and `n < m + 1` are comparisons of two ℕ-elements.
