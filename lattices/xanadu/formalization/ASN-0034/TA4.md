**TA4 (PartialInverse).** `(A a, w : Pos(w) ∧ k = #a ∧ #w = k ∧ (A i : 1 ≤ i < k : aᵢ = 0) : (a ⊕ w) ⊖ w = a)`, where `k` is the action point of `w`.

The precondition has three parts. First, `k = #a` — the action point falls at the last component of `a`. This is necessary because addition replaces `a`'s trailing structure below the action point with `w`'s trailing structure (tail replacement, defined below). When `k < #a`, components `aₖ₊₁, ..., a_{#a}` are discarded by addition and cannot be recovered by subtraction. Concretely: `[1, 5] ⊕ [1, 3] = [2, 3]` (action point 1, position 2 replaced by `w`'s trailing `3`), then `[2, 3] ⊖ [1, 3] = [1, 3] ≠ [1, 5]`.

Second, `#w = k` — the displacement has no trailing components beyond the action point. When `#w > k`, the result acquires trailing components from `w` that were not present in `a`. The trailing `7` from `w` persists through subtraction: `[0, 5] ⊕ [0, 3, 7] = [0, 8, 7]`, then `[0, 8, 7] ⊖ [0, 3, 7]` yields `[0, 5, 7] ≠ [0, 5]`.

Third, `(A i : 1 ≤ i < k : aᵢ = 0)` — all components of `a` before the action point are zero. This ensures the subtraction's divergence-discovery mechanism finds the action point at the right position. If `a` has a nonzero component at some position `j < k`, then the result of addition has `rⱼ = aⱼ ≠ 0`, and the subtraction's divergence falls at `j`, not at `k`. Concretely: `[5, 3] ⊕ [0, 7] = [5, 10]`, then `[5, 10] ⊖ [0, 7]`: divergence at position 1, producing `[5, 10] ≠ [5, 3]`.

When all three conditions hold, recovery is exact. The restriction is not a deficiency but a precise statement of when the operations are inverses.

The precondition `Pos(w)` (TA-Pos) guarantees that the action point `k` exists; by TA-Pos, this excludes every all-zero displacement regardless of length.

*Proof.* We show that under the stated preconditions, the round-trip `(a ⊕ w) ⊖ w` recovers `a` exactly. Throughout, `k` denotes the action point of `w` — the least position `i` with `wᵢ > 0` — so by definition `wᵢ = 0` for all `i < k` and `wₖ > 0`.

**Step 1: the structure of `r = a ⊕ w`.** By TumblerAdd (applicable since `k = #a` gives `k ≤ #a`, satisfying TA0's precondition), the result `r` is built in three regions relative to the action point: `rᵢ = aᵢ` for `i < k` (prefix copy), `rₖ = aₖ + wₖ` (single-component advance), and `rᵢ = wᵢ` for `i > k` (tail copy from displacement). We determine each region under the preconditions.

For `i < k`: the precondition `(A i : 1 ≤ i < k : aᵢ = 0)` gives `rᵢ = aᵢ = 0`.

At `i = k`: `rₖ = aₖ + wₖ`, and since `wₖ > 0` (definition of action point), `rₖ ≥ wₖ > 0`.

For `i > k`: by the result-length identity (TA0), `#r = #w`. The precondition `#w = k` gives `#r = k`, so there are no positions beyond `k` — the tail-copy region is empty. The precondition `k = #a` ensures that no components of `a` beyond position `k` are discarded by tail replacement.

Therefore `r = [0, ..., 0, aₖ + wₖ]` — a tumbler of length `k` with zeros at all positions before `k`.

**Step 2: computing `s = r ⊖ w`.** TumblerSub requires `r ≥ w` (T1). Since `r = a ⊕ w`, TumblerAdd's dominance postcondition gives `r ≥ w`, discharging this obligation. TumblerSub's construction is keyed on `k = zpd(r, w)` (ZPD) — the first position at which the zero-padded projections of `r` and `w` disagree — with the three-region production rule (`sᵢ = 0` for `i < k`, `sₖ = rₖ − wₖ`, `sᵢ = rᵢ` for `i > k`) evaluated at this `k` and the no-divergence branch (producing the zero tumbler of length `L`) gated on `zpd(r, w)` being undefined; the common length `L` over which zpd is evaluated is the longer of `#r` and `#w`, named by NAT-order's trichotomy on the length pair `(#r, #w)` (sub-case (α) `#r = #w` with `L = #r`; sub-case (β) `#r < #w` with `L = #w`; sub-case (γ) `#w < #r` with `L = #r`) rather than by a primitive binary-maximum operator on ℕ, reproducing ZPD's and TumblerSub's length-pair dispatch for the same operand pair. Step 1's identity `#r = k` combined with the precondition `#w = k` places `(#r, #w)` in sub-case (α) `#r = #w` with `L = #r = #w`, furnishing `L = k`; both operands are already at this length, so the padding region is empty and the padded projections coincide with `r` and `w` throughout `{1, ..., k}`. At each position `i < k`, both `rᵢ = 0` (established above) and `wᵢ = 0` (definition of action point), so `rᵢ = wᵢ` — no padded-disagreement occurs before position `k`.

Two cases arise at position `k`. The dichotomy `aₖ > 0 ∨ aₖ = 0` — restricted to two disjuncts rather than admitting a third negative branch `aₖ < 0` that NAT-order's trichotomy on `(0, aₖ)` alone would not exclude — is sourced per step: `aₖ ∈ ℕ` since `k = #a` places position `k` within `a` and `a ∈ T` supplies ℕ-valued components at each position; NAT-zero's axiom `(A n ∈ ℕ :: 0 ≤ n)` instantiated at `n = aₖ` supplies the lower bound `0 ≤ aₖ`; NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = 0, n = aₖ` unfolds this lower bound into `0 < aₖ ∨ 0 = aₖ` — equivalently, the dichotomy. This reproduces the NAT-zero + NAT-order unfolding TumblerAdd's two dichotomy sites (the dominance proof's `aⱼ > 0 ∨ aⱼ = 0` for `j < k` and the inner `aₖ > 0 ∨ aₖ = 0` at the action point) and TumblerSub's case (ii) (`aₖ ≠ 0 ⟹ aₖ > 0` at the prefix-divergence zpd point) apply at the structurally identical pattern.

*Case 1: `aₖ > 0`.* We derive the strict `rₖ = aₖ + wₖ > wₖ` via the same four-step chain TumblerAdd's dominance proof applies at the structurally identical site (with NAT-zero's lower bound on `aₖ` already supplied for Step 2's outer dichotomy): NAT-addcompat's right order-compatibility (`n ≥ p ⟹ n + m ≥ p + m`, instantiated at `n = aₖ, p = 0, m = wₖ`) lifts NAT-zero's lower bound `0 ≤ aₖ` into the non-strict `aₖ + wₖ ≥ 0 + wₖ`; NAT-closure's additive identity `(A n ∈ ℕ :: 0 + n = n)` instantiated at `n = wₖ` rewrites the right-hand side, yielding `aₖ + wₖ ≥ wₖ`; NAT-cancel's summand absorption in its symmetric form `n + m = m ⟹ n = 0` (instantiated at `n = aₖ, m = wₖ`) rules out the equality disjunct `aₖ + wₖ = wₖ` — that equality would force `aₖ = 0` and contradict the sub-case hypothesis `aₖ > 0` via NAT-order's irreflexivity (substituting `aₖ = 0` into `0 < aₖ` yields `0 < 0`); and NAT-order's defining clause `m ≤ n ⟺ m < n ∨ m = n` instantiated at `m = wₖ, n = aₖ + wₖ`, applied to the non-strict `aₖ + wₖ ≥ wₖ` together with the just-discharged non-equality, returns the strict `aₖ + wₖ > wₖ`, i.e., `rₖ > wₖ`. The mirrored route through NAT-addcompat's left order-compatibility with `aₖ ≥ 1` would require `1 + wₖ > wₖ`, which is not available from T0's exhaustive NAT-* enumeration (it lacks commutativity of addition). Hence `rₖ ≠ wₖ`; combined with the pre-divergence agreement `rᵢ = wᵢ` at positions `i < k` established at Step 2's opening, ZPD's minimality clause — applied to the pair `(r, w)` on the common length `L = k` — identifies `k` as the least position at which the padded projections disagree, so the action point `k` of `w` coincides with `zpd(r, w)`. This identification is the premise TumblerSub's three-region production rule consumes at the named `k`; the tumbler-level precondition `r ≥ w` that the call owes TumblerSub was already discharged at Step 2's opening via TumblerAdd's dominance postcondition, and the componentwise strict `rₖ > wₖ` derived here is not a TumblerSub precondition at all — it is the premise ZPD's minimality clause consumed to identify `k = zpd(r, w)`, and it coincides with TumblerSub's precondition-consequence at the same `k` (a derived fact internal to TumblerSub's proof, not an obligation TA4 must discharge). TumblerSub produces: `sᵢ = 0` for `i < k` (zeroing pre-divergence positions), `sₖ = rₖ - wₖ = (aₖ + wₖ) - wₖ = aₖ` (reversing the advance) — the equality `(aₖ + wₖ) - wₖ = aₖ` is NAT-sub's right-telescoping clause applied to the naturals `aₖ, wₖ ∈ ℕ` — and `sᵢ = rᵢ` for `i > k` (tail copy). Since `#r = k`, there are no positions beyond `k`, so the tail-copy region contributes nothing. The result length is `L = k` (by the trichotomy dispatch at Step 2's opening), giving `s = [0, ..., 0, aₖ]` of length `k`. By the precondition, `aᵢ = 0` for all `i < k` and `#a = k`, so `s = a`.

*Case 2: `aₖ = 0`.* Every component of `a` is zero: `aᵢ = 0` for `i < k` by precondition, and `aₖ = 0` by the case hypothesis, so `a` is the zero tumbler of length `k`. At the action point the construction gives `rₖ = aₖ + wₖ`, and substitution of the case hypothesis `aₖ = 0` yields `rₖ = 0 + wₖ`; NAT-closure's additive identity `(A n ∈ ℕ :: 0 + n = n)` instantiated at `n = wₖ` rewrites the right-hand side to `wₖ`, yielding `rₖ = wₖ` — the same per-step citation convention TumblerAdd's dominance sub-case `aₖ = 0` applies at the structurally identical site (rewriting the same `0 + wₖ` to `wₖ`) and ActionPoint's third postcondition applies at `n = 1` (rewriting `0 + 1` to `1`), rather than elided as a syntactic identification of numerals. Combined with `rᵢ = 0 = wᵢ` for `i < k` and `#r = k = #w`, this yields `r = w`. Now `s = r ⊖ w = w ⊖ w`: the padded projections of `r` and `w` agree at every position of the common domain `{1, ..., k}` (the padding region being empty by sub-case (α)), so by ZPD's case-split clause `zpd(r, w)` is undefined, and TumblerSub's no-divergence branch yields the zero tumbler of length `L = k` (by the same trichotomy dispatch, unchanged by the case-2 specialisation `r = w` since the length pair `(#r, #w) = (#w, #w)` remains in sub-case (α)). This zero tumbler of length `k` is exactly `a`.

In both cases, `(a ⊕ w) ⊖ w = a`. ∎

Gregory's analysis confirms that `⊕` and `⊖` are NOT inverses in general.
