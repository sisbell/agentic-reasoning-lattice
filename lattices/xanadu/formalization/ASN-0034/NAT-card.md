**NAT-card (NatFiniteSetCardinality).** For every `n ∈ ℕ` and every subset `S ⊆ {1, 2, …, n} ⊆ ℕ`, we postulate `|·|` as a primitive operator with `|S| ∈ ℕ` the unique `k` for which `S` admits a strictly increasing enumeration `s₁ < s₂ < … < s_k` under `<` on ℕ with `S = {s₁, s₂, …, s_k}` (empty enumeration when `S = ∅`, so `|∅| = 0`), and `|S| ≤ n`. Existence-and-uniqueness of `k` — the definite article binds both — and the upper bound `|S| ≤ n` are clauses of the axiom, not consequences. Hence `|·|` is a well-defined total function on subsets of every initial segment `{1, …, n}` of ℕ with codomain ℕ solely by virtue of the postulate. The operator `|·|` is distinct from T0's tumbler-length `#· : T → ℕ`, which acts on sequences rather than sets.

*Formal Contract:*
- *Axiom:* `(A n ∈ ℕ, S : S ⊆ {1, 2, …, n} :: |S|` is the unique `k ∈ ℕ` such that `S` admits a strictly increasing enumeration `s₁ < s₂ < … < s_k` under `<` on ℕ with `S = {s₁, s₂, …, s_k})` (empty enumeration when `k = 0`, giving `|∅| = 0`) — the definite article "the unique" places existence and uniqueness of `k` inside the quantifier body, and `|S| ∈ ℕ` is thereby entailed rather than stated as a separate clause (enumeration characterisation, codomain entailed); `(A n ∈ ℕ, S : S ⊆ {1, 2, …, n} :: |S| ≤ n)` (upper bound — a further clause of the axiom, resting on NAT-order's `≤` to render the inequality well-formed and on the range `S ⊆ {1, …, n}` to pin the right-hand side). `|·|` acts on subsets of ℕ and is distinct from T0's tumbler-length `#·`, which acts on sequences.
- *Depends:*
  - NAT-order (NatStrictTotalOrder) — supplies the strict order `<` (used in the strictly increasing enumeration `s₁ < s₂ < … < s_k`) and the non-strict companion `≤` (used in the upper bound `|S| ≤ n` and in the unfolding `{1, 2, …, n} = {j ∈ ℕ : 1 ≤ j ≤ n}` that grounds the initial-segment notation); the strict-total-order discipline (irreflexivity, transitivity, trichotomy `m < n ∨ m = n ∨ n < m`) makes "strictly increasing enumeration" a well-formed predicate on ℕ-indexed sequences.
  - NAT-closure (NatArithmeticClosureAndIdentity) — supplies `1 ∈ ℕ`, the lower bound in the unfolded initial segment `{j ∈ ℕ : 1 ≤ j ≤ n}`; combined with NAT-order's `≤` and the outer `n ∈ ℕ`, this grounds `S ⊆ {1, 2, …, n} ⊆ ℕ` with ℕ-typed elements. Also supplies `0 < 1`, which separates the empty case `n = 0` (`{j ∈ ℕ : 1 ≤ j ≤ 0} = ∅`, forcing `|S| = 0` at `S = ∅` with empty enumeration `k = 0`) from the nonempty case `n ≥ 1` (`1 ∈ {j ∈ ℕ : 1 ≤ j ≤ n}`, so enumerations with `k ≥ 1` become admissible).
  - NAT-zero (NatZeroMinimum) — supplies `0 ∈ ℕ` for the literal `0` appearing in the empty-enumeration case `k = 0` and in the empty-set cardinality `|∅| = 0`.
  - T0 (CarrierSetDefinition) — supplies the tumbler-length operator `#· : T → ℕ` referenced in the disambiguation remark distinguishing `|·|` from `#·`.
