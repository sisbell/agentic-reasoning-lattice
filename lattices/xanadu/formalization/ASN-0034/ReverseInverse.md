**ReverseInverse (ReverseInverse).** `(A a, w : a ≥ w ∧ Pos(w) ∧ k = #a ∧ #w = k ∧ (A i : 1 ≤ i < k : aᵢ = 0) : (a ⊖ w) ⊕ w = a)`, where `k` is the action point of `w`.

*Proof.* We show that subtracting `w` from `a` and then adding `w` back recovers `a` exactly, under conditions that make the two operations mutually inverse. Throughout, `k` denotes the action point of `w` — the least position with `wₖ > 0` — so by definition `wᵢ = 0` for all `i < k`.

**Step 1: the structure of `y = a ⊖ w`.** By TumblerSub, subtraction scans `a` and `w` for the first position where they differ, zero-padding the shorter to a common length `L` furnished by NAT-order's trichotomy on the length pair `(#a, #w)` — (α) `#a = #w` with `L = #a`, (β) `#a < #w` with `L = #w`, or (γ) `#w < #a` with `L = #a` — rather than by a primitive binary-maximum operator on ℕ, reproducing TumblerSub's own length-pair dispatch at the same operand pair. The preconditions `k = #a` and `#w = k` give `#a = k = #w`, placing `(#a, #w)` in sub-case (α), so `L = #a = k` and the padding region is empty — no padding is needed. At each position `i < k`, both `aᵢ = 0` (by the zero-prefix precondition) and `wᵢ = 0` (by definition of action point), so the operands agree before position `k`.

NAT-order's trichotomy at the component pair `(aₖ, wₖ)` enumerates three outcomes — `aₖ < wₖ`, `aₖ = wₖ`, or `aₖ > wₖ` — of which we rule out the first. Under the hypothetical `aₖ < wₖ`, the pre-`k` agreement `aᵢ = wᵢ = 0` for `1 ≤ i < k` (just established from the zero-prefix precondition and the action-point definition) together with the lengths `k ≤ #a` and `k ≤ #w` (read off from `k = #a` and `#w = k`) witness `a < w` via T1 case (i) at divergence position `k`; this contradicts the precondition `a ≥ w`. Two cases remain. If `aₖ = wₖ`, then `a` and `w` agree at every position — there are no positions beyond `k` since both have length `k` — and TumblerSub produces the zero tumbler of length `k`, so at position `k` the component `yₖ` is the zero tumbler's `k`-th component, i.e., `yₖ = 0` read directly off TumblerSub's no-divergence-branch output rather than computed via `aₖ - wₖ`. If `aₖ > wₖ`, the strict inequality entails `aₖ ≠ wₖ`, so the padded projections of `a` and `w` (coinciding with the originals at position `k` since the padding region is empty under sub-case (α)) disagree at position `k` — whence ZPD's case-split clause applied affirmatively identifies `zpd(a, w)` as defined. The pre-`k` agreement `aᵢ = wᵢ = 0` for `1 ≤ i < k` (from the zero-prefix precondition and the action-point definition, evaluated on the common length `L = k` where the padding region is empty) rules out every position `i < k` as a padded-disagreement, so ZPD's minimality clause — applied to the pair `(a, w)` on the common length `L = k` — identifies `k` as the least padded-disagreement position, fixing `k = zpd(a, w)`. At this zpd-identified `k`, TumblerSub's three-region production rule produces `yᵢ = 0` for `i < k`, `yₖ = aₖ - wₖ > 0` (the membership `aₖ - wₖ ∈ ℕ` by NAT-sub's conditional closure under `aₖ ≥ wₖ` from NAT-order; the strict positivity `aₖ - wₖ > 0` by NAT-sub's strict-positivity clause under `aₖ > wₖ`), and no components beyond `k` (since `L = k` by the sub-case (α) dispatch above). In either case, `y` has two uniform properties and a branch-indexed third property we record for later use:

- (Y1) `#y = k`
- (Y2) `yᵢ = 0` for all `1 ≤ i < k`
- (Y3a) in the equality branch `aₖ = wₖ`: `yₖ = 0` (read directly off TumblerSub's no-divergence-branch output, since `y` is the zero tumbler of length `k` furnished by that branch — no NAT-sub or NAT-closure inference is consumed to establish `yₖ = 0` in this branch)
- (Y3b) in the divergence branch `aₖ > wₖ`: `yₖ = aₖ - wₖ` and `yₖ > 0` (the value by TumblerSub's divergence-branch output composed with NAT-sub's conditional closure, the strict positivity by NAT-sub's strict-positivity clause under `aₖ > wₖ`)

We promote this hypothesis to the strict component inequality `(y ⊕ w)ₖ = yₖ + wₖ > wₖ` via the same four-step chain consuming NAT-addcompat, NAT-closure, NAT-cancel, and NAT-order together that TumblerAdd's dominance sub-case `aₖ > 0` applies at its structurally identical site — TumblerAdd's exported dominance postcondition `a ⊕ w ≥ w` is non-strict, so the strict form this step consumes to feed T1 case (i) must be derived per-step rather than read off transitively. *Strict ordering between `y ⊕ w` and `a`.* By T1 (trichotomy), since `y ⊕ w ≠ a`, exactly one of `y ⊕ w < a` or `y ⊕ w > a` holds. *Case `y ⊕ w < a`:* We invoke TA3-strict at the operand renaming `TA3-strict's a := y ⊕ w, b := a, w := w`. Both cases are impossible, so the assumption `y ⊕ w ≠ a` is false. Therefore `(a ⊖ w) ⊕ w = a`. ∎
