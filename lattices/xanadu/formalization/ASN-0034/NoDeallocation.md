**NoDeallocation (NoDeallocation).** The system defines no operation that removes an element from the set of allocated addresses. No "deallocate", "free", or "reclaim" mechanism exists. This is a design constraint accepted as an axiom, not a derived property.

Nelson: "any address of any document in an ever-growing network may be specified by a permanent tumbler address." The permanence guarantee requires that the allocated set can only grow — no mechanism exists to contract it.

Let Σ denote the system's transition vocabulary — the complete set of operations the system may perform — and let 𝒮 denote the state space of the allocation system (the set of allocator-tree configurations defined in AllocatedSet, written here in script form to avoid clashing with the unrelated symbol S used for positive-component ordinals in TA7a and the Vocabulary). We fix the signature of elements of Σ once, here, so that it is available to every property that cites this axiom: each `op ∈ Σ` is a partial function `op : 𝒮 ⇀ 𝒮`, written with the half-arrow to mark that `op` need not be defined at every state. Under this signature the predicate `op(s) defined` — used both in the axiom below and in T8's monotonicity conclusion, which restricts its state-pair quantifier to pairs `(s, s')` arising from a defined application — abbreviates `s ∈ dom(op)`, and when it holds `op(s) ∈ 𝒮` denotes the unique successor state produced by applying `op` at `s`; the transitions `s → s'` that AllocatedSet speaks of are exactly the pairs `(s, op(s))` with `op ∈ Σ` and `s ∈ dom(op)`. Recording the signature as a frame is needed because the axiom's `op(s) defined` clause is load-bearing: at states where `op` is not defined no constraint is placed on `op`, and without the signature it would be ambiguous whether such pairs are to be read as a permission (partial function, no transition at all at such `s`) or as a silent universal over all candidate successors (relational reading). We take the first reading. Let `allocated(s)` denote the set of addresses allocated in state `s ∈ 𝒮`. The axiom asserts: for every operation `op ∈ Σ` and every state `s ∈ 𝒮` in which `op(s)` is defined, `allocated(s) ⊆ allocated(op(s))`. No operation in Σ removes any element from the allocated set. Because Σ is taken to be closed — every transition the system can undergo is an application of some `op ∈ Σ` — the axiom constrains not merely the currently defined operations but any operation the system could ever admit.

*Formal Contract:*
- *Signature of Σ:* Each element of Σ is a partial function `op : 𝒮 ⇀ 𝒮`. The predicate `op(s) defined` abbreviates `s ∈ dom(op)`; when it holds, `op(s) ∈ 𝒮` is the unique successor state. A state transition `s → s'` is exactly a pair `(s, op(s))` with `op ∈ Σ` and `s ∈ dom(op)` — no transition exists at `s` for operations undefined at `s`. This signature is the frame against which the axiom's `op(s) defined` clause is to be read.
- *Axiom:* `(A op ∈ Σ, s ∈ 𝒮 :: op(s) defined ⟹ allocated(s) ⊆ allocated(op(s)))`, where Σ is the system's complete (closed) transition vocabulary of partial functions on 𝒮 (per the signature above) and 𝒮 is the state space of the allocation system (the script-S form is used here to keep the state-space symbol distinct from the unrelated symbol S in TA7a/Vocabulary, which denotes positive-component ordinals). Frame assumption: Σ is closed, so every reachable state transition is governed by this constraint.
- *Depends:* AllocatedSet (AllocatedSet) — supplies the carrier `𝒮` over which the axiom's quantifier `s ∈ 𝒮` ranges (AllocatedSet's *State* clause names `𝒮` as the state space of the allocator-tree configurations) and supplies the symbol `allocated(s) = ⋃ { domₛ(A) : A activated in s }` whose inclusion `allocated(s) ⊆ allocated(op(s))` is the entire content of the axiom; also fixes the domain-embedding facts (i)–(iii) that determine what `allocated(s)` denotes at each state, so that any future tightening of `𝒮` or of `allocated(s)` propagates to NoDeallocation through this Depends edge. The dependency is mutual — AllocatedSet reciprocally cites NoDeallocation (with a forward-reference note) for the transition vocabulary Σ — and this clause records the reverse direction. Without this citation, NoDeallocation would consume AllocatedSet's vocabulary load-bearingly while leaving the connection invisible to the Depends graph that T8's proof, which cites both AllocatedSet and NoDeallocation, traverses.
