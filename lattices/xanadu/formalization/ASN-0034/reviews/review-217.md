# Cone Review — ASN-0034/TA-strict (cycle 5)

*2026-04-18 01:05*

### ActionPoint's third-postcondition derivation elides the same `0 + 1 = 1` rewrite that Previous Findings required TA-Pos to source
**Foundation**: NAT-closure (NatArithmeticClosureAndIdentity) — additive identity `0 + n = n`; ActionPoint's Depends does not list NAT-closure.
**ASN**: ActionPoint — "NAT-discrete's axiom `m ≤ n < m + 1 ⟹ n = m` in its contrapositive form `m < n ⟹ m + 1 ≤ n` ... instantiated at `m = 0, n = w_{actionPoint(w)}` yields `1 ≤ w_{actionPoint(w)}`."
**Issue**: NAT-discrete's contrapositive instantiated at `m = 0, n = w_{actionPoint(w)}` actually yields `0 + 1 ≤ w_{actionPoint(w)}`, not `1 ≤ w_{actionPoint(w)}`. The rewrite `0 + 1 ↦ 1` requires NAT-closure's additive identity `0 + n = n` at `n = 1` — an axiom step, not a syntactic identification of numerals. Previous Findings flagged this exact elision at a structurally identical site in TA-Pos (the "0 < 0 + 1 → 0 < 1" passage); the current ASN resolves TA-Pos's instance by citing NAT-closure explicitly and adding it to TA-Pos's Depends. ActionPoint performs the mirror rewrite (`0 + 1 ≤ · ↦ 1 ≤ ·`) at the close of the third-postcondition chain but (i) does not name NAT-closure at the step, (ii) does not list NAT-closure in its Depends, and (iii) writes the conclusion directly as `1 ≤ w_{actionPoint(w)}` as if the identity were syntactic. Because TA-Pos's Depends declares NAT-closure as "retained as a transitive sourcing handle through ActionPoint" for the structurally identical ActionPoint-postcondition-(iii) cascade, a reviser tracing NAT-closure's load-bearing sites through ActionPoint finds no direct citation at the step that consumes the axiom — the transitive handle at TA-Pos points at a site where the axiom is unnamed.
**What needs resolving**: ActionPoint's third-postcondition proof must either cite NAT-closure's additive identity `0 + n = n` at `n = 1` for the rewrite from NAT-discrete's raw conclusion `0 + 1 ≤ w_{actionPoint(w)}` to the postcondition's `1 ≤ w_{actionPoint(w)}` — adding NAT-closure to ActionPoint's Depends in parallel to TA-Pos's treatment — or the ASN must document a uniform convention under which `0 + 1` and `1` are identified without axiomatic mediation, and retrofit TA-Pos (along with TumblerAdd's dominance sub-case `aₖ > 0`, which cites NAT-closure at the analogous `0 + wₖ ↦ wₖ` rewrite) to elide the citations now made explicit there.
