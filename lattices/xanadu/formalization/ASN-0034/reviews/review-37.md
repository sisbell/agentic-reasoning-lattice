### GlobalUniqueness Case 2 — T10 invoked without establishing the prefix-extension bridge
**Foundation**: (internal consistency — T10 precondition chain)
**ASN**: GlobalUniqueness proof, Case 2: *"The two allocators have prefixes `p₁` and `p₂` such that neither is a prefix of the other... By T10 (partition independence), for any tumbler `a` extending `p₁` and any tumbler `b` extending `p₂`, `a ≠ b`."* Also T10a formal contract: *"child-spawning uses exactly one `inc(·, k')` ... to establish the child's prefix."*
**Issue**: T10 requires `p₁ ≼ a` and `p₂ ≼ b` — every output must *extend* some prefix in the ≼ sense. Case 2 asserts that allocators "have prefixes" and their outputs "extend" them, but never defines the mapping from allocator to prefix, and the natural candidate fails. T10a calls the child-spawning output `c₀ = inc(t, k')` "the child's prefix," suggesting the base address is the prefix. But `inc(base, 0)` produces a same-length sibling differing at position `sig(base)` — for valid addresses, `sig(base) = #base` (TA5-SigValid) — so `base ⋠ inc(base, 0)` (same-length tumblers are prefix-related only if identical, by Prefix and T3). The allocator's base address is not a prefix of its own sibling outputs. A viable prefix exists: the parent sibling from which the allocator was spawned, which is strictly shorter than the base. All outputs preserve the parent sibling's components (by TA5(b) for the child-spawning step, then TA5(b)+TA5(c) for subsequent `inc(·, 0)` steps), so `parent_sibling ≼ output` holds. Two sibling allocators spawned from different parent siblings have non-nesting parent-sibling prefixes (same length, differing at `sig`). But neither the identification of this prefix nor the verification of the extension property appears in the proof. The term "prefix" is used for two incompatible things: T10a uses it for the base address (which doesn't satisfy ≼ with outputs), T10 uses it for an address that all outputs extend (which must be shorter than the base).
**What needs resolving**: Define the "prefix" of an allocator (the parent sibling from which it was spawned, or an equivalent formulation), establish that all allocator outputs extend this prefix in the ≼ sense, and show that sibling allocators' prefixes are non-nesting — so that T10's preconditions are actually met when Case 2 invokes it. The root allocator (which has no parent sibling) needs separate treatment — its cross-domain pairs are all handled by length separation (Case 4), so it may simply be excluded from Case 2's scope.
