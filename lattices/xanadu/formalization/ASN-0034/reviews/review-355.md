### NAT-cancel commentary cites a NAT-* enumeration in T0 that no longer exists
**Foundation**: T0 (CarrierSetDefinition) — body: "T is the set of nonempty finite sequences over ℕ … For each `a ∈ T`, write `#a` for the length … `aᵢ ∈ ℕ` at each `i ∈ {1, …, #a}`." T0 contains no enumeration of NAT-* axioms (consistent with the cycle-7 revision noted in the recent commits).
**ASN**: NAT-cancel (NatAdditionCancellation) — explanatory paragraph: "Both summand-absorption forms are stated independently because **T0's NAT-* enumeration** does not include commutativity of addition on ℕ, so neither form is derivable from the other. The same reason governs the independent statement of left and right cancellation."
**Issue**: NAT-cancel's justification for stating two summand-absorption forms (and for separating left/right cancellation) cites "T0's NAT-* enumeration" as the inventory whose absence of commutativity forces the redundancy. T0 in the current ASN exposes only the carrier-set definition and projection signature; it does not enumerate NAT-* axioms at all. The justification therefore points at a structural feature the ASN no longer has. The substantive claim (no commutativity axiom is in scope, so symmetric forms must be stated independently) may still be true, but the citation is stale and gives a precise reader nothing to verify against.
**What needs resolving**: Either drop the "T0's NAT-* enumeration" reference and rephrase the justification in terms of the actual scope of NAT-* axioms in this ASN, or restore an enumeration in T0 (or elsewhere) that NAT-cancel can legitimately cite.

### T1 Depends attributes a Case 3 use to NAT-discrete that is actually NAT-addcompat
**Foundation**: T1 (LexicographicOrder) — Depends entry for NAT-discrete: "forward direction `m < n ⟹ m + 1 ≤ n`, used contrapositively in Case 1 … and in part (c) Case `k₂ < k₁` case-(ii) branch … **used in Case 3 to supply the successor-comparison step**."
**ASN**: Case 3 of part (b), the only Case 3 in the proof: "Both clauses force `m ≠ n`: (β) gives `m + 1 ≤ n`, hence `m < n` via **NAT-addcompat's `m < m + 1`**; (γ) gives `n < m` symmetrically." The subsequent step "If `m < n`, then `k = m + 1 ≤ n`" reuses the m+1 ≤ n already supplied by clause (β) and does not invoke NAT-discrete either.
**Issue**: NAT-discrete supplies `m < n ⟹ m + 1 ≤ n`. The successor-comparison step in Case 3 goes the other way: from `m + 1 ≤ n` it derives `m < n`, by composing NAT-addcompat's `m < m + 1` with NAT-order transitivity. NAT-discrete's forward direction is not a fit for that step, and the proof body explicitly attributes it to NAT-addcompat. The Depends list therefore tells a reader that NAT-discrete carries a Case 3 obligation that the proof never lays on it. NAT-discrete genuinely does the work in Case 1 (contrapositive) and in the transitivity case k₂ < k₁ branch (ii), so the dependency is not spurious overall — only the third use-site claim is misattributed.
**What needs resolving**: Remove the "used in Case 3 to supply the successor-comparison step" clause from NAT-discrete's Depends entry, or reword it to describe what NAT-discrete actually licenses in Case 3 (if any). The misattributed step's true citation, NAT-addcompat (`n < n + 1`), is already in NAT-addcompat's Depends entry, so no addition there is needed.
