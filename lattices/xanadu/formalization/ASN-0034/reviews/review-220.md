### TA-Pos's "T0 makes Zero ⟺ ¬Pos coincide" justification is incorrect — the equivalence is unconditional
**Foundation**: TA-Pos (this ASN) — Definition block prose and T0-citation in Depends.
**ASN**: TA-Pos prose: "T0's requirement `#t ≥ 1` on every `t ∈ T` makes `Zero(t)` and `¬Pos(t)` logically equivalent on the carrier — **an empty conjunction would vacuously satisfy `Zero(t)` while a non-empty existential over no indices would falsify `Pos(t)`, but T0 rules this case out**". Depends restates it: "T0's requirement `#t ≥ 1` is additionally what makes `Zero(t)` and `¬Pos(t)` coincide on the carrier — the vacuous case in which both would hold is ruled out".
**Issue**: `Zero(t) ⟺ ¬Pos(t)` holds unconditionally as the DeMorgan dual `¬(∃i. tᵢ ≠ 0) ⟺ (∀i. tᵢ = 0)`, with no precondition on `#t`. The text's argument tries to identify the empty-index-set case as one where the equivalence could fail, but in that case **both** predicates hold vacuously (the empty universal `Zero(t)` is true, the empty existential `Pos(t)` is false, hence `¬Pos(t)` is true), so the predicates *agree* there — they do not diverge. T0's `#t ≥ 1` does eliminate the vacuous case, but it is not what aligns Zero and ¬Pos. The motivational paragraph thus rests its convention on a false premise. This matters because downstream properties citing Zero(t) — TA-PosDom (per its forward reference), TA6 in the ordinal section — will be sourced through this paragraph and Depends, so a reviser auditing why Zero is defined as a universal rather than as `¬Pos` will trace the rationale to a non-existent failure mode.
**What needs resolving**: Either correct the prose to articulate the actual reason for stating Zero as a universal (e.g., the unpacking-without-trichotomy benefit, which the next clause does identify but presents as additional rather than primary), or supply a different premise for the T0-dependence. As written, the T0 citation in TA-Pos's Depends — which the contract explicitly invokes "for the equivalence" — is justified by an argument the predicates do not actually need.

### TumblerAdd's dominance case split treats `(∀j: P ∨ Q) ⟹ (∃j: P) ∨ (∀j: Q)` as a "closure" of the per-position dichotomy
**Foundation**: TumblerAdd (this ASN), dominance proof — top-level case split.
**ASN**: TumblerAdd dominance: "The top-level case split on whether some `aⱼ > 0` for `j < k` is a dichotomy at `0` on ℕ, sourced per position: for each `j` with `1 ≤ j < k`, … NAT-order's defining clause … unfolds this lower bound into `0 < aⱼ ∨ 0 = aⱼ`, i.e., `aⱼ > 0 ∨ aⱼ = 0`; **the two top-level cases are the existential and universal closures of these two disjuncts over `1 ≤ j < k`**."
**Issue**: The two top-level cases used in the proof are `(∃j ∈ [1, k-1] : aⱼ > 0)` and `(∀j ∈ [1, k-1] : aⱼ = 0)`. The per-position dichotomy `(∀j ∈ [1, k-1] : aⱼ > 0 ∨ aⱼ = 0)` does not yield this disjunction by "existential and universal closure" — it yields the strictly weaker `(∀j : aⱼ > 0 ∨ aⱼ = 0)`, which is logically equivalent to the case split only via classical excluded middle on the existential (or, equivalently, double-negation elimination over the index range). The proof tacitly performs this move: it case-splits into "some `aⱼ > 0`" versus "all `aⱼ = 0`", which is `(∃j : aⱼ > 0) ∨ ¬(∃j : aⱼ > 0)`, then uses DeMorgan over the bounded existential to rewrite the second disjunct as `(∀j : aⱼ = 0)`. The ASN's per-step citation discipline elsewhere (e.g., NAT-order's defining clause for unfolding `≤` into `< ∨ =`, NAT-cancel for ruling out an equality disjunct) sources every classical move; here the bounded LEM step is described as a "closure" — a syntactic word that hides the logical content. No axiom in TumblerAdd's Depends underwrites this step (NAT-wellorder furnishes the least element *given* the existential, but does not produce the existential-or-its-negation case split itself).
**What needs resolving**: The case split must be sourced — either by adding a citation to whichever foundation supplies bounded LEM / DeMorgan-on-bounded-quantifiers (and listing it in Depends), or by reformulating the case analysis so that "some `aⱼ > 0` for `j < k`" versus "`aᵢ = 0` for all `i < k`" is presented as the bounded-LEM disjunction `(∃j : aⱼ > 0) ∨ ¬(∃j : aⱼ > 0)` with the second disjunct's reduction to a universal cited explicitly. As written, the move is hidden behind syntactic vocabulary while every smaller logical step in the same paragraph is named.
