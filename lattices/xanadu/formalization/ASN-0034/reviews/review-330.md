### Case 5's "WLOG k'ₓ = 1, k'ᵧ = 2" in T10a.5 is applied in a branch where WLOG is already consumed by a separate assignment
**Foundation**: Self-review (T10a.5 proof structure).
**ASN**: T10a.5 proof, Base case: "If `tₓ = tᵧ` and `k'ₓ ≠ k'ᵧ`: one of `{k'ₓ, k'ᵧ}` is 1 and the other 2; take `k'ₓ = 1` and `k'ᵧ = 2` without loss of generality. TA5(d) places a 1 at position `m + 1` of `bₓ` (the single appended position) and a 0 at position `m + 1` of `bᵧ` (the zero-valued separator preceding the final 1), so `j = m + 1` with `bₓ[j] = 1 ≠ 0 = bᵧ[j]`…"; subsequent inductive step: "When `j = #bₓ` — the case arising from `k'ₓ = 1`, `k'ᵧ = 2` — successive elements of `dom(Cₓ)` increment position j (since `sig(bₓ) = #bₓ = j`)…"; closing inductive step: "On the Cᵧ side, `j = m + 1 < m + 2 = #bᵧ` (since `k'ᵧ = 2`), so position j is interior to Cᵧ's base".
**Issue**: The inductive step following the base case reasons asymmetrically about Cₓ and Cᵧ — it argues that on Cₓ's side position j equals `sig(bₓ) = #bₓ` (so each `inc(·,0)` modifies j) and on Cᵧ's side position j is interior (so each `inc(·,0)` leaves j alone). This asymmetry is load-bearing: the two sides of the argument are genuinely different, not mirror images. The "WLOG" in the base case fixes which allocator is Cₓ and which is Cᵧ by tying Cₓ to spawn parameter 1 and Cᵧ to spawn parameter 2. The "Closure" step then claims results for "any `x ∈ domain(X)` and `y ∈ domain(Y)`", where X and Y were the original allocators, but the analysis only established the invariant for the specific (Cₓ=param-1, Cᵧ=param-2) assignment. If X's ancestor chain from the LCA happens to go through the param-2 child and Y's through the param-1 child, the assignment Cₓ=param-1 does not match the naming; either the labels X and Y must be swapped before invoking the inductive result (making the WLOG genuinely symmetric) or the subtree invariants must be re-derived with the opposite role assignment. As written, the closure step applies a non-symmetric invariant under a symmetric-seeming WLOG.
**What needs resolving**: State explicitly that after the "WLOG k'ₓ = 1, k'ᵧ = 2" choice, the labels X and Y are (if necessary) relabeled to match the Cₓ/Cᵧ role assignment, or give the mirror argument for the opposite assignment so both orderings of X and Y are covered without appeal to a symmetry that the asymmetric sig/interior analysis does not possess.

### "Distinct allocators" identity criterion is used before `U(d)` supplies it in GlobalUniqueness exhaustiveness routing
**Foundation**: Self-review (GlobalUniqueness proof structure, T10a Definitions).
**ASN**: GlobalUniqueness *Exhaustiveness*: "If the domain prefixes are value-identical (`p₁ = p₂`): we must first establish that both allocators were spawned by the same parent. Let `P₁` and `P₂` be the respective parent allocators; each has depth strictly less than its child, hence depth ≤ *d*. The shared prefix value `p₁` is a domain element of `P₁` (assigned by the event that produced it) and `p₂ = p₁` is a domain element of `P₂` (likewise). If `P₁ ≠ P₂`, these are two distinct allocation events at depth ≤ *d* yielding the same output value, contradicting `U(d)`." T10a *Identity of allocators*: "`X = Y` iff both are the root, or both are non-root with identical spawning triples — `parent(X) = parent(Y)`, `spawnPt(X) = spawnPt(Y)`, `spawnParam(X) = spawnParam(Y)`."
**Issue**: The `U(d)` induction is about distinct *allocation events* yielding distinct output *values*. The step "if `P₁ ≠ P₂`, then the events producing `p₁` (in P₁) and `p₂ = p₁` (in P₂) are distinct events" silently equates "P₁ ≠ P₂ (allocator-inequality per T10a's spawning-triple identity)" with "the producing events are distinct". These are not the same notion: two allocators being distinct means their spawning triples differ; two allocation events being distinct means they are different invocations of `inc` (or initialization). The inductive hypothesis `U(d)` only guarantees distinctness between *events*. If the same event happened to be counted under two allocator-labels (which T10a's identity criterion forbids, but that forbiddance is a separate T10a consequence that the exhaustiveness argument has not cited here), the step would be unsound. More subtly: the argument claims the output `p₁` "is a domain element of `P₁`" and "is a domain element of `P₂`", appealing to the producing-allocator assignment in both directions — but that assignment is precisely what assigns each event to a single allocator, and invoking it simultaneously for P₁ and P₂ presumes p₁ is produced by two events (one per allocator), which is the conclusion the step is trying to derive.
**What needs resolving**: Either justify directly that under `P₁ ≠ P₂` the events that add p₁ to their respective domains are different events (e.g., by citing the per-event producing-allocator taxonomy that the Preconditions introduce), or restate the exhaustiveness step to use T10a.6's domain disjointness over allocators at depth ≤ d (which would follow from `U(d)` via the same corollary argument) to conclude `P₁ = P₂` without routing through event-distinctness.
