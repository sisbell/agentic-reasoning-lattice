### GlobalUniqueness exhaustiveness — root-vs-non-root pairs routed to Cases 3/4 whose preconditions the root cannot satisfy
**Foundation**: (internal consistency — case analysis completeness)
**ASN**: GlobalUniqueness exhaustiveness claim: *"The root allocator has no domain prefix; its outputs have the shortest uniform length in the hierarchy (T10a.1), and every non-root output is strictly longer (T10a.3), so all root-vs-non-root pairs are separated by length and fall under Cases 3–4."* Case definitions: *"Case 3: Different allocators with nesting prefixes and different zero counts."* *"Case 4: Different allocators with nesting prefixes and the same zero count."*
**Issue**: The exhaustiveness argument acknowledges that the root has no domain prefix, then immediately routes root-vs-non-root pairs into Cases 3 and 4 — both of which are defined as requiring "nesting domain prefixes." A case whose precondition demands nesting prefixes cannot accept a participant that has no prefix. The length separation argument supplied is correct: root outputs have uniform length γ (T10a.1), all non-root outputs have length ≥ γ + 1 (T10a.3), and different lengths give distinct tumblers (T3). But this argument does not depend on domain prefixes, zero counts, or any other apparatus of Cases 3/4 — it is a self-contained one-step conclusion. The four-case partition is therefore not exhaustive over all distinct-allocator pairs as currently stated: root-vs-non-root pairs satisfy no case's preconditions.
**What needs resolving**: Either handle root-vs-non-root as a separate case (the length argument is simpler than Cases 3/4 and self-contained), or extend the domain-prefix concept to cover the root (e.g., the root's prefix could be defined as the empty sequence, which is a prefix of every tumbler — but then T10's mutual non-nesting precondition would never hold for root-vs-non-root pairs, confirming that these pairs need a different mechanism than Case 2 as well). The current routing to Cases 3/4 must be replaced with whichever approach is chosen.

### GlobalUniqueness Case 3 — "allocator level" used as a per-allocator concept without establishing uniformity of zero count across outputs
**Foundation**: (internal consistency — definition precision)
**ASN**: GlobalUniqueness Case 3: *"The injective correspondence between levels and zero counts means allocators at different levels produce addresses with zeros(a) ≠ zeros(b)."* Also: *"contradicting the hypothesis that the allocators operate at different hierarchical levels."*
**Issue**: Case 3 treats "allocator level" as a well-defined per-allocator property — an allocator *is at* a level, and all its outputs carry that level's zero count. T4c (cited as the basis) provides per-address level determination: given an individual address `t`, `zeros(t)` determines its level. The claim that all outputs of a single allocator share the same zero count — bridging per-address to per-allocator — is an unstated intermediate. The claim is true: `inc(·, 0)` modifies only position `sig(t) = #t` (TA5(c) + TA5-SigValid), which is a positive component in the last field (T4), so it stays positive and the set of zero-valued positions is unchanged; by induction the zero count is invariant across the sibling stream. But this reasoning appears nowhere in the proof. The proof's actual logical need is simpler: the case is discriminated by per-pair zero count (`zeros(a) ≠ zeros(b)`), and the conclusion follows in one step (`a = b` ⟹ `zeros(a) = zeros(b)`, contradiction). The T4c citation and the per-allocator level concept are not required for the logical argument. Their presence introduces an undefined term ("allocator level") and an unjustified derivation step (from per-address T4c to per-allocator uniformity).
**What needs resolving**: Either state and prove the zero-count uniformity lemma (that all outputs of a conforming allocator have the same `zeros` value) — making "allocator level" a defined concept — or simplify Case 3 to use the per-pair condition directly: "If `zeros(a) ≠ zeros(b)` then `a = b` implies `zeros(a) = zeros(b)`, contradiction." The latter removes the need for T4c entirely in Case 3 and avoids the undefined intermediate.
