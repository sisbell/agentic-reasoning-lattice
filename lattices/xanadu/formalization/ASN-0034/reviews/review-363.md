### Term mismatch between Nelson's quoted terminology and T4's formal field names is never bridged
**Foundation**: T4's main-text naming of the four fields
**ASN**: T4 quotes Nelson: "There are four major fields, each expandable indefinitely, with three major dividers between them. These fields are Server, User, Document and Contents." Immediately after, T4's formal naming gives "**Node field** `N₁. ... .Nₐ`: identifies the server." and "**Element field** `E₁. ... .Eδ`: identifies the content element." All subsequent claims (T4, T4a, T4b, T4c) use Node/User/Document/Element.
**Issue**: Nelson's text names four fields Server/User/Document/Contents. T4 silently renames two of them: Server → Node (with a single prose gloss "identifies the server") and Contents → Element (no gloss). A reader auditing the ASN against its cited primary source cannot verify term-by-term correspondence; the Nelson quote is justification by authority, but the authority speaks a different vocabulary. This matters because T4c's label set is `{node, user, document, element}` — the term "node" has no citation in the quoted sources.
**What needs resolving**: Either state the terminological translation explicitly (e.g., "We write *node* for Nelson's *server* and *element* for Nelson's *contents*; the renaming is notational"), or adopt Nelson's terms, or cite a different primary source that uses Node/Element.
